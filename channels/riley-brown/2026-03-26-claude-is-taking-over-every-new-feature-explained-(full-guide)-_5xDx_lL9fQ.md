## 第一部分：AI 的三大进化浪潮——从对话到创造再到行动 `[00:00-04:00]`

### 核心观点

在深入解析 Claude 的新功能之前，我们首先需要理解一个更宏大的背景：AI 从 2022 年末至今，已经经历了三个截然不同的发展阶段。每个阶段都在为下一个阶段铺路，而我们现在正处于从“创造”到“行动”的关键转折点。

### 深度阐述

**第一波浪潮：对话时代 (2022 年末 - 2024 年中)**

ChatGPT 在 2022 年末的横空出世，开启了一个全新的时代。Riley Brown 将其描述为“魔法”——一个似乎知道所有答案的文本生成器。当然，它会产生幻觉，它并非完美，但它的能力足够令人惊叹：写论文、撰写邮件、回答各种问题。这个阶段的核心范式是 **“文本到文本”**。

到了 2023 年末，这个范式开始扩展。用户不仅可以上传图片，还可以上传文件，模型不仅能回复文字，还能输出代码。你可以向它询问某一行代码的功能，或者让它帮你编写一个函数。但本质上，这仍然是“文本”层面的交互——模型输出的代码仍然是原始文本，需要你自己复制、粘贴、运行。

**第二波浪潮：Vibe Coding 时代 (2024 年中至今)**

真正的分水岭出现在 2024 年 5 月，而开创者正是 Claude。他们推出了一个改变游戏规则的功能：**Claude Artifacts (工件)**。这背后的突破在于：AI 不再只是生成代码文本，而是直接在应用内渲染出代码的实际运行结果。

想象一下，你让 AI 创建一个 React 组件来解释什么是 Claude Artifacts，它不仅能写出代码，还能立即在界面上显示一个可交互的组件。这就是 Artifacts 的核心价值——它让“代码”变成了“应用”，让 AI 的创造力有了可见的形态。

Riley 回忆道：“当这个功能推出时，我立刻就全身心投入了。”很快，Cursor 等工具进一步扩展了这个概念。Cursor 的 Composer (现称为 Agent) 可以让你用一句话就创建一个完整的生产级应用：“创建一个新的应用，前端用 React，后端用 Firebase，做一个能赚钱的完整应用。”Cursor 会自己编写所有代码文件，生成的不只是前端界面，而是一个完整的产品。

这个阶段催生了大量“Vibe Coding”工具：Lovable、Replit、Bolt 等，它们的目标是让没有编程经验的人也能快速创建应用。而在这个过程中，**Claude Code** 的发布成为另一个关键节点。

**第三波浪潮：通用代理时代 (2025 年末至今)**

Claude Code 最初是一个在终端中运行的工具，让开发者可以用它来创建应用、内部工具、营销素材甚至幻灯片。但人们很快发现，它的能力远不止于编程。它可以自动完成营销任务、回复邮件、连接其他工具并执行任务。

2026 年初，OpenClaw 的诞生标志着“通用代理”时代的正式开启。OpenClaw 的底层代理引擎被称为 “Pi”，它与 Claude Code 类似，但被设计为通用目的代理，可以完成任何类型的工作任务。

Riley 将这个阶段描述为：“我们正在试图将这种能创建任何应用的技术，应用到通用代理上。”这引出了一个重要的竞争格局：OpenAI 在录制视频前一小时宣布彻底放弃 Sora 应用，转而聚焦于打造一个统一的超级应用（整合 ChatGPT、Codex 和 Atlas 浏览器），因为 An**thropic 的超级应用——Claude Desktop——已经成为了一个真正的巨无霸**。

**个人感受**

Riley 清晰地表达了这种“时代感”：“ChatGPT 是魔法，它曾让人觉得它会接管世界。但现在，感觉每个人都在转向 Claude。”这种情绪的转变不是突然发生的，而是功能迭代的自然结果。他提到，Anthropic 在过去两个月的发布节奏“疯狂”，那种兴奋感贯穿整个视频。

**延伸思考**

这一部分揭示了 AI 发展的核心逻辑：**从对话工具 → 创造工具 → 行动工具**。而“行动”的终极形态，就是 AI 能够像人类一样使用计算机。当我们进入这个阶段，AI 不再只是一个助手，而是真正意义上的“同事”——它能够完成你交付的任何数字任务，而不仅仅是回答问题或生成代码。

## 第二部分：Claude Co-work——安全与自由的完美平衡点 `[04:00-12:00]`

### 核心观点

Claude Co-work 是 Anthropic 推出的全新协作平台，它巧妙地填补了 Claude Chat (安全但功能受限) 和 Claude Code (强大但危险) 之间的空白。通过在沙箱环境中运作，Co-work 让用户既能享受 AI 代理的强大能力，又不必担心它破坏电脑上的其他文件。

### 深度阐述

**Claude 产品的三个层级**

为了更好地理解 Co-work 的价值，我们需要先看 Claude 的产品矩阵：

1. **Claude Chat**：在网页或 App 中与 Claude 对话。它几乎不接触你的电脑，所有操作都在应用内部完成。安全，但能力受限。
1. **Claude Code**：在终端运行，具有完整的系统访问权限。当使用 `dangerously-skip-permissions` 参数时，它可以做任何事情：重新整理你的桌面文件夹、上传文件到云端、甚至调用 API 生成视频。Riley 形容道：“你可以让 Claude Code 做任何你想象力能想到的事情。” `[05:30]` 但这也意味着极高的风险——它可能把你的数据发送给错误的人。
1. **Claude Co-work**：介于两者之间的理想方案。它在你的电脑上创建一个“沙箱”环境（一个隔离的文件夹），**只能在这个文件夹内操作**。你可以给 Co-work 添加其他文件夹的上下文（比如添加某个文件作为参考），但它不会主动修改沙箱外的任何内容。
**Co-work 的实操演示**

Riley 在视频中做了完整的演示。当你在 Claude 中进入 Co-work 并创建一个新项目（比如“Marketing”文件夹）后，这个项目就成为了 Claude 的工作空间。你可以通过“Connectors”连接各种外部工具：

- **Figma**：Claude 可以读取你的 Figma 文件，甚至生成新的设计图。
- **Gmail**：授权后，Claude 可以获取你的邮件列表、草稿，甚至发送邮件。
- **Canva、Excel、Google Drive** 等都可以一键连接。
Riley 惊讶地发现：“在短短两分钟内，我连接了所有对我业务有用的工具。” `[07:30]`

**一个真实的 Co-work 场景**

他演示了一个实际案例：让 Claude Co-work 撰写一个视频的引言，解释 Co-work 和 Claude Code 的区别。Claude 会：

1. 自动找到他存放视频脚本的文件夹。
1. 读取现有脚本以“理解”他的写作风格。
1. 生成一个完全符合他风格的引言。
生成的引言是这样的：

> “Claude 刚刚发布了一个改变一切的东西。大多数人都知道 Claude Code，它是世界上最强大的编程代理，我每天都用它，还制作了几十个视频。但现在有了 Claude Co-work，大多数人根本不知道它是什么，它和 Claude Code 有什么区别，为什么它甚至存在。”
这正是 Riley 自己的风格。

**Skills：软件的终结者？**

在演示中，Riley 还展示了“Skills”的概念。当他要求 Claude 创建 PDF 时，Claude 使用了特定的技能文件（Markdown 格式的指令集）来完成任务。Riley 指出：“现在很多人说，大多数软件将被 Skills 取代。因为任何你需要在外部软件上做的事情，都可以通过一个带有特殊指令集的代理直接在 Co-work 中完成。” `[09:30]`

这意味着，你不再需要打开 PDF 编辑器、图片处理软件或简单的计算工具——你只需告诉 Claude，它就会用内置的技能为你生成。

**个人感受**

Riley 对 Co-work 的演示充满热情，尤其是当他发现 Claude 生成的引言“听起来完全像我”时。这种对“个性化”的还原，正是 AI 从工具向“同事”演进的关键。

**精华收获**

- **Co-work 是 Claude 的战略核心**：它提供了强大的自动化能力，同时解决了安全性的核心痛点。
- **Connectors 让 AI 真正融入工作流**：Claude 不再是一个孤立的对话窗口，而是连接你所有数字工具的枢纽。
- **Skills 可能是未来软件的形态**：与其购买专门的软件，不如让 AI 通过特定的指令集完成相同任务。

## 第三部分：2026 年初的 Claude 功能全景扫描 `[12:00-25:00]`

### 核心观点

Anthropic 在 2026 年初的发布节奏堪称“狂暴”，几乎每周都有新功能上线。从模型升级到平台扩展，从语音控制到远程操控，Claude 正在从单一对话工具演变为完整的操作系统级智能代理。

### 深度阐述

Riley 将所有新功能分为两大类：**Claude Co-work 相关** 和 **Claude Code 相关**。以下是对每个功能的深度解析：

Co-work 相关功能 (2026 年 1-3 月)

1. **Slack MCP 全计划可用** `[12:30]`
用户可以授权 Claude 访问 Slack，在离岗后自动总结错过的消息，并草拟回复。视频中的示例是：“我出差了三天，请总结我离开期间 Slack 上发生的所有事情，并起草一条消息发送到公共频道。”Claude 会在界面中生成一个可预览的消息草稿，用户可以一键确认并发送。

1. **Gmail 集成简化** `[12:50]`
连接 Gmail 的过程大幅简化，Claude 可以直接读取邮件、列出草稿、甚至发送邮件。

1. **Claude Opus 4.6 模型升级** `[13:00]`
新模型可以维持更长时间的代理任务（agentic tasks）。这意味着，当你让 Claude 执行一个复杂的多步骤任务时，它不会“迷失”或“忘记”最初的指令。

1. **Opus 4.6 极速版** `[13:20]`
另一个版本的 Opus 4.6 速度快了 2.5 倍，但成本是原来的 4 倍。Riley 分享了一个惊人的数据：“我和一些公司的开发者聊过，他们每天在 token 上的花费超过 1000 美元，但雇主觉得完全值得，因为他们完成的工作量远超这个价值。”

1. **Claude for PowerPoint** `[14:00]`
Claude 可以直接集成到 PowerPoint 中，自动生成完整的幻灯片。这主要面向企业用户，但预示着 Claude 将嵌入更多生产力工具。

1. **Co-work 登陆 Windows** `[14:20]`
2 月 10 日，Windows 用户终于可以使用 Claude Co-work。

1. **桌面端的“危险模式”** `[14:30]`
Claude Code 的 `dangerously-skip-permissions` 模式被引入桌面应用。当你启用“Act Mode”时，顶部会出现一个黄色的警告条：“高风险：Claude Code 可以不经询问采取行动。”Riley 分享了他的使用经验：“我经常让它搜索网络，找到某个应用并直接安装到我的电脑上，它就会照做。”

1. **免费计划的扩展** `[15:20]`
Anthropic 向免费用户开放了更多功能。Riley 认为，相比 ChatGPT 开始投放广告的策略，Anthropic 选择通过增强生产力用例来吸引用户，并且提供相当慷慨的免费套餐。

1. **文件夹指令 (Folder Instructions)** `[15:40]`
用户可以在 Co-work 的每个项目文件夹中创建一个指令文件，Claude 会严格遵守这些指令。这为团队协作提供了统一的标准。

1. **Co-work 插件系统** `[15:50]`
Claude Code 桌面版支持插件。插件本质上是指令文件集合，可以赋予 Claude 访问特定工具的权限（如公司内部数据库）。团队管理员可以创建插件并分享给所有成员。

1. **Claude Sonnet 4.6** `[16:20]`
2 月 17 日发布的 Sonnet 4.6 更便宜、更快，虽然不如 Opus 强大，但拥有 100 万 token 的上下文窗口（beta 版），性价比极高。

1. **Figma MCP 服务器** `[16:50]`
设计师可以更轻松地将 Figma 设计板转换为代码。Riley 更喜欢在 Claude Code 中使用 Figma，因为“你需要一个目录来存放代码文件”。

1. **Co-work 插件管理功能** `[17:30]`
管理员可以创建包含 Skills、Agents、Connectors 和 Commands 的插件，并分享给团队成员。这允许团队将高效的 AI 工作流标准化和复制。

1. **定时任务 (Scheduled Tasks)** `[18:00]`
Co-work 支持调度任务。Riley 演示了输入 `/schedule` 后的场景：“每天早上为我总结邮件中的重要任务，并草拟回复。”Claude 会检查可用的工具和上下文，然后设置每日任务。

1. **Claude for Excel** `[18:50]`
Claude 可以同时编辑 Excel 和 PowerPoint，并能保持两者同步：“更新模型，然后为明天的会议构建我的幻灯片。”

1. **内建图表生成** `[19:20]`
当你在对话中要求 Claude 生成图表时，它会创建可交互的图表，这些图表可以点击和进一步探索。Riley 对这项技术感到惊叹：“它有点像 Excalidraw，可以创建非常有趣的图表。”

1. **100 万上下文窗口成为默认** `[20:00]`
所有模型默认支持 100 万 token 的上下文窗口。这意味着你可以将完整的代码仓库或数年的笔记一次性输入给 Claude。

1. **Dispatch：持久对话代理** `[20:30]`
这是 Co-work 的一个研究预览版功能。Dispatch 是一个在你的电脑上持续运行的对话，你可以通过手机向它发送消息，它会执行任务，你可以随时回来继续。这反映了当前 AI 的一个巨大趋势：人们希望用手机远程控制电脑执行任务。

1. **Channels：通过 Telegram 控制 Claude Code** `[21:30]`
Claude 发布了“channels”功能，允许通过 Telegram 和 Discord 控制 Claude Code 会话。Riley 评论道：“我不知道为什么这如此重要，但似乎人们非常渴望用手机控制电脑。”

1. **项目工作空间** `[22:00]`
多个 Co-work 会话可以共享同一个项目文件夹，上下文会随着时间累积，形成对项目的持续理解。

1. **计算机控制 (Computer Use)** `[22:30]`
这是最令人震惊的功能：Claude 可以控制你的鼠标、键盘和屏幕，使用任何应用程序。Riley 称之为“AI 代理的终局”。虽然这类技术自 Claude 3.5 以来就已存在，但现在正在变得更好。他预测：“到今年年底，AI 在计算机使用上可能比普通人更快、更好。”

Claude Code 相关功能

1. **会话分享 (Session Sharing)** `[23:20]`
你可以公开你的 Claude Code 对话历史，并分享链接。Riley 提到，他们在招聘时不仅看开发者的最终成果，还会查看他们与 Claude Code 的对话历史。“通过观察一个人与超级智能 AI 的交互方式，你可以学到很多关于他们解决问题的能力。”

1. **/insights 命令** `[23:50]`
输入 `/insights` 会生成一个详细的总结报告，包括你提交了多少行代码、你的使用习惯、常用工具等。

1. **代理团队 (Agent Teams)** `[24:10]`
研究预览版中，一个“领导代理”可以将任务分配给多个“同事代理”，它们并行工作，进行调研、调试和构建，并相互协调。Riley 认为这将成为下一个大趋势：“那些真正擅长 AI 的聪明人，会同时操作 5 到 10 个 AI 代理来构建应用。”

1. **回退 (Rewind)** `[24:40]`
你可以通过 `/re` 命令回退到对话的某个节点，探索不同的路径，然后将学到的知识“带回”到原始路径。这大大增强了探索的灵活性。

1. **内置安全扫描** `[25:00]`
Claude Code 现在内置了安全扫描功能。当你在使用 Claude Code 时，可以点击“Security Scan”按钮，它会扫描你的代码库中的漏洞。这对“Vibe Coding”尤为重要——你可以快速确保应用没有安全隐患。

1. **远程控制** `[25:50]`
在终端中运行 Claude Code 时，输入 `/remote-control` 会生成一个链接，让你可以用手机控制这个会话。

1. **自动记忆 (Auto-memory)** `[26:10]`
Claude 现在会跨会话记住它学到的东西：项目上下文、调试模式、偏好的方法等。这是 OpenClaw 受欢迎的重要原因，现在 Claude 也具备了类似功能。

1. **终端语音模式** `[26:40]`
在终端版的 Claude Code 中，你可以通过 `/voice` 命令激活语音模式，按住空格键即可说话，Claude 会接收语音指令。

1. **定时代码审查** `[27:00]`
你可以设置每天自动运行的代码审查任务。

1. **/btw (By The Way) 命令** `[27:20]`
这是一个有趣的功能：如果 Claude 正在执行一个需要 20 分钟的长任务，你可以输入 `/btw` 并问它一个小问题（比如“这个重试逻辑是做什么的？”）。Claude 会在不影响主任务的情况下，在旁边回复你。

### 精华收获

- **功能即生态**：Anthropic 不是在“添加功能”，而是在构建一个操作系统级的智能层。Connectors、Skills、Plugins、Channels 这些概念表明，Claude 正在成为所有数字工具的统一界面。
- **安全性是护城河**：从 Co-work 的沙箱到安全扫描，再到危险模式的明确警告，Anthropic 在推动前沿能力的同时，始终没有忽视安全责任。
- **多代理是未来**：Agent Teams 功能的预览，揭示了下一代 AI 工作的模式：不是一个人与一个 AI 对话，而是一个人与一支 AI 团队协作。
- **从“工具”到“伙伴”**：记忆功能、语音模式、持久对话、远程控制，这些都在将 AI 从“用完即走”的工具转变为“持续陪伴”的伙伴。

## 第四部分：总结与展望——为什么 Claude 可能成为最重要的工具 `[27:40-结束]`

### 核心观点

与 OpenAI 分散产品线（ChatGPT、Codex、Sora、Atlas）的策略不同，Anthropic 选择将所有力量集中在一个“超级应用”——Claude Desktop 上。这种专注正在收获回报，使 Claude 成为当前全球最受欢迎的软件。而未来，AI 代理将从“被动执行”进化为“主动协作”，成为真正的数字同事。

### 深度阐述

Riley 在视频的最后部分，用一个有趣的实验展示了 Claude 的当前能力。他上传了自己录制视频时生成的 SRT 字幕文件，然后让 Claude 根据内容创建一个图表。他特意测试了内置图表功能：“Use your built-in diagram view.” Claude 不仅生成了图表，而且这些图表是可点击的，点击后会触发后续问题。这展示了 Claude 在视觉推理和交互设计上的进展。

**战略差异：Anthropic vs OpenAI**

Riley 总结了两家公司的战略差异：

- **OpenAI**：创造了多个独立产品（ChatGPT、Codex、Sora、Atlas 浏览器），试图用不同的产品覆盖不同的场景。
- **Anthropic**：只做了一个超级应用——Claude Desktop，将所有功能整合在一起。
结果是，Claude Desktop 正在成为全球最受欢迎的工具。虽然 OpenClaw 曾是最快增长的开源项目，但 Anthropic 没有选择收购它，而是专注于自己的产品，这个策略正在奏效。

**个人展望**

Riley 透露，他仍然在使用 OpenClaw，因为它更“自主”——它运行在一台专门的电脑上（通常是 Mac Mini 或虚拟电脑），永不关机。他正在构建一个“代理团队”，希望 AI 能更主动、更像一个有个性的人。他计划在未来继续深入探讨 OpenClaw 和代理团队，这预示着一个更有趣的 AI 协作时代即将到来。

### 精华收获

- **专注的力量**：在一个快速变化的行业，保持专注比追逐热点更难。Anthropic 拒绝了收购 OpenClaw 的诱惑，而是坚持打磨自己的产品，最终实现了更完整的集成体验。
- **从“工具”到“同事”**：当 AI 能够控制鼠标、键盘、屏幕，能够记住你的偏好，能够主动定时执行任务时，它不再是一个需要你“使用”的工具，而是一个你可以“委托”的同事。
- **未来是“代理团队”**：最高效的工作方式可能不是一个人与一个 AI 协作，而是一个人同时与多个专业化的 AI 代理协作，每个代理负责一个领域，由领导代理协调。
- **学习 AI 交互是核心技能**：正如 Riley 所说，观察一个人如何与 Claude 对话，就能判断他的解决问题的能力。未来，与 AI 交互的能力可能比传统的编程能力更重要。

## 结束语

从 2022 年末的 ChatGPT 魔法，到 2024 年的 Vibe Coding 浪潮，再到 2026 年初的通用代理时代，AI 的进化速度已经超出了大多数人的想象。而在这场进化中，Anthropic 用两个月密集发布的功能清单，不仅重新定义了 AI 代理的可能性，也重新定义了我们与数字世界交互的方式。

Claude Co-work、Claude Code、Skills、Agent Teams、Computer Use、Dispatch……这些名字背后，是 AI 正在从一个“能聊天的程序”进化为“能完成任何数字任务的同事”。当 AI 能像人一样使用电脑时，它就不再需要等待人类的指令——它可以主动观察、思考、执行。

正如 Riley 在视频开头所说：“如果你是开发者、企业家，或者只是想跟上 AI 步伐的人，这是你需要观看的那个视频。”而在我们完成这篇深度解析之后，或许我们可以说：这是你需要理解的那个未来。
