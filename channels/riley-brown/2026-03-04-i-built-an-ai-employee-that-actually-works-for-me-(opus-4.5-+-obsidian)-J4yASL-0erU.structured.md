# 将 Claude Code 打造为全能 AI 员工：基于 Obsidian 的智能工作空间构建指南

在人工智能技术飞速发展的今天，**Claude Opus 4.5** 已经不仅仅是一个强大的编程助手。通过与 **Claude Code** 的结合，它实际上是目前全球功能最强大的通用型 AI 智能体。在本教程中，我将一步步向你展示，如何将 Claude Code 转变为一个通用的智能代理，并与名为 **Obsidian** 的笔记应用配合使用。

通过这种方式，你可以创建一个由 Claude Code 自动组织和维护的工作空间。它能胜任各种任务，例如**搜索网络、进行内容研究、撰写脚本、制定计划**，如果你赋予它合适的工具，它甚至可以进行**图像生成**。

如果设置得当，你完全可以将 Claude Code 视为你的**联合 CEO 或一名员工**，将完整的任务委派给它。在本视频中，我将精确演示如何在你的电脑上将 Claude Code 设置为一个通用智能体，你可以将任何任务委派给它，而它会将所有成果井然有序地整理在一个漂亮的工作空间里。看完这个视频，你将能够利用这个全球最强大的通用智能体来完成数百种不同的任务。让我们直接开始吧。

## 一、 准备工作：所需工具与初始设置

在本教程中，我们将使用三个核心工具：

1. **Obsidian**：一个非常流行的本地笔记应用。所有文件都将存放于此，并能在其简洁的界面中进行查看和编辑。它最大的优点是所有数据都存储在本地。
1. **Cursor**：这是我的首选代码编辑器，因为我喜欢在其中使用 Claude Code 扩展的界面。我们将主要在 Cursor 中操作。
1. **Claude Code**：我们将通过 Cursor 中的扩展来使用 Claude Code。
> **注意**：本教程的所有操作步骤，在完全免费的 **VS Code** 中同样可以完成。如果你不想使用 Cursor，可以下载免费的 VS Code，并安装我将要在视频中演示的同一个 Claude Code 扩展。
### 第一步：在 Obsidian 中创建工作空间（Vault）

首先，打开 Obsidian。你将看到启动页面。

1. 点击创建新的仓库（Vault）。你可以将 Vault 理解为一个智能体工作空间。
1. 为简单起见，我们将其命名为 **`agent-workspace`**。
1. 选择一个你容易找到的本地存储位置（例如“下载”文件夹），然后点击创建。
创建完成后，Obsidian 会打开一个空白的笔记本。我们可以关闭左侧的图表视图（Graph View），暂时用不到它。

现在，我们创建一个核心文件：

1. 删除默认的欢迎笔记（Welcome）。
1. 新建一个文件，命名为 **`readme.md`**。
这个 `readme` 文件至关重要，它是你的智能体（Claude Code）每次执行任务时都会首先读取的“指令手册”。即使未来这个工作空间扩展到包含成千上万个页面，这个文件的作用也始终不变。无论你是将它用于公司、学校还是任何其他目的，`readme` 文件都将告诉智能体所有信息的位置。

### 第二步：在 Cursor（或 VS Code）中打开工作空间

1. 打开 Cursor 应用。
1. 选择“打开项目”（Open Project），找到并打开我们刚刚在 Obsidian 中创建的 `agent-workspace` 文件夹。
1. 你会注意到，Cursor 的界面与 Obsidian 有些相似。你可以点击打开 `readme.md` 文件，输入一些文字（例如“Hello”）并保存，它也会立刻同步显示在 Obsidian 中。
Claude Code 之所以擅长编程，是因为它非常了解代码库中发生的所有事情。在这里，我们虽然只存放 Markdown 文件，但本质上也是在利用一个“代码库”的概念。

### 第三步：安装并启动 Claude Code 扩展

1. 在 Cursor 中，点击左侧的“扩展”（Extensions）图标。
1. 搜索并安装 Claude Code 扩展。如果你已注册 Claude Code 服务，按照提示登录 Anthropic 网站即可完成设置。
1. 安装完成后，打开 Claude Code 扩展界面。
1. 为了获得最佳体验，我建议这样布置你的屏幕：
现在，我们来测试一下 Claude Code 的基本操作能力。让我们先创建一个临时文件 `test.md`，里面随便写点东西。然后，在 Claude Code 对话框中输入指令：

> “请删除测试文件，它叫 test。”
由于 Claude Code 是在当前项目（`agent-workspace`）中启动的，它知道上下文。发送指令后，你会看到它成功删除了那个测试文件。这证明了它可以像操作员一样管理你的文件系统。

## 二、 构建核心：创建智能体的“大脑”（Readme 文件）

接下来，我们要创建一个详细的 `readme` 文件，作为智能体每次执行任务前都必须阅读的“总章程”。这将是整个工作空间的核心。

1. 首先，确保 Cursor 的项目文件夹中是空的（可以手动清理一下）。
1. 在 Claude Code 中输入以下指令，让它创建基础的 `readme` 文件和初始文件夹结构：
> “我打算把这个工作空间变成我个人的工作空间，我是 **Vibe Code** 应用的 **CMO**。这个工作空间的主要目的是我可以把任何任务都委派给你。
>

> 首先，请在一个 `readme` 文件中，用非常简洁的段落把这个背景写出来。这个 `readme` 文件将是你每次与我对话时都必须阅读的。直接创建 `readme` 文件并写下这段话。
>

> 同时，请创建四个不同的文件夹，用于存放我将要创建的各类内容：
> 1.  ads
> 2.  content
> 3.  research
> 4.  strategy-documents”
发送指令后，Claude Code 会开始工作。我们可以看到它创建了 `readme.md` 文件，并在其中写下了对工作空间的定义。同时，它也在项目中创建了我们要求的四个新文件夹。这一切都实时地反映在 Obsidian 的界面中。

## 三、 实战演示：委派任务，让 AI 员工真正工作

现在，我们的“AI 员工”已经初步成型，可以开始委派实际任务了。

### 任务一：整理和分析现有素材

假设我已经搜集了一些竞争对手的广告文案，总共 16 条广告的完整转录文本，并粘贴到了一个名为 `top-ads` 的新文件中。现在，我希望 Claude Code 能帮我整理这些素材。

1. 在 Obsidian 中创建一个新文件 `top-ads`，将所有广告文案粘贴进去。
1. 在 Claude Code 中输入指令：
> “我在我的笔记里刚创建了一个新文件，叫 `top-ads`。我想让你帮我把它整理一下：
> *   我希望每个广告都有一个清晰、漂亮的二级标题（Heading 2），而不是现在括号里的‘新脚本’字样。*
> *   为了便于阅读，请把每个句子单独放在新的一行上，而不是保持段落格式。*
> *   将文件标题改成更合适的名字。*
> *   把它放到你认为最合适的文件夹里。*
>

> 另外，请更新 `readme` 文件：我希望你建立一个类似‘examples’的文件夹，这样你每次在写东西时都能参考。以后当我要求你写广告时，你就有权限去读取这个 `examples` 文件夹里的文件。这次的这个广告集，就作为第一个范例放进去吧。
>

> 请执行以上清理和归类工作。”
发送指令后，Claude Code 会开始处理这个相当长的任务。你可以看到它如何解析你的指令，并逐步执行。

### 任务二：根据指令和要求创作新广告

现在，我们可以着手创作新的广告了。

1. 我在 Obsidian 中又创建了一个新文件，列出了想要创作的五种广告风格：**一个 UGC 风格、两个教育风格、一个副业风格、一个针对妈妈的风格**。
1. 在 Claude Code 中，我可以这样委派任务：
> “我希望为 **Vibe Code 应用** 创作这五种风格的广告。这是我们公司自己的应用，专门用来构建移动应用。
>

> 这里有我们应用的官方文档：`https://vibecodeapp.com/docs`。请你去研究一下这些文档，了解我们应用的全部功能。然后，把这些信息也更新到 `readme` 文件里，确保你自己以后在创作广告时，始终记得使用这些资料。你可以在 `readme` 里再增加一段，从一个宏观的层面，用非常简洁的两段话概括一下 Vibe Code 应用是什么。
>

> 在创作广告时，请务必参考 `examples` 文件夹里的‘high-performing-ad-scripts’范例。
>

> 重要的一点是：**不要让广告听起来像是一个‘万能应用’**。要专注于具体的用例流程。意思是，视频不应该只是罗列所有功能，那样的广告效果不好。我希望你能围绕一个具体的用例来展开，比如‘你是如何用 Vibe Code 先创建了前端，又创建了后端，然后添加了支付功能，现在可以给应用收费了’，并讲述你创建它的特定目的。请确保广告中功能的使用流程是合理的。
>

> 广告创作完成后，把它们放到 `ads` 文件夹里，并再次更新 `readme`，说明所有广告都应存放在正确的位置，并简单描述一下我们工作空间里所有文件夹的组织方式。”
任务完成后，我们会在 Obsidian 的 `ads` 文件夹下看到 Claude Code 为我们创作的五份广告脚本。每份脚本都被自动分成了独立的文件，并且完全按照我们的要求，采用了单句成行的格式，便于后续编辑。例如，针对妈妈的广告脚本是这样开头的：

> “当妈妈意味着当一个没人雇佣的项目经理。我厌倦了反复唠叨家务：‘你打扫房间了吗？’‘喂狗了吗？’‘谁去倒垃圾了？’所以，我用 Vibe Code 自己建了一个家庭家务管理应用……”
### 任务三：自主研究与策略建议

一个好的员工不仅仅会执行命令，更应该能主动研究和提出建议。让我们看看 Claude Code 能否做到。

1. 在 Claude Code 中输入指令：
> “非常感谢你的工作，做得很好。
>

> 现在，我希望你做一些深入的研究。去调研一下我们的竞争对手，以及营销这类工具的其他成功方法，为我们创作更好的广告找到新的切入角度。
>

> 请将你的研究成果整理成一份研究报吿，放在 `research` 文件夹里。
>

> 研究结束后，请向我提出后续创作的建议，比如我们应该再写多少条广告，以及每一条广告应该采用什么角度。我的目标是到本次会话结束时，我们能有 15 条高质量的广告脚本。”
发送指令后，Claude Code 会立刻开始行动。你可以看到它在后台进行多次网络搜索，分析各种信息。

几分钟后，任务完成。我们打开 `research` 文件夹，看到一份名为 `competitor-ad-research.md` 的报告。报告内容极其详细，列出了做得好的品牌（如 Adobe, Shopify），并提出了十个全新的广告角度。

我回复它：“除了第八条（‘小型企业解决方案’）之外，其他九条都很好。请你为第八条想一个不同的角度，然后用和之前一样的格式，把所有这九条新广告脚本都创作出来。”

很快，`ads` 文件夹里又多了九份高质量的新脚本，每一份都遵循了我们设定的格式和风格。

### 任务四：学习优秀范例，模仿创作风格

除了广告，我们还需要创作高质量的长视频内容。我们可以让 Claude Code 学习优秀创作者的风格。

1. 我先找了一段优秀创作者 **Andre Jikh** 的热门视频的 YouTube 转录文本。
1. 在 Obsidian 的 `examples` 文件夹下，新建一个文件，将转录文本粘贴进去。
1. 在 Claude Code 中下达指令：
> “我刚刚粘贴了一个高质量的长视频脚本范例，创作者是 Andre Jikh。请帮我清理一下这份转录稿，然后把它也放到 `examples` 文件夹里。
>

> 请根据你的分析，为这个文件起一个合适的标题（保留 Andre Jikh 的名字，并在括号里注明脚本类型）。
>

> 同时，请在 `examples` 文件夹下创建两个子文件夹：`ads` 和 `content`，以便将来分类存放不同类型的范例。把之前那个‘high-performing-ad-scripts’移动到 `ads` 文件夹里，把这个 Andre 的脚本移动到 `content` 文件夹里。
>

> 最后，请更新 `readme` 文件：以后我给你的任何高质量范例中，如果包含付费广告植入部分（比如视频中段的赞助商广告），请自动将其移除。这对我们来说不是需要学习的部分（除非它本身就在 `ads` 范例文件夹里）。”
Claude Code 完美地执行了指令，创建了文件夹结构，完成了文件移动，并记住了自动移除广告部分的规则。

现在，是时候让它模仿 Andre Jikh 的风格来创作了：

> “现在，我想创作一个高质量的长视频脚本。请你研究一下 **Riley Brown**（也就是我本人）的 YouTube 频道，看看哪些视频表现得好。然后，结合 Vibe Code 应用的文档，用 Andre Jikh 的风格，为我写一个关于 Vibe Code 应用的脚本。
>

> 记得参考我们刚刚放进 `examples/content` 文件夹里的 Andre Jikh 范例。”
Claude Code 再次开始了工作：搜索我的 YouTube 频道、复习 Vibe Code 文档、同时参考 Andre Jikh 的写作风格。最终，它在 `content` 文件夹里生成了一份完整的脚本。脚本开头是：

> “嗨，我是 Riley Brown，希望你今天过得不错。今天不谈钱，我们来聊聊 AI。请留下来，因为我将要展示一些在两年前绝对不可能做到的事情：**在不写一行代码的情况下，在一天之内构建五个功能完备、可以直接上架 App Store 的移动应用。**”
这个脚本质量非常高，甚至给了我拍摄新视频的灵感。

## 四、 总结与未来展望

回顾整个视频，我们实际上做成了以下几件事：

1. 创建了一个 `readme` 文件，它作为每次启动智能体时的“大脑”，包含了核心指令和背景信息。
1. 建立了清晰的文件夹结构，让所有内容（广告、内容、研究、策略）都能各归其位。
1. 通过委派任务，让 Claude Code 完成了素材整理、广告创作、竞品研究、策略建议和风格模仿写作等一系列复杂工作。
这个工作流的未来发展方向是什么？我有个愿景：未来我们可以轻松地为 Claude Code 制作各种工具（MCP 工具），让它不仅能写作，还能直接生成图像、制作视频缩略图等。将来越来越多原本需要手动完成的事情，都可以通过这个智能体界面来完成。

同时，AI 模型本身也在飞速进步。Claude Opus 5 在处理上下文和管理海量信息的能力上，可能会比现在的 Opus 4.5 强上数倍。到那时，我们就能创建拥有数百万个文件夹的超级工作空间，并在所有信息之间进行深度分析。

我们团队已经开始在各个部门推广这种工作模式：内容团队有自己的工作空间，客户支持团队也通过 Claude Code 来分析用户反馈的问题，找出产品痛点。**Claude Opus 4.5 在模仿特定写作风格方面做得非常出色，而成功的关键就在于：给予它相关的背景信息、明确的目标，以及最重要的——高质量的范例。**

因此，我强烈建议你立刻动手，为自己创建一个这样的工作空间，尝试生成不同类型的内容，进行研究。你甚至可以为你的研究设定更精细的指令，比如“不要引用 X、Y、Z 网站的信息，多参考这个账号的内容”。

在未来的视频（大约三四个月后）中，我将更详细地展示如何为 Claude Code 添加 MCP 工具，让它变得更加强大。敬请期待。感谢观看，下次见！
