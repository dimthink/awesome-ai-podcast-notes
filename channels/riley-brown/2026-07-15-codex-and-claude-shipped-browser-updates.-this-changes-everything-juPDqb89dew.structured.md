# Codex与Claude内置浏览器更新：AI代理正在重塑你的工作方式

## 引言

OpenAI 和 Anthropic 在同一天对 Claude 桌面应用和 Codex 应用进行了相同的更新。我认为这将对 AI 代理的业务应用产生巨大影响。这次更新就是它们内置的浏览器——现在，在 Claude 桌面版和 Codex ChatGPT 应用中，你可以在每个聊天会话中打开多个标签页，这对于使用这些工具经营业务、做营销和内容创作尤其有用。

## 更新实战：多标签同时运作

最新版的 Claude Code（Anthropic 的编码工具）现在可以在一个会话中打开10个不同的浏览器标签页。在 Codex 应用中，我可以运行这样一个提示：“打开我的 YouTube Studio 分析，打开我的 Notion 视频数据库，打开我的 X (Twitter) 通知，我最重要的5封邮件，以及挪威足球比赛的比分，并将所有这些作为新标签页在应用内置浏览器中打开。”我还可以指示它使用子代理来处理任务。子代理正在启动，所有不同的浏览器标签页都在打开：YouTube 分析、Twitter 通知、视频数据库、挪威比分（2:1，挪威输了）。我可以全屏浏览，更多标签页持续打开，包括不同的重要邮件以及一份自动创建并打开的“最近支持票据摘要”文档。这表明，Codex 和 Claude Code 中的 AI 代理可以为你执行操作，然后将结果打开在不同标签页中。

## Claude Code 新功能：浏览器可访问任意网页 + 注释工具

三天前，Claude Code 的浏览器还只能打开本地主机链接（例如在本地开发应用时可以打开），但无法访问实际网站。现在，你可以直接要求 Claude：“我想写一篇关于 Claude Code 最佳使用方式的脚本，请快速打开一些相关的链接，不用想太久，直接快速打开就行。”它会立刻打开最佳实践页面并新建标签页，现在我们看到两个标签页打开了。不过我不太喜欢的是它一个一个打开，而没有在后台批量处理。

几周前我做过一个视频解释这个功能为什么重要——我认为它正在重新定义我们使用计算机的方式。以前你需要导航到浏览器（如 Arc），再找到需要的标签页；现在你可以直接对 AI 代理说：“嘿，我要做一个关于某主题的视频，请打开所有相关内容。”它就可以为你打开所有信息。当然，我不使用 Claude Code 来做知识工作，这方面我用 Codex。所以对我而言，Claude Code 不太擅长这种情况。

另一个 Anthropic 新增的功能是注释工具（annotation tool），它很像 Excalidraw，在底部打开。无论你在哪个页面，都可以与 Claude 沟通。看，我可以选中这部分文字，添加文字“我们真的需要这个吗？”，然后加入聊天，Claude 就会看到这个注释。如果我已经将 Claude 连接到 Google Docs（目前 Codex 有连接而 Claude Code 我还没设），我就可以让它直接删除那段内容。

## Codex 内置浏览器深度体验：四种高效用法

接下来展示如何在 Codex 中使用这一功能。在新标签页中，可以按 Command + T 打开侧边浏览器，然后像普通网页浏览器一样输入 Google Docs 并导航过去。你可以点击全屏按钮，降低聊天窗口，完全用浏览器模式工作，同时能与 AI 交流。例如创建一个新文档，标题设为“Riley 对超级应用浏览器的痴迷”。由于 Codex 知道我们正在做什么，我可以直接打开这个文档，用 5.6 soul 模型说：“请用你对我的一切了解，写两段关于这个主题的文字。别想太久，直接在这个文档里写出来。”我调低模型聪明程度以更快响应，然后发送提示，Codex 就会直接在这个文档里写作。因为 Codex 几乎负责我的一切，它非常了解我——Google 虽然有内置 AI，但只知道我 Google Drive 里的内容；而 Codex 知道我的 Notion、我的 Drive、我执行过的所有任务的记忆，几乎一切。它刚刚写好了内容，我可以要求“第一段太长，请改成推特风格”。而且任何时候我可以按 Command + T 新建浏览器标签页，打开 Twitter 进行后续操作。

接下来，我可以说：“我想从我的 Agent Native 账号发布这条推文，但还不完美。请制作三个版本，每个作为一个新浏览器标签页预填好推文，不要实际发布，让我选择发送。”AI 会为我生成三个版本。

以下是四种在 Codex 中使用浏览器的主要方式：

### 1. 手动打开浏览器（Command + T）
最直接的方式。我常常在 Codex 里新建一个聊天会话，直接按 Command + T 打开侧边浏览器，像普通浏览器一样导航到任何网站，然后让 Codex 编辑或执行操作。

### 2. 让 AI 打开特定内容
你可以直接对 AI 说：“嘿，我们正在制作一个关于 Devin 的长视频，它在 Notion 里，请把它在浏览器中打开。”由于 Codex 已经集成了 Notion，它能轻松找到该内容并在新标签页中打开。

### 3. 让 AI 打开多个标签
你可以命令：“我不知道接下来该做哪个长视频，请打开我所有正在进行的长视频，同时把我的短视频项目页面也在浏览器标签页中打开。”AI 会一次性打开所有相关标签。正如你所见，所有16个正在进行的视频页面都打开了，包括我现在正在拍的这一个。

### 4. 将浏览器操作嵌入“技能”（Skill）
这是最有用也最有趣的一种方式。你可以创建一个技能：当某件事发生时，在浏览器中打开某个内容。例如，我设置了一个“邮件草稿”技能。在 ChatGPT（现已合并为 Codex）中，我新建一个线程说：“我需要你查看当前的支持工单，为每个工单起草回复。”然后用斜杠调用我的邮件草稿技能，该技能的一部分就是将这些工单在每个新标签页中打开。它会自动执行：搜索邮箱、识别每一个当前支持票据、阅读完整线索、保存回复、验证，然后将每个回复草稿打开在一个单独的 Codex 浏览器标签页中，并提示什么都不会发送。然后我可以查看这些草稿并选择发送或编辑。这实际上是我发送所有邮件的真实流程。

## 从“告诉代理做什么”到“问代理应该做什么”：个人操作系统

现在我想花点时间谈谈这些工具正在成为我的个人操作系统。最好的描述来自 Jack Dorsey（Twitter 前CEO，现任 Block CEO）的一条推文：“我已经从告诉代理做什么转变为向它们询问应该做什么，然后拉取最好的一条线索。”

举个例子：基于你对我的一切了解和即将到来的重要截止日期，思考邮件、日历，我可以问：“我接下来应该处理什么任务？给我三个选择。” AI 连接着我的所有工具——Google 日历、Notion、邮件——它会评估然后做出决定。它正在读取我的 Google 日历、Notion、邮件，然后向我呈现应该做的事情。比如它说：“你需要录制的两个播客在周四，非常重要。”然后我就可以拉动线索：“打开这两个播客的 Google 文档，在另外的标签页中打开所有相关调研以及外部资料。同时在聊天区（而不是任何文档里）针对每位嘉宾列出10个问题和关于结构如何获得最高观看量的建议。”我只需要确认，AI 就开始打开我需要的所有标签页：每个播客的文档、嘉宾的 LinkedIn、之前录制的播客链接等。我可以说：“请根据你对我的了解和我讨论过的兴趣（比如喜欢用纸笔之类），为 Steven 那集写出一个精彩的开场介绍，我会选择是否编辑。”AI 就能直接编辑文档，这使用了 Google Docs 插件——我可以 @mention 插件，或直接说出来，它就会执行。我说：“请将其放入 Steven Haney 那集的文档中。”它就立即将开场白添加到了文档，并提供了链接。如果我没有打开这个文档，我可以轻松地在浏览器中打开并查看。

这就是我如何将 Codex 当作操作系统来使用——AI 帮我决定该做什么，我只需要顺势展开行动。

## 实用工作流展示

接下来展示几个使用 Codex 内置浏览器的特别酷的工作流。

### 工作流 1：邮件草稿技能
如前面所展示，我告诉 Codex：“查看所有支持工单，为每个工单草拟回复。”利用“邮件草稿”技能，AI 自动将每个回复草稿在新的浏览器标签页中打开，我可以直接发送或编辑。

### 工作流 2：Google 图片搜索视频 B-roll
当我在写一个视频脚本时，我有一种视频风格：录制视频时同时显示图片作为 B-roll。以前需要手动搜索 Twitter 或 Google 图片。现在我可以让 Codex 做这件事：“我要在每一句台词的时候显示至少15张图片。请根据脚本，按出现顺序在 Google 图片中搜索相关的画面，每个搜索作为一个新的浏览器标签页打开。”AI 分析每一行脚本，决定搜索内容，然后在我眼前快速打开所有标签页。我可以直接浏览每个标签页的搜索结果，挑选合适的图片作为 B-roll。例如，针对一个脚本，它搜索了“OpenAI GPT 5.6 sole model announcement”、“OpenAI logo”、“新的AI模型发布”、“Claude fable”、“Anthropic CEO AI safety warning government”等，所有结果都在标签页中，我可以直接引用。

### 工作流 3：准备推文（从脚本或下载图片）
另一个例子：我可以要求 AI：“请根据这段文字写出三个版本推文，每个在一个新浏览器标签页中预填好，不要发布，让我选择发送。”44秒后，三个推文标签页打开，我可以逐条审阅，关闭不需要的，然后发送我喜欢的。或者，我还可以使用下载文件夹中的图片：“分析这些图片，并写出配图推文，每个图片在一个新标签页中准备好发送。”AI 会自动从电脑中找到图片，上传到 Twitter 草稿并生成文案，每个标签页一个推文，我只需点击发送。你可以在页面中看到它正在手动上传文件并撰写文案。

### 工作流 4：并行多任务处理
这个工作流展示在一个 Codex 会话中同时启动三个独立的研究线程。我对 AI 说：“请创建三个独立的 Codex 会话线程。每个线程都需要打开许多浏览器标签页。第一个线程的任务：为我的下一个视频查找10个视频的钩子，并在第一个标签页创建一个 Google Doc，同时将每个视频在各自的标签页中打开。第二个线程的任务：为升级一个适合多摄像头拍摄和使用 vMix 的 Windows PC 做深入产品研究，并在浏览器中打开所有相关链接。第三个线程的任务：深度研究并找到10个理想的播客嘉宾，在浏览器标签页中打开显示他们为何合适的链接，如果是 YouTube 视频则直接指向时间戳。这些应该是独立的会话，不要放在文件夹里，放在未分类中。”AI 立即创建了三个新的会话线程（研究1、研究2、研究3），每个都正在打开浏览器标签页。我可以点击第一个线程，看到它正在查找适合我视频的钩子，并创建了 Google Doc，同时打开了相关 YouTube 视频页面。第二个线程提供了多款 Windows PC 选项和测评链接。第三个线程列出了嘉宾如 Nate B Jones（很适合上节目）和 Nate Hurk，并打开了他们的视频链接。所有研究结果和标签页都单独整理，我可以随时深入查看。

## 总结

我认为，想要一个应用能使用 AI 完成几乎所有事，一个完整的内置浏览器是必不可少的。当 AI 代理工具弹出外部浏览器（如 Chrome）并把你带离当前工作平台时，非常令人烦恼。而 Codex 和 Claude 桌面应用都在内部直接增加了这个浏览器功能。我们讨论了使用它的不同方式，也讨论了为什么我相信它正在成为类似操作系统的存在，还展示了许多工作流。
