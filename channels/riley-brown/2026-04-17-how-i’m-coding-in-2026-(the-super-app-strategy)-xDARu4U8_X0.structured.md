## OpenAI 的“超级应用”泄露

两天前，推特上泄露了 **OpenAI** 的新应用 —— **Codeex**，他们称之为“**超级应用**”（super app）。他们基本上将 **Atlas**（他们的智能体浏览器）、**Codeex** 和 **ChatGPT** 全部整合到一个工具中。人们实际上不想要分离的应用，他们想要一个**智能中枢**（central hub for intelligence）——至少 OpenAI 是这么认为的。

从泄露的截图可以看到，这个应用有一个全新的浏览器部分。在此之前，Codeex 的当前版本没有这个右侧面板来显示浏览器。这与当前的 **Claude** 桌面应用几乎一模一样：你有聊天线程、智能体、预览窗口。完全相同。

## 为什么左侧面板成为标配？

我认为人们喜欢这种左侧面板的原因是：**这是在不同聊天之间切换的最简单方式**。当智能体需要较长时间运行时，问题就来了：你在等待的时候做什么？合乎逻辑的做法是**切换聊天，同时处理多项任务**。这就是人们想要的。

我认为所有这些公司都逐渐得出结论：**这种界面最适合此类活动** —— 能够快速切换智能体和正在处理的任务。

## 新范式：多任务处理变得至关重要

在这种新范式、新世界中，**多任务处理**变得重要得多。你需要能够进行多任务，因为当你把大量工作委托给智能体时，你需要找其他事情做。

我身处旧金山，认识很多顶尖开发者。我与许多公司的许多人聊过，他们都在**寻找方法，通过并行地与智能体对话，同时处理代码库的多个部分**。他们常常同时处理五到十件事情。如果你擅长**智能体编程**（agentic coding），你就能在更短的时间内完成更多工作。

## 两种优势：模型 vs. 集成

我认为，思考每个工具拥有何种优势是有用的。主要有两种优势：

- **模型优势**
- **集成优势**
### 谁拥有模型优势？

**重要说明**：对于 OpenAI、Claude 和 Anti-Gravity 这三家，如果他们的模型在某个类别上成为世界最佳，他们可以以比任何人都便宜的价格在自己的平台上提供这些模型。例如，在 Claude 桌面应用上花 200 美元购买积分，可以获得相当于 4000 美元的 token 使用量——他们在补贴你的使用，以吸引更多用户到他们的平台。而在 Cursor 上，你可以使用 Claude Opus 4.6，但 200 美元很快会用完，这就是为什么 Cursor 试图让你使用他们更便宜的模型。问题在于，他们还没能训练出与 OpenAI 和 Claude 比肩的模型。

### 谁拥有集成优势？

我认为最重要的两个赢家是 **Manis**（Meta）和 **Anti-Gravity**（Google）。

- **Manis (Meta)**：Meta 拥有 Facebook、Instagram、WhatsApp、Threads、Facebook Ads Manager 等。他们正在向 Manis 开放广告管理器，你可以通过 Manis 完全控制 Facebook 广告，还可以自动联系 Instagram 网红。这在其他任何智能体平台上都做不到。Meta 是一个非常封闭的公司，不轻易分享算法，API 也关得很紧，但他们允许 Manis 这样做。
- **Anti-Gravity (Google)**：谷歌拥有庞大的工具套件，可以与 Gmail、Google Docs、Google Sheets、演示文稿等集成，并且收集了大量用户数据，这对通用智能体平台可能非常有用。

## 深入：如何使用 Claude 桌面应用创建有用的智能体

接下来，我将使用 Claude 桌面应用来创建一个对你有用的智能体或系统。

### Claude 桌面的三种模式

Claude 桌面分为三种模式：**Chat**、**Co-work** 和 **Code**。你可以将它们视为：**简单、中等、困难**，或者 **低风险、中风险、高风险**。

- **Claude Code**：可以完全控制你的整个计算机。我设置它默认“绕过权限”（dangerously skip permissions），这样智能体可以在我电脑上任何地方自由行动。
- **Claude Co-work**：隔离到计算机上的特定文件夹，设计上更安全。
- **Chat**：基础模式。
我认为未来的产品会是 **Claude Code** 或 **Codeex** 的样子。如果你要求它做一个演示文稿、电子表格或文档，它会直接做，识别出这是通用智能体任务而非编码任务，然后打开正确的窗口执行。它可能将其标记为“co-work”标签，而不是一个单独的沙盒式 co-work 功能。我个人觉得沙盒 co-work 有点烦人，我一般只用 Chat 或 Code。但有些人（如法律行业或需要反复创建文档的人）非常喜欢 Co-work。

对于有用的通用智能体，我希望它是**不受限制的**。虽然有风险，但我更想快速行动，不想接受一堆请求、授予特殊权限、花大量时间设置。我宁愿让我的智能体自由发挥，如果做错了，我就说“别再那样做”。我希望它**不受限制**，能够完全访问计算机和集成，这样它就能**自主工作**，我不需要干预。

### 让智能体更自主：计划任务（Scheduled Tasks）

Claude Code 新增了**计划任务**功能。你可以输入 `/schedule`，然后说：“请每天中午执行此任务，每次寻找新公司。这些应该是AI智能体公司。” 这会创建一个新的计划任务，每天自动运行。

### 让智能体始终在线：远程控制（Remote Control）

人们创建“始终在线”智能体的方法之一是设置**远程控制通道**。例如，**OpenClaw** 很受欢迎，你可以把一个智能体放在 Mac Mini 上，然后通过 Telegram 给它发消息。计算机保持开机，你可以从外部设备（iMessage 或 Telegram）随时消息它，无论你是在商店还是度假。

Claude 也创建了一个类似功能叫 **Dispatch**，但我用的是另一个功能叫 **Remote Control**。在终端中运行 `claude` 会话，输入 `/remote/control`，就会激活远程控制。然后你可以从手机发消息：“请查看我今天早些时候保存在下载文件夹里的 PDF，并总结给我，内容是关于 AI 智能体的。” 只要你的电脑开着，你可以在任何地方使用它，它有权访问你电脑上的所有文件。

Claude 有一个内置通道 —— **Claude iOS App 通道**，很多人喜欢这个。许多公司也在创建 Telegram 和 WhatsApp 集成，以便你可以给 AI 智能体发消息。

### 心跳功能（Heartbeat）与模仿方案

**OpenClaw** 还有一个非常实用的功能叫 **Heartbeat**：每隔30分钟，它会醒来，检查 `heartbeat.md` 文件，然后根据文件中列出的内容检查各种事情。例如，你可以说“每30分钟检查我的邮件，如果需要回复，请回复”。

要模仿这个功能，你可以在 Claude 中设置集成，比如：“每30分钟检查邮件，如果重要就回复。” 理想情况下，你应该提供大量**上下文**，比如你的目标。以我作为内容创作者为例，我每天收到 50-100 封赞助邮件。你可能需要设置过滤器：只想与符合特定类别、特定预算的公司合作。你需要给智能体详细的说明文档。如果我花 2-3 小时来做，我可以给它很好的指令，从而创建一个**完全自主的 AI 智能体来帮我谈判品牌合作**。

让智能体更主动的另一种方式是使用 **schedule** 功能，并让它更开放，比如“每30分钟检查我的邮件，做点有用的事”。

## 最重要的部分：定制化（插件/技能/连接器）

如果你想最有效地使用智能体，你需要专注于**定制化**：**插件/技能/连接器**。

在 Claude 中，他们已将“插件”选项卡改为“**定制**”（Customize）。点击后，你会看到 **Skills** 和 **Connectors**。这些名称总是在变（skills、connectors、MCPs、integrations、plugins），但你需要意识到：**技能本质上就是一个 Markdown 文件**。

### 第一步：坐下来写文档

在你深入使用 Claude Co-work 或 Claude Code 之前，你应该花三个小时，审视你的工作。

以我为例，我是内容创作者，我花时间研究某个细分领域（AI、氛围编码、智能体），也研究其他创作者的成功内容（主题和形式）。我还是公司的 CMO 和创始人，有另一套任务清单。**每项任务都有一个标准操作程序（SOP）**。

对于每项任务，你想创建一个**一页纸的 SOP**。例如，研究其他创作者的有效内容时，首先考虑需要访问什么：YouTube / Supa Data（一个连接器/API，可以快速拉取 YouTube 字幕）。然后列出步骤：列出我喜欢的创作者（他们的信号表明我想制作类似内容），列出我感兴趣的所有主题（OpenClaw、Claude Code 等工具），然后创建文档。

你可以在 **Notion** 或 **Obsidian** 中做这件事。**最关键的部分是提供真正优秀的例子**。你需要创建高质量的例子。

### 我的知识库实践

我在 Notion 中创建了一个新的团队空间，叫 **Master Database**，里面有：

- Riley 作为 CMO 的角色
- Riley 作为内容创作者的角色
- 其他公司上下文
- 受众与消息传递
每个都有大量信息。我还有一个叫 **Paragon Templates** 的东西：使用 Supa Data API 拉取我喜欢的内容创作者的视频字幕，然后创建了一个技能，精确指导如何像 Callaway（我的一位朋友，很擅长做视频开场白）那样制作开场白。我把这些称为“典范模板”——高质量的例子。

当我的 AI 智能体被指示“查看 Notion”时，它会进入这个知识库，读取文件或页面。**我是为智能体而写的**。这是我的知识库，包含关于 Riley 的信息。智能体会读取它，知道一切如何组织。任何时候我需要智能体获取上下文，我就说“检查 Notion”。它首先查看的页面是 **Knowledge Base**。

### 技能与文档的联动

你创建的文档越多，你的智能体就会**越像你一样思考**。然后你可以专注于插件和技能。例如，你可能有一个“脚本撰写者”技能，并且把所有脚本写在 Google Docs 中。你可以在技能中指定：“总是在这个特定的 Google 文档上写，或者在这个账户中创建一个新的 Google 文档”。作为脚本撰写者技能的一部分，它应该引用文档：“如果需要了解 Riley 的观点，请查阅文档中的这个页面。”

因此，你需要为你自己或你的公司创建一个**内部文档**，作为**单一事实来源**，你的智能体在执行任务时可以访问它。你甚至可以创建一个工作流，让你的 AI 分析并建议你应该更新哪些内容。它会问你问题，你回答，然后 AI 去更新文档的那部分。我还在研究这个工作流，因为我想创建一个能自动更新文档的系统。

你应该把这些技能导出并保存在一个文件夹中，这样**可移植性**就很高。假设现在 Claude Code 最好，但两个月后 Codeex 明显更好，你希望能够把这些技能轻松迁移到新平台上。因此，我建议把它们放在 Notion 或类似的地方。

### 当前最佳编码模型

目前，**Claude Code** 和 **Codeex** 是世界上最好的两个编码模型。我认为 Codeex 略微领先于 Claude Code。我使用的是 **GPT-4 Extra High**（effort 设为 extra high）。这个模型编码能力极强，我仅用几个提示词就用 Codeex 构建了一个完整的 Replicate 克隆版 iOS 应用，它运行了 40 分钟，直接就能用。

当你使用 Codeex 和 Claude 时，注意：在它们自己的应用（桌面应用或终端）中使用，**性价比最高**。如果你在 Cursor 中使用 Claude，会贵得多。所以，如果你有大约 50% 的工作是编码，我推荐使用 Claude Code 或 Codeex。这些智能体默认就非常擅长编码，你不需要做太多前期定制。

## 记忆（Memory）：AI 领域最有趣的事情

**记忆**可能是目前 AI 领域最有趣的事情。很多公司正在尝试解决记忆问题：如何有效地在智能体中存储信息，让它记住关于你的一切。这些创新发生在**记忆层面**和**文件层面**。许多公司正在创建特殊的数据库，你可以放入项目中，让智能体拥有更好的记忆。

**Anthropic** 和 **OpenAI** 正在致力于将记忆直接烘焙到模型中。未来一年左右，我们将在记忆方面看到最大的进步（除了原始编码能力）。他们会显著改善记忆。

### 你现在能做什么：记录一切 + 定时任务

你可以设置一个 **cron job**（自动化任务）。我的 OpenClaw 运行在楼下的 Mac Mini 上，我设置了一个 cron job：每晚 9 点，它在一个 Notion 文档中写日记。

Claude Code 最近泄露了一个正在开发的功能叫 **Dreaming**。我相信这类似于上述做法：每天某个时间（比如夜间你不使用智能体时），你的智能体会去清理、记录你当天做了什么，并整理成小笔记。这样，将来你问智能体问题时，它可以参考这些笔记，更好地理解你想做什么。

这是一个**数十亿甚至万亿美元**的产业 —— 仅仅是为了让智能体拥有更好的记忆。目前你能做的最好的事情就是：**在你的笔记和文档中保持高度组织性**。在记忆领域取得重大突破之前，这是最好的做法。

## 总结：提升 AI 智能体能力的六个关键方向

最后，无论你的目标是什么（在商业环境中更好地使用 AI 智能体），你应该思考以下**六件事**：

1. **文档（Documentation）**：为你的工作、你的角色创建非常非常好的文档。可以把它看作一个**维基**（wiki），你的智能体在需要时可以访问。
1. **工具（Tools）**：列出你使用的所有工具。例如 Gmail、Granola（会议笔记）、Zoom、YouTube（如果你是我这样的 YouTuber，YouTube 集成对内容创作非常有用）。
1. **文档列表（Documents）**：列出你创建的所有文档。如果你在商业领域工作，你会经常创建文档。找出优秀范例，保存到一个文件夹，让你的智能体访问这些技能（也可以把它们看作 **SOP**）。对于你做的每一件事，列出具体步骤。由于 AI 在分析视频方面越来越好（Gemini 可以摄取视频），你甚至可以制作一个 **Loom 视频**，精确描述你如何做事，这可以作为 AI 智能体的有用上下文。
1. **时间浪费点（Time Wasters）**：列出你浪费时间的所有领域。通常你可以把这些变成自动化。如果你希望你的智能体更主动、更自主，你应该列出浪费你时间的事情 —— 你不想让它随机乱动，而是希望它解决**对你来说真正痛苦**的事情。
1. **明确的目标（Goals）**：拥有非常清晰、有文档记录的目标。最好再有一个电子表格来跟踪你的进度，这样你的 AI 智能体就能知道你是否在朝着特定目标前进。如果它理解你的目标，它就可以将其自主行动对准这些目标。**你的智能体最需要理解的是：你想去哪里，以及如何验证你是否到达或正在轨道上**。例如，如果你雇用一个帮你投广告的顾问，他们会想知道你的预算和目标（例如 2027 年你想达到多少收入或利润），然后反向推算每月需要花多少钱在广告上。如果智能体不理解你的目标，你只会漫无目的地做事。
1. **存储位置**：将这些信息存储在不绑定特定平台的地方，最好是**本地**或使用 **Notion**（因为本视频中提到的所有智能体工具都有 Notion 集成）。如果你存储在 Notion 上，你可以将 AI 智能体与 Notion 集成，并告诉它确切地在 Notion 的哪里找到所有这些信息。
## 最后的鼓励

记住，**你不会选错**。如果你使用 Manis，也不会错 —— 它们在底层使用 Anthropic，拥有技能，而且可以说是一个最好的智能体工具之一。只需专注于其中几个。**退一步，审视你正在做（或试图做）的工作，记录一切，然后让你的智能体开始工作**。因为 AI 正在变得更好，AI 会变得更好。
