## 理解AI发展的三个阶段

在我们深入探讨 Claude 和 Claude Co-work 的所有全新更新之前，我想先谈谈自 2022 年底 ChatGPT 问世以来，我们所经历的 AI 发展的三个阶段。我保证视频的大部分时间还是会用来讨论所有 Claude 的新功能，但让每个人了解我们所经历的 LLM 的不同阶段是很重要的。

**第一时代：文本交互时代 (始于 2022 年底)**

当 ChatGPT 刚发布时，它就像一个奇迹。它是一个似乎无所不知的文本生成器。是的，它有时会产生幻觉，它并不总是完美的，但它是一种神奇的技术，可以为你写文章、写邮件，做各种不同的事情。它统治了世界。这个阶段在很大程度上是“文本到文本”的。你可以问 ChatGPT 一个问题，它用文本回复。后来，你也可以向 ChatGPT 输入图像，再后来，你可以在 ChatGPT 中上传文件，它不仅能回复文本，还能回复代码。你可以询问关于特定代码行的问题，或者让它为你创建一个函数或代码页面。

**第二时代：氛围编程时代 (始于2024年中期)**

区分 2023 年和 2024 年大部分时间与下一个时代（氛围编程时代）的关键在于，公司开始意识到你可以直接在生成代码的应用中渲染代码。第一家做到这一点的公司实际上是 **Claude**，他们创造的产品叫做 **Claude Artifacts**。

在早期的 ChatGPT 时代，它只能生成一个代码文件，但无法实际渲染它。而 Claude Artifacts 在 2024 年 5 月左右问世，它不仅能生成代码，还能直接在界面中渲染代码。你可以与之交互。这让我立刻投入其中，去学习如何用 AI 创建应用。之后，我很快从 Claude Artifacts 转向了像 Cursor 这样的工具。Cursor 有一个功能，你可以直接创建一个新项目，然后说“创建一个新的应用，包含前端和后端，使用 Firebase”，Cursor 就会尝试编写所有代码，生成完整的代码文件。这不仅仅是像 Claude Artifacts 那样的前端，而是一个完整的生产级应用。

但归根结底，Cursor 是为开发者准备的工具。而从这个领域衍生出了许多公司，如 Lovable、Replit、Bolt、Vibe Code、Emergent 等，它们试图让那些没有或只有很少编程经验的人，比如想快速创建登录页面的营销人员，也能更容易地进行氛围编程。直到最近，在这些平台上，你甚至可以创建完整的生产级应用。

**第三时代：通用智能体时代 (现在)**

然后，在去年下半年，一个名为 **Claude Code** 的工具发布了，并迅速风靡全球。Claude Code 是一个可以直接在终端使用的工具。我发现我可以用 Claude Code 创建任何我想做的应用。我在为我们公司制作内部工具，创建登录页面、营销材料，甚至幻灯片，只需要几个提示词，它就能直接把这些东西存在我的电脑上。这就是为什么这么多人爱上 Claude Code 的原因。

到 2025 年底，人们开始意识到，你可以用 Claude Code 自主或半自主地完成大量其他任务，比如营销任务、回复邮件、连接其他工具并代表你执行任务。

紧接着，在 2026 年初，我们见证了 **OpenClaw** 的诞生。OpenClaw 是第一个通用 AI 智能体，其背后的核心引擎叫做 Pi。它与 Claude Code 有些相似，但 OpenClaw 的设计初衷是成为一个通用智能体。

因此，这就是第三个时代：通用智能体时代。我们正试图将所有能让你创建任何类型应用的技术，应用到通用智能体上。这就是像 OpenClaw、Manus、Perplexity Computer、Claude Co-work 和 OpenClaw 等公司正在做的事情。这场竞赛变得如此重要，以至于 OpenAI 今天（在我录制视频前一小时）决定砍掉他们所有的副项目，完全放弃 Sora 应用，专注于创建一个统一的 OpenAI 超级应用，它将结合 ChatGPT、Codex 和 Atlas（他们的浏览器），以应对 Anthropic 的超级应用——**Claude Desktop**。Claude Desktop 已经成为一个强大的应用，它的优秀程度简直令人难以置信。

## Claude Co-work：安全的环境与强大的连接

Claude Desktop 应用的起飞始于他们推出 **Claude Co-work**，这是 Claude Code 在工作领域的延伸。他们创建了一个平台，介于 Claude Code 和 Chat 之间。

### 安全的环境：沙盒机制

Claude Code 可以控制你的电脑，在目录（即你电脑上的文件）中工作，而 Chat 则主要在应用内工作，不触及你的电脑。Claude Co-work 则创建了自己的独立环境，可以把它想象成电脑上一个隔离的“小文件夹”，它只会触及这个文件夹。

当你创建新项目时，可以选择一个现有文件夹。Co-work 只会在该文件夹内操作，不会做任何超出范围的事情。你可以通过添加特定文件来引用电脑上其他文件夹的内容，但总体上它是隔离的，因此**更加安全**。

### 强大的连接器

Claude Co-work 让连接新工具变得非常简单。你可以通过“连接器”功能，轻松连接 Figma、Gmail、Canva、Excel 等众多应用。例如，连接 Figma 后，它就能读取或生成 Figma 上的图表；连接 Gmail 后，它就能获取你的邮件资料、列出草稿等。几分钟内，你就可以连接所有对你业务有用的工具。

### 实战演示

在 Co-work 中，你可以指定一个项目文件夹，并给出任务指令，例如“请查看我的 Riley 内容数据库，了解我正在制作的视频和我的声音，然后为一段解释 Claude Co-work 及其与 Claude Code 区别的视频写一个开头”。Co-work 会快速找到相关文件夹，尝试理解你的声音风格，抓取更多脚本来模仿你的语气，并查看当前进度，然后执行任务。

它生成的文案听起来会很像你本人。你还可以要求它“把这个做成一个 PDF”，它会直接使用一个“技能”来创建 PDF，并保存在你指定的项目文件夹中。这体现了“技能”的概念：技能本质上是一个 Markdown 文件，解释了如何执行特定任务。许多人认为，未来大多数软件将被技能取代，因为任何你需要在外部软件上做的事情，都可以由一个带有特定指令集的智能体（即技能）直接在 Co-work 这样的工具中完成。

## 近期所有Claude新功能速览（2024年底 - 2026年）

接下来，我将快速浏览过去几个月里 Claude 发布的所有新功能。

### Claude Co-work 相关更新

- **Slack MCP 对所有计划开放**：例如，当你休假回来，可以让它总结你离开期间 Slack 上发生的一切，并起草一条消息发到公共频道。它会在 Co-work 中起草消息，并提供 UI 界面让你查看和发送。
- **更易连接 Gmail**。
- **Claude Opus 4.6 模型升级**：可以更长时间地维持智能体任务。
- **新版 Opus 模型**：速度提升 2.5 倍，但成本是原来的 4 倍。这对于大型团队来说虽然昂贵，但效率的提升让雇主觉得物超所值。
- **Claude 集成至 PowerPoint**：可以直接在 PowerPoint 中生成完整的幻灯片，对企业用户非常实用。
- **Claude Co-work 登陆 Windows 平台**（2月10日）。
- **Claude Code 的“危险跳过权限”模式**：可以在桌面应用中开启“操作模式”，让 Claude Code 无需询问即可采取行动，拥有最高自由度（可以编辑任何文件、安装程序等）。
- **免费计划功能升级**：免费用户可以访问更多有趣功能。
- **Co-work 文件夹指令**：可以在特定文件夹内创建一个文件，为该文件夹提供专属的指令，Claude 会遵守这些指令。
- **Claude Code 桌面版支持插件**：插件是一组指令文件，可以赋予 Claude 访问特定资源（如内部数据库、Slack）的能力，并可以轻松在团队成员间共享。
- **Claude Sonnet 4.6 发布**（2月17日）：这是目前最强大的 Sonnet 模型，速度更快、成本更低。它拥有 100 万 token 的上下文窗口（测试版）。
- **Figma MCP 发布**：可以更轻松地将 Claude 连接到 Figma，方便设计师将设计稿转换为代码。
- **Co-work 插件的管理功能**：管理员可以创建插件（包含技能、智能体、连接器和命令），并与团队成员共享。
- **Co-work 支持计划任务**：你可以使用 `/schedule` 命令创建定时任务，例如“每天早上总结我的邮件，并草拟重要邮件的回复”。
- **Claude for Excel 和 Claude for PowerPoint 同步**：可以同时使用多个扩展，无缝编辑表格和幻灯片。
- **增强的图表生成能力**：生成的图表不仅视觉效果好，而且是可点击、可交互的，具备出色的视觉推理能力。
- **所有模型默认支持 100 万上下文窗口**：可以处理更大规模的代码库或多年的笔记记录。
- **Dispatch 功能（研究预览）**：一个持续运行的对话，你可以从手机上发消息给它，稍后回到电脑前继续完成工作。这是对 OpenClaw 核心功能的回应，让用户可以用手机远程指挥电脑。
- **Channels 功能**：通过选择 MCP（如 Telegram 和 Discord）来控制你的 Claude Code 会话。你可以通过 Telegram 向你的 Claude Code 会话发送消息，延续了“手机远程控制”的主题。
- **项目管理功能**（3月20日）：可以从一个特定文件夹开始工作，所有文件都创建在该项目内。你可以创建多个 Co-work 会话，它们都工作在同一个项目下，共享上下文，随着时间的推移积累文件和工件。
- **计算机控制功能**（发布前一天）：允许 Claude 控制你的鼠标、键盘和屏幕，使其能够使用任何应用。这是 AI 智能体的终极形态。虽然目前还不够完美，但到年底，AI 在使用电脑方面的能力可能会超越普通人。这也导致一些公司通过录制员工工作流程来训练这些计算机使用智能体，以便未来用 AI 取代他们，这引发了一些争议。
### Claude Code 相关更新

- **会话分享功能**：可以将会话历史设为公开，并分享链接。这对于招聘开发者尤其有用，通过观察他们与 AI 的交互方式，可以深入了解其问题解决能力。
- **/insights 功能**：输入 `/insights`，它会生成一个关于你所有项目的长摘要，显示你提交的代码行数、使用习惯、常用工具等。
- **智能体团队（研究预览）**：不再是一个智能体按顺序完成任务，而是一个主导智能体将任务委派给多个“团队成员”智能体，它们并行工作，相互协调，进行调研、调试和构建。这是多智能体协同的大趋势。
- **Rewind 功能**：输入 `/re` 或按两次 ESC 键，可以回退到对话的某个节点，并让 Claude 总结被回退部分的内容，用于探索不同路径。
- **内置安全扫描**：可以直接在 Claude Code 中对代码库进行漏洞扫描。这对于氛围编程者来说非常实用，可以一键检查应用的安全性。
- **远程控制终端会话**（2月底）：在终端运行的 Claude Code 可以通过生成的链接，用手机进行远程控制。
- **自动记忆功能**：Claude 现在可以跨会话记住它学到的东西，包括项目上下文、调试模式、首选方法等。这让智能体在使用过程中不断变得更好。
- **终端版语音模式**：输入 `/voice`，然后按住空格键即可说话，例如“重构身份验证中间件”，它会将语音转换为指令。
- **计划任务**：可以安排定时任务，例如每天进行代码审查。
- **/btw 功能**：当智能体在执行长时间任务时，你可以输入 `/btw` 并提问（例如“这个重试逻辑是做什么的？”），它会在旁边回答你，而不会干扰主任务。
## 总结与展望

总的来说，Claude 的新功能主要集中在三个方面：

- **Co-work 更新**：更多的连接、更多的沟通渠道，让你能更容易地将 Claude 与你现有的所有工具集成。你可以同时处理 PowerPoint 和 Excel，当更新表格时，它能同步更新幻灯片。它正变得越来越像一个同事。
- **Claude Code 更新**：更强的协作性（会话分享）、多智能体协同、更好的记忆能力等。
- **集成更新**：所有功能正融合成一个通用的智能体工具。
与 OpenAI 分散开发不同产品（ChatGPT、Codex、Sora、Atlas）的策略不同，Anthropic 选择专注于打造一个“超级工具”。这使得许多人正在转向 Claude。它目前是世界上最受欢迎的软件之一。
