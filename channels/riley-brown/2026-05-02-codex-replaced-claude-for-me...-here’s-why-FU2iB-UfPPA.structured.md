# 2026年AI代理大爆发：从超级应用到个人代理，全面解析

**本文基于Riley Brown与Ross Mike的深度对话整理，涵盖OpenClaw、Claude、Codex、Cursor等前沿AI工具的最新进展与未来趋势。**

## 引言：2026年，AI代理的元年

2026年无疑是AI代理的一年。所有代理公司都在不断发布更新，让人很难跟上节奏。今天，我们将全面梳理这些进展。

在过去的四个月里，**OpenClaw** 已成为历史上增长最快的开源软件，甚至导致了Mac Mini的缺货。**Anthropic** 推出了超过50项新功能，全力打造他们的新桌面应用。**OpenAI** 发布了 **Codex**，它正迅速成为我进行“氛围编程”（vibe coding）和知识工作的首选平台。而如今，**SpaceX** 似乎正在收购 **Cursor**，以与所有这些公司竞争。

今天，我和我的朋友、创作者兼软件工程师 **Ross Mike** 一起，探讨所有这些进步及其意义。Ross Mike 是YouTube上最好的教育者之一，对AI和软件工程的内部动态有着独到的见解。

在本期视频中，我们将首先讨论什么是“超级应用”（super app），以及为什么所有大型实验室都在构建同样的东西。我们将探讨 Anthropic 如何开始在对 OpenAI 的竞争中失去领先地位。我们会谈到 **Claude Code**、**Codex** 和 **OpenClaw**，讨论**计算机使用**（computer use）和**浏览器使用**（browser use），还会聊到**代理支付**（agentic payments）。最后，我们将以AI图像模型和知识工作的未来作为结尾。

## 一、2026年Q1回顾：Anthropic的辉煌与OpenAI的反击

> **Riley Brown (R):** 今天我们来做一点回顾。2026年已经过去了四个月，感觉像是过了十年。第一季度（1月、2月、3月）属于Anthropic。从他们大概1月10日推出功能开始，直到3月底，几乎每天都在发布新功能。这期间还有OpenClaw的热潮，我Instagram上的每个人都在买Mac Mini。现在都很难买到。
>

> **Ross Mike (M):** 在加拿大，多伦多要等三个月，我都不想买了。
R：确实，我们经历了OpenClaw热潮，还有像Perplexity Computer等其他公司。现在，我玩得最开心的工具是刚发布的**Codex**。一个共同的主题是：**AI代理在操作电脑方面变得越来越强**——无论是通过Codex的计算机使用，还是它们操控文件和控制电脑的能力。

R：你对2026年至今有哪些看法？亮点是什么？

M：对我来说，**Opus 4.5** 是一个转折点。这个模型刚发布时效果并不立刻显现，但大约一两周后，我们都集体意识到：“哇，这太棒了。”

R：我的“热辣观点”是，转折点是Andre Karpathy的那条推文：“**我作为一名程序员，从未感到如此落后。**” 所有对AI持怀疑态度的开发者试用了之后，都不得不接受它。Opus 4.5大概在去年11月或12月发布。当时我试了一下，并没有完全理解。但后来在Cursor中使用它时，我发现：这个模型懂我，速度快，感觉太棒了。

M：从那以后，Anthropic 确实赢得了第一季度。他们接连不断发布更新。当我看到Instagram上有人说“每个人都在用ChatGPT，但能让你的业务增长10倍的秘密工具是Claude”时，我就知道他们赢了。朋友们（比如当老师的）也开始谈论Claude Co-work。

R：但过去一个月，OpenAI已经迎头赶上了。

## 二、超级应用（Super App）之争：从分散到聚焦

R：为了让讨论更具体，我认为在2月底、3月初，OpenAI 宣布重新聚焦公司，将所有精力投入到 **Codex** 或所谓的“超级应用”上。他们将 Atlas 和其他所有成果（如 GPT-4o 图像生成）整合在一起，并砍掉了 Sora。这是一个很好的决定。

之前，OpenAI 被认为是“不聚焦”的公司，而 Anthropic 因为专注于编码模型（没有图像、视频模型）而显得非常聚焦。他们有人们喜爱的终端界面。但后来，Anthropic 也开始变得“不聚焦”，而 OpenAI 则开始聚焦。

M：看看现在的情况：有 **Claude Code** 和 **Co-work**，它们是相似的技术但需要分开学习。Claude Code 可以用远程（remote）功能，但必须输入 `/remote` 来设置，而且不能在桌面应用使用，必须在网页端。还有 routines，而 Co-work 里有 schedule……它们其实是同一件事，却被分散了。

R：我跟你说过，我想要一个像 Claude Code 那样能构建** artifacts** 的工具，能切换任务、创建文档、做演示文稿、做 Excel。我知道你很喜欢 Excel。我希望在一个平台里完成所有事。然后 OpenAI 就发布了这样的东西。没有 automations、schedules、dispatch 或 remote control，他们很快会发布一个 Codex 应用。

M：我100%同意。我注意到 AI 实验室有两种发布：一种是面向用户的，另一种是制造头条新闻以便融资的。Anthropic 展示的迭代速度带来了大量关注，从 PR 角度看是净收益。对于圈外人来说，看起来 Anthropic 正在赢——而且不是小幅领先，而是**数量级上的领先**。我认识的每个准技术人员都认为 Anthropic 赢了。但从产品角度看，OpenAI 正在清理产品、集中焦点。

R：一个关键认识是：**一个优秀的编码模型就是一个优秀的通用模型**。早期有人认为会有特定领域的模型，但对于一般知识工作（创建 Excel、抓取数据、做数学），优秀的编码模型恰好也是优秀的通用知识模型。Anthropic 早期赢在构建了优秀的编码模型，然后发现这些模型能做电子表格、分析数据等等。因为代码文件本质上就是计算机上的文件。

现在每个人都在努力构建最好的编码模型，然后用 **GUI（图形用户界面）** 包装起来，让用户可以用模型做不同的事情。模型都很棒，但**如何使用模型、使用什么工具**变得至关重要。而 **Codex 在这个领域100%处于领先地位**。

## 三、OpenClaw：开源代理的革命性范式

R：**OpenClaw** 最初让我持怀疑态度，觉得有很多骗局。但我把它看作一个**在计算机上运行、拥有完全控制权的AI代理**。它有一个“个性文件”，随着使用会不断更新。理论上，你用得越多，它越了解你，能为你做得越多。你可以用自然语言创建技能（skills）。这创造了一个全新的范式。

M：对我来说，OpenClaw 最大的亮点是**心跳机制**（heartbeat）。每15分钟，代理会被 ping 一次，查看所有内容，然后可以主动给你发消息。这模拟了人类的思考模式。如果底层的编码模型越来越好，代理拥有记忆和上下文，那对无论是你的业务还是个人生活，都将是净收益。

M：我是怎么使用 OpenClaw 的？作为创作者，我每天收到很多赞助邮件。我的代理有权访问我的赞助邮箱。每当邮件到达，它会做三件事：

1. 检查是否是诈骗（例如 `@gmail.com` 直接忽略）。
1. 如果是公司邮箱，研究这家公司（在 Google 和 Reddit 上搜索“公司名 scam”）。
1. 如果合法，进行预研（融资金额、粉丝数等）。
每天早上，我会收到一份“早晨报告”。我只负责查看报告。第二个工作流：一旦我批准了要联系的品牌，代理会发送第一封邮件询问报价，然后检查回复和谈判意愿。直到这时我才接手。这为我每天节省了1.5到2小时，聚合到每周、每月、每年，就是大量的时间和家庭时光。我对这种**知识工作**的自动化比对编码改进更兴奋。

M：有趣的是，Anthropic 正试图添加 OpenClaw 的功能（如 schedule、dispatch、remote control），而 OpenAI 则**半收购了 OpenClaw**（雇佣了其创始人 Peter）。这证实了每个人都会去构建类似的代理。

## 四、代理工具的使用技巧：如何有效提示和管理上下文

R：既然大家都在使用编码模型，那么有哪些跨工具的通用要点？你提到过**提示**（prompting）非常重要。

M：核心是理解：**LLM 本质上只是一个“下一个词元”的预测器**。它并不真正“思考”，只是预测最可能的下一个答案。所以，你使用的**英语（语言）和词汇**比想象中更强大。模型无法像人类一样理解语境，但它擅长预测。这就是为什么像 **Whisper Flow** 这样的工具很好——大多数人说话比打字更好。能够清晰地口述你想要模型做什么，是预期价值最高的行为之一。

另外，**行业知识**很重要。你无法表达自己不懂的东西。这就是为什么营销人员迷恋“氛围编程”工具，而很多“垃圾”工具被推出——非行业专家既不会清晰表达需求，也无法验证输出质量。**通才型人才**（对营销、编码、设计都有六七分了解）现在表现极佳，尤其是那些有**高能动性**（high agency）、不认为技能之间有壁垒的人。

M：很多人给了模型**不必要的上下文**。例如，不需要告诉它“这是一个 React 代码库”，它读文件就会知道。**保持你的代理栈精简**，技能精简，并且技能应该是**特定于你和公司的领域专长和工作流**。我曾经让 OpenClaw 生成一份频道和业务报告，结果一团糟，因为我没解释报告长什么样。后来我创建一个技能，清晰说明如何计算 CTR、赞助视频等，一次性就成功了。

如果你给模型垃圾，它就会输出垃圾。如果你给好的输入，它输出好的概率就大得多。

R：对于知识工作，我甚至不把重点放在提示上，而是**先得到一个好的输出，然后逆向工程出技能**。比如，我的“YouTube Researcher”技能使用 SERP API 搜索 YouTube，再用另一个 API 抓取完整字幕。当我需要一个完美的钩子（hook）时，我会让代理分析所有教育类 YouTuber，找出超出其平均观看量的视频，提取最佳的钩子风格，应用到我的视频上。这更像是**上下文管理**，可以发送到正确的 API 或给出优秀的示例。

关键是要**为生产力而规模化**（scale for productivity）。这种工作流非常个人化。很多人从别人那里安装50-100个技能然后抱怨不工作，因为**代理应该是个人化的**。学习创建一个技能，比得到100个现成技能的列表更有价值。

## 五、浏览器使用与计算机使用的加速演进

R：代理们开始使用计算机了。Yann LeCun 前几天发推说：“享受这段你还能看清浏览器使用过程的短暂时期吧，因为很快它会快得让我们无法跟上。” 你用过 **browser-use** 吗？

M：我用它构建了一个棋盘。在一个提示下，它构建了一个国际象棋游戏并让两个 AI 对弈。第一次它四步就把自己将死了，我让它玩久一点，它就点击重启，然后迅速对弈，速度比我任何对手都快。它们甚至能**围绕自己的“ harness ”编写代码**，构建持久化的工具，就像机器人可以构建自己的扩展。

R：计算机使用需要训练数据。它的工作原理是：截屏，喂给 LLM，LLM 说“点击 DOM 元素 D4”，然后执行并再次截图。目前还很慢，但会越来越快。Opus 新模型分析图像的分辨率已经提高到普通 MacBook 的尺寸，这就是因为他们押注计算机使用。

M：Meta 曾试图收购 Manis，还想收购一家叫 **Versep** 的公司（创始人 Matt 后来被 Zuck 以2亿美元收购，公司最后去了 Anthropic）。可见 Meta 对这个领域也很有兴趣。不过，Manis 虽然是第一个“超级应用”，但它没有捕获到巨大的上升空间。Codex 和最终整合好的 Claude 的成长高度，是无法与 Manis 当前水平相比的。你不一定要做第一，但要做最好。

## 六、Cursor 的困境与 SpaceX 的收购

R：Cursor 是创新者。很多人不想从 Cursor 转用 Claude 或 OpenAI，因为这些公司有全球最好的模型并且提供高额补贴。Cursor 最弱的环节是**它们不是模型提供商**，没有补贴价格的优势。例如，Claude Max 每月200美元的订阅，实际给你提供了价值5000美元的计算资源。

M：Cursor 的 $200 Ultra 套餐无法与 Codex 或 Claude 的 $200 套餐相比，除非你使用其 composer 模型（但模型本身无法比拟）。他们在 K2.5 模型上做构建很聪明，但无法在模型领域获胜。所以他们与 SpaceX 达成了**收购协议**（10亿美元退出的条款）。这很合理：xAI 应该专注于模型，Cursor 则在 GUI 上领先（他们最早做内联编辑、agents 标签）。他们只是无法在模型上获胜。

R：Cursor 的应用内浏览器非常出色，还能持久化。还有沙盒功能可以测试应用并发送视频。他们在接近构建超级应用上可能比 Claude 更近，但最终，驱动超级应用的是**编码模型**。

M：而且，Anthropic 的产品做得不好。你的线程可能在刷新后消失，而 Codex 和 Cursor 更流畅。

## 七、Claude Design 与 Codex 设计模式的对比

R：我试用了 **Claude Design**。它是最清新的一个，因为它与众不同。它不是为设计师准备的，而是为那些有品味但不懂设计的人准备的。你可以上传图片，分享 Figma 信息，它会给你建议。但问题是：**一个优秀的设计工具本质上就是一个编码工具**。所有这些设计模式应该集成在你的聊天中，而不是让用户去一个叫 Claude Design 的网页应用。而且它只有网页版，没有桌面应用。如何导出？复制后拖拽到 Claude Code——它们甚至没有连接。

M：我希望 Anthropic 在重建整个应用，否则真的太失败了。我需要去浏览器用 Claude Design，复制到 Claude Code，如果我想用 Co-work，还得再开一个……甚至连共享会话都没有，需要重新解释。

## 八、Google 的困境：组织问题阻碍了超级应用

R：Google 在哪里？他们似乎在酝酿着什么，但感觉一直在隐藏。

M：Google 有无限的钱和优秀的人才。问题是**团队之间不允许充分协作，信息非常孤立**。Google 内部的人会和公众同时知道新模型的发布。如果 Gemini 团队能像一个创业公司那样在自己的“盒子”里运作，允许犯错和快速行动就好了。Gemini 在知识上是目前最聪明的模型，但**工具调用能力不强**。我用它做计算机使用，它甚至无法正确点击。回想 Bard 有多糟糕，再到后来 Gemini 的巨大飞跃，他们有再次飞跃的能力，但组织问题是巨大的。

M：想想看：要做 Google 搜索你去 Gemini，要学习你去 Notebook LM，要做快速氛围编程你去 Google AI Studio，要做严肃编程你去 Anti-Gravity，要做设计你去 Stitch……它们有这么多产品界面。为什么不做 OpenAI 做的事？把所有团队整合成一个超级应用。如果我是 Google，我会把每个团队最优秀的人放到一起，把 **Google AI Studio 或 Anti-Gravity 变成超级应用**。他们应该让 Gemini 像一个独立的初创公司那样运作。

## 九、代理支付（Agentic Payments）与 commerce 的未来

M：一个重要的趋势是：公司开始意识到**代理也是客户**。Stripe 最近宣布代理可以：使用你的 Stripe 信用卡/借记卡发卡（无需向代理提供敏感信息），代理可以进入支付环节并询问你“我应该支付吗？”。

R：我构建了一个迷你版本给 Uber Eats 用。使用虚拟卡 API 可行，但有些网站不接受虚拟卡。Stripe 为代理发行的卡是绑定到你业务的合法卡，有地址。你可以设置限额（比如每周2000美元）。**代理 commerce** 将会是大趋势。

M：还有像 **X42** 这样的协议（由 Coinbase 支持），允许代理通过加密发送支付。Stripe 有自己的实现。目前问题是协议碎片化，但未来一年将是**代理执行支付**的关键年。比如公司午餐，为什么不让代理处理？

R：我的联合创始人就用一个“购物机器人”订购蓝莓和酸奶送到办公室，通过亚马逊 API，还没给卡。但 Stripe 可能是让这成为现实的公司。还有很多公司只做代理支付，比如 Crossmint、Natural Pay、agentcard.sh 等。

## 十、AI 图像与视频模型的爆发：双刃剑

R：下一个“可怕”的进展是**图像和视频模型的飞速进步**。ChatGPT-4o 图像生成 2 代非常强大。我能让 GPT 生成一张我和 LeBron James 的合影，我的家人一定会相信。你们可能也有“安全密码”——因为图像生成、视频模型（如 Seed Dance）和**语音克隆**都在变好，这意味着更大的诈骗风险，尤其是在 Facebook 上的老年人群体。这很可怕。

M：对我来说，Midjourney 在 2022 年中期是我的 AI 入口。奇怪的是，随着图像生成变好，我的兴趣反而递减。早期的“不完美”有种魔力。现在它们太好了，我反而麻木了。但这意味着，**Photoshop 被模型取代了**。只要你能清晰表达修改意愿（又回到行业知识和表达能力），模型就能做到。

R（展示一张图片）：看这个——我用一个提示生成的：“让我看起来像是为猛龙队签约了 NBA”，放上我的照片。看细节：汗水、反光……太不可思议了。这将发生在所有知识工作上：演示文稿、报告……不是 OpenAI 发布一个设计界面，而是 **模型本身能够做到，甚至不需要界面**。

## 十一、AI 时代的内容创作与个人机遇

R：对于内容创作者来说，所有重复性的、流程化的事情都会被 AI 接管。视频教程已经“死了”——没人再看逐行打字的编码教程（除了那个叫 Code with Anton 的家伙，他是大神）。胜出的教程是那些**有伟大个性的创作者**（像 Theo、Primeagen），他们谈论“Anthropic 烂透了，OpenAI 这个那个”，人们喜欢的是这种关系和讨论。

M：我告诉那些害怕的开发者：**打开你的电脑，学一点相机基础**（不需要买4000美元的相机），买个麦克风，然后开启直播，拍自己，玩得开心。人们不是在看教程，他们想找到**灵感**，想了解你能做什么，然后自己去捣鼓。AI 让开始变得容易，但**社区、聊天和信息的需求仍然存在**。诚实、真实地分享的人会赢。

## 十二、最终思考：这是最好的时代，动起来
