## 一、什么是编程智能体？

你可能听说过 Lovable，不过这个视频并不是关于 Lovable 的。我只是想快速说明一个观点。现在很多人都在讨论“氛围编程”（Vibe Coding）。你可以使用像 Lovable、Replit 这样的工具来进行“氛围编程”。这些都是你可以用来构建应用程序的编程智能体。

当我在 Lovable 中输入“现在制作一个关于披萨的落地页”时，它所做的就是启动一个沙箱（基本上就是一台虚拟计算机），并在其中运行一个 AI 智能体（或 AI 编程智能体）。然后，它会在一个目录或代码库中生成一堆文件，这些文件最终构成一个应用程序。这就是 AI 编程智能体在构建应用程序时的实际工作过程。

好，现在它完成了。Lovable 的编程智能体接受我的提示“制作一个关于披萨的落地页”，生成了所有文件。这个编程智能体决定了需要生成哪些文件，生成了它们，然后可能检查了这些文件，最后在某个虚拟计算机上运行程序，这样我就可以在浏览器中看到结果（因为这是在网页浏览器内）。这是一个专门用于创建简单应用程序的智能体示例，一个非常狭窄的智能体体验，专门用于使用特定技术栈创建特定类型的应用程序。

**我们也可以在自己的电脑上做同样的事情，不是吗？** 我们完全可以在自己的电脑上完成相同的操作。如果我们打开访达（Finder），直接进入“下载”文件夹，新建一个文件夹，比如命名为“using-claude-code”。我们甚至不需要任何像 Cursor 或 Lovable 这样的 IDE。我可以在该文件夹下打开一个终端窗口，然后直接运行 `claude` 命令。现在我们就在这个名为“using-claude-code”的空文件夹中运行着 Claude Code。我可以直接说：“请为我的披萨店‘Riley's Pizza Shop’创建一个漂亮又令人惊叹的落地页。采用新粗野主义（neo brutalism）风格，并融合友好的披萨氛围。”就像 Lovable 一样，我们可以运行这个命令，Claude Code 会在这个空文件夹中创建应用程序。

看！它刚刚生成了一个 `index.html` 文件。就这样，它创建了一个文件，就像 Lovable 在某个虚拟机的文件夹中创建各种文件一样。Lovable 使用的是虚拟计算机，而我们在自己的电脑上做同样的事情。Claude Code 所做的和 Lovable 完全相同。

**我想强调的重点是：你完全可以在自己的电脑上使用这些编码模型来处理各种文件，不仅仅是代码文件。** 这里生成的是 HTML 文件和 CSS 文件，但 Claude Code 可以创建**任何类型**的文件。现在它完成了。如果我们复制这里的链接（`localhost:80`）并在浏览器中粘贴，就可以在本地运行这个页面。而 Lovable 只是创建了一个虚拟机并在那台计算机上运行，然后向你展示它在浏览器中创建的内容。现在我们完全在本地运行，没有使用任何中间软件。我们用 AI 编程智能体在本地电脑上生成了代码文件，它使用 Anthropic 服务器的 AI 模型来创建这些文件（这是需要付费的部分），但我们没有使用除 Claude 生成文件之外的任何其他软件。我们在自己的 MacBook Pro 上运行它，这就是它创建的网站。

以我的观点，我测试过所有这些编程智能体，**原生的 Claude Code 是世界上最好的**。Replit 和 Lovable 有它们自己的编程智能体，但开箱即用的 Claude Code 比它们都好。

## 二、将 Claude Code 用作通用智能体

现在我想解释的是，如何将 Claude Code 用作**通用智能体**。

这是 Obsidian，一个基础的笔记应用。这里的每个文件不再是 HTML、JavaScript 或 CSS 文件（那种最终构成可运行应用程序的文件），而是 Markdown 文件。我使用通用智能体的方式是，我实际上就在我的笔记所在的同一位置运行 Claude Code。这是我的智能体工作区。我把它看作一个笔记本，我（Riley Brown）把它用作个人笔记本，这样我就能成为 Vibe Code 应用更好的 CMO 和联合创始人。

我在里面记录我所有的目标。你可能会问：我为什么要把这些写下来？我自己知道我的目标啊。因为我不是自己单独使用这个笔记本，我是和我的通用智能体——Claude Code 一起使用它。

我可以在这里点击文件，或者右键点击并选择“在访达中显示”。这里是我的“Riley 内容创作”文件夹，基本上就是这个文件的所在地。你会看到这里有很多不同的文件，它们只是显示在我电脑上的文件集合。和我之前在新文件夹中运行 Claude Code 一样，我也可以在这个文件夹中运行 Claude Code。所以我可以“在访达中显示”，然后右键点击，选择“服务”->“在文件夹中新建终端”。现在可以看到终端路径已经定位到这个文件夹，也就是所有这些笔记所在的文件夹。现在我们可以运行 `claude` 命令。这样，Claude Code 就在我存放所有笔记的文件夹中运行了。

因此，我可以将它与 Claude Code 结合使用。比如，我有一个“任务”笔记，里面记录着我的待办事项。其中一个待办事项是“向我的观众解释 Claude Skills 并制作一个视频”。我把这个事项拖拽到当前笔记中，并告诉 Claude Code：“做我队列里的任务。不要创建或编辑任何其他文件。只需制定一个计划并执行。”然后我只需对 Claude Code 说：“执行我队列里的任务。”

因为 Claude Code 要做的第一件事就是分析 README 文件，它会了解整个笔记的组织方式，知道哪里是任务队列，然后直接去处理队列。由于 Claude Code 是一个非常强大的智能体，擅长查看代码库（在这里就是一堆我的笔记），它会非常聪明地找到正确的位置，找到这个页面，然后执行我想要它做的事情。我们只需等待 Claude Code 完成。

请注意，我们从来没有告诉它要看这个特定文件，它并不知道我打开了这个文件。它只是找到了队列里的内容。你会看到它正在“检查笔记本以获取技能创意”，找到了正确的文件，然后开始创建相关内容。允许它编辑这个会话……看！它刚刚在这个文档中添加了所有这些内容。它告诉我们什么是技能，从 Anthropic 的技能代码库中找到了相关信息，然后告诉我们如何创建技能：在 `.claude` 文件中，你可以通过创建一个以技能名称命名的文件夹来创建技能，在该文件夹内需要有一个 `claude.md` 或 `skill.md` 文件，还可以在该文件夹中放置可选的资源文件（如资产、参考资料等）。稍后我会详细说明。

我想说明的是，就像 Lovable 是使用智能体的一个编码特定示例，这里是一个使用智能体的笔记特定示例。我保证我们正在逐步深入，最终创建 Claude Skills。我只是想先说明这个观点：先有像 Lovable 这样能创建应用的领域特定智能体，然后我展示了一个笔记示例，说明如何将 Claude Code 用作通用智能体。现在，我们将在 Cursor 这个更开放的环境中使用它，我们可以做任何想做的事情。

## 三、在 Cursor 中创建 Claude Skills

我喜欢使用 Cursor，因为在我看来，它是快速使用 Claude Code 的最佳工具。这有点讽刺，但我们可以直接说“claude-code-skills”来创建一个新项目，然后它会在一个 IDE 中打开这个目录。Cursor 就像 VS Code 一样的 IDE，你可以查看所有文件。

我的“氛围编程”方式是：进入项目后，首先按 `Command+J` 打开终端，输入 `claude`，然后按旁边的按钮打开侧边栏。这样，Claude Code 就在右侧控制着这个“claude-code-skills”目录。

接下来，我会说：“请查看这个目录（或这个代码库）”，然后粘贴 Anthropic Skills 的 GitHub 链接。“请按照正确格式创建一个简单的技能，这样你在需要时就会使用这个技能。” 现在它正在创建一个示例技能。它会创建一个 `.claude` 文件。正如你看到的，它创建了这个 `.claude` 文件夹，稍后还会在其中创建一个 `skills` 文件夹。好了，它创建了一个名为 `.claude` 的文件夹，然后在其中创建了 `skills` 文件夹，里面还有其他子文件夹。它创建了一个 `summarize` 文件夹，里面有一个 `skill.md` 文件。这就是创建一个技能所需的一切：只需将其放在 `.claude` 文件夹中，因为 Claude Code 知道在 `.claude` 文件夹中查找某些内容，比如子智能体（sub-agents）和现在的技能（我们这次不讨论子智能体，只讨论技能）。

我们创建了 `skills` 文件夹，在这个文件夹里，我们看到了这个 `skill.md` 文件。**每个 ****`skill.md`**** 文件的格式都应包含 ****`name`**** 和 ****`description`****。** 它应该有一个名称和描述。你可以看到这里创建的这个技能叫“品牌声音”（brand voice），是第一个技能。名称只是一个人性化的标题，可以是任何内容。但关键部分是**描述**，因为它告诉 Claude Code 何时使用这个技能。

我们现在已经达到了一个阶段：仅用一个 Markdown 文件就能让智能体执行任务。但你需要确保格式正确，以便 Claude Code 知道何时遵循你的指令。因为在我看来，技能本质上就是**智能体的标准操作程序（SOP）**。它们是针对特定场景的指令手册。如果你提供高质量的描述，它就会知道在需要执行特定任务时该使用哪本说明书（或 SOP）。

我不太想用这个总结技能，我想创建一个与我更相关的技能。我想展示一下我一直在做的事情：我一直给我的 Twitter 帖子添加注释，因为我可以把这些注释交给 Claude，放进 Claude Skills 里，这样智能体就能了解我的发帖风格。例如，我会在图片上直接标注“此帖子附有短视频”，“资深氛围编码者（senior vibe coder）是流量诱饵（ragebait）并且有效，但我不常这样做”等等。这有助于我注释图片。我保存这些图片，然后把它们全部拖进 Cursor 项目里。

现在我要创建一个新技能。我将这些图片拖到一个新文件夹中，然后对 Claude Code 说：“我想让你替换掉这个 `summarize` 技能。把这个 `summarize` 技能整个替换成一个 **Twitter 发帖（X Post）技能**。这个技能应该能让智能体以我的风格创建推文，并使用这个新技能文件夹中的图片作为参考。请创建这个技能。” 这样，我们基本上是在告诉 Claude Code 分析这些图片，寻找我的语言模式。另外，我可以说明一下：图片中的红色文字是我的注释，不是帖子内容。这些是我对你（智能体）做的注释，解释我为什么做某些事情。在构建这个技能时请考虑这些因素。所以，我注释了我为什么这样做，这为 Claude Code 提供了额外的上下文，它会把所有信息打包成一个技能。理论上，以后只要在这个代码库中向 Claude Code 请求 Twitter 帖子，它就应该能按照我的风格生成，因为它已经有了相关的上下文。

好了，它创建了这个 `skill.md` 文件。它分析了所有这些图片，把它们转换成 Markdown 格式，并封装成了一个技能。它没有做的是……我忘了指定：那只是一个临时文件夹，请移除那个文件夹，但把这些图片添加到该技能文件夹中作为参考，因为临时文件夹不需要存在，图片可以直接放进技能文件夹。现在，我们有两个技能了。每个技能都需要一个 `claude.md` 文件。它刚刚创建了这个技能，现在有了可以直接使用的参考图片。我们可以把这些图片全部选中，移动到技能文件夹里。所以，我们现在有 `references`（参考图片）和 `skill.md` 文件。你需要这个 `skill.md` 文件。

接下来，我让 Claude Code 在根目录（最初打开的“claude-code-skills”文件夹）创建一个 Markdown 文件，基于我上一个关于 Claude Skills 的视频，生成 10 条推文点子。并告诉它：“请使用 `xpost` 技能来生成这些点子。” 稍等，它完成了。你会看到这里出现一个绿点，显示“expost”（即 `xpost` 技能），意味着它正在使用那个技能。我们允许它创建这些文件。看，生成了“Twitter 创意”文件。Claude Code 变成了一个通用智能体，而不是一个聊天机器人，而是一个执行你工作流程的智能体。

它生成的推文点子包括：

- “Anthropic 刚刚发布了 Skills：可重复使用的提示模板，可通过斜杠命令调用。营销团队注意了。”
- “哦天，这太不可思议了。我把 Claude 变成了我的个人营销团队。社交媒体文案随叫随到，几秒钟生成邮件序列，品牌声音永久锁定。每次都是零提示。Claude Skills：一个 Markdown 文件，无限工作流。”
写得非常好，它用我的风格写出了这些，用到了刚刚创建的技能。

## 四、增强技能：集成 Typefully API

但这还可以变得更酷。我们打开 Typefully 网站，进入设置，找到 API 部分，新建一个 API 密钥（测试用，视频结束后会删除）。然后查看 Typefully 的 API 文档，里面详细说明了如何使用这个 API 密钥。我的目标是：不只是让它想出点子，而是希望它直接在这个工具中草稿推文，这样我就可以直接发布到社交媒体。理论上，我可以让智能体生成推文，并自动在 Typefully 中创建草稿，因为这就是该平台支持的功能。我们可以让它直接以草稿形式发布。

所以，我告诉 Claude Code：“我想给我们的 `xpost` 技能添加功能。我使用的软件是 Typefully，我希望你不再把推文点子放进文档，而是直接将它们作为草稿调度到 Typefully。我给你这些 API 文档链接，这是我的 API 密钥（放在 `key.txt` 文件中，我不想暴露它）。请使用 Claude Skills 添加到这个技能中，并确保这个技能能将其作为草稿调度到我的 Typefully 账号，不要直接发布或正式调度。”

现在等待它修改 `xpost` 技能。完成后，我们可以看到它已经识别到账户信息。然后我测试一下：“请测试这个技能，把我们之前生成的列表中的一条推文草稿到 Typefully。然后我们检查一下。” 它完成了。我们回到 Typefully，看到在 “Vibe Coding Explained” 账号下确实创建了一个草稿。然后我又让它在 “Riley Brown” 账号下草稿一条“你好，这是测试”的推文。完成后，回到 Typefully，刷新，看到测试推文已经存在，显示“34秒前”。非常棒！我们可以让它自动草稿推文。

为了演示它真的使用了技能，我重置上下文（重新打开一个 Claude 实例），然后说：“我刚刚创建了一个 Typefully 视频，解释了我如何创建一个 Claude Code 技能以便通过 Typefully 在我所有的 Twitter 账号上发帖。我想让你草稿三条推文，并草稿到 Typefully 的 Vibe Coding Explained 账号。” 注意，我拼错了 “Typefully”，但它应该能理解。过程中它提示中断，但继续使用了 `xpost` 技能。最终，它创建了草稿，而且推文里还包含了一个链接：指向 Claude Skills 的代码库，并配文“零编码要求。刚刚发布了一个完整教程，教你如何做到这一点。” 太棒了！它使用 `xpost` 技能（技能里包含访问 Typefully API 的代码）创建了这些推文。

最后，我需要删除我的 API 密钥，因为可能已经暴露了。

## 五、技能的工作原理与最佳实践

好了，以上就是我们刚刚完成的事情：创建了一个通用智能体，并赋予它一个技能。我认为这些足够让你入门了。我不想过度深入，比如创建智能体群或带有大量技能的智能体，但我们刚刚在 Cursor 中使用了 Claude，创建了一个智能体，然后只创建了一个简单的技能。这足以让你开始。

正如你所看到的，我只需要提供链接（我会放在视频描述中）指向 Claude Code Skills 的官方资料，Claude Code 就能为你创建技能。记住，你的项目结构应该是：

- 根目录
- 在根目录下，有一个 `.claude` 文件夹（注意文件夹名以点开头）
- 在 `.claude` 文件夹内，有一个 `skills` 文件夹
- 在 `skills` 文件夹内，每个技能一个子文件夹（例如 `skill-1`）
- 在每个技能子文件夹内，有一个 `skill.md` 文件
- 在 `skill.md` 文件中，必须包含 `name` 和 `description` 字段。
当你创建这些技能时（你可以创建多个，但不要过量），确保描述清晰准确。如果你想创建一个通用型智能体并使用 Claude Code，这就是方法。你可以创建一堆不同的技能，每个技能对应一个标准操作程序。
