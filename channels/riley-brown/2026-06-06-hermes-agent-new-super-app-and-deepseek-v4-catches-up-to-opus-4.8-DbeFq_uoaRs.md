### 一、超级应用大战：从"写代码"到"一键部署"的范式迁移 `[视频 00:00 - 04:00]`

**核心观点**

以 **Codex** 和 **Cursor** 为代表的 AI 编程平台，正从"代码生成器"进化为"内部工具即时部署平台"，这被 Riley 视为诞生"AI 时代的 GitHub"的第一步。

**深度阐述**

本周最引人注目的动态之一，是 OpenAI 的 **Codex** 推出了名为 **Sites** 的新功能。Riley 认为，这个功能是 OpenAI 将 Codex 转变为"全面的 Vibe Coding 平台"的第一步，它甚至被许多人誉为 **Lovable 和 Replit 的终结者**。

**Sites 到底做了什么？**

它本质上是一个插件，它会让 Codex 在生成代码后，自动将其部署成一个内部网站。这意味着，任何通过 Vibe Coding（即通过对话式指令生成应用）产出的成果，都会立刻变成可访问的网页。Riley 给出了一个非常生动的例子：他告诉 Codex "请为本构建一个包含本周最重要邮件和 Slack 更新的仪表盘"。由于 Codex 已经连接了他的 Gmail 和 Slack，它能够构建出极具相关性的个性化站点，并且只有团队内部可以访问。

这仅仅是开始。Riley 进一步展示了 Sites 的威力——**通过插件和 Skill 构建一个持久化、跨平台的个人工作流**。

他演示了一个 "康加班 (Kanban Board)" 的内部工具。具体做法是：

1. 他使用了名为 **Convex** 的数据库插件（类似于 Supabase 或 Neon）。
1. Codex 生成的所有任务项都存储在这个 Convex 数据库中。
1. 他创建了一个名为 `/con` 的 **Skill**。这个 Skill 的神奇之处在于，它不仅仅可以在 Codex 的对话中使用，**任何 AI 代理**（甚至是在不同平台上的代理）只要拥有这个 Skill，就能向这个数据库写入数据。
然后，他在另一个完全不相关的聊天窗口中输入：`/con 请添加“制作 Chorus 长视频内容”到在研中`。眨眼之间，他的个人看板就更新了。Riley 对此的评论是：“这就是在构建一个持久化的后端代理。”

**Riley 的个人感受与延伸思考**

在演示中，Riley 的声音充满了创造的兴奋感：“There you go. See, it added it to the board.” 他为自己创造的这个“数字乐高”世界感到由衷的快乐。但他并未止步于此。他用另一个更新——Cursor 发布的 **Canvases**，引出了一个战略层面的预测。

Cursor 的 Canvases 几乎是 Codex Sites 的翻版：允许用户创建内部站点，并可以通过 URL 与团队成员分享。同时，Cursor 还推出了 **Profiles**（`cursor.com/profile`），用户可以像在 GitHub 上一样认领自己的专属 ID。

Riley 将所有线索拼凑起来，提出了一个大胆的预测：

> **“Cursor 正在试图打造 AI 编程时代的 GitHub。”**
他阐述了 Cursor 的两步走战略：

- **第一步（现在进行时）**：让用户轻松创建与公司数据相连的内部工具。这是相对“安全”的，因为数据仅对内可见。
- **第二步（未来时）**：让任何人都可以创建应用，并与任何人分享。其他人可以 Fork 并进行贡献。这彻底重新定义了软件的分享方式——完全非技术的人也能向完全非技术的人分享应用。
Riley 认为，这也是为什么 Cursor 和 Codex 都从“内部工具”入手，而没有直接开放外部共享。因为一旦允许任何人创建连接了公司数据的应用并与外部共享，“这极其危险，事情可能会变得非常糟糕。”

**精华收获**

1. 未来软件的创建和交付将融为一体。Prompt 即是产品，对话即是开发。
1. “AI 时代的 GitHub”的构想是革命性的：开源精神将进入 Vibe Coding 时代，代码审查、协作和分发的属性将彻底改变。
1. 安全与数据治理将是这场变革从“内部友好”走向“大众市场”最大的拦路虎。平台方必须在创新与风险之间找到平衡。

### 二、降维打击：DeepSeek V4 如何用 1/23 的成本改写市场规则 `[视频 04:00 - 07:00]`

**核心观点**

开源模型在 Agent 任务上的迎头赶上，叠加其惊人的成本优势（DeepSeek V4 Pro 比 Anthropic Opus 4.8 便宜 23 倍），正在彻底颠覆商业模型和 AI 代理的商业模式。

**深度阐述**

这是整篇综述中最具冲击力的部分。Riley 分享了一个来自他好友 **Flo Crello（Lindy 公司创始人）**的真实故事：

> Linay 将旗下 **100% 的流量切换到了 DeepSeek V4**，彻底告别了 Anthropic 的模型。这不仅为他们节省了数百万美元，而且在他们关注的许多核心用例上，性能竟然还得到了提升。
Riley 强调，即使 Flo 的话只有一半的准确性，这件事也极其严重。这意味着，在越来越多的 Agentic 任务上，开源模型已经足以与顶级闭源模型匹敌，甚至更好。

让我们仔细看 Riley 列出的成本数据：

这个差距是毁灭性的。Riley 点出了关键：**为什么 Anthropic 和 OpenAI 拼命地冲向企业市场，构建超级应用？**

因为企业用户愿意为了试错和创新，支付每月 1 万、2 万甚至 3 万美元的 API 费用。但消费者市场行不通。

**延伸思考：消费者代理的“50美分”生死线**

Riley 提出了一个明确的行业目标：

> “如果你想让 AI 代理陪你聊一整天（All Day），每月要花费 3000-4000 美元。消费者和许多小企业根本负担不起。消费者代理公司的首要任务，就是把这个成本降到 **50 美分/天**。”
如果成本降不到这个量级，消费者级 AI 代理永远只是一个昂贵的玩具。而 DeepSeek V4 的出现，让这个目标第一次变得触手可及。对于像 Riley 自己的公司 **Chorus**，以及市场上的 **Poke** 等消费级代理公司来说，DeepSeek 不仅是救命稻草，更是唯一的出路。它们可以以闭源模型 1/20 的成本，提供几乎相同质量的智能服务。

**精华收获**

1. 模型能力的“天花板”不再是唯一的护城河。**成本** 正在成为决定 AI 应用渗透率的关键变量。
1. 一场针对 API 定价的“价格战”已经打响。闭源模型的暴利时代即将终结。商业模型必须转向垂直应用、分发和生态！
1. 对于 AI 创业者来说，现在是一个绝佳的窗口期：你可以在拥有顶级智能的同时，保持极其低廉的成本结构。

### 三、心理滤镜：当“安全警告”遇上“IPO 估值” `[视频 07:00 - 08:30]`

**核心观点**

我们需要用一套“心理滤镜”来看待大公司的安全声明，因为公司行为背后交织着对安全、估值和地缘政治的复杂激励。

**深度阐述**

在介绍了 DeepSeek 的崛起后，Riley 迅速将矛头指向了最不可能喜欢这个局面的人，Antropic。他捕捉到了 Anthropic 正在发生的微妙变化：

Anthropic 在闭门会议中警告，他们的模型开始出现 **递归自我改进** 的迹象，即不再需要人类干预，模型就能自己改进自己。他们认为这很危险，呼吁 **暂停开发**，并希望所有 AI 实验室制定一个协调计划，在风险升级时共同停止研发。

与此同时，Anthropic 刚刚提交了 **IPO 申请**。Riley 敏锐地指出了这两件事之间潜在的逻辑联系。

**Riley 提出的“心理滤镜”框架**

他恳请观众在看待类似的新闻时，一定要带上以下滤镜，因为所有的“事实”背后都有动机：

1. **滤镜一：真刀真枪的安全担忧（善意假设）**
An tropic 内部确实有一批人，特别是那些持有“AI 末日论”信仰的员工，他们认为 AGI 的风险是真实且迫在眉睫的。他们呼吁暂停是出于纯粹的善意。

1. **滤镜二：竞争打压与 IPO 估值（商业动机）**
如果 DeepSeek V4 或任何其他开源模型真正具备了媲美甚至超越 Opus 4.8 的能力，那么 Anthropic 的核心价值——它的模型——就受到了威胁。面对即将到来的 IPO，他们最希望看到的是什么？是让“强 AI 变得非法”或者“减缓开源发展的速度”。通过将“开源模型追上来”与“安全风险升高”进行捆绑，他们可以成功游说政府出台对自己有利的监管政策。这既能抬高自己的估值（我们是安全可控的），又能打击竞争对手（你们是危险的）。

1. **滤镜三：地缘政治竞赛（国家动机）**
美国不希望在经济和军事上与中国的 AI 竞赛中落后。DeepSeek 是中国的模型。如果美国政府将安全问题升级为国家安全问题，可能会出台直接针对中国开源模型的限制措施。

Riley 非常克制地强调：**“我并不是说 Anthropic 真的在这么做，我没有证据，甚至没有在预测他们这么做。”** 他的目的是提供一个思维工具。

> “我只是希望，在未来 6 个月里，当你看到关于 AI 模型变得多强大、AI 代理多危险、新法规出台等新闻时，你都能通过这套关于‘激励机制’的滤镜来审视它们。”
**精华收获**

- 不要天真地相信任何一家公司发布的“安全公告”。要思考，这个公告是服务于“全人类的安全”，还是服务于“公司的股价”，或是服务于“国家的利益”？
- 监管是双刃剑。它可能避免灾难，也可能成为巨头们打压新锐竞争者的最强武器。
- 作为普通用户和观察者，保持一种批判性的、多维度的思维方式，是在信息爆炸时代生存的必备技能。

### 四、消费者的超级应用：iMessage 就是新时代的 App Store `[视频 08:30 - 10:00]`

**核心观点**

对于普通消费来说，“超级应用”可能不是 Codex 或 Cursor 这样的 IDE，而是你每天早上都会打开的 **iMessage**。Apple 的正式入局，开启了 AI 代理的分销革命。

**深度阐述**

Riley 将目光投向了与底层模型和开发工具截然不同的战场：**消费级代理的分销渠道**。

故事的主角是 **Poke.com**，一个消费级代理平台。Riley 提到，尽管很早就有公司（包括他自己的 Chorus）通过模拟用户的方式接入了 iMessage，但 **Poke 是第一个获得 Apple 官方批准，正式成为 iMessage 上“官方代理”的公司**。当用户给 Poke 发消息时，气泡显示的不是普通的蓝色，而是代表官方服务的**灰色气泡**。

投资人 **GMO Roush** 对此评论道：**“这可能是第一个让所有人都觉得有意义的代理。”**

**Riley 的深度战略分析：**

Riley 认为，这件小事背后隐藏着巨头的重大布局：

1. **超级应用的分化**：对于发烧友和开发者，超级应用是 Codex 或 Claude Desktop。但对于普通大众，超级应用就是你的即时通讯软件。你不需要记住复杂的 IDE 快捷方式，你只需要在聊天框里说“嘿，帮我把这个会推到后天”。
1. **Apple 的“代理商店”计划**：Riley 爆料，在即将到来的 WWDC 大会上，许多人猜测 Apple 将会宣布一套“代理审批流程”。任何 SaaS 公司或开发者，只要能通过 Apple 的安全检查（类似于 App Store 的上线审核），就可以将自己的业务逻辑变成一个可以驻留在 iMessage 里的 AI 代理。
1. **对创业者的冲击**：这个预测对 Riley 的 Chorus 来说，是巨大的机遇，也是巨大的威胁。
**个人感受**

Riley 在谈到这个问题时，语气变得非常严肃，因为他意识到自己的公司正处在这个历史变革的漩涡中心。他接下来要做的事就是：“尽可能深入地研究如何在 WWDC 后，第一时间让你的代理通过 iMessage 发布。”

**精华收获**

- AI 代理的分发渠道正在从“应用商店”向“消息商店”转移。谁掌握了你的首选聊天框，谁就掌握了 AI 代理的流量入口。
- Apple 正在用其强大的生态控制力，试图建立一个壁垒森严但又安全可靠的“官方代理市场”。这将深刻改变消费者如何接触和使用 AI 的未来。

### 五、开源的火种与巨人的觉醒：Hermes 和微软的差异化布局 `[视频 10:00 - 12:00]`

**核心观点**

在封闭的超级应用和官方的封闭生态之外，开源社区（Hermes）和传统巨头（微软）正在提供另一条路径：自由选择与深度集成。

**深度阐述**

**01. Hermes Desktop：****`[视频 10:00 - 11:00]`**

Riley 将 **Hermes Desktop** 称为 **OpenClaw 的“开放替代品”**。如果说 Codex 不能切换模型、Cursor 有所倾向，那么 Hermes 的价值就在于 **“让你自由地使用任何模型**”——Anthropic 的、OpenAI 的、DeepSeek 的、MiniMax 的、Kim K2 的…… 你想用哪个就用哪个。

他提到，自己和许多朋友都在用 Hermes 开展业务，他们认为 Hermes 比原版 OpenClaw 更好，拥有更成熟的技能系统、更好的记忆力和个性。本周，Hermes 摆脱了繁琐的命令行安装，发布了桌面版。这个版本看起来已经非常像一个专业的超级应用了：你可以创建代理、添加技能、连接消息渠道（iMessage/Telegram）、查看文件、生成图片等。

Riley 特别强调了这个“开放”的意义。在一个日益被几家巨头（Apple、Anthropic、OpenAI）定义的生态里，Hermes 提供了一个自由的避风港，它保证了选择权和可转移性。

**02. 微软的姗姗来迟：****`[视频 11:00 - 12:00]`**

Riley 非常坦率地承认了他通常忽视微软的原因：**“我认为 Copilot 没那么好用，它在上一个 AI 浪潮中的表现简直一团糟。”**

但本周，微软发布了一些让他无法忽视的更新：

- **深度融合**：微软推出了 **深度集成到 Windows 生态系统的 OpenClaw 代理**。这意味着你可以在 Windows 上拥有一个始终在线的、官方支持的 OpenClaw 体验。
- **Dev Box 硬件**：微软推出了一款类似 Mac Mini 的“Dev Box”。它足够强大，能让你**在本地运行开源模型**！这意味着你终于可以摆脱高昂的 API 费用，直接用本地算力运行代理。
- **世界级模型**：微软发布了自己的首批 AI 模型，并声称它们是世界级的。
Riley 的回应是：“我对微软的模型持观望态度，但我必须去亲自测试它们。因为它们如果真的足够好，对于 Windows 和微软生态的用户来说，这可能是一个非常可行的选择。”

**延伸思考**

这两条新闻看似无关，实则互为补充。Hermes 代表了社区的力量和不可驯服的自由精神；微软则代表了平台的力量和无处不在地渗透。它们共同构成了对 Codex、Cursor 和 Apple 完美“围墙花园”的牵制。在一个健康的市场里，我们既需要苹果式的严格与安全，也需要 Hermes 和微软式的开放与包容。

**精华收获**

1. 不要将所有希望寄托于单一平台或单一模型。多样化是抵抗风险的关键。
1. 本地运行模型正在从愿景变为现实（微软 Dev Box），这将进一步颠覆 API 定价模式。
1. 对于 Windows 用户而言，AI 代理的生态正在快速成熟，这可能会改变整个开发工具的市场格局。

### 结语：一个创始人的“元视角”

看完 Riley 的这期内容，最强烈的感受是：**他不是一个坐在屏幕前念新闻的主播，他是在战场第一线拼杀的士兵。**

在整个视频中，他穿插介绍了他自己公司的产品——**Billy**。乍一看是个广告，但实际上，Billy 就是他将上述所有趋势集于一身的完美注脚。

- Billy 是一个运行在 **iMessage** 上的代理。
- 它需要像 **DeepSeek** 这样廉价的模型才能实现“全天候待命”的愿景。
- 它拥有自己的 **技能市场**（类似于 Codex/Cursor 的插件生态）。
