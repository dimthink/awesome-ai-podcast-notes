## 开篇引入

想象一下，你只需在笔记软件里写下一行「完成队列中的任务」，一个智能代理就会自动分析你的待办事项、查阅相关资料、生成内容草稿，甚至直接发布到社交媒体——全程无需你动手打开任何编辑器。这不再是科幻电影的场景，而是 Riley Brown 在视频中展示的真实工作流。

大多数人知道 Claude Code 是目前最强的编程代理，能够从零构建完整的应用。但鲜有人意识到，它完全可以跳出代码的藩篱，成为你的「通用代理人」。Anthropic 最近推出的「Claude Skills」机制，让这件事变得异常简单：只需几个 markdown 文件，你就能赋予 Claude Code 任意领域的专业技能，无论是内容创作、市场营销还是项目管理。本文将带你深入 Riley 的实践，一步步拆解如何用 Claude Code + Skills 打造一个属于你的 AI 代理团队。

## 详细内容

### 一、Claude Code：超越「代码」的代理引擎 `[视频起始-03:00]`

**核心观点**

Claude Code 不仅是生成代码的工具，它本质上是一个能操作文件系统、执行多步骤任务的通用代理。与 Lovable、Replit 等专用编码代理相比，Claude Code 更开放、更强大，可以直接在你的电脑上运行。

**深度阐述**

Riley 用一个简单对比开场：Lovable 这类「vibe coding」工具，本质上是在云端创建一个沙盒环境，然后让一个编码代理在里面生成文件并运行。当你输入「做一个披萨落地页」，Lovable 会启动一个虚拟电脑，代理生成一堆 HTML/CSS/JS 文件，然后通过浏览器展示结果。

同样的过程，用 Claude Code 在本地也能完成。Riley 新建一个空文件夹，在终端运行 `claude`，输入同样的指令：「请为我的披萨店 Riley's Pizza Shop 创建一个漂亮的落地页，采用新粗野主义风格混合友好的披萨能量。」Claude Code 立刻开始思考，几秒后生成了 `index.html` 和 `style.css` 文件。Riley 双击打开，一个完整的网页就运行在本地浏览器中。

> “这完全是在我的 MacBook Pro 上运行，没有借助任何中间软件。Claude Code 只是调用了 Anthropic 的模型，生成了文件，然后我直接打开它们。” `[视频 02:15]`
**个人感受**

Riley 强调，他测试过所有主流编码代理，原生 Claude Code 是最好的。这种「直接在本地工作」的体验，让他感到前所未有的自由——代理不再是云端黑盒，而是你电脑里的一个协作伙伴，可以读写你本地的任何文件。

**精华收获**

Claude Code 的本质是一个「文件操作代理」，它不关心你操作的是代码文件还是普通文档。只要给它合适的目录和指令，它就能执行任何基于文件的任务。这正是它成为通用代理的基础。

### 二、把笔记变成代理的「大脑」：通用代理的雏形 `[03:00-07:30]`

**核心观点**

将 Claude Code 指向你的笔记文件夹，它就能理解你的目标、任务和个人知识库，从而成为一个真正的「通用代理」。

**深度阐述**

Riley 使用 Obsidian 作为个人笔记工具。他的笔记库包含目标列表、任务队列、草稿等。有一天，他在「队列」中写下一条任务：「向我的观众解释 Claude Skills，并制作一个视频。分享 Claude Skills 的文件结构。检查我的笔记，看看哪些技能对我是有用的。」然后他在笔记文件夹的终端运行 `claude`，只说了一句话：「执行我的队列中的任务。」`[04:50]`

Claude Code 首先自动读取了笔记库中的 README 或其他组织文件，理解笔记的结构。它找到「队列」文件，解析出任务，然后开始行动：它查阅了 Anthropic 的 Claude Skills 仓库，生成了一个新笔记，详细解释了什么是技能、如何创建技能文件夹、技能文件的格式。整个过程 Riley 没有给任何具体指示，代理完全自主地完成了研究、整理和输出。

**视觉/结构信息描述**

Riley 展示了 Obsidian 界面，左侧是文件列表：`Tasks.md`、`Queue.md`、`Drafts/` 等。他右键点击笔记文件夹，选择「在终端中新建终端」，然后输入 `claude`。右侧终端窗口中，Claude Code 逐步显示它的思考过程：「检查队列… 找到任务… 搜索 Claude Skills 仓库… 正在创建笔记…」最终，一个名为 `Claude`*`Skills`*`Explanation.md` 的新文件出现在笔记库中，里面包含了结构化的技能介绍。`[05:30-06:45]`

**个人感受**

Riley 激动地说：「这太酷了！我根本没有指定具体文件，它就自己找到了队列，完成了任务。」这种体验让他感觉笔记不再是死文档，而是代理可以理解和执行的「活指令」。

**延伸思考**

这个例子揭示了通用代理的关键：代理需要「上下文」——你的个人知识库、目标、偏好。而笔记软件恰好是存放这些上下文的最佳场所。通过 Claude Code，笔记从「被动记录」变成了「主动执行」的起点。

**精华收获**

要让 Claude Code 成为你的通用代理，只需将终端定位到你的笔记根目录，然后用自然语言下达指令。代理会利用整个笔记库作为背景知识，自动规划并执行任务。

### 三、Claude Skills：让代理掌握专业技能 `[07:30-12:00]`

**核心观点**

Claude Skills 是一种特殊的 markdown 文件，它们像「标准操作程序（SOP）」一样，告诉代理在特定场景下应该遵循哪些步骤、参考哪些资源。通过技能，你可以让代理具备高度专业化的能力。

**深度阐述**

Riley 在 Cursor（一个与 VS Code 类似的 IDE）中创建了一个新项目 `claude-code-skills`。他在终端运行 `claude`，并打开侧边栏与代理对话。他要求代理参考 Anthropic 官方技能仓库，创建一个示例技能。代理自动生成了一个 `.cloud` 文件夹，并在其中创建了 `skills/summarize/skill.md` 文件。这个文件包含了技能的元数据：名称和描述。

Riley 解释：技能文件的核心是 `description` 字段。当 Claude Code 执行任务时，它会根据当前上下文，读取所有技能的描述，判断哪个技能最匹配当前需求，然后使用该技能文件中的指令来指导自己的行为。因此，描述要写得足够清晰、具体，让代理能准确触发。`[09:20]`

**视觉/结构信息描述**

Riley 打开 Cursor 的文件树，展示：

```plain text
.cloud/
  skills/
    summarize/
      skill.md
    twitter-post/
      skill.md
      references/ (内含多张截图)
```

每个技能文件夹下必须有一个 `skill.md` 文件，格式示例：

```markdown
name: Twitter Post Creator
description: 当需要创建 Twitter 帖子时使用此技能。它会分析用户的历史帖子风格，并生成符合品牌语调的文案。
```

此外，技能文件夹内可以包含任意辅助文件，如参考截图、文档、代码片段等，代理在执行时会自动访问它们。`[10:15]`

**个人感受**

Riley 将技能比作「代理的 SOP」——就像公司给员工的操作手册。以前你需要反复给 AI 提示，现在只需要一次性定义好技能，代理就能在需要时自动调用，实现零重复提示。

**精华收获**

技能的本质是「可复用的提示模板 + 相关资源」。通过将你的工作流程固化到技能文件中，你可以让 Claude Code 在任何项目中都能以一致的方式执行专业任务。

### 四、实战：打造个人化的 Twitter 发帖技能 `[12:00-16:30]`

**核心观点**

利用 Claude 自身的能力，我们可以让它分析我们过往的内容，自动生成一个符合我们风格的技能。

**深度阐述**

Riley 想要一个能模仿他 Twitter 风格的技能。他收集了自己过往的一些帖子截图（上面用红色标注了他自己对每篇帖子的思考注解，例如「这条用了 rage bait，效果很好」）。他把这些截图拖入 Cursor 项目中的一个临时文件夹，然后对 Claude Code 说：

> “请用这些图片分析我的发帖风格。创建一个新的 Twitter 技能，替换掉之前的 summarize 技能。技能应该能生成我风格的推文，并使用这些图片作为参考。图片上的红色文字是我的注解，不要把它们当成推文内容。” `[12:40]`
Claude Code 分析了所有图片，将每张图片转换为 markdown 格式的描述，提取了风格特征，并生成了一个新的 `skill.md` 文件，同时将原始图片移入技能文件夹作为参考。接着，Riley 要求代理根据他刚刚录制的关于 Claude Skills 的视频（他提供了视频链接），生成 10 条推文创意。代理立即调用了刚创建的 Twitter 技能，生成了一个 `twitter-ideas.md` 文件，里面是 10 条完全符合 Riley 语气和风格的推文。

**视觉/结构信息描述**

Riley 展示生成的推文样例，例如：“Anthropic just dropped Skills — reusable prompt templates for Claude Code. Invoke them with slash commands. Marketing teams, this changes everything.” 另一条：“How Claude Skills turn one markdown file into infinite workflows.” 这些推文语气活泼，带有感叹号和短句，确实很像 Riley 的风格。`[15:00]`

**个人感受**

Riley 惊叹道：「天哪，这不应该可能做到！我刚刚把 Claude 变成了我个人的营销团队。按需生成社交媒体文案，品牌语调永久锁定，每次零提示。」这种从「分析」到「生成」的闭环，让他感到无比强大。

**精华收获**

创建技能的过程本身也可以由 AI 完成。你只需要提供一些示例和注解，Claude 就能提取出风格模式，封装成可重复使用的技能。这极大降低了构建专业代理的门槛。

### 五、技能升级：集成外部工具，实现端到端自动化 `[16:30-20:30]`

**核心观点**

技能不仅能指导代理生成内容，还能赋予它「行动能力」——通过集成 API，让代理直接与外部服务交互，比如自动在社交媒体平台创建草稿。

**深度阐述**

Riley 希望他的 Twitter 技能不仅能生成文案，还能自动把草稿发布到 Typefully（一个社交媒体管理工具）。他找到 Typefully 的 API 文档，复制了相关链接。他创建了一个 `key.txt` 文件存放自己的 API 密钥（之后立即删除）。然后对 Claude Code 说：

> “请更新我们的 Twitter 技能，使用 Typefully 的 API。参考这份文档（粘贴链接），并读取 `key.txt` 中的密钥。让技能在生成推文后，直接将它们作为草稿调度到 Typefully 账户，不要发布，只保存草稿。” `[17:10]`
Claude Code 理解了任务，它读取了 API 文档，修改了 `skill.md` 文件，在其中加入了调用 Typefully API 的步骤说明（包括如何构造请求、处理认证）。随后，Riley 测试技能：「请用我们刚刚生成的推文列表，将其中一条草稿到 Typefully 的 Vibe Coding Explained 账户。」代理立刻执行，几秒后 Riley 刷新 Typefully 后台，看到一条新草稿：「Hi, this is a test.」显示 34 秒前创建。`[18:40]`

为了验证技能真正通用，Riley 清空 Claude Code 的上下文（重启终端），然后直接说：“我刚做了一个关于 Claude Skills 的视频，请根据它生成三条推文，并草稿到 Vibe Coding Explained 账户。”代理再次调用 Twitter 技能，不仅生成了贴合视频内容的推文，还自动将它们通过 API 存为草稿。其中一条推文甚至自动包含了 Claude Skills 仓库的链接，非常智能。`[19:30]`

**个人感受**

Riley 兴奋地展示结果：「看！它甚至加上了那个 repo 链接。天哪，它真的做到了！」这一刻，他真切地感受到自己拥有了一支 24 小时工作的自动化团队。

**精华收获**

技能的终极形态是「思考 + 行动」。通过结合 API，代理不再局限于生成文本，而是能操作真实世界的工具，实现完整的业务流程自动化。从生成到发布，全程无人干预。

**延伸思考**

Riley 预告了下一期视频：他将直播创建一个更复杂的「代理界面」，包含多个技能，并能与自定义应用交互，实现人与代理共享的工作空间。这暗示着未来我们可以构建高度个性化的 AI 协作环境。
