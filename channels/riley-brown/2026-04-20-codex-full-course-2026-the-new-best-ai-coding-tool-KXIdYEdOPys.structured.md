## 项目与聊天管理

### 创建项目与文件夹

Codeex 的强大之处在于，它是一个能够**控制你整个电脑**的 AI 代理：它可以创建文件、编辑文件，甚至可以使用计算机（完全控制鼠标和键盘）。因此，最有效的方法是给它一个**起始的工作位置**。

当你创建一个项目时，它会打开你电脑上的文件夹（Finder）。我创建了一个名为 “Codeex Projects” 的主文件夹，你也可以放在 “文档” 文件夹里。为了保持有序，我会在这个主文件夹下再创建子项目。

例如，我想研究 Codeex 新桌面应用的所有酷炫功能。我可以：

1. 在 “Codeex Projects” 下新建一个文件夹，命名为 “Codeex Desktop Research”。
1. 选择此文件夹并点击“打开”。
你会注意到，一个名为 “Codex Desktop Research” 的项目被创建了。点击项目名称，你就可以在这个项目下开启一个新的聊天。

### 同时进行多个聊天

在同一个项目下，你可以创建多个聊天，它们可以同时工作。

- 当你创建一个新聊天并提问时（例如：“请告诉我今天发布的 Codeex 新桌面应用功能”），聊天会自动命名（如 “Find New Codex App Features”）并显示在侧边栏。
- 你可以再创建一个新聊天，问：“人们是怎么评价新的 Codeex 应用的？去博客和 X.com 上看看，大家兴奋的点是什么？”
你会注意到聊天旁边有一个旋转的圆圈，表示任务正在进行中。当任务完成时，会有一个**蓝色的小圆点**提示。你可以点击查看未读消息。

### 创建和管理文件

假设第一个聊天已经给出了新功能列表，你可以继续对它说：“请根据你刚才告诉我的内容，创建一个新功能的电子表格。”

代理会创建一个 **Excel 电子表格（.xlsx 文件）**，并将其存储在项目文件夹中。要查看项目文件夹，可以点击聊天旁的三个点，选择 “Open in Finder”。你会看到新创建的 “outputs” 文件夹，里面就是那个电子表格。

预览和全屏文件

点击文件名旁边的“打开”按钮，文件会在侧边视图中打开。你还可以：

- 点击右上角的按钮，将文件**全屏**显示。
- 移除侧边栏，专注于当前文件。
你甚至可以继续对话进行修改，例如：“请移除‘来源页面’这一列。” 代理会完成修改，你可以看到源列已被移除。所有文件都存储在 Finder 的对应文件夹中，方便你管理和引用。

### 引用文件（@ 提及）

在同一个项目下的新聊天中，你可以使用 **@** 符号来引用已有的文件。例如，输入 `@codeex`*`desktop`*`features.xlsx`，然后说：“做一下研究，看看我是否遗漏了任何关于 Codeex 新功能的信息。” 这种方式非常方便。

## 搜索、插件与技能

### 搜索功能（Command + G）

侧边栏的第一个选项是“搜索”。即使你之前将项目从侧边栏移除（这并不会删除电脑上的文件夹），也可以通过搜索找回。

例如，我记得我让代理做过一个“Karpathy 风格”的研究报告，但项目被我移除了。我在搜索框输入 “Karpathy”，它就会搜索所有聊天记录。点击结果，聊天会重新打开。你只需将项目文件夹拖回侧边栏，所有聊天记录就会再次出现。

你也可以使用快捷键 **Command + G** 快速调出搜索菜单。

### 插件（Plugins）与技能（Skills）

很多人会混淆插件和技能。它们之间的界限有点模糊。简单来说：

- **技能**：是为特定类型任务准备的**可复用的工作流程包**，就像一个**可重复使用的配方**。
- **插件**：是可安装的单元，用于**扩展 Codeex 的能力**。
在不同的工具中，它们可能有不同的叫法（如 “Customize” 标签下的 “Skills” 和 “Connectors”）。我个人倾向于将它们都视为“技能”，即**扩展模型能力的方式**。

示例：安装 Google Calendar 插件

如果你想让 Codeex 查看你的邮件或日历，你需要给它相应的技能。

1. 点击 “Google Calendar” 插件，选择 “Install”。
1. 浏览器会打开，引导你登录 Google 账户并授权。
1. 授权完成后，Google Calendar 即成功连接。
之后，你可以在聊天中对它说：“请列出我这周的所有事件。” 代理会调用 Google Calendar 插件，并将事件列表显示在聊天中。

示例：结合 Gmail 发送邮件

你可以进一步要求它：“请给我发一封这周的周报邮件。” 它会调用已连接的 Gmail 技能，将周报发送到你的邮箱。

### 创建自动化（Automations）

你可以要求 Codeex 将一个任务创建为自动化流程。例如：

“你能把这个任务做成每周五下午 4 点运行的自动化吗？”

Codeex 会创建一个自动化任务。你可以在顶部的 “Automations” 选项卡中查看和管理所有自动化任务：

- 查看状态（Active/Inactive）
- 查看下次运行时间
- 测试运行（点击播放按钮）
- 编辑自动化内容
**你只需通过对话就能创建自动化**，就是这么简单。

### 探索插件的功能

如果你不确定某个插件能做什么，只需创建一个新聊天，输入 `@插件名` 并询问。例如：

“@Figma，请告诉我这个插件能执行哪些操作。描述一下你的能力，列出所有功能。”

代理会详细列出该插件的所有能力，例如：

- 检查现有的 Figma 文件
- 生成新的视觉设计稿
- 将 Figma 连接到你的代码库
测试 Figma 插件

你可以立刻测试它。例如，在 Figma 中创建一个名为 “Riley new design” 的新画板，然后对 Codeex 说：“请在这个设计上放上 ‘Hello World’ 文字。” 代理会操作 Figma 完成这个任务。

结合语音输入（Whisper Flow）

我通常使用 **Whisper Flow** 工具进行语音输入，只需按住 Fn 键即可说出我的想法。例如：

“现在去 Figma，忽略刚才的 Hello World 测试。我想让你为我的新鞋公司 ‘New Shoe’ 制作一个落地页的模拟设计。如果需要新鞋的图片，就用你内置的图像生成工具生成一张。”

代理会生成一张鞋子图片（无背景），并尝试在 Figma 中进行设计。

### 更优的设计工具集成：Paper

相比之下，**Paper** 是与 AI 代理配合进行设计的更优工具。它专门用于连接 AI 代理，可以将设计稿直接转换为代码。很多设计师正在转向使用 Paper。

你可以创建一个新文件，并使用你已经创建好的 “Paper” 技能。例如：

“使用 Paper MCP，为我的新公司 ‘New Shoe’ 创建一个落地页。使用之前生成的无背景图片，在打开的 Paper 画板上创建。”

代理会直接生成一个落地页设计。你甚至可以将 Codeex 聊天窗口**打开为迷你窗口**，并最小化 Codeex，让设计工作在其他应用中并行进行。

“转向”（Steer）功能：实时干预

Codeex 有一个独特的 “**转向**”（Steer）功能。通常，你在代理工作时输入新指令，会被排队等待。但如果点击 “Steer”，指令会**立即被输入**，在代理当前任务进行中就能纠正方向。

例如，你看到设计上有重叠问题，可以截图并说：“顺便说一下，这里有点重叠，你在工作时可以顺便修复它。” 按回车默认排队，但点击 “Steer” 则会立即执行，无需等待当前任务完成。

### 创建自定义技能（Skill）

场景：自动拉取 YouTube 视频转录稿

假设你经常需要拉取 YouTube 视频的转录稿。你可以让 Codeex 帮你创建一个自定义技能。

1. **寻找 API**：询问 Codeex：“我想用一个 API 来拉取 YouTube 转录稿，你能推荐前五个给我吗？” 它会列出几个选项，如 **Super Data**、Transcript API 等。
1. **获取 API 密钥**：选择 Super Data，注册并获取 API 密钥（免费套餐每月有 100 次转录额度）。
1. **创建技能**：告诉 Codeex：“请创建一个技能，让我可以要求你总结某个 YouTube 频道最新的 10 个视频。你需要研究如何使用 Super Data，并创建一个技能。这是我的 API 密钥。” 你可以使用内置的 **`skill-creator`** 技能来帮助它更好地理解如何创建。
技能创建完成后，你可以在 **Plugins -> Skills** 中找到它，例如名为 “YouTube Researcher” 的技能。

使用自定义技能

要使用新技能，你需要**创建一个新的聊天会话**。然后输入：

“使用 YouTube Researcher 技能，查看 Riley Brown 最新的 10 个 YouTube 视频，拉取转录稿，并总结成一个文档。另外，告诉我哪些视频表现好，哪些不好，做一个受众兴趣分析。再做一个大纲分析，根据我的视频开场白，判断哪些开场效果好。报告中还要包含 YouTube 缩略图。”

代理会生成一份详细的报告文档（例如使用内置的 Word 文档技能），包含以下分析：

- **成功的开场**：如“Claude is taking over”、“Big market shift”等。
- **效果不佳的开场**：如“Build a sync web app and mobile app in 16 minutes”等。
- **你应该加倍投入的内容建议**。
将技能与自动化结合

你还可以为这个报告创建自动化：“在每月的最后一天，用 YouTube Researcher 技能和 Word 文档技能，为当月视频创建一份类似的报告。稍微简洁一些，表格按观看次数排序。”

这样，一个全自动的月度报告系统就完成了。

### 小结：插件、技能与自动化

我们通过以下步骤创建了一个强大的工作流：

1. 找到一个外部 API（Super Data）。
1. 获取 API 密钥。
1. 让 Codeex 用这个 API 密钥创建一个**技能**。
1. 使用该技能拉取数据并生成**文档**。
1. 将技能和文档生成过程组合，创建一个**自动化**任务。

# 第二部分：多任务实战

在第二部分，我将进行一个角色扮演。我的主要目标是创建一个 iOS 应用，但同时也需要做市场推广。

我们将同时创建以下六项内容：

1. **iOS 应用** (Chorus App)
1. **落地页 / Web 应用** (用于收集用户邮箱)
1. **高质量的启动视频**
1. **投资人演示文稿**
1. **社交媒体自动化** (例如 X 平台的自动发帖)
随着 AI 代理执行任务的时间越来越长（有时长达 1-2 小时），学会**多任务处理**是提高效率的关键。最好的工程师和运营者都是高效的多任务处理者：他们会给一个代理发送指令，信任它去完成，然后立刻转向另一个代理或新任务。

在这个角色扮演中，我的应用名叫 **Chorus**。它是一个让人们学习 AI 代理知识、了解最佳代理平台、学习代理工作原理，以及获取可复制粘贴的技能库的平台。

### 第一步：创建项目总计划

1. 在 Codeex 中创建一个名为 “My New Business” 的新项目。
1. 开启一个新聊天，并说：“我有一个计划，创建一个 Markdown 文件来记录我的计划。” 然后截屏并附上你的想法。
1. 要求 Codeex：“将我列出的所有项目（iOS 应用、落地页、启动视频、投资人演示文稿、自动化等）做成一个简单的清单，放在一个 Markdown 文件中。在顶部写上我的应用想法：Chorus，一个关于 AI 代理的学习和技能库。”
这样你就有了一个总计划，可以随时勾选完成的项目。

### 任务 1：创建 iOS 应用设计（Mobile App Design）

1. 开启一个新聊天，使用一个自定义的 **`mobile-design`** 技能。这个技能是从 Anthropic 的新设计工具中提取出来的。
1. 对代理说：“使用移动端设计技能，为这个应用创建所有屏幕，采用非常基础的苹果风格设计。我们之后再添加一些亮点，现在先确保功能正确。”
1. 将总计划中的应用想法复制粘贴给代理。
### 任务 2：设置 iOS 开发环境（Xcode）

1. 你需要 **Xcode**（Apple 的开发平台）和 **iOS 模拟器**。你可以让 Codeex 搜索如何下载这些工具。你需要一台 Mac 和大约 20 GB 的可用存储空间。
1. 在另一个聊天中，告诉代理：“创建一个名为 ‘Chorus’ 的新项目，这是一个 Swift 移动应用。现在只让它显示 ‘Hello World, This is Chorus’ 的简单文本在屏幕中央。完成后打开 Xcode 项目。”
1. 代理会操作 Xcode 并打开项目。在 Xcode 中选择一个模拟器（如 iPhone 17），点击运行按钮，模拟器会启动并显示你的应用。
### 任务 3：同时进行多项工作

设计屏幕与实现

- 第一个设计任务完成后，代理会提供一个原型链接。你可以查看设计，它可能包含 “Learn”、“Platforms”、“Skills”、“Saved” 等标签页。
- 你可以指示代理：“请将这些屏幕集成到 ‘My New Business’ 文件夹的移动应用中，让应用拥有所有这些屏幕。我们之后会填充真实数据并完善设计。”
创建落地页（Web App）

1. 开启一个新聊天，说：“我想创建一个非常基础的 Web 应用落地页，用来收集意向用户的邮件。只需要一个标题 ‘Chorus App’，和一个用户可以填写邮箱的表单。”
1. 使用外部工具 **Tally.so** 来收集表单提交。在 Tally 上创建一个 “Waitlist” 表单，包含姓名和邮箱，并获取嵌入代码。
1. 回到 Codeex，告诉它：“我正在使用 Tally.so。请把这个表单嵌入到网站中。我们之后再做设计。把它做成一个 React 应用并在本地运行。”
完善 iOS 应用（UI 调整）

1. 回到 Xcode 项目，运行应用。你可以要求代理进行 UI 调整，例如：“每个页面的标题（如 ‘Learn’）应该固定在顶部，滚动时不要移动。底部标签栏再往下移一点，滚动时列表内容在到达底部栏时应该有淡出效果。”
1. 代理完成后，在 Xcode 中重新运行应用，查看效果。你会看到漂亮的顶部和底部模糊效果。
为 iOS 应用添加数据库（Supabase）

1. 询问代理：“我希望应用中的平台和技能数据存储在数据库中，这样我可以方便地用 AI 添加或编辑内容。用户无法编辑。并且支持用户登录。我应该用什么数据库系统？” 代理会给出建议（如 Supabase + PostgreSQL）。
1. 根据代理的指引，在 Supabase 创建一个新项目。
1. 如果 Supabase 没有现成的插件，让 Codeex 搜索网络并找到一个可用的技能或 MCP（Model Context Protocol）。
1. 按照代理的指示连接 Supabase 并创建数据库表。你可能需要重启 Codeex 来使新的 MCP 生效。
1. 完成后，在 Xcode 中再次运行应用，检查数据是否真的存储到了 Supabase 中。你可以登录 Supabase 后台，在 “Table Editor” 中看到创建的表和数据。
生成应用图标
