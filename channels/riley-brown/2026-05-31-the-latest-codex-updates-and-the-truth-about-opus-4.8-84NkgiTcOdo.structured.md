# Codex 重大更新与 Opus 4.8 的真相：AI 模型步入“iPhone 时代”

本周，Anthropic 发布了 **Claude Opus 4.8**，并宣称它是世界上最先进的 AI 模型。然而，也有声音指出，我们已经进入了 AI 模型的 **“iPhone 时代”** ——每次升级之间的差异微乎其微，用户几乎难以察觉。与此同时，**OpenAI 为其超级应用 Codeex（OpenAI 推出的 AI 编程平台）推出了多项重磅更新**，其中一些功能甚至没有在公开场合宣布。本文将全面回顾这些动态，并探讨 AI 模型与超级应用平台的最新趋势。

## Opus 4.8：难以感知的升级与“iPhone 时代”的到来

Anthropic 在周四正式发布了 Claude Opus 4.8。官方公告称，该模型在 Opus 4.7 的基础上，**判断力更加敏锐，对自身进展的诚实度更高，并且能够比前代模型更长时间地独立工作**。根据模型卡的数据，Opus 4.8 在编码（SWEBench Pro 基准测试）方面表现更好，在终端编码能力上虽然不及 GPT 5.5（OpenAI 的最新模型），但在推理、控制计算机、文档/表格/演示文稿等知识工作以及其他财务任务上，**优于包括 Opus 4.7 和 GPT 5.5 在内的所有其他模型**。

然而，实际体验却引发了广泛质疑。视频作者 Riley Brown 表示：“我原本计划做一期完整的 Opus 4.8 视频，但花费了 3 个小时对比 Opus 4.8 和上一代 Opus 4.7 之后，**我 literally（确实）无法区分这两个模型的区别**。”这一感受并非个例：

- **Greg Eisenberg （节目常客）** 直言：“我没有在播客中报道 Opus 4.8，因为我不认为它比 GPT 5.5 有实质性的改进。”他补充道，“而且它同样不比 4.7 有明显提升。**我们正在进入一个模型发布开始让人感觉像 iPhone 发布的时代**。还记得当初每一部新 iPhone 都带来真正飞跃吗？现在不过是摄像头好了一点点，你几乎看不出差别。模型领域正朝着这个方向发展：4.6、4.7、4.8……每一个都略有不同，但很难说哪个最好。”
- **Matt Wolf** 对此表示赞同：“我在近期的新闻汇总中只用了 1 分钟讨论 Opus 4.8，说实话真的没什么可说的。”——相比之下，重大更新往往需要 5 到 10 分钟的篇幅。
Riley 还分享了自己的实际使用经验：“我目前在 iMessage 中运行 AI 代理（类似 OpenClaw），使用的是 **Opus 4.6**。我认为它是通用代理任务的最佳选择（至少基于我的使用方式），而且 **我 literally（确实）无法区分这三个模型**。”

### Deep Suite 对比数据：GPT 5.5 效率更高

专注于前沿编码代理长期任务评估的公司 **Deep Suite** 发布了一组有趣的数据。他们从三个维度进行比较：**成本**、**时间**、**输出 token 数**，并将它们与评分进行对比。结果显示：

- **OpenAI 的 GPT 5.5**（高、中、超高配置）在更低成本下获得了更高评分。
- **成本方面**：GPT 5.5 的每任务平均成本明显低于 Opus 4.8。
- **效率方面**：GPT 5.5 花费更少时间却取得更高分数。
- **信任度**：有用户表示：“**我可以信任 GPT 5.5 处理那些我绝不会交给 Opus 4.8 的任务**。”Opus 4.8 用起来感觉很好甚至让人上瘾（尤其是在“找感觉”时），但那更多是表面工夫。
Riley 补充道：“总体而言，Opus 系列在**设计**方面更优——幻灯片、登录页等视觉呈现更好看，Anthropic 在 Claude 的设计上投入了很多精力。但是，**当你需要执行真正长期、深度的代理任务时**——比如深度编码、控制计算机甚至直接从应用内控制短信——**我强烈推荐使用 GPT 5.5**。”

## Codeex 超级应用迎来重量级更新

Riley 将大型实验室的公告分为两类：**模型更新**和**超级应用更新**。他表示，9 个月前自己更兴奋于模型更新，因为每次更新都带来阶段性变化；但过去 4 个月他“痴迷于超级应用”——像 Claude 桌面或 Codeex 这样能轻松与 AI 代理交互的平台。OpenAI 本周为 Codeex 推出的正是这类创新。

### 1. Windows 计算机控制

现在，在 Windows 版 Codeex 中，你可以键入 `@computer use`，让内置的 GPT 5.5 模型**完全控制你的计算机**。例如，说“控制 Canva 执行任务”，即可在 Windows 上实现。

### 2. Windows 端 Codeex 远程控制（通过 iPhone）

如果你同时拥有 iPhone 和 Windows 电脑，可以**通过手机上的 ChatGPT 应用直接控制 Codeex**。在 Codeex 中找到电话图标，扫描二维码，即可从 iPhone 发送提示词，让 ChatGPT 控制 Codeex，进而控制你的电脑。这一功能同样适用于 Mac，并且同步完美——同一聊天线程在桌面端和手机端同时显示。

### 3. 浏览器保持登录且支持多标签页

在 Codeex 内部，你可以打开浏览器。**新升级后，浏览器将保持登录状态**。例如：打开 twitter.com 自动登录你的账号；连接 Notion 插件后，可以让 Codeex 搜索视频相关 Notion 文档并返回链接，点击即可在 Codeex 浏览器内直接打开 Notion（自动登录）。此外，**现在支持打开多个浏览器标签页**——通过 `Command + 点击` 即可开启新标签页，在标签页间自由切换，仿佛将 Chrome 浏览器直接嵌入 Codeex。

### 4. 代理可以生成子代理

这是许多人最喜欢的功能：**现在 Codeex 中的代理可以创建新的聊天线程（子代理）**。用户可以输入一个“超级提示”，例如让主代理创建 6 个新的 Codeex 线程，每个线程都带有具体的任务提示。主代理会自动编写提示并启动线程，新线程会显示在聊天列表中，由 Codeex 自动命名。这意味着你可以启动 10 个线程同时工作，甚至未来可以创建一个自动化机制来检查这些子代理的完成情况。

### 5. 更好的搜索功能

按下 `Command + G`，现在可以**在 Codeex 中搜索所有聊天会话的全文**，而不仅仅是标题。例如搜索“OpenAI”，所有提及该关键词的聊天记录都会呈现。

### 6. GitHub 活动页面

新增的个人资料页面显示了用户的 Codeex 使用统计：**连续使用天数、最长任务时长（如 3 小时 7 分钟）、总消耗 token 数**等。

## 趋势预测：Vibe Coding 平台将被超级应用取代

许多用户从 **Lovable、Replit、Bolt** 等专用 vibe coding 平台转向 Codeex 或 Claude Code。Riley 认为，大约再过一两个月，这些超级应用就会成为完整的 vibe coding 平台。目前，Replit 等平台的优势在于：一键生成代码、实时预览、设置认证和数据库、托管到互联网。而 **Codeex 通过一个提示就能实现这一切**：

> “请为我的公司建立一个内部工具，用于追踪视频数据。使用 **Neon Postgres （数据库服务）** 作为数据库（需注册并配置插件），使用 Google 登录，使用 **Vercel （托管平台）** 进行托管，使用 **AI Gateway** 实现 AI 功能（只需一个 API 密钥即可使用任何 AI 模型），使用 **Gen Media by FAL （图像与视频模型接口）** 集成 AI 视频能力。运行大量安全检查——GPT 5.5 超高配置在漏洞检测方面表现出色。”
这个提示涵盖了 Replit/Lovable 的完整价值。Riley 预测，**2026 年内将有人构建一个完全 AI 原生的 Replit/Lovable 替代品，作为 Codeex 的插件**。用户无需自建代理和支付 token 费用（OpenAI 大力补贴 GPT 5.5 的使用），只需每月付费约 10 美元即可使用该插件。未来，“vibe coding 将只是任何 AI 代理都能掌握的一项技能”。

## Agent Mini A超级应用的终极形态？

Riley 在过去两个月最痴迷的概念是 **Agent Mini App （代理迷你应用）**。这个概念源于 Codeex 的内部浏览器，并最终将扩展到所有代理平台。

### 现有局限：插件与生成的应用无法打通

Codeex 目前支持 30 多种插件（Gmail、Slack、Typefully、GitHub、Vercel 等），用户可以登录并授权。但 **无法在代理即时生成的迷你应用中使用这些插件身份认证**。例如，代理可以生成一个邮件处理迷你应用，但如果想真正发送邮件，还需要重复登录。

### 理想场景：代理即时生成“Tinder 版邮箱”

想象你在 Codeex 中说：“我需要处理邮件。”代理可以生成一个迷你应用，像 **Tinder 那样呈现**：上面是你收到的邮件，下面是 AI 生成的回复草稿。你可以选择 **“归档”**（不发送）、 **“发送”** 或 **“编辑”**。每一次操作（归档、编辑、发送）都会作为数据被记录，代理会逐渐学习你的偏好，最终做到只在极高置信度下才生成建议回复。这个迷你应用直接连接了你的 Gmail 插件，无需离开超级应用就能完成全部操作。

Riley 指出：“**为什么我要使用其他人的外部平台，如果我的 AI 代理可以在我需要时即时生成一个 UI？**”目前 Codeex 的浏览器是完美的承载环境，但插件 API 还没有开放到允许生成的应用直接调用集成。

### 现实案例与未来方向

Riley 的公司（chorus.com）正在尝试通过 iMessage 实现类似功能：AI 代理发送一个链接，点击后变成迷你应用，这些迷你应用充当代理的“操作系统”。他相信，**所有主要平台都会围绕这一概念演进，最终带来真正的“Jarvis”体验**。

## 结语：成为 Agent Native

Riley 将当前社会比作社交媒体时代：有创作者（利用平台）和消费者（被算法利用）。**在 AI 革命中，他选择成为生产者**。他呼吁观众学习成为 **“Agent Native”**（代理原生），了解 AI 代理存在的各种界面，否则就会像被算法支配的普通用户一样被代理支配。

他在纽约的 Airbnb 中录制了这期视频，公司刚从旧金山搬到纽约。未来他将每周更新，重点分享 AI 代理平台的最新动态与深度洞察。
