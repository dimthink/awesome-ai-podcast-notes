### 详细内容

开启 Vibe Coding 之旅：从想法到原型 `[00:00-03:45]`

**核心观点**

Vibe Coding 的起点是一个清晰的“提示”，它将你的自然语言描述转化为 AI 编码代理可理解的指令，并迅速生成应用的基础结构，包括用户可见的“前端”和幕后运行的“后端”。

**深度阐述**

Riley Brown 开宗明义地指出，Vibe Coding 的核心在于其直观性，它允许用户通过自然语言与 AI 编码代理沟通，从而构建应用程序。这一过程的第一个，也是最关键的步骤，就是提供一个有效的“**提示 (Prompt)**” `[00:50]`。提示是你告诉 AI 编码代理你想要构建什么应用程序的自然语言描述。Riley 强调，“你的第一个提示是最重要的提示”，因为它奠定了整个应用的基础。

在演示中，Riley 选择了 Vibecode.dev 平台，并决定构建一个全栈 Web 应用。他输入了一个具体的提示：“我希望你构建一个像 ChatGPT 一样的应用，但每个聊天都称为一个‘任务’，并且它应该看起来像这样。” 这里的“像这样”至关重要，因为 Vibe Coding 平台允许用户在提示中“**附加图像 (Attach Images)**” `[01:30]`。Riley 截取了 Manis 应用的屏幕截图，将其粘贴到提示中，明确指示 AI 模仿其视觉风格。这种能力极大地提升了用户表达设计意图的精确性，超越了纯文本描述的局限。

除了视觉参考，Riley 还通过提示指定了应用的“**最小化 (Minimal)**”设计风格，并选择了特定的“**AI 模型 (AI Model)**” `[01:50]`——GPT 5.2（尽管在实际生成时，Vibecode.dev 默认使用了 Claude Code 作为编码代理，具体是 Claude Opus 4.5）。这表明用户不仅能控制应用的外观和功能，还能选择底层 AI 的能力。

当 AI 编码代理完成构建后，Riley 展示了生成的应用。他随即引入了两个核心概念：“**前端 (Front End)**” `[02:40]` 和“**后端 (Back End)**” `[03:00]`。

- **前端**：是用户在应用程序中看到和交互的部分，包括按钮、文本、图像、表单以及所有视觉元素。它是用户体验的直接界面。在演示中，Riley 展示了用户可以输入文本并与前端交互。
- **后端**：则是应用程序的服务器端逻辑，用户无法直接看到。它负责处理数据、执行身份验证、管理数据库操作以及进行 API 调用。Riley 解释说，他所使用的 AI 模型就存储在后端，这增强了安全性。
Riley 演示了前端的初步功能，但发现 AI 生成的响应格式不佳。他没有手动修改代码，而是通过“**后续提示 (Follow-up Prompt)**” `[03:45]` 解决了问题。他截取了格式错误的响应截图，并将其粘贴回提示框，简单地写道：“请看这个响应，格式不好，请修复。” AI 编码代理能够理解这个请求并修正前端的显示问题。这引出了一个关键概念：“**上下文 (Context)**” `[04:10]`。AI 编码代理拥有整个对话的上下文，这意味着它会记住之前的所有交互和指令，从而能够理解后续提示的意图，并在此基础上进行修改。这种持续的对话能力是 Vibe Coding 效率的关键。

**视觉/结构信息描述**

视频中展示了 Vibecode.dev 界面，包括一个大的“提示框”用于输入指令和粘贴图片。应用生成后，界面分为左右两部分，左侧是实时运行的应用预览（前端），右侧是代码编辑器（显示“web app”和“backend”文件夹）。当 Riley 粘贴截图并发送后续提示时，AI 代理会重新处理并更新左侧的预览界面，实时展示修复后的格式。

**个人感受**

Riley 对 AI 编码代理的上下文理解能力表现出明显的赞叹，认为这是 Vibe Coding 能够高效迭代的关键。他强调，这种能力让用户无需精确描述所有细节，AI 就能基于历史对话进行智能修正。

**延伸思考**

提示工程的艺术在 Vibe Coding 中被提升到了新的高度。如何通过清晰、富有上下文的提示，以及巧妙地利用图像和后续指令，来引导 AI 代理生成符合预期的应用，是 Vibe Coding 开发者需要深入探索的技能。这不仅仅是技术问题，更是人机协作的艺术。

构建用户系统与数据存储：认证与数据库 `[06:00-09:30]`

**核心观点**

为了使应用具备用户管理和数据持久化能力，Vibe Coding 平台提供了简便的方式来集成“身份验证”和“数据库”，确保用户数据安全存储并与特定用户关联。

**深度阐述**

在应用具备基本的前后端功能后，Riley 转向了构建一个更具交互性和个性化的应用——这意味着需要管理用户和他们的数据。他引入了两个紧密相关的概念：“**身份验证 (Authentication)**” `[06:00]` 和“**数据库 (Database)**” `[06:00]`。

- **数据库**：是应用程序中所有数据存储的地方。它可以被想象成一个巨大的电子表格，其中包含各种信息，如用户资料、任务列表、聊天记录等。
- **身份验证**：则是验证用户身份的过程，例如登录、注册、密码重置等。它确保只有经过授权的用户才能访问其数据或执行特定操作。Riley 形象地解释说，如果你登录了 Twitter，你可以发布推文并将其存储在你的账户下，但你不能以他的账户发布，这就是身份验证的作用。
Vibecode.dev 平台极大地简化了这一过程。Riley 演示了如何通过简单的点击“数据库”选项卡并启用数据库功能，就能触发 AI 代理自动为应用添加数据库和身份验证功能。这种“一键式”的集成能力，是 Vibe Coding 平台赋能非技术用户的典型体现。

当 AI 代理完成加载后，应用界面立即更新，显示出登录/注册界面：“欢迎来到任务，请输入您的电子邮件以登录。” Riley 输入了他的电子邮件，并通过邮件接收到的验证码完成了登录。这个过程生动地展示了身份验证如何确保安全，以及用户如何通过验证码登录。

登录后，Riley 在应用中输入了一条消息“Hi”，并展示了这条消息如何作为“任务”被存储。他随后打开了数据库界面，详细展示了数据库的内部结构。在这里，他指出了数据库中的“**表 (Table)**” `[09:00]` 概念。

- **表**：是数据库中相关数据的集合，类似于电子表格中的一个工作表，每一行代表一个记录。在演示中，Riley 展示了“用户 (Users)”、“会话 (Session)”和“任务 (Tasks)”等表。
- **模式 (Schema)**” `[08:50]`：Riley 进一步解释说，数据库的“模式”定义了数据是如何组织的，例如一个用户有哪些字段（姓名、邮箱等），一个任务有哪些字段（ID、标题、创建者 ID、创建时间、更新时间等）。这些字段共同构成了数据的结构。
他强调，每个任务都有一个唯一的 ID，可以被引用，并且与创建它的用户 ID 相关联。所有这些数据，包括用户消息和 AI 助手的回复，都被实时存储在数据库中。Riley 认为，理解数据库的工作原理是 Vibe Coding 中最被低估的技能之一，尤其是在未来可能构建多个共享同一数据库的应用场景中（例如一个移动应用和一个 Web 应用共享数据）。他鼓励那些有数据科学、金融背景或熟悉 Excel/Notion 的人，会发现数据库概念很容易上手。

**视觉/结构信息描述**

视频展示了 Vibecode.dev 界面中的“数据库”选项卡，点击后应用预览区会显示登录/注册流程。登录成功后，数据库选项卡会展开，显示多个表格（如 Users, Tasks, Sessions, Messages），每个表格都列出了其字段（Schema）和具体的数据行。

**个人感受**

Riley 对数据库在 Vibe Coding 中的重要性表达了强烈认同，认为它是构建复杂、互联应用的基础。他对于平台能够如此便捷地集成这些复杂功能感到兴奋。

**延伸思考**

数据库和身份验证的自动化集成，极大地降低了开发门槛，使得非专业开发者也能快速构建具备用户管理和数据持久化能力的应用程序。这对于快速原型开发和内部工具的构建尤其有价值。同时，理解数据结构和关系，即使在 AI 辅助下，依然是构建健壮应用的关键。

扩展应用能力：API、资源与代理 `[09:30-17:00]`

**核心观点**

通过集成“API”和各种“资产”，Vibe Coding 应用可以扩展其功能和用户体验。更进一步，“代理”模型能够利用工具和循环思维，执行复杂的多步骤任务，而“环境变量”和“代理技能”则增强了其安全性和可定制性。

**深度阐述**

在应用具备用户和数据管理能力后，Riley 转向了如何扩展其功能和交互性。他首先深入探讨了“**API (Application Programming Interface)**” `[09:30]`。

- **API**：是一组定义，允许不同的软件组件进行通信。在 Vibe Coding 中，API 通常指外部服务提供的接口，让你的应用能够调用其功能。Riley 提到，他们正在使用 GPT-5 API，并且所有 API 路由都设置在后端以确保安全。他强调，每次使用 API 都会产生费用，Vibecode.dev 平台会透明地将这些费用计入用户账户。
为了展示 API 的灵活性，Riley 通过提示要求 AI 代理允许用户在 GPT 5.2 和 Gemini 3 Pro 之间切换。AI 代理成功地修改了应用，增加了模型切换功能。当 Riley 询问 Gemini 模型“你是哪个模型？”时，它正确地回复“我是 Gemini，一个由 Google 开发的大型语言模型。”这生动地展示了如何通过简单的提示集成和切换不同的第三方 AI 服务。

随后，Riley 转向了提升应用视觉吸引力的“**图像资产 (Image Assets)**” `[11:00]`。

- **图像资产**：指应用程序中使用的所有图片、图标、背景等视觉元素。Vibecode.dev 允许用户直接在平台内通过文本提示生成 3D 渲染的卡通图标，或者上传外部生成的图像。Riley 生成了一个日本风格的剑形图标，并指示 AI 将其用作应用的 Logo，放置在左上角。这种直接在平台内生成和集成视觉元素的能力，极大地加速了设计迭代。
紧接着是“**声音资产 (Sound Assets)**” `[12:20]`。

- **声音资产**：指应用程序中使用的所有音频文件，如音效、背景音乐等。与图像资产类似，用户可以在平台内生成简单的提示音，或上传自定义音频。Riley 生成了一个“令人愉悦的铃声”，并指示 AI 在代理响应时以 40% 的音量播放。他解释说，添加这个声音是为了在未来将应用从简单的聊天机器人转变为一个“**代理 (Agent)**” `[13:00]` 时，能够提醒用户长时间运行任务的完成。
这引出了视频中一个非常重要的概念：“**代理 (Agent)**” `[13:40]`。

- **代理**：不仅仅是一个 AI 模型，它是一个能够“在循环中使用工具”的 AI 模型。这意味着代理可以执行一系列复杂的步骤：使用 AI 模型生成思考，然后调用外部“**工具 (Tools)**”（如网络搜索），再根据工具的输出继续思考，甚至再次调用工具，如此循环，直到它认为任务完成并“退出循环”。Riley 强调，像 GPT 5.2 或 Gemini 3 Pro 这样的普通 AI 模型不具备内置的工具使用能力，而 Claude Agent SDK 则提供了这种能力。他演示了如何通过上传一个 PDF 文件（关于 Claude Agent SDK 的设置说明）作为“**文件上传 (File Upload)**” `[13:00]`，并提示 AI 将应用转变为一个使用该 SDK 的代理。
通过提示，Riley 指示 AI 使用 Anthropic API 和“超思考 (ultra think)”模式，并特别指定使用“**网络搜索工具 (Web Search Tool)**” `[14:40]`。他随后要求代理搜索 NFL 季后赛的最新信息。在执行过程中，界面清晰地显示代理正在“使用网络搜索工具”，并经过几个步骤后生成了输出。这直观地展示了代理如何利用外部工具来完成任务。

代理的强大之处还在于其“**代理技能 (Agent Skills)**” `[15:50]`。

- **代理技能**：是代理可以执行的特定、预定义的功能，通常通过调用外部服务实现。Riley 演示了如何使用一个名为“技能创建器 (Skill Creator)”的内置技能来为 Claude 代理创建一个新的技能。他希望创建一个 Slack 技能，让代理能够向 Slack 发送消息。这需要配置“**环境变量 (Environment Variables)**” `[16:30]`。
- **环境变量**：是存储在代码外部的秘密值，如 API 密钥、数据库密码等，用于保护敏感信息。Riley 在后端配置了一个名为 `Slack`*`webhook`** 的环境变量，并粘贴了一个 Slack Webhook 地址。他通过提示将这个密钥提供给 AI 代理，指示它使用该密钥来配置 Slack 技能。*
在配置过程中，Riley 引入了“**队列 (Queue)**” `[17:00]` 的概念。

- **队列**：是一个按顺序执行的提示列表。当前一个提示完成后，队列中的下一个提示会自动开始执行。这使得用户可以预设一系列操作，而无需等待每个步骤手动触发。
当 Slack 技能配置完成后，Riley 成功地通过代理发送了一条消息到 Slack 频道，并展示了 Slack 中收到的消息。他进一步演示了如何让代理执行更复杂的任务：先通过网络搜索研究 Remotion 技能，然后将一段总结发送到 Slack。这展示了代理如何结合多个工具和技能来完成多步骤任务。

**视觉/结构信息描述**

视频展示了 Vibecode.dev 界面中 API 切换器、图像生成器、音频生成器、文件上传区（显示 PDF 内容）、代理执行步骤（显示“使用网络搜索工具”）、环境变量配置界面（后端文件夹下的 ENV 变量列表）、以及队列视图（显示当前正在执行的提示和等待的提示）。Slack 消息的截图也清晰展示了代理发送的信息。

**个人感受**

Riley 对代理及其技能的强大功能感到非常兴奋，认为这开启了 Vibe Coding 的无限可能性。他特别强调了环境变量在安全存储敏感信息方面的重要性。

**延伸思考**

代理模型是 Vibe Coding 走向智能自动化的关键一步。它将传统的“一次性”指令转变为“循环思考与行动”的智能体，能够处理更复杂、更动态的任务。环境变量的使用，则体现了现代软件开发中安全性和可配置性的最佳实践，即使在低代码/无代码环境中也同样重要。

迭代与维护：预览、调试与日志 `[19:00-22:45]`

**核心观点**

在 Vibe Coding 的开发过程中，“预览”环境允许实时查看应用变化，“错误”和“Bug”是常见现象，而“调试”则通过“日志”和清晰的描述来定位并解决问题。在开始新功能开发前，“清除上下文”有助于保持 AI 代理的专注。

**深度阐述**

在开发过程中，Riley 强调了实时反馈的重要性，这通过“**预览 (Preview)**” `[19:00]` 功能实现。

- **预览**：是应用程序的开发版本，允许开发者在部署到生产环境之前，实时查看应用的外观和功能。在预览模式下，当对应用进行更改时，可能会发生“**热重载 (Hot Reloads)**” `[19:15]`。
- **热重载**：是指在不完全重启应用的情况下，实时应用代码更改并立即反映在预览界面中。这极大地加快了开发迭代的速度。
在完成一个功能模块后，Riley 引入了“**清除上下文 (Clear Context)**” `[19:50]` 的概念。

- **清除上下文**：是指指示 AI 代理忘记之前的对话历史和指令。当开发者开始一个与之前功能完全不相关的新功能时，清除上下文可以避免 AI 代理受到旧信息干扰，从而更专注于新的任务。这就像给 AI 一个“新的开始”，确保其思考过程的纯粹性。
Riley 随后转向了软件开发中不可避免的一部分：“**错误 (Error)**” `[20:20]` 和“**Bug (Bug)**” `[21:15]`。

- **错误**：是指导致代码停止运行的严重问题。在 Vibe Coding 中，当出现错误时，通常会收到明确的错误消息。Riley 故意让 AI 代理制造了一个错误，以演示如何处理。当错误发生时，用户可以简单地复制错误消息，并将其粘贴回提示框，指示 AI 代理“请修复这个错误”。AI 代理通常能够识别并修正这些导致代码中断的问题。
- **Bug**：则是一种更微妙的问题，代码可能仍在运行，但程序的行为不符合预期。Riley 解释说，这种情况下没有明确的错误消息可供复制。解决 Bug 的关键在于“**调试 (Debugging)**” `[21:40]`。
- **调试**：是查找、识别并修复代码中错误和 Bug 的过程。对于 Bug，Riley 建议提供尽可能详细的描述：“当我做这个时，会发生那个。我不希望发生那个，我希望发生这个。” 越多的上下文和清晰的期望，AI 编码代理（如 Claude Opus 4.5）就越能有效地定位并解决问题。
为了更好地理解应用在幕后的运行情况，Riley 引入了“**日志 (Logs)**” `[22:00]`。

- **日志**：是应用程序在运行过程中打印出的消息，记录了各种事件和操作。它们对于理解应用内部发生了什么、诊断问题以及进行调试非常有用。Riley 演示了如何通过提示要求 AI 代理创建详细的日志。一旦日志被生成，用户就可以在专门的“日志”选项卡中查看它们。这些日志可以被复制，并作为上下文提供给 AI 代理，帮助其进行更深入的调试。
Riley 随后展示了一个例子：他要求代理搜索关于他联合创始人 Ananda 的信息，写一首诗，然后发送到 Slack。在代理执行过程中，日志选项卡中显示了详细的调试信息，如“debug AI stream started”、“AI stream web search”等，这些都为理解代理的思考和行动提供了透明度。

**视觉/结构信息描述**

视频展示了 Vibecode.dev 界面中的应用预览区（显示热重载后的变化）、清除上下文按钮、错误消息弹窗（可复制文本）、以及日志选项卡（显示详细的后端日志信息，可选择复制）。

**个人感受**

Riley 强调了在 Vibe Coding 中处理错误和 Bug 的简易性，认为 AI 代理在这方面提供了巨大帮助。他对日志的详细程度和其在调试中的作用表示赞赏，认为这对于更技术化的用户尤其有用。

**延伸思考**

Vibe Coding 平台通过集成预览、自动化错误修复和可视化日志，极大地简化了开发和维护流程。它将传统开发中复杂的调试过程，转化为与 AI 代理的对话和协作，降低了非专业开发者解决问题的门槛。然而，理解问题的本质和提供清晰的描述，仍然是有效利用 AI 代理的关键。

走向生产：部署、托管与商业模式 `[26:00-30:00]`

**核心观点**

完成开发后，应用需要通过“部署”和“托管”上线，使其可通过“域名”访问。Vibe Coding 平台还提供了“密码保护”功能。在商业模式上，Riley 建议初学者应先专注于构建“个人工具”或“内部工具”，而非直接追求“SaaS”盈利。

**深度阐述**

当应用开发和调试完成后，下一步就是让它对用户可用。Riley 介绍了“**部署 (Deployment)**” `[26:00]` 和“**托管 (Hosting)**” `[26:20]`。

- **部署**：是将你的代码推送到服务器的过程，使其可以在互联网上访问，标志着你的应用正式上线。
- **托管**：是提供运行你的应用程序并使其在线可用的服务。Vibecode.dev 平台提供了内置的托管服务，用户只需点击“部署”按钮，即可将应用推送到生产环境，获得一个可供用户访问的 URL。对于其他 Vibe Coding 工具，可能需要使用 Vercel、Netlify、AWS 等外部服务。
Riley 展示了一个已部署应用的 URL（例如 `frame.io.vibecode.run`），并解释说，用户可以通过这个 URL 访问他们的应用。他还提到了“**密码保护 (Password Protection)**” `[27:00]` 功能。

- **密码保护**：允许用户为整个应用设置一个密码，这对于构建内部工具或仅供少数人使用的应用非常有用，因为它比完整的身份验证系统更简单。
为了让应用拥有更专业的地址，Riley 引入了“**域名 (Domain)**” `[27:40]` 的概念。

- **域名**：是应用程序在互联网上的地址，例如 `your`*`app.com`**，而不是一个随机生成的 URL。Vibecode.dev 计划在未来提供自定义域名集成，用户也可以通过 Namecheap 等服务购买域名，然后指示 AI 代理进行设置。*
最后，Riley 深入探讨了 Vibe Coding 应用的商业模式，区分了三种类型的应用：

- **SaaS (Software as a Service)** `[28:10]`：指通过订阅模式向用户收费以获取利润的应用程序。Riley 坦率地指出，许多初学者在 Vibe Coding 时急于构建 SaaS 产品以赚钱，但这往往是徒劳的。他认为，大多数开发者在尝试构建 SaaS 工具时，都未能从中赚取一分钱，因为这需要构建出真正受用户喜爱的工具。
- **个人工具 (Personal Tool)** `[28:30]`：是为自己日常使用而构建的工具，旨在解决个人痛点或提高效率。
- **内部工具 (Internal Tool)** `[28:40]`：是为团队或公司内部使用而构建的工具，旨在优化工作流程或解决团队面临的问题。
Riley 强烈建议 Vibe Coding 初学者应从构建“个人工具”或“内部工具”开始。他认为，最好的方法是先构建一个自己或团队喜欢并每天使用的工具，当这个工具被证明有价值并受到欢迎时，再考虑如何通过设置“付费墙 (Paywall)” `[29:00]` 来实现盈利。他将此比作内容创作者，不可能在发布第一个视频后就期待赚钱。这种务实的建议，旨在帮助初学者建立坚实的基础，避免过早地陷入商业化的困境。

**视觉/结构信息描述**

视频展示了 Vibecode.dev 界面中的“部署”按钮、部署成功后的 URL 链接、托管详情页面、以及设置中的“密码保护”开关。

**个人感受**

Riley 对初学者直接追求 SaaS 盈利的现象表达了担忧，并给出了非常真诚和实用的建议。他强调了从解决实际问题出发，构建有价值工具的重要性，这体现了他对 Vibe Coding 社区的责任感和对学习者成长的关怀。

**延伸思考**

Vibe Coding 降低了技术门槛，但商业成功的法则并未改变。Riley 的建议提醒我们，技术只是实现目标的手段，真正的价值在于解决问题和创造用户体验。从个人或团队的实际需求出发，是构建任何成功产品的基石，无论其最终是否走向商业化。

总结与挑战：Vibe Coding 的未来 `[29:40-30:40]`

**核心观点**

Vibe Coding 赋能用户快速构建功能丰富的应用，包括代理、数据库、身份验证和技能集成等。Riley 鼓励学习者从解决具体痛点开始，精确定义需求，并分享他们的创作。

在视频的最后，Riley Brown 总结了他们在本次教程中通过 Vibe Coding 所取得的成就。他们成功地构建了一个功能强大的“**代理应用 (Agentic App)**” `[29:40]`，该应用利用了 Claude SDK，具备了“**数据库 (Database)**” `[29:45]` 和“**身份验证 (Authentication)**” `[29:45]` 功能，允许用户登录和注册，甚至可以切换账户。此外，应用还集成了“**Slack 技能 (Slack Skill)**” `[29:50]` 和多个“**API (APIs)**” `[29:50]`，展示了 Vibe Coding 在功能扩展方面的巨大潜力。

Vibe Coding 的核心价值在于其赋能性。它不仅仅是关于技术，更是关于如何将想法快速转化为现实，解决实际问题。Riley 的挑战提醒我们，无论技术如何发展，清晰的问题定义和以用户为中心的解决方案始终是成功的关键。
