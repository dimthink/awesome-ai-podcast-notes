## 详细内容

### 一、超级应用的定义与博学者时代 `[视频 00:00 - 02:00]`

**核心观点**

“AI 超级应用”不只是聊天界面或代码编辑器。它是一个**统一的智能工作平台**，同时涵盖对话、知识工作、编程、集成和自动化，并具备一个全功能的内嵌浏览器。这个定义将决定未来哪些公司能吃掉万亿级的企业市场。

**深度阐述**

Riley 一开场就抛出了他的核心框架：**什么是 AI 超级应用？**

这不是营销概念，而是一个极其务实的定义。一个真正的超级应用必须同时具备五重能力：

1. **对话**：基础的聊天交互，这是入口。
1. **知识工作**：处理文档、表格、幻灯片，这是办公的中枢。
1. **编程任务**：从生成代码片段到构建完整的端到端应用，这是技术生产力的核心。
1. **集成与自动化**：无缝接入现有工具（Slack、Gmail、Google Drive 等），并能创建自动化任务。
1. **内嵌浏览器**：这不是搜索框，这是一个**完整的浏览器**。AI 可以在其中预览它生成的网页，用户可以在这里进行网页操作，甚至更重要的是——AI 本身可以通过“计算机使用”能力操控这个浏览器去执行复杂的网络任务。
这个定义之所以重要，是因为它画出了竞争格局的版图。目前，没有一个产品能完美满足所有五点，但所有玩家都在向这个目标狂奔。

与此同时，Anthropic 的 CEO Amu 提出了一个并行的重要观点——我们正在进入“博学的独立贡献者时代”。正是在这个背景下，Andrej Karpathy 的加入才显得如此具有标志性。

> "Individuals with a ton of talent and areas of expertise, especially within AI and tech, are being recruited for hundreds of millions of dollars." `[视频 04:00 - 04:20]`
> （那些身怀绝技、拥有深厚专业知识的人，正被以亿万美元的价格招募。）
Riley 强调，这不是个例。他看到 CTO 们从超级公司的高管座位上离开，转而以个人贡献者的身份加入 Anthropic。这预示着一个新的权力结构正在形成：在这个时代，创造力的价值正在超越管理能力。

这场战争的残酷性——他称之为“bloodbath”——对于用户而言，反而是最大的利好。因为竞争者们在厮杀中，会不断迭代出更好的产品。

**个人感受**

Riley 的热情极具穿透力。他不仅是在播报新闻。他在讲述一个自己亲身参与的故事。他的语气充满兴奋，同时又带着一种清晰的使命感："The goal of this series is simple. To make sense of all of the updates that actually matter."（这个系列的目标很简单，理清那些真正重要的更新。）

他不是在当旁观者，他是前线的记录者和解读人。

**延伸思考**

为什么这点如此重要？如果你理解了“超级应用”的定义，你就会明白：未来的企业 SaaS 市场可能面临一场彻底的洗牌。不再是 Slack 管通讯、Notion 管文档、GitHub 管代码。所有这些功能将汇聚到一个统一的平台。哪家公司能率先打通这五个维度，哪家公司就将占据未来十年的数字入口。

### 二、Anthropic 的三线作战与致命裂痕 `[视频 02:00 - 06:30]`

**核心观点**

Anthropic 在终端创新、人才招募和桌面应用上同时发力，但它也暴露了一个关键的内部一致性危机：Cursor 和 Claude Code 之间的割裂正在削弱用户的信任。

**深度阐述**

**终端新范式：Quad Agents**

Claude 的终端版本 Claude Code 一直是核心用户的重要武器。但这次，Anthropic 做了一个有趣的创新——不再是线性聊天，而是实现了**多任务并行**。

Riley 现场演示：他一次性往终端里发射了五个独立的研究任务——研究 Google I/O、查询 Cursor 的所有更新、整理 Codex 最新动态、查找 Claude Code 的更新、汇总所有 AI 新闻并生成一个落地页。

这些任务可以并行跑在后台。你可以通过左右键在不同的任务之间切换。每个任务都在独立推进，你在中间不用等待。这种方式打破了传统终端“一问一答”的线性束缚，让 AI 代理成为一种多线程的协作工具。

> "And so it allows me to... see that it's using the web research specialist, which is a custom sub-agent that I created a long time ago." `[视频 03:20 - 03:40]`
> （这让我可以看到它正在使用我很久以前创建的自定义子代理——网络研究专家。）
**人才大迁徙：Karpathy 效应的涟漪**

Karpathy 加入 Anthropic 的消息引发了巨大讨论，但 Riley 做的功课远超大多数媒体：

- 人们在猜测他的薪酬是否超过十亿美元。
- 有人将这比作“C 罗加盟曼城”的技术界等效事件。
- 许多人预测这将引发从 Codex 向 Claude 的用户迁移。
但 Riley 看到了更深的趋势。他列举了一份惊人的“叛逃名单”：

- Super.com 的 CTO 离职加入 Anthropic
- You.com 的 CTO 离职加入 Anthropic
- Workday 的 CTO 离职加入 Anthropic
- Bun（一个流行的 JavaScript 运行环境）直接被 Anthropic 收购
这是一种全新的现象。那些有资格管理大公司的人，放弃了几百万的薪酬和巨大的权力光环，去 Anthropic 做一名**个人贡献者（IC）**。

>"As Amu put it earlier this week, it is the era of the polymathic individual contributor." `[视频 04:00 - 04:10]`

>（正如 Amu 这周所说，这是博学型独立贡献者的时代。）

这是一个极其重要的信号：最聪明的人正在逃离管理层，回归创造层。他们押注于构建 AI 的未来，而不是管理一个现有的商业帝国。

**桌面端的进步与隐患**

Claude 桌面应用确实在变好。浏览器体验更流畅了，响应更快。Riley 赞扬了这一点。但紧接着，他抛出了一个尖锐的批评——这也揭示了他为什么目前仍然在使用 Codex 作为主力工具：

> "I still don't like the fact that I can't use my skills, right? This is a skill that I created on Cursor called 'Riley quality content'. If I go to Claude Code, I can't actually use that same skill. It doesn't exist. So that is why I'm using Codex. I don't understand the relationship between Cursor and Claude Code. Just give me one single app that can do anything." `[视频 05:10 - 05:50]`
> （我仍然不喜欢我无法使用我的技能这件事，对吗？这是我在 Cursor 上创建的技能叫‘Riley 品质内容’。如果我跑去用 Claude Code，我根本无法使用那个技能。它不存在。所以我用 Codex。我不理解 Cursor 和 Claude Code 的关系。给我一个能做所有事情的单一应用。）
这是他评价中最犀利的一部分。Anthropic 虽然拥有极佳的人才（Cursor 最初基于 Claude 构建），但它似乎无法提供一个统一的产品体验。Riley 认为，**Cursor 很可能是 Anthropic 历史上最大的战略失误之一**（"one of their biggest mistakes"）。

**个人感受**

Riley 对 Anthropic 的感情是复杂的。他为 Claude 的创新鼓掌，但又被产品矩阵的碎片化所困扰。他不像一个旁观者在评论，他像一个真实用户在自己搭建的工作流中感受到的挫折。这种真实的体验感，让他的分析超出了新闻本身。

**延伸思考**

这引出了一个核心的哲学问题：**什么才是真正的平台？** 多个好产品串在一起，还是“一个”能做完所有事的统一界面？用户的答案似乎越来越清晰——他们要的是后者。

### 三、OpenAI/Codex：长跑冠军的生态扩张 `[视频 06:30 - 17:00]`

**核心观点**

Codex 凭借革命性的 `/goal` 功能突破了 AI 代理的时间限制，并通过插件共享、设计模式和应用快照（Appshots）打造了目前最接近“超级应用”定义的产品体验。

**深度阐述**

**从指令到目标：****`/goal`**** 的革命**

这是本集视频最具突破性的更新。传统的 AI 代理交互是“指令驱动”的。你告诉它一个步骤，它执行，然后停下。但 Codex 现在引入了一个全新的模式——`/goal`。

当你输入 `/goal`，你就进入了一个新的范式。

> "You can think of goal in Codex as a way to tell Codex the objective for the current task so that it can stay oriented around the outcome not just the command." `[视频 07:20 - 07:40]`
> （你可以把 Codex 中的 goal 理解为一种方式，告诉 Codex 当前任务的目标，让它始终围绕最终结果，而不仅仅是执行指令。）
Riley 现场演示了一个宏大目标：“在结束时，我想让你创建 30 个 iOS 应用。首先做研究，然后创建代码，然后给我发送构建链接。”

这会跑多久？几个星期前，这听起来像天方夜谭。但现在——Riley 报告，已经有人让 Codex 跑过 **1 天 14 个小时**，完成一个深度编码任务。

这个功能是里程碑式的。它意味着 AI 代理的抗压能力和续航能力得到了质的飞跃。Codex 不再被当成一个即时问答的工具，它正在变成一个你可以真正委托大项目的**数字员工**。

**插件共享：团队维度的胜利**

下一个维度是团队协作。OpenAI 发布了插件共享功能。

你可以创建一个自定义插件——一个包含一系列技能、应用程序、配置文件的集合。然后，你可以像在 Slack 生态里一样，通过分享链接，将这个插件直接发送给团队成员或整个工作区。

这个模式之前在 Cursor 或 Claude Code 中也存在（它们叫“连接”或“技能”），但这次 Codex 的更新让分享变得极其简单——点击分享按钮，选择同事或工作区，搞定。

> "What this means is after I click save, now I've shared this plugin for anybody in the workspace." `[视频 09:20 - 09:30]`
> （这意味着我点击保存后，我已经为工作区里的任何人分享了这个插件。）
更强大的是，你甚至可以直接用自然语言命令 Codex 帮你创建一个插件：

> "Hey, can you please create a plugin that does X, Y, Z? It should interact with this API and do this."
或者直接使用 `/plugin-creator` 命令让 Codex 引导你一步步完成创建。这极大降低了自定义 AI 工作流的门槛。

**设计模式：代码的“注释交互”时代**

Codex 正在努力成为一个完整的 Vibe Coding 平台。这次的新功能是拖拽式的 UI 编辑。

Riley 给出了一个非常重要的对比：

- **Cursor 的方式**：直接在代码上编辑，所见即所得。
- **Codex 的方式**：你在界面上点击一个元素，系统不是直接改代码，而是生成一个**注释**（Annotation）。比如你点击一个标题，说“改大一点”，它不会立刻修改代码，而是把这个指令存为注释。然后 AI 会在后台批量执行这些注释。
这种哲学上的差异反映了两种不同的产品思路。Codex 更适合非程序员或需要精细控制的场景。AI 在后台执行列表式的修改，而不是在前端实时改动代码。这是更安全、也更具可审计性的方式。

Riley 还演示了一个简洁但强大的截图功能：你在界面上看到的任何内容，可以一键截图并自动复制到剪贴板。然后你可以在这个截图的基础上加上注释，再扔给 AI 去修改。这提供了视觉反馈闭环。

**Appshots (Cmd+Cmd)：打通所有应用的万能开关**

但这集视频里，Riley 个人最喜欢的功能是 Appshots。它提供了让 Riley 震惊的体验：**在任何应用中，双击 Command 键，Codex 就会自动获取那个应用的上下文并截图。**

> "No matter what application you're on, you can press command command, please research this company and write a draft response." `[视频 13:30 - 13:50]`
> （无论你在哪个应用中，按下 Command Command，请研究这家公司并撰写回复草稿。）
他演示了核弹级别的场景：他在使用 Google Docs 写东西。按 Cmd+Cmd。Codex 立即捕捉到这正是他的视频大纲文档。因为他已经连接了 Google Doc 插件，Codex 直接理解了上下文。他说：“帮我围绕这个提纲写四段话。”

然后奇迹发生了。

> "Oh my god, it's literally writing in the Google Doc with... it's using computer use. If we go back to Codex, you can see that it's actually typing it directly in the document." `[视频 14:40 - 15:00]`
> （我的天，它真的在 Google Doc 里写……它用了计算机使用功能。如果我们回到 Codex，你可以看到它真的在文档里逐字输入。）
Codex 没有把文本复制粘贴过去。它使用了 Computer Use 能力，控制光标，在 Google Doc 里像人类一样逐字打字。

这就是 Riley 定义的未来：**超级应用 + 计算机使用 = 跨应用的真正自动化。**

**个人感受**

"I'm telling you this is one of the most fun features to use ever."（我告诉你，这是有史以来最好用的功能。）Riley 的语气在这里饱含惊叹。他不是在麻木地展示，他是真的被这个技术击中了。Appshots 让他看到了一个不再需要深度 API 集成的未来——AI 可以自己“看”屏幕，自己“操作”电脑。

**延伸思考**

Appshots 打破了围绕 API 集成建立的旧世界。它不再需要 AI 工具去“理解”你的日历模块的具体 API，它只需要能“看见”你的日历。这意味着，任何桌面应用都可能被 AI 代理操控。这场竞争的维度从技术集成，变成了 AI 的“实境感知”能力。

### 四、Google 的困惑：云端的雄心和桌面端的迷惘 `[视频 17:00 - 22:00]`

**核心观点**

Google I/O 在 AI 层面令人失望。虽然 Gemini Spark 展现了一定潜力，但 Google 缺乏一个明确的“超级应用”战略，导致其在各个细分赛道的产品都不具有领先优势。

**深度阐述**

**“Nothing Burger”**

Riley 给了 Google I/O 一个毫不留情的评价：**“It was a nothing burger.”** （基本上啥也不是。）

虽然他承认 Google 谈了很多 AI 代理，但问题是——当你想要用 AI 干活时，你到底该用 Google 的哪个产品？

> "With Google, you don't know. If you want to do agentic tasks, do you use Antigravity? Or do you use Gemini? Or do you use Google AI Studio?" `[视频 18:00 - 18:20]`
> （用 Google，你不知道。如果你想要做代理任务，你用 Antigravity？还是 Gemini？还是 Google AI Studio？）
对比之下，他再次强调，Codex 和 Claude Desktop 都给出了清晰的答案。

**Gemini Spark：潜力无限，方向不明**

尽管整体失望，Riley 承认 Gemini Spark 是 I/O 上唯一引起他注意的产品。但从他的讲述中可以看出，这更多是惋惜而非兴奋。

他展示了一个泄露的视频。Spark 位于 Gemini 应用内部，类似于 Claude 桌面版里 Chat 和 Cursor 的区分。Spark 提供了一系列强大的功能：

- 连接本机文件夹
- 在云端启动虚拟计算机
- 展示代理的完整执行轨迹
- 深度连接 Google 全家桶（Drive、Photos、NotebookLM）
- 生成音乐、视频、图像
- 拥有 Canvas、深度研究、引导式学习
> "This has so much potential." `[视频 19:50 - 20:00]`
> （这有太多的潜力。）
但 Riley 话锋一转：

> "But where I lose interest, I just don't trust them to focus enough." `[视频 20:00 - 20:10]`
> （但我的兴趣就在这里消失了，我不信任他们有足够的专注力。）
为什么？

因为 Google 同时还在弄 Antigravity（编程代理平台）、AI Studio（Vibe Coding 平台）、新的视频模型。他们同时在押注太多的产品，没有一个得到他们全部的资源和专注。

**Not the Best at Anything**

这是 Riley 对 Google 的终评，也是这段分析中最残酷也最客观的总结：

> "They are not the best at anything. Their video models worse than Seed Dance (Baidu). Nano (Google's product) worse than ChatGPT Image Gen. Their LLMs are nowhere close to frontier level. Their vibe coding tools are not the best. Their chat experience is not nearly as good as the normal chat version on Claude. Nothing that they've created is state-of-the-art." `[视频 20:40 - 21:10]`
> （他们哪方面都不是最好的。他们的视频模型不如百度的 Seed Dance。Nano 不如 ChatGPT 的图像生成。他们的大模型离前沿还差得远。他们的 Vibe Coding 工具并不是最好的。他们的聊天体验远不如 Claude 的标准聊天版。他们没有创造出任何一件作品是业界顶级的。）
唯一的好东西？**视频分析功能**。但你很难靠一个单品撑起一套操作系统。

> "I hope they just announce a code red and focus all their attention on one of the apps." `[视频 21:30 - 21:40]`
> （我希望他们宣布一个“红色代码”，把所有注意力集中在其中一个应用上。）
**个人感受**

Riley 的失望感非常真实。作为 AI 领域的深度用户，他对 Google 的迟钝感到惋惜。他并不是在贬低 Google 的技术实力，而是在批评 Google 的产品战略。他像是一个看着天才队友瞎搞的球员，痛心疾首地喊着："Just focus!"

**延伸思考**

Riley 对 Google 的评价揭示了一个残酷的事实：在 AI 这个领域，**技术优势如果不能转化为聚焦的产品战略，就毫无意义。** Google 拥有 DeepMind 这样的瑰宝和巨大的资源，但缺乏一个统一的中心战略来把这些力量汇聚成一个面向用户的产品。当 OpenAI 和 Anthropic 倾尽全力打造“唯一的那个应用”时，Google 在分散自己的力量。

### 五、Cursor 的极速崛起：新巨头的诞生 `[视频 22:00 - 26:00]`

**核心观点**

被 xAI 以 600 亿美元收购的 Cursor，凭借 Composer 2.5 模型的极速和高性价比，正在成为一个不可忽视的超级应用竞争者。

**深度阐述**

**一场改变格局的收购**

Cursor 的更新是本集视频中最令人兴奋的硬件消息。xAI/SpaceX 同意以 **600 亿美元** 的价格收购 Cursor（含 100 亿美元的退出条款）。

600 亿美元意味着什么？意味着 Cursor 突然拥有了像 OpenAI 或 Anthropic 一样充足的算力。他们立即利用这些资源训练了 **Composer 2.5**。Riley 透露，一个更大的模型 Composer 3 已经在路上。

**Composer 2.5：快、便宜、接近前沿**

Riley 现场进行了实机测试：要求创建一个类似 Linktree 的落地页，并使用 Cursor 的品牌风格。

> "This model is pretty close to frontier especially for front-end work." `[视频 23:20 - 23:30]`
> （这个模型非常接近前沿，尤其是在前端工作上。）
但最令 Riley 感兴趣的是它的速度和价格：

> "This brand new composer 2.5 model which hardly uses any tokens... you could use it for like an hour and I don't think it would cost you a dollar." `[视频 23:00 - 23:20]`
> （这个全新的 Composer 2.5 模型几乎不怎么消耗 Token……你可以用它狂用一个小时，我觉得花费不会超过一块钱。）
他在现场演示了速度对比：当他提出设计要求时，落地页几乎立刻生成。他评价说：**"This would not go nearly as fast if you use Codex or even Claude. It is significantly faster."**（如果你用 Codex 或 Claude，这根本不可能这么快。它快太多了。）

**功能追赶：成为真正的超级应用**

Cursor 正在全面追赶 Codex。

- **内嵌浏览器**：已经和 Codex 一样完善。
- **应用市场**：已经上线，拥有各种集成。
- **自动化功能**：已经开始推出。
SpaceX 在收购公告中明确表示，他们的目标是：**"build the next generation platform for coding and knowledge work."**（构建下一代代码和知识工作平台。）

这意味着 Cursor 不打算只是一个编辑器，它要成为一个 Super App。不过，与 Codex 目前的差异在于，Cursor 还不能像 Codex 或 Claude 那样原生地创建文档、PPT 和电子表格，并在侧边栏打开。

> "But my guess is that's coming very soon." `[视频 25:30 - 25:40]`
> （但我猜这很快就会到来。）
**个人感受**

Riley 对 Cursor 的更新充满了认可。他甚至有些急迫地想做一集 Cursor 的完整指南。这种热情源于他体验到的“速度”的质变。在一个 AI 工具逐渐变得同质化的市场里，速度成为了一个极具差异化的竞争力。

**延伸思考**

Cursor 的崛起让这场超级应用战争变得更加复杂和有趣。三个主要玩家——OpenAI (Codex)、Anthropic (Claude Desktop)、xAI (Cursor)——都获得了巨额资金和算力支撑。这不是一个市场，这是一个**三体问题**。他们的竞争将会非常激烈，用户将成为受益者。

### 六、战场延伸：从桌面到 iMessage `[视频 26:00 - 结束]`

**核心观点**

这场战争的最终形态不仅仅是桌面端的厮杀。移动端（通过 iMessage 或短信触达的 AI 代理）将会是下一个巨大的战场。

**深度阐述**

**Chorus.chat：手机里的超级应用**

Riley 没有只是分析其他人的产品。他也在自己的试验田里实践他的理念。

他带着团队创建了 **Chorus.chat**。核心思路：

- 把 Claude Code 和 Codex 接入了 iMessage。
- 你可以在手机上和一个 AI 代理对话（它拥有一个名字，Riley 的叫 James Bond）。
- 你的 AI 代理运行在云端沙盒里，可以完成编程、研究等复杂任务。
- 它甚至可以在后台执行 Cron Job 自动化任务。
> "My AI agent, James Bond. Hey, James, what are you up to? And so I'm talking to Claude Code. Very soon, you can talk to Codex. It can do anything." `[视频 27:00 - 27:30]`
> （我的 AI 代理，James Bond。嗨，James，你在干嘛？我在和 Claude Code 对话。很快你也可以和 Codex 对话了。它什么都能做。）
这是 Riley 的“移动优先的迷你超级应用”的实践。他的核心论断是：**AI 代理不应该只是存在于一个专门的桌面应用里，它应该可以主动地通过你日常使用的任何通讯渠道触达你。**

**对未来的预期**

Chorus.chat 不仅会支持 iMessage。Slack 正在开发中，WhatsApp 即将推出，还将提供电话号码。用户可以直接和自己的 AI 发短信，就像它是你通讯录里的一个真实联系人。

> "We want to make one that's a lot easier to get started... It will start doing tasks for you and you can create Cron jobs." `[视频 28:10 - 28:30]`
> （我们希望让它更容易上手……它会开始为你做事，你可以创建定时任务。）
**个人感受**

这个自荐产品充满了 Riley 的个人风格。他不是在卖一个软件，他是在展示一个他认为必然会发生的未来的原型。他就像是科技界的约翰·亨利——不仅记录铁路建设，还在亲手铺铁轨。

**延伸思考**

想象一下，你不再需要打开一个特定的应用去给 AI 下达任务。你会在 Slack 里说：“帮我把明天的会议准备做好，生成文档”，或者给 AI 发一条短信：“在 Codex 里创建一个新的 React 组件，并把它提交到仓库”。这就是 Riley 想用 Chorus.chat 实现的世界：AI 代理成为我们的**同事**，存在于我们的聊天列表里，而不是一个我们需要主动去操作的“工具”。

### 精华收获

1. **终极定义**：未来的“AI 超级应用”必须同时满足对话、知识工作、编程、集成和自动化，并拥有一个内嵌浏览器。谁能打通这五重维度，谁就能统治企业桌面。
1. **长跑能力**：Codex 的 `/goal` 功能让 AI 从“问答机器”进化为“数字员工”。AI 代理的运行时间突破到 24 小时甚至更长，这改变了项目的规模和性质。
1. **人才密码**：科技巨头的 CTO 们为做个人贡献者而加入 Anthropic，预示了“创造价值 > 管理价值”的新范式。人才流向即投资方向。
1. **Google 的困境**：不是没有技术，而是缺乏聚合。多平台战略正在稀释 Google 的竞争潜力，“不是最好”的标签正成为其 AI 产品最大的污点。
1. **速度即竞争力**：Cursor 被 xAI 收购后，凭借更快的模型和更低的成本，正在迅速抹平与 Codex 的功能差距。用户体验的竞争正在回归最本质的“快”。
1. **移动端是下一城**：桌面战争如火如荼，但 AI 代理的移动端入口（通过 iMessage、Slack、短信触达的代理）可能是下一个决定性的战场。AI 不应该是一个应用，它应该是你通讯录里的一个联系人。
