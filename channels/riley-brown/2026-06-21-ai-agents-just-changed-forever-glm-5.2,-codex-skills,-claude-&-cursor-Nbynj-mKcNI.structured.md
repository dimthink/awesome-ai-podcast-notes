今天，我们从我认为 AI 智能体领域最重要的新闻开始。公司 **Z.ai** 发布了一款开源模型，我认为它比 **GPT 5.5** 便宜五六倍，有些人说它实际上能与 **Opus 4.8** 和 GPT 5.5 相媲美，几乎一样好。**GLM 5.2** 就是 Z.ai 发布的模型，这家公司来自中国，该模型是开源的，并且比前沿模型便宜得多。顺便说一句，我将向你展示如何在 Cursor 中一秒完成设置。但首先，我想谈谈基准测试。

以下是一些基准测试，这是它在各个方面的表现。你会看到 **GLM 5.2** 与 Opus 和 GPT 5.5 不相上下。目前，我认为除 **Fable** 外最好的模型是 GPT 5.5，Opus 稍逊一筹，但当我实际测试时，这个模型确实表现出色。因为通常情况下，当新开源模型发布时，会有基准测试结果发布，但并不能反映全貌，甚至完全不准确。但由于他们在 Twitter 上发布了这些炫酷的图表，引起了大量炒作，很多人制作视频说这个模型真的很好。而我通常去测试那个模型时，往往大失所望。我实际测试后发现它通不过 **“体验检验（vibe check）”**。正如我今天早些时候在推特上所说，这次并非如此。在花了很多时间实际使用这个模型后，我确实相信它通过了体验检验。我认为它正在接近前沿实验室的水平，特别是 **Opus 4.8** 和 **GPT 5.5**，我认为这实际上会促使 OpenAI 和 Anthropic 等前沿实验室发布更智能的模型。我认为很多人意识到他们每天依赖的模型可能会被取代。然而，有了这些开源模型，你可以实际下载权重。我认为很多人正在利用这个时机测试这些开源模型。

所以，在我看来，试用这个新模型 **GLM 5.2** 的最佳地点是直接在 **Cursor** 中。稍后我将展示如何设置。我使用了 **Convex** 插件，它一键生成了一个 **Trello** 应用，基本上包含了 Trello 的所有不同功能，带有数据库和身份验证，而且它几乎完美运行——实际上完美运行。我还让 GLM 5.2 去研究我，然后创建一个落地页，并在本地运行。我还将 GLM 5.2 连接到了我的 **Notion**、**Slack** 和大量其他集成，让它为我执行一般的智能体任务，它做得非常出色，就像我使用 4.8 一样好。所以，是的，我认为这个模型非常好，你会在互联网上看到很多人说这个模型非常非常好。但你不应该只听我们一面之词，你应该亲自去尝试。我将向你展示在 Cursor 中试用的最简单方法。

在 Cursor 中，请按照以下确切步骤操作，只需 3–5 分钟即可将模型添加到 Cursor。为了将模型添加到 Cursor，我们将使用另一个工具叫 **OpenRouter**。通常，如果你想使用大量不同的 AI 模型，你需要大量的 API 密钥。OpenRouter 允许我们只使用一个密钥，就能访问 GPT 5.5、Claude Opus、DeepSeek V4，以及这里最重要的 GLM 5.2，还有成千上万的其他模型。这个视频不是 OpenRouter 赞助的。我只是解释为什么我通常使用它。OpenRouter 允许我们将任何模型添加到 Cursor。

1. 在 Cursor 中，进入计划设置，点击“设置”，然后点击“模型”，接着点击“API 密钥”。
2. 开启 **“覆盖 OpenAI 基础 URL”** 开关（通常关闭），然后输入你自己的 API 密钥。
3. 在“覆盖 OpenAI 基础 URL”字段中，粘贴 OpenRouter 的基础 URL：`https://openrouter.ai/api/v1`（具体链接见视频描述）。你实际上是将这个 OpenAI 密钥转换为 OpenRouter API 密钥。
4. 点击“查看所有模型”，再点击 **“添加自定义模型”**。
5. 去 [OpenRouter](https://openrouter.ai) 点击“模型”，搜索 `z.ai/glm-5.2`，你会看到一个复制按钮，点击复制。
6. 将复制的模型名称粘贴到 Cursor 的自定义模型框中，点击添加。由于我已经添加过，它显示已可用，但对你来说，它应该出现在模型中，看起来像 `z-ai/glm-5.2`。

恭喜，你现在可以在 Cursor 中直接使用最好的开源模型了。我们来测试一下。在 Cursor 中新建一个智能体会话，你可以选择任何模型。我选择 **Z-AI/GLM2**（注：对应 `z-ai/glm-5.2`），然后说“嗨，你是什么模型？”，它回答“我是 Z-AI 的 **GLM 5.2**”。你现在可以测试这个最好的模型了。我希望你在评论区告诉我你用这个模型做了什么，它表现如何。我真心想知道你对此模型的看法。请告诉我。

我认为你应该开始使用并测试这些开源模型的原因之一是，创建 GLM 5.2 的 Z.ai 创始人表示，他们计划在今年内推出一个与 **Fable**（Anthropic 的最强模型）水平相当的开源模型。有人问：“目前中国达到 Fable 级别或与 Fable 5 一样好的时间表是什么？”**埃隆·马斯克** 评论说：“可能是第一季度。”然后创始人说：“不需要那么久。”这意味着他认为年底前就能完成。也就是说，大约 5 个月后，我们可能会得到一个开源模型，比 Fable 更好，而且可能便宜得多。

## 二、Codex 新增录屏技能功能

现在在 Codex 中，你可以使用一个名为 **记录与重放（record and replay）** 的插件。我将演示添加一个 **Typefully** 草稿的过程。请创建一个名为“手动推文草稿”的技能。现在，你只需通过这个“记录与重放”技能告诉 Codex 你想展示如何做某事。它会说：“我将使用记录与重放工作流来捕获 Typefully 步骤。”现在，看这个！它自动打开了录制，并说：“录制已开启。请向我展示 Typefully 草稿过程。”

好了，现在技能创建完成。我们可以输入 `/manual tweet draft`。现在说：“嘿，请将最新视频上传到 Typefully 作为一个草稿。它在我的下载文件夹中，就是最新的那个视频。”然后它开始执行，我们可以打开 **Comet**。可以看到，计算机使用功能正在工作，那是它的小鼠标。它点击了“新草稿”，现在应该上传一个视频或至少为它输入草稿。然后上传图片，找到最新视频。哇，太疯狂了。哦，需要升级。视频太大了，不能上传超过 512 MB 的。但意思你明白了：我录了屏，教 Codex 如何使用 Comet 上传东西到另一个软件，然后立即变成了一个技能。要实现这一点，你只需要使用“记录与重放”功能，说“我要录制一些东西，观察并把它变成技能”，并告诉它你想给技能取什么名字。就是这么简单。

## 四、Claude 更新：设计模式与迷你应用

好的，在结束这一集之前，我想谈谈 **Claude** 的一些更新。我觉得我们所有人对 Claude 都有一种说不出的别扭感，我想我们都处于这种 **Mythos** 或 **Fable 的失落**中。这是我截取的一条推文，但我已经看到了成百上千类似的推文，内容大概是：

> “我不知道是不是安慰剂效应，但使用 Fable 的那几天，感觉它从不放弃问题，一直尝试各种疯狂的方法来完成你想要的任何事情。现在回到 Opus，它有点懒。觉得事情太艰巨，总是问你是否确定。使用 Fable 时有一种感觉，你基本上可以做任何事情。”

AI 模型最好的基准之一就是你实际上能有多雄心勃勃。Fable 给我的感觉是，我根本不够聪明，想不出一个 Mythos 或 Fable 无法胜任的事情。所以在过去四个月我在硅谷时，我和每个人交谈，每个人都在说 GPT 5.5 有多好。但现在他们只能用上 Fable 4 天，就再也回不到 GPT 5.5 或 Opus 4.8 了。他们真的处于这种 **Fable/Mythos 失落**中，只是等待这个模型回来，因为他们知道一旦回来，他们就能在极短的时间内完成他们试图做的事情。这就是 Fable 有多好。

现在是东部时间 **6 月 19 日下午 3 点 26 分**，Fable 还没有回到任何 Claude 产品中，仍然禁止使用。Anthropic 正在与政府合作，试图找到让这个模型回到我们手中的方式，我们完全不知道它什么时候会回归。

但除了这种 Mythos 失落，还有两个更新我想谈谈。第一个触及我一直在谈论的主题：智能体原生应用。但首先我要谈谈 **Claude 设计模式**的新更新。Claude 设计的新特性：**跨项目保持品牌风格**、**直接在画布上编辑**、**与 Claude Code 同步**、**连接更多你已在使用的工具**。如果你还不知道，在 `Claude.ai`（仅限网页版，不支持桌面）上，有这个“设计”功能。首先他们宣布的是“跨项目保持品牌风格一致”，我还没用足够长时间来测试。但我可以测试的是“直接在画布上编辑”。我注意到这里有一个编辑功能。我可以直接编辑吗？例如写“开源对手在此。哇。一个便宜的通过了体验检验的中国模型。记录 GLM 5.2 是一个非常好的模型。”这非常有趣。当然，你也可以做标注。比如“这里不要这些，我不喜欢这些。”真的很棒。Claude 增加的另一个功能是很容易将这些设计分享到其他工具。所以我可以发送到 **Lovable**、**Base 44**、**Gamma**、**Miro** 和 **Replit**。理论上，如果设计了一个落地页或网站，我可以发送到 Lovable 并连接，然后部署到 Lovable，或者直接部署到 **Vercel**。我已经设置好了，现在可以发送到 Vercel。这样它就可以真正上网了。

最后，**Claude Code** 新增了一个功能：**组件（artifacts）**。你知道 Claude 网页版和桌面应用在普通 Claude 模式下已经有 artifacts 了。但现在 Claude Code 也可以创建 artifacts，并发送可交互的页面。例如，它说“研究自上一个版本以来用户在哪里流失。”然后我们看到在这个视频中，它创建了一个小程序或者智能体原生应用，你可以与其他人分享。它创建了一个 artifact，有自己的链接。然后说“提出解决方案”，所以它在小程序中展示了当前和提案。你可以让智能体或 Claude Code 创建这个小 artifact（我称之为 **迷你应用**）。你可以查看，然后说“好的，这个主意不错，但这是他们提议的，好的，可以这么做。”如果你想与团队分享，只需点击复制链接，然后发送给任何人。示例中是一个手机，链接便于分享。你可以将这些小程序（或 artifacts）轻松分享给团队。
