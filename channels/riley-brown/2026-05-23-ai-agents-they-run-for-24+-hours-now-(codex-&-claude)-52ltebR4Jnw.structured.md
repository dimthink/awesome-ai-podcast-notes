## Anthropic / Claude 更新

### 1. Claude Code 终端引入多代理模式（Quad Agents）

几周来，我第一次又回到了终端。我之前在进行“氛围编程（Vibe Coding）”，有点怀念终端环境。Anthropic发布了一个非常有趣的新功能。

长久以来，要运行 **Claude Code**，你需要打开终端输入 `claude`，然后在一个线性对话中向下工作。而现在，你可以输入 **`quad agents`**——这是在终端与AI代理交互的一种全新方式。它让你能够进行多任务处理。

例如，我可以同时发起以下任务：

- “请研究Google IO大会的最新消息，特别是 **Spark**。”
- “请研究 **Cursor** 的所有更新，给我一份非常清晰的列表。”
- “请查找 **Codex** 的所有不同更新。”
- “请告诉我 **Claude Code** 的最新更新。”
- “请搜索AI世界的其他更新，并且把它们整合成一个小型着陆页（Landing Page）给我。”
按下回车后，这些任务会立即启动，并在终端中显示工作状态。同时触发了五个独立的任务。

导航方式非常直观：你可以滚动到每个任务，然后按下键盘上的 **右键** 进入聊天界面，按下 **左键** 返回。我可以看到，它在使用“网络研究专家”——一个我很久以前创建的定制子代理（Sub Agent）。这是一种在终端与AI代理互动的有趣方式，Anthropic刚刚发布了这项功能。

接下来是一个大消息。**Andre Karpathy** 加入了 **Anthropic**。

很多人都在猜测这笔交易的规模、他拿到的薪酬。有人猜测薪酬超过 **10亿美元**。有人说这是科技界的 **C罗加盟曼城**。还有人认为，这会促使人们从 **Codex** 回流到 **Claude**。

如果你看过我的视频，就会知道我已经切换到了 **Codex**，至少我所有的代理任务都交给它，因为它拥有更统一的产品。但看看 Anthropic 吸引而来的顶尖人才：

人们真的在离开世界上最顶级的工作岗位，离开运营大公司的职位，转而成为 Anthropic 的一名 **个人贡献者（Individual Contributor）**。

正如 **Anu**（Anthropic CEO）本周早些时候所说：“这是多才多艺的个人贡献者（Polymathic Individual Contributor）的时代。” 我们正在看到这一趋势：才华横溢、在多个领域拥有深厚专长的人，尤其是在AI和科技领域，正被以数亿美元的薪酬招募。这是一个非常有趣的趋势。

### 3. 桌面应用改进与“超级应用”讨论

今天关于Claude的最后一项更新是他们对桌面应用进行了一些整体改进。

有些人喜欢用终端界面来使用Claude，但我已经不是了——因为他们已经做出了桌面应用。Claude创造了最好的 **超级应用（Super App）**之一。

我认为超级应用的定义是：一个AI平台，你可以用**它来聊天、进行知识工作、编写代码**，并且它能够**连接到所有你现有的集成**。这些AI代理平台应该连接所有现有工具，并且能够创建自动化。无论你的AI代理在做什么，它都应该能在右侧打开一个小界面。我将一直在谈论超级应用，因为我认为这就是企业用例的未来。它将演变成一个完整的浏览器。这就是我所定义的AI驱动的超级应用。

我认为Claude桌面应用在过去几周内有了很大提升。**技能（Skills）**的使用变得更容易了。但我仍然不满意的一点是：我无法使用我的技能。我在 **Co-work** 中创建了一个名为“Riley quality content”的技能，当我切换到 **Claude Code** 时，却无法使用同一个技能——它不存在。这就是我现在用 **Codex** 的原因。我不理解 **Co-work** 和 **Claude Code** 之间的关系。给我一个单一的应用，能够完成所有事情：编码任务或知识工作。我认为 **Co-work** 是他们最大的错误之一。

我要说的是，在这个桌面应用上做编码任务时，浏览器的工作效果好多了。最开始他们刚发布Claude Code功能时，浏览器还不太稳定，现在已经好多了，响应更快。

以上就是今天关于Claude的所有更新。

## OpenAI / Codex 更新

与Claude拥有其桌面超级应用类似，OpenAI则拥有 **Codex** 作为他们的超级应用版本。

### 1. /goal：长任务模式

他们正式在Codex应用中添加了 **`/goal`** 命令。你现在可以输入 `/goal` 回车，然后给它一个超级长的任务。

你可以把 Codex 中的 **goal** 理解成告诉 Codex 当前任务的目标，这样它就能围绕最终结果（Outcome）来保持方向，而不仅仅是执行命令。例如，我按下 Command+N 新建任务，输入 `/goal`，就会看到目标指示。我可以让它：

> 我希望你最终创建30个iOS应用，为30个不同的iOS应用编写代码。首先，进行一个研究步骤，研究30个iOS应用。我希望这些应用能下载到我的手机上。请给我发送构建链接，并且可以使用 @porous iOS 技能。
现在它正在运行，任务被设定为目标。这是一种不同的模式，Codex 知道它需要工作更长时间，不仅仅关注指令，而是始终对齐目标，它会做更多规划，并且不会停下来直到达成目标。我已经看到有人运行这个功能长达 **4小时**，甚至有人在Twitter上发帖说，他的一次深度编码任务运行了 **1天零14小时**。

### 2. 插件分享

下一个大更新是：你现在可以在 Codex 中分享插件了。

从 OpenAI 开发者账号的推文我们看到：

> 今天我们发布了 Codex 中的**插件分享**功能，让与你一起工作的同事可以轻松共享插件。例如，我构建了一个“First Week Quest”插件。在插件目录中，我可以找到这个插件，查看默认提示，然后点击分享按钮，弹出一个窗口让我指定要分享的团队成员，或者也可以设置为工作区任何人都能看见。保存后，我就把这个插件分享给了工作区的所有人。我还可以复制链接发给团队，他们就能在自己的目录中看到这个插件。反过来也一样，在“分享给你”的标签下，我可以看到团队成员分享给我的所有插件，比如一个叫“finalized code”的插件，在我完成PR之前非常有用。
在 Codex 中，你可以在“插件”标签下看到所有不同的插件，它们本质上是由一系列应用、技能和其他文件组合而成的包。你可以直接让 Codex 创建一个插件，比如“嘿，请为我创建一个做 X、Y、Z 的插件，它应该与这个API交互并执行相应操作。” 或者输入 `/plugin creator`，让它引导你完成创建插件的步骤。创建完成后，就可以分享给团队，所有被分享给你的插件都会显示在一个页面中，非常酷。

### 3. 全设计模式（Annotations & Screenshots）

另一个让 Codex 更接近完整“氛围编程（Vibe Coding）”平台的更新是 **全设计模式（Full Design Mode）**。以前你需要用 Replit、Lovable 或 Bolt 才能直接在应用内获得类似设计模式的功能，而现在 Codex——并且可能很快 Claude Code——正在成为完整的 Vibe Coding 平台。

具体操作是：在 Codex 的浏览器中，你可以点击上方的“注释”按钮，对页面元素进行标注。你可以添加注释，也可以点击设置按钮直接修改文本颜色、字号、字重等。但当前这个功能并不是像 Cursor 那样直接编辑代码，而是通过将操作添加为注释，然后让AI去执行这些注释。例如，我点击某段文字，选择“增大字号”，它会创建一个注释。然后我可以告诉它执行这些注释，AI就会据此修改。你也可以直接进行截图，把屏幕截图添加到上下文中，然后对截图添加注释。总之，你拥有两个工具：**注释功能**和**截图功能**。屏幕上的任何内容，你都可以立即截图并自动复制到剪贴板，然后添加上下文。

### 4. Appshots：全局快捷键操作

OpenAI 针对 Codex 还推出了一项我觉得最好用的功能之一，那就是 **Appshots**。

你只需要同时按下 **两个 Command 键**，会立刻截取你当前所在的应用程序（无论是浏览器、邮件客户端、Google Docs等）并带回 Codex。例如：

- 当我在查看网页时，按两次 Command 键，Codex 截取了网站，我可以说“请告诉我关于这些内容的信息，做一些研究。”
- 当我在 **Superhuman**（邮件客户端）中时，按两次 Command 键，我可以说“请研究这家公司并起草回复邮件。” 它就可以直接控制我的电脑在后台完成。
- 当我有一个 **Google Docs** 集成时，我可以打开一个文档，按两次 Command 键，然后说“Codex，请在这里写一段四段的文章。” 因为 Codex 知道这个文档，并且有 Google Docs 技能，它会直接使用计算机操作（Computer Use）在这个 Google Docs 里直接输入内容。返回 Codex 可以看到它正在文档中直接打字。
无论你正在使用什么应用程序，按下 Command+Command 就可以给 Codex 提供上下文，让它执行你需要的任务，并在后台控制你的电脑。这就是超级应用的未来。

## Google / Gemini 更新

### 1. Google IO 总体观感

接下来是平台 **Google / Gemini**。本周谷歌举行了他们的年度大会 **Google IO**，发布了大量内容。从 AI 的角度来看，我认为这次大会基本上是“无足轻重（nothing burger）”的。除了**一件事情**让我无法停止思考——**Gemini Spark**（稍后细说）——其他几乎没什么亮点。整个大会他们不断提到 AI 代理，但我最大的问题是：**我不知道他们的“超级应用”是什么？**

我向你们展示了 Codex 和 Claude 桌面应用，它们分别是 OpenAI 和 Anthropic 明确的代理平台。如果你想用 AI 作为高级用户来完成任务、实现长期目标、连接所有工具，你很清楚该去哪个平台。但 Google 让人困惑：如果你要做代理任务，应该用 **Anti-gravity**（他们的编码代理平台）？还是用 **Gemini**（现在多了一个叫 **Spark** 的东西，类似 Manis 或 Perplexity Computer）？或者用 **Google AI Studio**？还是说它只用于构建应用？

我的核心观点是：所有类型的任务（代理任务、应用构建、营销、自动化），人们会希望从**单一平台**完成。他们希望所有技能、集成都在一个平台中，而 **Web 浏览器**就是提供界面的那个工具。因为这些平台（比如 Codex）自带一个完整功能的浏览器，根据任务的不同，你可以在这个浏览器中打开任何界面，或者让AI通过 Browser Use 控制浏览器。我认为这就是未来的范式。但 Google 让人不知该选哪款超级应用。

### 2. Gemini Spark：潜力与困惑

我认为 **Spark** 的宣布实际上很有意思。我们看到的这段视频展示了一个集成在 Gemini 中的 Spark，它看起来有点像 Claude 桌面应用中 **Chat** 和 **Co-work** 的区分，现在变成了 **Chat** 和 **Spark**。

这个 App（Chat + Spark）有成为超级应用的潜力，因为它可以：

- 添加文件夹，控制你电脑上的文件夹；
- 启动虚拟计算机，在云端执行代理任务；
- 拥有各种连接；
- 显示代理的执行轨迹，能够长时间思考；
- 连接到所有 Google 应用和其他工具；
- 拥有 Canvas、深度研究、引导式学习、创作音乐、视频、图像等功能；
- 可以使用 **NotebookLM**；
- 可以搜索你的 Google Drive 和照片。
潜力巨大。但为什么我失去了兴趣？因为我不信任 Google 保持足够专注。他们同时还在更新 Anti-gravity（AI 开发者平台）、AI Studio（他们的 Vibe Coding 平台，类似 Lovable）。我的想法是：

- 作为代理平台，它不会比 Codex 更好；
- 作为 Vibe Coding 工具，它不会比 Lovable 或 Replit 更好；
- 作为真正的工程工作平台，它不会比 Cursor 或 Anti-gravity 更好（而 Anti-gravity 本身也不是最顶尖的）。
这就是 Google 和 Google DeepMind 目前的故事：作为 AI 的开拓者，**他们没有任何东西是行业最好的**。

- 视频模型不如 **Seed Dance**（字节跳动）；
- 图像生成不如 **ChatGPT Image 2**；
- 他们的 LLM 远未达到前沿水平；
- Vibe Coding 工具不是最好的；
- 聊天体验远不如 Claude 的标准聊天（我认为 Claude 是最好的聊天应用）。
他们创造的东西没有一样是最先进的。你必须非常深入挖掘才能找到亮点：**视频分析**是 Gemini 应用中的一个模态，你可以上传视频进行分析——这可能是他们唯一最擅长的事情。除此之外，我希望他们能发布“红色警报（code red）”，把所有注意力集中在其中一个应用上。也许就是 Gemini 应用，集中精力优化 Chat 和 Spark。谁知道呢？我们拭目以待。

### 3. 新模型 Gemini 3.5 Flash 与 Google 现状

说到“不是最好”，Google 的最后一个更新是他们的新模型——**Gemini 3.5 Flash**。大家都在期待 **Gemini 3.5 Pro**（他们的前沿模型），但看起来它实际上并不前沿。在创造强大的代理模型方面，Google 似乎明显落后于 OpenAI（如 5.5 模型）和 Anthropic（如 Opus）。

如果你有兴趣，可以去看看 Theo 的视频，他对 Google 讲了很多负面评价。我不打算赞同或反对，只是我跟聊过的人都说这个模型不值得测试——它很快但很贵，而且不如 **Cursor 的 Composer 2.5**（我们稍后会讲到）。所以，在 AI 前沿方面，Google 这次的表现确实令人失望。

他们还发布了一个新的视频模型叫 **Gemini Omni**，但生成的视频质量不如中国公司字节跳动的 **Seed Dance**。再次印证：他们什么都不拔尖。

## Cursor 更新

### 1. 新模型：Composer 2.5

首先，Cursor 发布了全新的模型。大家都知道，xAI / SpaceX 宣布将以 **600亿美元**（附带100亿美元的退出条款）收购 Cursor。这使得 Cursor 获得了更多的计算资源，并且能够训练出全新的 **Cursor 2.5**，同时更大的模型 **Composer 3** 正在训练中，据说会更加强大。

我可以告诉你，这个模型真的非常好。我非常喜欢用 Cursor，它的内置浏览器甚至比 Codex 还要好。如果你有一段时间没用 Cursor，它现在看起来很像 Codex：左侧是工作区，可以添加新项目，打开文件夹。比如我新建一个项目，说“请创建一个类似 Linktree 的漂亮着陆页”，然后使用全新的 **Composer 2.5** 模型。这个模型消耗的 token 很少，用起来非常便宜，你可能用一个小时都花不了一美元。因此它极快，而且充满乐趣。它还拥有和 Codex 一样的完整浏览器，可以搜索任意网站。

在我下达指令后，模型很快完成了一个着陆页，并识别了我的身份（Riley Brown）。然后我让它“请按照 Cursor 的风格进行美化”。它比 Codex 甚至 Claude 快得多，还可以上网搜索。这个模型对于前端开发来说，已经非常接近前沿水平。它能快速生成代码并修改，刷新页面后，风格已经变成 Cursor 的品牌风——效果很不错。

### 2. 收购与自主平台愿景

Cursor 有一个全新的模型，速度更快。我还没有制作关于 Cursor 的完整视频，但可能会出一期全面教程。它跟 Codex 非常相似，只是目前还不能像 Codex 或 Claude Co-work 那样自然地在侧边栏打开文档、幻灯片、电子表格。但我猜测这个功能很快就会到来，因为 **xAI** 在宣布收购 Cursor 的声明中表示：“我们正在构建下一代**编码和知识工作平台**。” 我知道他们希望进入 **AI 智能体在企业中应用**这个市场。Cursor 已经是一个完整的超级应用竞争者，我将密切关注其发展。他们正在变成全功能的超级应用，这意味着他们将与 Codex 和 Claude Code 实现功能对等。

目前 Cursor 不仅拥有包含所有集成的市场（类似 Codex 和 Claude Code），现在还加入了 **自动化（Automations）**。

### 3. 自动化与集成

在 Cursor 界面中，你不仅可以看到市场（Marketplace），还有我提到的新的自动化功能。这标志着它正加速成为一个全功能平台。

## 项目推荐：Chorus — 将 AI 代理带入 iMessage

顺便，如果你想支持我，并且看看我和团队正在做的项目——我们找到了一种方法，将 **Claude Code** 和 **Codex** 加入 **iMessage**。

我们购买了域名 **chorus.com**，这是我的 AI 代理“James Bond”。我可以直接说：“嘿，James，你在做什么？” 现在我在和 **Claude Code** 聊天，很快你也能和 **Codex** 对话。
