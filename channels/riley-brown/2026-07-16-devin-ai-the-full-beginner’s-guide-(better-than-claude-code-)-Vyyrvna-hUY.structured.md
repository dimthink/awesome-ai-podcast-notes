# Devin AI 完全指南：无需编写代码创建专业登陆页、线索磁铁与营销仪表板

## 引言与概述

本文将完整介绍如何使用 **Devin** —— 一款能够调用全球所有前沿模型的 AI 代码编辑器。借助 Devin，你可以创建极为专业的网站、移动应用、演示文稿以及许多其他对营销有用的内容。今天，我们将使用 Devin 构建以下三样东西：

1. **一个精心设计的个人登陆页面。**
2. **一个线索磁铁（Lead Magnet）**：允许我们从发布的内容中收集潜在客户信息。
3. **一个仪表板（Dashboard）**：供整个营销团队日后使用。

整个过程中，我们**不需要编写任何一行代码**。

本指南将展示如何一步步实现上述目标。真正的精彩部分在于：我们会先创建一个个人登陆页面，然后为它关联一个线索磁铁（通常是你免费提供给受众、用以获取其联系信息的东西）。对于任何服务型或信息型业务来说，线索磁铁都是捕获潜在客户、为后续销售作准备的绝佳工具。接着，我们还会把线索磁铁收集到的提交数据转化成一个可供整个营销团队使用的仪表板。

## 什么是 Devin？

**Devin** 是一个平台，你只需要输入想要的指令，AI 就会自动为你构建整个网站或应用。例如，我可以输入“为 Riley 制作一个登陆页，搜索 Rileybrown.xyz（我的网站），然后做一个更好的版本”。

它更有趣的功能在于 **会话标签页（Sessions Tab）**。这里像一块看板（Kanban Board），展示所有正在运行的不同的 AI 代理（Agents）。你可以同时运行不限数量的 AI 代理，各自构建你业务所需的东西。这就是 Devin 的核心本质——它不仅可以让你命令一个代理构建应用，还可以指挥一群代理同时处理多个应用上的多项任务。你可以让多个代理协同工作于同一个应用，也可以让每个代理负责一个独立的应用。本质上，你就是在管理这些代理，为你的业务创造一切所需。

## 开始使用 Devin：下载与安装

### 下载 Devin

1. 打开一个新标签页，访问 **devin.ai**。
2. 点击 **下载** 按钮，根据你的电脑系统选择对应版本（作者使用 MacBook Pro M4，Windows 及其他电脑同样支持）。
3. 安装完成后，启动 Devin。

### 连接 GitHub 账户

登录 Devin 时，你会看到 **用 GitHub 登录** 的选项。**强烈建议创建一个 GitHub 账户并将其与 Devin 关联**（首次注册 GitHub 完全免费）。登录后，你就可以开始使用了。

### 界面介绍：编辑模式与代理模式

Devin 提供两种视图：

- **编辑器视图（Editor）**：适合高级开发者，可对代码文件进行更精细的控制。
- **代理视图（Agents）**：适合像作者一样从未写过一行代码的用户。作者本人从未写过代码，却通过 Devin 管理着 5 个日常应用、2 个拥有付费用户的应用，并在 GitHub 上创建了 139 个仓库（项目）。

在代理视图下，界面非常简洁：左侧是聊天会话列表（即所有你与之对话的代理），右侧是你可以输入指令的区域。你可以在这里输入任何应用创意，并在上方选择要使用的模型。Devin 提供了多种模型选择，例如内置的 SW 1.7、Claude Fable 5、GPT 5.6 Luna，以及一些免费的开源模型（比如 GLM 5.2 High——这可能是注册赠品）。

## 创建第一个应用：个人登陆页面

一切就绪后，我们就可以开始构建应用了。

### 下达第一个指令

作者对 Devin 中的某个代理输入了以下指令：

> “请为我（Riley Brown）创建一个非常漂亮的个人登陆页面。先在互联网上搜索我的信息，然后创建一个漂亮的登陆页面。我们将在接下来的几分钟内不断完善它，让它变得非常棒。请构建它，然后本地运行。”

### 本地预览与代码界面

Devin 很快完成了任务。你可能注意到右侧有大量代码，但**完全不必关注它**，忽略即可。我们真正需要看的是 **本地预览（Local Preview）**，它显示着正在你电脑上本地运行的应用。点击预览，浏览器会打开你刚刚创建的网站——但请注意：**这只是一个本地站点，只有你自己能看见**，他人无法访问。

### 连接 Vercel 准备部署

要将网站真正发布到互联网上，我们可以使用多种服务。作者推荐 **Vercel**，因为 AI 代理非常擅长使用它。连接方法如下：

1. 回到 Devin，点击底部的 **Open MCP Marketplace**（MCP 市场）。
2. 找到 **Vercel** 选项，点击 **连接（Connect）**。
3. 选择你要关联的 Vercel 子账户（作者选择了“添加所有项目”），然后点击 **允许**。

Vercel 的账户设置是免费的，甚至可以免费将你的站点部署（即发布）到互联网上。

## 构建线索磁铁页面

### 明确需求

作者计划发布一个名为 **Agent Tree** 的产品（类似为代理技能定制的 Linktree）。在此之前，他想先验证想法，看看有多少人感兴趣。因此，他需要创建一个登陆页面，该页面指向一个 **线索磁铁**——一份名为“如何将你的代理技能变现”的 PDF。用户需要填写姓名、邮箱等字段才能获取这份 PDF，而我们则需要将这些信息存储起来，以便后续使用。

### 调整登陆页面风格

作者提前用 AI 处理了自己的照片，并准备了两张图片放在下载文件夹中：一张是本人的无背景照片（用于网站），另一张作为风格参考图。然后他通过 Devin 下达了详细的 prompt：

> “请完全改变这个登陆页面的风格。我的下载文件夹中有两张图片：一张是本人无背景照（用于网站），另一张作为风格参考图（只需参考大体风格，不要照搬）。此网站的目的是推动人们获取线索磁铁，所以请先创建一个非常有说服力的登陆页面，并包含一个指向线索磁铁的按钮。**不要**创建线索磁铁页面本身，留待下一步。请更新当前在本地运行的应用。”

**注意**：作者将模型优先级调至“高”，并使用了最佳的 GPT 模型（接近 Fable 的水平）。Devin 开始工作，预计需要 3-5 分钟。此时可以在会话标签页中看到有一个代理正在运行。

### 页面生成结果与修改

几分钟后，页面生成完毕。效果如下：

- 标题：“Riley Brown, AI Agents for Creators, Get the Guide.”
- “Get the Guide”按钮点击后会返回首页（因为我们尚未创建线索磁铁页面）。
- 文案：“Send me the playbook. Monetize your AI agents. Learn how to turn the AI agents you’re already building into product services and recurring revenue. Your audience doesn’t need more information. They need your help. And a product that you can sell.”

作者对这个页面基本满意，但觉得字体颜色难看（一种奇怪的棕色），于是截图指出并让 Devin 修复。同时，他还想替换其中一张照片，并希望将某个部分的背景色改为浅蓝色，作为整个网站的强调色。他依次将这些新 prompt 加入队列，Devin 会逐一执行。

### 创建线索磁铁页面（PDF 页面）

紧接着，作者让 Devin 创建 PDF 页面本身：

> “请制作一个非常酷的 PDF 页面。包含所有需要填写的字段。暂时不需要连接到 Supabase，稍后我会处理。先让当用户点击按钮时能跳转到这个页面。”

Devin 内置的浏览器很好用，窄屏和宽屏下显示效果都不错。最终生成的 PDF 页面让作者眼前一亮，背景样式非常出色。

## 连接 Supabase 存储数据

要让所有表单提交数据被保存下来，我们需要使用 **Supabase**（一个后端即服务平台，同样可以免费起步）。操作如下：

1. 回到 Devin 底部的 MCP 市场，找到 **Supabase**，点击安装。
2. 前往 Supabase 官网，进入仪表板 → 账户 → **Tokens（令牌）** 页面（链接见视频描述），**生成一个新 Token** 并复制。
3. 回到 Devin，将 Token 粘贴到弹窗中，点击 **添加**。Devin 会自动安装并集成所有 Supabase 工具。

### 将表单数据存入数据库

接着作者向 Devin 下达新的指令：

> “请将所有提交数据使用 Supabase 存入数据库。我希望稍后能创建一个密码保护的管理仪表板。请在页面底部添加一个仪表板链接，预设密码为 **3694**，这个密码要安全地存储在数据库中。现在先只存储数据，仪表板内容留空。稍后我会指示你填充。”

由于之前已经成功连接 Supabase，Devin 可以轻松调用数据库工具创建存储。测试如下：

- 作者在网站中填写信息：名字“Riley Brown”、邮箱“...”、总粉丝数“10,000”、最喜欢的平台“TikTok”，点击“Send me the guide”。
- 显示“You’re on the list”。
- 回到 Supabase 的 **Table Editor**，进入名为“Devon”的项目下的表“guide leads”，发现刚刚填写的数据已经成功存入。

这证明数据库连接和数据存储完全正常。

## 部署到互联网：GitHub + Vercel

现在应用已经完成，但仍在本地运行（localhost），他人无法访问。我们需要将其部署到公网。

### 推送到 GitHub

作者向 Devin 发出指令：

> “请在 GitHub 上创建一个新的公开仓库，命名为 **Riley-land-lead-dash**。然后将这个项目推送上去。”

GitHub 可以理解为开发者的 Google Drive，用来备份代码。上传后，每当在本地更新项目并推送到 GitHub，Vercel 会自动重新部署，保持网站更新。

### 部署到 Vercel

接着作者要求：

> “将仓库部署到 Vercel，让它上线到互联网。”

Devin 自动完成了部署，项目在 Vercel 上获得一个公开 URL：**Riley-land-lead-dash.vercel.app**。作者打开该链接测试，所有功能正常工作：可以浏览登录页面、提交表单、下载 PDF、进入仪表板（密码 3694），且仪表板数据显示实时。**这个网站现在是真正的公开网站了**。
