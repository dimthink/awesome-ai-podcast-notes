# 2026年AI编程初学者指南：从零构建网页、桌面与移动应用

## 引言：什么是“Vibe Coding”？

使用AI代理来构建你想要的任何应用程序，这被称为**Vibe Coding**。Vibe Coding是2026年人工智能领域中最令人兴奋、发展最迅速的方向之一。这些AI编程模型每个月的性能都在变得更强，初学者们正越来越多地利用AI来编写几乎所有的代码，从而构建出越来越复杂的应用程序。

如果你正在旁观这一切，可能会感觉自己正在落后。但事实是，只要你专注地学习几个小时，Vibe Coding是非常容易理解的。

最棒的是，如果你有一个**ChatGPT订阅**，你现在就可以使用世界上最好的初学者编程工具和模型：**Codex**和**GPT-5.5**。即使你没有任何编程经验，这也将是你看到的关于Vibe Coding最全面的视频。

在本视频中，我们将构建三种类型的应用程序：

- 一个**网页应用 (Web App)**
- 一个**移动应用 (Mobile App)**

我们将在构建过程中学习进行高质量Vibe Coding所需的关键概念。

## 第一部分：初识 Codex

### 1. 下载与安装

这个界面就是**Codex**，作者进行所有Vibe Coding工作的地方。要下载Codex，你需要：

1. 打开任何一个浏览器。
1. 在地址栏输入：`openai.com/codex`。
1. 页面加载后，点击下载适用于 **Mac OS** 的版本。
1. 按照步骤完成安装。
当你启动Codex后，就会看到它的主界面。恭喜你，一旦Codex在你的电脑上安装并启动，你就拥有了使用世界上最好的AI编程工具来创建应用程序的能力，例如网页应用、桌面应用和移动应用。

在本视频中，我们将创建一个网页应用，然后把它分别转换成一个桌面应用和一个移动应用。

### 2. 核心概念与操作

在开始之前，我们需要了解一些在Codex上进行Vibe Coding的基础知识和必要词汇。

基本上，你只需要输入你的应用想法，它就会为你创建整个应用程序，然后你就可以开始使用它了。

- **新建项目 (New Project)**：在Codex的侧边栏，你应该创建一个新项目。你可以按下“**Start from scratch**”（从零开始），或者选择你电脑上想要存放应用代码的文件夹位置。
- **项目文件夹**：你的电脑有文件系统。例如，在Mac上，你可以将项目放在“文稿 (Documents)”文件夹中。新建一个文件夹（例如命名为 `simple app`）并打开它。项目所在的文件路径会显示在界面上。
- **创建聊天 (New Chat)**：在项目内，你可以按下按钮创建一个新的聊天窗口，然后就可以输入指令了。
- **应用的实质**：一个应用本质上就是一个包含代码文件的文件夹，这些文件运行起来就是软件。在项目文件夹没有文件时，你是无法创建应用的，这时就需要用到 **GPT-5.5** 这个AI模型来生成构成应用的文件。
### 3. 动手实践：创建一个“微软画图”网页应用

作者用一个例子演示了全过程：输入提示词“**I want you to create a Microsoft Paint app simple web app then run it locally.**”（我要你创建一个简单的微软画图网页应用，并在本地运行它。）

AI编程模型GPT-5.5会开始为这个应用编写所有的代码文件。大约5分44秒后，它响应并构建完成了一个简单的React Vite画图应用，并在本地启动了它。

**应用效果**：

- 界面中出现了一个名为“Paint Website”的组件，点击“Open”即可打开。
- 你可以在上面直接绘画，测试画笔、颜料桶和颜色选择器等功能。
- 一个简单的“微软画图”程序就这样通过单个提示词创建出来了。
**迭代修改**：

你可以继续提出修改要求，例如：“**Please make this look like Apple instead of Microsoft in the top and bottom panel. Please allow me to use undo.**”（请让顶部和底部面板看起来像苹果风格，而不是微软风格。请允许我使用撤销功能。）

当你提出修改时，AI代理会更新项目文件夹中的文件。即使你从未写过一行代码，也无需关心这些文件具体是做什么的，因为AI在编程方面已经足够优秀。你只需要知道，你的应用就存在于这些文件中。当它运行时，是在你的电脑本地运行的，这让你可以预览应用。你可以在Codex中通过一个按钮来切换预览。

### 4. 使用 GitHub 保存你的工作

如果你想将工作保存到云端，以便在另一台电脑上继续工作，或者发送链接给他人合作，该怎么做？答案是 **GitHub**。

即使你从未写过代码，也不要觉得GitHub令人生畏，因为AI代理可以处理一切。

**步骤**：

1. **创建GitHub账户**。
1. **创建新的仓库 (Repository)**：建议设置为“**Private**”（私有），这样更安全。为仓库命名，例如 `Paint by Riley`。你可以把这个仓库想象成Google Drive里的一个新文件夹。
1. 回到Codex，对AI说：“**Hey, I like this app. I want to upload it to GitHub so I can save my work. I created a new repo. Please upload the code here.**”（嘿，我喜欢这个应用。我想把它上传到GitHub以便保存我的工作。我创建了一个新仓库，请把代码上传到这里。）然后粘贴你的GitHub仓库链接。
1. 如果你之前没有在电脑上设置过GitHub，可以要求Codex：“**I want to set up GitHub**”（我想设置GitHub）。它会引导你完成身份验证。
**“提交 (Commit)” 的概念**：

当你对应用做了修改（例如“把画图顶栏的标题和左边的三个点变大”），这些修改只存在于你电脑的本地文件夹中，GitHub上的保存版本并未改变。要让云端版本同步，你需要对AI说：“**Please commit the changes to GitHub**”（请将更改提交到GitHub）。这就像点击Google Drive的“更新”按钮，是保存工作进度的方式。

**小结**：

到目前为止，我们所做的全部工作是：

- 使用桌面应用Codex来构建一个**网页应用**（可以在浏览器中打开）。
- 过程是：在Codex中输入 **提示词** → **AI代理**（运行GPT-5.5模型）**编辑、创建、删除文件** → 文件位于电脑上的**项目文件夹**中 → 在电脑上**本地运行**并**预览应用** → 根据预览效果再次**提示修改**（形成一个循环）。
- 为了保存和分享工作，我们使用了 GitHub，创建了一个**仓库**（云端文件夹），并将代码上传。

## 第二部分：构建一个复杂的网页应用 —— “共享大脑”

现在，是时候构建一个更复杂的应用程序了。我们从网页应用开始。

### 1. 应用构想：“Shared Brain”

作者的应用构想是一个名为 **“共享大脑 (Shared Brain)”** 的内部网页应用，用于管理视频创意。

**核心需求**：

- 一个美观的内部网页应用，作为团队共享的视频创意“第二大脑”。
- 团队成员可以保存任何可能成为未来内容的东西：推文、油管视频、Ins帖子、文章、截图、反应、参考、灵感片段或零散想法。
- 所有内容都应存放在一个视觉化的资料库中，感觉像在使用自己的大脑一样。
- **自动拉取元数据**：保存的每个项目都应自动获取最佳的可用元数据（例如，保存推特链接时显示卡片），使其更具视觉性。
- **支持分类**：应用应支持类别，类似于Notion，可以创建不同的分类。
- **包含日期字段**：每个创意应有一个可以手动更改的“归档日期”。
- **为人与代理共同设计**：这是一个内部工具，作者希望自己的AI代理或任何正在使用的AI代理都能够更新和读取这个数据库。例如，AI代理可以浏览数据库，找到最适合制作反应视频的内容。
- **包含“代理技能”**：在用户个人资料中，应有一个可复制的技能或指令块，可以提供给任何代理（如Codex或Claude Code）。该技能应解释如何访问和操作这个“第二大脑”，并包含所需的密码或访问令牌，以确保只有拥有凭证的代理才能使用该系统。
- **设计感觉**：应用应感觉高级且平静，不像一个通用的管理后台。应该像一个私有的、用于视觉化/视频创意的创意操作系统。视觉化、可搜索、快速、组织有序，足以让团队日常使用。
作者将这些想法总结为：“**这是一个共享的视觉化第二大脑**”，并将其整理成一个提示词，要求AI在项目文件夹中创建一个名为 `my`*`idea.md`** 的Markdown文件来存储这个构想。*

### 2. 设置后端：Firebase (身份验证、数据库、存储)

在让AI构建整个应用之前，必须先设置好身份验证、数据库和存储。如果浏览器刷新，之前做的“微软画图”应用数据会消失，因为它只临时存储在浏览器中。对于真正的网页应用，需要将所有数据存储在一个数据库中。这里选择使用 **Firebase**。

Firebase将用于三件事：

1. **身份验证 (Authentication)**：让用户可以登录。
1. **数据库 (Database)**：存储所有数据（如用户名、类别等）。
1. **存储桶 (Storage Bucket)**：存储图片、视频等文件。
**设置步骤 (在 Firebase 控制台完成，这是唯一需要离开 Codex 的步骤)**：

1. 访问 `firebase.google.com`，点击“Get started in console”，然后创建或获取一个新项目。
1. 为项目命名（例如 `shared space`），禁用 Gemini 和 Google Analytics，然后继续。
1. 项目创建成功后，添加一个 **Web App**。
1. 初始化 **Firestore 数据库**：点击“Create database”，选择“Start in production mode”，完成创建。
1. 设置 **Storage**：需要升级项目（可能会产生小额费用，可以设置预算上限），然后创建存储桶。
1. 设置 **Authentication (身份验证)**：启用“**Google**”作为登录提供商，并选择支持邮箱。
完成以上步骤后，你将把Firebase密钥提供给Codex，让它知道要连接哪个项目，并且已经启用了身份验证、存储和数据库功能。现在，Codex可以与你整个Firebase项目进行通信。

### 3. AI构建应用与迭代调试

回到Codex，输入提示词，要求AI使用Firebase构建整个应用，实现计划中的所有功能。

经过约25分51秒的构建，AI给出了一个本地预览地址。但首次测试时，登录和保存功能遇到了权限问题。

**调试技巧**：

- 在浏览器中右键点击，选择“**Inspect**”（检查），打开开发者工具。
- 进入“**Console**”（控制台）面板，复制所有错误日志。
- 回到Codex，将错误日志粘贴给AI，并描述问题（例如：“我可以登录，但尝试保存链接时失败，提示权限不足”）。
- AI会根据错误信息进行修复。
经过修复，应用可以正常保存数据到Firebase了。你可以在Firebase控制台的“Authentication”标签页看到注册用户，在“Firestore Database”中看到用户保存的项目数据。

### 4. 进一步优化和添加 AI 标题生成功能

**优化元数据拉取和布局**：

对于推特（X）的帖子，作者不喜欢蓝色的圆圈；对于YouTube视频，缩略图没有正确加载。同时，保存在网格中的项目没有并排显示。作者将这个反馈和需求告知Codex，要求它研究“My Mind”等应用是如何拉取相关数据的，并对X链接和YouTube链接的显示进行优化，同时解决网格布局问题。AI很快完成了修复。

**添加 AI 自动生成标题**：

保存的项日标题（特别是截图）格式混乱。作者希望使用一个轻量级AI模型，通过OpenAI API来分析任何类型的数据，并快速生成一个更好的标题。要求标题简洁（少于6个词），如果描述或截图中有清晰的标题，应使用最保守、最简单、字面意义的标题。

**设置 OpenAI API**：

1. 为了在应用中使用ChatGPT技术，需要申请一个 **API Key**。
1. 访问 OpenAI 平台，登录后点击“API Keys”，创建一个新的密钥。
1. 将生成的密钥复制到Codex中，告诉AI这是你的API Key，并要求它分析截图并生成标题。
AI修复后，应用中的YouTube视频和X帖子的显示都得到了改善，并且会自动生成更准确的标题。

### 5. 改进用户界面和过滤器

作者继续通过截图和语音向Codex提出修改要求：

- **界面优化**：更换字体；将背景改为更灰的颜色，并添加从灰色到稍深灰色的微渐变；为卡片添加轻微的边框，让应用更有质感。

AI根据这些要求持续迭代，最终的应用看起来已经很不错了。

## 第三部分：从网页应用到桌面应用和移动应用

### 1. 制定转换计划

如何将一个网页应用转换成桌面应用和移动应用？作者直接在Codex中提出了需求：

- 使用 **Electron** 将网页应用转换为桌面应用。
- 使用 **Swift** 开发iOS移动应用。
- 要求所有新应用与原来的后端通信，共享数据库、存储和身份验证，确保团队可以在任何平台上使用。
- 要求AI制定一个详细的计划，并生成一个Markdown计划文件。
AI建议所有项目（网页、桌面、iOS）都放在同一个项目文件夹中，创建一个 `apps` 目录来分别存放。

### 2. 生成桌面应用 (Electron)

作者要求AI根据计划，将三个应用都制作完善。经过约30分钟的工作，AI完成了任务。

当要求AI运行桌面应用时，一个独立的桌面应用窗口弹出了。它甚至在电脑上创建了一个名为“Shared Brain”的应用图标。

- 应用可以全屏，但最初窗口无法移动（作者随即要求AI修复）。
- 登录后，桌面应用的功能与网页版几乎完全相同，并且数据是同步的。
- 这个桌面应用在本地运行，是一个真正的Electron应用。
### 3. 生成iOS移动应用 (Swift/Xcode)

要运行iOS应用，需要在Mac上安装 **Xcode**（苹果官方的开发者平台）。Xcode自带 **Simulator**（模拟器），可以在电脑上模拟运行iOS应用。

AI自动开始构建模拟器版本，处理Firebase和Google登录的依赖，编译原生应用，并启动一个iPhone模拟器。

模拟器成功运行了“Shared Brain”应用。虽然初次登录遇到了“钥匙串错误”，但将错误信息提供给AI后，问题迅速被修复。

**结果**：

- 一个网页应用，拥有后端数据库和存储。
- 在不到7个提示词内，这个网页应用被成功转换成了一个功能完美的**桌面应用 (Electron应用)**，可以直接下载到电脑上使用。
- 同时，也成功创建了一个**iOS应用**，可以在Xcode的模拟器中运行。
作者惊叹，基本上在一个提示词中就完成了98%的工作。

### 4. 迭代iOS应用与代理功能演示

**迭代iOS应用**：

作者对iOS应用的界面提出了修改要求（例如：简化详情页、将添加按钮改为右下角的悬浮大按钮），AI迅速完成了修改。

**代理功能演示**：

回到最初的应用构想——代理可以访问和操作这个数据库。作者复制了AI生成的“代理技能”指令，在一个新的聊天中告诉AI：“使用这个技能，将这些推文添加到我的共享大脑。”

AI完成后，在应用的“所有用户”视图中，这些新添加的推文确实出现了。这说明代理功能正常工作：AI代理可以代表作者向数据库中添加内容。
