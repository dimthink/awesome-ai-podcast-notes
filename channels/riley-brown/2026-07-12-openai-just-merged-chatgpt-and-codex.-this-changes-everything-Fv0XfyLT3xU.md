## 开篇引入

想象这样一个场景：你打开 ChatGPT，却发现它不再只是一个聊天窗口。左侧是浏览器标签、右侧是正在后台运行的 Agent 测试环境、底部是刚刚通过一句话生成的托管网站——你进入了 AI 操作系统。这不再是科幻设定，而是在 Riley Brown 的节目中，Ross Mike 为我们描绘的正在发生的未来。

OpenAI 刚刚完成了一次极其隐秘却影响深远的战略整合：关闭独立的 Codex 应用，将之与 ChatGPT 合并，同时推出全新的模型家族（GPT-5.6 Soul、Terra、Luna）以及一套足以颠覆我们对“工作”定义的 AI 原生操作系统。这不仅仅是一次产品更新，它正在重新定义人类在计算生态中的角色——**我们从操作员，正在走向指挥家**。

## 详细内容

### 一、超级应用的第一步：Work Tab 的“牧羊犬”效应 `[章节：工作标签与Codex定位]`

**核心观点**

Work Tab 不是为高级用户准备的，它是 OpenAI 争夺 9.83 亿普通用户的“认知入口”。高级用户请留在 Codex Tab。

**深度阐述**

Ross 以一个醒目的数据开场：“OpenAI 拥有 9.83 亿用户，比美国人口的两倍还多。但绝大多数人根本不知道 ChatGPT 除了聊天还能做什么。” 这带出一个核心问题：品牌认知陷阱。

“我身边教堂里的人会走过来跟我说，‘ChatGPT 挺酷的，但 Claude 才能真正干活啊。’” Ross 苦笑道。这句话揭示了 OpenAI 面临的深层困境——ChatGPT 被贴上了“聊天工具”的标签，尽管其底层能力的边界远不止于此。这是一个严重的品牌认知问题，也是一个巨大的教育缺口。

Work Tab 的诞生正是为了堵住这个缺口。这个标签明确写着“用于完成工作（Getting Work Done）”，它的受众不是已经熟稔 Codex 的极客，而是那 9.83 亿沉默的大多数。

> “这个下拉菜单告诉人们：‘也许我可以用它来做点知识工作。’——Ross Mike `[工作标签讨论]`

这是一种极为高明的产品策略：利用低门槛的“知识工作”场景激活海量用户，一旦用户的复杂度需求上升，自然会被吸引到 Codex 这个真正的旗舰平台。Ross 直言：“如果你已经在用 Codex，请留在 Codex。Work Tab 不是给你我的。”

**延伸思考**

这是一个标准的平台化漏斗。先让用户尝到“有用”的甜头，再引导他们进入更深度的生态。对于 OpenAI 来说，这不仅是功能扩展，更是一个市值管理策略——只有让市场看到 ChatGPT 的“超级应用”潜力，投资故事才能继续讲下去。

### 二、模型矩阵的“田忌赛马”：GPT-5.6 vs. 那个名为“Fable”的怪物 `[模型探讨：5.6家族与Fable对比]`

**核心观点**

GPT-5.6 Soul 不是 Fable 的对手，但它在成本与效率上的优势，使其成为开发者必须重视的“次旗舰”选择。

**深度阐述**

“Fable 是一个怪物。GPT-6 才是 Fable 的对手。”

Ross 在谈到模型对比时毫不含糊。OpenAI 此次发布的三款模型——5.6 Soul（旗舰）、5.6 Terra 以及 5.6 Luna——构成了一个清晰的矩阵：Luna 和 Terra 将常驻 Work Tab，处理 70% 的日常轻量任务；而 Soul 则面向开发者和重度用户。

但直接拿 5.6 Soul 与 Fable 对比是不公平的。Ross 提出了一个被很多人忽视的能力维度：**工具调用效率（Tool Call Efficiency）**。

“我当然会测试模型的智能水平，但在开发实践中，我更关心它调用工具的方式。Gemini 在纯智商测试上表现极佳，但在 Agent 场景里，它就像陷入了自我催眠一样，连续调用 15-20 个无关工具。而 GPT-5.6 极其冷酷高效，它知道你什么时候需要什么工具。”

更重要的差异在于**主动性**：

“5.5 版本特别粘人。你给了它完整的执行计划，它做完第一步会停下来问：‘我该继续第二步吗？’你明明把整件事都告诉它了！但 5.6 拿到计划后，直接一口气执行到底，像一个成熟的项目经理，把事情做完再回来汇报。”

然而，Ross 也坦率地承认，Fable 在真正的“怪兽级任务”上仍然无人能及。他分享了一个令人瞠目的经历：

> “我把 Eve 框架（Vercel 的 Agent 框架）完整移植到 Convex 上，全部测试通过，部署在 Vercel，一个完整可用的聊天应用。这通常需要我专注地投入 3-4 个月的全职工作。Fable 用了 8 小时。一个 Prompt。什么都没落下。” `[Fable能力]`

这句话背后蕴含着一种复杂的情绪：敬畏与兴奋交织，既为工具的强大感到震撼，也为过去自己日复一日的磨砺感到了某种释怀。

**延伸思考**

Ross 反复强调一个被 Kool-Aid 喝晕的人忽视的残酷现实：**补贴会结束**。目前 AI 公司普遍在烧钱补贴生态，但这一天不会永远持续。学会用 5.6 完成 70% 的日常任务，把 Fable 留给最硬核的攻坚，是在这个即将进入“成本意识”时代的生存之道。

### 三、工作流三支柱：计算机使用、通用循环与积木经济 `[工作流分解]`

计算机使用、循环机制和构建块工具，构成了这个新操作系统的三大核心能力支柱。

#### 3.1 计算机使用：从“屏幕共享”到“后台矩阵” `[计算机使用]`

**核心观点**

Codex 的计算机使用支持后台静默运行，这一特性使其与所有竞品拉开了一个维度的差距。

**深度阐述**

Ross 在分享这个功能时情绪明显高涨起来：

“这个模型太……太不可思议了。它可以在你的电脑上做各种事情——填写表单、操作浏览器、测试应用——而且它是在后台工作的。我为此专门买了一台迷你 PC，远程远程桌面进去，让 Agent 自由使用它。”

关键是“后台”二字。对比 Claude Code 的计算机使用，Ross 描述了一个极具画面感的场景：

> “当你使用 Anthropic 的计算机使用时，屏幕上会蒙上一层棕色的渐变，你的鼠标被接管了，你只能坐在那里等着它用完你的电脑。但在 Codex，我可以切换到另一个 App 标签页，看到它正在默默工作。这是完全不同的体验。它让我想买一台顶配电脑，然后启动 100 个 Agent 全力开工。这太疯狂了。” `[计算机使用对比]`

**个人感受**

Ross 的声音里带着一种发现新大陆般的兴奋。他描绘的画面——一排排微型 PC 在后台被 Agent 支配着自动运行测试、构建应用——这已经不是一个“辅助工具”，而是一个数字工厂。

更令人兴奋的是**记录与回放（Record and Replay）**技能功能：

“这是我认为计算机使用未来的终极形态——非开发者录制自己 30 分钟的操作，Agent 把这段记录转化成一个‘技能’，以后自动复现。你不需要写提示词，你只需要做一遍。”

---

#### 3.2 通用循环：开发与营销共享的底层架构 `[循环机制]`

**核心观点**

循环（Loop）不是黑客的专属，它是任何知识工作者都可以掌握的通用框架：明确定义的提示 + 反馈引擎 + 清晰的成功标准。

**深度阐述**

如果说计算机使用是手，循环就是大脑。

Ross 给出了一个极其优雅的抽象模型：

> “一次循环的公式非常简单：一个明确定义的提示（Prompt），一个反馈引擎（Feedback Engine），以及一个清晰的成功标准（Success Criteria）。只要这三个要素到位，任何行业的任何任务都可以放入循环。” `[循环公式]`

**开发者场景**：
Ross 让 Agent 构建一个新功能，然后直接告诉它：“用计算机使用测试这个功能。如果测试失败，自己修复再测。直到全部通过再回来。” Agent 自动打开浏览器、模拟用户交互、检查 UI 响应、报错、修改代码、重新测试……这是一个全自动的 QA + 开发闭环。

**营销者场景**：
Riley 接过话题，展现了非编码场景下的循环应用：

“作为创始人兼营销者，我的场景可能更让你惊讶。我们利用 Foreplay 这个 API 抓取竞品跑了好几个月的高转化广告——因为如果一家公司持续跑一个广告，它一定在赚钱。然后我们用这些高质广告作为‘反馈引擎’，让 AI 创作我们的广告脚本。AI 每写完一版，就拿它与竞品基准对比打分，持续迭代直到所有脚本都达到那个质量线。”

**深度剖析**：Riley 提出了一个尖锐的洞察——为什么营销、设计等领域被认为是 AI 不擅长的地方？因为“好”的判断标准是模糊的。

> “在编码中，代码是否通过测试、是否运行成功，这是一种确定性反馈。但设计是主观的。所以要让 AI 在这些领域工作，你必须给它一个‘北极星’——高质量的例子。你给它 20 个你最喜欢的 YouTube 缩略图，它就会形成一个关于‘什么是好’的认知。然后你让它不断迭代直到输出与这些例子神似。” `[营销循环原理]`

**延伸思考**

这揭示了知识工作的未来基础：任何输出都可以被评估，任何评估都可以被量化，任何量化都可以被循环。关键是你是否拥有那个“基准”。拥有高质量例子库的人，就是拥有“反馈引擎”的人。

---

#### 3.3 积木经济：AI 时代的乐高游戏 `[工具与构建块]`

**核心观点**

未来的开发者不再是写每一行代码的“高级工匠”，而是最了解有哪些“构建块（Building Blocks）”的架构师。

**深度阐述**

Ross 引用了 Vercel 的 Geff Rory 在那篇著名的 Naval 播客中提出的概念：构建块经济（Building Block Economy）。

“你可以看到，现在的 AI 不会从零去用 AWS 那一套繁琐的原始工具。它们默认使用 Convex 作为数据库、Eve 作为 Agent 框架、Daytona 作为沙箱、Vercel AI SDK 作为推理层——这就是我的技术栈。AI 帮我把这些积木连接起来。” `[构建块经济]`

Ross 分享了他最近在搭建的一个疯狂项目：一个多人 GTA 罪恶都市的浏览器版本。

“我 Fork 了一个 Vice City 的在线版本，然后问 Agent：‘我要多人模式。’ 它花了四小时——四小时后，我有了多人联机版。虽然还有 Bug，但你想过没有，如果不是 AI，这需要多少工程师、多少时间才能搭建出雏形？”

**延伸思考**

Riley 提出“Replet Clone”作为验证基准测试。GPT-5.6 Soul 在一个 Prompt 内利用 Daytona、Convex、Vercel AI SDK 搭建了一个完整的 Web App 生成器。它没有从零造轮子，它只是完美地选择了合适的积木。

这意味着：**真正的核心技能正在转移**。不再是“写代码”，而是“知道有什么代码积木”，以及“如何在正确的时机把它递给 Agent”。

---

#### 3.4 多线程并行：从“第一员工”到“CEO” `[多线程管理]`

**核心观点**

最被低估的功能是 Codex 可以应你的要求生成多个独立线程同时工作。这是从操作员到经理的质变。

**深度阐述**

Ross 提出了一个我个人觉得最有冲击力的观点：

“我建立了一个主线程，里面列出十个不同的 Feature。我告诉 Codex：‘请针对这十个 Feature 生成十个独立的线程，用计算机使用分别测试它们，全部通过后回来汇报。’它真的生成了十个线程！然后我去陪老婆了，让 Codex 自己干活。”

**为什么这至关重要？**

> “你必须理解上下文的重要性。一个干净的新线程，它的大脑是清醒的，上下文是清爽的。一个被 120K token 垃圾填满的旧线程，它的表现远不如新线程。所以，多线程不只是并行，它是让每个 Agent 保持在巅峰状态的方法。” `[多线程价值]`

Riley 补充了一个关键应用场景：“你甚至可以要求 Codex‘总结我今天所有线程的工作进展’。它会创建一份简洁的一页报告，并附带每个线程的链接。点开直接跳转到工作现场。这对于像我这样同时处理营销、运营、产品等多工种任务的人来说，简直是救星。”

**延伸思考**

人机协作正在经历一次“管理模式的跃迁”。过去你是一个人，用一台机器，在一个窗口里完成任务。现在，你是一个“AI 经理”，管理着 N 个 Agent，它们各自在不同的线程/窗口中为不同的目标工作。你的价值不是“做得多”，而是“决策做得对”。

### 四、从聊天机器到振动编码平台：Sites 与内置浏览器 `[Sites 功能与浏览器]`

**核心观点**

`@sites` 命令与内置浏览器的深度集成，将 ChatGPT / Codex 从“软件”升维为“操作系统”。

**深度阐述**

Riley 打开了 Codex 的新界面。全新的“Sites”标签——一个以往只针对团队用户开放的功能，现在全员可用。

“只需打 `@sites` 加上你的需求——‘帮我建一个时尚的图片分享应用’——它就会直接在 OpenAI 的服务器上生成并托管一个网站，生成一个独立链接，你可以直接分享给任何人。无需部署，无需配置。” `[Sites功能]`

而更令人震惊的是，OpenAI 正在做的是**关闭独立浏览器 Atlas 团队**。

“他们已经裁掉了 Atlas 独立浏览器团队，把全部精力集中在优化这个内置浏览器上。这意味着什么？OpenAI 不想让你离开它的生态。你可以在里面搜索、浏览、编码、部署——你不需要跳出。” `[内置浏览器战略]`

**视觉描述摘要**

Riley 的屏幕上，Agent 正在打开多个浏览器标签页。一个标签页在测试用户注册流程，另一个在检查 API 响应，第三个在运行刚刚通过 `@sites` 生成的网站预览。所有这一切都在同一个“超级应用”的视图内完成。Ross 将这种体验称为“氛围编码（Vibe Coding）”的终极形态。

**延伸思考**

“超级应用（Super App）”这个词正在从愿景变成现实。如果说微信是中国的超级应用，Codex 很可能是 AI 原生的全球超级应用。它集成了大脑（推理模型）、眼睛（内置浏览器）、双手（计算机使用）和话语权（部署能力）。你不需要“打开电脑工作”，你只需要“打开 ChatGPT”。

### 五、结语与未来：最大的鸿沟不是工具，而是教育 `[教育与未来]`

**核心观点**

工具的差距正在缩小，教育的差距正在拉大。谁能填补这个教育鸿沟，谁就拥有下一个时代最大的机会。

**深度阐述**

对话接近尾声时，两人都将话题转向了一个更深远的角度。

“现在最大的机会不是建造新工具，而是**教育**。YouTube 和 Instagram 上充斥着大量糟糕的 AI 内容，比如有人在吹嘘某个模型能省钱，但那个模型本身就很差。人们渴望真正理解工作流的指导。” —— Riley Brown `[教育机会]`

Ross 对此深表赞同。他分享了自己正在刻意训练的两项传统但却极其重要的能力：

**第一：设计审美**
“AI 生成的网站越来越同质化，但真正的美感依然稀缺。我建立了一个‘设计灵感库’，专门收集顶尖的视觉案例。随着 AI 生成的内容泛滥，能辨别什么是‘好看’，并告诉 AI 如何调整的人，极具价值。”

**第二：精准的表达力**
“模型本身越来越强。但在实际使用中你会发现，你与 AI 沟通的能力——你能否清晰地定义目标、解释上下文、设定反馈标准——才是决定输出质量的关键。你的文字质量，决定了 AI 的效能上限。”

**最精彩的洞察来自 Ross 的一句话：**

> “如今，大多数产品本质上都只是一个 Markdown 文件加几个 API 密钥。别把时间花在构建 Markdown 文件上。把你的时间花在构建真正有用的东西上。” `[产品本质]`

**延伸思考**

这句话极为锋利。它说明基础设施已经被充分简化——数据库、认证、代理框架、部署管线都已作为积木存在。AI 可以帮你把积木拼好。唯一不可替代的是：你对“什么是真正有用的”的判断力。

这就是人的价值所在。

## 精华收获

1. **构建你的循环**：无论你在哪个行业，明确“Prompt + 反馈引擎 + 成功标准”三要素，就能构建属于自己的 AI 工作流。
2. **掌控后台权限**：允许 Agent 在后台使用计算机，你才真正开始解放自己的注意力。
3. **拥抱构建块经济**：不再从零造轮子，成为最了解“乐高积木”的人。Convex、Eve、Daytona、Vercel AI SDK……这些是你必须掌握的积木。
4. **多线程经理思维**：学会委派任务。用 10 个干净的 Agent 上下文并行工作，你只做决策和验收。
