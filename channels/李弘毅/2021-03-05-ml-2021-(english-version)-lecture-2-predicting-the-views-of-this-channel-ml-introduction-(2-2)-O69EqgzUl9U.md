### 线性模型的局限性与“模型偏差” `[00:00-01:40]`

**核心观点**

线性模型因其固有的直线关系假设，在面对真实世界中复杂多变的非线性数据时显得力不从心，这种由模型结构本身带来的限制被称为“模型偏差”，它与模型中的“偏置项b”是两个截然不同的概念。

**深度阐述**

在机器学习的旅程中，我们通常从最简单的模型——线性模型开始。一个典型的线性模型可以表示为 `y = w `*` x1 + b`**，其中 **`x1`** 是输入特征，**`y`** 是预测输出，**`w`** 是权重，**`b`** 是偏置项。李弘毅老师首先抛出了一个核心问题：为什么说线性模型可能过于简单？他通过一个生动的例子来解释：假设我们正在预测视频的观看量，**`x1`** 代表前一天的观看量，**`y`** 代表第二天的观看量。线性模型强制 **`x1`** 和 **`y`** 之间存在一种直线关系：随着 **`x1`** 的增加，**`y`** 总是以固定的斜率增加。你可以通过调整 **`w`** 来改变直线的斜率，通过调整 **`b`** 来改变直线与 **`y`** 轴的截距，但无论如何调整，它始终是一条直线。这意味着，如果前一天的观看量 **`x1`** 越高，第二天的观看量 **`y`** 就会越大。*

然而，真实世界往往比这复杂得多。李老师指出，也许当 `x1` 小于某个阈值时，`y` 确实与 `x1` 成正比；但当 `x1` 超过这个阈值，例如前一天观看量已经非常高，高到可能意味着人们已经看腻了或者进入了某个饱和期，那么第二天的观看量反而可能下降。这种“先增后减”或更复杂的非线性关系，用一条简单的直线是无论如何也无法拟合的。视频中用一条“红线”形象地展示了这种复杂的非线性关系，它有多个转折点，呈现出分段线性的特征，甚至可能包含平滑的曲线。无论你如何调整线性模型中的 `w` 和 `b`，都无法生成这条“红线”。

这种由于模型本身结构（例如线性模型只能表达直线关系）所导致的，无法模拟真实世界复杂情况的限制，被李老师称为**模型偏差（Model Bias）**。他特别强调，这个“模型偏差”与线性模型中的“偏置项b”（bias term）是两个容易混淆但意义完全不同的概念。偏置项 `b` 是模型中的一个可学习参数，它决定了直线的截距；而模型偏差则是指模型在表达能力上的根本性缺陷。理解这一区别至关重要，因为它指出了我们需要超越线性模型，去构建更复杂、更灵活的函数来捕捉数据中蕴含的真实模式。

**重要原话**：

- "linear models may be too simple... no matter how you change w and b it's always a straight line." `[00:00]`
- "maybe when x1 is less than a certain value the number of views on the day before is directly proportional to the number of views on the next day but when x1 is greater than the value y goes in the opposite direction after a threshold..." `[00:30]`
- "the limitations that come from the model are called model bias... it is different from the bias b." `[01:10]`
**延伸思考**

模型偏差是机器学习中一个核心概念，它直接关系到模型的欠拟合（underfitting）问题。当模型偏差过高时，即使训练数据再多，模型也无法学到数据的真实分布，导致在训练集和测试集上表现都不佳。这促使我们思考如何设计出具有足够表达能力（低偏差）的模型，同时又要避免过度拟合（高方差）的问题。

**精华收获**

线性模型虽然简单，但其表达能力有限，无法捕捉非线性关系。这种结构性限制导致了“模型偏差”，是促使我们探索更复杂模型的根本原因。

### 构建复杂函数：从“蓝函数”到分段线性近似 `[01:40-04:40]`

**核心观点**

为了克服线性模型的局限，我们需要更复杂、更灵活的函数。李老师提出，任何复杂的分段线性曲线都可以通过一个常数项和一系列具有特定形状的“蓝函数”的叠加来近似。

**深度阐述**

既然线性模型不足以描述复杂的非线性关系，那么我们应该如何构建一个更复杂、更灵活的函数呢？李弘毅老师巧妙地引入了一个概念性的“蓝函数”来解决这个问题。他首先展示了之前提到的那条“红线”，它是一个典型的分段线性曲线（piecewise linear curve），由多个线段连接而成，具有明显的转折点。

李老师提出一个核心思想：这条“红线”可以被看作是一个常数函数加上一系列特殊的“蓝函数”的叠加。那么，这个“蓝函数”究竟长什么样？它有一个非常独特的性质：

1. 当输入值 `x` 小于某个阈值时，输出是一个常数（水平线）。
1. 当输入值 `x` 介于两个阈值之间时，输出有一个斜率（向上或向下倾斜的线段）。
1. 当输入值 `x` 大于另一个阈值时，输出又是一个常数（另一条水平线）。
简而言之，这个“蓝函数”的形状是“水平-倾斜-水平”。

**视觉/结构信息描述**：

想象一个坐标系，`x` 轴是输入，`y` 轴是输出。

- **蓝函数**：从左边看，`y` 值保持不变（水平），到达某个 `x` 值后，`y` 值开始变化（斜线），到达另一个 `x` 值后，`y` 值再次保持不变（水平）。整个函数看起来像一个平躺的“S”形或者一个“阶梯”的中间部分。
李老师接着详细解释了如何通过叠加这些“蓝函数”来构造任意分段线性曲线：

1. **确定常数项**：首先，找到“红线”与 `y` 轴的截距，将其作为我们叠加的起始常数项。
1. **叠加第一个蓝函数**：找到“红线”的第一个转折点。我们可以选择一个“蓝函数”，使其斜率的起始点与“红线”的起始点对齐，斜率的结束点与“红线”的第一个转折点对齐。并且，这个“蓝函数”的斜率要与“红线”在第一个线段上的斜率相同。通过将这个“蓝函数”与常数项相加，我们就得到了“红线”从起点到第一个转折点的部分。
1. **叠加第二个蓝函数**：接着，找到“红线”的第二个转折点。我们再选择一个“蓝函数”，使其斜率的起始点与“红线”的第一个转折点对齐，斜率的结束点与第二个转折点对齐。同样，这个“蓝函数”的斜率要与“红线”在第二个线段上的斜率相同。将其与之前的叠加结果相加，我们就得到了“红线”从第一个转折点到第二个转折点的部分。
1. **依此类推**：对于“红线”的每一个线段，我们都可以通过叠加一个相应的“蓝函数”来构造。最终，将所有这些“蓝函数”和一个常数项相加，就能精确地重构出这条分段线性“红线”。
**重要原话**：

- "this red curve can be seen as a constant function plus a bunch of blue functions like this." `[02:00]`
- "the blue function has a special property when the input value... is less than a certain threshold the output is a constant when it is larger than another threshold the output is another constant there is a slope in the middle." `[02:10]`
- "no matter what kind of piecewise linear curve i draw... you will find that these piecewise linear curves are composed of a constant term and a lot of blue functions." `[03:40]`
**延伸思考**

这个“蓝函数”的叠加思想是理解神经网络如何逼近复杂函数的关键。它告诉我们，即使是最复杂的曲线，也可以通过组合足够多的简单“基函数”来构建。这为我们从简单模型过渡到复杂模型提供了理论基础。

**精华收获**

任何分段线性函数都可以分解为一个常数项和一系列“水平-倾斜-水平”形状的“蓝函数”的线性组合。这种叠加思想是构建复杂模型的基础。

### 逼近连续曲线：从分段线性到平滑S形 `[04:40-06:50]`

**核心观点**

即使是平滑的连续曲线，也可以通过足够密集的点连接形成的分段线性曲线来近似。而之前提到的“蓝函数”，则可以通过一种名为“Sigmoid函数”的平滑S形曲线来近似，Sigmoid函数的参数 `w, b, c` 控制着其形状和位置，使其能够灵活地模拟各种“蓝函数”。

**深度阐述**

李弘毅老师进一步拓展了模型的表达能力。他提出了一个更深刻的问题：如果真实世界中的关系不是分段线性的，而是一个平滑的、连续的曲线呢？例如，视频中展示了一条带有弧度的平滑曲线。他给出了一个令人信服的答案：没关系！我们可以通过在平滑曲线上选择足够多的点，并将这些点用直线段连接起来，从而将这条平滑曲线**近似**为一个分段线性曲线。如果选择的点足够多，或者选择的点的位置足够巧妙，这条分段线性曲线就可以非常接近原始的平滑连续曲线。

**重要原话**：

- "even if it's a curve like this it doesn't matter we can choose some points on the curve and connect them transforming them into a piecewise linear curve." `[04:50]`
- "as long as i have enough blue functions i may be able to create any continuous curve." `[05:30]`
这意味着，我们今天学到了一件非常重要的事情：

1. 任何连续曲线都可以用分段线性曲线来近似。
1. 任何分段线性曲线都可以由一个常数项和一系列“蓝函数”的组合来构成。
因此，只要我们有足够多的“蓝函数”，理论上就可以近似任何连续曲线。

那么，下一个关键问题是：这个概念性的“蓝函数”在数学上应该如何表示呢？李老师指出，直接写出“蓝函数”的精确公式可能并不容易，但我们可以用一个已知的、具有类似形状的函数来**近似**它。这个函数就是**Sigmoid函数**。

**视觉/结构信息描述**：

- **Sigmoid函数**：在坐标系中，Sigmoid函数呈现出经典的“S”形曲线。当 `x` 趋近负无穷时，`y` 趋近于0；当 `x` 趋近正无穷时，`y` 趋近于一个常数 `c`。在中间部分，曲线有一个平滑的上升（或下降）过程，其斜率在中间点最大。这与“蓝函数”的“水平-倾斜-水平”形状非常相似，只是Sigmoid是平滑过渡而非尖锐转折。
Sigmoid函数的标准形式通常是 `1 / (1 + e^(-z))`，但李老师在这里给出了一个更通用的形式，包含了缩放和平移：`y = c `*` sigmoid(w `*` x1 + b)`。他详细解释了Sigmoid函数的数学表达式和其参数的作用：

- `x1`：输入值。
- `w`：权重，控制S形曲线的**陡峭程度**。`w` 越大，曲线越陡峭。
- `b`：偏置，控制S形曲线的**水平移动**，使其左右平移。
- `c`：常数，控制S形曲线的**最大高度**，即当 `x1` 趋近正无穷时，`y` 的收敛值。
通过调整 `w, b, c` 这三个参数，我们可以生成各种不同形状、不同位置和不同高度的Sigmoid函数，从而灵活地近似我们所需的各种“蓝函数”。李老师还提到，我们最初设想的“蓝函数”其实有一个更专业的名称，叫做“硬Sigmoid”（hard sigmoid），而我们现在引入的平滑S形函数是“软Sigmoid”（soft sigmoid）。由于“硬Sigmoid”的公式可能更复杂，所以我们选择用“软Sigmoid”来近似它，因为“软Sigmoid”的数学表达和性质更易于处理。

**重要原话**：

- "we can use a sigmoid function to approximate this blue function." `[05:50]`
- "if we alter the variable w the slope changes... if we alter the variable b the sigmoid function shifts horizontally... if we alter the variable c the height changes." `[07:00]`
- "this blue function is commonly known as the hard sigmoid." `[06:30]`
**延伸思考**

Sigmoid函数作为一种“激活函数”的引入，是神经网络发展史上的一个重要里程碑。它使得模型能够引入非线性，从而具备了逼近任意复杂函数的能力。理解Sigmoid的参数如何影响其形状，对于理解神经网络中权重和偏置的作用至关重要。

**精华收获**

任何连续函数都可以被分段线性函数近似，而分段线性函数又可由“蓝函数”组合而成。Sigmoid函数作为一种平滑的S形曲线，能够通过调整其参数 `w, b, c` 来灵活地近似各种“蓝函数”，从而为构建复杂模型提供了数学工具。

### 从Sigmoid叠加到神经网络的构建 `[06:50-13:30]`

**核心观点**

通过将多个具有不同参数的Sigmoid函数进行线性组合（叠加），我们可以构建出能够近似任意复杂函数的模型。当引入多个输入特征时，这种叠加结构自然地演变为一个具有输入层、隐藏层（由Sigmoid函数组成）和输出层的神经网络。

**深度阐述**

既然单个Sigmoid函数可以近似一个“蓝函数”，那么将多个Sigmoid函数叠加起来，就可以近似任何分段线性函数，进而近似任何连续函数。李老师通过一个具体的例子来展示这个过程。

**视觉/结构信息描述**：

- **Sigmoid叠加**：视频中展示了如何将四个不同形状的Sigmoid函数（每个由 `c`*`i  sigmoid(wi `*` x1 + b`*`i)`** 表示）相加，再加上一个常数项 **`b`**，最终得到一条复杂的“红线”——一个分段线性函数。*
- **多特征输入**：当输入特征不再是单一的 `x1`，而是 `x1, x2, x3` 等多个特征时，每个Sigmoid函数的输入 `(w`*`i  x1 + bi)`** 就会扩展为 **`(wi1  x1 + w`*`i2 `*` x2 + wi3  x3 + b`*`i)`，即每个Sigmoid函数会接收所有输入特征的加权和。
- **网络图示**：李老师将这种结构绘制成了一个直观的神经网络图。
李老师详细解释了如何用数学公式来表示这个过程：

1. **每个Sigmoid的输入**：对于第 `i` 个Sigmoid函数，其输入 `r`*`i`** 是所有输入特征 **`x`*`j` 的加权和加上一个偏置 `b`*`i`**：**`r`*`i = w`*`i1  x1 + wi2 `*` x2 + w`*`i3  x3 + bi`**。这里的 **`wij`** 表示第 **`j`** 个特征在第 **`i`** 个Sigmoid函数中的权重。*
1. **矩阵/向量表示**：为了更简洁地表示，李老师引入了线性代数的概念。
1. **Sigmoid激活**：向量 `r` 中的每个元素 `r`*`i`** 都会通过Sigmoid函数，得到输出 **`a`*`i`。这可以表示为 `a = sigmoid(r)`，其中 `sigmoid` 函数是逐元素应用的。
1. **最终输出**：最终的输出 `y` 是 `a` 向量中每个元素 `a`*`i`** 乘以一个系数 **`c`*`i` 的和，再加上一个总偏置 `b`。
**重要原话**：

- "the red line is the result of stacking these four blue functions." `[07:30]`
- "we use j to represent the number of features... we just replace the items in sigmoid... changing to bi plus summation over j w i j x j." `[09:00]`
- "this looks like this when it is drawn out." `[09:40]` (指代网络图)
- "r = W * x + b... a = sigmoid(r)... y = c^T * a + b." `[12:30]`
**视觉/结构信息描述**：

- **完整的网络结构图**：
李老师强调，这个模型中的所有 `w, b, c` 参数，以及最后的 `b`，都是待学习的未知参数。它们共同构成了模型的参数集合，通常用一个长向量 `theta` 来表示。这个 `theta` 包含了 `W` 矩阵中的所有权重、两个 `b` 向量中的所有偏置，以及 `c` 向量中的所有系数。

**延伸思考**

这个从Sigmoid叠加到神经网络的推导过程，是理解多层感知机（MLP）和深度学习基础的关键。它展示了如何通过简单的非线性单元（Sigmoid神经元）的组合，构建出具有强大函数逼近能力的模型。矩阵和向量的引入，不仅简化了数学表达，也为后续的并行计算和优化算法奠定了基础。

**精华收获**

通过叠加多个Sigmoid函数，我们可以构建出灵活的非线性模型。当处理多特征输入时，这种结构自然演变为一个包含输入层、隐藏层（Sigmoid神经元）和输出层的神经网络，其数学表达可通过线性代数简洁表示为 `y = b + c^T `*` sigmoid(W `*` x + b)`。

### 机器学习的第二步与第三步：损失函数与优化 `[13:30-17:20]`

**核心观点**

即使模型变得更加复杂，机器学习的第二步（定义损失函数）和第三步（优化寻找最佳参数）的核心思想保持不变。损失函数依然衡量模型预测与真实值之间的差距，而优化过程仍通过梯度下降法迭代更新参数以最小化损失。

**深度阐述**

在构建了更灵活、更复杂的模型之后，李弘毅老师将注意力转向了机器学习的另外两个核心步骤：定义损失函数（Loss Function）和优化（Optimization）。他明确指出，尽管模型结构发生了巨大变化，但这两个步骤的基本原则和方法并没有改变。

**第二步：定义损失函数**

- **概念不变**：损失函数的核心目标是衡量模型预测结果与真实标签之间的“好坏”或“差距”。如果预测结果与真实值越接近，损失就越小；反之，损失就越大。
- **符号简化**：由于新模型包含了大量的未知参数（`W` 矩阵中的权重、两个 `b` 向量中的偏置、`c` 向量中的系数），如果一一列举会非常繁琐。因此，我们用一个统一的符号 `theta` 来代表所有这些未知参数的集合。`theta` 是一个很长的向量，包含了模型中所有需要学习的参数。
- **损失函数表示**：现在，损失函数被表示为 `L(theta)`，它表示在给定一组特定参数 `theta` 值的情况下，模型的预测效果有多差。
- **计算方法**：计算损失的方法与线性模型时完全相同。给定一组 `theta` 值，我们将其代入模型，对每个输入 `x` 计算出预测的 `y` 值。然后，将这个预测 `y` 与真实的标签值进行比较，计算出误差 `e`。将所有数据点的误差累加起来（例如，使用均方误差），就得到了总的损失 `L(theta)`。
**第三步：优化**

- **目标不变**：优化的目标是找到一组参数 `theta`（我们称之为 `theta`*），使得损失函数 **`L(theta)`** 达到最小值。*
- **方法不变**：对于参数数量较少的情况，理论上可以进行暴力搜索（brute force search）。但对于像我们现在这样参数数量庞大（例如，数千个甚至更多）的模型，暴力搜索是不可行的。因此，我们仍然依赖于**梯度下降法（Gradient Descent）**。
- **梯度下降的步骤**：
**重要原话**：

- "our current loss function becomes l of theta... the calculation method is exactly the same." `[15:00]`
- "the optimization's algorithm is still gradient descent." `[15:50]`
- "you have to calculate its differential to l for each unknown parameter... it makes up a vector altogether we use g to represent this vector... we call it gradient." `[16:20]`
- "theta 0 minus the learning rate times g is theta 1." `[17:00]`
**延伸思考**

尽管模型结构变得复杂，但机器学习的“三步走”框架（定义模型、定义损失、优化）依然稳固。这表明了机器学习方法论的普适性。梯度下降作为核心优化算法，其原理在参数数量增加后依然适用，只是计算梯度的方式会变得更复杂（例如，需要用到链式法则，这在后续的“反向传播”中会详细讨论）。

**精华收获**

复杂模型下的损失函数和优化过程与简单模型保持一致：损失函数 `L(theta)` 衡量模型表现，所有参数集合用 `theta` 表示；优化通过梯度下降法，迭代计算梯度 `g` 并更新 `theta` (`theta`*`new = theta`*`old - learning`*`rate  g`**) 来最小化损失。*

### 实践中的优化：批次、更新与超参数 `[17:20-22:00]`

**核心观点**

在实际训练中，为了提高效率和稳定性，通常不使用全部数据计算梯度，而是将数据分成小批次（batches）。每次使用一个批次的数据计算梯度并更新参数，称为一次“更新”（update）。处理完所有批次的数据则称为一个“周期”（epoch）。学习率、Sigmoid函数数量和批次大小都是需要人工设定的“超参数”。

**深度阐述**

在理论上，梯度下降法要求我们使用所有训练数据来计算损失函数的梯度。然而，在实际应用中，尤其是当数据量非常庞大时，每次迭代都遍历所有数据来计算梯度会非常耗时且计算成本高昂。为了解决这个问题，李弘毅老师引入了**随机梯度下降（Stochastic Gradient Descent, SGD）**的思想，尽管他没有直接使用这个术语，但其描述的正是SGD的核心机制。

**批次（Batches）与更新（Updates）**

- **数据划分**：我们将总的训练数据 `N` 分成若干个小批次。每个批次包含 `B` 个数据点。这些批次是随机划分的。
- **局部损失与梯度**：在每次迭代中，我们不再使用全部数据计算总损失 `L`，而是只取一个批次的数据（例如，第一个批次），计算这个批次数据上的损失 `L1`。然后，根据 `L1` 计算梯度，并用这个梯度来更新模型参数 `theta`。
- **迭代更新**：接着，我们选择下一个批次的数据，计算其上的损失 `L2`，再根据 `L2` 计算梯度并更新参数。这个过程会一直重复，直到所有批次的数据都被使用过一次。
- **“更新”的定义**：每次使用一个批次的数据计算梯度并更新参数，被称为一次“更新”（update）。
**周期（Epoch）**

- **定义**：当所有批次的数据都被使用过一次，并且参数也相应地更新了多次之后，这个过程就完成了一个“周期”（epoch）。
- **更新次数与周期**：在一个周期内，参数被更新的次数等于批次的数量。例如，如果有10000个数据点，批次大小为10，那么总共有 10000 / 10 = 1000 个批次。因此，在一个周期内，参数会被更新1000次。李老师强调，论文中常提到的“一个周期训练”并不意味着只更新一次参数，实际更新次数取决于批次大小。
**超参数（Hyperparameters）**

李老师借此机会引入了一个非常重要的概念：**超参数**。

- **定义**：超参数是那些由人类（而不是机器）在训练模型之前设定的参数。它们不是通过模型训练过程自动学习出来的。
- **例子**：
**重要原话**：

- "we divide the n pieces into individual batches... we only take the data in one batch just take the first batch and calculate a loss... and update the parameters by the gradient." `[18:20]`
- "processing all the batches once is called an epic." `[19:20]`
- "the so-called hyperparameter is a parameter decided by yourself parameters set by human not machine is called hyperparameter." `[21:00]`
- "learning rate is a hyper parameter the numbers of sigmoids are also hyperparameters the batch size is also a hyperparameter." `[21:20]`
**延伸思考**

批次训练（Mini-Batch Gradient Descent）是现代深度学习训练的基石。它在计算效率和梯度估计的准确性之间取得了平衡，使得训练大型神经网络成为可能。超参数的选择对模型的性能至关重要，通常需要通过实验和经验来调整。

**精华收获**

实际优化中，数据被划分为批次，每次用一个批次计算梯度并更新参数（一次“更新”）。所有批次处理完为一个“周期”。学习率、Sigmoid数量和批次大小是影响模型训练的关键“超参数”，需人工设定。

### 激活函数的演变：从Sigmoid到ReLU `[22:00-24:40]`

**核心观点**

Sigmoid函数并非唯一的选择，还有其他更高效的激活函数。例如，ReLU（Rectified Linear Unit）是一种更简单的激活函数，它可以通过组合来模拟“硬Sigmoid”，并且在实践中表现出更好的性能。

**深度阐述**

他首先回顾了之前提到的“硬Sigmoid”，并指出虽然其公式可能看起来复杂，但实际上它可以通过其他更简单的函数来构建。这里，他引入了另一种非常流行的激活函数——**ReLU（Rectified Linear Unit）**。

**激活函数（Activation Functions）**

无论是Sigmoid还是ReLU，它们在机器学习中都有一个共同的名称：**激活函数（Activation Functions）**。它们是神经网络中引入非线性的关键组件。没有激活函数，无论网络有多少层，都只能表达线性关系，因为线性函数的组合仍然是线性函数。激活函数使得神经网络能够学习和表示复杂的非线性模式。李老师提到，Sigmoid和ReLU是目前最常见的两种激活函数，并预告了在后续课程中会讨论哪种激活函数更好。

**延伸思考**

ReLU函数因其计算简单、能够缓解梯度消失问题等优点，在现代深度学习中已经取代Sigmoid成为主流激活函数。理解不同激活函数的数学特性和它们如何影响网络的学习能力，是设计高效神经网络的关键。

**精华收获**

ReLU（Rectified Linear Unit）是另一种重要的激活函数，其形状为“0-斜坡”，可通过参数 `w, b, c` 调整。两个ReLU函数可以组合成一个“硬Sigmoid”。Sigmoid和ReLU统称为“激活函数”，它们是神经网络引入非线性、学习复杂模式的核心。

### 模型的命名：从神经元到深度学习 `[29:50-34:50]`

**核心观点**

为了赋予复杂模型一个响亮且具有吸引力的名称，我们将Sigmoid或ReLU函数称为“神经元”，将多层神经元堆叠的网络称为“神经网络”，而当网络层数足够多时，则被称为“深度学习”。这个命名过程既有科学的类比，也包含了市场推广的考量。

**深度阐述**

在经历了模型从简单到复杂、从线性到非线性的演变，以及对损失函数和优化过程的深入理解后，李弘毅老师提出了一个幽默而又深刻的问题：我们还缺少什么？答案是——一个“好名字”！他风趣地指出，就像一个人穿上西装会显得更帅气，一个模型拥有一个响亮的名字也能提升其“身价”。

**神经元（Neuron）与神经网络（Neural Network）**

- **命名起源**：我们将之前讨论的Sigmoid或ReLU函数，这些接收输入、进行加权求和、并通过激活函数产生输出的单元，形象地称为**神经元（Neuron）**。
- **类比生物**：这个名字来源于生物学。人脑中有大量的神经元，它们相互连接，形成一个复杂的神经网络。我们的模型，通过模拟这种结构，也由许多相互连接的“神经元”组成，因此被称为**神经网络（Neural Network）**。
- **历史背景**：李老师提到，神经网络并非一项新技术，早在20世纪80年代和90年代就已经存在。然而，由于当时对神经网络的能力过度夸大，导致其声誉受损，甚至在学术界被视为“脏词”，研究论文难以发表。
**深度学习（Deep Learning）的诞生**

- **重塑形象**：为了恢复神经网络的声誉，需要给它一个“新名字”。
- **“层”的引入**：在我们的模型中，每一排神经元（例如，所有接收相同输入并输出到下一层的Sigmoid或ReLU函数）被称为一个**层（Layer）**，通常也称为**隐藏层（Hidden Layer）**。
- **“深度”的含义**：当一个神经网络拥有许多个隐藏层时，我们就称之为**深（Deep）**。因此，这种具有多层隐藏层的神经网络就被赋予了新的名字——**深度学习（Deep Learning）**。
- **重要原话**：
**“深”与“胖”的争论**

李老师接着提出了一个引人深思的问题：为什么我们要让网络“深”（Deep），而不是让它“胖”（Fat）？

- **“胖网络”**：如果只使用一层，但包含非常多的神经元（例如，1000个ReLU），这可以被称为一个“胖网络”。理论上，一个足够“胖”的单层网络也能够近似任何连续函数。
- **“深网络”的优势**：那么，为什么还要费力去堆叠多层呢？为什么深度比宽度更受青睐？李老师将这个问题留给了未来的课程，暗示了深度网络在表达能力、特征学习和训练效率等方面可能具有独特的优势。他幽默地指出，“深度神经网络”听起来比“胖神经网络”更酷。
**深度学习的发展里程碑**：

李老师简要回顾了深度学习在2012年之后取得的突破性进展：

- **AlexNet (2012)**：8层，图像识别错误率16.4%。
- **VGG (2014)**：19层，图像识别错误率7.3%。
- **GoogLeNet (2014)**：22层，图像识别错误率6.7%。
- **ResNet (2015)**：152层，甚至比台北101大楼还高！
这些例子展示了深度学习模型层数不断增加的趋势，以及由此带来的性能飞跃。

**延伸思考**

深度学习的命名和发展历程，反映了科学研究与社会认知、市场推广之间的复杂关系。理解“神经元”、“神经网络”和“深度学习”这些术语的由来，有助于我们更好地把握其核心概念和历史演变。而“深”与“胖”的争论，则预示了网络结构设计中的重要理论和实践问题。

**精华收获**

Sigmoid/ReLU函数被命名为“神经元”，多层神经元堆叠形成“神经网络”。当网络层数足够多时，则被称为“深度学习”，这是一个旨在重塑神经网络形象的新名称。深度网络相比于宽度网络具有独特的优势，是未来课程探讨的重点。

### 实际预测与人类洞察的价值 `[34:50-40:00]`

**核心观点**

在模型选择时，我们应优先考虑其在未见数据上的表现，以避免过拟合。即使是先进的深度学习模型，也可能因缺乏人类常识而做出“愚蠢”的预测，这凸显了人类领域知识在机器学习应用中的不可替代性。

**深度阐述**

在课程的最后，李弘毅老师将理论与实践相结合，通过一个真实的预测案例，不仅巩固了之前关于过拟合的讨论，更深刻地揭示了机器学习模型在面对真实世界复杂性时的局限性，以及人类洞察力的不可或缺性。

**模型选择：3层 vs. 4层**

- **问题回顾**：我们之前实验发现，3层网络在训练数据上的损失是0.14k，在未见数据上的损失是0.38k。而4层网络在训练数据上的损失是0.1k（更低），但在未见数据上的损失却是0.44k（更高）。
- **选择标准**：当我们需要用训练好的模型来预测未来的未知数据时，应该选择3层网络还是4层网络？李老师通过现场提问的方式，引导学生思考。
- **正确答案**：我们应该选择**3层网络**。
- **原因**：虽然4层网络在训练数据上表现更好，但我们真正关心的是模型在**未见数据**上的泛化能力。2月26日的观看量是未见数据，因此我们应该选择在未见数据上表现更好的模型，即3层网络。这再次强调了**过拟合**的危害以及评估模型时应关注泛化性能。
**实际预测案例：2月26日观看量预测**

李老师使用3层网络对2月25日和2月26日的频道观看量进行了预测。

- **2月25日预测**：5250次观看。
- **2月26日预测**：3960次观看。
- **模型行为分析**：模型预测2月26日的观看量非常少。李老师解释说，这是因为模型“知道”周五的观看人数相对较少，所以做出了较低的预测，这听起来似乎合理。
**模型的“盲点”与人类洞察**

然而，李老师随即抛出了一个更深层次的问题：这个预测真的准确吗？他展示了之前模型预测曲线中一个有趣的现象：

- **视觉/结构信息描述**：在预测曲线中，红色（真实数据）在某些特定日期会出现明显的低谷（例如，每隔一段时间出现一个两天的低点），而蓝色（预测数据）在这些低点上预测得相当准确。然而，在某个特定的日期，真实数据出现了一个非常明显的低谷，但模型却没有预测到，反而将低谷预测晚了一天。
- **原因揭示**：李老师询问学生是否知道原因。答案是：**除夕（New Year's Eve）**！
- **模型局限**：模型仅仅根据过去56天的数据进行预测，它并不知道“除夕”这个概念，也不知道人们在除夕夜不会学习机器学习。因此，它无法理解这种由人类社会文化事件导致的特殊低谷。
- **人类价值**：这个例子生动地说明了，即使是最复杂的深度学习模型，也可能因为缺乏人类的**常识（common sense）**和**领域知识（domain knowledge）**而做出“愚蠢”的预测。在实际应用中，人类的洞察力仍然是不可替代的，我们需要将模型预测与人类经验相结合。

**进一步学习**

李老师最后提到了**反向传播（Backpropagation）**，指出它是一种比直接计算梯度更高效的方法，并提供了相关视频链接。这为学生指明了未来深入学习的方向。

**延伸思考**

这个案例是理解机器学习模型局限性的绝佳范例。它提醒我们，模型是基于数据学习的，它只能从数据中发现模式，而无法理解数据背后的深层语义和人类社会文化背景。因此，在部署机器学习系统时，必须结合人类专家知识进行审慎评估和干预。

**精华收获**

模型选择应以在未见数据上的泛化能力为准，以避免过拟合。深度学习模型虽强大，但缺乏人类常识和领域知识，可能在特殊情境下做出不准确预测，这凸显了人类洞察力在机器学习应用中的不可替代性。
