### 终身学习：从人类到机器的宏伟愿景 `[00:00-01:30]`

**核心观点**

终身学习（Lifelong Learning, LLL）的字面含义是“活到老学到老”，虽然在日常语境中多指人类的持续学习，但在机器学习领域，它代表着机器能够像人类一样，不断学习新任务、积累新技能，并最终融会贯通，实现通用人工智能的宏伟目标。

**深度阐述**

当我们首次听到“终身学习”这个词时，大多数人的第一反应是与人类教育和个人发展相关的概念——即“活到老学到老”的精神。如果你在搜索引擎中输入“lifelong learning”，最先跳出来的结果也往往是关于人类如何持续提升自我、适应社会变化的讨论。这无疑是终身学习最直观、最普遍的理解。

然而，李弘毅教授在课程伊始便巧妙地将我们的注意力从人类世界引向了机器智能的未来。他指出，尽管人类的终身学习备受关注，但机器同样需要进行终身学习。而且，机器实现终身学习的方式，与我们对人工智能最原始、最浪漫的想象惊人地相似。

在许多人尚未接触机器学习之前，对AI的想象往往是这样的：我们首先教会机器一项特定任务，比如语音识别，它便能熟练掌握。接着，我们教授它第二项任务，例如图像识别，它也能轻松应对。然后是第三项任务，比如翻译。最终，这个AI将能够同时进行图像识别、语音识别和翻译，并且随着我们不断教授它新的技能，它将学会成千上万、甚至数百万种技能，最终演化成一个无所不能的“天网”（Skynet），甚至可能统治人类。

这种对AI的憧憬——一个能够持续学习新任务，并不断变得更强，最终达到人类无法企及高度的智能体——其核心目标正是“终身学习”。李教授强调，这种愿景并非科幻独有，而是机器学习研究者们孜孜以求的终极目标之一。

为了方便讨论，终身学习通常被缩写为“LLL”（Triple L），而非“LOL”。它还有许多听起来更“花哨”的别名，例如“持续学习”（Continuous Learning）、“永无止境学习”（Never-ending Learning），以及一个相对不那么时髦的名称——“增量学习”（Incremental Learning）。这些不同的称谓，都指向了同一个核心理念：让机器在不断获取新信息、面对新挑战的过程中，持续地进化和成长。

**个人感受**

李教授在这里的开场白非常引人入胜，他没有直接抛出技术定义，而是从我们对AI的普遍想象入手，将“天网”这样的科幻元素与严肃的机器学习概念联系起来，瞬间拉近了听众与主题的距离。他幽默地提醒我们“LLL”不是“LOL”，也展现了他轻松活泼的教学风格。这种从宏大愿景切入的方式，不仅激发了听众的兴趣，也为后续深入探讨终身学习的挑战埋下了伏笔。

**延伸思考**

这种对“天网”式AI的想象，在现实中究竟有多大的可能性？如果机器真的能够实现无限制的终身学习，那么它与人类智能的边界又将如何定义？这不仅仅是技术问题，更是伦理、哲学和社会层面的深远议题。终身学习的终极目标，是否就是实现通用人工智能（AGI）？如果不是，那么它的实际应用边界又在哪里？

### 终身学习的现实意义与核心挑战：灾难性遗忘 `[01:30-04:50]`

**核心观点**

终身学习不仅关乎宏大的“天网”愿景，在实际应用中也具有重要价值，例如模型在线更新。然而，其核心挑战在于“灾难性遗忘”（Catastrophic Forgetting），即模型在学习新任务时，会迅速且严重地遗忘之前学到的旧任务知识。

**深度阐述**

或许有人会觉得，“天网”的目标过于宏大，与日常应用相去甚远。但李教授紧接着指出，终身学习在实际应用中同样具有不可忽视的价值。他举了一个非常贴近工程实践的例子：在实验室中开发并训练好一个模型后，将其部署上线。模型上线后，会不断从用户那里收集到新的反馈数据。我们期望这个过程能够形成一个自动化的循环：新数据被收集，模型参数得以更新，更新后的模型继续收集更多数据，并再次更新参数。通过这种持续的在线学习和更新，系统将变得越来越强大。

在这个场景中，我们可以将旧数据视为“旧任务”，而来自用户反馈的新数据则构成“新任务”。机器持续在线收集数据并用于更新模型，这本身就是一个典型的终身学习问题。

然而，终身学习并非简单地让机器不断收集新数据并更新参数就能实现。如果真如此，它就不值得作为一个独立的研究课题来深入探讨了。那么，终身学习的真正难点究竟在哪里？

李教授通过一个简单的手写数字识别示例，生动地揭示了这一核心挑战。假设我们有两个任务：

- **任务一**：识别带有大量噪声的手写数字（例如，识别一个模糊的“0”）。
- **任务二**：识别相对简单、没有噪声的手写数字。
有人可能会争辩说，这并非两个完全不同的任务，而是在不同领域（或不同难度级别）下的同一个任务。李教授承认这种观点，并指出在终身学习的论文中，所谓的“不同任务”确实常常表现为同一任务在不同领域或条件下的变体，而非语音识别、图像识别、翻译这样截然不同的任务。但他强调，即使是这些非常相似的任务，在进行终身学习时，也可能出现严重问题。

他展示了一个简单的三层神经网络，每层50个神经元。

1. **首先，模型在任务一上进行训练。** 训练完成后，模型在任务一上达到了90%的准确率。令人惊喜的是，即使它从未见过任务二的数据，在任务二上也能达到96%的准确率。这表明任务一中学到的知识具有很好的迁移性，能够帮助模型解决任务二。
1. **接着，让同一个模型在任务二上继续学习。** 这里的“继续学习”至关重要，意味着模型不是从随机初始化的参数开始训练，而是基于在任务一上已经训练好的参数继续学习任务二。
1. **结果令人震惊：** 模型在任务二上的准确率确实略有提升，从96%提高到97%。然而，它却“忘记”了如何执行任务一！原本在任务一上高达90%的准确率，在学习任务二之后，骤降至80%。机器遗忘了它过去学到的技能。
有人可能会认为，这不足为奇，毕竟神经网络的容量有限，学习了新任务自然会挤占旧任务的记忆。但李教授通过另一个实验驳斥了这种观点：

- **将任务一和任务二的数据合并在一起，同时训练网络。** 结果显示，模型在任务一上达到89%的准确率，在任务二上达到98%的准确率。这意味着，对于这个网络架构而言，它完全有能力同时学习这两个任务，即使每层只有50个神经元，也足以达到令人满意的准确率。
这个对比实验清晰地揭示了问题所在：网络并非没有能力学习多个任务，而是当它被要求“顺序学习”时，它就无法记住旧任务。李教授形象地比喻道：这就像一个“脑袋上有个洞的人”，新东西一进去，旧东西就漏出去了，它永远无法学会多项技能。

这种现象在机器学习领域被称为“灾难性遗忘”（Catastrophic Forgetting）。之所以在“遗忘”前加上“灾难性”这个形容词，是因为遗忘在人类身上也会发生，这并不奇怪。但机器的遗忘程度过于严重，以至于它几乎无法学习新技能。这种程度的遗忘，已经超出了正常遗忘的范畴，因此被冠以“灾难性”之名，以强调其问题的严重性。

**重要原话**：

- "The machine continuously collects data online and uses it to update model, it is a lifelong learning problem." `[02:05]`
- "The bad thing is that the machine forgets how to do task 1." `[03:40]`
- "When the new task comes in, the old thing comes out. It will never learn multiple skills. This situation is called catastrophic forgetting." `[04:35]`
**个人感受**

李教授在阐述灾难性遗忘时，通过两个对比实验，将一个抽象的技术难题具象化。他先是展示了顺序学习的失败，再通过同时学习的成功来证明模型本身具备学习多任务的能力，从而有力地反驳了“容量不足”的常见误解。这种层层递进的论证方式，让听众对“灾难性遗忘”的本质有了更深刻的理解。他用“脑袋上有个洞的人”来形容，既生动又富有画面感，让人印象深刻。

**延伸思考**

灾难性遗忘的根本原因是什么？是神经网络的权重更新机制导致旧知识被新知识覆盖？还是模型在学习新任务时，其内部表示发生了根本性的改变，从而破坏了对旧任务的编码？理解其深层机制，对于寻找有效的解决方案至关重要。此外，人类大脑是如何避免这种灾难性遗忘的？这是否能为我们设计更有效的终身学习算法提供生物学上的启发？

### 灾难性遗忘的普遍性：以自然语言处理为例 `[04:50-07:50]`

**核心观点**

灾难性遗忘并非手写数字识别的特例，它在自然语言处理（NLP）等其他领域也普遍存在，尤其是在顺序学习多个简单任务时，模型会立即遗忘旧任务。

**深度阐述**

为了进一步证明灾难性遗忘是一个普遍现象，而非仅仅局限于图像识别领域，李教授接着举了一个自然语言处理（NLP）的例子：问答系统（Question Answering, QA）。他提到，这与他课程的第七次作业（Homework 7）相关，该作业要求机器阅读文档并回答问题。

然而，为了简化问题，他没有使用Homework 7的数据，而是引入了一个更简单的QA任务集，名为“bAbI”。bAbI是一个早期QA任务，由Facebook定义了20个简单的QA任务，旨在帮助研究者在深度学习刚开始应用于QA领域时，克服当时人们认为QA“太难”的挑战。

李教授解释说，bAbI任务与真实的QA任务（如Homework 7）相比，要简单得多。bAbI中的文章和问题都是通过固定句式模板生成的，例如：

- “玛丽把蛋糕给了弗雷德。弗雷德把蛋糕给了比尔。比尔把牛奶给了杰夫。”
- 问题：“谁把蛋糕给了弗雷德？”
- 答案：“玛丽。”
再比如第15个任务：

- “羊害怕狼。猫害怕狗。老鼠害怕猫。”
- “如果格特鲁德是一只羊，那么它害怕什么？”
- 答案：“狼。”
这些问题对于人类来说简直是“小菜一碟”，甚至对于能够解决Homework 7的复杂模型来说，似乎也不应该有任何问题。然而，几年前，让模型能够回答这些简单问题，就已经让研究者们感到震惊了。

接下来，李教授再次展示了灾难性遗忘在bAbI任务上的表现。通常，处理bAbI任务有两种方式：

1. 将所有20个任务的数据合并在一起，同时训练一个模型。
1. 为每个任务分别训练一个模型。
但李教授在这里选择了一种不同的实验设置：他让机器顺序学习这20个QA任务，从第一个任务到第二十个任务，观察模型能否学会所有任务。

他展示了一张图表，纵轴表示正确率，横轴表示学习过程。我们重点关注任务5的正确率。

- 在学习任务1到任务4的过程中，任务5的准确率始终为0。这并不奇怪，因为模型还没有学习任务5，所以它不知道如何解决。
- 当模型开始学习任务5时，任务5的准确率迅速上升到100%。这表明模型一旦接触到任务5的数据，就能立即学会并完美解决它。
- 然而，问题出现在模型继续学习后续任务时。当模型完成任务6的学习后，任务5的准确率立即跌回0。学习任务7之后，任务5的准确率依然是0。
这个结果再次印证了灾难性遗忘的严重性：模型一旦学习新任务，就会立即遗忘旧任务。

为了再次排除“模型容量不足”的解释，李教授展示了另一张图表。这张图表展示了模型同时学习所有20个bAbI任务的结果。纵轴不再是单个任务的准确率，而是所有任务的平均准确率。结果显示，当模型同时学习20个任务时，虽然有些任务（如任务19）可能仍然难以学习，但模型确实能够同时学习多个任务，并在其中几个任务上达到非常高的准确率。

这再次证明，机器并非没有能力掌握多个任务，而是当它被要求顺序学习时，它就“拒绝”这样做。它有能力掌握多项技能，但它就是不愿（或不能）在顺序学习中保持这些技能。这就像李教授之前所说的“脑袋上有个洞的人”，新知识一进来，旧知识就流失了，它永远无法积累多项技能。

**重要原话**：

- "This situation we just saw is not a special case, it is a common phenomenon." `[05:05]`
- "When the machine finishes learning task 6, the accuracy of the model on task 5 returns to 0." `[07:05]`
- "It has the capability to master multiple tasks but it refuses to do so." `[07:40]`
**个人感受**

通过bAbI QA任务的例子，李教授进一步巩固了“灾难性遗忘”的普遍性和严重性。他选择了一个看似简单却能清晰展示问题的NLP任务，让听众意识到这个问题并非特定于某个数据类型或模型架构。他再次用对比实验（顺序学习 vs. 同时学习）来排除模型容量的误解，使得论证更加严谨和令人信服。这种反复强调和多角度验证的方式，加深了听众对核心问题的理解。

**延伸思考**

如果灾难性遗忘是如此普遍和严重，那么我们目前广泛使用的预训练-微调（pre-train & fine-tune）范式，在面对连续任务学习时，是否也存在类似的风险？例如，一个在通用语料上预训练的大模型，在针对特定下游任务进行微调后，是否会遗忘其在其他通用任务上的能力？这对于构建真正通用且持续进化的AI系统，提出了巨大的挑战。

### 多任务学习并非终极方案：存储与计算的瓶颈 `[07:50-10:50]`

**核心观点**

虽然将所有任务数据合并进行“多任务训练”（Multi-task Training）可以有效避免灾难性遗忘，但它并非终身学习的终极解决方案，因为它在实际应用中面临着巨大的存储和计算成本，随着任务数量的增加，其效率会变得极其低下。

**深度阐述**

在深入探讨如何解决灾难性遗忘问题之前，李教授预料到听众可能会提出一个疑问：“既然将所有任务的数据合并在一起进行训练（即多任务训练）可以避免灾难性遗忘，并让机器学会多个任务，那为什么还要研究终身学习呢？”

他随即解释了多任务训练的局限性。设想一下，如果我们要让机器学习第1000个任务。为了防止它遗忘前面999个任务的知识，我们必须将这999个任务的所有数据，与第1000个任务的数据合并起来，然后将这总共1000个任务的所有数据一起进行训练。

这种做法在实践中会带来一系列严重问题：

1. **存储问题**：为了能够同时训练所有任务，机器必须存储它迄今为止所见过的所有数据。这意味着随着学习任务数量的增加，所需存储的数据量也会呈线性甚至指数级增长。在许多实际场景中，我们可能根本没有足够的存储空间来保存如此庞大的数据。
1. **计算问题**：将1000个任务的数据合并在一起进行训练，意味着训练时间将变得异常漫长。数据量的激增会导致计算资源的巨大消耗，使得训练过程效率低下，甚至无法在合理时间内完成。
李教授用一个生动的比喻来形容这种低效：这就像一个学生要学习一门新课程，但在上新课之前，他必须把这辈子上过的所有课程都重新听一遍，把所有读过的教科书都重新读一遍。这显然是非常低效的。随着需要学习的任务越来越多，训练时间会越来越长，需要存储的数据也会越来越多。

因此，尽管多任务训练能够让机器学会多个任务，但它并非终身学习的最终解决方案。在学术文献中，多任务训练通常被视为终身学习的“上限”（Upper Bound）。这意味着，如果我们将所有数据都放在一起进行训练，虽然在任务数量很多时这种方法不切实际，但它确实能够让机器掌握多个任务。所以，多任务学习的结果，往往被认为是终身学习方法所无法超越的理想性能。

在终身学习的研究中，研究人员通常会首先进行多任务训练，以确定这个“上限”在哪里，然后他们的目标就是设计出能够尽可能接近这个上限的终身学习算法，同时避免多任务训练的存储和计算弊端。

有学生提问：“多任务学习是否等同于让机器复习旧材料？”李教授肯定了这种理解，并指出这正是问题所在：我们现在要问的是，在不允许复习旧材料的前提下，机器能否记住它以前学过的东西？而不是每次学习新东西时，都要把旧材料全部复习一遍。

**重要原话**：

- "If we want the machine to learn the 1000th skill and it needs data for the previous 999 tasks, that means that the machine needs to store all the data it has seen." `[08:40]`
- "If the machine must do multi-task learning to be able to learn multiple tasks, it's like a student taking a new class but before he takes the new class he has to retake all the lessons he has taken in his life and read all the textbooks he has read again too learn new tasks." `[09:30]`
- "In the literature, multi-task training is usually treated as the upper bound of life-long learning." `[10:10]`
**个人感受**

李教授在这里的论证非常清晰和务实。他不仅指出了多任务学习的优点（避免遗忘），更深刻剖析了其在实际应用中的致命缺陷（存储和计算瓶颈）。他用学生复习的例子，将复杂的计算资源问题变得通俗易懂，让听众深刻理解了为什么多任务学习无法成为终身学习的最终答案。将多任务学习定义为“上限”的观点，也为后续的技术讨论设定了明确的性能目标。

**延伸思考**

既然多任务学习存在这些问题，那么终身学习的解决方案就必须在不存储所有历史数据、不重复训练所有历史任务的前提下，有效地保留旧知识。这可能需要模型具备某种“选择性记忆”或“知识蒸馏”的能力，将旧知识以更紧凑、更高效的形式编码，以便在学习新任务时能够被唤起和利用，而无需再次访问原始数据。

### 为什么不为每个任务训练一个独立模型？ `[10:50-12:30]`

**核心观点**

为每个任务训练一个独立模型虽然可以解决灾难性遗忘问题，但这种方法同样不适用于终身学习的宏伟目标，因为它会导致模型数量爆炸、无法共享知识，且与人类大脑的运作方式相悖。

**深度阐述**

在讨论了多任务学习的局限性之后，李教授又提出了另一个自然而然的疑问：“为什么我们不直接为每个任务训练一个全新的模型呢？这样不就能彻底解决灾难性遗忘的问题了吗？”

他承认，如果为每个任务都训练一个独立的模型，灾难性遗忘问题确实可以得到解决。然而，这种方法同样存在几个关键问题，使其无法成为终身学习的理想方案：

1. **模型存储问题**：如果我们的目标是构建一个像“天网”那样拥有数百万甚至数亿种技能的AI，那么为每种技能都存储一个独立的模型将是不可行的。所需的模型数量将是天文数字，我们根本没有足够的空间来存储所有这些模型。
1. **知识无法共享**：当每个任务都由一个独立的模型来处理时，不同任务之间学到的知识就无法共享。这意味着，如果某个知识点在多个任务中都有用，每个模型都需要从头学习，无法利用其他模型已经掌握的经验。这不仅效率低下，也限制了模型从更广泛的经验中学习和泛化的能力。
1. **与人类智能的对比**：李教授指出，人类只有一个大脑，但这个大脑却能够学习无数不同的任务，并持续不断地获取新技能。我们不需要为每项技能都配备一个独立的“大脑”。那么，为什么机器就不能做到这一点呢？终身学习正是试图回答这个问题：一个模型能否像人类大脑一样，学习并掌握多项任务？
因此，尽管为每个任务训练独立模型在技术上可行，但它与终身学习追求的“一个智能体持续进化”的愿景背道而驰，也无法实现知识的有效积累和复用。

**重要原话**：

- "If we did learn a model for each task separately, it is true that the catastrophic forgetting problem would indeed be solved." `[11:05]`
- "We can't have a model for every skill right? With this method you may not be able to save all the models." `[11:20]`
- "Humans only have one brain yet the brain can learn many different tasks and keep on learning new skills. We don't need to store every task using a separate brain so why can't machines do the same?" `[11:50]`
**个人感受**

李教授在这里的论证，再次强调了终身学习的“一体化”核心理念。他通过模型数量爆炸和知识共享的视角，以及与人类大脑的对比，清晰地阐明了为什么独立模型并非终身学习的正确方向。这种对“一个模型，多种技能”的执着，正是终身学习区别于其他多任务学习范式的关键。

**延伸思考**

如果一个模型要学习多项任务，并且不能为每个任务都存储一个独立模型，也不能每次都重新训练所有历史数据，那么它必须具备某种机制，能够在学习新任务的同时，有效地保护和利用旧知识。这可能需要更精巧的模型架构、更智能的参数更新策略，或者某种形式的知识压缩与检索机制。

### 终身学习与迁移学习的异同 `[12:30-13:40]`

**核心观点**

终身学习和迁移学习都涉及机器学习多个任务，但它们的关注点截然不同：迁移学习侧重于旧任务知识对新任务的帮助，而终身学习则更关注在学习新任务后，模型是否仍能保持对旧任务的性能。

**深度阐述**

在讨论到这里时，一些听众可能会觉得终身学习与另一个机器学习领域——迁移学习（Transfer Learning）——听起来有些相似。李教授敏锐地捕捉到了这一点，并专门花时间对两者进行了区分。

他指出，迁移学习主要关注的是：机器在任务一上学到的技能，能否有效地“迁移”到任务二上，从而帮助任务二的学习。因此，在迁移学习中，我们主要关心的是模型在**第二个任务**上的表现如何。例如，一个在ImageNet上预训练的模型，能否在新的小数据集上（如猫狗识别）表现出色？这里的核心是预训练知识对新任务的加速和性能提升。

然而，终身学习的关注点则完全不同。终身学习的核心问题是：当机器学会了执行第二个任务之后，它是否**仍然能够很好地执行第一个任务**？换句话说，终身学习更关注的是模型在学习新任务后，对**旧任务的记忆和性能保持**。

李教授总结道，尽管终身学习和迁移学习都涉及两个或多个任务的学习，但它们的侧重点有着本质区别：

- **迁移学习**：关心旧知识对新任务的“前向”帮助。
- **终身学习**：关心新学习对旧任务的“后向”影响（即是否会遗忘）。
理解这种区别至关重要，因为它决定了我们在设计算法和评估系统时所关注的指标和目标。

**重要原话**：

- "In transfer learning we care about whether the skills learned by the machine on the first task can be helpful for the second task so we only care about the performance of the second task." `[13:05]`
- "In life-long learning the main focus is whether the machine can still perform the first task after it has learned how to perform the second task." `[13:20]`
**个人感受**

李教授在这里的区分非常及时和精准，避免了听众可能产生的混淆。他用简洁明了的语言，清晰地界定了两个看似相似但目标迥异的领域，这对于初学者建立正确的概念框架非常有帮助。这种对概念边界的清晰界定，体现了严谨的学术态度。

**延伸思考**

虽然终身学习和迁移学习的关注点不同，但它们之间是否存在协同作用？例如，一个能够有效进行知识迁移的模型，是否更有可能在终身学习中避免灾难性遗忘？或者说，终身学习的某些技术，能否反过来促进更高效的知识迁移？这两个领域的研究成果，或许可以相互借鉴，共同推动AI的进步。

### 终身学习任务的定义与评估方法 `[13:40-结束]`

**核心观点**

为了评估终身学习算法的有效性，研究者需要定义一系列顺序任务，并采用特定的指标来衡量模型在学习新任务后对旧任务的记忆保持能力，以及知识迁移的效果。

**深度阐述**

在探讨终身学习的具体技术之前，李教授首先介绍了如何定义终身学习任务序列以及如何评估其性能。他指出，当前终身学习论文中的任务通常相对简单，这与之前讨论的bAbI任务类似。

他举了几个常见的任务设置示例：

1. **置换MNIST（Permuted MNIST）**：任务一是对原始手写数字进行识别。任务二则是对经过特定模式打乱（置换）后的手写数字进行识别。每个任务对应一种不同的置换模式。这种置换可能很复杂，也可以很简单，例如仅仅将数字图像顺时针或逆时针旋转15度，形成新的任务。在这种情况下，图像仍然是数字，只是角度不同，但仍然可以用于终身学习。
1. **类增量学习（Class-Incremental Learning）**：任务一可能要求模型区分数字0和1。任务二要求区分数字2和3。任务三要求区分数字4和5，以此类推。在这种设置下，例如，0、2、4可能属于第一类，而1、3、5属于第二类。当给定一个小于6的奇数（如1、3、5），模型需要将其归为第二类；而0、2、4则归为第一类。

**延伸思考**

这些评估指标是否足够全面？除了准确率，我们是否还需要考虑其他因素，例如学习效率、计算资源消耗、模型复杂度等？在实际应用中，不同指标的重要性可能有所不同。例如，在某些安全关键型系统中，对旧任务的零遗忘可能比新任务的性能提升更为重要。如何设计更全面的评估框架，以适应不同应用场景的需求，是值得进一步思考的问题。
