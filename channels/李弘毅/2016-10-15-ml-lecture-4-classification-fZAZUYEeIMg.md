## 详细内容

### 分类任务的定义与广泛应用 `[00:00-02:00]`

**核心观点**

分类任务旨在找到一个函数，其输入是一个对象，输出则是该对象所属的预定义类别之一。这是一个在现实世界中拥有无数应用场景的基础机器学习问题。

**深度阐述**

李弘毅教授开篇便直指分类任务的核心：寻找一个“函数”（function）。这个函数的神奇之处在于，它能够接收一个复杂的“对象 x”作为输入，然后精准地判断出这个对象究竟属于“n 个类别”中的哪一个。这听起来抽象，但其应用却遍布我们生活的方方面面，几乎是信手拈来，俯拾皆是。

教授首先列举了金融领域的应用。例如，银行可以利用分类模型来决定是否向某人发放贷款。这个函数的输入将是申请人的多维度信息，包括但不限于其“收入（income）”、“储蓄（saving）”、“工作状况”、“年龄”，以及至关重要的“过往财务记录”，比如是否有“欠债”等。而函数的输出则是一个二元决策：是“接受（accept）”贷款，还是“拒绝（refuse）”贷款。这是一个典型的二元分类（binary classification）问题。

接着，医疗诊断也被提及。在这里，函数的输入是患者的“症状”、“年龄”、“性别”以及“过往就医历史”等。模型的目标是自动诊断出患者可能患有的“疾病类型”。这无疑能极大地辅助医生，提高诊断效率和准确性。

手写数字或文字识别是另一个经典案例。当我们手写一个字（例如教授风趣地写下“金”字，并自称“大金”），机器接收到的是这个字的图像。分类模型的任务就是识别出这个图像所代表的“字符”。如果涉及到中文手写识别，由于中文字符数量庞大（至少 8000 个常用字），这就演变成了一个拥有“8000 个类别”的复杂分类问题，模型需要从这海量的类别中选出唯一正确的那个。

最后，人脸识别也被提及，其目标是输入一张人脸图像，然后识别出“这个人是谁”。这些例子共同描绘了分类任务的强大能力和广泛适用性，它不仅仅是理论概念，更是解决实际问题的强大工具。

**精华收获**

分类是机器学习的核心任务，通过构建一个输入对象、输出类别的函数，能够解决金融、医疗、识别等诸多领域的实际问题。其本质是从多维信息中提取模式，进行决策。

### 宝可梦分类实战：将抽象概念具象化 `[02:00-07:00]`

**核心观点**

为了深入理解分类，教授以宝可梦属性分类为例，展示了如何将一个现实世界的复杂对象（宝可梦）转化为数值输入，并阐明了分类结果在实际应用中的重要性。

**深度阐述**

为了让分类这个抽象概念更加具象化，李弘毅教授再次祭出了他钟爱的“宝可梦”例子。他幽默地表示，虽然之前在预测 CP 值时用过宝可梦，但这次他“又做了一些有关宝可梦的研究”，展示了对这个主题的持续热情。

本次研究的目标是构建一个宝可梦属性分类器。我们知道，宝可梦拥有多达“18 种属性”（到第六代为止），包括水、火、电、草、冰等。分类任务就是找到一个函数，输入是“一只宝可梦”，输出则是它所属的“属性类型”。例如，输入“皮卡丘”，输出“电”；输入“杰尼龟”，输出“水”；输入“妙蛙种子”，输出“草”。

然而，这里首先遇到的挑战是：如何将一只宝可梦作为函数的输入？函数通常处理的是数值，而宝可梦是一个复杂的生物。教授指出，关键在于“数值化”——我们需要用“数字来表示一隻宝可梦”。

宝可梦的“特性”是数值化的基础。虽然教授提到这并非《Pokémon Go》的数据，而是电玩中的设定，但其核心思想是共通的。一只宝可梦可以用一个“数字向量”来描述其各种特性，这些特性包括：

- **整体强度 (Total Strong)**：衡量宝可梦的综合实力。
- **生命值 (HP)**：决定宝可梦的耐久度。
- **攻击力 (Attack)**：物理攻击能力。
- **防御力 (Defense)**：物理防御能力。
- **特殊攻击力 (Special Attack)**：特殊攻击能力。
- **特殊防御力 (Special Defense)**：特殊防御能力。
- **速度 (Speed)**：决定战斗中谁先行动。
以“皮卡丘”为例，它的这些数值构成了一个“7 个数值组成的向量”：[320 (Total), 35 (HP), 55 (Attack), 40 (Defense), 50 (Special Attack), 50 (Special Defense), 90 (Speed)]。这样，每一只宝可梦都被转化为了一个数学上的“向量”，可以作为函数的输入。

那么，为什么要费力去预测宝可梦的属性呢？教授强调了其“非常重要”的实际意义。在宝可梦对战中，存在着“属性相克”的关系，这是一个“18x18 的属性相克表”。例如，“格斗系”对“一般系”的攻击力会“乘以 2”。这意味着，如果能提前知道对手宝可梦的属性，就能选择克制它的属性进行攻击，从而占据优势。

你可能会问：“宝可梦图鉴上不是都有属性了吗？”教授巧妙地回应了这个问题。他指出，在对战中，我们可能会遇到“图鉴上没有的”、“没见过的”新宝可梦。如果此时我们有一个“预测模型”，就能在第一时间预测出新宝可梦的属性，从而“用正确的属性来对付它”。这不仅展示了分类模型的实用性，也暗示了机器学习在处理未知数据时的强大能力。教授甚至幽默地建议，应该把这种预测模型加入到宝可梦图鉴中，使其具备“影像辨识”和“预测新宝可梦”的功能。

**个人感受**

教授在讲解中穿插宝可梦的例子，不仅让枯燥的机器学习概念变得生动有趣，也让人感受到他对教学的热情和对学生理解的考量。他甚至能预判到学生可能提出的疑问，并给出巧妙的解答，这种互动感极大地提升了学习体验。

**精华收获**

将复杂对象数值化是机器学习的第一步。宝可梦的七维特征向量化，使得分类模型得以处理。属性相克机制凸显了分类预测的实战价值，尤其是在面对未知对象时，模型能提供关键决策依据。

### 数据收集与训练/测试集划分 `[07:00-08:00]`

**核心观点**

构建分类模型的第一步是收集数据，并将其合理划分为训练集和测试集，以模拟真实世界中模型对新数据的泛化能力。

**深度阐述**

在明确了分类任务和宝可梦的数值表示后，下一步自然是“收集数据”。教授提出了一个实际的划分策略：将“宝可梦编号 400 以下的”作为“训练数据（training data）”，而“编号 400 以上的”作为“测试数据（testing data）”。

这个划分并非随意，而是基于对宝可梦世界演变的一种模拟。教授回忆起自己小时候只有 150 只宝可梦，而现在已经有 800 多只，宝可梦的数量是“不断增加的”。因此，“编号比较前面的”代表“比较早发现的”宝可梦，可以用来训练模型；而“编号比较后面的”则模拟了“新的宝可梦”，用于测试模型在面对未见过的宝可梦时的预测能力。这种划分方式，巧妙地模拟了机器学习模型在实际应用中需要处理“新数据”的场景，确保了模型评估的有效性。

收集到的数据是“pair”（对）的形式，即“function 的 input, output 是什么”。例如：

- Input: 皮卡丘 -> Output: 电
- Input: 杰尼龟 -> Output: 水
- Input: 妙蛙种子 -> Output: 草
这些成对的数据构成了模型学习的基础，让模型能够从已知的输入-输出关系中学习模式。教授还提到，如果想要更严谨，可以在训练数据中再切分出一部分作为“验证数据（validation data）”，用于模型调优。

**精华收获**

数据是机器学习的基石。通过将早期发现的宝可梦作为训练集，后期发现的作为测试集，可以有效模拟模型在面对未来新数据时的泛化能力，确保模型评估的科学性。

### 误区：将分类问题硬解为回归问题 `[08:00-15:00]`

**核心观点**

将分类问题简单地当作回归问题来处理，虽然直观，但由于回归和分类对“好坏”的定义不同，会导致模型学习到次优甚至错误的决策边界。

**深度阐述**

在讲解如何解决分类问题时，教授首先提出了一个常见的“误区”：那些只学过回归（Regression）的人，可能会尝试“硬解”分类问题，将其当作回归问题来处理。教授明确劝告：“我劝你不要这么做。”

他以二元分类为例进行说明。假设我们只有两个类别：Class 1 和 Class 2。如果将其视为回归问题，一个直观的想法是：

- 将 Class 1 的目标值（target，即 y*head）设为 1。*
- 将 Class 2 的目标值设为 -1。
然后，像训练回归模型一样进行训练。在测试时，由于回归模型的输出是一个连续的数值，而不是离散的 0 或 1，我们需要设定一个阈值：如果输出值“比较接近 1”，就判定为 Class 1；如果“比较接近 -1”，就判定为 Class 2。例如，可以以 0 为分界点：大于 0 判为 Class 1，小于 0 判为 Class 2。

然而，这种做法会遇到严重问题。教授通过一个二维空间中的例子（`[09:00]` 视觉/结构信息描述：投影片展示了一个二维平面，其中红色点代表 Class 1，蓝色点代表 Class 2。一个线性回归模型 `y = b + w1`*`x1 + w2`*`x2` 被用来拟合这些点。）来阐释。

在这个例子中，蓝色点希望输出接近 1，红色点希望输出接近 -1。如果数据分布良好，回归模型可能会找到一条绿色的线，这条线代表 `b + w1`*`x1 + w2`*`x2 = 0`，作为 Class 1 和 Class 2 的分界线。这条线看起来“满好的”，因为它直观上是最佳的分类边界。

但问题出在回归模型的“好坏定义”上。教授展示了另一种 Class 1 的分布（`[10:00]` 视觉/结构信息描述：投影片上，Class 1 的红色点分布发生了变化，一部分点远离了决策边界）。在这种情况下，如果仍然使用回归模型，它会认为那些“远大于 1”的蓝色点是“错误”的，因为它们“太大了”，偏离了目标值 1。回归模型的损失函数会“惩罚”这些“太正确”的点，试图将它们拉回到 1 附近。

结果是，回归模型不会选择直观上更好的“绿色线”作为边界，而是会选择一条“紫色线”（`[11:00]` 视觉/结构信息描述：投影片上出现了一条偏离绿色线的紫色线）。这条紫色线对于回归模型来说是“更好的函数”，因为它能让那些远离边界的蓝色点（其回归输出值可能远大于 1）的输出值“压小”，使其更接近 1，从而减小回归损失。然而，从分类的角度看，这条紫色线显然是一个“不好的”决策边界，因为它错误地将一些红色点划分到了蓝色区域。

这揭示了核心矛盾：

- **回归模型**：目标是预测一个精确的数值，其损失函数（如均方误差）会惩罚预测值与真实值之间的任何偏差，无论偏差方向。它会惩罚“太正确”的预测。
- **分类模型**：目标是正确划分类别，只要预测的类别正确，预测值具体是多少并不重要。它不应该惩罚“太正确”的预测。
因此，回归模型对“模型好坏的定义”与分类任务的需求是“不适用”的。

除了二元分类，教授还提到了多类别分类的另一个问题。如果将 Class 1 设为目标 1，Class 2 设为目标 2，Class 3 设为目标 3，那么模型就会“假设 Class 3 和 Class 2 比较近”，以及“Class 2 和 Class 1 比较近”，它们之间存在某种“关系”。然而，在实际的分类问题中，类别之间往往没有这种自然的“序关系”或“距离关系”。例如，猫、狗、鸟这三个类别，它们之间并没有说“狗比猫更接近鸟”这种数值上的远近。这种人为的数值映射会误导模型，导致“没有办法得到一个好的结果”。

**个人感受**

教授在讲解这一部分时，语气中带着一丝无奈和强调，反复劝告“我劝你不要这么做”，显示出这是一个非常普遍且容易犯的错误。通过视觉化的例子，他清晰地展示了直觉与模型内在机制之间的差异，令人印象深刻。

**延伸思考**

这个例子深刻说明了选择正确模型和损失函数的重要性。不同的任务有不同的目标，盲目套用模型可能导致南辕北辙。理解模型背后的数学原理和假设，是避免“硬解”误区的关键。

**精华收获**

将分类问题简单转化为回归问题是错误的。回归模型会惩罚“过于正确”的预测，导致决策边界次优；多类别回归中的数值映射会引入不合理的类别间关系。分类需要专门的损失函数和模型来处理。

### 理想分类方法的初步构想与挑战 `[15:00-17:00]`

**核心观点**

理想的分类模型应直接输出离散的类别，其损失函数应基于错误分类的次数。然而，这种基于计数（0/1损失）的损失函数是不可微的，给传统的梯度下降优化带来了挑战。

**深度阐述**

既然将分类问题硬解为回归行不通，那么“理想上的做法”应该是什么样子呢？教授指出，与回归模型输出连续的“实数（real number）”不同，分类模型的输出应该是“离散的（discrete）”，即直接输出“某一个 class”。

一个可能的想法是，对于二元分类问题，我们可以寻找一个函数 `f`，它内部包含另一个函数 `g`。如果 `g(x)` 的值“大于 0”，则判定为 Class 1；否则，判定为 Class 2。这里的 `g` 也是通过训练数据自动学习得到的。

那么，在训练这个模型时，“损失（loss）”应该如何定义呢？最直观的想法是，我们希望找到一个函数 `f`，使其在训练数据上“预测错误的次数”越少越好。这个损失函数可以表示为：

`Loss(f) = Σ δ (f(x^n) ≠ (y`*`head)^n)`*

其中，`δ` 是一个指示函数：如果 `f(x^n)` 的输出与正确答案 `(y`*`head)^n`** 不符，**`δ`** 就为 1；否则为 0。将所有训练样本的 **`δ`** 值加起来，就得到了模型分类错误的次数。我们自然希望这个值“越小越好”。*

然而，教授立即指出了这种损失函数的核心挑战：“这个没办法微分啊！”我们熟悉的梯度下降（Gradient Descent）优化算法，依赖于损失函数的可微性来计算梯度，从而更新模型参数。但这种基于计数的 0/1 损失函数是“不可微的”，这意味着我们无法直接使用梯度下降来优化它。

教授简要提到了存在一些方法可以解决这个问题，例如“Perceptron”（感知机）和“SVM”（支持向量机），但表示“今天先不讲这个方法”，为后续课程埋下伏笔。这表明，虽然直观的损失定义面临技术障碍，但机器学习领域已经发展出多种巧妙的解决方案。

**精华收获**

理想分类模型的损失函数应直接衡量错误分类次数。但这种0/1损失函数不可微，导致无法直接使用梯度下降优化，需要其他优化策略或替代损失函数。

### 从概率视角看分类：生成式模型 `[17:00-22:00]`

**核心观点**

通过引入概率论，我们可以将分类问题转化为计算给定输入属于各个类别的后验概率，并选择概率最大的类别。这种方法被称为生成式模型，因为它隐含了生成数据的能力。

**深度阐述**

由于直接优化不可微的损失函数存在困难，教授引导我们转向“另外一个 solution”——从“机率的观点”来看待分类问题。他强调，这种解决方案最终也会与机器学习的三个步骤（模型、好坏、优化）保持一致。

为了引入概率概念，教授首先回顾了我们“国中的时候，机率会问的问题”：两个盒子，每个盒子装有不同比例的蓝球和绿球。假设我们不知道球是从哪个盒子取出的，但知道取出的球是蓝色的，那么这颗蓝球来自盒子 1 或盒子 2 的概率分别是多少？（`[18:00]` 视觉/结构信息描述：投影片展示了两个盒子，B1 和 B2，其中包含不同数量的蓝球和绿球，并给出了从每个盒子取球的先验概率 P(B1) 和 P(B2)，以及在每个盒子中取到蓝球的条件概率 P(Blue|B1) 和 P(Blue|B2)。）

通过贝叶斯定理，我们可以计算出“给定一颗蓝球，它从盒子 1 里面抽出来的机率”，即 `P(B1|Blue)`。教授自信地表示：“我相信大家都秒懂。”

接着，教授将这个“盒子与球”的类比巧妙地转化到分类问题中：

- **盒子** 变成了 **类别（Class）**，例如 Class 1 和 Class 2。
- **球** 变成了 **要分类的对象 x**，例如一只宝可梦。
现在的问题是：给定一个对象 `x`（比如一只宝可梦），它“从某一个 class 里面 sample 出来的机率是多少呢？”即计算 `P(Class|x)`。

为了计算 `P(Class|x)`，我们需要知道以下四个概率值：

1. **`P(C1)`**：对象 `x` 属于 Class 1 的先验概率。
1. **`P(C2)`**：对象 `x` 属于 Class 2 的先验概率。
1. **`P(x|C1)`**：如果 `x` 属于 Class 1，那么观察到 `x` 这个特定对象的概率。
1. **`P(x|C2)`**：如果 `x` 属于 Class 2，那么观察到 `x` 这个特定对象的概率。
有了这四个值，我们就可以利用贝叶斯定理计算后验概率 `P(C1|x)`：

`P(C1|x) = [P(x|C1) `*` P(C1)] / [P(x|C1) `*` P(C1) + P(x|C2) `*` P(C2)]`*

（`[20:00]` 视觉/结构信息描述：投影片上展示了贝叶斯定理的公式，将盒子 B1 替换为 C1，Blue 替换为 x。）

如果能够计算出 `P(C1|x)` 和 `P(C2|x)`，那么“问题就解决了”。我们只需选择“机率最大那个 case”，作为 `x` 的分类结果。

教授进一步解释了这种方法为何被称为“生成式模型（Generative model）”。这是因为，如果我们知道了 `P(C1)`、`P(C2)`、`P(x|C1)` 和 `P(x|C2)`，实际上就知道了 `P(x)`（即 `P(x) = P(x|C1)P(C1) + P(x|C2)P(C2)`）。知道了 `P(x)`，就知道了 `x` 的“分布（distribution）”，从而可以“用这个 distribution 来产生 x、sample x 出来”。换句话说，模型不仅能分类，还能“生成”与训练数据分布相似的新数据，因此得名。

**精华收获**

通过概率模型，分类问题转化为计算后验概率 P(Class|x)。这需要估计先验概率 P(Class) 和条件概率 P(x|Class)。这种方法被称为生成式模型，因为它能描述数据的生成过程。

### 估计先验概率 P(C) `[22:00-24:00]`

**核心观点**

先验概率 P(C) 可以通过统计训练数据中每个类别的样本数量来直接估算，这是一种相对简单的计算。

**深度阐述**

在生成式模型中，我们需要估算四个概率值。教授首先从相对简单的“P(C1) 跟 P(C2)”入手，称它们为“Prior”（先验概率），并指出它们“是比较好算的”。

他以宝可梦分类为例，选择了“水系”和“一般系”作为两个类别（Class 1 和 Class 2），暂时忽略了其他 16 种属性，以简化问题为二元分类。选择这两个类别的原因是，教授统计发现，在所有 18 种宝可梦中，“水系跟一般系是最多的”，这使得它们成为一个合适的二元分类示例。

接着，他利用之前划分好的训练数据（编号 ID < 400 的宝可梦）进行统计：

- 训练数据中共有“79 隻是水系的”宝可梦。
- 训练数据中共有“61 隻一般系”的宝可梦。
有了这些计数，估算先验概率就变得非常直观：

- 从 Class 1（水系）中抽出一只宝可梦的概率 `P(C1)` = `79 / (79 + 61)` = `0.56`。
- 从 Class 2（一般系）中抽出一只宝可梦的概率 `P(C2)` = `61 / (79 + 61)` = `0.44`。
教授总结道：“这个是比较容易、比较简单可以理解的。”这部分内容强调了从实际数据中直接统计频率来估算先验概率的实用性。

**个人感受**

教授在选择水系和一般系作为例子时，特意提到了“不过，我试了一下，如果你要把 18 种都分类正确，好像是做不太出来的”，这种坦诚和对实际困难的预判，让课程显得更加真实和可信。

**精华收获**

先验概率 P(Class) 可以通过统计训练集中各类别样本的比例来直接估算，是生成式模型中相对容易确定的参数。

### 估算条件概率 P(x|C)：高斯分布与最大似然估计 `[24:00-42:00]`

**核心观点**

估算 P(x|C) 是生成式模型的关键挑战。通过假设数据特征服从高斯分布，并利用最大似然估计（MLE）来确定高斯分布的参数（均值 μ 和协方差 Σ），可以解决这个问题。

**深度阐述**

估算 `P(x|C)` 是生成式模型中最具挑战性也最核心的部分。教授提出了一个关键问题：如果给我们一只“海龟”（`[24:00]` 视觉/结构信息描述：投影片展示了一只海龟的图片），它是水系的，但它“没有在我们的 training data 里面”，那么“从水系的神奇宝贝里面挑一隻神奇宝贝出来，它是海龟的机率，到底应该有多大呢？”

直观地，如果训练数据中没有海龟，我们可能会认为这个概率是 0。但教授立刻指出：“这样显然是不对的。”因为海龟明显是水系的，它的概率不应该是 0。我们需要一种方法来“估测”这种未见过的数据点的概率。

**1. 特征表示与数据可视化**

首先，教授再次强调，每只宝可梦都用一个“向量来描述”，这个向量包含了一组“特征（feature）”。为了可视化，虽然每只宝可梦有 7 个数值特征，但我们只能在二维平面上绘制。因此，他选择了“防禦力（Defense）跟特殊防禦力（SP Defense）”这两个特征进行展示。（`[26:00]` 视觉/结构信息描述：投影片展示了一个二维散点图，横轴是 Defense，纵轴是 SP Defense。图上密布着 79 个蓝色点，代表水系宝可梦的分布，并标注了杰尼龟、可达鸭等具体宝可梦的位置。）

这些点代表了训练数据中 79 只水系宝可梦的分布。现在，如果有一只“海龟”，它的防禦力是 103，特殊防禦力是 45，它在图上的位置大概在某个区域。我们不能因为训练数据中没有它就说它的概率是 0。

**2. 引入高斯分布假设**

解决这个问题的关键假设是：这 79 只宝可梦“只是冰山的一角”，它们实际上是从一个“机率的分布里面”采样（sample）出来的。更具体地说，我们假设水系宝可梦的防禦力与特殊防禦力这两个特征，是从一个“高斯分布（Gaussian distribution）”中采样出来的。

高斯分布（也称正态分布）是一个由两个参数决定的函数：

- **均值（mean, μ）**：一个向量，代表分布的“中心点”或“最高点”。
- **协方差矩阵（covariance matrix, Σ）**：一个矩阵，代表分布的“散布程度”和特征之间的“相关性”。
（`[28:00]` 视觉/结构信息描述：投影片展示了高斯分布的数学公式，以及通过改变 μ 和 Σ 导致分布形状变化的示意图。例如，相同的 Σ 但不同的 μ 会导致分布中心不同；相同的 μ 但不同的 Σ 会导致分布的散布程度和方向不同。）

高斯分布的函数 `f`*`μ,Σ(x)`** 的输入是一个向量 **`x`**（宝可梦的特征），输出是 **`x`** 从这个分布中“被 sample 出来的机率”（严格来说是概率密度，但为简化理解，此处当作概率）。**`x`** 越接近分布的中心 **`μ`**，其被采样的概率就越大。*

**3. 最大似然估计（Maximum Likelihood Estimation, MLE）**

现在的问题是：如果给定这 79 个点，我们如何找到那个“高斯分布”的 `μ` 和 `Σ` 呢？这里引入了“最大似然估计（Maximum Likelihood）”的概念。

核心思想是：虽然任何高斯分布都可能采样出这 79 个点（因为任何点的概率都不为 0，只是有些很低），但不同的高斯分布“sample 出这 79 个点的可能性（Likelihood）是不一样的”。我们要做的是“找到那一个 Gaussian”，它“sample 出这 79 个点的机率是最大的”。这个能最大化似然的 `μ` 和 `Σ`，就是我们认为最能代表这些数据的参数，记作 `μ`* 和 **`Σ`*。

似然函数 `L(μ, Σ)` 的计算方式是：由于所有 79 个点是“独立被 sample 出来的”，所以总似然是每个点被采样的概率的“乘积”：

`L(μ, Σ) = P(x^1|μ, Σ) `*` P(x^2|μ, Σ) `*` ... `*` P(x^79|μ, Σ)`*

（`[34:00]` 视觉/结构信息描述：投影片展示了似然函数的公式，以及高斯分布的概率密度函数。）

**4. 求解 μ***** 和 Σ***

那么，如何找到 `μ`* 和 **`Σ`* 呢？教授指出，这可以通过微积分求解似然函数的极值（导数为 0 的点），或者直接背诵公式。结果“是很直觉的”：

- **`μ`***：就是所有 79 个 **`x`** 向量的“平均值”。*
- **`Σ`***：在计算出 **`μ`* 后，通过计算每个 `(x^n - μ`*`)`** 与 **`(x^n - μ`*`)^T` 的乘积的平均值来得到，这实际上是样本的协方差矩阵。
（`[37:00]` 视觉/结构信息描述：投影片展示了 μ* 和 Σ* 的计算公式。）

**5. 实际计算结果**

教授展示了实际计算出的结果：

- 对于 79 只水系宝可梦（Class 1），计算得到其均值 `μ1` 和协方差矩阵 `Σ1`。
- 对于 61 只一般系宝可梦（Class 2），同样计算得到其均值 `μ2` 和协方差矩阵 `Σ2`。
有了这些参数，我们就可以计算任何一个新宝可梦 `x` 从 Class 1 或 Class 2 分布中采样的概率 `P(x|C1)` 和 `P(x|C2)`。

**个人感受**

教授在讲解高斯分布和最大似然估计时，通过形象的比喻和直观的解释，将复杂的数学概念变得易于理解。他强调“这个结果是很直觉的”，并提供了“秒解”的公式，体现了教学的实用性。

**延伸思考**

最大似然估计是统计推断中的一个核心思想，它不仅仅用于高斯分布，也广泛应用于其他概率模型的参数估计。理解 MLE 的原理，对于理解各种机器学习算法的参数学习过程至关重要。

**精华收获**

P(x|C) 的估算通过假设特征服从高斯分布实现。利用最大似然估计，可以从训练数据中计算出每个类别的高斯分布参数（均值 μ 和协方差 Σ）。μ* 是样本均值，Σ* 是样本协方差，它们最大化了观测到训练数据的可能性。

### 初步分类结果与高维空间的挑战 `[42:00-48:00]`

**核心观点**

基于高斯分布的生成式模型在二维特征空间中可能表现不佳，因为类别分布可能高度重叠。然而，机器学习的优势在于能够处理高维数据，高维空间可能揭示出二维空间中无法看到的类别分离。

**深度阐述**

在估算完所有必要的概率（`P(C1)`、`P(C2)`、`P(x|C1)`、`P(x|C2)`）之后，我们就可以进行分类了。对于任何一个给定的宝可梦 `x`，我们计算 `P(C1|x)`。如果 `P(C1|x) > 0.5`，则将其归为 Class 1；否则归为 Class 2。

教授展示了使用“防禦力跟特殊防禦力”这两个二维特征进行分类的结果。（`[43:00]` 视觉/结构信息描述：投影片展示了两个二维散点图。第一个图显示了水系（蓝色点）和一般系（红色点）宝可梦在 Defense-SP Defense 平面上的分布，以及模型计算出的 `P(C1|x)` 的等高线图，用颜色深浅表示概率大小。第二个图则根据 `P(C1|x) > 0.5` 划出了一个非线性的决策边界，将平面分为红色区域（Class 1）和蓝色区域（Class 2）。）

看到这个结果，教授坦言“我有点紧张”，因为“觉得分不出来，我用人眼看就知道不太 ok”。从图上可以看出，蓝色点（水系）和红色点（一般系）的分布“比较多蓝色的点，在这个红色的区间”，而“红色的点，是比较多在这个蓝色的区间”，这意味着类别之间存在严重的“重叠”，没有一个“明确的 boundary”。

将这个模型应用到“测试集”（编号大于 400 的宝可梦）上，结果是“正确率是 47 %”。教授直言：“很糟”，因为这甚至低于随机猜测（二元分类随机猜测的正确率是 50%）。

面对如此糟糕的结果，教授提出了一个关键的思考点：“会不会是这题有可能分不出来，这也有可能。”但他随即强调了机器学习的“厉害的地方”：我们现在只用了“二维的空間”，而“机器学习厉害的地方就是...高维空间也可以处理”。

他解释说，每只宝可梦实际上是分布在“七维的空間里面”。虽然在二维空间中，红色和蓝色点看起来“叠在一起”，但“搞不好七维分的出来啊”。就像从上面往下看，两个物体可能重叠，但从侧面看它们可能是分开的。在七维空间中，这些点“搞不好是分开的”。

因此，即使在七维空间中，我们仍然可以计算 Class 1 和 Class 2 的“μ1 跟 μ2”（七维向量）以及“Σ1 跟 Σ2”（7x7 的矩阵）。然而，当教授真的用所有七个特征进行计算后，结果仍然是“正确率就 50%，很糟”。他无奈地表示：“这样就跟你 random 猜，大概也是这个样子”，并以一句“so sad”表达了失望。这为下一节课如何改进模型埋下了伏笔。

### 改进模型：共享协方差矩阵 `[48:00-58:00]`

**核心观点**

为了减少模型参数、避免过拟合并提高泛化能力，可以强制不同类别共享同一个协方差矩阵。这种假设简化了模型，并可能在实践中带来性能提升，同时使决策边界变为线性。

**深度阐述**

面对之前模型在七维空间中仍只有 50% 正确率的困境，教授开始探讨如何改进。他指出，之前使用的模型（每个 Gaussian 都有自己的 `μ` 和 `Σ`）在实践中“是比较少见的”。更常见的方法是，不同的类别可以“share 同一个 covariance 的 matrix”。

**1. 为什么共享协方差矩阵？**

教授解释了共享协方差矩阵的理由：

- **参数数量**：协方差矩阵的大小与特征维度的平方成正比。当特征维度很高时（例如 7 维，协方差矩阵是 7x7），如果每个类别都有独立的协方差矩阵，模型参数会“非常快”地增长。
- **过拟合（Overfitting）**：模型参数过多，会导致“variance 就大”，即模型容易“overfitting”（过拟合）训练数据，而泛化能力差。
- **简化模型**：通过“强迫它们共用 covariance matrix”，可以“有效减少参数”，从而得到一个更简单的模型。
具体来说，现在 Class 1（水系）和 Class 2（一般系）的宝可梦，都假设是从一个高斯分布中生成出来的，但它们拥有不同的均值 `μ1` 和 `μ2`，却“共用同一个协方差矩阵 Σ”。

**2. 共享协方差矩阵的似然计算**

在这种情况下，计算总似然函数的方式略有不同。它仍然是所有数据点被采样的概率的乘积，但现在 Class 1 的数据点使用 `μ1` 和 `Σ`，Class 2 的数据点使用 `μ2` 和 `Σ`。

**3. 求解 μ1*****, μ2***** 和 Σ**

- **`μ1`***** 和 ******`μ2`***：它们的计算方式与之前相同，仍然是各自类别内数据点的平均值。
- **`Σ`***：由于现在是共享的，**`Σ`** 的计算方式需要综合两个类别的数据。教授指出，这个结果“非常的直观”，它实际上是两个类别各自计算出的协方差矩阵 **`Σ1`** 和 **`Σ2`** 的“加权平均”。权重是每个类别的数据点数量。例如，**`Σ`*` = (Σ1 `*` N1 + Σ2 `*` N2) / (N1 + N2)`。
（`[53:00]` 视觉/结构信息描述：投影片展示了共享协方差矩阵的 Σ 计算公式，并引用了 Bishop 的教科书。）

**4. 共享协方差矩阵的几何意义与结果**

教授再次用二维特征（Defense 和 SP Defense）来可视化共享协方差矩阵的效果。（`[54:00]` 视觉/结构信息描述：投影片展示了两个二维散点图。第一个图显示了在不共享协方差矩阵时，决策边界是“曲线”的。第二个图则显示了当共享协方差矩阵时，决策边界“变成是一个直线”。）

这是一个非常重要的发现：当两个类别“强迫他们的 covariance matrix 必须共用同一个的话”，那么分类时的“boundary 就会变成是一条直线”。这种模型因此被称为“线性模型（linear model）”。

最令人惊喜的是，当我们将这个“共用 covariance matrix”的假设应用到所有七个特征上时，模型的性能发生了显著变化。之前只有 50% 的正确率，现在“神奇的是，当我们共用 covariance matrix 的时候，我们就得到 79% 的正确率了！”教授兴奋地表示：“显然是有分对东西。”

他承认，在高维空间中，“很难分析”这个边界是如何切分的，但这就是“machine learning fancy 的地方”——“人没有办法知道怎么做，但是 machine 可以帮我们做出来”。如果特征很少，人一眼就能看出如何分类，那就不需要机器学习了。

**个人感受**

从“so sad”到“神奇的是”，教授的情绪转变体现了机器学习研究中发现有效改进方法时的喜悦。他再次强调了机器学习在处理高维复杂问题上的独特优势，令人对算法的力量充满敬畏。

**延伸思考**

共享协方差矩阵是模型正则化的一种形式，通过减少模型自由度来降低过拟合风险。这种从非线性决策边界到线性决策边界的转变，也为后续理解线性分类器（如逻辑回归）奠定了基础。

**精华收获**

共享协方差矩阵是减少模型参数、避免过拟合的有效策略。它将非线性的决策边界简化为线性边界，并在宝可梦分类任务中显著提升了准确率。这体现了机器学习在处理高维复杂问题时，通过合理模型假设超越人类直觉的能力。

### 机器学习三步骤回顾与模型选择 `[58:00-01:03:00]`

**核心观点**

生成式模型完美契合机器学习的三个核心步骤：定义模型（函数集）、评估模型好坏（似然函数）和优化模型（最大似然估计）。模型选择（如高斯分布）是人类智慧的体现，需要在偏差与方差之间权衡。

**深度阐述**

教授在此时回顾了机器学习的“三个 step”，并指出之前讲解的概率模型也完全符合这三个步骤：

**1. 模型（Model / Function Set）** `[58:00]`

- **定义**：模型就是“你的 function set”，即所有可能的分类函数。
- **构成**：在这个概率模型中，一个函数由 `P(C1)`、`P(C2)`（先验概率）和 `P(x|C1)`、`P(x|C2)`（条件概率分布）决定。
- **参数**：这些概率值和分布（例如高斯分布的 `μ` 和 `Σ`）就是模型的“参数”。选择不同的参数组合，就得到了不同的函数。
- **决策**：最终的分类决策是基于后验概率 `P(C1|x)` 是否大于 0.5。
**2. 评估函数好坏（Goodness of Function）** `[01:00:00]`

- **定义**：如何衡量一个模型（或一组参数）的好坏？
- **方法**：在这个概率模型中，我们使用“似然（Likelihood）”来评估。一个 `μ` 和 `Σ` 的组合，如果它能“最大化产生这些 data 的 likelihood”，那么它就是好的。
**3. 寻找最好的函数（Finding Best Function）** `[01:00:30]`

- **定义**：如何找到那组最好的参数？
- **方法**：通过“最大似然估计（Maximum Likelihood Estimation）”来求解 `μ`* 和 **`Σ`*。教授再次强调，其结果是“很 trivial 的”（即平均值和协方差矩阵）。
**模型选择的艺术：为什么是高斯分布？**

教授提出了一个关键问题：“为甚么要用 Gaussian，为甚么不选别的？”他幽默地回答：“如果你选了别的机率模型，你也会问我同样的问题。”这强调了一个核心观点：**选择哪种概率分布（或模型），是“你人的智慧”，而不是“人工智慧”**。

这种选择涉及到“偏差（bias）”和“方差（variance）”的权衡：

- **简单模型**（参数较少，如 Naive Bayes 或共享协方差的高斯）：`bias` 就大，`variance` 就小。它可能无法很好地捕捉数据的复杂性，但对新数据更稳定。
- **复杂模型**（参数较多，如独立协方差的高斯）：`bias` 就小，`variance` 就大。它可能能更好地拟合训练数据，但容易过拟合，对新数据不稳定。
因此，我们需要根据“data set”的特点来决定“要用怎么样子的机率模型”。

**个人感受**

教授在这里对机器学习三步骤的总结，将前面零散的知识点串联起来，形成了完整的理论框架。他对模型选择的哲学思考，更是点睛之笔，提醒我们机器学习并非完全的自动化，人类的经验和判断力依然至关重要。

**精华收获**

生成式模型是机器学习三步骤的完美实践。模型选择是人类智慧的体现，需要在模型复杂度（偏差）和泛化能力（方差）之间进行权衡，以适应不同的数据集。

### 简化模型：朴素贝叶斯分类器与特征独立性假设 `[01:03:00-01:06:00]`

**核心观点**

朴素贝叶斯分类器通过假设特征之间相互独立来进一步简化模型。这种假设在某些情况下能提供良好性能，但在特征间存在强相关时会导致模型偏差过大。

**深度阐述**

在模型选择的讨论中，教授引入了另一种常见的简化假设：**特征独立性**。

**1. 特征独立性假设**

我们知道，宝可梦的 `x` 是由多个特征（例如 7 个数值）组成的向量。之前的高斯分布模型考虑了特征之间的协方差。现在，我们可以假设“每一个 dimension 它从机率模型，产生出来的机率是 independent 的”。这意味着：

`P(x|C) = P(x1|C) `*` P(x2|C) `*` ... `*` P(xK|C)`*

（`[01:03:00]` 视觉/结构信息描述：投影片展示了特征独立性假设下的 P(x|C) 拆解公式。）

如果每个维度都是独立的，那么每个 `P(xi|C)` 都可以用一维的高斯分布来建模。这意味着，原来高维高斯分布的“covariance matrix”，其“不是对角线的地方，值都是 0”，只有“对角线的地方，有值”。这进一步“减少你的参数量”，得到一个“更简单的模型”。

**2. 朴素贝叶斯分类器（Naive Bayes Classifier）**

当我们将这种特征独立性假设应用于分类模型时，它就被称为“朴素贝叶斯分类器（Naive Bayes Classifier）”。教授指出，它前面有一个“Naive”（朴素），因为“它真的很 naive”，真的很简单。

**3. 朴素贝叶斯的适用性**

那么，朴素贝叶斯分类器是强还是不强呢？教授解释说，“它强不强是 depend on 你的假设是不是精准的”。

- 如果“不同的 dimension 之间是 independent 这件事情是很切合实际的”，那么朴素贝叶斯确实能提供“很好的 performance”。
- 但如果这个假设“是很不成立的话”，例如宝可梦的攻击力与防御力之间可能存在正相关，那么朴素贝叶斯模型的“bias 就太大了”，它就“不是一个好的 Classifier”。
教授提到，他尝试了这种简化模型，结果是“坏的”，这表明“model 不同的 feature 间的 covariance 我看也是必要的”。例如，攻击力和防御力之间可能存在正相关，这种相关性是不能被忽略的。

**4. 非高斯分布的特征**

此外，教授还提醒，并非所有特征都适合用高斯分布建模。例如，如果一个特征是“binary 的”（二元的），如“是不是神兽”，它的输出是 0 或 1。此时，用高斯分布来建模就“太自欺欺人了”。更合适的选择是“伯努利分布（Bernoulli distribution）”。这再次强调了模型选择的灵活性和对数据特性的理解。

**精华收获**

朴素贝叶斯分类器通过假设特征独立性来简化模型，减少参数。它在特征独立性假设成立时表现良好，但若假设不成立，则会导致高偏差。模型选择需根据特征类型（如连续、二元）和特征间关系来决定合适的概率分布。

### 从生成式模型到判别式模型：数学推导与线性边界 `[01:06:00-01:10:00]`

**核心观点**

通过对生成式模型中后验概率 P(C|x) 的数学推导，可以发现当类别共享协方差矩阵时，其决策边界自然地简化为线性形式，从而引出判别式模型的直观性。

**深度阐述**

在课程的最后一部分，教授进行了一系列重要的数学推导，旨在“分析这项 Posterior Probability”，即 `P(C1|x)`。

**1. 后验概率的 Sigmoid 形式**

首先，通过简单的代数整理，可以将 `P(C1|x)` 转换为一个更简洁的形式：

`P(C1|x) = 1 / (1 + exp(-z))`

其中，`z = ln [P(x|C1) `*` P(C1) / (P(x|C2) `*` P(C2))]`。

教授指出，这个 `1 / (1 + exp(-z))` 函数就是著名的“Sigmoid function”（S 型函数）。（`[01:07:00]` 视觉/结构信息描述：投影片展示了 Sigmoid 函数的图像，其输出值在 0 到 1 之间，随着 z 的增大从 0 趋近于 1，随着 z 的减小从 1 趋近于 0。）

**2. z 的复杂推导（一般情况）**

接着，教授开始推导 `z` 的具体形式。`z` 涉及到 `ln(P(C1)/P(C2))` 和 `ln(P(x|C1)/P(x|C2))`。

- `ln(P(C1)/P(C2))` 可以简化为 `ln(N1/N2)`，其中 `N1` 和 `N2` 是训练数据中 Class 1 和 Class 2 的样本数量。
- `ln(P(x|C1)/P(x|C2))` 则涉及到两个高斯分布的概率密度函数之比取对数。这个推导过程“是数学比较多”，展开后会得到一个非常复杂的表达式，其中包含 `x` 的二次项（`x^T `*` Σ^(-1) `*` x`）。（`[01:08:00]` 视觉/结构信息描述：投影片展示了 z 的详细推导过程，包括高斯分布公式的代入、对数运算、展开和合并项。）
教授幽默地表示：“如果你觉得这很无聊的话，你就睡一下，听一下结论就好。”这表明他理解这部分数学的复杂性，但其结论至关重要。

**3. z 的简化推导（共享协方差矩阵情况）**

关键的简化发生在“我们刚才有说过呢，一般我们会假设，covariance matrix 是共用的”，即 `Σ1 = Σ2 = Σ`。

在这种情况下，`z` 的表达式会大大简化。许多包含 `Σ1` 和 `Σ2` 的项会相互抵消，特别是那些 `x` 的二次项。

最终，`z` 被简化为：

`z = (μ1 - μ2)^T `*` Σ^(-1) `*` x + b`

其中，`b` 是一个不含 `x` 的常数项，而 `(μ1 - μ2)^T `*` Σ^(-1)`** 也是一个常数向量，我们可以将其记为 **`w^T`**。*

所以，`z` 最终的形式是：`z = w^T `*` x + b`**。*

**4. 结论：线性决策边界与判别式模型**

这意味着，当类别共享协方差矩阵时，后验概率 `P(C1|x)` 可以写成 `σ(w^T `*` x + b)`** 的形式。*

教授强调：“从这个式子，你就可以看出来说，为甚么我们今天把 Σ1 跟 Σ2 共用的時候，你的 class 1 跟 class 2 的 boundary 会是 linear。”因为 `w^T `*` x + b = 0`** 正是一个线性方程，它定义了一个超平面（在二维空间中是直线），作为分类的决策边界。*

**5. 从生成式到判别式**

教授在课程的最后提出了一个发人深省的问题：

“假设我们最终的目标，都是要找一个 vector w，都是要找一个 constant b。我们何必先去搞个机率，算出一些 μ, Σ 甚么的，然后再把它搞起来，再得到 w 跟 b？这不是舍近求远吗？”

他指出，我们最终需要的只是 `w` 和 `b` 来定义决策边界。那么，“我们能不能够直接把 w 跟 b 找出来？”这个问题，正是从“生成式模型（Generative Model）”转向“判别式模型（Discriminative Model）”的关键。生成式模型先建模 `P(x|C)` 和 `P(C)`，再通过贝叶斯定理得到 `P(C|x)`。而判别式模型则直接建模 `P(C|x)`，或者直接学习决策边界。这为下一份投影片（下一堂课）的内容埋下了伏笔，预示着我们将探索更直接的分类方法，如逻辑回归。

**个人感受**

这部分数学推导虽然复杂，但教授通过清晰的步骤和最终的简化，揭示了生成式模型与线性分类器之间的深层联系。他最后提出的“舍近求远”的疑问，更是巧妙地引导听众思考不同模型范式的优劣，激发了对未来学习的兴趣。

**延伸思考**

这个推导是理解逻辑回归（Logistic Regression）等判别式分类器的重要桥梁。它表明，在某些假设下（如共享协方差的高斯分布），生成式模型可以导出与判别式模型形式相同的决策函数。这两种看似不同的方法，在特定条件下殊途同归。

**精华收获**

通过数学推导，当生成式模型假设类别共享协方差矩阵时，后验概率 P(C|x) 可以表达为 Sigmoid 函数作用于一个线性函数 (w^T * x + b)。这直接解释了为何共享协方差矩阵会产生线性决策边界，并引出了直接学习 w 和 b 的判别式模型思想。*
