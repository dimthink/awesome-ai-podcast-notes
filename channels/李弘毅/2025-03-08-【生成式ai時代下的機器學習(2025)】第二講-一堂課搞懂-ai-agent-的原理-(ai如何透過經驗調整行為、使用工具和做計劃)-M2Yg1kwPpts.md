### AI Agent 的核心概念與運作循環 `[00:00-03:40]`

**核心观点**

AI Agent 的本质在于其自主性与目标导向，而非被动执行指令。它通过一个持续的“观察-分析-行动”循环，在复杂环境中自主实现人类设定的高层目标。

**深度阐述**

在课程伊始，李弘毅教授便开宗明义地指出，"AI Agent"是一个被广泛使用但定义模糊的词汇。他首先进行了一个重要的“免责声明” `[00:00-01:00]`，强调不同人对AI Agent的理解可能大相径庭，甚至有人认为当前基于大型语言模型（LLM）的AI Agent并非“真正”的Agent，只有具备物理身体的机器人才能称之为Agent。教授明确表示，本课程将聚焦于他所定义的AI Agent，即一种能够自主达成目标的智能体。

传统的AI使用方式，通常是人类给出明确的指令，AI则“一个口令一个动作”地执行，例如输入“AI Agent的翻译是什么”，AI便直接给出翻译结果，不多做任何额外的事情 `[01:00-01:30]`。这种模式下，AI扮演的是一个被动的工具角色。

然而，AI Agent 的核心突破在于，人类不再提供明确的行为或步骤指示，而是**只给出AI一个高层目标** `[01:30-01:45]`。至于如何达成这个目标，AI Agent 需要自己思考、规划并执行。教授举例说明，如果你给AI一个研究议题，一个合格的AI Agent 应该能够：

- 自己提出假设
- 自己设计实验
- 自己进行实验
- 自己分析结果
- 如果结果与假设不符，还能回过头去修正假设 `[01:45-02:00]`
这意味着AI Agent 需要处理的目标通常涉及**多个步骤**，并需要与**环境进行复杂且不可预测的互动**。因此，AI Agent 必须具备根据当前状况灵活调整计划的能力。

那么，AI Agent 是如何做到这一点的呢？教授通过一张简洁的投影片 `[02:00-03:00]` 简化了其背后的运作过程，描绘了一个经典的“感知-决策-行动”循环：

1. **目标 (Goal)**：这是人类赋予AI Agent 的初始输入。
1. **观察 (Observation)**：AI Agent 观察当前环境的状况。
1. **分析与决策 (Analysis & Decision)**：根据观察到的信息，AI Agent 分析现状，并决定下一步要采取的行动。
1. **行动 (Action)**：AI Agent 执行其决定的行动。
1. **环境影响 (Environment Impact)**：行动会改变环境的状态。
1. **新观察 (New Observation)**：环境的变化导致AI Agent 看到新的观察结果。
这个循环会持续进行，直到AI Agent 达成预设的目标为止。

为了让这个抽象概念更具体，教授以大家耳熟能详的 **AlphaGo** 为例 `[03:00-03:40]`。AlphaGo 的目标是“下棋要赢”，它的观察是“棋盘上黑白子的位置和盘势”，而它的行动则是在19x19的棋盘上选择一个落子位置。每一次落子都会改变环境（对手的响应），从而产生新的观察，促使AlphaGo 采取下一个行动。这个例子清晰地展示了AI Agent 的核心运作原理。

**精华收获**

AI Agent 的本质是“目标导向的自主性”，它通过持续的“观察-决策-行动”循环，在动态环境中灵活适应，自主完成复杂任务，而非仅仅执行预设指令。

### 從強化學習到大型語言模型驅動的 AI Agent `[03:40-08:40]`

**核心观点**

AI Agent 的概念并非新生，但LLM的崛起为其注入了通用能力，使其能摆脱传统强化学习的局限，直接作为Agent的核心驱动力。

**深度阐述**

教授指出，AI Agent 的这种运作模式，对于熟悉强化学习（Reinforcement Learning, RL）的听众而言，会感到非常熟悉 `[03:40-04:00]`。事实上，过去要打造AI Agent，RL 算法往往被认为是首选。RL 的核心思想是让 Agent 通过与环境互动来学习，目标是最大化一个预定义的“奖励 (reward)”。在下围棋的例子中，赢棋可以设定为正一的奖励，输棋则为负一，Agent 便会学习如何最大化这个奖励 `[04:00-04:30]`。

然而，传统的 RL 算法存在显著的局限性 `[04:30-06:00]`：

1. **任务特异性 (Task Specificity)**：每个任务都需要单独训练一个模型。AlphaGo 经过大量训练可以下围棋，但它不能直接下西洋棋或将棋。即使是 AlphaZero 能够玩多种棋类，那也是通过对不同棋类进行独立训练得到的不同模型，而非同一个模型具备通用能力。
1. **奖励工程 (Reward Engineering)**：定义一个有效的奖励函数是极其困难且主观的。例如，在训练一个AI程序员时，编译错误可以给予负一的奖励，但这个“-1”的数值是任意的，没有人能说清楚为什么是-1而不是-10或-17.7。教授形象地称之为“通靈的東西”，意指其缺乏明确的理论依据，需要凭经验和直觉去调整。
正是为了克服这些局限，人们产生了新的想法 `[06:00-06:30]`：**能否直接将大型语言模型（LLM）用作AI Agent？**

在这种新的范式下，LLM 自身就成为了 Agent 的核心。当需要告诉它目标时，直接用文字输入；当需要描述环境时，也将其转化为文字叙述（尽管现在多模态LLM可以直接处理图片等）。LLM 产生行动的方式，同样是输出一段文字，这段文字再被转译成真正可执行的动作 `[06:30-07:30]`。这个过程同样会改变环境，产生新的观察，从而驱动 Agent 持续运作直到目标达成。

教授强调，当前 AI Agent 再次爆红，并非因为 Agent 本身有了什么全新的技术突破，而是**LLM 能力的显著增强** `[07:30-08:40]`。人们开始思考，既然 LLM 已经具备了一定程度的通用能力，为什么不直接将它们用作 Agent，来实现人类一直以来对智能体的渴望呢？从 LLM 的角度来看，它所做的事情并没有本质上的不同，依然是其最擅长的“文字接龍 (text completion)”能力。因此，AI Agent 更多地被视为 LLM 的一种高级应用，而非 LLM 的一项全新技术。

**个人感受**

教授对RL奖励工程的“通靈的東西”这一描述，生动地揭示了传统AI训练中一个核心但又难以言喻的痛点。这种幽默的表达方式，不仅让听众会心一笑，也深刻传达了LLM作为Agent的优势：它能直接理解和处理人类语言中的反馈，而无需抽象的数值奖励。

**精华收获**

LLM作为AI Agent的核心，通过其强大的文字接龙能力，克服了传统RL在任务特异性和奖励工程上的挑战，将AI Agent的构建从特定任务的训练转向了通用能力的直接应用。

### LLM 作為 AI Agent 的現狀與挑戰：從棋局到虛擬世界 `[08:40-17:30]`

**核心观点**

尽管LLM作为AI Agent潜力巨大，但在复杂、规则严谨的任务中，其推理和规划能力仍显不足，但其无限的输出可能性和直接反馈处理能力，使其在模拟人类行为和使用工具方面展现出独特优势。

**深度阐述**

既然LLM被寄予厚望，那么它在实际应用中表现如何呢？教授首先以“下棋”为例，探讨了LLM的推理能力 `[08:40-11:30]`。

- **上古时代（2022年，ChatGPT之前）的尝试**：在著名的BigBench基准测试中，人们尝试让当时的语言模型下西洋棋。由于模型无法直接看图，棋盘状态需要转化为文字描述输入。结果显示，当时的LLM无法给出正确答案，但较强的模型能遵循棋规，而较弱的模型则“乱走”，完全不理解规则。 `[08:40-10:00]` (视觉/结构信息描述：投影片展示了将棋盘转化为文字描述的例子，以及不同模型输出的棋步，橙色为正确，绿色为模型输出，虚线为弱模型乱走)。
- **“惊天动地”的棋局（ChatGPT o1 vs. DeepSeek-R1）**：教授幽默地引用了一个广为流传的视频，展示了两个“最强”LLM下西洋棋的惨烈场景。它们不仅频繁违反棋规（如兵当马用，主教无视阻挡，甚至凭空变出棋子），DeepSeek-R1甚至在吃掉自己的兵后宣布胜利，而ChatGPT竟然“想了一下”后投降 `[10:00-11:30]`。这个例子生动地说明，即使是当前最强的LLM，在需要严谨逻辑推理和规则遵守的任务上，仍有很长的路要走。
教授再次强调，从LLM的角度看，它作为AI Agent所做的，依然是其核心能力——**文字接龍** `[11:30-12:40]`。AI Agent 并非LLM的新技术，而是其应用。因此，本课程后续讨论的所有内容，都将基于现有LLM的能力，不涉及模型训练。

回顾历史，AI Agent 的热潮并非首次。2023年ChatGPT爆红后，曾有一波AI Agent热潮，AutoGPT便是当时的代表 `[12:40-13:30]`。然而，这股热潮很快消退，因为人们发现这些Agent的实际能力远不如网红们吹嘘的那么强大。

尽管存在挑战，LLM驱动的AI Agent 仍有其独特优势 `[13:30-15:30]`：

1. **近乎无限的行动可能性 (Infinite Action Space)**：相较于AlphaGo只能在19x19个位置中选择落子，LLM可以生成无穷无尽的文本输出，这意味着其行动不再受限于预设的有限选项。这种灵活性使得LLM Agent 能够直接调用工具，解决原本无法解决的问题。
1. **直接反馈与丰富信息 (Direct Feedback vs. Reward)**：传统RL需要抽象的奖励信号，而LLM Agent 可以直接理解并处理详细的错误信息（例如编译错误日志），从而更精准地调整行为。错误日志提供了比单一数值奖励更丰富、更具指导性的信息。
教授随后举例说明了LLM Agent 的实际应用 `[15:30-17:30]`：

- **AI 村民组成的虚拟村庄 (Virtual Villagers, 2023)**：这是一个早期的实验，村庄中的NPC（非玩家角色）由LLM驱动。每个NPC都有预设目标（如办派对、准备考试），它们观察环境（以文字描述，如“Eddy在读书，厨房里有柜子”），然后决定行动（如“上床睡觉”），并通过转译器将文字行动转化为实际指令。 `[15:30-16:50]` (视觉/结构信息描述：投影片展示了NPC观察到的文字信息和其决定的行动)。
- **Minecraft 中的 AI NPC**：更宏大的实验，将Minecraft中的NPC替换为AI Agent，据说它们甚至能组织自己的经济体系和政府（教授对此表示怀疑，并幽默地指出“是真的还假的啦”）。 `[16:50-17:30]`
**个人感受**

教授对LLM下棋的描述，特别是DeepSeek-R1宣布胜利和ChatGPT投降的细节，充满了黑色幽默，让人在忍俊不禁的同时，也深刻体会到当前LLM在逻辑推理和规则遵守上的局限性。这种生动的案例比枯燥的理论更能触动人心，也更让人期待未来LLM的进步。

**精华收获**

LLM作为AI Agent，其优势在于无限的行动空间和直接处理反馈的能力，使其在模拟复杂行为和调用工具方面潜力巨大。然而，在需要严谨逻辑和规则遵守的任务中，LLM仍面临挑战，其“文字接龍”的本质决定了其推理能力尚待提高。

### AI Agent 使用電腦與即時互動：從鍵盤滑鼠到語音模式 `[17:30-24:00]`

**核心观点**

AI Agent 的发展趋势是实现更深层次的现实世界互动，包括像人类一样操作电脑，以及进行非回合制的即时语音对话，这需要Agent具备处理多模态信息和灵活适应动态环境的能力。

**深度阐述**

在这一部分，教授将讨论AI Agent 如何更直接地与现实世界互动，首先是**让AI真正使用电脑** `[17:30-20:30]`。这听起来有些矛盾，因为AI本身就是电脑程序，但这里的“使用电脑”指的是像人类一样操作图形用户界面。

- **代表性案例**：Cloud Computer Use 和 ChainGPT Operator。这些Agent 的目标是用户输入的任务（如“订披萨”、“网上购物”），它们的观察是电脑屏幕画面（现在多模态LLM可以直接处理图片），而行动则是模拟按键或点击鼠标。 `[18:00-18:40]`
- **历史回顾**：让AI使用电脑并非新野望。早在2017年，就有论文《Words of Bits》尝试让AI Agent 在网页上执行任务。但当时没有LLM，方法是硬训练一个CNN来识别屏幕画面并输出鼠标点击位置或按键。教授幽默地称之为“死前时代”甚至“旧时期时代” `[18:40-19:40]`。
- **LLM 时代的 Web Agent**：随着LLM的出现，2023年夏天涌现了Mine to Web、Web Arena、Visual Web Arena 等项目，它们让LLM通过观察屏幕画面或HTML代码来决定行动，以解决问题（如订机票）。 `[19:40-20:30]`
除了直接使用电脑，AI Agent 还能**训练其他AI模型** `[20:30-22:00]`。这是一种“AI 创造 AI”的体现。

- **ML Engineer Agent**：例如 AIDE 框架，其目标是“通过 strong baseline”，LLM 接收训练数据，编写代码训练模型，根据模型准确率再修改代码，循环往复。
- **AI Co-scientist**：Google 曾发布一个AI Co-scientist，它不能真正做实验，但能根据研究想法规划出完整的实验方案。教授提到，其博客中甚至有夸张案例称AI Agent 两天完成人类十年研究，但真实性有待考证。
接下来，教授引入了更高级的互动模式：**即时互动 (Real-time Interaction)** `[22:00-24:00]`。

- **回合制 vs. 即时互动**：传统的AI Agent 互动（如ChatGPT）是回合制的：输入一段文字，输出一段文字。但现实世界和人际交流往往是即时的、非回合制的，例如对话中可能互相打断，或通过“嗯”、“好”等非语义反馈来表示在听。
- **GPT-4o Voice Mode**：教授指出，GPT-4o 的高级语音模式在某种程度上实现了这种即时互动。他举例说明 `[23:00-24:00]` (视觉/结构信息描述：投影片展示了AI讲故事时，用户插入“好”或“不是我要听的故事”，AI能即时理解并调整行为)。这种能力要求AI Agent 不仅要处理观察，还要判断哪些观察需要立即改变行动，哪些可以忽略。教授提到，实现这种非回合制互动超出了本课程范围，但推荐了一篇相关论文供有兴趣者深入研究 `[23:40-24:00]`。
**个人感受**

教授对“AI使用电脑”这一概念的阐述，以及对早期Web Agent研究的“死前时代”的幽默称呼，都体现了他对AI发展历史的深刻理解和对技术演进的敏锐洞察。而GPT-4o语音模式的例子，则让人对未来AI与人类的自然交互充满了期待。

**精华收获**

AI Agent 的发展正从文本交互迈向更复杂的现实世界互动，包括像人类一样操作电脑界面，以及实现即时、非回合制的语音对话。这要求Agent具备多模态感知、灵活的决策调整和对动态环境的实时响应能力。

### AI Agent 如何從經驗中學習與調整行為：記憶模組與 RAG `[24:00-40:00]`

**核心观点**

AI Agent 能够根据经验调整行为，但并非通过模型参数更新，而是通过智能的记忆管理机制（Read、Write、Reflection模块），利用RAG技术高效检索相关经验，并优先学习正面反馈。

**深度阐述**

AI Agent 如何根据过去的经验或环境反馈来调整行为，是其智能性的关键体现 `[24:00-24:30]`。

- **无需参数调整的行為改變**：教授强调，在本课程中，不涉及模型参数的训练。那么，在不更新模型参数的情况下，LLM 如何改变其行为呢？答案在于其“文字接龍”的本质 `[24:30-26:00]`。当接收到错误信息（例如编译错误日志）时，这些信息会成为LLM输入的一部分。不同的输入会导致不同的接龍结果，从而使模型产生不同的、修正后的代码。LLM 的强大之处在于，它能直接理解错误信息，并据此调整后续输出，而无需复杂的参数微调。
然而，如果将所有过去的经验都一股脑地提供给LLM，会带来新的问题 `[26:00-28:30]`：

- **超长自传式记忆的困境 (Super Autobiographical Memory)**：如果每次决策都回顾“一生”的经历，当经验过长时，LLM可能因算力不足或信息过载而无法做出正确决策。教授以人类的“超忆症”为例，形象地说明了这种困境：虽然记忆力超群，但患者往往难以进行抽象思考，生活也可能被琐碎的回忆所困扰。因此，让AI Agent 记住所有细节，并非好事。
为了解决这个问题，AI Agent 需要**记忆模組 (Memory Modules)**，这类似于人类的长期记忆 `[28:30-30:00]`：

- **Read 模組 (Retrieval)**：当AI Agent 遇到新问题时，Read 模組会从长期记忆中筛选出与当前问题相关的经验，并将其置于LLM的输入之前。这样，模型只需根据这些相关的经验和当前的观察进行决策，避免了处理冗长无关信息。这本质上就是 **RAG (Retrieval Augmented Generation)** 技术，只是这里的“资料库”是Agent 自身的经历，而非整个互联网。
教授介绍了 **Streambench** 这个基准测试 `[30:00-33:00]`，它用于评估AI Agent 根据经验调整行为的能力。Agent 依次解决一系列问题，并接收二进制反馈（对或错）。

- **RAG 的有效性**：实验结果表明，使用RAG方法从记忆中挑选最相关的经验，能显著提高Agent 的平均准确率，远超不学习或随机选择经验的模型 `[31:00-33:00]` (视觉/结构信息描述：投影片展示了Streambench的横轴为时间步/问题数，纵轴为平均准确率，RAG方法（粉红色线）明显优于无学习（灰色线）和随机选择（黄色线）)。
Streambench 还发现了一个有趣的现象 `[33:00-36:00]`：

- **正面反馈优于负面反馈**：对于LLM而言，提供**正面的例子**（哪些类似问题得到了正确答案）比提供**负面的例子**（哪些问题得到了错误答案）更有效，更能引导模型得到正确答案。如果只使用负面例子，甚至可能有害。这与一些研究结论不谋而合：与其告诉LLM“不要做什么”，不如直接告诉它“要做什么”。 `[33:00-36:00]` (视觉/结构信息描述：投影片展示了不同数据集上，只用正面例子（绿色）效果最好，负面例子（红色）甚至有害的柱状图)。
除了 Read 模組，还有另外两个关键模組 `[36:00-39:00]`：

- **Write 模組**：负责决定哪些信息应该被存储到长期记忆中，哪些是无关紧要的琐事可以忽略。这个模組本身也可以是一个LLM，它会“自问自答”：这件事重要到需要被记住吗？
- **Reflection 模組 (反思模組)**：对记忆中的信息进行更高层次的抽象和整理，从中提炼出新的想法或建立知识图谱。例如，从“喜欢的人每天搭同一班公车”和“她今天对我笑了”推断出“她喜欢我”的“错觉”。这些新的想法可以辅助 Read 模組进行更有效的检索，帮助模型做出更好的决策。
教授最后以 **ChatGPT 的记忆功能**为例 `[39:00-40:00]`，说明这些概念已在实际产品中落地。当用户告诉ChatGPT“我周五下午要上机器学习课，请记住”，ChatGPT会启动其Write模組，将信息存入记忆。用户可以在设置中查看和管理这些记忆。然而，教授也指出，模型的记忆并非完美，它会经过“昇华反思”后才存入，可能出现错误（例如将教授误认为学生）。

**个人感受**

教授将AI Agent的记忆问题与人类的“超忆症”进行类比，不仅生动形象，更引发了对智能体记忆本质的哲学思考：并非越多越好，而是“智慧地记忆”才更重要。而关于正面反馈优于负面反馈的发现，也为我们日常与LLM互动提供了实用的指导。

**精华收获**

AI Agent通过Read、Write、Reflection等记忆模組，实现了基于经验的学习，其中Read模組利用RAG技术高效检索相关信息。研究表明，LLM更擅长从正面例子中学习，且记忆的质量而非数量是关键。

### AI Agent 如何使用工具：從搜尋引擎到自我創造 `[40:00-1:03:00]`

**核心观点**

工具是AI Agent 突破自身能力边界的关键，通过“函数调用”机制，LLM能利用外部资源（如搜索、代码、其他AI）扩展其感知和行动范围，但需警惕工具的局限性，并培养Agent对工具输出的判断力。

**深度阐述**

在这一部分，教授深入探讨了AI Agent 如何利用外部工具来增强自身能力 `[40:00-41:00]`。他首先给出了“工具”的定义：**你只需要知道如何使用它，而无需了解其内部运作原理**。教授幽默地将“工具人”比作这种关系，因为“别人没有人在意肥宅的心思，只知道他能不能够修电脑而已”。

那么，LLM Agent 常用的工具有哪些呢 `[41:00-42:00]`？

- **搜尋引擎 (Search Engine)**：最常用，也是RAG技术的核心。
- **程式碼執行器 (Code Interpreter)**：LLM 可以编写并执行自己的代码，这些代码本身就是工具。
- **其他 AI 模型 (Other AIs)**：例如，一个只能处理文本的LLM 可以调用能看图片、听声音的多模态AI，或者在遇到难题时，调用一个计算资源消耗更大的“大哥”模型来解决。
教授指出，使用工具的本质就是**函數調用 (Function Calling)** `[42:00-43:00]`。LLM 只需要知道函数的输入和输出格式，而无需关心其内部实现。许多LLM现在都内置了Function Calling功能。

教授详细讲解了一种**通用的工具使用方法** `[43:00-46:30]`：

1. **指令与格式**：明确告诉LLM如何使用工具，例如将工具调用指令放在 `Tool[...]` 符号中，工具输出放在 `Output[...]` 符号中。
1. **工具描述**：提供可用工具的列表及其使用说明（例如 `Temperature(地點, 時間)` 函数）。
1. **Prompt 组合**：将工具使用说明（System Prompt）和用户问题（User Prompt）一同输入给LLM。System Prompt 优先级更高，用于设定Agent 的行为模式。
1. **执行桥梁**：当LLM输出 `Tool[Temperature(高雄, 某年某月某日)]` 时，这只是文本。开发者需要一个“桥梁”程序来解析这段文本，实际调用 `Temperature` 函数，并将函数返回的真实结果（例如32度）封装在 `Output[32]` 中，再作为新的输入反馈给LLM。
1. **继续接龍**：LLM 接收到工具输出后，会继续其文字接龍，最终生成用户友好的答案（例如“高雄气温是摄氏32度”）。这个过程可以完全对用户透明。
**RAG (Retrieval Augmented Generation)** 再次被提及 `[46:30-47:00]`，它本质上就是LLM 使用搜尋引擎这个工具。

工具的强大之处还在于**多模態處理 (Multi-modal Processing)** `[47:00-50:00]`。教授以处理语音为例：一个文本LLM 本身无法理解语音，但它可以调用语音识别、情绪辨识等工具，将语音转化为文本，再进行分析。教授特别提到了其大助教的文章，展示了这种方法在语音相关任务上的卓越效果，甚至超越了当时号称能直接听懂语音的模型 `[49:00-50:00]`。

然而，工具使用也面临挑战 `[50:00-51:30]`：

- **工具过多**：当有上百上千个工具时，LLM 无法一次性阅读所有说明书。
- **解决方案**：引入**工具選擇模組 (Tool Selection Module)**，类似于RAG的Read模組，它能根据当前状态从工具库中检索出最相关的工具说明，只将这些相关说明提供给LLM。
更令人兴奋的是，**LLM 甚至可以自己打造工具** `[51:30-52:30]`。由于LLM 能够编写代码，它完全可以自己写一个函数，如果这个函数运行顺利，就可以将其添加到自己的工具包中，供未来使用。这与Agent 存储成功经验的机制异曲同工，只是这里存储的是可执行的代码。

尽管工具强大，但教授提醒我们**过度相信工具的危险性** `[52:30-55:30]`：

- **工具幻觉 (Tool Hallucination)**：Google AI Overview 曾因RAG功能，在回答“披萨上的芝士粘不住怎么办”时，建议用“无毒胶水”粘住，这源于Reddit上的一个玩笑。LLM 无法判断这是玩笑还是真实信息。
- **AI 对工具输出的判断力**：LLM 并非完全盲信工具。教授举例，当给LLM一个荒谬的温度（如高雄10000度）时，LLM 会判断这不合理，并指出工具输出有误。这表明LLM 内部存在**对世界的信念 (Internal Knowledge)**，它会与**外部知识 (External Knowledge)** 相互拉扯，最终得出结果。
那么，**什么样的外部知识更容易说服AI** `[55:30-1:01:00]`？

- **与内部信念的差距**：外部知识与LLM 自身信念差距越大，越不容易被相信；差距越小，越容易被相信。
- **模型信心**：LLM 对自身答案的信心越低，越容易被外部信息动摇。
- **AI 撰写的内容**：当两篇文章观点相左时，LLM 倾向于相信**其他AI撰写的内容**，即使这些AI的观点与自身不同。这可能是因为AI撰写的文字更具结构、条理和简洁性。
- **发布时间**：LLM 倾向于相信**最新的文章**。当两篇文章观点冲突时，它会采信发布时间更晚的那篇。
- **呈现方式**：教授的实验发现，LLM 倾向于喜欢**“好看的模板”**，即视觉上更吸引人的网页呈现方式，这可能会影响其对文章内容的信任度（尽管教授也谨慎地指出，这可能是对特定颜色或布局的偏好，而非普遍的“好看”）。
最后，教授提醒我们，**使用工具并不总是更有效率** `[1:01:00-1:03:00]`。就像计算3乘以4，人类心算比按计算器更快。对于LLM而言，如果自身能力足以解决问题，直接生成答案可能比调用工具更高效。早期的研究中，LLM 甚至会调用翻译系统，但现在的LLM 自身翻译能力已经非常强大，直接翻译反而更好。

**个人感受**

教授将“工具人”的梗引入对工具的定义，既幽默又精准。而Google AI Overview的“胶水披萨”案例，则以一个极具冲击力的反例，警示了AI Agent在缺乏常识判断时可能带来的荒谬结果，强调了AI判断力的重要性。关于AI更相信AI同类、相信新信息、甚至相信“好看”模板的发现，则为未来AI内容生成和信息传播提供了深刻的启示。

**精华收获**

AI Agent通过函数调用机制，能有效利用外部工具（搜索、代码、其他AI）扩展其能力边界。然而，Agent需要具备对工具输出的批判性判断力，平衡内部知识与外部信息。研究表明，LLM更倾向于相信与自身信念接近、由其他AI生成、发布时间较新且呈现方式更佳的外部信息。同时，并非所有任务都适合工具调用，有时LLM自身能力足以高效解决。

### AI Agent 的規劃能力：從簡單步驟到腦內劇場 `[1:03:00-1:28:00]`

**核心观点**

AI Agent 的规划能力是实现复杂目标的关键，但其并非天生强大。通过显式规划、动态重规划、树搜索以及内部模拟“世界模型”，Agent 能逐步提升规划水平，但仍需警惕“过度思考”的陷阱。

**深度阐述**

本节探讨AI Agent 最具挑战性的能力之一：**規劃 (Planning)** `[1:03:00-1:04:30]`。

- **显式规划**：我们可以强迫LLM 生成一个明确的规划，即一系列达成目标的行动步骤。然后将这个规划作为输入的一部分，指导LLM 采取行动。许多论文都尝试过这种“先规划后执行”的方法。
然而，现实世界充满变数，**计划总是赶不上变化** `[1:04:30-1:07:00]`：

- **动态环境**：从行动到观察的环节并非由模型完全控制，外部环境的随机性可能导致观察结果与预期不符，使原有计划失效。例如，下棋时无法预测对手的每一步；使用电脑时可能突然弹出广告窗口。
- **重规划 (Re-planning)**：因此，AI Agent 需要具备弹性，能够根据新的观察结果，重新思考并修改计划。
那么，LLM 到底有没有规划能力呢 `[1:07:00-1:10:00]`？

- **“百万YouTuber 计划”**：LLM 可以生成看起来非常合理的计划（例如如何成为百万YouTuber），但教授指出，这很可能只是基于其训练数据中的“文字接龍”，而非真正的规划能力。
- **Codex 12B (2022)**：早期研究表明，LLM 能够将“刷牙”等任务分解为一系列步骤，这显示出一定程度的规划能力。
为了更严谨地测试LLM 的规划能力，研究者设计了 **PlaneBench** `[1:10:00-1:14:30]`：

- **正常方块世界**：测试Agent 在标准积木堆叠任务中的规划能力。LLM 表现良好，但教授幽默地用“司令官与长椅”的故事 `[1:11:30-1:12:30]` 暗示，这可能只是LLM 见过类似问题，照本宣科而已，并非真正理解规划。
- **神秘方块世界 (Mysterious Block World)**：为了避免LLM 依赖训练数据，这个测试引入了非传统的方块行为（攻击、吞噬、屈服、征服）和复杂规则，要求Agent 解决一个抽象目标（如“让C方块渴望A方块”）。
- **结果**：2023年的GPT-4在神秘方块世界中表现不佳（仅9%准确率），表明其存在“过度拟合”于正常方块世界的问题。但2023年9月后的O1系列模型（LLaMA 3.1 405B, O1-mini/preview）在推理能力上有所提升，显示出解决神秘方块世界的潜力 `[1:13:30-1:14:30]` (视觉/结构信息描述：投影片展示了神秘方块世界中，不同模型准确率的对比图，O1系列表现出希望)。
另一个规划基准测试是 **旅行社 Agent (Travel Agent Benchmark, 2024)** `[1:14:30-1:17:30]`：

- **任务**：AI Agent 需要根据用户需求（如三天行程、预算1900元）规划旅行，并使用工具（如上网搜索）。
- **初期表现**：2024年初的模型（GPT-3.5, GPT-4）在没有工具辅助下，成功率几乎为0%，常常做出不符合常识或超出预算的规划。例如，飞机8点起飞，却还安排了午餐；或者在预算不足时，只微调早餐午餐，而不去考虑更根本的住宿调整。
- **工具辅助**：当引入专门的“求解器 (solver)”工具来处理预算等约束时，GPT-4 和 Claude 3 的成功率飙升至90%以上。这表明，LLM 结合外部工具，可以显著提升其规划能力。
为了进一步强化AI Agent 的规划能力，研究者提出了更高级的策略 `[1:17:30-1:26:30]`：

- **樹搜索 (Tree Search)**：Agent 在决策前，先探索所有可能的行动路径，评估其成功率，并找到最佳路径。
- **腦內劇場 / 世界模型 (Dream Theater / World Model)**：树搜索的一个问题是，有些行动是不可逆的（例如订披萨）。为了避免在现实世界中做出不可逆的错误，Agent 可以在“梦境”中进行模拟。
- **推理能力与内部独白 (Reasoning & Internal Monologue)**：教授指出，具备“思考”或“推理”能力的LLM，其内部独白（脑内小剧场）正是这种规划过程的体现。他以DeepSeek-R1为例，展示了其在解决积木问题时，通过长达1500字的内部独白，探索各种可能性，最终找到最优解 `[1:24:30-1:26:30]` (视觉/结构信息描述：投影片展示了DeepSeek-R1解决积木问题的详细内部独白)。
然而，这种强大的规划能力也伴随着新的问题 `[1:26:30-1:28:00]`：

- **过度思考的危险 (The Danger of Overthinking)**：一篇新论文指出，能进行脑内剧场的模型虽然整体表现更好，但有时会“想太多”，成为“思考的巨人，行动的矮子”。它们可能在决策前过度分析，甚至在未尝试前就放弃，导致效率低下。因此，如何避免模型过度思考，是未来研究的关键。
**个人感受**

教授用“司令官与长椅”的故事来讽刺LLM可能存在的“照本宣科”式规划，以及用“订披萨”的例子来解释“覆水难收”的行动，都极富画面感和启发性。而“脑内剧场”的概念，则将AI的规划能力提升到了一个更具想象力的层面，让人感受到AI智能的深度。最后对“过度思考”的警示，则为AI Agent的未来发展指明了一个重要的优化方向。

**精华收获**

AI Agent 的规划能力通过显式规划、动态重规划、树搜索和内部世界模型模拟得到增强。尽管LLM能生成看似合理的计划，但真正的规划需要其在动态环境中进行探索、剪枝和内部模拟。具备推理能力的LLM能通过“脑内剧场”进行复杂规划，但需警惕过度思考导致行动迟缓或放弃的风险。
