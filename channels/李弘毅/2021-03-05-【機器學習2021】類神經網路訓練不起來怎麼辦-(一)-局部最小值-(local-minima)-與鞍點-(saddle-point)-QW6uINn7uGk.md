## 详细内容

### 优化失败：超越局部最小值 `[00:00-01:50]`

**核心观点**

当深度学习模型训练停滞，损失函数不再下降时，我们往往会归咎于陷入了局部最小值，但这种说法在学术界被视为不够严谨，因为除了局部最小值，鞍点也是梯度为零的“临界点”之一，且在实践中更为常见。

**深度阐述**

在深度学习的训练过程中，我们常常会遇到一个令人沮丧的场景：尽管模型参数不断更新，但训练损失（training loss）却停止了下降。此时，我们可能会对模型的表现感到不满，尤其是在与更简单的模型（如线性模型或浅层网络）比较时，发现深度网络并未展现出其应有的强大潜力。更糟糕的情况是，有时模型甚至在一开始就“撑不起来”，无论如何更新参数，损失函数都纹丝不动。这种优化过程的失败，是深度学习研究者和实践者普遍面临的挑战。

过去，一个普遍的猜测是，模型陷入了“局部最小值”（local minima）。这个概念深入人心，因为它直观地描绘了一个“山谷”的底部，四周都是更高的点，使得梯度下降算法无法再找到更低的方向。当参数对损失函数的微分（即梯度）为零时，梯度下降算法确实会停止更新参数，训练也就此停滞。因此，许多人会说：“深度学习使用梯度下降会卡在局部最小值，所以梯度下降不行，深度学习也不行。”

然而，李弘毅老师在这里提出了一个重要的纠正和警示：**“如果你要写跟deep learning相关paper的时候，你千万不要讲什么卡在local minima这种事情，别人会觉得你非常没有水准。”** `[00:40]` 这句话并非苛刻，而是强调了学术严谨性。因为梯度为零的点并非只有局部最小值一种情况。

除了局部最小值，还有另一种重要的临界点——“鞍点”（saddle point）。鞍点之所以得名，是因为它的形状酷似马鞍：在某些方向上，它是局部最高点（例如马鞍的前后方向），而在另一些方向上，它又是局部最低点（例如马鞍的左右方向）。在鞍点处，损失函数对参数的梯度同样为零，因此梯度下降算法也会在此停滞。但与局部最小值不同的是，鞍点并非真正的“死胡同”，它在某些方向上仍然有路可走，可以进一步降低损失。

**视觉/结构信息描述**：视频中通过一个红点在三维曲面上的示意图，形象地展示了鞍点的特征：在X轴方向上，红点是局部最高点，而在Y轴方向上，红点是局部最低点，整体呈现出马鞍状。

因此，更准确的说法是，当梯度为零时，我们可能被困在了一个“临界点”（critical point）。临界点是所有梯度为零的点的统称，它包括了局部最小值、局部最大值（local maxima）以及鞍点。区分这些临界点至关重要，因为如果被困在局部最小值，可能真的无路可走；但如果被困在鞍点，则意味着仍有逃脱并进一步优化损失的可能。理解这一点，是深入探讨优化策略的第一步。

**精华收获**

- 训练停滞不前，损失函数不再下降，是深度学习优化中常见的问题。
- 梯度为零的点统称为“临界点”，包括局部最小值、局部最大值和鞍点。
- 简单地将优化失败归咎于“局部最小值”是不准确的，鞍点同样会导致梯度为零，但其性质与局部最小值截然不同。
- 区分局部最小值和鞍点至关重要，因为鞍点提供了逃逸并继续降低损失的可能性。
### 区分临界点：Hessian矩阵的数学之美 `[01:50-05:00]`

**核心观点**

为了区分局部最小值和鞍点，我们需要借助损失函数在临界点附近的二阶泰勒展开，特别是其中的Hessian矩阵。通过分析Hessian矩阵的特征值，我们可以判断临界点的性质。

**深度阐述**

既然区分局部最小值和鞍点如此重要，那么我们如何才能做到这一点呢？李弘毅老师指出，这需要一点数学工具——微积分和线性代数，但其核心思想并不复杂。

首先，我们需要理解为什么区分它们很重要。如果一个模型卡在局部最小值，那意味着它已经到达了当前“山谷”的最低点，无论向哪个方向移动，损失都会增加。这就像走到了一个盆地的底部，四周都是高山，确实无路可走。但如果模型卡在鞍点，情况就大不相同了。鞍点虽然梯度为零，但它周围仍然存在可以降低损失的路径。只要我们能识别出这些路径并沿着它们移动，就有机会逃离鞍点，继续优化模型。

那么，如何识别这些路径呢？答案在于分析损失函数在临界点附近的“地形”。虽然我们无法完整地描绘整个复杂深度网络的损失函数（error surface），但我们可以通过泰勒级数展开（Taylor Series Approximation）来近似描述损失函数在一个特定参数点θ'附近的形状。

损失函数L(θ)在θ'附近的泰勒展开式可以写为：

L(θ) ≈ L(θ') + (θ - θ')ᵀg + ½(θ - θ')ᵀH(θ - θ')

这里：

- L(θ') 是在参数θ'处的损失值。
- g 是损失函数在θ'处对参数θ的一阶导数向量，即梯度（gradient）。**“我们用绿色的这个g来代表gradient，它是一个向量。”** `[03:00]` 梯度描述了损失函数在θ'处最陡峭的上升方向。
- H 是损失函数在θ'处对参数θ的二阶导数矩阵，即Hessian矩阵。Hessian矩阵的第i行第j列的元素Hᵢⱼ表示损失函数L对参数θᵢ和θⱼ的二次偏导数。
当我们在一个临界点时，根据定义，梯度g为零向量。这意味着泰勒展开式中的第二项`(θ - θ')ᵀg`会完全消失。此时，损失函数在临界点θ'附近可以近似表示为：

L(θ) ≈ L(θ') + ½(θ - θ')ᵀH(θ - θ')

现在，我们就可以通过分析第二项，即`½(θ - θ')ᵀH(θ - θ')`，来判断临界点θ'的性质。为了简化表示，我们用向量v = (θ - θ')来代替参数的偏移量。那么，我们需要关注的是`vᵀHv`这一项的符号。

- **局部最小值 (Local Minima)**：如果对于所有可能的非零向量v，`vᵀHv > 0`，这意味着在θ'附近，L(θ)总是大于L(θ')。此时，Hessian矩阵是“正定”（positive definite）的。这表明θ'是一个局部最低点。
- **局部最大值 (Local Maxima)**：如果对于所有可能的非零向量v，`vᵀHv < 0`，这意味着在θ'附近，L(θ)总是小于L(θ')。此时，Hessian矩阵是“负定”（negative definite）的。这表明θ'是一个局部最高点。
- **鞍点 (Saddle Point)**：如果`vᵀHv`有时大于零，有时小于零（取决于v的方向），这意味着在θ'附近，有些方向损失会增加，有些方向损失会减少。此时，Hessian矩阵是“不定”（indefinite）的。这表明θ'是一个鞍点。
当然，我们不可能遍历所有可能的向量v来检查`vᵀHv`的符号。幸运的是，线性代数提供了一个更简便的方法：检查Hessian矩阵的特征值（eigenvalues）。

- 如果Hessian矩阵的所有特征值都是正的，那么它就是正定的，θ'是局部最小值。
- 如果Hessian矩阵的所有特征值都是负的，那么它就是负定的，θ'是局部最大值。
- 如果Hessian矩阵的特征值有正有负，那么它就是不定的，θ'是鞍点。
**延伸思考**

Hessian矩阵的分析为我们提供了一个强大的数学工具，能够从理论上精确地识别不同类型的临界点。这不仅加深了我们对优化景观的理解，也为后续设计更有效的优化算法奠定了基础。虽然计算Hessian矩阵及其特征值在实际深度学习中可能计算量巨大，但其概念对于理解问题本质至关重要。

**精华收获**

- 区分局部最小值和鞍点，关键在于分析损失函数在临界点附近的二阶泰勒展开。
- 梯度为零时，损失函数局部形状由Hessian矩阵决定。
- Hessian矩阵的特征值符号是判断临界点性质的依据：全正为局部最小值，全负为局部最大值，有正有负则为鞍点。
- 鞍点并非优化死胡同，因为它周围仍有降低损失的路径。
### 实例解析：最简网络的鞍点识别 `[05:00-08:50]`

**核心观点**

通过一个极简的神经网络和数据集，我们可以具体演示如何计算梯度和Hessian矩阵，并利用Hessian的特征值来识别出鞍点。

**深度阐述**

为了让抽象的数学概念变得具体，李弘毅老师举了一个“史上最废”的神经网络和“史上最废”的训练数据集作为例子。这个例子虽然简单，却能清晰地展示Hessian矩阵在识别临界点中的作用。

**视觉/结构信息描述**：视频中展示了一个三维的损失函数曲面图，其中横轴和纵轴分别代表w₁和w₂，垂直轴代表损失L。曲面呈现出在(0,0)点有一个鞍点，以及沿着w₁w₂=1和w₁w₂=-1两条线形成的“山谷”（局部最小值）。

**最简网络与数据集**：

- **网络结构**：输入x，经过一个神经元乘以w₁，再乘以w₂，得到输出y。即 `y = w₁ `*` w₂ `*` x`。这个网络没有激活函数，也没有偏置项（bias），只有两个参数w₁和w₂。
- **训练数据**：只有一笔数据，当x=1时，期望输出y=1。
- **损失函数**：采用平方误差（square error），由于只有一笔数据，所以损失函数为 `L = (ŷ - y)² = (1 - w₁w₂x)²`。将x=1代入，得到 `L = (1 - w₁w₂)²`。
**识别临界点**：

首先，我们需要找到损失函数的梯度，即L对w₁和w₂的偏导数：

- ∂L/∂w₁ = 2(1 - w₁w₂) * (-w₂) = -2w₂(1 - w₁w₂)*
- ∂L/∂w₂ = 2(1 - w₁w₂) * (-w₁) = -2w₁(1 - w₁w₂)*
当梯度为零时，我们找到了临界点。观察上述梯度表达式，当w₁=0, w₂=0时，两个偏导数都为零。因此，原点(0,0)是一个临界点。

此外，当w₁w₂=1时，两个偏导数也同时为零。这意味着在w₁w₂=1这条线上，所有的点都是临界点。

**计算Hessian矩阵**：

为了判断(0,0)这个临界点是局部最小值还是鞍点，我们需要计算Hessian矩阵。Hessian矩阵H是一个2x2的矩阵，其元素是损失函数的二阶偏导数：

H = [[∂²L/∂w₁², ∂²L/∂w₁∂w₂],

[∂²L/∂w₂∂w₁, ∂²L/∂w₂²]]

计算这些二阶偏导数：

- ∂²L/∂w₁² = ∂/∂w₁(-2w₂ + 2w₁w₂²) = 2w₂²
- ∂²L/∂w₂² = ∂/∂w₂(-2w₁ + 2w₁²w₂) = 2w₁²
- ∂²L/∂w₁∂w₂ = ∂/∂w₁(-2w₁ + 2w₁²w₂) = -2 + 4w₁w₂
- ∂²L/∂w₂∂w₁ = ∂/∂w₂(-2w₂ + 2w₁w₂²) = -2 + 4w₁w₂ (根据Schwarz定理，混合偏导数相等)
将临界点(w₁, w₂) = (0, 0)代入Hessian矩阵：

H = [[2(0)², -2 + 4(0)(0)],

[-2 + 4(0)(0), 2(0)²]]

H = [[0, -2],

[-2, 0]]

**分析Hessian特征值**：

现在，我们需要计算这个Hessian矩阵的特征值。对于矩阵 `[[a, b], [c, d]]`，特征值λ可以通过解方程 `det(H - λI) = 0` 得到，即 `(a-λ)(d-λ) - bc = 0`。

对于H = `[[0, -2], [-2, 0]]`：

`(0 - λ)(0 - λ) - (-2)(-2) = 0`

`λ² - 4 = 0`

`λ² = 4`

`λ = ±2`

我们得到了两个特征值：λ₁ = 2 和 λ₂ = -2。由于特征值有正有负，根据之前的理论，这表明原点(0,0)是一个鞍点。这与我们通过“爆搜”所有参数值绘制出的损失曲面图的结论完全一致。

**个人感受**

这个简单的例子极大地增强了对Hessian矩阵作用的理解。从抽象的数学公式到具体的计算结果，再到与直观图像的印证，整个过程让人感到豁然开朗。原来，那些看似复杂的数学工具，正是帮助我们洞察深层机制的关键。

**精华收获**

- 通过一个简单的 `y = w₁w₂x` 网络和 `L = (1 - w₁w₂)²` 损失函数，具体演示了临界点的识别。
- 计算了在(0,0)点的梯度和Hessian矩阵。
- 发现Hessian矩阵在(0,0)点的特征值为2和-2，有正有负，从而确认(0,0)是一个鞍点。
- 实践证明了Hessian矩阵特征值在区分临界点类型上的有效性。
### 逃离鞍点：Hessian的指引与实践困境 `[08:50-12:30]`

**核心观点**

如果模型停滞在鞍点，Hessian矩阵不仅能识别它，还能指出逃离的方向。沿着对应负特征值的特征向量方向更新参数，可以降低损失。然而，在实际深度学习中，计算Hessian的成本过高，使得这种方法难以直接应用。

**深度阐述**

当我们确认一个临界点是鞍点时，这并非绝望的终点，反而可能是一个转机。因为鞍点意味着“有路可走”，而Hessian矩阵不仅能告诉我们这是一个鞍点，还能进一步指明逃离的方向。

回顾泰勒展开式：`L(θ) ≈ L(θ') + ½(θ - θ')ᵀH(θ - θ')`。

我们知道，如果Hessian矩阵H的特征值λ有正有负，那么它就是一个鞍点。假设我们找到了一个负的特征值λ < 0，以及它对应的特征向量u。

如果我们让参数的偏移量v等于这个特征向量u，即 `(θ - θ') = u`，那么 `vᵀHv` 这一项就变成了 `uᵀHu`。

根据特征向量的定义，`Hu = λu`。所以，`uᵀHu = uᵀ(λu) = λ(uᵀu) = λ‖u‖²`。

由于‖u‖²（向量u的模的平方）必然是正数，而我们假设λ是负数，那么 `λ‖u‖²` 这一项就必然是负数。

这意味着，如果沿着特征向量u的方向进行参数更新，即 `θ = θ' + u`，那么 `L(θ)` 将会小于 `L(θ')`。换句话说，沿着对应负特征值的特征向量方向移动，损失函数会下降！

**具体例子**：

回到我们“最废”的神经网络例子，在(0,0)点，Hessian矩阵的特征值是2和-2。负特征值是-2。

对应特征值-2的特征向量之一是 `u = [1, 1]ᵀ`。

这意味着，如果我们从(0,0)点沿着向量 `[1, 1]ᵀ` 的方向更新参数，即 `w₁ = 0 + 1`，`w₂ = 0 + 1`，那么新的损失值会比在(0,0)点的损失值更低。

在(0,0)点，`L = (1 - 0`*`0)² = 1`**。*

如果更新到(1,1)点，`L = (1 - 1`*`1)² = 0`**。损失确实从1降到了0。*

这清晰地展示了Hessian矩阵如何为我们指明了逃离鞍点的方向。

**个人感受**

听到这里，一种“柳暗花明又一村”的感觉油然而生。原来，鞍点并非绝境，而是蕴含着继续优化的机会。Hessian矩阵就像一张地形图，不仅标出了我们所处的位置，还指明了通往更低谷的路径。这种理论上的突破，无疑给深度学习的优化带来了新的希望。

**实践困境**：

然而，李弘毅老师也坦诚地指出了这种方法的实际局限性：**“在实作上，你几乎没有看到，有人用这一个方法来逃离saddle point。”** `[12:00]`

原因在于，计算Hessian矩阵本身就需要巨大的计算量，因为它涉及到所有参数的二次偏导数。对于拥有数百万甚至数亿参数的深度神经网络来说，Hessian矩阵将是一个天文数字般大小的矩阵（例如，一百万参数的网络，Hessian矩阵就是一百万乘一百万）。更不用说，计算如此庞大矩阵的特征值和特征向量，其计算复杂度更是难以承受。

因此，尽管Hessian矩阵在理论上为逃离鞍点提供了精确的指导，但在实际的深度学习训练中，这种方法几乎不被采用。研究者们通常会寻找其他计算成本更低的方法来应对鞍点问题。

**延伸思考**

虽然直接计算Hessian来逃离鞍点不切实际，但理解其原理对于设计和改进优化算法至关重要。例如，一些二阶优化方法（如牛顿法及其变种）在理论上利用了Hessian的信息，但通过近似或稀疏化等技术来降低计算成本。此外，理解鞍点的性质也促使我们思考，是否可以通过引入噪声、动量或其他策略，让优化器在遇到鞍点时能够“跳出”困境。

**精华收获**

- 鞍点并非优化死胡同，Hessian矩阵可以指示逃离方向。
- 沿着对应负特征值的特征向量方向更新参数，可以有效降低损失。
- 理论上，Hessian矩阵提供了精确的逃离路径，但实际计算成本过高，使其在深度学习中难以直接应用。
- 尽管如此，理解Hessian的原理对于优化算法的设计和改进具有重要指导意义。
### 鞍点与局部最小值：高维空间的真相 `[12:30-18:00]`

**核心观点**

在高维参数空间中，鞍点远比局部最小值更为常见。在低维看似无路可走的“局部最小值”，在高维空间中可能只是一个可以逃离的鞍点。

**深度阐述**

在理解了鞍点的性质和逃逸机制后，一个更深层次的问题浮出水面：在实际的深度学习训练中，我们究竟更容易遇到局部最小值还是鞍点？李弘毅老师通过一个引人入胜的科幻故事和经验证据，揭示了一个令人振奋的真相：在高维参数空间中，鞍点远比局部最小值更为普遍。

**科幻故事的启示**：

李弘毅老师引用了刘慈欣《三体》中的一个情节：1543年，君士坦丁堡被围困，一位名叫狄奥伦娜的魔法师声称能从万军中取下敌方苏丹的首级。她展示了从密封石棺中取出圣杯并放入葡萄的能力，这让人们震惊。狄奥伦娜解释说，从三维空间来看，石棺是完全封闭的，但她可以进入四维空间，从高维找到进入石棺的路径。这个故事的寓意是：**“从三维的空間來看，是沒有路可以走的東西，在高維的空間中是有路可以走的。”** `[15:00]`

这个故事与深度学习的优化过程有着惊人的相似之处。我们通常习惯于在二维或三维空间中想象损失函数的“地形”，例如一个碗状的局部最小值。但在这种低维视角下看似无路可走的“局部最小值”，在更高维的参数空间中，可能仅仅是一个鞍点。这意味着，在某个方向上它确实是最低点，但在其他未被我们观察到的高维方向上，却可能存在更低的路径。

**高维空间的特性**：

深度学习模型的参数数量动辄百万、千万甚至上亿。这意味着我们的损失函数实际上是在一个非常高维的空间中（维度等于参数数量）。在一个一千万维的空间中，想象一个点在所有一千万个方向上都是最低点，这概率是极其微小的。相反，更有可能的是，在某些方向上是最低点，而在另一些方向上是最高点——这正是鞍点的定义。维度越高，这种“既高又低”的鞍点出现的可能性就越大，而真正的“四面楚歌”的局部最小值反而越稀有。

**经验证据**：

李弘毅老师进一步提供了经验上的证据来支持这一假说。他展示了一张图表，其中每个点代表一个训练完成并卡在临界点的神经网络。图表的纵轴是训练停滞时的损失值，横轴是“最小比率”（minimum ratio），定义为正特征值的数量除以Hessian矩阵的总特征值数量。

- 如果一个临界点是局部最小值，那么它的Hessian矩阵所有特征值都应为正，最小比率应为1。
- 如果是鞍点，则特征值有正有负，最小比率会小于1。
**视觉/结构信息描述**：图表显示，所有训练停滞的神经网络的最小比率都远小于1，通常在0.5到0.6之间。这意味着即使在最“像”局部最小值的情况下，也只有大约一半的特征值是正的，另一半是负的。这表明，在所有可能的维度中，有一半的方向会导致损失上升，而另一半的方向则会导致损失下降。

这个结果强有力地支持了“鞍点比局部最小值更常见”的观点。在绝大多数情况下，当我们觉得训练停滞、梯度接近零时，我们很可能不是卡在了一个真正的局部最小值，而是一个鞍点。

**个人感受**

这个部分是整个课程中最具启发性的。科幻故事的引入，将复杂的数学概念与直观的想象力结合起来，让人对高维空间有了全新的认识。经验证据的呈现，更是将理论与实践完美结合，彻底颠覆了“深度学习容易卡在局部最小值”的传统观念。这不仅减轻了对优化失败的焦虑，也为未来的研究和实践指明了方向。

**延伸思考**

如果鞍点确实比局部最小值更常见，那么我们就不应该过分担心“卡在局部最小值”的问题。相反，我们应该将注意力放在如何有效地逃离鞍点上。这可能包括设计更鲁棒的优化器，例如那些能够利用二阶信息（即使是近似的）或者通过引入随机性来帮助模型跳出鞍点的算法。

**精华收获**

- 借鉴《三体》的四维空间概念，形象说明了低维中的“死胡同”在高维中可能有“出路”。
- 深度学习模型参数众多，损失函数存在于高维空间，这使得真正的局部最小值变得稀有。
- 经验证据表明，在深度学习训练中，遇到的临界点大多数是鞍点（Hessian特征值有正有负），而非纯粹的局部最小值。
- 这一发现极大地缓解了对深度学习优化陷入局部最小值的担忧，并将焦点转向如何有效逃离鞍点。
