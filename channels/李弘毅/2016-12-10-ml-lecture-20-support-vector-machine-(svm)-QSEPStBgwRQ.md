### Hinge Loss的深层解析：从理想损失到实际优化 `[00:30-09:30]`

**核心观点**

Hinge Loss是二元分类问题中理想但不可微损失函数的一种可微分近似，它通过引入“间隔”概念，鼓励模型不仅正确分类，还要以足够的置信度进行分类，并在实际优化中展现出比平方损失和交叉熵更鲁棒的特性。

**深度阐述**

为了深入理解Hinge Loss，李教授首先将我们带回了二元分类（Binary Classification）问题的基本设定。他回顾了机器学习解决方案的三个步骤：定义函数集、定义损失函数、进行优化。在二元分类中，我们定义一个函数 `g(x)`，其内部包含 `f(x)`。当 `f(x) > 0` 时，输出为 `+1`（代表一个类别）；当 `f(x) < 0` 时，输出为 `-1`（代表另一个类别）。值得注意的是，教授特意强调了使用 `+1` 和 `-1` 作为标签，而非传统的 `0` 和 `1`，因为**“这边如果用 +1 和 -1 的话，等一下写式子是会比较顺”** `[Approx. 01:20]`，这体现了他在教学中对数学表达简洁性的考量。

理想的损失函数（Ideal Loss Function）被定义为：当 `g(x)` 的输出与真实标签 `y`*`hat`** 不一致时，损失为 **`1`**；一致时，损失为 **`0`**。这本质上是一个 **`delta`** 函数，统计模型犯错的次数。然而，教授立即指出了其核心问题：****“你的 loss 是不可微分的。这个东西是没办法微分的，所以根本没有办法用 Gradient Descent 来解他。”**** **`[Approx. 02:00]`** 这种不可微性，使得梯度下降等优化算法无法直接应用，因此，寻找一个可微分的近似损失函数成为必然。*

教授通过绘制 `y`*`hat  f(x)`** 与损失值的关系图（****视觉信息描述：横轴为 ******`yhat `*****` f(x)`****，纵轴为损失值。理想损失函数在 ****`y`*****`hat  f(x) < 0`****** 时为 ******`1`******，在 ******`yhat `*****` f(x) >= 0`**** 时为 ****`0`****，呈现阶梯状** `[Approx. 02:15-02:50]`），直观地展示了不同近似损失函数的特性。他首先批判了平方损失（Square Loss），指出其在分类问题中的不合理性。平方损失旨在让 `f(x)` 尽可能接近 `y`*`hat`**（即 **`+1`** 或 **`-1`**），但其曲线在 **`y`*`hat `*` f(x)`** 很大时仍有较大损失，这与分类的直觉相悖。****“我们不希望说 y n hat 跟 f(x) 乘起来很大的时候，居然有一个很大的 loss。”**** **`[Approx. 04:00]`*

接着，教授比较了Sigmoid函数结合平方损失和交叉熵损失。他解释了交叉熵损失在Logistic Regression中的优越性，并展示了其数学形式 `log(1 + exp(-y`*`hat  f(x)))`**。通过曲线对比（****视觉信息描述：绿色曲线代表交叉熵损失，在 ******`yhat `*****` f(x)`**** 趋近无穷大时趋近于 ****`0`****，在 ****`y`*****`hat  f(x)`****** 趋近负无穷大时趋近于无穷大，且在负值区域梯度较大**** **`[Approx. 05:00-06:00]`**），教授强调了交叉熵作为理想损失函数的“上界”（upper bound）的特性，以及其在训练中的优势。他生动地比喻：****“如果把 y n hat  f(x) 从 -2 移到 -1 的时候，如果是 sigmoid + square error 他的变化很小，如果是 sigmoid + cross entropy 他的变化就非常大。”**** **`[Approx. 07:00]`** 这意味着交叉熵在模型表现不佳时能提供更强的梯度信号，促使模型快速调整，从而更容易训练。*

最终，教授引出了Hinge Loss，其表达式为 `max(0, 1 - y`*`hat  f(x))`**。他详细解释了Hinge Loss的工作机制：*

- 当 `y`*`hat = +1`** 时，损失为 **`max(0, 1 - f(x))`**。这意味着只要 **`f(x) > 1`**，损失就为 **`0`**。*
- 当 `y`*`hat = -1`** 时，损失为 **`max(0, 1 + f(x))`**。这意味着只要 **`f(x) < -1`**，损失就为 **`0`**。*
**“他如果对一个 positive 的 example 来说 f(x) > 1 的时候就是完美的 case，对一个 negative example 来说 他的 f(x) < -1 的时候他就是一个完美的 case。”** `[Approx. 08:00]`

Hinge Loss的独特之处在于引入了“间隔”（margin）的概念。它不仅要求分类正确（即 `y`*`hat  f(x) > 0`**），还要求分类结果与决策边界之间保持一定的距离（即 **`yhat `*` f(x) > 1`）。如果 `y`*`hat  f(x)`** 介于 **`0`** 和 **`1`** 之间，即使分类正确，Hinge Loss仍然会产生损失，促使模型进一步优化。教授解释了为何这个间隔通常是 **`1`**：因为它能使Hinge Loss成为理想损失函数的一个“紧密上界”（tight upper bound）。*

在比较Hinge Loss与交叉熵时，教授用了一个非常形象的比喻来阐述它们对“做得好”的样本的不同态度：

- 交叉熵：**“他会想要好，还可以更好。”** `[Approx. 08:50]` 即使 `y`*`hat  f(x)`** 已经很大，交叉熵仍会鼓励其变得更大，以进一步降低损失。这就像一个学生，即使考了90分，也想冲刺100分。*
- Hinge Loss：**“他是一个及格就好的 loss function。”** `[Approx. 09:00]` 一旦 `y`*`hat  f(x)`** 超过了间隔（例如 **`1`**），损失就变为 **`0`**，模型便不再对其进行优化。这就像一个学生，只要达到及格线，就不再花额外精力。*
这种“及格就好”的特性，使得Hinge Loss在处理异常值（outlier）时表现出更好的鲁棒性。教授用一个生动的期末考试比喻来解释：**“cross entropy 就好像是说现在期末考有很多科目要读，但你就是想要拚死读一科而已... 最后可能就被二一了。hinge loss 会顾全所有的科目，所有的科目考到及格就好，他其实是会 all pass 的。”** `[Approx. 09:15-09:30]` 这意味着Hinge Loss更倾向于让所有样本都达到“及格”标准，而不是为了少数“优秀”样本而牺牲整体表现，从而使得模型对异常值不那么敏感。

**个人感受**

李教授对不同损失函数的对比，特别是通过生动比喻来解释Hinge Loss和Cross Entropy的“性格”差异，让我对这些抽象概念有了更直观和深刻的理解。他不仅传授了知识，更传递了对这些算法设计哲学的思考，令人印象深刻。

**延伸思考**

Hinge Loss的“间隔”概念，是SVM作为“最大间隔分类器”的直接体现。这种设计哲学在许多实际应用中都非常有用，尤其是在对分类边界的清晰度和鲁棒性有较高要求的场景。理解不同损失函数的内在机制，对于选择合适的模型和优化策略至关重要。

**精华收获**

Hinge Loss是理想分类损失的可微近似，通过“间隔”强制模型以高置信度分类。它与交叉熵相比，更注重“及格就好”，对异常值具有更好的鲁棒性，是SVM实现最大间隔分类的关键。

### 线性SVM的构建与梯度下降优化 `[09:30-13:30]`

**核心观点**

线性SVM通过将Hinge Loss与线性函数 `f(x)` 结合，形成一个凸优化问题，尽管Hinge Loss在某些点不可微，但仍可通过梯度下降有效求解，甚至可以构建深度版本的SVM。

**深度阐述**

在理解了Hinge Loss的独特之处后，李教授将我们带入了线性SVM的具体构建。他首先定义了线性SVM中的函数 `f(x)`：**“我们现在 的 f(x) 就是 x 里面的每一个 feature，x i 乘上他对应的 weight，w i summation 起来再加上 b。”** `[Approx. 09:50]` 这正是典型的线性模型形式，可以简洁地表示为权重向量 `w` 和输入特征向量 `x` 的内积加上偏置 `b`，即 `f(x) = w^T `*` x + b`**。为了简化表示，教授巧妙地将偏置 **`b`** 融入到 **`w`** 和 **`x`** 中，通过在 **`x`** 向量末尾添加一个 **`1`**，并将 **`b`** 视为 **`w`** 的最后一个分量，从而将 **`f(x)`** 统一表示为 **`w^T `*` x`（这里 `x` 是增广后的特征向量）。这种表示方法在机器学习中非常常见，它使得后续的数学推导更加简洁。

接下来，教授强调了线性SVM的优化目标：最小化Hinge Loss与正则化项之和。他特别指出，这个损失函数是一个**凸函数（convex function）**。**“这个 loss function 他是一个 convex 的 function... 所以所有的 loss 都是 convex 的 function，regularization 也是 convex 的 function，当你把这些 convex 的 function 叠加起来以后会发生甚么事情呢？其实你可以很轻易的证明，把 convex 的 function 叠加起来仍然是 convex function。”** `[Approx. 10:40-11:00]` 凸函数的特性对于优化而言至关重要，因为它保证了任何局部最优解都是全局最优解，这意味着无论从哪个初始点开始进行梯度下降，最终都能收敛到相同的最优解。

一个常见的问题是，Hinge Loss在 `y`*`hat  f(x) = 1`** 处存在“尖角”，即不可微。教授对此进行了巧妙的解释，将其与深度学习中的ReLU激活函数和Maxout网络进行类比：****“Hinge Loss 他在某些点是不可微的... 但其实你都可以用 Gradient Descent 去做 optimization。”**** **`[Approx. 11:15-11:30]`** 这种在“不可微”点仍能进行梯度下降的实践，在现代机器学习中已是常态，通常通过次梯度（subgradient）的概念来处理。*

教授进一步指出，线性SVM与Logistic Regression的本质区别仅在于损失函数的选择：**“如果我们比较 Logistic Regression 跟 Linear SVM 的差别，他唯一的差别就只是我们怎么定 Loss Function。你用 Hinge Loss 就是 Linear SVM，Cross Entropy 的 Loss 就是 Logistic Regression。”** `[Approx. 12:00-12:15]` 这一洞察极大地简化了我们对这两种看似不同算法的理解，揭示了它们在底层结构上的高度相似性。

更令人兴奋的是，教授提出了**“SVM是可以用 Gradient Descent 来 train 的”** `[Approx. 12:20]` 这一观点，这与许多传统教材中通过二次规划（Quadratic Programming）求解SVM的印象有所不同。他详细推导了Hinge Loss对权重 `w`*`i`** 的偏导数：*

`∂L/∂w`*`i = ∂L/∂f  ∂f/∂wi`*

其中，`∂f/∂w`*`i`** 对于线性函数 **`f(x) = w^T  x`** 而言，就是 **`xi`**。*

而 `∂L/∂f` 的值取决于 `1 - y`*`hat  f(x)`** 是否大于 **`0`**：*

- 如果 `1 - y`*`hat  f(x) > 0`**（即 **`yhat `*` f(x) < 1`），则 `∂L/∂f = -y`*`hat`**。*
- 否则，`∂L/∂f = 0`。
因此，梯度更新规则可以写为：`w`*`i = w`*`i - η `*` Σn (δ(yhatn  f(x`*`n) < 1) `*` (-yhatn) `*` x`*`n`*`i)`，其中 `δ` 是指示函数。这个推导清晰地展示了如何将Hinge Loss融入到梯度下降框架中。

最后，教授提出了一个前瞻性的观点：SVM也可以拥有“深度”版本。**“当你今天在做 Deep Learning 的时候，你不是用 Cross Entropy 当作你 minimization 的对象... 而是想要用 Hinge Loss 的话，你其实就是有一个 deep 版本的 SVM。”** `[Approx. 13:00-13:20]` 这打破了传统上将SVM和深度学习视为完全不同领域的观念，揭示了它们在损失函数选择上的潜在融合，为未来的研究和应用提供了新的思路。

**个人感受**

教授对Hinge Loss可微性的解释，以及将SVM与深度学习中的ReLU进行类比，极大地拓宽了我对优化算法适用性的认知。他不仅解决了技术上的疑问，更在思想层面将不同领域的知识融会贯通，令人茅塞顿开。

**延伸思考**

理解SVM作为凸优化问题的重要性，以及其可以通过梯度下降求解的灵活性，对于实际工程应用具有指导意义。将SVM的损失函数应用于深度学习模型，可以探索结合两种范式优势的新型架构，例如在对抗性训练中Hinge Loss的应用。

**精华收获**

线性SVM结合Hinge Loss和正则化形成凸优化问题，可通过梯度下降求解，即使在不可微点也适用。SVM与Logistic Regression仅损失函数不同，且SVM可拥有深度版本，这揭示了算法间的深层共通性。

### 从Hinge Loss到SVM的约束优化形式：Slack Variable的引入 `[13:30-17:30]`

**核心观点**

Hinge Loss的最小化问题，在数学上等价于传统SVM的带松弛变量（slack variable）的约束优化问题，这揭示了SVM“软间隔”分类器的本质。

**深度阐述**

李教授接下来展示了一个非常巧妙的数学转换，将我们之前讨论的Hinge Loss最小化问题，与传统SVM教科书中常见的约束优化形式联系起来。他首先引入了一个新的符号 `epsilon`*`n`** 来表示Hinge Loss的值：****“epsilon n 就等于 max(0, 1 - y n hat  f(x n))。”**** **`[Approx. 14:00]`** 这里的 **`epsilonn`** 代表了第 **`n`** 个样本的损失值。*

这个等式可以被分解为两个不等式约束：

1. `epsilon`*`n >= 0`** (因为 **`max`** 函数的结果不可能为负)*
1. `epsilon`*`n >= 1 - y`*`hat`*`n  f(xn)`** (因为 **`epsilonn`** 必须大于等于 **`max`** 函数的另一个参数)*
教授提出了一个关键问题：仅仅有这两个约束，`epsilon`*`n`** 是否就等同于 **`max(0, 1 - y`*`hat`*`n  f(xn))`**？他通过互动式提问引导听众思考，并明确指出，在没有其他条件的情况下，这两个形式并不完全等价。因为 **`epsilonn`** 可以取一个非常大的值（例如一兆），它仍然满足这两个约束，但显然不是 **`max`** 函数的实际结果。****“理论上它不需要正好等于 0，它不需要正好等于 1- y n hat  f(x)。它可以是任何值。”**** **`[Approx. 16:00]`*

然而，当我们将目标设定为**最小化总损失 ****`L`** 时，情况就发生了根本性的变化。**“当我们要做的事情是要 minimize L 的时候，我们就要想办法让 epsilon n 越小越好。”** `[Approx. 16:30]` 在最小化 `L` 的目标下，`epsilon`*`n`** 自然会被推向其可能的最小值。而 **`epsilon`*`n` 的最小值，正是由 `0` 和 `1 - y`*`hat`*`n `*` f(xn)`** 中的较大者所决定。因此，在最小化损失的语境下，这两个看似不同的表达形式变得完全等价。*

通过这个转换，我们可以将原始的Hinge Loss最小化问题重写为：

最小化 `Σ epsilon`*`n`** (通常还会加上 **`w`** 的正则化项)*

受限于：

1. `y`*`hat`*`n `*` f(xn) >= 1 - epsilonn`*
1. `epsilon`*`n >= 0`*
这里的 `epsilon`*`n`** 就是所谓的****松弛变量（slack variable）****。教授解释道：****“这个 epsilon n 叫做 slack variable，slack 大概是松弛的意思，让你的 margin 变得比较松。”**** **`[Approx. 17:00]`** 这个变量允许某些样本不完全满足 **`y`*`hat`*`n  f(xn) >= 1`** 的严格间隔要求，从而实现“软间隔”（soft margin）分类。如果 **`epsilonn > 0`**，则意味着该样本违反了间隔约束，或者甚至被错误分类。通过最小化 **`epsilonn`** 的总和，我们希望尽可能少地违反间隔约束。*

教授总结道，这个带有约束的优化问题，正是我们所熟悉的SVM的**二次规划（Quadratic Programming, QP）**形式。**“这个 formulation 是一个 Quadratic Programming 的 problem，你就可以带一个 Quadratic Programming 的 solver 然后把它解出来。”** `[Approx. 17:15]` 这不仅将Hinge Loss与传统SVM理论完美衔接，也揭示了SVM作为最大间隔分类器的深层数学原理。

**延伸思考**

理解Hinge Loss与带松弛变量的约束优化问题的等价性，是深入掌握SVM理论的关键一步。这不仅展示了数学形式的灵活性，也解释了SVM如何通过允许少量样本违反间隔约束来提高模型的泛化能力和鲁棒性。这种“软间隔”思想在处理噪声和重叠数据时尤为重要。

**精华收获**

Hinge Loss最小化问题等价于传统SVM的带松弛变量的二次规划问题。松弛变量 `epsilon`*`n`** 允许“软间隔”存在，使得模型在最小化总损失时，能灵活处理不满足严格间隔约束的样本，从而提升模型的鲁棒性。*

### Kernel Trick的魔力：从线性组合到高维映射 `[17:30-29:30]`

**核心观点**

Kernel Trick是一种计算上的巧妙，它利用最优权重 `w` 是数据点线性组合的特性，允许SVM在不显式计算高维特征的情况下，通过核函数在高维空间中进行内积运算，从而高效处理非线性问题。

**深度阐述**

李教授将Kernel Trick誉为SVM“最厉害的地方”，并以其独特的视角，从梯度下降的推导结果出发，而非传统的Lagrange乘子法，来阐释其核心思想。他首先提出了一个关键的结论：**“实际上我们找出来的 weight... 它其实是我们 data 的 linear combination。”** `[Approx. 18:00]` 也就是说，最优的权重向量 `w`* 可以表示为训练数据点 **`x^n`** 的线性组合：**`w`*` = Σ`*`n (alpha`*`n `*` x^n)`**。*

教授通过回顾梯度下降更新 `w` 的过程来证明这一点。如果我们将 `w` 初始化为零向量，那么每次更新 `w` 时，都是在 `w` 上加上一个与数据点 `x^n` 成比例的项。因此，最终得到的 `w` 必然是所有训练数据点 `x^n` 的线性组合。**“每次你 update w 的时候，你都是加上 data points 的 Linear Combination，最后得到的 solution 用 Gradient Descent 解出来的 w 得到的结果就是 w 的 Linear Combination。”** `[Approx. 19:00]`

更进一步，由于Hinge Loss的特性，许多 `alpha`*`n`** 会是零。****“当你在用 Hinge Loss 的时候你的这一项往往都是 0，也就是不是所有的 x^n 都会被拿来加到 w 里面去。”**** **`[Approx. 19:30]`** 那些 **`alpha`*`n` 不为零的 `x^n`，被称为**支持向量（Support Vectors）**。这些支持向量是决定最终模型参数 `w` 的关键数据点，而那些 `alpha`*`n = 0`** 的数据点则对模型没有影响。****“如果 alpha = 0 就一点作用都没有，对这个 model 完全没有影响力... 这些可以决定 parameter 长甚么样子的 data point 他们就叫做 support vector。”**** **`[Approx. 20:00]`** 这种稀疏性（sparsity）使得SVM对异常值具有更好的鲁棒性，因为异常值通常不会成为支持向量。相比之下，Logistic Regression由于其交叉熵损失函数在任何地方的梯度都不为零，导致其 **`alpha`*`n` 不会稀疏，每个数据点都会对模型产生影响。

理解了 `w` 是数据点线性组合的特性后，Kernel Trick的魔力便得以展现。我们知道 `f(x) = w^T `*` x`**。将 **`w = Σn (alphan `*` x^n)` 代入，得到：

`f(x) = (Σ`*`n (alpha`*`n `*` x^n))^T `*` x = Σ`*`n (alpha`*`n `*` (x^n)^T `*` x)`

这里，`(x^n)^T `*` x`** 正是数据点 **`x^n`** 和输入 **`x`** 之间的内积。教授将这个内积操作定义为一个函数 **`k(x^n, x)`**，称之为****核函数（Kernel Function）****。****“这个 function 叫做 Kernel Function。”**** **`[Approx. 22:00]`*

这意味着，在预测 `f(x)` 时，我们不再需要显式地计算 `w`，而是计算输入 `x` 与所有支持向量（或所有训练数据点，如果 `alpha`*`n`** 不稀疏）的内积，然后进行加权求和。****“我们不再需要真的知道 x 的 vector 是多少，真正需要知道的其实只有 x 跟另外一个 vector z 他们之间的 inner product 值。”**** **`[Approx. 23:00]`** 这一洞察是Kernel Trick的核心：我们只需要知道数据点在高维空间中的内积结果，而无需知道高维空间本身。*

教授进一步解释了Kernel Trick的通用性：它不仅限于SVM，也可以应用于Logistic Regression和Linear Regression，从而产生基于核的Logistic Regression和核回归。

接下来，教授通过一个具体的例子展示了Kernel Trick如何实现非线性映射。假设我们想对二维数据 `(x1, x2)` 进行特征变换 `phi(x) = (x1^2, √2 `*` x1 `*` x2, x2^2)`，然后在变换后的高维空间中应用线性SVM。直接计算 `phi(x)` 和 `phi(z)` 的内积会比较复杂。然而，教授展示了：

`phi(x) · phi(z) = (x1^2 `*` z1^2) + 2(x1`*`x2`*`z1`*`z2) + (x2^2 `*` z2^2)`*

这个表达式可以巧妙地简化为 `((x1`*`z1) + (x2`*`z2))^2`，而 `(x1`*`z1) + (x2`*`z2)` 正是原始二维空间中 `x` 和 `z` 的内积 `x · z`。因此，`phi(x) · phi(z) = (x · z)^2`。

**“所以说我们把 x 跟 z 做 Feature Transform 再做 Inner Product 等同于他们在原来 Feature Transform 之前的 space 上面先做 Inner Product 以后再平方。”** `[Approx. 25:00]`

这个例子完美诠释了Kernel Trick的精髓：我们无需显式地计算高维特征 `phi(x)`，只需在原始低维空间中计算一个简单的核函数 `K(x, z) = (x · z)^2`，其结果就等同于在高维空间中的内积。当原始特征维度 `k` 很高，而 `phi(x)` 映射到的维度更高时（例如 `C(k, 2)` 甚至无穷维），这种计算上的优势就变得尤为显著。直接计算 `(x · z)^2` 只需要 `k` 次乘法和一次平方，而显式映射到高维再计算内积则需要处理大量高维特征。

**个人感受**

教授从梯度下降的角度推导出 `w` 是数据点线性组合的特性，再自然过渡到Kernel Trick，这种逻辑链条非常清晰且富有说服力。特别是通过多项式核的例子，直观地展示了“无需显式映射”的计算优势，让我对Kernel Trick的“魔力”有了更深刻的体会。

**延伸思考**

Kernel Trick的本质是利用了“表示定理”（Representer Theorem），该定理表明许多正则化学习算法的最优解都可以表示为训练数据的线性组合。这不仅是SVM的核心，也是许多其他核方法的基础。理解Kernel Trick的计算优势，对于设计高效的机器学习算法至关重要。

**精华收获**

Kernel Trick利用最优权重是数据点线性组合的特性，通过核函数 `K(x, z)` 直接计算高维特征的内积，避免了显式高维映射的计算开销。这使得SVM能高效处理非线性问题，且对异常值更鲁棒，是SVM实现强大分类能力的关键。

### 深入探索不同的核函数及其应用 `[29:30-35:30]`

**核心观点**

核函数是Kernel Trick的核心，通过设计不同的核函数，SVM能够隐式地将数据映射到各种高维空间，处理复杂的非线性关系，甚至处理结构化数据，从而极大地扩展了其应用范围。

**深度阐述**

在详细阐述了Kernel Trick的原理之后，李教授进一步深入探讨了各种具体的核函数，以及它们如何赋予SVM处理复杂问题的能力。他强调，一旦有了Kernel Trick，我们就可以直接设计核函数 `K(x, z)`，而无需关心原始特征 `x` 和 `z` 的具体向量形式，只要这个函数能够给出它们在高维空间中内积的值即可。**“你根本就不用理会 x 跟 z 的 feature 长甚么样子，只要有一个 Kernel Function 可以把 x 跟 z 这两个东西带进去，它可以给你一个 value，这个 value 代表了 x 跟 z 在某一个高维平面上的 vector 的 Inner Product 的话，你根本就不需要在意 x 跟 z 他们 的 vector 长甚么样子。”** `[Approx. 29:50-30:10]`

教授首先回顾了之前提到的多项式核（Polynomial Kernel），例如 `K(x, z) = (x · z)^2`，它隐式地将数据映射到一个包含特征两两之间关系的二次多项式空间。

接着，他介绍了更强大的**径向基函数核（Radial Basis Function Kernel, RBF Kernel）**，其形式为 `K(x, z) = exp((-1/2) ||x-z||^2)`。这个核函数衡量了 `x` 和 `z` 之间的相似度：如果 `x` 和 `z` 越相似（距离越近），核函数值越大；如果 `x = z`，值为 `1`；如果 `x` 和 `z` 完全不同（距离很远），值趋近于 `0`。教授揭示了RBF核的惊人特性：**“这个式子... 其实也可以化成两个 high dimension 的 vector 做 Inner Product 以后的结果，这两个 vector 其实他们的 dimension 是有無窮多維。”** `[Approx. 31:00-31:20]` 他通过泰勒展开式（Taylor Expansion）展示了 `exp(x · z)` 可以展开为无穷多项的和，每一项都对应着原始特征在不同阶次上的组合。这意味着RBF核能够将数据隐式地映射到一个**无穷维**的特征空间中。虽然在无穷维空间中显式计算特征是不可能的，但通过RBF核，我们却能高效地完成等价的内积运算。然而，教授也提醒了RBF核的潜在风险：**“如果用 RBF Kernel 有时候要小心，可能在 training data 上得到很好的 performance，但是在 testing data 上得到很糟的 performance。”** `[Approx. 33:00]` 这暗示了RBF核强大的拟合能力也可能导致过拟合。

第三种核函数是**Sigmoid核（Sigmoid Kernel）**，其形式为 `K(x, z) = tanh(x · z)`。教授指出，如果使用Sigmoid核，那么SVM的预测函数 `f(x) = Σ`*`n (alpha`*`n `*` K(x^n, x))`** 就可以被视为一个****单层隐藏层神经网络****。****“如果用的是 Sigmoid Kernel 这个 f(x) 就可以想成它其实是一个只有一个 hidden layer 的 Neural Network。”**** **`[Approx. 34:00]`** 在这个神经网络中，每个隐藏层神经元的权重就是某个支持向量 **`x^n`**，它计算输入 **`x`** 与该支持向量的内积，并通过 **`tanh`** 激活函数，然后所有隐藏层输出再通过 **`alphan`** 加权求和。这种类比进一步模糊了SVM与神经网络之间的界限。*

核函数最令人兴奋的应用之一是处理**结构化数据（structured data）**。对于序列（sequence）、树结构（tree structure）等难以直接表示为固定长度向量的数据，传统的线性模型或固定特征的SVM难以处理。然而，通过Kernel Trick，我们无需将这些结构化数据显式地转化为向量，只需定义一个能够衡量两个结构化对象之间相似度的核函数即可。**“只要你可以定一个 Function，它是 evaluate x 跟 z 的 similarity，就算 x 跟 z 它是有 structure 的 object... 也没有关系。”** `[Approx. 30:30-30:50]` 教授以语音识别为例，说明如何为变长音频片段定义核函数，从而在不将音频转化为固定向量的情况下进行分类。他提到了Mercer's theory，该理论为判断一个函数是否能作为有效核函数（即其背后是否存在一个隐式的高维内积空间）提供了数学依据。

**个人感受**

RBF核能够隐式映射到无穷维空间，以及Sigmoid核与单层神经网络的类比，这些都让我对核函数的强大和灵活性感到震撼。特别是处理结构化数据的能力，极大地拓展了我对SVM应用场景的想象。

**延伸思考**

核函数的选择对SVM的性能至关重要。理解不同核函数的数学特性和隐式映射的空间，有助于根据数据特点选择最合适的核。同时，核函数在处理结构化数据方面的潜力，为自然语言处理、生物信息学等领域提供了强大的工具。

**精华收获**

核函数是SVM处理非线性及结构化数据的核心。多项式核、RBF核（隐式映射至无穷维）和Sigmoid核（类比单层神经网络）各有特点。通过直接设计衡量相似度的核函数，SVM能处理难以向量化的结构化数据，极大地扩展了其应用边界。

### SVM的变体与与深度学习的融合 `[35:30-38:00]`

**核心观点**

SVM不仅限于二元分类，还发展出多种变体以适应回归、排序等任务；同时，SVM与深度学习在功能上存在共通之处，两者都涉及特征转换和线性分类，甚至可以相互借鉴。

**深度阐述**

在讲座的最后部分，李教授简要介绍了SVM的几种重要变体，并将其与当前热门的深度学习技术进行了富有洞察力的对比，揭示了两者之间深层的联系。

首先是**支持向量回归（Support Vector Regression, SVR）**。教授用简洁的语言概括了其核心思想：**“它的精神是，原来在做 Regression 的时候希望 model 的 output 跟 target 越近越好，如果做 Support Vector Regression 的时候，近到某一个距离里面，它的 loss 就是 0。”** `[Approx. 36:00-36:20]` 这与Hinge Loss的“及格就好”哲学一脉相承，SVR引入了一个“不敏感区域”（epsilon-insensitive zone），在这个区域内的预测误差不会产生损失，从而使得模型对落在该区域内的误差不敏感，具有更好的鲁棒性。

其次是**排序SVM（Ranking SVM）**。当任务目标是生成一个有序列表（例如推荐系统中的商品排序）时，Ranking SVM便派上用场。它直接优化排序的相对顺序，而非简单的回归分数。教授指出，在推荐系统中，**“你可以说我把它当作是一个 regression 的题目，我给每一个 element 一个分数，然后按照分数由高到低做搜索。但这样并没有直接 optimize 你的问题。其实可以直接考虑这个 list 的 ranking。”** `[Approx. 36:40-37:00]` 这强调了针对特定问题设计优化目标的必要性。

最后是**单类别SVM（One-class SVM）**。这种变体主要用于异常检测，它旨在学习一个能够包围正常数据点的决策边界，从而识别出不属于该类别的新样本。**“它是会希望说属于 positive 的 example 都自成一类，negative 的 example 就散佈在其他地方。”** `[Approx. 37:10-37:20]`

在介绍了SVM的多种应用后，教授将目光投向了SVM与深度学习的对比，这是一个极具启发性的环节。他指出，两者在处理数据的方式上有着惊人的相似之处：

- **深度学习**：**“Deep Learning 的前几个 layer 可以看作是 Feature Transformation，最后一个 layer 可以看作是 Linear Classifier。”** `[Approx. 37:30-37:40]` 深度神经网络通过多层非线性变换，自动学习数据的抽象特征表示，最终通过一个线性层进行分类或回归。
- **SVM**：**“它前面先 apply 一个 Kernel Function，把 feature 转到 high dimension 上面，在 high dimension space 上面就可以 apply Linear Classifier。”** `[Approx. 37:40-37:50]` SVM通过核函数隐式地将数据映射到高维特征空间，然后在该空间中应用线性分类器。
这种对比揭示了两者在功能上的异曲同工：都是先进行特征转换，再进行线性分类。区别在于，深度学习的特征转换是多层、可学习的，而传统SVM的核函数是预定义的。然而，教授也提到，SVM的核函数也可以是**可学习的（learnable）**，例如通过组合多个不同的核函数并学习它们的权重。**“当你只有一个 kernel 的时候 SVM 就好像是只有一个 Hidden Layer 的 Neural Network，当你把 kernel 在做 Linear Combination 的时候，他就像一个有两个 layer 的 Neural Network。”** `[Approx. 38:00-38:20]` 这进一步拉近了SVM与神经网络的距离，表明它们并非完全独立的范式，而是在不同抽象层次上解决相似问题的工具。

**个人感受**

教授对SVM变体的简要介绍，让我看到了SVM在分类之外的广阔应用。而他将SVM与深度学习进行对比，并指出两者在特征转换和线性分类上的共通性，以及核函数可学习的潜力，极大地启发了我对机器学习模型架构的理解，打破了固有的分类思维。

**延伸思考**

SVM与深度学习的对比，促使我们思考不同模型范式之间的融合与借鉴。例如，在深度学习中引入Hinge Loss作为损失函数，或探索可学习的核函数在神经网络中的应用，都可能带来新的突破。这种跨领域的思考，是推动机器学习发展的重要动力。

**精华收获**

SVM拥有SVR、Ranking SVM、One-class SVM等多种变体，以适应不同任务。SVM与深度学习在“特征转换+线性分类”的范式上共通，SVM通过核函数进行特征转换，深度学习通过多层网络。核函数甚至可以是可学习的，这模糊了SVM与神经网络的界限，预示着模型融合的潜力。
