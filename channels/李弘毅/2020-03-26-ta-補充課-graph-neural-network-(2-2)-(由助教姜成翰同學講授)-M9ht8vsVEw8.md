### 1. 从图像卷积到图滤波：核心挑战与频谱视角 `[00:00-04:00]`

**核心观点**

传统的卷积神经网络（CNN）通过固定大小的卷积核在规则网格数据（如图像）上进行局部特征提取。然而，图结构数据的非规则性使得直接应用这种“九宫格”式的卷积核变得不合理，因此需要引入傅里叶变换的频谱视角，将图上的卷积操作转化为频谱域的乘法。

**深度阐述**

助教首先回顾了CNN中卷积操作的本质：给定一个固定大小（例如3x3）的卷积核（filter），它在图像上滑动，对局部区域进行特征学习。这种操作在图像这种规则的网格数据上是直观且高效的。然而，当我们将目光转向图数据时，一个根本性的问题浮现出来：我们如何在图上定义一个“3x3的九宫格”？图上的节点没有固定的空间位置，也没有统一的邻居数量或排列方式。一个节点可能只有两个邻居，另一个可能有十个，它们之间也没有明确的“上下左右”关系。因此，直接将CNN的卷积核概念移植到图上是行不通的。

为了解决这个难题，研究者们借鉴了经典信号处理领域的一个强大工具——傅里叶变换（Fourier Transform）。在传统的信号处理中，我们知道在时域（或空间域）进行卷积操作，等价于在频域（Fourier domain）进行简单的乘法操作。这个性质为图上的“卷积”提供了新的思路：

1. **图信号的傅里叶变换**：将图上的信号（即每个节点的特征）从“顶点域”（vertex domain）转换到“频谱域”（spectral domain）。
1. **滤波器的傅里叶变换**：将我们希望学习的“滤波器”也转换到频谱域。
1. **频谱域乘法**：在频谱域中，将图信号的傅里叶变换结果与滤波器的傅里叶变换结果进行逐点乘法。
1. **逆傅里叶变换**：将乘法结果再通过逆傅里叶变换（Inverse Fourier Transform）转换回顶点域，得到经过滤波后的信号。
这个过程听起来有些抽象，但它提供了一个优雅的数学框架，绕过了在非规则图上定义局部卷积核的困难。然而，要实现这一构想，我们必须首先解决如何在图上定义“傅里叶变换”以及其背后的数学基础。助教强调，接下来的内容将大量涉及信号与系统（Signal and System）的知识，并特别提到他所学的是李琳山老师的课程，这为后续的讲解奠定了严谨的数学基础。

**重要原话**：

- "我们有没有办法放一个 3 x 3 的 kernel 在 graph 上面 期待说这个 3 x 3 的 kernel 放在这个节电上面 他就可以看到他九宫格里面的元素 这好像是不太合理的" `[02:00]`
- "我们知道在 fourier domain 里面 如果你在原本这个 domain 要做 convolution 在 fourier domain 里面他做的事情其实就是 只要把他们两个乘起来做 multiplication 就好了" `[02:40]`
**延伸思考**

这种从空间域到频谱域的转换思想，体现了数学在解决复杂问题时的普适性。它不仅仅是GNN的基础，也是许多物理、工程领域分析复杂系统行为的通用范式。理解这一转换的本质，有助于我们跳出具体领域的限制，从更宏观的视角审视问题。

### 2. 信号与系统的基础回顾：合成、分析与傅里叶基 `[04:00-09:00]`

**核心观点**

任何信号都可以被视为一组基（basis）的线性组合（合成），而傅里叶变换就是一种将信号分解到特定频率基上的分析过程。

**深度阐述**

为了铺垫图上的傅里叶变换，助教首先带领我们回顾了信号与系统中的基本概念：合成与分析。

一个信号可以被看作是N维空间中的一个向量。在向量空间中，任何向量都可以由一组基向量（basis vectors）通过线性组合（linear combination）合成。例如，在时域（time domain）中，我们常用的一组基就是一系列在时间轴上排开的Delta函数（脉冲函数），任何时域信号都可以看作是这些Delta函数的加权和。

当我们需要知道这个线性组合中每个基成分（component）的大小（即系数ak）时，就需要进行“分析”。分析的方法就是将整个信号（或向量）与某个特定的基向量进行内积（inner product）。得到的内积值就代表了该信号在这个特定基方向上的分量大小。当然，这里的前提是这组基是正交归一的（orthonormal basis）。

在周期性信号的分析中，傅里叶级数（Fourier Series）就是通过选择一组由余弦（cos）和正弦（sin）函数（或等价的复指数函数e^(jkω0t)）组成的基，来分解周期信号。每个ak系数就代表了信号中对应频率成分的大小。

傅里叶变换（Fourier Transform）则是傅里叶级数向非周期信号的推广，它将时域信号分解到连续的频率成分上。其核心思想依然是：一个信号x(t)可以由一组复指数基e^(jωt)合成，而傅里叶变换X(jω)就是通过内积的方式，分析出信号在每个频率ω上的成分大小。

**个人感受**

助教在这里的讲解，仿佛将我们带回了大学课堂，用最基础但最本质的语言，重新激活了我们对信号处理的记忆。他强调了“合成”与“分析”这对核心概念，并用大家熟悉的傅里叶变换作为例子，为后续将这些概念推广到图上做好了充分的准备。这种循序渐进的教学方式，使得即使对信号处理不甚熟悉的听众，也能抓住核心要义。

**重要原话**：

- "我们说一个信号 我们可以把它看成是一个在 N 维空间里面的一个向量 这个向量我们知道在向量空间里面 它是由一组 basis 经过线性组合合成的" `[04:30]`
- "分析的方式就是我用整个信号 或是整个 vector 跟某一个特定的 basis 某一个特定的成分去做内积 得到的东西就会是他这个 component 的大小" `[05:00]`
**延伸思考**

基的选择是信号表示和分析的关键。不同的基可以揭示信号的不同特性。傅里叶基揭示了信号的频率成分，而小波基则能同时提供时域和频域的局部信息。在图神经网络中，选择合适的“图傅里叶基”将直接决定我们如何理解和操作图上的信息。

### 3. 谱图理论：图拉普拉斯算子与图频率 `[09:00-22:00]`

**核心观点**

谱图理论引入了图的邻接矩阵、度矩阵和拉普拉斯矩阵，其中图拉普拉斯矩阵的特征值被定义为图的频率，其特征向量被定义为图的傅里叶基。

**深度阐述**

进入谱图理论（Spectral Graph Theory）部分，助教开始定义图上的核心数学工具。

1. **图的定义**：一个图G由顶点集V和边集E组成，顶点数量用N表示。
1. **邻接矩阵 (Adjacency Matrix, A)**：一个N×N的矩阵，A*ij表示节点i和节点j之间是否存在边，以及边的权重。如果A*ij=0，则无边；否则为边的权重。当前课程只考虑无向图，因此邻接矩阵是对称的。
1. **度矩阵 (Degree Matrix, D)**：一个N×N的对角矩阵，D*ii表示节点i的度（即与节点i相连的边的权重之和，对于无权图就是邻居数量）。*
1. **图信号 (Graph Signal, f)**：定义在图的每个节点上的值，可以是标量或向量。例如，在一个城市路网图中，每个节点（城市）的信号可以是气温或人口增长数。
**图拉普拉斯算子 (Graph Laplacian, L)** 是谱图理论的核心。它的定义非常简洁：`L = D - A`。

助教强调了拉普拉斯矩阵的几个重要性质：

- **对称性**：由于D是对角矩阵且A是对称矩阵，L也是对称矩阵。
- **半正定性 (Positive Semi-definite)**：这意味着L的所有特征值（eigenvalue）都大于等于0。最小的特征值一定是0。
对于一个对称矩阵，我们可以对其进行谱分解（Spectral Decomposition）：`L = UΛU^T`。

- **Λ (Lambda)**：是一个对角矩阵，其对角线上的元素是L的特征值（λ0, λ1, ..., λN-1）。在谱图理论中，这些特征值被定义为**图的频率 (Graph Frequency)**。
- **U**：是一个由L的特征向量（eigenvector）作为列向量组成的矩阵。这些特征向量是相互正交归一的，它们被定义为**图的傅里叶基 (Graph Fourier Basis)**。
助教通过一个简单的4节点图的例子，详细展示了如何计算其邻接矩阵、度矩阵和拉普拉斯矩阵，并进一步计算出其特征值和特征向量。他将这些特征向量可视化在图上，解释了每个特征向量在每个节点上的分量大小。

**频率的直观理解**：

为了让大家更好地理解“特征值代表频率”这个概念，助教将其与离散时间傅里叶基（Discrete Time Fourier Basis）进行了类比。在离散时间信号中：

- 最低频率（DC分量）的信号是所有点都相同的常数。
- 随着频率升高，相邻信号点之间的变化量（振荡）越大。
- 最高频率的信号，相邻两点之间可能是完全相反的（正负交替）。
这个“频率越大，相邻两点之间信号变化量就越大”的核心概念，被推广到图上。助教通过分析拉普拉斯算子作用于图信号的物理意义来解释这一点：

`Lf = Df - Af`。当我们将Lf的第i个分量展开时，会发现它等于节点i的信号值乘以其度，再减去所有邻居节点信号值的和。进一步整理，` (Lf)`*`i = Σ`*`{j∈N(i)} (f`*`i - f`*`j) `。这意味着，拉普拉斯算子作用于一个节点i的信号，实际上衡量了节点i的信号值与其所有邻居信号值之间的差异总和。

更进一步，`f^T L f = Σ`*`{i,j} A`*`ij (f`*`i - f`*`j)^2`。这个表达式被称为**信号变化的能量 (Power of Signal Variation)**。它量化了图信号的“平滑度（smoothness）”。

- 如果一个图信号非常平滑，相邻节点之间的信号差异很小，那么`f^T L f`的值就会很小。
- 如果一个图信号变化剧烈，相邻节点之间的信号差异很大，那么`f^T L f`的值就会很大。
当我们将特征向量（作为信号）代入这个表达式时，会发现`u`*`k^T L u`*`k = λ`*`k`**。这表明，****拉普拉斯矩阵的特征值λk，精确地量化了其对应的特征向量uk作为图信号时的“平滑度”或“信号变化能量”****。*

- `λ = 0`时，对应的特征向量是所有节点值都相同的常数向量（DC分量），表示信号最平滑，没有变化。
- `λ`越大，对应的特征向量作为信号时，相邻节点之间的差异越大，表示信号的频率越高。
助教通过一个20个节点的一维线图（Line Graph）的例子，直观地展示了随着特征值λ的增大，对应的特征向量在图上呈现出越来越高频（振荡越剧烈）的模式，从而进一步巩固了“特征值即频率”的理解。

**个人感受**

这一部分是整个课程的数学核心，助教的讲解非常精彩。他不仅给出了数学定义，更通过物理意义和直观类比，将抽象的矩阵运算与我们对“频率”的日常理解联系起来。特别是`f^T L f`的推导，清晰地揭示了拉普拉斯算子在图上衡量信号平滑度的本质，这是理解谱图理论的关键。

**重要原话**：

- "Graph Laplacian 他的定义就是 degree matrix - adjacency matrix" `[11:00]`
- "我们把不同的 Λ 这一些 Λ 叫做 frequency graph 的 frequency 把这一组 igon vector 叫做对的 basis" `[12:30]`
- "频率越大，相邻两点之间信号变化量就越大" `[17:00]`
- "这个东西代表相邻两个节电 graph signal 的能量差异 我们就可以用这个东西来量化 某一个 graph signal 的频率大小是多少" `[20:30]`
**延伸思考**

拉普拉斯算子在许多物理现象中都有出现，如热传导、扩散等。在图上，它也自然地描述了信息在节点间的传播和扩散过程。将特征值解释为频率，不仅为GNN提供了理论基础，也为我们理解图上信息流动的模式提供了新的视角。

### 4. 图傅里叶变换与逆变换：在图上实现信号分析与合成 `[22:00-28:00]`

**核心观点**

在图上，图傅里叶变换（GFT）通过将图信号与拉普拉斯矩阵的特征向量（图傅里叶基）进行内积，实现信号到频谱域的分析；逆图傅里叶变换（IGFT）则通过基的线性组合，将频谱域信号合成回顶点域。

**深度阐述**

有了图拉普拉斯矩阵及其特征值和特征向量的定义，我们终于可以正式定义图上的傅里叶变换。

**图傅里叶变换 (Graph Fourier Transform, GFT)**：

目标是将一个顶点域的图信号x（即节点特征向量f）转换到频谱域，得到其在不同频率成分上的大小。

其数学形式为：`x`*`hat = U^T x`**。*

这里的`U^T`是一个N×N的矩阵，其每一行都是一个图拉普拉斯矩阵的特征向量（即图傅里叶基）。

这个操作的本质，正如助教所解释的，就是**分析 (Analysis)**。`U^T x`的每一个分量`x`*`hat`*`k`，实际上是图信号x与第k个特征向量`u`*`k`**的内积：**`x`*`hat`*`k = u`*`k^T x`。

这与传统傅里叶变换中通过内积计算每个频率分量大小的原理完全一致。`x`*`hat`*`k`表示了图信号x在频率`λ`*`k`**上的成分有多少。*

**逆图傅里叶变换 (Inverse Graph Fourier Transform, IGFT)**：

目标是将频谱域的信号`x`*`hat`**转换回顶点域，得到原始的图信号x。*

其数学形式为：`x = U x`*`hat`**。*

这个操作的本质是**合成 (Synthesis)**。它将频谱域中每个频率分量的大小`x`*`hat`*`k`，乘以其对应的图傅里叶基`u`*`k`**，然后将所有这些加权基向量相加，从而重构出原始的顶点域信号x。*

`x = Σ`*`k x`*`hat`*`k u`*`k`。

助教通过类比传统时域信号的逆傅里叶变换，解释了如何通过每个频率分量（如`a`*`k`**）乘以其对应的基函数（如**`cos(kωt)`**）再求和，来还原时域信号。在图上，这个过程就是将每个**`x`*`hat`*`k`**乘以对应的特征向量**`u`*`k`，再求和。

至此，我们已经建立了在图上进行傅里叶分析和合成的数学工具，为在频谱域进行滤波操作奠定了基础。

**重要原话**：

- "假设我有一个信号 x 我就把他乘以一个 U^T 这个 U^T 就是我们刚才前面讲到的那个 igon vector 的矩阵 我把他乘出来以后就会得到这个信号他在 frequency domain 的一个信号" `[23:40]`
- "他做的事情其实就是分析 他要分析每一个 component 的大小是什么" `[24:20]`
- "假设说你已经有这样一个信号 你想要把他还原到 vertex domain 上面 你做的事情很简单，你就是看 这个 λ0 对应到的 igon value 是他 我们就看这个 igon value 假设我想要还原出 他在 vo 这个节电上面信号大小是多少" `[26:30]`
**延伸思考**

GFT和IGFT的引入，使得图信号的处理可以借鉴传统信号处理的丰富经验。它将图的拓扑结构信息编码到拉普拉斯矩阵的特征基中，从而允许我们像处理时间序列或图像一样，在频率域对图信号进行操作。

### 5. 图滤波（卷积）的实现与挑战 `[28:00-36:00]`

**核心观点**

在图的频谱域中，滤波操作等价于将图信号的傅里叶变换结果与一个频率响应函数进行乘法。然而，直接实现这种滤波会带来参数量大、缺乏局部性以及计算成本高昂等问题。

**深度阐述**

现在，我们已经可以在图上进行傅里叶变换和逆变换，那么图上的“滤波”（或“卷积”）就可以按照信号处理的经典范式来实现了。

**图滤波的数学形式**：

1. **傅里叶变换**：将输入图信号x转换到频谱域：`x`*`hat = U^T x`**。*
1. **频谱域乘法**：在频谱域中，将`x`*`hat`**与一个频率响应函数**`g`*`θ(λ)`进行逐点乘法，得到滤波后的频谱域信号`y`*`hat`**：**`y`*`hat = g`*`θ(Λ) x`*`hat`。这里的`g`*`θ(Λ)`**是一个对角矩阵，其对角线元素是**`g`*`θ(λ`*`k)`**，表示滤波器在每个频率**`λ`*`k`上的响应。
1. **逆傅里叶变换**：将`y`*`hat`**转换回顶点域，得到最终的滤波后信号y：**`y = U y`*`hat`。
将这三步结合起来，我们可以得到图滤波的完整表达式：`y = U g`*`θ(Λ) U^T x`**。*

这个表达式是基于频谱的图神经网络的核心。我们的目标就是学习这个频率响应函数`g`*`θ(λ)`**，或者说学习**`g`*`θ(Λ)`这个对角矩阵。

然而，助教随即指出了这种直接实现方式的**三大挑战**：

1. **参数量与泛化性问题 (Learning Capacity)** `[32:00]`：
1. **缺乏局部性 (Non-Localize)** `[33:00]`：
1. **计算成本高昂 (High Computational Complexity)** `[35:00]`：
这些挑战使得直接基于谱分解的图卷积方法在实践中难以应用。因此，需要寻找更高效、更具泛化性且能保持局部性的近似方法。

**重要原话**：

- "你要去学一个 Laplacian 的多项式函数 不一定是多项式函数，你要去学一个 Laplacian 的函数 经过这个 Laplacian 的函数之后 跟 x 相乘可以得到经过 filter 的信号 y" `[30:00]`

- "假设你给他弄到 n 次方他只有 n 个节点的话 经过一次 filter 他就可以把所有的 整个 graph 上面的东西都看过 他就只有考虑道 neighborhood 里面的东西 所以这就是我们说第二个问题，他不是 localize" `[34:30]`

### 6. ChebNet：利用切比雪夫多项式解决挑战 `[36:00-47:00]`

**核心观点**

ChebNet通过将频率响应函数近似为拉普拉斯矩阵的切比雪夫多项式，成功解决了参数量、局部性和计算效率问题，实现了K-局部化的图卷积。

**深度阐述**

为了克服上述挑战，第一个重要的基于频谱的GNN模型——ChebNet（Chebyshev Spectral CNN）被提出。

**解决参数量与局部性问题**：

ChebNet的核心思想是：不再直接学习`g`*`θ(Λ)`**的N个对角元素，而是将频率响应函数**`g`*`θ(λ)`近似为一个关于拉普拉斯矩阵L的K阶多项式函数：

`g`*`θ(L) = Σ`*`{k=0}^K θ`*`k L^k`*

- **参数量**：现在需要学习的参数是多项式的系数`θ`*`k`**，总共有**`K+1`**个参数。这个参数量**`O(K)`**与图的节点数量N无关，只与多项式的阶数K有关。K是一个超参数，通常远小于N。这解决了参数量依赖于图大小的问题。*
- **局部性**：由于多项式的最高阶数是K，根据谱图理论，`L^k`操作只会影响到距离为K的邻居节点。因此，这个滤波器是**K-局部化 (K-localized)** 的。这解决了缺乏局部性的问题，使得模型能够像CNN一样，通过堆叠多层来逐步扩大感受野。
**解决计算成本高昂问题**：

虽然多项式近似解决了前两个问题，但直接计算`Σ`*`{k=0}^K θ`*`k L^k x`仍然需要计算`L^k x`，这可能涉及多次矩阵乘法，并且仍然需要特征分解来得到`U`和`Λ`（虽然理论上可以避免，但实际计算`L^k`仍然复杂）。ChebNet引入了**切比雪夫多项式 (Chebyshev Polynomials)** 来进一步优化计算。

**切比雪夫多项式**：

切比雪夫多项式`T`*`k(x)`**是一个特殊的正交多项式家族，它通过递归关系定义：*

- `T`*`0(x) = 1`*
- `T`*`1(x) = x`*
- `T`*`k(x) = 2x T`*`{k-1}(x) - T`*`{k-2}(x)`*
其中，x的取值范围通常被限制在[-1, 1]。为了将拉普拉斯矩阵的特征值`λ`（范围是[0, λ*max]）映射到这个范围，需要进行一个线性变换：**`λ~ = (2λ / λ`*`max) - 1`。

ChebNet将`g`*`θ(λ)`**近似为切比雪夫多项式的线性组合：*

`g`*`θ(λ) ≈ Σ`*`{k=0}^K θ`*`k' T`*`k(λ~) `

其中`θ`*`k'`**是新的可学习参数。*

**计算优化**：

关键在于，`T`*`k(L~)x`**（其中**`L~`**是经过归一化和缩放的拉普拉斯矩阵）可以通过递归关系高效计算，而无需进行谱分解或复杂的矩阵幂运算：*

定义`x`*`0`*`bar = x`

定义`x`*`1`*`bar = L~x`

对于`k >= 2`，`x`*`k`*`bar = 2L~x`*`{k-1}`*`bar - x`*`{k-2}`*`bar`

最终的滤波结果`y`可以表示为：`y = Σ`*`{k=0}^K θ`*`k' x`*`k`*`bar`。

通过这种递归计算方式，每次计算`x`*`k`*`bar`只需要进行一次矩阵-向量乘法（`L~x`），其复杂度为`O(E)`（E是边的数量，因为L是稀疏矩阵）。因此，计算所有`x`*`k`*`bar`直到K阶的复杂度是`O(KE)`。这大大降低了计算成本，解决了`O(N^2)`的复杂度和特征分解的问题。

**多通道扩展**：

与CNN类似，ChebNet也可以扩展到多通道（multi-channel）输出。如果需要F*out个输出通道，就可以学习F*out组`θ`*`k'`**参数，每组对应一个滤波器。*

**个人感受**

助教在这里巧妙地引入了高中数学中多项式近似的例子，来解释为什么我们要用切比雪夫多项式来近似`g`*`θ(L)`**。这个类比非常生动，帮助听众理解了这种“换基”的思路——虽然原始多项式直接计算可能复杂，但通过转换为另一组基（切比雪夫多项式），在特定点或特定场景下计算会变得异常简单。这种将复杂数学概念与直观例子结合的教学方式，是其讲解的亮点。*

**重要原话**：

- "他做的方式很简单 就是刚才说 gθ(L) 可以随便乱选 他说我们不要随便乱选，我们选东西必须要是 Laplacian 的一个多项式函数" `[37:00]`
- "透过你选择让他是多项式函数的方式，你就可以让他是 K-localized" `[37:30]`
- "他要学的是 O(K) 因为你现在这个 filter 就是 Laplacian 的一个多项式函数 你只要决定这个多项式函数细数次多少就好" `[38:00]`
- "他的 complexity 就只会有 O(E) 你算 k 个的话 你用递迴式算只要 O(KE)" `[45:30]`
**延伸思考**

ChebNet是谱图理论向实用GNN模型迈出的关键一步。它展示了如何通过巧妙的数学近似和计算技巧，将复杂的理论转化为高效可行的深度学习模块。切比雪夫多项式的应用，不仅降低了计算成本，也为后续更简化的GCN模型提供了理论基础。

### 7. GCN：ChebNet的简化与直观解释 `[47:00-54:00]`

**核心观点**

图卷积网络（GCN）是ChebNet的一个极端简化版本，通过将多项式阶数K设为1，并引入一系列近似和归一化技巧，最终得到一个极其简洁且直观的节点特征聚合公式。

**深度阐述**

GCN（Graph Convolutional Network）是目前最流行、最广泛使用的GNN模型之一，其成功在于其极度的简洁性和直观性。助教指出，GCN可以被理解为ChebNet的一个特殊简化版本。

**从ChebNet到GCN的简化路径**：

1. **K=1的近似**：GCN将ChebNet中切比雪夫多项式的阶数K设为1。这意味着滤波器只考虑一阶邻居信息。
1. **参数共享**：为了进一步减少参数，GCN假设`θ`*`0' = -θ`*`1'`（或`θ`*`0' = θ`**，**`θ`*`1' = -θ`）。
1. **Renormalization Trick (重归一化技巧)**：
**GCN的最终形式与直观解释**：

将上述简化和技巧整合，GCN的层间传播规则可以写为：

`H^(l+1) = σ(D`*`tilde^(-1/2) A`*`tilde D`*`tilde^(-1/2) H^(l) W^(l))`*

其中：

- `H^(l)`是第l层的节点特征矩阵。
- `W^(l)`是可学习的权重矩阵（对应于`θ`）。
- `σ`是非线性激活函数（如ReLU）。
助教进一步将这个矩阵形式展开，给出了GCN最直观的解释：

**对于图中的任意节点v，其在下一层****`H^(l+1)`****的特征，是通过对其所有邻居节点u（包括节点v自身，因为有自环）在当前层****`H^(l)`****的特征进行加权平均，然后经过一个线性变换（****`W^(l)`****）和一个非线性激活函数得到的。**

具体来说，每个邻居节点u的特征`x`*`u`**首先通过**`W^(l)`**进行变换，然后这些变换后的特征被加权求和（权重由**`D`*`tilde^(-1/2) A`*`tilde D`*`tilde^(-1/2)`决定，本质上是一种度归一化），最后加上一个偏置项`b`并经过激活函数。

**个人感受**

GCN的推导过程虽然经过了多步近似，但最终得到的公式却异常简洁和直观。助教在讲解时，从复杂的谱图理论一步步简化到这个易于理解的“邻居聚合”公式，让听众感受到了数学的精妙和工程的智慧。他特别强调“信號與系統已經全部結束了”，标志着从纯理论到实践应用的过渡，让大家卸下了数学的包袱，专注于GCN的直观含义。

**重要原话**：

- "GCN 的全名叫 graph convolution network 他做的方法很簡單，就是我們剛才講的 Chebyshev polynomial ChebNet 他用的是一個 K-localized polynomial filter 如果 polynomial filter 選擇讓 k = 1 的時候" `[48:00]`
- "這個式子告訴我們說你要 update 某一個 v 他的 feature map 的話 你要做的事情就是把他所有的 neighbor 包括他自己全部加起來 你把他所有的東西加起來 先把他的 filter 經過一個 transform 之後 加起來取平均再加一個 b 經過一個 nonlinear activation 你就會得到他再下一層" `[52:30]`
**延伸思考**

GCN的成功在于其简洁性和高效性，它将复杂的谱图理论转化为一个易于实现和理解的局部聚合操作。然而，这种简化也可能带来一些局限性，例如对不同频率信息的区分能力可能不如更复杂的谱方法。但无论如何，GCN为后续GNN模型的发展奠定了基础。

### 9. 深度GNN的挑战：过平滑问题与DropEdge `[01:04:00-01:09:00]`

**核心观点**

GNN模型，特别是GCN，在堆叠多层时容易出现“过平滑”（Over-smoothing）问题，导致不同社区的节点特征趋于一致，丧失区分度。DropEdge是一种简单的解决方案，通过随机丢弃边来缓解过平滑。

**深度阐述**

在深度学习中，通常认为模型层数越多，其表达能力越强。然而，助教指出，对于GNN，尤其是GCN，情况并非如此。

**深度GNN的性能下降**：

- 在TSP数据集上，GCN在没有残差连接（residual connection）的情况下，堆叠到32层时，性能会急剧下降到0.361的F1-score，远低于浅层模型。
- GIN等其他模型虽然没有GCN那么惨，但也普遍存在层数增加导致性能瓶颈的问题。
**过平滑问题 (Over-smoothing)**：

研究者们分析发现，GCN在堆叠多层时会产生一个严重的现象——**过平滑**。

- GCN的更新机制是聚合邻居特征并取平均。当模型层数很深时，每个节点的感受野会指数级扩大，最终会覆盖到整个图。
- 这意味着，经过多层聚合后，同一个连通分量（connected component）内的所有节点，其特征向量会变得非常相似，甚至收敛到同一个值。
- 例如，在一个包含两个社区的图上，经过多层GCN后，第一个社区的所有节点特征会收敛到一点，第二个社区的所有节点特征会收敛到另一点。虽然不同社区之间可能仍有区分，但社区内部的节点区分度完全丧失。
- 这种现象类似于图像处理中的过度模糊，导致细节信息丢失，模型无法区分细粒度的节点特征。
**解决方案：DropEdge**：

为了缓解过平滑问题，一种简单而有效的方法是**DropEdge**。

- DropEdge的思路非常直接：在训练过程中，随机地丢弃图中的一些边。
- 通过随机丢弃边，可以有效地打断节点之间过度频繁的信息传递，从而避免特征向量过快地收敛到相同的值。
- 实验结果表明，加入DropEdge后，GCN的性能确实有所提升，比没有DropEdge时要好。然而，助教也指出，即使加入了DropEdge，GCN的性能仍然可能不如其他更先进的GNN模型。
**重要原话**：

- "你把 GCN 叠到 32 层，他的 performance 在没有 residual connection 的状况下会烂到只剩 0.361" `[01:04:30]`
- "总而言之他的重点就告诉我们说 GCN 到最后 你的 node feature 经过很深很深的 GCN 之后 同样的一个 subgrade 里面 很多东西会收敛到同样的地方" `[01:06:00]`
- "DropEdge 告诉你说不要把所有 edge 取平均 在取平均的时候随边 drop 掉一些 neighbor 就可以避免刚刚讲的问题" `[01:07:30]`
**延伸思考**

过平滑是GNN领域的一个核心挑战，它限制了GNN模型的深度和表达能力。DropEdge是一种启发式的解决方案，但更根本的解决之道可能在于设计新的聚合机制、引入残差连接、或者结合自注意力机制等，以更好地平衡局部信息聚合和全局信息传播。
