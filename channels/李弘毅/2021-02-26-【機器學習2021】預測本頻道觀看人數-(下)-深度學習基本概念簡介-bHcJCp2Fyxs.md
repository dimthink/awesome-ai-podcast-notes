### 线性模型的局限性与“模型偏差” `[00:00-01:30]`

**核心观点**

简单的线性模型无法捕捉数据中复杂的非线性关系，这种由模型结构本身带来的限制被称为“模型偏差”（Model Bias），它与优化过程中的“偏置项b”是两个不同的概念。

**深度阐述**

在机器学习的旅程中，我们通常从最基础的线性模型（Linear Model）开始。以预测YouTube频道观看人数为例，如果我们将前一天的观看人数（x1）作为输入，预测第二天的观看人数（y），一个线性模型会假设x1和y之间存在一种简单的直线关系：`y = b + w `*` x1`**。这意味着，无论x1如何变化，y总是随着x1的增加而线性增加或减少，其关系图永远是一条直线。我们可以通过调整权重（w）来改变直线的斜率，通过调整偏置（b）来改变直线与y轴的截距，但无论如何调整，它始终是一条直线。*

然而，现实世界的数据往往远比一条直线复杂。李老师生动地指出，例如，前一天的观看人数（x1）与第二天观看人数（y）的关系可能并非总是正比。也许在x1较小的时候，它们成正比；但当x1达到某个峰值后，物极必反，如果前一天的观看人数过高，可能导致第二天观看人数反而减少。这种关系在图上可能表现为一条先上升后下降的曲线，或者带有多个转折点的复杂曲线。

**重要原话**："但你不管怎么摆弄你的 w 跟 b，你永远制造不出红色那一条线。你永远无法用 Linear 的 Model 制造红色这一条线。所以怎么办呢？显然 Linear 的 Model 有很大的限制。这一种来自于 Model 的限制，叫做 Model 的 Bias。" `[01:00-01:30]`

这种由模型结构本身过于简单，无法拟合真实数据复杂关系而导致的限制，被称为“模型偏差”（Model Bias）。李老师特别强调，这个“Model Bias”与线性模型中的偏置项“b”（Bias）在用词上容易混淆，但它们指代的是完全不同的概念。模型偏差是模型能力上的根本限制，它决定了模型能够表达的函数类型。如果模型本身就无法表达真实世界中的复杂函数，那么无论我们如何优化参数，都无法得到一个好的结果。这就像你试图用一把直尺去描绘一个完美的圆，无论你如何努力，直尺的本质决定了你无法完成任务。因此，为了应对现实的复杂性，我们需要一个更复杂、更具弹性、拥有更多未知参数的函数模型。

**延伸思考**

“模型偏差”的概念在机器学习中至关重要。它提醒我们，选择合适的模型结构是解决问题的第一步。如果模型偏差过高，即使训练数据再多，优化算法再先进，模型也无法学习到数据的真实分布。这引出了一个核心问题：如何构建一个足够灵活的模型，以逼近任意复杂的函数？

### 逼近复杂函数：分段线性曲线与“蓝色函数”的构建 `[01:30-05:00]`

**核心观点**

任何连续的复杂曲线都可以通过足够多的分段线性曲线来近似，而这些分段线性曲线又可以由一系列具有特定形状的“蓝色函数”叠加组合而成。

**深度阐述**

既然线性模型过于简单，无法捕捉复杂关系，那么我们该如何构建一个更强大的模型呢？李老师巧妙地引入了一个核心思想：任何连续的曲线，无论多么复杂，都可以通过足够多的“分段线性曲线”（Piecewise Linear Curves）来近似。想象一下，一条平滑的曲线，如果我们在这条曲线上取足够多的点，然后将这些相邻的点用直线段连接起来，得到的锯齿状折线（即分段线性曲线）就会与原始曲线非常接近。点取得越多，近似效果就越好。

那么，如何用数学方式来构建这些分段线性曲线呢？李老师接着提出了一个天才般的构想：我们可以将任何分段线性曲线看作是一个常数项，再加上一系列特殊的“蓝色函数”的组合。这个“蓝色函数”具有独特的形状：它在某个阈值之前保持一个定值（水平），然后在一个区间内呈现斜坡状（线性增长或下降），最后在另一个阈值之后又恢复为另一个定值（水平）。用李老师的话来说，它“先水平的，然后再斜坡，然后再水平的”。

**视觉/结构信息描述**：想象一条横轴为x1，纵轴为y的坐标系。一个“蓝色函数”的图形会像一个平躺的“S”形，但中间的斜坡是直线段，两端是水平线。

李老师详细演示了如何通过叠加这些“蓝色函数”来构建一个复杂的分段线性曲线。首先，确定曲线的起始点作为常数项。然后，第一个“蓝色函数”的斜坡起始点与曲线的起始点对齐，斜坡的终点与曲线的第一个转折点对齐，并且斜率与曲线在该段的斜率一致。这样，常数项加上第一个“蓝色函数”就构成了曲线的第一段。接着，第二个“蓝色函数”的斜坡起始点与曲线的第一个转折点对齐，终点与第二个转折点对齐，斜率也与曲线在该段的斜率一致。通过不断叠加这样的“蓝色函数”，我们就可以精确地重构出任意复杂的分段线性曲线。

**重要原话**："不管我画什么样 Piecewise Linear 的 Curves，你有办法用常数项加一大堆的蓝色 Function 组合出来。" `[05:00]`

这意味着，只要我们有足够多的“蓝色函数”，并且能够灵活地调整它们的起始点、斜率和高度，我们就能组合出任何形状的分段线性曲线。而既然分段线性曲线可以逼近任何连续曲线，那么理论上，我们只要有足够多的“蓝色函数”，就能逼近任何连续的复杂函数。这个“蓝色函数”就是我们构建复杂模型的基石。

**延伸思考**

这个“蓝色函数”的构想是理解神经网络中“激活函数”作用的关键。它将一个看似复杂的任务（拟合任意曲线）分解为简单的、可叠加的模块。这种模块化思想是深度学习模型设计的基础。

### 从“蓝色函数”到Sigmoid：数学表达与参数控制 `[05:00-08:00]`

**核心观点**

虽然“蓝色函数”概念直观，但其数学表达可能复杂。Sigmoid函数作为一种平滑的S形曲线，能够很好地近似“蓝色函数”，并通过调整其参数（w, b, c）来控制其形状、位置和高度，从而实现灵活的组合。

**深度阐述**

我们已经理解了“蓝色函数”作为构建分段线性曲线的基本单元的重要性。然而，直接写出这种“先水平、再斜坡、再水平”的函数形式可能并不那么直接。李老师接着引入了一个在机器学习中更为常见且易于处理的函数来近似这个“蓝色函数”——那就是**Sigmoid函数**。

Sigmoid函数，顾名思义，因为它长得有点像字母“S”，所以被称为S形函数。它的数学表达式是：

`y = c / (1 + exp(-(b + w `*` x1)))`*

让我们来剖析这个表达式的含义：

- **x1**：是输入值，对应于我们例子中的前一天观看人数。
- **w (Weight)**：权重，它控制着Sigmoid曲线的“坡度”或“斜率”。当w越大，曲线的坡度越陡峭；当w越小，曲线的坡度越平缓。
- **b (Bias)**：偏置，它控制着Sigmoid曲线在x轴上的“左右移动”。调整b可以使曲线向左或向右平移。
- **c (Constant)**：常数项，它控制着Sigmoid曲线的“高度”或“幅度”。它决定了Sigmoid函数能达到的最大值。
李老师通过分析x1趋近于无穷大和负无穷大时的Sigmoid函数行为，形象地解释了其S形特征：

- 当x1趋近于正无穷大时，`-(b + w `*` x1)`**会趋近于负无穷大，**`exp(-(b + w `*` x1))`会趋近于0，因此y会趋近于`c / (1 + 0) = c`。这意味着曲线会收敛在高度为c的地方。
- 当x1趋近于负无穷大时，`-(b + w `*` x1)`**会趋近于正无穷大，**`exp(-(b + w `*` x1))`会趋近于无穷大，因此y会趋近于`c / (1 + 无穷大) = 0`。这意味着曲线会收敛在高度为0的地方。
**视觉/结构信息描述**：Sigmoid函数在x轴上从左到右，y值从0平滑地过渡到c，中间有一个明显的S形上升坡度。这与“蓝色函数”的“水平-斜坡-水平”结构非常相似，只是Sigmoid是平滑的，而“蓝色函数”是分段线性的。

**重要原话**："所以你可以用這樣子的一個 Function 來試著畫出這一條曲線，用這一條曲線來逼近這一個藍色的 Function。那這個東西它的名字叫做 Sigmoid。" `[07:00-07:20]`

通过调整w、b、c这三个参数，我们可以制造出各种不同形状、不同位置、不同高度的Sigmoid函数。这些灵活可变的Sigmoid函数，就能够像“蓝色函数”一样，作为基本构建块，通过叠加组合来逼近任意复杂的分段线性曲线，进而逼近任何连续的函数。这种将复杂问题分解为简单、可参数化单元的思想，是神经网络能够学习复杂模式的关键。

**个人感受**

李老师在解释Sigmoid函数时，从直观的“蓝色函数”过渡到具体的数学表达式，再通过参数的物理意义（斜率、移动、高度）来解释其可塑性，使得一个看似复杂的数学概念变得触手可及。这种循序渐进的讲解方式，极大地降低了学习门槛，让人对神经网络的构建原理有了更深刻的理解。

### 构建复杂模型：多Sigmoid叠加与多特征扩展 `[08:00-11:00]`

**核心观点**

通过将多个Sigmoid函数（每个具有独立的参数）叠加起来，并引入多个输入特征，我们可以构建一个高度灵活的函数，以逼近任意复杂的连续函数。

**深度阐述**

既然单个Sigmoid函数可以近似一个“蓝色函数”，那么要构建一个能够近似任意复杂连续函数的模型，就需要将多个Sigmoid函数叠加起来。李老师解释道，一个复杂的红色曲线（代表真实世界的复杂关系）可以被看作是一个常数项，加上一系列不同的“蓝色函数”的组合。当我们将“蓝色函数”替换为Sigmoid函数时，这个组合就变成了：

`y = b + Σ [ci `*` Sigmoid(bi + wi `*` x1)]`

其中：

- `b`：是一个全局的常数项，用于调整整个曲线的基线。
- `Σ`：表示求和，我们将多个Sigmoid函数的输出叠加起来。
- `i`：代表第i个Sigmoid函数。
- `ci, bi, wi`：分别是第i个Sigmoid函数的常数项、偏置和权重。这意味着每一个Sigmoid函数都有自己独立的参数，可以独立地调整其形状、位置和高度。
**重要原话**："所以今天，假设我们的 x 跟 y 的关系它也许非常地复杂，那也没关系，我们就想办法写一个带未知数的 Function，这个带未知数的 Function 它表示的就是一堆蓝色的 Function 加上一个 Constant。" `[09:00-09:30]`

这个模型的核心思想是：通过足够多的、参数各异的Sigmoid函数，我们可以组合出任意复杂的分段线性曲线，进而逼近任何连续函数。Sigmoid函数的数量越多，模型能够拟合的曲线就越复杂，细节捕捉能力也越强。

进一步地，李老师将这个模型从单一输入特征（x1）扩展到多个输入特征（x*j）。在实际应用中，预测YouTube观看人数可能需要考虑前一天、前两天、甚至前56天的观看人数等多个因素。因此，每个Sigmoid函数内部的线性组合也需要扩展：*

`y = b + Σ [ci `*` Sigmoid(bi + Σj (wij `*` xj))]`

这里：

- `xj`：代表第j个输入特征（例如，前j天的观看人数）。
- `wij`：是第i个Sigmoid函数中，作用于第j个输入特征的权重。
- `bi`：是第i个Sigmoid函数的偏置。
- `ci`：是第i个Sigmoid函数的输出权重。
这个扩展后的模型，每一个Sigmoid函数都会接收所有的输入特征，并对它们进行加权求和，然后通过Sigmoid激活，最后再乘以一个输出权重。所有这些结果相加，再加上一个全局偏置，就得到了最终的预测值y。这个模型拥有大量的未知参数（所有的`wij`, `bi`, `ci`以及全局`b`），使其具有极高的灵活性和表达能力，能够学习到输入特征与输出之间极其复杂的非线性关系。

**延伸思考**

这个多Sigmoid叠加、多特征输入的模型，实际上已经初步展现了神经网络的结构。每一个Sigmoid函数可以被看作是一个“神经元”，它们并行地处理输入信息，并将各自的输出贡献给最终的预测。这种并行处理和层级组合的模式，正是神经网络强大能力的基础。

### 统一参数表示与优化目标：θ与Loss Function `[17:00-20:00]`

**核心观点**

为了简化表示和优化，模型中所有的未知参数（权重和偏置）被统一表示为一个长向量`θ`。优化目标是找到一组`θ`，使得模型在训练数据上的损失函数`L(θ)`最小。

**深度阐述**

在构建了如此复杂的神经网络模型之后，我们面临一个实际问题：模型中包含了大量的未知参数。这些参数包括了连接输入层到隐藏层的权重矩阵`W`，隐藏层的偏置向量`b`*`hidden`**，连接隐藏层到输出层的权重向量**`c`**，以及最终的全局偏置**`b`*`output`。如果每次都分别提及这些参数，将会非常繁琐。

为了解决这个问题，李老师引入了一个统一的符号：**`θ`**** (theta)**。

**重要原话**："我们把这个黄色的这个 w，把这个 b，把这个 c，把这个 b 统统拿出来，集合在这边，它们就是我们的 Unknown 的 Parameters，就是我们的未知的参数。我们把这些东西通通拉直，拼成一个很长的向量，我们把 w 的每一个 Row 或者是每一个 Column 拿出来... 拼成一个长的向量，把 b 拼上来，把 c 拼上来，把 b 拼上来。这个长的向量，我们直接用一个符号叫做 θ 来表示它。" `[17:50-18:40]`

`θ`是一个非常长的向量，它包含了模型中所有的可学习参数。无论是矩阵`W`中的每一个元素，还是向量`b`*`hidden`**、**`c`**、**`b`*`output`中的每一个元素，都被“拉直”并拼接到了`θ`这个单一的向量中。这样，我们就可以用一个简洁的符号`θ`来代表整个模型的参数集合。

接下来，我们需要定义如何衡量模型的好坏，这就是**损失函数（Loss Function）**。

损失函数的目的是量化模型预测值与真实值之间的差异。对于我们预测YouTube观看人数的例子，损失函数通常是预测值与真实值之间差的平方和（或其他形式的误差度量）。

**重要原话**："我们现在接下来的未知的参数很多了，你再把它一个一个列出来太累了，所以我们直接用 θ 来统设所有的参数... 所以我们现在 Loss Function 就变成 L(θ)。这个 Loss Function 要问的就是，这个 θ 如果它是某一组数值的话，会有多不好或有多好。" `[19:30-20:00]`

现在，由于所有的参数都统一表示为`θ`，我们的损失函数也相应地表示为`L(θ)`。这个函数的作用是：给定一组特定的参数值`θ`，它会计算出模型在所有训练数据上的总误差。我们的目标就是找到一组`θ`，使得`L(θ)`的值最小。这组最小化损失的参数`θ`，就是我们模型学习到的最佳参数。

**个人感受**

将所有参数打包成一个`θ`向量，是机器学习中一种非常优雅且实用的抽象。它不仅简化了数学表示，也为后续的优化算法（如梯度下降）提供了统一的操作接口。这种从具体到抽象的思维转换，是理解现代机器学习框架的关键一步。

### 优化算法：梯度下降与实践细节 `[20:00-26:00]`

**核心观点**

梯度下降（Gradient Descent）是寻找最小化损失函数参数`θ`的核心算法。它通过迭代地沿着损失函数梯度（最陡峭的上升方向）的反方向更新参数。在实践中，为了效率和稳定性，通常采用小批量梯度下降（Mini-batch Gradient Descent），引入了“Batch”和“Epoch”的概念。

**深度阐述**

在定义了损失函数`L(θ)`之后，我们的下一个任务就是找到能够最小化`L(θ)`的参数`θ`。这个过程被称为**优化（Optimization）**。李老师指出，即使模型变得更复杂，核心的优化算法仍然是**梯度下降（Gradient Descent）**。

**梯度下降原理**：

1. **初始化参数**：首先，我们随机选择一组初始参数`θ^0`。
1. **计算梯度**：在当前参数`θ^t`处，计算损失函数`L(θ)`对每个参数的偏导数。这些偏导数组成的向量就是**梯度（Gradient）**，通常用`∇L(θ^t)`或`g`表示。梯度指向损失函数上升最快的方向。
**重要原话**："你要对每一个未知的参数... 都去计算它对 L 的微分。那把每一个参数都拿去计算对 L 的微分以后，集合起来它就是一个向量，那个向量我们用 g 来表示它... 这个向量有一个名字叫做 Gradient。" `[20:50-21:30]`

1. **更新参数**：为了最小化损失，我们需要沿着梯度的反方向移动。因此，参数更新公式为：
`θ^(t+1) = θ^t - η `*` g`*

其中，`η` (eta) 是**学习率（Learning Rate）**，它是一个超参数（Hyperparameter），决定了每次参数更新的步长。学习率过大可能导致震荡甚至发散，过小则训练速度慢。

1. **迭代**：重复步骤2和3，直到满足某个停止条件（例如，梯度接近零，损失不再显著下降，或者达到预设的迭代次数）。
**实践中的梯度下降：Batching与Epochs**

在实际应用中，尤其当训练数据量非常大时，每次计算所有数据的损失并更新参数会非常耗时。因此，我们通常采用**小批量梯度下降（Mini-batch Gradient Descent）**。

**重要原话**："实际上我们做 Gradient Descent 的时候，我们会这么做：我们这边有大 N 笔资料，我们会把这大 N 笔资料分成一个一个的 Batch，就是一包一包的东西，一组一组的。怎么分？随机分就好。" `[22:50-23:20]`

1. **Batch（批次）**：我们将总数据集（大N笔资料）随机分成若干个小批次（Batch），每个批次包含大B笔资料。
1. **Update（更新）**：每次参数更新不再基于全部数据，而是只基于当前选定的一个Batch的数据计算损失`L`*`batch`**和梯度，然后更新参数。*
1. **Epoch（周期）**：当所有的Batch都被用来更新过一次参数后，就完成了一个**Epoch**。一个Epoch中会发生多次参数更新（Update），更新次数等于Batch的数量。
**重要原话**："把所有的 Batch 都看过一次，叫做一个 Epoch。每一次更新参数叫做一次 Update。所以你在文献上，常常会有人听到 Update 这个词汇，常常有人听到 Epoch 这个词汇。那 Update 跟 Epoch 是不一样的东西。" `[24:30-25:00]`

李老师通过具体例子解释了Update和Epoch的区别：

- 如果10000笔数据，Batch Size为10，那么一个Epoch包含1000个Batch，参数会更新1000次。
- 如果1000笔数据，Batch Size为100，那么一个Epoch包含10个Batch，参数会更新10次。
这表明，仅仅说“训练了一个Epoch”并不能说明模型更新了多少次，还需要知道Batch Size。Batch Size也是一个**超参数（Hyperparameter）**，需要我们手动设定。至此，我们已经遇到了多个超参数：学习率、Sigmoid的数量（或ReLU的数量）、Batch Size。这些超参数的选择对模型的训练效果至关重要。

**延伸思考**

小批量梯度下降是现代深度学习训练的基石。它在计算效率和收敛稳定性之间取得了良好的平衡。理解Batch和Epoch的概念，对于阅读深度学习论文和实践模型训练至关重要。同时，超参数的选择艺术也在此处初现端倪，它需要经验、实验和一定的直觉。

### 替代激活函数：ReLU的崛起与多层网络 `[26:00-30:00]`

**核心观点**

除了Sigmoid，Rectified Linear Unit (ReLU) 是另一种常用且高效的激活函数，它能更简单地构建分段线性函数。通过堆叠多个这样的“层”，我们可以构建更深层次的神经网络，进一步提升模型的表达能力。

**深度阐述**

在前面的讨论中，我们主要使用了Sigmoid函数来近似“蓝色函数”。然而，李老师提出，Sigmoid并非唯一的选择，甚至可能不是最佳选择。他引入了另一种在现代深度学习中更为流行的激活函数：**Rectified Linear Unit (ReLU)**。

**ReLU函数**的数学表达式非常简洁：

`y = c `*` max(0, b + w `*` x1)`

这意味着：

- 如果 `b + w `*` x1`** 小于0，输出就是0（水平线）。*
- 如果 `b + w `*` x1`** 大于0，输出就是 **`c `*` (b + w `*` x1)`**（斜坡）。*
**视觉/结构信息描述**：ReLU的图形就像一个“折棍”，在某个点之前是水平的0，在该点之后则以一定的斜率线性上升。

李老师指出，一个“蓝色函数”（即Hard Sigmoid）实际上可以由两个ReLU函数叠加而成。一个ReLU负责从0到斜坡的上升段，另一个ReLU负责从斜坡到另一个水平段的下降或保持。因此，我们完全可以用ReLU来替代Sigmoid作为神经元的激活函数。如果用ReLU来构建模型，原来的Sigmoid部分就会被替换为`max(0, bi + Σj wij `*` xj)`**。由于一个Hard Sigmoid需要两个ReLU来合成，所以如果原来有**`i`**个Sigmoid，现在可能需要**`2i`**个ReLU才能达到相似的表达能力。*

**重要原话**："这种线呢在机器学习里面，我们叫做 Rectified Linear Unit，它的缩写叫做 ReLU，名字念起来蛮有趣的，它真的就唸 ReLU。" `[27:50-28:10]`

**重要原话**："这个 Sigmoid 或是 ReLU，它们在机器学习里面，我们就叫它 Activation Function，它们是有名字的，它们统称为 Activation Function。" `[28:40-29:00]`

Sigmoid和ReLU都被统称为**激活函数（Activation Function）**。它们是神经网络中引入非线性的关键组件。没有激活函数，无论网络有多少层，都只会是线性模型的堆叠，无法学习复杂的非线性关系。

**深化网络：多层感知机（Multi-layer Perceptron）**

除了更换激活函数，我们还可以通过**堆叠多层**来进一步增强模型的表达能力。

之前我们讨论的模型可以看作是一个“单隐藏层”的神经网络：输入 `x` 经过一层计算（加权和 + 激活）得到 `a`，然后 `a` 再经过一层计算（加权和 + 激活）得到最终输出 `y`。

李老师提出，我们可以将这个过程重复多次。例如，我们可以将 `a` 再次作为输入，乘以另一组权重 `W'`，加上另一组偏置 `b'`，再通过一个激活函数（Sigmoid或ReLU），得到新的输出 `a'`。然后 `a'` 又可以作为下一层的输入，如此反复。

**视觉/结构信息描述**：神经网络图上，在输入层和输出层之间，增加更多的“隐藏层”。每一层都由多个神经元组成，每个神经元接收前一层所有神经元的输出，进行加权求和并激活，然后将结果传递给下一层。

**重要原话**："我们可以把这个同样的事情再反覆地多做几次。刚才我们把 x 乘上 w 加 b 通过 Sigmoid Function 得到 a，我们可以把 a 再乘上另外一个 w' 再加上另外一个 b' 再通过 Sigmoid Function 或 ReLU Function 得到 a'。" `[29:40-30:10]`

这种堆叠多层隐藏层的结构，就是**深度神经网络（Deep Neural Network）**的基础。每一层都学习到输入数据不同层次的抽象特征，层数越多，理论上模型可以学习到的特征越复杂，表达能力也越强。当然，每增加一层，就会引入更多的未知参数（新的权重矩阵和偏置向量），这也会增加模型的复杂性和训练难度。至于要堆叠多少层，这也是一个需要我们自己决定的**超参数**。

**延伸思考**

ReLU的引入是深度学习发展史上的一个重要里程碑，它解决了Sigmoid函数在深层网络中可能出现的梯度消失问题，使得训练深层网络变得更加容易。多层网络的思想，让模型能够从原始数据中自动学习出层次化的特征表示，这是深度学习“深度”的真正含义。

### 实验结果、过拟合与模型选择 `[30:00-34:00]`

**核心观点**

通过实验对比不同模型复杂度（ReLU数量、网络层数）对性能的影响，我们发现增加模型复杂度可以降低训练损失，但过度复杂可能导致“过拟合”，即在训练数据上表现良好，但在未见过的新数据上表现下降。因此，选择模型时应以在未见过数据上的表现为准。

**深度阐述**

理论的探讨最终要回归到实践。李老师通过一系列真实的实验结果，展示了不同模型配置对预测YouTube观看人数任务的影响。

**实验一：增加ReLU数量（模型宽度）**

- **线性模型 (Linear Model)**：在56天的数据上，训练损失0.32k，测试损失0.46k。
- **10个ReLU**：性能与线性模型相近，提升不明显。
- **100个ReLU**：训练损失从0.32k降至0.28k，测试损失也有所改善。这表明模型宽度增加，能够捕捉更复杂的模式。
- **1000个ReLU**：训练损失进一步降低，但在测试数据上没有显著进步。这暗示着模型可能已经足够复杂，或者即将面临过拟合。
**实验二：增加网络层数（模型深度）**

在固定每层100个ReLU的情况下，李老师比较了不同层数的网络：

- **1层 (1-layer)**：训练损失0.28k，测试损失0.43k（与100个ReLU的结果一致）。
- **2层 (2-layer)**：训练损失显著下降到0.18k，测试损失也改善到0.38k。
- **3层 (3-layer)**：训练损失进一步下降到0.14k，测试损失改善到0.38k。
- **4层 (4-layer)**：训练损失继续下降到0.1k（在训练数据上表现最好），但**测试损失却上升到0.44k**。
**重要原话**："这种训练资料跟没看过的资料，它的结果是不一致的状况，这个状况叫做 Overfitting。那你常常听到有人说，机器学习会发生 Overfitting 的问题，指的就是在训练资料上有变好，但是在没看过的资料上没有变好这件事情。" `[33:30-34:00]`

**过拟合 (Overfitting)**：

4层网络的结果是一个典型的**过拟合**现象。模型在训练数据上表现得越来越好（损失持续降低），因为它开始学习训练数据中的噪声和特有模式，而不是普遍规律。然而，这种对训练数据过度学习导致它在面对未见过的新数据时，泛化能力下降，表现反而变差。

**模型选择**：

面对过拟合，我们必须做出选择。当我们需要预测未来的、未见过的数据时，我们应该选择在**未见过数据（测试数据）上表现最好的模型**，而不是在训练数据上表现最好的模型。在上述实验中，3层网络在测试数据上的损失是0.38k，而4层网络是0.44k。因此，尽管4层网络在训练数据上损失更低，但为了更好地预测未来，我们应该选择**3层网络**。

**重要原话**："我们应该选一个在训练的时候，没看过的资料上表现会好的模型。所以我们应该选 3 层的 Network。" `[40:30-40:50]`

**个人感受**

李老师通过实际的实验数据，直观地展示了模型复杂度和过拟合之间的关系，以及在实践中如何进行模型选择。这种“眼见为实”的教学方式，比纯理论讲解更具说服力。他引导学生思考“为什么不选4层”，并强调了“我们不在意训练资料的结果，我们在意的是没看过的资料”这一核心原则，这对于机器学习实践者来说是金玉良言。

### 真实预测案例与深度学习的命名之谜 `[34:00-39:00]`

**核心观点**

模型在实际预测中可能因缺乏特定背景知识而出现偏差，揭示了数据和特征工程的重要性。同时，深度学习的命名并非纯粹的技术考量，而是包含了历史、营销和重塑形象的策略。

**深度阐述**

在展示了模型选择的原则后，李老师将目光投向了真实的预测任务。他用3层网络来预测未来几天YouTube频道的观看人数。

**真实预测与挑战**：

模型预测了2月25日和2月26日的观看人数，并指出2月26日的预测值较低，因为模型“知道”周五的观看人数通常较少，这听起来很合理。然而，在回顾历史数据时，李老师展示了一个有趣的现象：模型在某个特定日期（除夕）的预测出现了显著偏差，它高估了观看人数，未能预测到那个明显的低谷。

**重要原话**："这天最低点是什么？这天最低点就是除夕啊！谁除夕还学机器学习？对不对？" `[35:30-35:50]`

**重要原话**："它根本不知道除夕是什么，它只知道看前 56 天的值来预测下一天会发生什么事。所以它不知道那一天是除夕，所以你不能怪它预测地不准。" `[36:00-36:20]`

这个例子生动地说明了，即使是复杂的深度学习模型，也只能从它所“看到”的数据中学习。如果数据中没有包含“除夕”这样的节假日信息，或者模型没有被设计来理解这种周期性的人类社会行为，它就无法做出准确的预测。这强调了**特征工程**（为模型提供更多有意义的输入特征）和**领域知识**的重要性。

**深度学习的命名之谜**：

接下来，李老师以一种极具幽默感的方式，揭示了“深度学习”这个名字背后的故事。他将模型比作“死臭酸宅”，强调一个好的名字和包装的重要性。

1. **Neuron (神经元)**：我们之前讨论的Sigmoid或ReLU函数，在神经网络的语境中，被称为“神经元”，因为它模拟了生物神经元接收输入、处理并产生输出的机制。
1. **Neural Network (神经网络)**：当有大量的神经元连接在一起时，就形成了“神经网络”，这听起来就像是模拟人脑。
**重要原话**："这些 Sigmoid 或 ReLU 啊，它们叫做 Neuron。我们这边有很多的 Neuron，很多的 Neuron 叫什么？很多的 Neuron 就叫做 Neural Network。Neuron 是什么？Neuron 就是神经元。人脑中就是有很多神经元，很多神经元串起来就是个神经网络，跟你的脑是一样的。接下来你就可以到处骗麻瓜说，看到没有，这个模型就是在模拟人们脑，知道吗？这个就是在模拟人脑，这个就是人工智慧，然后麻瓜就会吓得把钱掏出来。" `[36:40-37:30]`

然而，李老师紧接着揭露了历史的真相：神经网络并非新技术，早在80、90年代就已经存在。但由于当时的过度吹捧和技术限制，导致其声誉受损，甚至“Neural Network”这个词在学术界一度成为“脏话”，写在论文里都可能被拒。

为了重振这项技术，需要一个新的名字。

1. **Hidden Layer (隐藏层)**：神经网络中，输入层和输出层之间的层被称为“隐藏层”。
1. **Deep Learning (深度学习)**：当神经网络拥有多个“隐藏层”时，它就被称为“Deep”（深）。于是，这项技术被重新命名为“Deep Learning”。
**重要原话**："所以后来为了要重振 Neural Network 的雄风，所以怎么办呢？需要新的名字。怎么样新的名字呢？这边有很多的 Neuron，每一排 Neuron 我们就叫它一个 Layer，它们叫 Hidden Layer。有很多的 Hidden Layer 就叫做 Deep。这整套技术就叫做 Deep Learning。好，我们就把 Deep Learning 讲完了，就是这么，就是这么回事，就是这样来的。" `[37:50-38:40]`

这个故事揭示了技术命名中除了技术内涵，还包含着市场营销、历史包袱和形象重塑的复杂考量。

**延伸思考**

“除夕”的例子提醒我们，模型并非万能，它受限于训练数据和我们提供的特征。真正的智能需要更深层次的理解和推理能力。而深度学习的命名故事，则是一个有趣的案例，说明了科学术语的演变，以及在技术推广中“包装”的重要性。

### “深”的意义何在？与模型选择的最终考量 `[39:00-43:00]`

**核心观点**

尽管理论上一个足够宽（胖）的单层网络可以近似任何函数，但实践中，深层网络（多层）通常表现更好，这引出了“深”的内在优势的哲学问题。在实际应用中，模型选择的最终依据是其在未见过数据上的泛化能力，而非训练损失。

**深度阐述**

在揭示了“深度学习”的命名由来后，李老师提出了一个深刻且引人深思的问题：既然我们知道一个足够多的Sigmoid或ReLU（即一个足够“宽”或“胖”的单隐藏层网络）理论上就可以逼近任何连续函数，那么**“深”的意义到底何在？**为什么我们要反复堆叠这些激活函数，而不是把它们都排成一排，构建一个“肥胖的神经网络”？

**重要原话**："我们一开始说，我们想要用 ReLU 或者是 Sigmoid 去逼近一个复杂的 Function，实际上只要够多的 ReLU，够多的 Sigmoid，就可以逼近任何的连续的 Function 对不对？... 所以我们只要一排 ReLU，一排 Sigmoid，够多就足够了。那深的意义到底何在呢？把 ReLU Sigmoid Function 反覆用，到底有什么好处呢？为什么不把它们直接排一排呢？直接排一排也可以表示任何 Function 啊！所以把它反覆用没什么道理啊！" `[38:40-39:30]`

这个问题触及了深度学习理论的核心。虽然万能近似定理（Universal Approximation Theorem）表明一个足够宽的单隐藏层网络可以近似任何连续函数，但在实践中，深层网络往往在学习效率、特征表示能力和泛化性能上优于同等参数量的浅层网络。深层结构允许模型学习到层次化的特征表示，每一层都提取不同抽象级别的特征，从而更有效地捕捉数据的内在结构。李老师将这个问题留作后续课程的伏笔，激发了听众对“深”的内在价值的思考。

**模型选择的最终考量**：

课程的最后，李老师再次强调了模型选择的实践原则。他以3层和4层网络的实验结果为例，再次向听众提问：如果我们要预测2月26日的观看人数，应该选择3层还是4层网络？

**重要原话**："我们要做的事情就是预测未知的资料。但是如果我们要预测未知的资料，我们应该选 3 层的 Network 还是 4 层的 Network 呢？" `[39:50-40:10]`

尽管4层网络在训练数据上的损失更低（0.1k），但它在未见过数据（2021年数据）上的损失却更高（0.44k），表现出了过拟合。而3层网络在未见过数据上的损失是0.38k，表现更好。因此，为了预测未知的未来数据，我们必须选择在**未见过数据上表现最好的模型**，即3层网络。

**重要原话**："我们并不在意训练资料的结果啊，我们在意的是没有看过的资料。而 2 月 26 号是没有看过的资料，我们应该选一个在训练的时候，没有看过的资料上表现会好的模型。所以我们应该选 3 层的 Network。" `[40:40-41:10]`

李老师通过一个互动环节，让听众投票选择，并肯定了大多数人选择3层网络的“Sense”。这不仅巩固了过拟合和模型选择的概念，也让听众感受到了将理论应用于实践的乐趣。

**最终预测与幽默互动**：

课程以一个轻松幽默的预测环节结束。李老师用3层网络预测了2月25日和2月26日的观看人数，并与听众打趣，如果预测不准，可能是因为大家回去“点爆”了视频，从而“骗大家去点影片”。

**重要原话**："当然我想应该是不会准啦，因为看这么多人都觉得误差会大，你们回去每个人都去点那个影片的话，哇，误差就大了啊！今天讲这么久，其实就是骗大家去点影片而已啦！" `[42:30-43:00]`

最后，李老师简要提到了**反向传播（Backpropagation）**，指出它是一种更高效计算梯度的算法，但其核心思想与梯度下降并无本质区别。

**个人感受**

这一部分将理论的深度与实践的考量完美结合。李老师提出的“深”与“胖”的哲学问题，是深度学习领域一个持续研究的课题，它鼓励我们超越表象，探究更深层次的原理。而对模型选择的反复强调，则为学习者提供了宝贵的实践指导，避免了陷入“训练损失越低越好”的误区。整个课程以幽默收尾，既缓解了学习的疲惫，又加深了对内容的印象。

**精华收获**

1. **模型偏差**：简单的模型有结构性限制，无法拟合复杂数据，需构建更灵活的模型。
1. **万能近似**：任何连续函数都可用足够多的分段线性函数近似，而分段线性函数可由“蓝色函数”（如Sigmoid或ReLU）叠加组合。
1. **神经网络结构**：通过多层神经元（激活函数+加权和）的堆叠，形成深度神经网络，能够学习层次化的特征表示。
1. **参数与优化**：所有模型参数统一为`θ`向量，通过梯度下降算法迭代更新`θ`以最小化损失函数`L(θ)`。
1. **实践优化**：小批量梯度下降（Batching）和周期（Epoch）是高效训练大型模型的关键。
1. **超参数**：学习率、Sigmoid/ReLU数量、Batch Size、网络层数等都需要手动调整。
1. **过拟合**：模型在训练数据上表现好，但在未见过数据上表现差。
1. **模型选择**：选择模型时，应以其在未见过数据（测试集）上的泛化能力为最终依据。
1. **深度学习命名**：其命名包含历史和营销考量，旨在重塑“神经网络”的形象。
1. **“深”的价值**：深层网络在实践中通常比宽层网络更有效，尽管理论上单层宽网络也能近似任何函数，这背后有更深层次的原理。
