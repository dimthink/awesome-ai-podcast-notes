**开篇引入**

在人工智能的璀璨星空中，强化学习（Reinforcement Learning, RL）无疑是最耀眼、也最令人着迷的领域之一。从战胜人类围棋大师的AlphaGo，到自动驾驶、机器人控制，RL的应用无不彰显着其颠覆性的潜力。然而，对于许多初学者而言，RL常常被视为一个高深莫测、难以企及的领域。台湾大学李弘毅教授的这堂课，正是为了打破这种认知壁垒。他以其一贯深入浅出、幽默风趣的风格，旨在揭示强化学习的本质——它并非与机器学习格格不入的异类，而是遵循着同样的“三步曲”框架，只是在具体实现上，面对着独特的挑战。这篇深度重构的文章，将带你跟随李教授的思路，一步步解开强化学习的神秘面纱，让你在理解其核心概念的同时，感受到“原来如此，我也能做”的启发与信心。

## 第一章：强化学习的独特视角：当“正确答案”不再明确时 `[00:00-03:40]`

**核心观点**

强化学习解决的是在缺乏明确“正确答案”的情况下，通过与环境互动、获取反馈（奖励）来学习最优行为的问题。它与我们熟悉的监督学习有着本质的区别，并在特定场景下展现出不可替代的价值。

**深度阐述**

李弘毅教授开宗明义地指出，强化学习（RL）在当今AI领域拥有众多“很潮”的应用，例如AlphaGo等，其背后都离不开RL技术的支撑。然而，RL本身是一个极其广阔且深奥的领域，绝非一堂课所能尽述。教授明确了本堂课的目标：并非要穷尽RL的一切，而是为听众建立一个基本的认知框架，让大家对RL有一个初步的了解，并期望听众在课后能感受到RL并非高不可攀，而是“原来就只是这样而已，我自己应该也做得起来”。这种降低门槛的教学理念，贯穿了整个课程。

为了更好地理解RL的独特之处，教授首先将其与我们此前课程中反复提及的“监督学习”（Supervised Learning）进行了对比。在监督学习中，无论是图像分类器还是其他模型，我们总是需要为机器提供明确的输入（Input）和对应的“正确输出”（Output），即所谓的“标签”（Label）。例如，在图像分类中，你不仅要给机器一张图片，还要告诉它这张图片是“猫”还是“狗”。即使是自监督学习（Self-Supervised Learning）或自编码器（Auto-encoder），虽然标签可能无需人工标注而能自动生成，但其本质上仍然存在一个明确的“标签”作为学习目标。

然而，强化学习所面对的问题则截然不同。在RL的场景中，当我们给机器一个输入时，我们往往**不知道最佳的输出应该是什么**。教授以围棋为例进行了生动阐述：如果你想用监督学习的方法教机器下围棋，你需要告诉它“看到当前盘势时，下一步应该落子在哪里”。但问题是，哪一步才是“神之一手”？哪一步是最好的下一步？人类棋手本身可能都无法给出确切的答案。尽管我们可以让机器学习大量职业棋手的棋谱，从中找到某种盘势下人类的落子作为“答案”，但这并不能保证这些答案就是“最好”的。

**重要原话**： "在这个你不知道正确答案是什么的情况下，往往就是 RL 可以派上用场的时候。所以当你今天你发现你要蒐集有标註的资料很困难的时候，正确答案人类也不知道是什么的时候，也许就是你可以考虑使用 RL 的时候。" `[03:00-03:40]`

因此，当面临以下两种情况时，强化学习便能大显身手：一是**难以收集带有明确标注的数据**，二是**人类本身也无法确定“正确答案”**。在这种“无标签”或“弱标签”的环境下，RL的学习机制是让机器与环境进行互动，虽然不知道“正确答案”，但机器能够通过环境反馈的“奖励”（Reward）来判断当前行为是“好”还是“不好”。通过这种试错和反馈循环，机器最终也能学习到一个模型，知道在特定情境下应该采取何种行动。

**个人感受**

教授在开篇就点明了RL的核心价值和应用场景，并以通俗易懂的围棋例子，清晰地对比了RL与SL的根本区别。他强调RL并非要让听众望而却步，而是要让大家感受到其内在的逻辑和可操作性，这种亲和力极大地激发了学习兴趣。

**延伸思考**

这种“不知道正确答案”的学习范式，在现实世界中有哪些更广泛的应用？除了围棋和游戏，自动驾驶系统如何通过与真实或模拟环境的互动来学习驾驶策略？资源调度、金融交易、推荐系统等领域，又如何利用RL来优化决策，而无需预设“最优解”？RL的这种特性，使其在解决复杂、动态且缺乏明确指导的问题上，展现出独特的优势。

**精华收获**

强化学习是解决“探索未知”和“延迟反馈”问题的强大工具，其核心在于通过与环境互动获取“奖励”信号，而非依赖预设的“正确答案”来指导学习。

## 第二章：强化学习的本质：万变不离其宗的机器学习三部曲 `[03:40-06:50]`

**核心观点**

尽管强化学习看起来复杂，但其核心框架与任何机器学习任务一样，都遵循“定义函数”、“定义损失”和“优化”这三个基本步骤。李教授通过强调这一共通性，旨在消除RL的神秘感，将其置于机器学习的通用框架之下。

**深度阐述**

为了进一步降低RL的理解难度，李弘毅教授提出了一个核心论点：强化学习（RL）与我们这门课一开始就介绍的机器学习（Machine Learning）基本框架是完全一致的。他反复强调，机器学习的本质就是三个步骤：

1. **找一个函数（Function）**：这个函数中包含一些未知数（Unknown Variables），这些未知数是我们需要通过学习找出来的。
1. **定义一个损失函数（Loss Function）**：这个函数用来衡量我们找到的函数表现好坏的程度。
1. **优化（Optimization）**：通过某种方法（如梯度下降）来调整未知数，以最小化损失函数。
教授明确指出，RL也完全遵循这三个步骤。那么，在RL中，这三个步骤具体对应着什么呢？

首先，在RL中，我们要找的这个函数被称为“**Actor**”（执行者）。Actor是机器的“大脑”，它负责根据当前环境的观察来决定下一步的行动。与Actor互动的是“**Environment**”（环境）。Actor和Environment之间会进行一个持续的互动循环：

- Environment会给Actor一个“**Observation**”（观测），这个Observation就是Actor的输入。
- Actor看到Observation后，会产生一个“**Action**”（行动），这个Action就是Actor的输出。
- 这个Action会去影响Environment，导致Environment的状态发生改变。
- Environment在被Action影响后，会给Actor一个新的Observation，并同时给予Actor一个“**Reward**”（奖励），告诉Actor它刚才的Action是好是坏。
- 这个循环会不断进行下去：新的Observation成为Actor的输入，Actor产生新的Action，Action影响环境，环境给出新的Observation和Reward……
**视觉/结构信息描述**： 想象一个循环图：Environment -> Observation -> Actor -> Action -> Environment。在Environment到Actor的箭头上，还伴随着Reward的输出。Actor本身就是一个Function，其输入是Observation，输出是Action。

而Actor这个函数的目标，是**最大化它可以从Environment中获得的Reward的总和**。换句话说，我们希望找到一个Actor，它能够通过与环境的互动，在整个过程中积累到最大的总奖励。

**重要原话**： "所以 Actor 本身啊 它就是一个 Function 其實 Actor 它就是我們要找的 Function 这个 Function 它的轮入 就是环境给它的 Observation 輸出就是这个 Actor 要採取的 Action 而今天在这个互动 的过程中呢 这个 Environment 会不断地给这个 Actor 一些 Reward 告诉它说 你现在採取的这个 Action 它是好的还是不好的 而我们今天要找的这个 Actor 我们今天要找的这个 Function 可以拿 Observation 当作 Input Actor 当作 Output 的 Function 这个 Function 的目标 是要去 Maximizing 我们可以从 Environment 获得到的 Reward 的总和" `[05:30-06:50]`

教授通过这种方式，将RL的核心概念——Actor、Environment、Observation、Action、Reward——都巧妙地融入到机器学习的通用框架中，为后续的深入讲解打下了坚实的基础。

**精华收获**

强化学习的本质是寻找一个“Actor”函数，通过与“环境”的持续互动，接收“观测”并输出“行动”，最终目标是最大化从环境中获得的“总奖励”。这一过程与机器学习的“找函数、定义损失、优化”三步曲高度契合。

## 第三章：Actor与环境的舞蹈：以游戏为例具象化RL互动 `[06:50-16:50]`

**核心观点**

通过具体的游戏案例（Space Invaders和围棋），详细阐述Actor如何接收观测、采取行动、影响环境并获取奖励的整个互动过程，从而将抽象的RL概念具象化。

**深度阐述**

为了让听众更好地理解Actor与环境的互动过程，李教授举了两个非常具体的例子：经典的街机游戏“Space Invaders”（太空侵略者）和复杂的策略游戏“围棋”。

### 案例一：Space Invaders（太空侵略者） `[08:00-13:00]`

在Space Invaders游戏中，玩家操控一艘绿色的太空梭，目标是击杀屏幕上方的黄色外星人。

- **Actor**：在这里，Actor就是那个坐在“人”的位置上，操控摇杆来控制太空梭的机器。它站在玩家的角度，与外星人对抗。
- **环境（Environment）**：游戏的主机就是环境。它控制着外星人的移动和攻击，并根据Actor的行动更新游戏状态。
- **观测（Observation）**：对Actor来说，它看到的与人类玩家无异，就是**游戏的画面**。这个画面包含了外星人的位置、太空梭的位置、子弹的轨迹等所有视觉信息。
- **行动（Action）**：在这个游戏中，Actor可以采取的行动是预先定义好的，只有三种：**向左移动、向右移动、开火**。
- **奖励（Reward）**：游戏中的**分数**被直接定义为奖励。当你击中并杀死外星人时，你会得到分数；有时屏幕上会出现补给包，击中它也会获得高分。这些分数就是环境给予Actor的即时反馈。
- **游戏终止**：当所有外星人被消灭，或者太空梭被外星人击毁时，游戏就结束了。
教授详细描述了互动过程：Actor看到游戏画面（Observation），决定向右移动（Action），此时因为没有击杀外星人，Reward为0。游戏画面更新（新的Observation），Actor看到新画面后决定开火（Action），如果恰好击杀了一只外星人，就会得到5分（Reward）。这个过程不断重复，直到游戏结束。Actor的目标就是学习一个策略，使得在整个游戏过程中，获得的**总分数（Total Reward）最大**。

**重要原话**： "这个 Score 呢 就是 Reward 就是环境给我们的 Reward" `[10:40]`

### 案例二：围棋（AlphaGo） `[13:00-16:50]`

将RL应用于围棋，其基本原理与Space Invaders并无本质区别，只是问题的规模和复杂性大大增加。

- **Actor**：在围棋中，Actor就是AlphaGo这样的AI棋手。
- **环境（Environment）**：人类对手（例如李世石）就是环境的一部分。环境会根据Actor的落子，做出相应的回应。
- **观测（Observation）**：Actor的输入是**棋盘的当前状态**，即棋盘上黑子和白子的位置。游戏开始时，棋盘是空的。
- **行动（Action）**：Actor的输出是**下一步的落子位置**。围棋棋盘有19x19个交叉点，因此有361种可能的行动。
- **奖励（Reward）**：围棋的奖励机制与Space Invaders不同，它是一种**延迟奖励**。在下围棋的过程中，每一步落子几乎都不会立即得到奖励。只有当一局棋结束时，如果Actor赢了，会得到**+1分**；如果输了，则得到**-1分**。中间过程的奖励都视为0分。
- **Actor目标**：同样是最大化它可能得到的总奖励，即最大化赢棋的概率。
**个人感受**

教授通过这两个对比鲜明的例子，成功地将RL的抽象概念具象化。Space Invaders展示了即时奖励的场景，而围棋则突出了延迟奖励的挑战。他强调，无论问题多么复杂，RL的核心互动模式都是一致的，这极大地增强了听众对RL普适性的理解。

**延伸思考**

延迟奖励是RL中一个重要的挑战。在围棋中，一个好的开局可能在几十步之后才显现出其价值，而一个糟糕的落子也可能在很久之后才导致失败。Actor如何在这种长期的因果链中，将最终的奖励归因到早期的行动上，是RL算法设计的关键。这引发了对“信用分配问题”（Credit Assignment Problem）的思考。

**精华收获**

通过游戏案例，清晰理解了Actor、环境、观测、行动、奖励在实际应用中的具体表现，特别是即时奖励与延迟奖励的区别。

## 第四章：构建Actor：从简单查表到深度神经网络的策略网络 `[16:50-22:00]`

**核心观点**

Actor本质上是一个带有未知参数的函数，在现代RL中通常表现为一个深度神经网络（策略网络），它将环境观测作为输入，输出采取各种行动的概率或分数。其行动选择通常采用基于概率的采样，而非简单的贪婪选择，以促进探索。

**深度阐述**

在强化学习的“三步曲”中，第一步是“找一个函数”，这个函数就是我们的Actor。李教授详细阐述了Actor的实现方式，特别是它如何演变为现代深度学习中的“策略网络”（Policy Network）。

在过去，当深度学习尚未普及到RL领域时，Actor可能是一个相对简单的结构，例如一个“查表”（Look-Up-Table）。它会记录“看到什么输入就产生什么输出”的规则。然而，在今天，Actor通常是一个复杂的**深度神经网络**。

这个神经网络的输入是环境的“观测”（Observation）。例如，在Space Invaders游戏中，输入就是**游戏画面的像素**。如果是围棋，输入就是棋盘上黑白子的位置信息。

神经网络的输出则代表了Actor可以采取的**每一个行动的分数或概率**。教授以Space Invaders为例：输入一个游戏画面，Actor（神经网络）的输出可能是“向左：0.7分，向右：0.2分，开火：0.1分”。

**与分类任务的类比**：教授指出，这与分类任务的神经网络非常相似。在分类中，输入一张图片，神经网络会为每个类别输出一个分数，通常通过Softmax层将其转换为概率，所有类别的概率总和为1。在RL中，Actor的策略网络也是如此，它为每个可能的行动（如向左、向右、开火）输出一个分数或概率，这些概率的总和也通常是1。

**网络架构的选择**：Actor的网络架构可以根据输入数据的类型自由设计。如果输入是图像，自然会想到使用**卷积神经网络（CNN）**。如果需要考虑时间序列信息，例如不仅看当前画面，还要看过去所有画面，那么**循环神经网络（RNN）**或更先进的**Transformer**模型都可能被采用。教授提到，在助教的示例代码中，可能为了简化，并未直接处理原始像素，而是使用了一些游戏参数作为输入，因此可能只用了简单的全连接网络。但如果直接处理游戏画面，CNN是常见的选择。

**行动选择的随机性**：一个关键的细节是，机器最终决定采取哪个行动，通常不是简单地选择分数最高的那个。更常见的做法是，将这些分数视为**概率**，然后根据这些概率**进行采样（Sample）**，随机决定要采取的行动。例如，向左0.7分，就意味着有70%的概率向左，20%的概率向右，10%的概率开火。

**重要原话**： "为什么不是看 Left 的分数最高 就直接向左呢 你也可以这么做 但是在助教的程式里面 还有多數 RL 應用的時候 你會發現 我們都是採取 Sample 採取 Sample 有一個好處是說 今天就算是看到同樣的遊戲畫面 你的機器每一次採取的行為 也會略有不同" `[18:47-19:30]`

教授解释了采取采样的两个重要原因：

1. **引入随机性**：即使面对相同的游戏画面，机器每次采取的行动也可能略有不同。这在某些场景下非常重要，例如在“剪刀石头布”游戏中，如果总是出石头，很容易被对手摸清规律而“打爆”。随机性使得策略更具鲁棒性。
1. **促进探索（Exploration）**：这是RL中一个非常重要的概念。如果机器总是选择当前看起来最好的行动（贪婪策略），它可能会陷入局部最优，而无法发现更好的、但需要先经历一些“不好”行动才能达到的策略。通过采样，机器有机会尝试那些当前分数不那么高、但可能带来长期更大奖励的行动，从而更好地探索环境。
这个作为Actor的神经网络，其内部的参数（权重和偏置）就是我们要通过学习找出来的“未知数”。

**精华收获**

Actor在现代RL中通常是一个深度神经网络（策略网络），它将环境观测作为输入，输出每个可能行动的概率分布。通过基于概率的“采样”来选择行动，引入随机性，这不仅能增强策略的鲁棒性，更重要的是促进了对环境的“探索”，避免陷入局部最优。

## 第五章：定义损失：最大化总奖励，最小化负回报 `[22:00-27:00]`

**核心观点**

强化学习的“损失”定义围绕着最大化一个“Episode”中所有即时奖励的总和（Total Reward或Return），这可以通过最小化其负值来实现，从而将其纳入机器学习的通用优化框架。

**深度阐述**

在机器学习的“三步曲”中，第二步是“定义损失函数”。在强化学习中，这个损失函数的定义方式，直接关系到我们如何衡量Actor的表现，并指导其学习。

教授首先重新审视了机器与环境的互动过程，并引入了几个关键概念：

- **互动序列（Sequence）**：从游戏开始到结束，机器与环境之间会产生一系列的观测（s）、行动（a）和奖励（r）。例如：s1 -> a1 -> r1 -> s2 -> a2 -> r2 -> ... -> sT -> aT -> rT。
- **Episode（回合）或 Trajectory（轨迹，τ）**：从游戏开始到结束的整个互动过程，被称为一个Episode或Trajectory。游戏结束的条件取决于具体任务，例如Space Invaders中所有外星人被杀光或飞船被毁，围棋中一方认输或棋局结束。
- **Reward (r)**：指机器采取某个行动后，立即从环境获得的反馈。例如，在Space Invaders中，击杀外星人立即获得5分。
- **Total Reward（总奖励）或 Return（回报，R）**：指在一个完整的Episode中，所有即时奖励的总和。即 R = r1 + r2 + ... + rT。教授指出，虽然Reward和Return在严格意义上略有不同（Reward是即时的，Return是累积的），但在本课程中，为了简化理解，将统一使用“Total Reward”来指代整个Episode的累积奖励。
**目标与损失的转换**：

强化学习的核心目标是**最大化（Maximize）**这个Total Reward (R)。我们希望找到一个Actor，它能让R的值尽可能大。

然而，在传统的机器学习优化中，我们通常是**最小化（Minimize）**一个损失函数。为了将“最大化R”的目标转化为“最小化损失”的形式，教授提出了一个巧妙的转换：

我们可以将**负的Total Reward (-R)** 定义为我们的损失函数。

**重要原话**： "Total Reward 是要越大越好 那負的 Total Reward 當然就是要它越小越好吧 就我們完全可以說負的 Total Reward 就是我們的 Loss 就是 RL 裡面的 Loss" `[26:00-26:20]`

通过这种方式，最大化Total Reward (R) 的问题，就等价于最小化负的Total Reward (-R) 的问题。这样，强化学习的第二步——定义损失函数——就与机器学习的通用框架完美衔接起来。

**视觉/结构信息描述**： 想象一个流程图，从s1, a1, r1...到sT, aT, rT，最终所有r累加得到R。这个R就是我们希望最大化的目标，而-R就是我们希望最小化的损失。

**精华收获**

强化学习的损失函数定义围绕着“Total Reward”（或Return），即一个Episode中所有即时奖励的总和。通过将损失定义为负的Total Reward，RL的优化目标（最大化奖励）得以转化为机器学习通用的“最小化损失”形式。

## 第六章：优化之困：强化学习的黑盒与随机性挑战 `[27:00-36:00]`

**核心观点**

强化学习的真正难点在于其优化过程，因为环境和奖励函数通常是不可微分的黑盒，且整个系统充满随机性，使得传统基于梯度的优化方法难以直接应用。这是RL与传统机器学习训练模型最显著的区别。

**深度阐述**

在机器学习的“三步曲”中，第三步是“优化”，即找到Actor（神经网络）的参数，使其能够最大化Total Reward (R)。乍看之下，如果Actor、环境和奖励函数都是可微分的神经网络，这似乎只是一个复杂的循环神经网络（Recurrent Network）的训练问题，我们可以直接使用梯度下降等方法来解决。然而，李教授强调，RL的优化问题远非如此简单，它面临着一系列独特的挑战，使其成为一个**非一般的优化问题**。

**非传统优化问题的核心原因**：

1. **Actor输出的随机性**：
1. **环境（Environment）和奖励（Reward）是黑盒（Black Box）**：
1. **环境和奖励的随机性**：
**与GAN的类比与区别**：

教授巧妙地将RL的优化问题与生成对抗网络（GAN）进行了类比：

- **相似之处**：在GAN中，生成器（Generator）试图调整参数，以最大化判别器（Discriminator）的输出（即让判别器认为生成的图片是真实的）。在RL中，Actor（可以看作Generator）也试图调整参数，以最大化Environment和Reward（可以看作Discriminator）的输出（即Total Reward）。
- **不同之处**：GAN中的Discriminator本身也是一个神经网络，因此我们可以通过梯度下降来训练Generator。然而，在RL中，Environment和Reward是**黑盒**，它们不是神经网络，我们无法直接计算梯度。这正是RL优化问题的核心难点所在。
**学生提问与解答**：

有同学提问，如果以前的模型没有固定随机种子，是否也算有随机性？教授对此进行了澄清：

- 传统模型训练时的随机性（如参数初始化、数据顺序）与RL的随机性不同。
- RL的随机性体现在**测试时**：即使Actor已经训练好，参数固定，但在与环境互动时，每次的Episode结果都可能不同，因为环境本身的回应是随机的。这使得RL的训练和评估都更具挑战性。
**精华收获**

强化学习的优化问题之所以困难，是因为其Actor的输出是随机采样，更重要的是，环境和奖励函数是不可微分的“黑盒”，且整个系统充满内在的随机性。这使得传统的基于梯度的优化方法无法直接应用，需要特殊的算法来应对。

## 第七章：策略梯度初探：从分类视角控制Actor行为 `[36:00-45:00]`

**核心观点**

策略梯度（Policy Gradient）是一种解决RL优化问题的方法，它通过将Actor的行为控制转化为类似分类任务的损失函数设计，来指导Actor学习期望的策略。其关键在于如何定义这些“标签”和“权重”，以反映环境的奖励信号。

**深度阐述**

既然RL的优化问题如此复杂，传统的梯度下降方法难以直接应用，那么我们该如何寻找Actor的最佳参数呢？李教授在这里引入了一种常用的算法——**策略梯度（Policy Gradient）**，并从一个直观的角度来解释其核心思想。

教授首先提出了一个问题：我们如何才能控制一个Actor的输出？即，如何让Actor在看到某个特定的Observation (s) 时，采取某个特定的Action (a^)？

**借鉴分类问题**：

教授指出，我们可以将这个问题想象成一个**分类问题**。

- 如果我们希望Actor在输入`s`时输出`a^`（例如，看到某个游戏画面时，就向左移动），那么`s`就是输入，`a^`就是我们的“标签”（Ground Truth）。
- 我们可以计算Actor的输出与`a^`之间的**交叉熵（Cross-entropy）**作为损失函数。通过最小化这个损失，我们就能让Actor的输出尽可能接近`a^`。
**反向控制：避免特定行为**：

更进一步，如果我们希望Actor在看到`s`时**不要**采取`a^`呢？

- 这也很容易实现：我们只需要将损失函数的定义反过来。定义损失为**负的交叉熵（-Cross-entropy）**。
- 然后，我们仍然去最小化这个负的交叉熵。最小化负的交叉熵，就等价于最大化正的交叉熵，这意味着Actor的输出会尽可能远离`a^`。
**加权控制：行为的程度差异**：

教授进一步扩展了这种控制方式。他提出，我们对每个行为的期望并非只有“做”或“不做”这种二元选择，而是有程度上的差异。有些行为我们非常希望Actor去做（例如，击杀Boss），有些行为我们只是希望它去做（例如，击杀普通小怪），有些行为我们不希望它做，有些行为我们非常不希望它做（例如，自毁）。

为了表达这种程度差异，我们可以为每个`(s, a)`对（即在观测`s`下采取行动`a`）赋予一个**分数 ****`A`*****`n`****。*

- `A`*`n`**为正值，表示我们希望Actor采取这个行为，数值越大，期望越强烈。*
- `A`*`n`**为负值，表示我们不希望Actor采取这个行为，数值越小（负得越多），不期望越强烈。*
- 此时，损失函数可以定义为：`Loss = A`*`n  Cross-entropy`**。*
- 通过最小化这个加权的损失函数，我们就可以训练Actor，使其行为符合我们带有程度差异的期望。
**重要原话**： "所以我们有办法控制我们的 Actor 做我们想要做的事 只要我们给它适当的 Label 跟适当的 Loss" `[40:50-41:10]`

**学生提问与解答**：

- **“一直射击”问题**：有同学提问，如果只有射中外星人才有奖励，那模型会不会一直倾向于射击？教授表示，这个问题将在后续内容中通过`A`*`n`**的定义来解决，暗示**`A`*`n`的计算会考虑长期奖励和探索。
- **“这不就是监督学习吗”**：另一个非常好的问题是，这种训练方式看起来与监督学习训练分类器一模一样。教授承认，**表面上看确实如此**。我们似乎在收集`(s, a^)`这样的数据，然后训练一个分类器。然而，他强调，RL与监督学习的关键区别在于：**这些****`s`****、****`a^`****以及最重要的****`A`*****`n`******（分数）是从哪里来的？****这正是策略梯度以及整个强化学习的核心挑战和创新之处。*
**精华收获**

策略梯度通过将Actor的行为控制转化为类似分类任务的损失函数设计，利用加权的交叉熵来指导Actor学习期望的策略。虽然形式上与监督学习相似，但其核心挑战和创新在于如何动态地生成这些带有“好坏程度”的“标签”（即`A`*`n`**），这正是RL与环境互动、获取奖励的关键所在。*
