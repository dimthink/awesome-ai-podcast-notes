### 1. 双重DQN (Double DQN, DDQN)：解决Q值高估的利器 `[00:00-05:50]`

**核心观点**

传统的Q-learning方法在实践中常导致Q值被系统性高估，这会影响策略的质量。Double DQN通过解耦动作选择与Q值评估，有效纠正了这一偏差，从而学习到更优的策略。

**深度阐述**

李弘毅教授首先指出，在Q-learning的实际应用中，一个普遍且棘手的问题是Q值（Q-value）倾向于被高估。这种高估并非偶然，而是算法内在机制所致。教授引用了Double DQN原始论文中的图表来直观展示这一现象：在多个Atari游戏中，Q网络估算出的平均Q值（红色锯齿线）在训练过程中持续上升，且远高于实际通过多次游戏模拟得到的真实累积回报（蓝色线）。这种持续上升的趋势是符合直觉的，因为随着训练的进行，智能体的策略不断增强，预期回报自然会增加。然而，关键在于估算值与真实值之间的巨大鸿沟。

那么，为什么Q值会被高估呢？教授深入剖析了Q-learning的更新公式。在计算目标值（target value）时，我们总是选择能够产生最大Q值的动作。具体来说，目标值 `Y = r`*`t + γ  maxa Q(s{t+1}, a)`**。问题就出在这个 **`maxa`** 操作上。假设有多个动作，它们的真实Q值可能相近，但由于Q网络在估计时存在误差，某些动作的Q值可能会被偶然高估。一旦某个动作的Q值被高估，**`maxa`** 操作就会倾向于选择这个被高估的动作，并将其高估的Q值纳入目标值计算。如此一来，目标值就会系统性地偏高，导致Q网络不断学习这些被高估的值，从而使得Q值整体被高估。教授形象地比喻：如果四个动作的真实Q值差不多，但其中一个被网络偶然高估了，那么**`maxa`**就会“盲目”选择这个被高估的动作，使得目标值偏大。*

**重要原话**：

- "Q value 往往是被高估的" `[00:30]`
- "你总是会选那个 Q value 被高估的" `[03:40]`
为了解决这一问题，Double DQN应运而生。其核心思想是**解耦动作选择（action selection）和Q值评估（value evaluation）**。在传统的DQN中，我们使用同一个Q网络来选择动作（即找到`max`*`a Q(s', a)`**中的**`a`**）和评估该动作的Q值（即**`Q(s', a)`**）。Double DQN则引入了两个Q网络：*

1. **在线Q网络 (Online Q-network)**：用于选择下一个状态`s'`中能带来最大Q值的动作`a`*。*
1. **目标Q网络 (Target Q-network)**：用于评估由在线Q网络选择出的动作`a`*的Q值，即**`Qtarget(s', a)`**。*
这样，目标值的计算公式变为 `Y = r`*`t + γ  Qtarget(s{t+1}, argmaxa Qonline(s{t+1}, a))`**。*

教授用了一个精妙的类比来解释其工作原理：“这一个Q，它只能提案，它不能够执行。Q负责提案，它负责选a；Q prime负责执行，它负责算出Q value的值。所以今天就算是前面这个Q，做了不好的提案，它选的a是被高估的，只要后面Q prime不要高估这个值就好了。那就算Q prime会高估某一个a的值，只要前面这个Q不提案那个a，算出来的值就不会被高估了。” `[04:50-05:30]` 这就像行政和立法部门的分立，一个负责决策，一个负责执行，互相制衡，避免单一环节的偏差被放大。

在实际实现上，Double DQN的改动极小，几乎不增加任何计算量。因为DQN本身就包含两个网络：一个是在线网络（不断更新参数），另一个是目标网络（参数定期从在线网络复制，然后冻结一段时间）。Double DQN只是改变了这两个网络的使用方式：用在线网络来选择动作，用目标网络来评估Q值。这通常只需要修改一行代码。

**个人感受**

教授强调，如果只能选择一个技巧来实现，Double DQN是“正常人都会选择”的，因为它简单有效，且改动成本极低。这体现了其在实践中的普适性和重要性。

**延伸思考**

Double DQN的成功揭示了强化学习中“偏差-方差”权衡的重要性。通过引入两个网络，它有效地降低了Q值估计的偏差。这种解耦思想在其他强化学习算法中也有类似体现，例如Actor-Critic方法中Actor和Critic的分离。

**精华收获**

- Q-learning存在Q值高估问题，源于`max`操作对估计误差的敏感性。
- Double DQN通过使用在线网络选择动作，目标网络评估Q值，解耦了选择与评估过程。
- 这种解耦有效降低了Q值高估，使学习到的策略更接近最优，且实现成本极低。
### 2. 对战DQN (Dueling DQN)：优化网络架构，提升数据效率 `[05:50-12:30]`

**核心观点**

Dueling DQN通过将Q网络分解为状态价值（V）和优势函数（A）两个分支，能够更有效地利用训练数据，尤其是在未被充分探索的状态-动作对上。

**深度阐述**

Dueling DQN的独特之处在于它不改变Q-learning的算法逻辑，而是**修改了Q网络的内部架构**。传统的Q网络直接输入状态`s`，输出每个动作`a`的Q值`Q(s,a)`。Dueling DQN则将Q值的输出分解为两个独立的路径：

1. **状态价值分支 (State Value Stream)**：输出一个标量`V(s)`，代表当前状态`s`的价值，与具体动作无关。
1. **优势函数分支 (Advantage Stream)**：输出一个向量`A(s,a)`，代表在状态`s`下采取每个动作`a`相对于平均水平的优势。
最终，Q值通过将这两个分支的输出相加得到：`Q(s,a) = V(s) + A(s,a)`。

教授通过一个表格的例子来具象化这个概念。假设Q(s,a)是一个表格，V(s)是每行的固定值，A(s,a)是每行中每个动作的相对调整值。将V(s)加到A(s,a)的每一列，就得到了Q(s,a)。

这种架构的优势在于**提高了数据利用效率**。教授举例说明：假设在某个状态`s`下，我们只采样到了两个动作`a1`和`a2`，并希望它们的Q值分别从3变为4，从-1变为0。在训练网络时，网络可以选择更新`V(s)`的值（例如从0变为1），而不是直接更新`A(s,a)`。当`V(s)`更新时，该状态下所有动作的Q值都会受到影响，包括那些未被采样的动作`a3`。这意味着，即使没有直接采样到某个状态-动作对，我们也可以通过更新`V(s)`来间接更新其Q值。这在某些状态下，智能体可能只执行少数几个动作，但我们仍然希望对所有动作的Q值有一个合理的估计时，显得尤为重要。

**重要原话**：

- "dueling DQN 它唯一做的事情，是改 network 的架构" `[06:20]`
- "你不需要把所有的 state action pair 都 sample 过，你可以用比较 efficient 的方式去 estimate Q value 出来" `[09:50]`
然而，这种分解也带来了一个潜在问题：网络可能会学到让`V(s)`始终为0，而让`A(s,a)`直接等于`Q(s,a)`，这样就失去了Dueling DQN的优势。为了避免这种情况，Dueling DQN对优势函数`A(s,a)`施加了一个**约束**。最常见的约束是要求在每个状态`s`下，所有动作的优势函数值之和为零（或平均值为零）。

教授解释了如何实现这个约束：在网络输出`V(s)`和`A(s,a)`后，在将它们相加之前，对`A(s,a)`进行一个归一化操作。具体做法是，计算`A(s,a)`向量的平均值，然后将`A(s,a)`的每个元素都减去这个平均值。这样，`A(s,a)`的每个列（对应一个状态）的和就变成了0。这个归一化步骤被整合到网络中，作为无参数的层进行联合训练（jointly trained），允许反向传播。

通过这个约束，网络在更新参数时，如果想要对某个状态下的所有Q值进行整体调整（例如都加1），它就不能通过更新`A(s,a)`来实现（因为`A(s,a)`的和必须为0），而必须通过更新`V(s)`来实现。这**强制网络倾向于利用****`V(s)`****来解决问题**，从而确保了Dueling DQN架构的优势得以发挥。

**延伸思考**

Dueling DQN的思路与特征工程中的“基线减除”有异曲同工之妙。它将状态的固有价值与动作的相对价值分离，使得网络能够更专注于学习动作的差异性，而不是重复学习状态的普遍价值。这种分解思想在其他深度学习架构设计中也值得借鉴。

**精华收获**

- Dueling DQN通过将Q网络分解为状态价值V(s)和优势函数A(s,a)两个分支来优化网络架构。
- 这种分解提高了数据利用效率，即使未采样到的状态-动作对也能通过V(s)的更新而受益。
- 通过对A(s,a)施加零和（或零均值）约束，确保网络有效利用V(s)分支，避免退化为传统DQN。
### 3. 优先经验回放 (Prioritized Replay)：智能地利用经验 `[12:30-14:30]`

**核心观点**

传统的经验回放均匀地采样数据，效率不高。优先经验回放根据TD误差的大小来优先采样那些“更值得学习”的经验，从而加速学习过程。

**深度阐述**

在DQN中，经验回放（Experience Replay）是一个关键机制，它将智能体与环境交互产生的经验`(s`*`t, a`*`t, r`*`t, s`*`{t+1})`存储在一个缓冲区（replay buffer）中，然后从中随机均匀地采样批次数据来训练Q网络。然而，李弘毅教授指出，这种**均匀采样不一定是最佳策略**。

教授解释道，有些经验可能比其他经验“更重要”或“更难学习”。这些“重要”的经验通常表现为**TD误差（Temporal Difference error）较大**。TD误差是Q网络当前输出值与目标值之间的差距。TD误差大的经验意味着Q网络对这些经验的预测与实际目标相去甚远，表明网络在这些方面学习得不好，或者这些经验包含了新的、有价值的信息。

Prioritized Replay的核心思想是：**给TD误差大的经验更高的采样概率**。这样，在训练过程中，那些网络预测不准确、学习效果不佳的经验会被更频繁地采样到，从而促使网络更快地纠正错误，提高学习效率。教授强调，这个想法“非常直觉”。

**重要原话**：

- "也许有一些 data 比较重要呢" `[12:50]`
- "你的 TD error 特别大，那这些 data 代表说你在 train network 的时候，你是比较 train 不好的。那既然比较 train 不好，那你就应该给它比较大的机率被 sample 到。" `[13:10]`
教授也提醒，Prioritized Replay不仅仅是改变了数据的采样分布。由于采样分布的改变，为了保证训练的无偏性，还需要对更新参数的方法进行相应的调整，例如引入**重要性采样（Importance Sampling）权重**。这部分细节教授留给了助教讲解，但强调了其复杂性。

**延伸思考**

优先经验回放的理念与人类学习中的“错题本”非常相似。我们倾向于反复练习那些容易出错的题目，而不是已经掌握的知识点。这种“有针对性”的学习策略，在机器学习中同样能带来显著的效率提升。

**精华收获**

- 均匀采样经验回放效率不高，未能充分利用经验的价值。
- 优先经验回放根据TD误差大小来优先采样经验，TD误差大的经验被赋予更高的采样概率。
- 这种方法能加速学习，但需要调整参数更新方法以保持无偏性。
### 4. 平衡MC与TD (Multi-step TD)：融合两种学习范式 `[14:30-17:30]`

**核心观点**

MC（蒙特卡洛）方法方差大但偏差小，TD（时序差分）方法方差小但偏差大。Multi-step TD通过结合N步真实回报和Q值估计，在两者之间取得平衡。

**深度阐述**

李弘毅教授回顾了蒙特卡洛（MC）方法和时序差分（TD）方法各自的优缺点。MC方法通过完成整个回合来计算真实回报，因此偏差较小，但由于回合长度不确定，方差较大。TD方法则只依赖一步回报和下一个状态的Q值估计，方差较小，但由于Q值本身是估计值，因此存在偏差。

Multi-step TD（N步TD）旨在**结合两者的优点，在偏差和方差之间取得平衡**。其核心思想是：不只存储一步的经验，而是存储N步的经验。具体来说，我们记录从状态`s`*`t`**采取动作**`a`*`t`开始，连续N步的奖励`r`*`t, r`*`{t+1}, ..., r`*`{t+N-1}`**，以及N步后的状态**`s`*`{t+N}`。

在更新Q值时，目标值不再仅仅是`r`*`t + γ  Q(s{t+1}, a')`**，而是：*

`Target Value = r`*`t + γ  r{t+1} + ... + γ^(N-1) `*` r`*`{t+N-1} + γ^N  maxa Q(s{t+N}, a)`*

这意味着，我们累积了N步的真实回报，然后才使用Q网络来估计N步之后的状态`s`*`{t+N}`**的Q值。*

**重要原话**：

- "它就是 MC 跟 TD 的结合，因为它就有 MC 的好处跟坏处，也有 TD 的好处跟坏处" `[16:00]`
- "你可以去调这个 N 的值，去在 variance 跟不精确的 Q 之间呢，取得一个平衡" `[17:00]`
这种方法的优点在于：

1. **降低Q值估计的偏差**：由于使用了更多的真实回报，Q网络估计部分的影响相对减弱，使得目标值更接近真实值。
1. **可调的平衡**：N是一个超参数。当N=1时，它退化为标准的1步TD。当N趋近于无穷大（或等于回合长度）时，它接近MC方法。通过调整N，我们可以在方差和偏差之间找到一个最佳平衡点，以适应不同的环境和任务。
缺点是，累积更多项真实回报会**增加方差**，因为每一项回报都带有随机性。

**延伸思考**

N步TD是强化学习中一个非常灵活且强大的工具，它允许我们根据任务的特点来调整学习的“深度”。在实际应用中，N的选择往往需要通过实验来确定，因为它直接影响到学习的稳定性和收敛速度。

**精华收获**

- Multi-step TD结合了MC和TD的优点，通过累积N步真实回报来计算目标值。
- 它降低了Q值估计的偏差，但可能增加方差。
- N是一个可调的超参数，用于在偏差和方差之间取得平衡。
### 5. 噪声网络 (Noisy Net)：系统性探索的利器 `[17:30-22:50]`

**核心观点**

传统的Epsilon-Greedy探索在动作空间中加噪声，导致策略不一致。Noisy Net通过在网络参数空间中加噪声，实现状态依赖的系统性探索，更符合真实世界策略的运作方式。

**深度阐述**

探索（Exploration）是强化学习中的一个核心问题。传统的Epsilon-Greedy策略通过以`ε`的概率随机选择动作，以`1-ε`的概率选择Q值最大的动作，从而在探索和利用之间进行权衡。然而，李弘毅教授指出，Epsilon-Greedy存在一个根本性的问题：它在**动作空间中加噪声**。这意味着，即使在同一个状态下，智能体也可能因为随机性而采取不同的动作。教授强调：“实际上你的policy并不是这样运作的啊，在一个真实世界的policy，给同样的state，他应该会有同样的回应。” `[20:50]` 这种不一致性使得探索不够系统化，也与真实世界的策略行为不符。

Noisy Net提供了一种更优雅的探索方式：它在**网络参数空间中加噪声**。具体做法是：

1. 在每个回合（episode）开始时，将Q网络的所有参数（权重）加上一个高斯噪声。
1. 得到一个“噪声Q函数”（Noisy Q-function），记作`Q`*`tilde`**。*
1. 在整个回合中，智能体都使用这个固定的`Q`*`tilde`**与环境进行交互，直到回合结束。*
1. 在下一个回合开始时，重新采样新的噪声，生成一个新的`Q`*`tilde`**。*
**重要原话**：

- "Epsilon Greedy 这样的 exploration 它是 在 action 的 space 上面加 noise" `[17:50]`
- "Noisy Net 它是在参数的 space 上面加 noise" `[18:00]`
- "在同一个 episode 里面，你的参数就是固定的" `[19:50]`
- "这是导致了为什么在 network 这个方法 Noisy Net 跟原来的 Epsilon Greedy 或是其他在 action 做 sample 方法本质上的差异" `[20:20]`
- "这个叫做 state dependent exploration" `[22:00]`
Noisy Net的关键在于**噪声在整个回合中是固定的**。这意味着，在同一个回合内，当智能体看到相同的状态时，它总是会采取相同的动作。这种探索方式被称为**状态依赖探索（state dependent exploration）**。它使得智能体能够以一种**系统化、一致性**的方式探索环境。例如，在一个回合中，智能体可能总是尝试向左走；在下一个回合中，它可能总是尝试向右走。这种有目的的探索比随机乱试更有效率。

教授还提到，Noisy Net是由OpenAI和DeepMind在ICLR 2018上同时提出的，只是它们在生成噪声的方法上略有不同（OpenAI直接加高斯噪声，DeepMind的噪声由一组可学习的参数控制）。

**延伸思考**

Noisy Net的探索方式更接近于人类在学习新技能时的策略。我们不会每次都随机尝试，而是在一个阶段内尝试一种特定的策略，观察其效果，然后在下一个阶段调整策略。这种系统性探索对于复杂环境中的长期规划和策略学习具有重要意义。

**精华收获**

- Epsilon-Greedy在动作空间加噪声，导致策略不一致，探索效率低。
- Noisy Net在网络参数空间加噪声，且噪声在回合内固定，实现状态依赖探索。
- 这种系统性探索方式更高效、更符合真实世界策略，有助于智能体更好地理解环境。
### 6. 分布Q函数 (Distributional Q-function)：捕捉回报的完整图景 `[22:50-28:00]`

**核心观点**

传统的Q函数只估计累积回报的期望值，忽略了回报的分布信息。Distributional Q-function通过建模整个回报分布，能够提供更丰富的决策依据，例如风险规避。

**深度阐述**

李弘毅教授提出了一个深刻的观点：传统的Q函数`Q(s,a)`代表的是累积回报的**期望值（expectation/mean）**。然而，在随机环境中，即使在同一个状态下采取同一个动作，实际得到的累积回报也可能是一个**分布**，而不是一个单一的确定值。不同的回报分布可能拥有相同的期望值，但它们背后的风险和不确定性却大相径庭。如果只关注期望值，就会丢失这些重要的分布信息。

**重要原话**：

- "Q function 是 accumulated reward 的期望值" `[23:20]`
- "不同的 distribution，他们其实可以有同样的 mean" `[24:20]`
- "假设我们今天只用一个 expected 的 Q value 来代表整个 reward 的话，其实可能有一些 information 是 loss 的" `[24:40]`
Distributional Q-function的核心思想是：**直接输出累积回报的整个分布，而不是仅仅输出其期望值**。由于直接输出一个连续的分布比较困难，实际做法是将回报空间（例如从-10到10）划分为一系列离散的“bin”（区间），然后让Q网络输出在每个bin中获得回报的**概率**。

教授通过一个图示来解释：如果回报空间被划分为5个bin，那么对于每个动作，Q网络将输出一个包含5个概率值的向量，这些概率值的和为1。这些概率值代表了在给定状态下采取某个动作后，累积回报落在每个bin中的可能性。

在测试时，虽然我们仍然需要选择一个动作来执行，通常会选择**期望值最大的动作**。然而，如果能够建模回报分布，未来就可以有更灵活的运用。例如，我们可以根据分布的方差来评估风险。如果一个动作的期望回报很高，但方差也很大，说明它可能伴随着高风险。有了分布信息，智能体就可以被训练成**风险规避型（risk-averse）**的，在两个期望值相近的动作中，选择风险（方差）较小的那个。

教授也坦言，尽管Distributional Q-function“蛮有道理的”，但它在实践中“没有红起来”，可能的原因是**实现起来比较复杂**。

**延伸思考**

Distributional Q-function为强化学习引入了风险管理的概念，这在金融、医疗等高风险决策领域具有巨大潜力。它将Q-learning从一个纯粹的“期望最大化”框架，扩展到了一个更接近人类决策的“风险-回报权衡”框架。

**精华收获**

- 传统Q函数只估计回报期望值，忽略了分布信息。
- Distributional Q-function通过输出回报的概率分布（通常通过分bin），提供更全面的回报信息。
- 能够支持更复杂的决策策略，如风险规避，但实现相对复杂。
### 7. 彩虹DQN (Rainbow DQN)：集大成者的性能飞跃 `[28:00-34:40]`

**核心观点**

Rainbow DQN将上述所有先进的Q-learning技巧（Double DQN, Dueling DQN, Prioritized Replay, Multi-step TD, Noisy Net, Distributional Q-function）整合在一起，实现了显著的性能提升，并揭示了这些技巧之间的相互作用。

**深度阐述**

李弘毅教授在课程的最后，介绍了“Rainbow”技术。顾名思义，Rainbow DQN就是将前面介绍的所有Q-learning高级技巧——Double DQN、Dueling DQN、Prioritized Replay、Multi-step TD、Noisy Net和Distributional Q-function——**全部整合到一起**。教授幽默地指出，虽然名字叫“彩虹”，但数了一下只有六种方法，可能把原始的DQN也算作一种。

教授展示了Rainbow论文中的性能对比图。图表的横轴是训练过程（时间），纵轴是智能体在十几个Atari游戏上的平均中位数分数。选择中位数而非平均值是为了避免少数高分游戏主导结果。

- **灰色线**代表标准的DQN，性能一般。
- **单色线**代表只使用某一种技巧。例如，紫色的Double DQN效果显著，Prioritized DQN、Dueling DQN和Distributional DQN也都表现出强大的性能。A3C（Actor-Critic方法，将在后续课程讲解）也比DQN强。
- **彩虹线**（所有技巧结合）的性能远超任何单一技巧，达到了当时的SOTA（State-of-the-Art）。
**重要原话**：

- "把刚才所有的方法都综合起来就变成 rainbow 啊" `[28:30]`
- "最后其实这些方法他们本身之间是沒有冲突的，所以全部都用上去，就变成七彩的一个方法，就叫做 rainbow，然后它很高这样" `[31:30]`
更重要的是，教授展示了Rainbow论文中的**消融研究（Ablation Study）**结果。这张图显示了从完整的Rainbow模型中**移除某一个技巧后性能的下降程度**，以此评估每个技巧的贡献。

- **Multi-step TD**：移除后性能下降最剧烈，表明其贡献巨大。
- **Prioritized Replay**：移除后性能也大幅下降，显示其重要性。
- **Distributional DQN**：移除后性能最终收敛到较低水平，尽管训练初期速度可能不受影响。
- **Noisy Net**和**Dueling DQN**：移除后性能也有所下降，但不如前三者显著。
- **Double DQN**：令人惊讶的是，从Rainbow中移除Double DQN后，性能几乎没有变化。
教授对Double DQN在Rainbow中作用不大的现象给出了一个合理的解释：**当使用了Distributional DQN时，Q值本质上就不会被高估，甚至可能出现低估**。Double DQN的主要目的是解决Q值高估问题。而Distributional DQN通过将回报限制在一个预设的范围（例如-10到10）内，如果实际回报超出这个范围（例如100），它会被“截断”，从而导致对极端高回报的低估。因此，在Distributional DQN的存在下，Q值高估问题已经得到缓解（甚至逆转），Double DQN的特定作用就变得冗余了。

**延伸思考**

Rainbow DQN的成功证明了“组合创新”在深度强化学习中的巨大潜力。它不仅仅是简单地堆叠技巧，更重要的是揭示了不同技巧之间的协同作用和潜在的冗余。这种消融研究对于理解复杂系统中的组件贡献至关重要。

**精华收获**

- Rainbow DQN通过整合Double DQN、Dueling DQN、Prioritized Replay、Multi-step TD、Noisy Net和Distributional Q-function，实现了显著的性能提升。
- 消融研究表明，Multi-step TD、Prioritized Replay和Distributional DQN对Rainbow的性能贡献最大。
- Distributional DQN的存在使得Double DQN在Rainbow中的作用变得不明显，因为它本身就能缓解Q值高估问题。
