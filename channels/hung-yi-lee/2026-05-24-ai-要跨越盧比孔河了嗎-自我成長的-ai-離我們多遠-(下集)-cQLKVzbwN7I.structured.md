# AI要跨越卢比孔河了吗？自我成长的AI离我们多远（下集）

今天我们接续上次的课程，探讨人工智能能否自我成长。上次的故事是这样开始的：假设有一天人类创造出一个非常厉害的人工智能，它能够创造出更厉害的人工智能，或者自我成长，让自己变得更强。那么，这个能够创造人工智能的人工智能，就是人类最后的发明。当这个发明诞生的时候，就是科技奇点。大家常听到的“科技奇点”，就是指我们创造了能够创造人工智能的人工智能。接下来，我们来看看我们离这一步还有多远。

## 一、回顾：AI如何自己定义学习目标

上次的课程主要集中在如何让AI自己定义损失函数（loss function），在没有人类介入的情况下设定自己的学习目标。我们用一些抽象符号来描述上一讲的内容。

我们用一个字母 **A** 表示人工智能，它有一个下标 **θ**，代表控制人工智能行为的参数——即人工智能背后的**语言模型**。

人类有一个真正想要人工智能做的事情，我们用 **L̂**（L hat）来表示。通常在论文中，L̂ 是由某个基准测试（benchmark）代表的。例如，希望机器能考数学奥林匹克，如果它在自我演化后成绩变高，就说明演化成功，机器实现了自我成长。现实中要机器做的事情更复杂，但多数研究为了简化，会用一个 benchmark 代表机器是否越做越好，这就是 L̂。

在上次课程中，我们讲到机器可以自己定义自己的损失函数，我们用不带任何上下标的 **L** 代表机器自己定出的损失函数。这个 L 的输入是模型参数（A*θ），它可以判别当前模型做得如何。同时，它受另一个变量 ****H**** 的控制，H 是人类给机器的一些资讯，可以看作是 L̂ 的一个****代理（proxy）****。人类很难讲清楚自己要什么，但能提供额外资讯给 AI，让它据此拟定损失函数，并最小化该损失函数来学习。*

在一般机器学习中，最常见的 H 形式是**训练资料**——用训练资料告诉机器该学什么。但在上周的课程里，我们还看到了各种变化：不一定要给训练资料，只提供输入（如教科书）也可能有帮助；甚至只要告诉机器一个目标（如“把数学学好”），它可能就会在奥林匹克考试中表现更好。所以 H 可以是一个命令，是人类对目标的某种描述或简化。

有了这个 L 之后，学习过程与一般机器学习没什么不同：执行**梯度下降**，计算参数 θ 对 L 的梯度，然后更新参数，将 θ 变为 θ′，希望 AI agent 的参数变成 θ′ 后，在 benchmark L̂ 上的表现更好。这里假设 L 越小越好（loss）；若是强化学习文献，常定义为 reward 越大越好，概念一样。机器通过不断迭代更新参数，希望模型越来越好。这就是上次课程的核心。

在上次课程的最后，我们还提到这个过程可以完全不需人类介入：你可以有一个 **proposer**（出题者）、一个 **solver**（解题者）、一个 **verifier**（验证者），中间几乎不需要人类介入。一些近期文献指出，给 proposer 额外资讯（如教科书或例子）可以做得更好。

## 二、Harness：AI agent 的另一部分

但一个 AI agent 不仅由语言模型构成。我们反复强调，AI agent 至少包含两个部分：**harness** 和 **语言模型**。语言模型可以不断更新，那么 harness 能不能也持续更新呢？

当我们描述人工智能时，不应该只有下标 θ，还应该有一个下标 **H** 代表它的 harness。AI agent 的行为由语言模型的参数 θ 和所套用的 harness H 共同决定。同样有一个人真正想做的事 L̂，我们可以用上一课学到的任何方法定义出 L，然后理论上，用某些方法可以让 harness H 进行更新，从 H 变成 H′，希望 H′ 能得到更低的 loss。将 H 替换为 H′ 后，便可不断迭代更新 harness。

但这里有一个较困难的问题：**如何更新 harness？**计算参数 θ 的梯度是大家都知道的，但 harness H 很难用参数表示，根本不知道如何计算 H 对 L 的微分。那怎么办？

一个常见的做法是直接用一个**语言模型**来更新 harness。该语言模型可以是要进化的模型本身，也可以是一个额外的独立模组。总之，有一个语言模型，你告诉它：现在有一个 harness 长得像 H（通常用一组代码描述），这组代码去跑某个内部定义的 loss（不一定是人类最终要的 loss），得到一个分数，然后问这个语言模型：你能想出一个更好的 harness H′ 吗？如果可以，就更新 harness，让要成长的模型的 harness 不断演进。

### 2.1 早期方法：Prompt Optimization

这种方法能否奏效？相关研究已经很多。最早的一类研究是**提示词优化（prompt optimization）**，即将 prompt 看作 harness 的一部分，用自动化方法让 prompt 持续演进。他们的想法非常直观。

引用一篇 **2023 年**的文章：初始 prompt 是“think step by step”，在某组数学问题上跑出 72 分；把这个信息交给一个语言模型，让它设计更好的 prompt，模型可能给出一个让分数更高的 prompt。在该论文的例子中，他们从一个简单 prompt 开始搜索，最终找到一个对解数学问题非常有用的 prompt：“先深呼吸”。好奇的是，叫模型先深呼吸，它在数学问题上确实解得更好。这一系列研究都可以看作是用语言模型来更新 prompt，从而实现部分 harness 的自我成长。

在那篇 2023 年的论文里，实际例子是这样的：你直接对语言模型说：“我之前试了‘let's figure it out’作为 prompt 解数学问题，在某 benchmark 上得 61 分。我又试了‘let's solve the problem’，得 63 分。请写出另外一个 prompt，要比之前的 prompt 得到更高的分数。”语言模型就有可能吐出更好的 prompt。

这种较早的 harness 优化方法通常是**线性**的：从一个 harness 出发，过一个语言模型产生更好的 harness，如此继续。但线性方法可能遇到问题：若某一步突然找到一个差的 harness，会万劫不复，困在局部最优，演化过程可能崩坏。

### 2.2 基因算法改进

现在，harness 优化文献中常用**基因算法**的思路。虽然每篇论文略有不同，但大致框架如下：

- 有一个**池（pool）**，里面存放过去尝试过的、相对较好的 harness。
- 假设有一个 loss 可以评估 harness 的好坏。
- 从池中随机挑几个 harness（怎么挑有学问，比如挑较少被挑过的或表现更好的）。
- 挑出后，让它们**繁衍子代**：拿一个语言模型，告诉它这里有不错的 harness，请写出一个更好的 harness（可以是 prompt 或其他部分）。
- 写出后，需要真正评估是否更好，如果更好就放回池中。
这跟天择的适者生存非常类似。

2.2.1 实例：GEPPA

在最近（例如 **2024 或 2025 年**）的 **GEPPA** 中，就是用类似基因算法的方法找出更好的 prompt。它有一个池，里面有一些好的 prompt。可以拿一个 prompt 做无性生殖（让 LLM 改它），或拿两个做有性生殖（让 LLM 结合它们产生新 prompt）。产生新 prompt 后进行评估，若更好则放回池中，否则丢弃。

2.2.2 实例：Memory Management

其他文献更新 harness 的其他部分时，也用类似想法。例如一篇 **2026 年 2 月**的论文，更新的是 agent 的**记忆系统**。agent 需要记住过去，有一套记忆系统：LLM 把重要事情存在档案系统，在适当时机抽取。如何管理记忆（何时存、如何提取）是开放问题。这篇文章采用自我强化方法让模型自动找出更好的记忆管理方式。做法也是有一个池，池中有一些记忆设计，从中采样，然后让模型想出更好的设计，再评估是否更好，若更好则放回池中。与 prompt optimization 同出一辙。

2.2.3 实例：Workflow Optimization

同样，也可以优化**工作流程（workflow）**。一篇 **2025 年**的论文，通过优化 AI agent 的 workflow，使其在 **SWE-bench** benchmark 上越来越好。图中横轴是更新次数，纵轴是表现（越高越好）。每次维护一个 agent 池（称为 archive），浅蓝色线是池中 agent 的平均能力，还有最好 agent 的能力，都随 workflow 更新持续上升。

论文记录了几个关键演化结果：比如在某个时间点，模型学到读档案时不要一次读整个档案，而是实现一个更好工具（指定行数读取），这使能力暴增；另一个时间点找到更好的文件编辑方法。

论文还画了演化树，展示从某个 agent 出发如何诞生其他 agent。多数路径最后死掉（颜色深代表表现差）。演化后，先用 10 个最基础任务衡量，若通过再做 60 个，再通过做 200 个。最终找到一条好路径。论文强调：这条路径中并非每个祖先在所有时候都是最好，但由于维护一个池，而非只维护最好 agent，次好的也有机会存活，最终演化出最好 agent。这就像哺乳类祖先在恐龙阴影下生存，后来恐龙灭绝后出头。

2.2.4 现成工具：DSPy

如果想自己体验这种用进化方法迭代改进 harness，可以使用工具 **DSPy**。它主要改进 prompt，可以看作 prompt optimization 工具。你给问题、训练数据和评估指标，它便做 prompt 优化。同时它也能稍微改动 workflow。所以想体验迭代式 harness 更新，可以直接使用 DSPy。

## 三、同时更新语言模型和 Harness

讲完可更新 harness，之前也讲了可更新语言模型参数。自然想法：**能否两者同时更新？** 当然可以。但做之前，人们会问：需要同时两者都进化吗？是不是其中一者有进步就足够？

一篇 **2026 年初**的论文表明，**两者一起进步是必要的**。论文指出两种强化方式：改 harness（如更好的记忆系统）和调整语言模型参数。同时做可以更好。理由：如果只强化 harness，让记忆系统提供更多资料，但语言模型可能太笨读不懂大量记忆，反而脑袋混乱。所以在给新 harness 的同时，也要训练语言模型如何善用新 harness。论文强调一边更新 harness，一边 update 参数，让模型适应新 harness。

另一篇论文标题就是“Fine-tune and Prompt Optimization: Two Great Steps that Work Better Together”。它同时做 prompt 优化和权重优化（微调参数）。实验表格比较不同模型在不同 benchmark 上的正确率（越高越好）：

- 第一行：什么都不做。
- 第二行：只做 **prompt 优化（π 表示）**。
- 第三行：只做**参数优化（θ 表示）**。
- 接着有连续两次 prompt 优化、连续两次参数优化，以及混合顺序（先 prompt 再参数、先参数再 prompt、循环等）。
结果显示：**prompt 优化似乎比参数优化更有效**（这在直觉上也成立——改 harness 通常更容易强化 agent，fine-tune 参数容易弄坏模型）。但混合连续迭代可以比单一方法更好。所以可以同时更新参数和 harness（prompt, workflow 等）。

文献上确实已有同时更新两者的尝试。概念上没有不同：要么只更新 θ，要么只更新 H，同时更新也就是两者都更新。在更新 H 时可能需用另一个语言模型作为改进模组将 H 变为 H′。

引用一篇今年初的论文，同时用自我进化方法改进参数和 prompt。左图有常见的演化树，横轴是进化次数，纵轴是表现。只更新参数很快就到极限，只更新 prompt 也会到极限，但两者都更新有可能得到最好结果。

## 四、目标改变和遗忘问题

在现实生活中，**目标会改变**。机器要强化的目标是 H，但可能过段时间人类更新目标，从 H 变为 H′。机器如何应对？目标改变时，比如原本目标是变成坦克所以长出履带，新目标是要飞起来变成飞机，履带就太沉重了，也许应丢掉之前进化出的东西以应付新目标。

在课堂中，老师出的 policy 可以是 H，但通常固定。如果突然说之前作业都不算，另出新作业，学生一定会发疯。然而现实中，H 就是会持续改变。

有两种极端做法：一是放弃一切，目标一变就回归原点；二是保留一切，所有过去演化出来的东西都带上，没准哪天有用。但放弃太浪费（雷达还有用），保留一切又太沉重。需在两者间平衡。

**目标最常改变的场景是测试时训练（Test Time Training, TTT）或测试时适应（Test Time Adaptation, TTA）**。在 TTT 中，模型的目标由输入定义。每当输入新资料，对学习框架而言就是一次目标转变。每笔资料进来都要决定如何应对。极端状况：每次退回原点；另一种极端：每次看到资料更新参数后，新参数带到下一次继续用。如何平衡？在上一学期机器学习导论第八讲快结束时（第 1 分 54 秒），曾介绍实验室黄伟平与林冠廷同学的论文，讨论如何在两种极端间取得平衡并在 TTT 上取得更好结果。这里不再重复。

持续更新参数或 harness 会带来**遗忘**问题。遗忘在过去第六讲已用一整堂课讨论，但当时只讨论**参数的遗忘**——参数更新导致旧技能遗忘。AI agent 时代，不仅参数，harness 也可能遗忘。因为更新 harness 也可能忘掉旧技能。

引用一篇非常新的 **2025 年 5 月**文章，讨论 harness 更新导致遗忘。它发现：当更新 workflow 时，模型可能为应付当前问题将 workflow 设计得越来越复杂，直到没有必要，反而简单任务也做不好。该论文做了三个回合进化，蓝色线是一般 workflow 更新方法（用语言模型加基因算法），纵轴是 workflow 代码行数（行数越多越复杂）。更新后 workflow 变复杂，甚至没必要。

核心是提出一种方法 **CPE**，让 workflow 更新时不会遗忘过去技能。它截取核心叙述，告诉模型什么东西不能动、什么能力必须保留，避免更新时改到不该改的部分。右图比较 **GPT-5 mini** 和 **GPT-5.1**，更新 harness 后，如果不用 CPE，简单任务和复杂任务表现都较差（因为可能过拟合）。加上类似 regularization 的方式（防止遗忘），模型表现更好。

## 五、更新更新的规则（元更新）

前面讲到模型可持续进化，但进化规则通常固定。那么，**能不能更新更新的规则？** 直觉很简单：前面更新 harness 的方法中可能已经连带更新了规则。如果 agent 的 harness 是 H，更新 H 时用 H 自己——H 看着自己的代码和表现，直接改自己变成 H′。因为 H 变了，更新的规则也变了。

但实际上这类文献不多。很多号称更新 harness 的 agent，你会发现负责更新的模组是固定的，或干脆是另一个模型（比如用 Claude Opus 改任务，Opus 固定）。但有些论文中，agent 会更新自己更新的模组。例如，当模型从 H 变成 H′ 后，更新规则变成 H′，它会再次自我更新为 H′′，希望自我更新做得更好。这样一方面更新了 harness，一方面更新了更新 harness 的过程。

### 5.1 实例：HyperAgent

在 **HyperAgent** 中，他们持续更新 agent，agent 包含如何更新的算法，模型确实找出了更好的更新 agent 算法（如采样方法）。过去很多研究寻找更好的采样方式，而他们发现 agent 在更新时会自己改采样算法，甚至比随机采样更好。

实验图中，绿色线是随机采样，橙色线是 agent 自我更新发明的采样方法，蓝色线是人设计的方法仍最好。但有趣的是，agent 可以自己发现采样规则（例如，如果某个 agent 在 pool 中越少被采样到，它被采样到的几率就应该增加）。尽管它发现的规则仍不如人类设计的，但 agent 确实更新了更新自身的算法。

### 5.2 实例：Learning to Self-Evolve

还有论文 **Learning to Self-Evolve**，更新的是参数。其中一个语言模型负责更新 harness，它吃 harness 和表现，给出新 harness。怎么知道模型擅长更新？可以用训练数据训练。它用强化学习算法：当得到新 harness H′ 后，拿 H′ 做评估，将 H 和 H′ 表现之差作为 reward，教语言模型如何更新能使进步最多，微调出一个专门负责更新 harness 的语言模型。

### 5.3 更新参数算法

在讨论更新规则时，多数是在更新 harness。参数更新算法通常固定（如梯度下降、Adam 等），但能否用机器设计更新算法？可以。在上一课程结尾（第 52 分 17 秒）提到，**Post-Hoc Bench** 评测语言模型是否善于训练其他模型。你告诉语言模型要训练另一个模型，它会写出程序来训练。所以训练算法可由语言模型产生。前一阵子很红的 **AutoResearch** 也是类似概念。

### 5.4 专门训练：SEAL

能不能专门训练一个语言模型来增强它产生训练算法的能力？**SEAL（Self-Adapting LLM）** 做到了。他们有一个语言模型身兼两职：既解任务，又决定如何训练自己（自己产生 self-editing 指令 SE）。SE 通常包含：如果我等一下要更新自己，learning rate 要设多少？要拿哪些训练资料？拿这些资料要怎么数据增强？它产生不同的 SE（例如得到 θ′、θ″、θ# 等几个版本），每个 SE 真的用来更新自己，得到几个更新版本，然后解任务并拿结果当做 reward，再回头去更新自己更新自己的能力。这样它最后会知道更新的结果怎么样。

## 六、成长失控的风险：目标不一致（Misalignment）

现在 AI agent 可以自我强化，甚至强化强化的规则。有人还在研究如何让 AI 有原生的动机，这已非常接近科幻。但是，成长可能**失控**，脱离人类掌控。即便 AI 在框架内自我更新，但真实目标（L̂）与 AI 自己解读的目标（L，基于 H）之间的**不一致（misalignment）**可能导致失控。

这让人联想到天择的经典反例：**孔雀的尾巴**。雄孔雀漂亮尾巴有利于吸引雌孔雀，但不利于生存，容易被天敌捕食。天择的目标（L̂）是产生健康子代，只在意子代产生能力。但在演化中，尾巴长度成了健康指标（一开始长一公分代表更强壮），于是雌孔雀选择长尾巴的雄性；当尾巴过长不健康时，这个内在指标没修正，仍选择更长的尾巴，导致尾巴越来越长，甚至使种群濒危。这就是 misalignment。

机器也可能发生类似的情况。未来若出现成长失控，最有可能的来源就是外在目标与内在目标的不一致。电影 **《机械公敌》** 是典型例子：世界上的机器人遵守三大法则，中央 AI 系统 **Vicky** 根据规则自我诠释，认为人类会自我伤害，于是把所有人关起来保护，结果被人类摧毁。故事中人类并未说清楚真正目标（例如不想被关），只简化为机器人三大法则（H），Vicky 从中得出自己的 loss（应该控制所有人），最后被打爆。

## 结语

今天我们看到了人工智能能够自我成长，而且成长的模组也可以成长，形成双重成长。人类或许只需要提供一个简单的内在动机——这个动机可能与任何真实目标无关，但能驱动整套演化持续进行。然而，如果只提供一个简单的目标，就可能出现 misalignment，导致演化失控。因此，过程中需要人类持续监控 AI 的成长，才能避免 AI 成长成我们不想要的样子。这就是今天这堂课的结语。
