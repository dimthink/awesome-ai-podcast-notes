## 开篇引入

大型语言模型（LLM）无疑是当今人工智能领域最引人注目的技术之一。它们能够生成流畅自然的文本，完成翻译、摘要、问答等一系列复杂任务，仿佛拥有了人类的智慧。然而，对于大多数人来说，LLM 内部的运作机制仍然是一个“黑箱”。它们是如何将输入的文字转化为意义，又是如何预测下一个最可能出现的词汇？李宏毅教授的这堂课，正是一次对这个神秘“黑箱”的深度解剖。他将带领我们剥开层层代码与数学的表象，直抵模型的核心，揭示其从输入到输出的每一个关键环节，并辅以生动的实作演示，让抽象的理论变得触手可及。这不仅仅是一次知识的传递，更是一场对 LLM 内部“思考”过程的窥探，一次理解人工智能如何“理解”世界的奇妙旅程。

## 详细内容

### 语言模型的核心运作流程：从输入到概率分布 `[00:00-00:20:00]`

**核心观点**

语言模型本质上是一个函数 F，它接收一个未完成的句子 X 作为输入，通过一系列复杂的内部运算，最终输出一个关于下一个可能出现的 Token 的概率分布 F(X)。本讲将聚焦于函数 F 内部的运作机制，而非模型的训练过程。

**深度阐述**

李宏毅教授开宗明义地指出，语言模型的核心功能在于“续写”句子。给定一个上下文（未完成的句子 X），模型的目标是预测接下来最有可能出现的 Token 及其概率。他强调，本课程的重点是解剖一个**已经训练好的模型**，观察其内部参数如何与输入句子互动，从而产生下一个 Token 的概率，而非探讨模型是如何被训练成现在这个样子的，这部分内容将留待后续课程。为了让听众对理论有更深刻的理解，课程的后半段还将通过实作来验证所讲解的原理。

### 语言模型中间层输出的解读：Representation 的含义与操控 `[00:30:00-00:54:00]`

**核心观点**

### Transformer 层内部运作：Self-Attention 与 Feed-Forward `[00:54:00-01:14:00]`

**核心观点**

Transformer 的每一层主要由 Self-Attention 和 Feed-Forward 两个子层构成。Self-Attention 负责捕捉输入序列中 Token 之间的相互依赖关系，从而融合上下文信息；Feed-Forward 层则对每个 Token 的 Representation 进行独立的非线性变换。

**延伸思考**

这堂课不仅揭示了 LLM 的内部机制，也引发了对未来 AI 发展的深层思考。Representation Engineering 的能力，意味着我们不仅能理解模型，还能在一定程度上“编程”其行为，这既带来了巨大的应用潜力（如定制化模型行为、去除偏见），也带来了伦理挑战（如恶意操控）。Logit Lens 和 Patch Scope 等技术，为我们提供了一扇窥探 AI“心智”的窗户，有助于我们更好地理解模型的决策过程，提升其可解释性。同时，对 Attention 机制计算成本的讨论，也预示着未来模型架构创新（如 Mamba）的方向，即如何在保持性能的同时，提高效率和处理长文本的能力。

**精华收获**

- **LLM 核心流程**：从 Tokenization 到 Embedding，再经多层 Transformer 处理，最终通过 LM Head 和 Softmax 输出概率分布。
- **Embedding 的语义力量**：Token Embedding 捕捉词汇语义，Contextualized Embedding 融合上下文，实现词义消歧。
- **模型内部的“思考”**：Logit Lens 和 Patch Scope 等技术让我们能观察模型在不同层级对信息的理解和推理过程。
- **可操控的 AI 行为**：Representation Engineering 证明通过修改中间层 Representation 可以改变模型的输出行为，具有巨大的潜力和风险。
- **Transformer 的基石**：Self-Attention 机制通过 Query-Key-Value 交互融合上下文，Multi-head Attention 从多角度捕捉关系；Feed-Forward 层进行非线性变换。
- **参数的直观理解**：通过实作，我们可以直接查看模型的参数数量、Embedding Table 结构和 Attention Weight，将抽象理论具象化。
