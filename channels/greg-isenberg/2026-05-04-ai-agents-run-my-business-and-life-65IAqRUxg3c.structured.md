今天的节目将深入揭秘 **Andrew Wilkinson** 如何使用 **AI Agent** 来管理他的个人生活与商业事务。这是一个拥有顶级配置的人。他将向我们精确展示他如何利用 **OpenClaw** 和各类 AI Agent 来提高生产力、赚更多钱；如何在家族办公室中使用它们；如何借助它们启动一家初创公司；以及如何利用它们思考下一代商业机会和资金投向。

这是一个他此前从未公开分享过的内部视角。在本期节目结束时，你的创意之泉必将涌动——我本人就是如此。我迫不及待地想让你也看到这些内容。

## Andrew 的 AI 觉醒时刻：从“尝试”到“上瘾”

>> 好的，我们开始。我要说，和许多其他人一样，我在去年 12 月经历了一个疯狂的时刻。当时我尝试了 **Claude Code** 以及 **Replit** 等所有 AI 工具，它们能做点事，很有趣，但还不够成熟。然而到了 **2025 年 12 月**，我感觉自己就像开始“吸食海洛因”一样。我开始在凌晨三四点醒来，脸上带着大大的笑容翻身下床，然后直接坐在终端里，打开 10 个 Claude Code 的标签页。从那时起，我就一直沉浸其中。

我的目标呢……我在生产力方面尝试了很多不同的事情。比如，在 OpenClaw 出现之前，我就试图自己构建一个类似的 Agent，但做得很差。所以，当我最终能够做到这一点时，我感到非常兴奋。但作为一个商人，最让我兴奋的是——对于那些不了解的听众，我基本上是收购和创办企业。我创办企业多半是出于爱好和乐趣。但我收购了很多企业，并且总是对一家 **SaaS（软件即服务）** 公司中庞大的**行政负担**感到震惊。

想象一下它如何扩展：客服、会计……所有这些非常枯燥、重复的任务，现在都可以实现自动化了。因此，在我们将 AI 应用于运营中的企业之前，作为一个测试，我构建了一个 SaaS 业务，然后使用 **OpenClaw** 完全自主地运营它。这是一个很酷的起点。

我必须强调一下 **OpenClaw** 。刚才我们提到了海洛因——我声明一下，我从没吸过，以防有人担心我。但据说吸海洛因时，第一次的感觉是最好的，他们称之为“追逐那条龙”，你总是试图找回那个特别的时刻。我就有过这样一个特别时刻：当时我去亚利桑那州旅行，忘了带笔记本电脑。但我在自己的电脑上设置好了一个 OpenClaw Agent。结果，我基本上可以在优步的后座上，用 OpenClaw 运营我的整个生意。我当时正要去参加一个会议，超级忙，但我惊讶于它的能力如此之强，而且没有人察觉到我发的每一封邮件其实都是 OpenClaw 写的。

从那以后，我一直在“追逐”那个时刻。问题在于，我感觉自己 50% 的时间花在调试上，30% 的时间花在改进它，只有 20% 的时间是在真正高效工作。这就是典型的“生产力跑步机”困境。不过，我们确实已经构建出了一些非常有趣的东西。

## 从亲密关系到商业机会：构建“Deep Personality”

让我通过屏幕共享来展示一下我构建的这个应用。这一切都始于我想为我和我女朋友的关系构建一个**自定义 GPT** 或 **ChatGPT 项目**。我们的想法是，我们可以查询它，比如：“嘿，我们正在为 XYZ 事情吵架”——我们确实经常吵架。我问它：“如果我要做一堆心理测试，你希望看到哪些？”它给了我一个包含 **15 种不同心理测试**的清单。

于是我去找 Claude Code，说：“嘿，你能把这些都做成一个选择题测试，设置好评分，然后放进一个 JSON 文件里吗？”我们俩都做了这个测试，花了大约 40 分钟。然后我们把 JSON 文件放进 ChatGPT，对它说：“在不了解这对情侣任何信息的情况下，告诉我他们的关系状况。”我们逐行阅读分析结果，下巴都惊掉了——它**完美地指出了我们每一次争吵和家里的每一个问题**。

我看到了这些**多选性格测试**的强大力量。我看到了产品机会，心想：“我要以此为基础建立一个生意。”于是我创办了 **Deep Personality**。让我来共享屏幕。

>> （播客中的简短广告插播：一个关于 AI 时代创业机遇的免费工作坊信息……）

>> 你拥有一家名为 **Metal** 的机构，你本可以叫 Metalab 的人来帮你构建这个，但你却决定自己动手，亲自产品化，亲自设计，亲自构建。你为什么做这个决定？

>> 我认为……我喜欢和人打交道，我是个外向的人，但生意中最糟糕的部分恰恰就是“人”。用一个编剧的比喻来说：在好莱坞，有很多才华横溢的编剧，他们写出很棒的剧本。他们有一个想法，知道想让它走向何方——比如从这里开始，想让它到达那里，拍出一部很棒的故事片。但在剧本和电影之间，横着 100 个需要说服的人、5000 万美元需要筹集、以及一百万个中间人。

我认为**创意总是会因为你的愿景和执行之间的人数而妥协**。对我来说，写作是一种非常纯粹的追求：我可以思考，然后写下来，如果需要可以用 AI 来改进。这感觉就像是你的愿景得到了最清晰的执行。我觉得 **“氛围编程”** （vibe coding）就是这样——我终于可以亲力亲为地完成每一个部分，并且按照自己的节奏来。

如果我用员工，他们不可靠，他们看不到我的愿景，他们不理解设计的每一个方面。你可能有一个很牛的设计师，但他不懂前端；或者他不懂如何设计出在前端能用的东西；或者一个很棒的设计师却不会写文案。而“氛围编程”把所有这一切都结合起来了。基本上，在几天狂热的工作中，我就能构建出这个相当疯狂的应用。

让我展示一下它是如何工作的。基本上，你做一个**40 分钟左右**的性格测试，只是点击多项选择题之类的。然后你会得到一份报告。例如，这是我的“原型”——你会被称为 **“炽热建筑师”** 。这是你的超能力，这是你的致命弱点。这只是基础部分。当你往下滚动，你会看到 AI 已经把你所有的答案写成了一份**长达 100 页的报告**。这份报告会一直持续下去，而且是以 **Robert Greene** 的风格写的——非常有冲击力、深刻、直击内心。它会带你了解你擅长什么、不擅长什么、你应该做什么工作、你的关系是否健康、你是否有多动症、强迫症、是否在谱系上……等等所有这些东西。

你可以看到（这个报告太大了），比如我的人格特质、我的依恋风格、我的内在家庭系统、我有多讨好别人等等。我构建了这个东西，但通常对我来说，事情会这样发展：我开始做一个项目，然后不想收尾，因为我不想雇人来处理它。但在这次，我最终做的就是**构建 Agent 来做所有事情**。

## “Harbor”与自动化运营：支持、营销与开发

我可以向你展示我们是如何使用 OpenClaw 的。我们目前正在使用一个我朋友构建的、叫做 **Harbor** 的工具。它本质上是一个 **Agent 的控制平台**。你可以看到这里运行着不同的 Agent：我们有**开发 Agent**、**营销 Agent**，还有一个**客服 Agent**。

基本上，它的工作方式是：当有人给客服发邮件时，客服 Agent 要么直接处理这个工单，要么如果它能修复问题，就会把它发送给开发 Agent。如果是一个 **P0 级问题**（比如一个超级可怕的安全漏洞），它会立即修复并合并 **PR（拉取请求）**。否则，我们醒来就会看到一堆 PR。然后它会回邮件给那个人说：“嘿，我修复了你的问题。”这基本上完美地工作，令人震惊。对我来说，**客服的工作很快就会被淘汰**。

真正令人兴奋的是**营销 Agent**。我们基本上把它连接到了 **PostHog**，所以它拥有应用的所有数据。它实际上管理着一个 **Meta** 和 **Reddit** 的广告账户。它会做**多变量测试**，创建广告创意，设定预算。我们基本上只需要给它发消息，比如：“嘿，你能把预算增加 1000 美元吗？”或者批准一个大型的 **SEO（搜索引擎优化）** 项目之类的。

我们目前的进展是，已经创造了大约 **2 万美元**的收入，规模还很小。我真正兴奋的下一步是，当我们给它**每月 10 万美元**的广告预算，并找到真正有效的广告和创意时，会发生什么。这太疯狂了。我认为我们大概只需要 3 到 6 个月，就能把生意完全交给 AI 去运营，至少像这样的基本业务是可以的。

>> Harbor 我从未听说过，我见过很多工具。它是我的朋友 **Gavin** 做的。你能把 Harbor 的网站链接给我吗？它让我想起了一点 **Paperclip**，你见过 Paperclip 吗？

>> 是的，我见过。它就是 `github/geekforbrains/harbor`。他是我的一个朋友，我一直在和他一起做这个项目。它运行得非常好。我们发现 **OpenClaw 还不够确定性**。OpenClaw 的另一个问题是，因为它是一个基于文本的界面，所以跟踪所有 Agent 和实际发生的事情有点困难。所以 Harbor 对我们来说效果很好。

>> 是的。我觉得它的酷之处在于，当你创办一家公司，思考组织架构图、角色和职责时，你通常不会在文本的来回交流中思考这些。你会想：“好，我要为 XYZ 任务启动这个 Agent 或员工，然后我该如何监控这个员工在做什么？”

>> 没错。所以它基本上就是 OpenClaw 之类工具的一个**图形用户界面**，但仍然运行在云端。但它拥有所有这些文档（就像一个知识库）、数据库、所有环境变量等等。它和 Paperclip 非常相似。我认为问题在于，这让我有点想起**加密货币**——加密货币是书呆子为书呆子打造的，所以它很书呆子气。

当我看到 Paperclip 时，我非常兴奋，因为我想：“天哪，这个我在视觉上能理解了。” 你有一个组织架构图，但很明显它还是书呆子做的，因为很混乱。我的信念是，很快 **Anthropic** 和 **OpenAI** 就会推出类似 **“CEO”** 的东西。你只需要说：“我要为我的公司雇一个 CEO。给你我的数据，你来搞定，你来运营整个业务。”

现在的问题是，两年前的感觉是，你有一个**患有阿尔茨海默症的天才**——上下文窗口太小了，你只能拥有短暂的瞬间。现在，有了 **100 万 token 的上下文窗口**，它们大概能记住一天，但就像电影《记忆碎片》一样。我觉得一旦上下文窗口达到 500 万到 1000 万，我们就能用这个来运营整个公司了。

>> 现在确实有人在卖“自主运营的公司”，不知道你见过没有？**Pulsia** 就是其中一家。

>> 我觉得 Pulsia……我在《纽约时报》上看到过那个人的故事，他创办了一个自主运营的、价值十亿美元的企业之类的。显然那是真实的，因为《纽约时报》报道了。但我分不清哪些是真实的，哪些是……

>> 我认为，我们录制这期节目是 **2026 年 4 月 29 日**，你现在还**无法运营一家完全自主的公司**，一个你可以 24/7 去海滩躺着享受的公司。话虽如此，我认为生意的某些部分可以是自主的。比如客服就是一个很好的领域。但在我看来，以目前的情况，你还无法将产品这类事情自主外包给 Pulsia 这样的服务。也许 6 个月、12 个月后会有变化。但我的看法是，今天那些销售“自主公司”的人，只是在**兜售一个梦想**。

>> 我有点同意。我认为 OpenClaw Agent 基本上还像是 **Zapier** 那样的 Zap，只不过它们能做基本的决定并拥有智能，但你仍然需要一步一步地告诉它们：“这是我想要你思考的方式，这是我想要你运作的方式。” 假设你雇了一个实习生，给了他们一个邮箱地址，你绝不会需要告诉他们：“你应该每 15 分钟检查一次邮箱，然后回复。回复时，你应该思考以下几点……” 这感觉就像你在**对付一个婴儿，一个天才婴儿**，但你必须教他们做每一件事。我真正兴奋的是，当我能够有一个“CEO 模型”时，我可以直接把任务委派给它，它会自己搞定。我以为 OpenClaw 就是那样的，但它完全不是。

## 产品反馈与真实性挑战

## 数据管理：构建中央化知识库

>> 让我们看看，还有什么可以展示的？另一个大块是**数据**。我感到非常沮丧，因为我在不同时期标准化使用了不同的 LLM。我会进入 Claude 生态系统，使用桌面应用；我也会用 ChatGPT，并构建一堆项目。我基本上已经接受了我会一直使用 Claude Code，直到有人构建出更好的东西（我相信未来几年内就会出现）。

所以我的策略基本上是把我所有的不同数据管道集中到一个地方。我用 **Fireflies** 记录我所有的会议。每天夜里，一个 **cron 任务**会进入 API，生成一个 Markdown 文件，然后放到 **GBrain** 里。我为我的公司构建了**向量数据库**。这真的很酷。一个向量数据库可以存储数百万甚至数亿的 token。你可以使用像 **Pinecone** 这样的服务来处理大量数据，它基本上能让这些数据对 LLM 来说变得高度可搜索。

我可以给你看一个例子。我训练了一个关于我的家族办公室 **Folly Partners** 的向量数据库，它基本上就是我的个人控股公司。这里我问它：“我做了多少少数风险投资？其中有多少是盈利、破产或减计的？” 它就去执行了这个查询，然后我就能看到我做了 132 笔直接投资，投资了 1600 万，现在价值 3600 万。然后它还能细分所有的减记等等。这真的非常强大。

我也为 **Tiny** 做了同样的事。在 Tiny，我有 24 家左右的公司，大量的历史数据。运营一个企业集团面临的挑战是，我不知道哪家公司在超支，哪个 CEO 在胡说八道，哪个业务趋势是向上还是向下，费用是否在上涨。对于一个人来说，数据太多了。虽然我不能展示，因为都是上市公司数据，但基本上，我现在可以在 Tiny 里问：“请回顾上一个季度，告诉我任何潜在的问题”或者问一个简单的问题，比如“我们在总部有多少会计员工？”我发现这非常有用。它并不完美，经常会说错某家公司的 CEO 或者数据有点偏差，但就能够在你的公司里扮演 **“索伦之眼”** 的角色而言，我认为它极其强大。

>> 好奇问一下，在 Tiny 内部，我看到你用它来与公司交互，来成为一个更好的董事长并提供建议。那么，在 Tiny 内部，是否有努力让公司成为 **AI First（AI 优先）** 之类的？

## 案例：CFO 使用 AI 构建软件

## 新时代的软件生意：“服务即新软件”

>> 是的。我的看法是，我总是以**殡仪馆**为例。殡仪馆是一个非常不性感的行业，没人想在里面工作。因此，你没有太多的竞争。如果你为某个拥有殡仪馆的人构建软件，比如他在 2010 年有个像你或我一样懂网络的外甥，他说：“嘿，你能为殡仪馆构建一个定制软件，来管理我的生意吗？” 然后你创办了软件公司，而你在这个市场里是唯一的玩家。没有其他拥有殡仪馆的人恰好也有一个懂网络的外甥会去做这件事。于是你拥有了**垄断地位**，成为了默认选择。

## 更多酷炫的应用：全天候录音、每日简报与自定义播客
