## 第一部分：什么是 Claude Code？

**Vin:** 好的。首先，什么是 Claude Code？Claude Code 是一个你可以**在命令行界面中使用**的代理。它基本上就是一个工具，可以控制你的电脑，你可以通过自然语言来使用它。比如，我可以说“在我的桌面上创建一个文件，内容是‘你好，Greg’，纯文本格式。”它就会去执行。这真的很酷，这是以前不可能实现的新功能。在这之前，我得自己去桌面，打开某个文本编辑器，然后创建那个文件。现在这个文件就在我桌面上了。我可以再说“打开文件”，看，出现了：“你好，Greg”。这很疯狂，对吧？

有趣的是，如果你有一个可以控制和操作你电脑的代理，那就意味着**任何你能描述给它的事情，它都能开始为你执行**。所以，当你向一个代理描述一个项目，或者你与它进行长时间的对话时，它就能执行越来越复杂的任务。它拥有的信息越多，它能做的复杂事情就越多。

但问题在于，假设你为一个特定项目写了一个非常长的描述，或者你与这个代理进行了一个小时的对话，你不想每次开启一个新会话时都重新解释一遍所有内容。很多人使用网页版的 Claude 或 ChatGPT，它们有类似记忆的功能，但你无法控制——你不知道记忆里有什么，不知道它知道什么、不知道什么。所以，需要有一种方法，能更轻松、更快速地将信息传递给这些代理。你能给它的信息越好、越快，它就能为你做越多的事情，你就能越快地把任务委派给它。

很多人非常喜欢使用 Obsidian 和类似工具，正是因为这种**建立相互关系的能力**。这是普通电脑文件夹无法做到的。随着时间的推移，当你不断建立这些相互关系时，事情就变得非常有趣了。这里有一个可视化视图。这些圆圈中的每一个都是一个文件，它显示了它与所有其他我写过的东西的文件是如何连接的。比如这里有“个人代理基础设施”，它链接到了“Agentic AI”，链接到了“Telegram”，链接到了 Shopify 创始人“Toby”，还有“存在感日志”、“Claudebot”等等。我还有一个播客叫“The Other Stuff”，你可以看到我显然对此思考了很多。

## 第四部分：Obsidian CLI + Claude Code = 模式识别器

**Vin:** 正是如此。接下来，Obsidian 发布了一个新工具，叫做 **Obsidian CLI**。它允许你使用 Claude Code，并且让 Claude Code 读取你 Obsidian 库中的所有文件（也就是那个文本文件文件夹）。但通过 Obsidian CLI，它不仅能给 Claude Code 提供它可以读取和访问的文件，还能提供这些文件之间**相互关系的信息**。所以，Claude Code 可以看到“哦，这个文件连接到了这个、这个和那个文件”。这在理解 Claude Code 能了解你什么，以及它能理解你正在处理的所有事物之间的关系方面，变得非常有趣。

它开始能够**浮现你自己都未曾注意到的思维模式**。你可能已经在库中写某个想法写了一年，它是一个潜在的想法，而它可以立刻说：“嘿，你知道吗，你在每个笔记中都在反复写关于创业的同一个模式？” 第一次看到这个，可能会产生巨大的“顿悟”效应。它能促使你在学习、理解、世界观以及你正在从事的项目上取得巨大进展。

## 第五部分：Vin 的自定义命令演示

**Vin:** 所以，我写了一些命令来演示如何将信息传递给代理，这是没有 Obsidian 和 Claude Code 就不可能做到的。这里有一些我使用的命令。我知道这些看起来可能有点复杂，但别怕。这只是我在终端里创建并运行在 Obsidian 中的命令。你不必非得用这个，你也可以在任何你想要的终端工具里做同样的事。但我把它放在 Obsidian 里，是因为我想把所有的东西都放在一起看，并向你展示如何集成和定制这个环境。

以下是一些关键命令：

### `/context` - 加载完整上下文

- **功能:** 读取上下文文件、每日笔记，并沿着反向链接构建一个关于我生活、工作和当前状态的完整图景。
- **演示:** 在一个新的 Claude 会话中，我只需输入 `/context demo`。它会立刻开始读取大量关于我当前状态的文件。你看，它正在读取 README，读取我正在做的媒体公司“New”的上下文，读取我的个人工作流程上下文。我不用担心它不知道关键信息，一个命令就搞定了。
### `/today` - 早晨回顾

- **功能:** 提取日历、任务、iMessage 信息和过去一周的每日笔记，整合成一个优先的今日计划。
- **为什么重要:** 当然，你可以让一个代理访问你的日历和任务，但它没有你正在思考和为什么思考的全部信息。如果我每天都在写关于某项技术或项目的每日笔记，我的日历能反映出这些吗？如果代理有这个上下文，它就能更有效地为你提供关于你应该做什么（或不该做什么）的信息，也能更有效地决定什么该进你的日历、什么不该进。
### `/closeday` - 日终处理

- **功能:** 提取行动项，浮现库中的连接，检查置信度标记，并更新需要更新的内容。
- **背景:** 我有很多假设，我会给它们打上置信度评分。这是我日常操作的一部分，但 Obsidian 对我来说最主要的用途是作为**思考工具**。我非常喜欢和大语言模型一起作为思考伙伴，这是我使用它们最喜欢的方式。
### `/ghost` - 以我的风格回答问题

- **功能:** 从库中构建一个声音画像，用那种声音写作，然后评估忠实度。
- **示例:** 我可以问“我对 AI 怎么看？”
### `/challenge` - 挑战当前信念

- **功能:** 使用库自身的历史来压力测试当前信念，找出矛盾、反证和思维转变。
- **为什么重要:** 确保自己作为人类和技能上不断发展，避免观点过度偏颇或受限。
### `/emerge` - 浮现未明说的想法

- **功能:** 从分散的前提中得出库隐含但从未明说的结论，找出未命名的模式、未阐明的方向。
- **价值:** 很多时候，我可能会多年 stuck 在浮现想法的不同方式上，有人简单地对我说一句“嘿，你知道吗，你一直在围绕这个模式转圈”，就能带来巨大突破。
### `/drift` - 检测意图与行为的偏差

- **功能:** 将我的陈述性注意意图与 30-60 天内的实际行为进行比较，浮现出我在回避的想法。
### `/deep` - 30天深度扫描

- **功能:** 对库进行 30 天深度扫描，结合跨域模式检测和图分析，生成跨所有领域的想法。
- **产出:** 不仅能给我关于工作项目的想法，还能给我关于该看的电影、该买的产品的想法，全都受我在库中写的东西影响。
### `/trace` - 追踪想法的演变

- **功能:** 追踪一个想法在库中随时间的演变过程。

## 第七部分：反思习惯、OpenClaw 与隐私

**Greg Isenberg:** 所以我认为，精通 Obsidian 的一部分听起来像是将**反思融入你的日常生活**。很多人从一个会议赶到另一个会议，我们很忙，我们是父母。随着我们长大，当然会在笔记本上写东西，但我觉得随着年龄增长，我们实际写作和反思得越来越少。你是如何将反思融入生活的？

**Vin:** 对我来说，我认为反思和大量做笔记之所以有趣，有两个原因。一是能够回顾它们很棒。作为一个不断变化的人，看到这些笔记，意识到自己的技能、项目、世界都在不断变化，这真是生活奇妙的一部分。二是有一个功能性的原因：**大量做笔记是我产生想法的方式**。当我坐在电脑前写下东西时，想法就来了。通过写作，我感觉更能内化它。我喜欢有好想法，喜欢进步。所以，如果你想培养写作习惯，首先得把它与“这是你进步和产生想法的方式”联系起来。

另外一点是，在当下，**写作是将任务委派给代理的一大方式**。这是一个全新的方面。如果你能养成写作习惯，你就能将更多的上下文传递给代理，这 dramatically 增加你可以委派和构建的事情的数量。

**Greg Isenberg:** 这与 OpenClaw 有什么关系？如果你把 OpenClaw 看作你（在最好情况下）的延伸，一个可以独立根据你的指导去做事的代理。那么你如何和谐地使用命令、Obsidian、OpenClaw 和反思呢？

**Vin:** 是的。比如这里有一个我用的 `/schedule` 命令。我问“我今天（2月20日下午2点）可以和 Greg Isenberg 开个会吗？”它不仅会查看我的日历，还会查看我的每日笔记，我在乎什么，然后给我一些观点。它说：“你今天日程已满，早上已经在录 Greg 的播客，然后是团队午餐外出，以及与 Peter 和 Vince 的会议。你 2 月 17 日的笔记显示 Greg 这期节目一直是你关注的重点。库中有一个专门的 Greg Isenberg 笔记。所以建议是：不行，不要下午 2 点，而且你可能根本不需要单独的会议。” 这确实是对的。

这与 OpenClaw 有什么关系？OpenClaw 是一个自主代理，如果你设置好，它可以去做事，不需要你每次都提示。现在，OpenClaw 可以像我刚才执行命令那样，自主地做同样的事。它可以去读取我的库，找到连接，然后**基于对我的更深入理解，代表我做出决策**。这样一来，我就不再是管理一个代理或与另一个人讨论工作，而是专注于管理这个库。这就是新的源头。我持续努力让这个库包含所有必要的信息，这样我就可以把任务委派给代理，而代理可以从这个库源头拉取信息并做决策。如果它做错了决策，我就修改库，而不是直接去和代理打交道。这是我的推测，我觉得非常有趣。

**Greg Isenberg:** 这让我有点担心：如果 Obsidian 真的是你的第二大脑，那么让 OpenClaw 访问你的第二大脑，这很吓人。
