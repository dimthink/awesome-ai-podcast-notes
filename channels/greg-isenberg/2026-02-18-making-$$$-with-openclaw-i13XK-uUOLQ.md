### 设计思维与工作流构建：从发现到部署 `[10:00-18:00]`

**核心观点**

将 OpenClaw 成功应用于商业场景，需要一套系统的设计思维方法，包括识别高价值低投入的自动化机会、详细映射工作流，并利用 AI 工具辅助整个设计过程。

**深度阐述**

Nick 强调，要将 OpenClaw 从一个“好员工”变成一个“卓越员工”，关键在于提供正确的上下文和指导。这需要一套设计思维的方法论来识别、规划和构建自动化工作流。

**1. 识别高价值、低投入的“低垂果实”** `[10:00-12:00]`

当进入一个企业或项目时，首先要做的不是立即自动化所有事情，而是运用设计思维，识别那些“高价值、低投入”的自动化机会。Nick 建议使用一个简单的二维矩阵来评估：

- **价值 (Value)**：自动化能带来多大的商业价值？（例如，节省时间、提高效率、增加收入）
- **投入 (Effort, Cost, Time)**：实现自动化所需的努力、成本和时间是多少？
目标是找到那些能够创造高价值但所需投入较低的任务，这些就是“低垂的果实”，是 OpenClaw 自动化的最佳切入点。他再次以之前提到的产品信息查找和上传 Zoho CRM 的案例为例，说明这就是一个典型的“低垂果实”。

**重要原话**：

- "What is the value that we can create by automating this thing and what's the relative effort, cost and time? And so we ultimately want to start with... high value and low effort, cost, and time." - Nick `[11:00]`
- **视觉/结构信息描述**：视频中展示了一个简单的图表，横轴代表“投入”，纵轴代表“价值”，并标示出“高价值、低投入”区域为优先选择。
**2. 详细映射自动化工作流** `[12:00-14:00]`

一旦确定了自动化机会，下一步就是详细地映射出整个工作流。Nick 建议使用工具如 Figma（或任何流程图工具）来可视化地描绘从头到尾的每一步。他举例说明了一个客户的场景：客户发送包含产品链接的邮件给她的客户，OpenClaw 需要查找所有产品信息，然后上传到 Zoho。这个过程被详细分解为“步骤一、步骤二、步骤三……”，确保 OpenClaw 能够端到端地完成整个任务。

**重要原话**：

- "Your next step after identifying the opportunity is to literally like map this out. I use Figma... map out the actual workflow process of like okay step one, step two, step three." - Nick `[12:30]`
**3. 利用 AI 辅助设计与规划** `[14:00-16:00]`

一个令人兴奋的“元”思考是：我们能否使用 OpenClaw 或其他 AI 工具来辅助设计和规划自动化本身？Nick 肯定了这一点。他提到，可以录制客户访谈，将转录文本上传给 AI，然后要求 AI 根据这些访谈内容，结合“高价值、低投入”的原则，识别出最佳的自动化机会，甚至直接生成 Figma 或 Mermaid 代码的流程图。他个人使用 Gemini for Google Meet 来记录会议，然后让 Gemini 总结工作流步骤，这大大简化了设计过程。

**重要原话**：

- "You could walk into a business and basically say, 'Hey, I want to figure out what we could automate here'... You upload the transcripts and then you say, 'Hey, based on that... give me the top three [automation opportunities] and then create, you know, Figma.'" - Greg `[14:30]`
- "You could literally ask it, yeah, like you said, based on this transcript, what is the automation workflow look like? You know, step by step." - Nick `[15:30]`
- **视觉/结构信息描述**：视频中展示了 Figma 界面，以及一个流程图示例，详细描绘了自动化任务的各个步骤。
**4. 触发机制与专业化 AI 员工** `[16:00-18:00]`

Nick 解释了 OpenClaw 自动化工作流的触发机制。例如，之前提到的产品信息上传流程，其触发条件可能是 OpenClaw 监测到一封包含特定链接的邮件。OpenClaw 可以设置 Cron Job 来监听这些事件，一旦触发，便启动预先编写好的 Python 脚本或其他自动化流程。

他强调，不应过度依赖 OpenClaw 自身的通用能力，而是应该创建“专业化的 AI 员工”（specialized AI workers）。这些“员工”是针对特定任务优化的 Python 脚本或 API 调用，由 OpenClaw 作为“协调者”（orchestrator）来调用。

**重要原话**：

- "The trigger of all of that is the OpenClaw being CC'd in an email and it seeing that email it has a link that is relevant for this type of workflow to be triggered." - Nick `[17:00]`
- "You're more of so creating specialized AI workers under each underneath the OpenClaw that it can call individually." - Nick `[17:30]`
**个人感受**

Nick 提出的设计思维方法论，为我们提供了一个清晰的框架，将抽象的 AI 能力转化为具体的商业解决方案。特别是利用 AI 辅助设计 AI 自动化的思路，展现了“AI 赋能 AI”的强大潜力，这无疑是未来工作方式的重要组成部分。

**延伸思考**

“高价值、低投入”的原则不仅适用于 AI 自动化，也适用于任何创新项目。它鼓励我们从小处着手，快速迭代，验证价值。同时，AI 辅助设计流程图的能力，预示着未来低代码/无代码开发将与 AI 深度融合，进一步降低技术门槛。

**精华收获**

采用设计思维，通过价值-投入矩阵识别“低垂果实”自动化机会，并详细映射工作流。利用 AI 工具（如 Gemini）辅助设计和规划，可以显著提高效率。通过创建专业化的 AI 员工和明确的触发机制，实现 OpenClaw 自动化工作流的端到端部署。

### 子代理（Sub-Agents）的威力：杠杆化与专业化 `[18:00-22:00]`

**核心观点**

子代理是 OpenClaw 实现高效、并行和专业化任务处理的关键。它们通过将主代理从繁重任务中解放出来，使其专注于协调和管理，从而极大地提升了 OpenClaw 的整体效能和商业价值。

**深度阐述**

Greg 提出了一个常见问题：子代理和任务有什么区别？Nick 对此进行了详细解释，揭示了子代理在 OpenClaw 生态系统中的核心作用。

**1. 子代理的定义与使用场景** `[18:00-19:30]`

Nick 指出，子代理可以从两个角度理解：

- **并行化任务**：OpenClaw 可以生成多个研究型子代理，让它们并行执行同一任务（例如，同时在五个不同实例上研究某个主题），或者将一个大任务分解成多个子任务，由不同的子代理分别处理。
- **专业化技能**：更重要的是，子代理可以被视为拥有特定“技能”的专业化 AI 员工。类似于 Anthropic 的“技能”概念，你可以为子代理提供特定的指令、规则和代码，让它专注于完成某项特定任务。例如，一个子代理可以专门负责在 Twitter 上寻找热门创意并收藏。
**重要原话**：

- "OpenClaw can spawn sub agents... each sub agent could have its own computer." - Nick `[03:50]`
- "You can have like these specialized instructions and rules along with actual code that you can provide to your agent for it to be able to go and do a given task." - Nick `[19:00]`
**2. 主代理与子代理的分工：协调者与执行者** `[19:30-21:00]`

Nick 强调，主代理（通用代理）应该被解放出来，充当“协调者”（orchestrator）的角色，而不是亲自执行所有任务。当主代理需要完成一项特定技能时，它可以调用一个专门的子代理来执行。这样，主代理就不会被长时间占用，可以继续处理其他协调工作。

他用一个生动的比喻来解释：如果你的 OpenClaw 实例正在“端着一杯热咖啡”，你就不能让它去搬桌子。子代理的作用就是为你的 OpenClaw 创造“杠杆”（leverage），让主代理能够专注于管理和协调，而将具体的、耗时的任务交给专业的子代理。这些子代理可以擅长 XYZ 领域，从而让主代理成为一个高效的“经理人”，负责检查工作质量、确保任务完成等。

**重要原话**：

- "I want my general agent to be freed up and to more so just be the orchestrator." - Nick `[19:30]`
- "What sub agents do is it basically creates leverage for your OpenClaw and it basically says like, 'Okay, I'm you're going to create a set of sub agents who are going to be good at XYZ thing.'" - Nick `[20:30]`
**3. 商业应用中的子代理** `[21:00-22:00]`

在为企业部署 OpenClaw 时，Nick 建议将所有 AI 自动化机会、工作流和任务都视为可以由特定子代理完成的“技能”。这意味着，每一个独立的自动化流程都可以封装成一个子代理，由主 OpenClaw 在需要时调用。这种模块化和专业化的方法，使得 OpenClaw 能够更灵活、高效地应对复杂的商业需求。

**个人感受**

子代理的概念是 OpenClaw 强大能力的核心体现。它将 AI 代理从单一的执行者提升为可扩展、可管理的“数字团队”。这种分工协作的模式，不仅提高了效率，也为构建更复杂的自动化系统提供了可能。Nick 的“热咖啡”比喻，形象地解释了子代理如何为主代理创造“杠杆”，令人印象深刻。

**延伸思考**

子代理的理念与软件工程中的“微服务”架构有异曲同工之妙。通过将复杂系统分解为独立、可复用的组件，可以提高系统的可维护性、可扩展性和鲁棒性。这预示着未来 AI 系统的构建也将越来越倾向于这种模块化、分布式的方式。

**精华收获**

子代理是实现 OpenClaw 商业价值的关键，它们可以是并行执行任务的实例，也可以是拥有特定技能的专业化 AI 员工。通过将主代理定位为协调者，子代理作为执行者，可以极大地提高自动化效率，并为企业提供更灵活、可扩展的 AI 解决方案。

### OpenClaw 的独特优势：全天候、可编程的“超级员工” `[22:00-24:00]`

**核心观点**

OpenClaw 的独特之处在于其拥有自己的“计算机”，能够全天候运行、可编程、可调度任务，并能通过文本消息进行交互，使其成为一个比人类员工更高效、更强大的“超级员工”。

**深度阐述**

Greg 提出一个核心问题：为什么 OpenClaw 如此特别？Nick 详细阐述了 OpenClaw 的几个关键特性，这些特性使其超越了普通的 AI 工具，成为一个真正具有颠覆性的存在。

**1. 拥有自己的计算机，全天候运行** `[22:00-23:00]`

OpenClaw 的一个核心优势是它拥有自己的“计算机”，并且能够 24/7 全天候运行。这意味着它不需要休息，不会疲劳，可以持续不断地执行任务。这与人类员工形成了鲜明对比，极大地提高了工作效率和产出。

**2. 可编程与任务调度** `[23:00-23:30]`

OpenClaw 能够编写代码（code），并调度任务。这意味着它不仅能执行预设的指令，还能根据需求生成新的代码来解决问题，并按照时间表或特定触发条件执行任务。这种可编程性和自主调度能力，使其能够适应各种复杂的业务场景。

**3. 文本交互能力** `[23:30-24:00]`

OpenClaw 可以通过文本消息进行交互，例如通过短信或 Telegram。这种简洁直观的交互方式，使得用户可以像与同事沟通一样，轻松地向 OpenClaw 下达指令、获取信息或调整任务。这种“聊天式”的界面，降低了技术门槛，使得更多非技术人员也能利用 OpenClaw。

**重要原话**：

- "It's the ability that it has its own computer. It's running 24/7. You can text it and you can schedule tasks." - Nick `[23:00]`
- "If we just removed OpenClaw from this card here and you just called this a really good employee, it would just make sense be like, 'Oh, works 24/7, can code, can schedule tasks, I can text it, and they have their own computer.'" - Nick `[23:30]`
**4. 克服“差劲员工”的挑战** `[24:00-24:30]`

Greg 提到，如果 OpenClaw 设置不当，也可能成为一个“差劲的员工”。Nick 认为，这通常是因为“经理”（即用户）没有提供正确的上下文和指导。这再次强调了设计思维和清晰指令的重要性。

**个人感受**

Nick 对 OpenClaw 独特优势的总结非常精辟。它将 AI 代理拟人化为“超级员工”，不仅强调了其技术能力，更突出了其在商业应用中的实际价值。这种将 AI 视为团队成员的视角，有助于我们更好地理解和利用这项技术。

**延伸思考**

OpenClaw 的这些特性，预示着未来企业组织结构可能发生根本性变化。当“数字员工”能够承担大量重复性、可编程的任务时，人类员工将更多地专注于创新、战略和人际互动。同时，这种“聊天式”的交互界面，也可能成为未来人机交互的主流模式。

**精华收获**

OpenClaw 之所以特别，在于它拥有自己的计算机，能够 24/7 运行，具备编程和任务调度能力，并通过文本消息进行交互。这使其成为一个高效、可扩展的“数字员工”，但其效能发挥的关键在于用户（经理）能否提供清晰的指导和上下文。

### 实时演示：构建 TikTok 趋势追踪代理 `[24:30-34:00]`

**核心观点**

通过实时演示，Nick 展示了如何从一个产品创意出发，利用 OpenClaw 及其生态系统（Orgo 平台、Claude Code 等）快速构建一个具有实际功能的 AI 代理，并强调了迭代、测试和利用 AI 辅助开发的重要性。

**深度阐述**

为了具体展示 OpenClaw 的构建过程，Nick 接受了 Greg 的挑战，尝试现场构建一个基于 Greg 的“Idea Browser”产品概念的 TikTok 趋势追踪代理。

**1. 从创意到技能：定义需求** `[24:30-26:00]`

Greg 的 Idea Browser 概念是“一个能捕捉病毒式浪潮的工具，在它们达到顶峰之前”。Nick 决定将其转化为 OpenClaw 的一个“专业技能”。他将 Greg 的产品描述直接粘贴到 OpenClaw 终端中，并询问：“你需要我提供什么来构建它？让我们制定一个计划。”这是他给出的第一个重要提示：**永远让 OpenClaw 提问，以获取所需信息。**

**重要原话**：

- "My number one tip is always ask it to ask you questions. So, what do you need from me to be able to build this out? Let's create a plan." - Nick `[25:30]`
**2. 最小可行产品（MVP）测试：探索能力** `[26:00-28:30]`

OpenClaw 回应称这是一个“宏伟愿景”，并要求提供数据访问、范围和利基焦点。Nick 决定从一个简单的 MVP 开始测试：让代理在 Orgo 的“Playground”模式中打开 Firefox，访问 TikTok，滚动页面，并总结“为你推荐”页面上最常见的视频。

**视觉/结构信息描述**：视频中展示了 Orgo 平台的“Playground”模式。当 Nick 输入指令后，一个虚拟计算机窗口弹出，Firefox 浏览器自动打开，导航到 TikTok，并开始滚动页面。整个过程是可视化的，代理通过视觉交互来操作界面。

Nick 解释说，这个代理正在通过视觉交互来操作屏幕，点击 Firefox，打开浏览器，输入 `tiktok.com`。他引用 Anthropic CEO Dario Amodei 的观点，强调“计算机使用代理”（Computer Use Agents）是实现通用人工智能（AGI）的关键瓶颈——让 AI 像人类一样操作计算机，但做得更好。OpenClaw 正是这种“计算机使用代理”的代表。

**重要原话**：

- "Can we get this agent to even just spin up Tik Tok and just scroll Tik Tok and identify what is on the for you page of Tik Tok? So let's like maybe start there." - Nick `[26:30]`
- "The constraint to getting there is computer use agents. The ability to have an AI that can operate a computer like you and I can, but better." - Nick `[28:00]`

Greg 观察到，这种构建过程是先从一个轻量级技能开始，进行测试，然后根据测试结果进行调试和优化。Nick 肯定了这种“设计思维”的重要性，他用“造车”的比喻来解释：如果目标是造车，也许不应该从车架开始，而是从一个滑板开始，先实现从 A 到 B 的核心功能。

在测试成功后，Nick 决定将这个功能转化为更具程序化的实现。他向 OpenClaw 提出，希望使用 Orgo API 文档来构建一个计算机使用代理，并询问需要什么。OpenClaw 立即提供了所需的架构、代码片段和 API 密钥要求（Orgo API Key, Anthropic Key）。

**重要原话**：

Nick 粘贴了 API 密钥，并让 OpenClaw 开始构建。他强调，这不仅仅是构建一个业务，更是在构建一个“资产”（asset）。在 AI 时代，人们将能够以前所未有的速度和规模创造数字资产。

**重要原话**：

- "We're building a business in a very short amount of time, but it's really like we're building an asset." - Greg `[32:00]`
- "Any idea you have, you can just build it." - Nick `[31:30]`

这个实时演示是整个视频的亮点，它将抽象的概念具象化，让观众亲眼看到 OpenClaw 如何将一个想法转化为实际运行的代理。Nick 在演示中展现出的快速迭代、测试和利用 AI 辅助开发的能力，令人印象深刻，也极大地增强了内容的说服力。

### 抓住机遇：垂直化、利基市场与“代理即服务” `[34:00-结束]`

**核心观点**

AI 代理将成为新的 SaaS（软件即服务），通过垂直化、专注于利基市场，并利用 OpenClaw 自动化现有市场需求，个人和企业可以抓住巨大的商业机遇。

**深度阐述**

视频的最后部分，Greg 和 Nick 总结了 OpenClaw 带来的巨大商业机遇，并提供了具体的行动建议。

**1. 市场需求：从 RPA 到智能代理** `[34:00-36:00]`

Nick 指出，Upwork 上仍然有大量关于“机器人流程自动化”（RPA）的任务。RPA 是一种过时、笨拙且不智能的自动化方式，它依赖于精确的 UI 位置，一旦界面变化就会失效。而 OpenClaw 这样的智能代理，能够通过视觉交互和上下文理解来操作计算机，从而克服了 RPA 的局限性。这意味着，现有的 RPA 市场是一个巨大的潜在客户群，他们正在寻求更智能、更灵活的自动化解决方案。

**重要原话**：

- "You just go to Upwork. Upwork's great because you get to see what the market's asking for and you just type in robotic process automation." - Nick `[34:30]`
- "If the button isn't in the exact UI space that you just delegated it to, it won't work." - Nick `[35:00]`
**2. 垂直化与利基市场：找到你的“不公平优势”** `[36:00-38:00]`

Nick 强调，不要试图成为所有人的所有。成功的关键在于**垂直化**（verticalize）和专注于**利基市场**（niche market）。选择一个你有所了解或有“不公平优势”的领域。这种优势不一定是 20 年的经验，可能仅仅是因为你的母亲是房地产经纪人，所以你了解这个行业的客户痛点。

他建议避开那些“红线”较多的行业，如医疗保健和金融，而是从制造业、分销业等领域开始。一旦在一个垂直领域构建了专业的自动化工作流，就可以将这些代理打包成一个“工作空间”，邀请新的客户加入，让他们立即拥有一个由 AI 员工组成的“团队”。

**重要原话**：

- "Focus like don't be everything to everyone. Like don't don't help any every business. help real estate agents, for example, or like pick a vertical that maybe you have some unfair advantage for some particular reason." - Nick `[36:30]`
- "Can you imagine you have a workspace and you invite some new company into this workspace that for automations, for manufacturing, for doors... they have all these AI employees in the workspace in Orgo set up ready to go." - Nick `[37:30]`
**3. 代理即服务（Agents as a Service）：新的商业模式** `[38:00-39:00]`

Greg 总结道：“代理是新的 SaaS。”过去，我们创建软件并将其出售给企业，企业员工使用软件。现在，我们将创建“代理”，并邀请企业使用这些代理，让代理为企业创造价值。这是一种根本性的思维转变。

**重要原话**：

- "Agents are the new SAS, like you know what I mean? So, yes, I agree with the like the vision you painted. I think that, you know, in the past, you know, we created software that we would sell to these businesses and then they would have people actually, you know, press the buttons, touch the knobs to to make it useful. Now, you don't you're not going to create software and invite them to the software. You're going to create agents and you're going to invite them to the agents and then the agents are going to do work that creates value for these companies." - Greg `[38:00]`
**4. 接口的消亡与聊天式交互的崛起** `[39:00-40:00]`

Sam Altman 曾说“每家公司都将成为 API 公司”。Nick 进一步指出，这意味着“接口正在消亡”。最终的接口似乎是聊天和文本消息。无论是 ChatGPT 还是 OpenClaw，都证明了聊天式交互的强大。用户不再关心 AI 如何完成任务，只关心它能否在后台高效运行并完成任务。

**重要原话**：

- "Every company is turning into an API company. ... And that's interesting because interfaces are in a sense dying in that way of like you know you won't interact like the ultimate interface for whatever reason seems to be chat and text message." - Greg & Nick `[39:00]`
**个人感受**

这部分内容为我们描绘了一个清晰的商业蓝图。它不仅指出了 AI 代理的巨大市场潜力，更提供了具体的策略和方向。特别是“代理即服务”的理念，预示着未来商业模式的深刻变革，令人对 AI 时代的创业充满期待。

**延伸思考**

“代理即服务”模式将催生大量新的创业公司和商业机会。这些公司将不再是传统的软件公司，而是“AI 劳务公司”或“自动化解决方案提供商”。同时，接口的消亡和聊天式交互的崛起，也对产品设计和用户体验提出了新的挑战和机遇。

**精华收获**

抓住 OpenClaw 商业机遇的关键在于：识别并自动化现有 RPA 市场的痛点；通过垂直化和专注于利基市场来建立“不公平优势”；将 AI 代理视为新的 SaaS 产品，提供“代理即服务”；并认识到聊天式交互将成为未来主流的用户界面。
