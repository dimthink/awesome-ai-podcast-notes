## 会话管理与成本控制

**AF**：几个事情：首先，**会话（Sessions）** 全都在这里了。以前在 Telegram 中，如果你想要会话或多线程的概念，你需要在 Telegram 里设置线程，这有点像高级操作且容易混乱。我知道很多极客会说：“你在说什么？这很容易。”但对于普通人来说，在 Telegram 中设置线程不是最容易的事。你需要创建单独的群组，然后添加机器人，还要编辑主题等等。而在这里，每次你和你的 Hermes 代理对话，它都会自动进入一个新的会话，你可以随时与它对话。这使得**分离上下文**变得非常方便。我认为大多数人默认使用 Hermes 代理的方式是有一个单一的主线程，就像一个大的对话，他们在里面谈论所有的事情。问题在于，你会用谈论的一百件不同的事情来**污染上下文**，导致你的**成本飙升**。这是对 Hermes 的第一大抱怨：“天哪，我每月付 1000 美元。”如果你管理好你的上下文和会话，你根本付不到 1000 美元，完全不可能。

**AF**：节省成本第一条：**合理管理会话上下文**。特别是使用 Opus 这种有百万参数上下文窗口的模型时，上下文很容易让你的账单膨胀 3 到 4 倍。如果你让每个会话保持独立，比如这里是我的“内容”会话，这里是“讨论本地模型”会话，“模式研究”是我的股票研究会话。上下文保持精简，你就能节省大量金钱，因为每次发送消息时，只会包含该线程本身的历史。

## 成果物管理：第二大脑的完美实现

## 技能与工具管理

**AF**：**技能与工具（Skills & Tools）** 部分有很多好东西。这是一个很好的界面，显示所有安装在 Hermes 代理上的技能。有一点很多人没有意识到：开箱即用，你的 Hermes 代理已经安装了超过 **150 个技能**，每个技能实际上都会增加上下文，从而增加你的成本。现在通过这个用户界面，你可以进入并关闭任何不必要的技能。你还可以看到基于你的操作而自动生成的**新技能**。很多人不知道 Hermes 有很强的**自我改进**能力，但不知道它实际上会在后台创建大量技能。比如我因为构建了很多 3D 游戏，就有了 3JS 游戏技能；还有 YouTube 新闻脚本、实时股票仪表盘等。很多东西在后台自动生成，你可以根据需要开关任何不相关的技能，以节省成本并提升效率。

**工具集（Toolsets）** 是工具的组合。这是一个新概念，正在逐步推出：你可以把多个技能和工具组合在一起。如果你正在做复杂的事情，它会把它们分组在一起。我还没有完全深入探索，但工具集肯定值得更多使用，因为它是一个新概念，可以让你将多个工具和技能组织起来。还有**消息（Messaging）** 功能：你可以非常方便地设置消息服务。以前你需要进入 CLI 的 Hermes 设置，导航到相应服务。普通人根本不想用 CLI。虽然过去几个月有股潮流说“CLI 是未来，所有人都会用 CLI”，但这并不能改变**普通人不想进入终端做任何事情**的事实。在我的设置教程视频下面，我收到很多评论说“就这么简单？在终端粘贴一下？”，还有人直接说“我不想做这个，我完全没兴趣，这不适合我”。所以桌面端让设置变得非常容易：点击即可设置 Telegram，然后它就能正常工作。你再也不需要进入终端或 CLI 了。设置东西变得非常简单。

## 定时任务：确认与优化

**AF**：另一个我喜欢的功能是**定时任务（Cron）部分**。如果你点击下面的 cron，你会看到你设置的所有定时任务。我收到的一个最大抱怨是：“Alex，你说要设置一个例程，我做了，但第二天没生效。例程没有发生。”问题在于，当你通过 Telegram 或 CLI 设置这些定时任务时，你实际上无法确认它们是否被成功创建。现在，通过 cron 界面，一目了然，你可以看到所有已安排的任务，甚至可以直接在这里手动创建。例如，你可以设置“每天早上 9 点给我晨间简报”，每日，交付至桌面，创建定时任务。这样你可以 **100% 确定这个定时任务被创建并且会每天按时执行**。

**GI**：定时任务和例程对释放 Hermes 的价值至关重要。你有什么建议，如何创建措辞恰当的定时任务以确保成功？

**AF**：当然。关键在于**反向提示（Reverse Prompting）**。永远如此。让我们现场演示。假设你是 Hermes 新手，想设置一个每天早上 9 点包含相关信息的晨间简报。你会说：“我想和你设置一个每天早上 9 点的晨间简报。”但很多人就停在这里了。这里你可以通过**反向提示**升级：你接着说：“我的兴趣一般是 AI、股票投资、科技，以及波士顿凯尔特人队。”然后问：“根据这些信息，最适合的提示是什么？”

我做了几件非常重要的事：第一，这是我最爱的组合：**大脑倾泻（Brain Dump） + 反向提示**。你把关于自己的所有信息（兴趣、目标、技能等）倾泻给代理，让它们了解你，然后它基于对你的了解反向请求你给出最好的方法。很多人只是直接告诉代理做什么，但忘了你正在和超级智能对话。所以为什么不从超级智能那里获得帮助，用最好的方式来做？我点击回答，它现在会给我一个非常详细、具体的提示。它说：“你是在为 Alex 编写每日晨间简报。当前是东部时间清晨。从过去 24 小时内拉取新鲜信息，不要依赖记忆或陈旧数据。使用网页搜索，获取真实头条、价格和分数。覆盖这四个领域：AI（过去 24 小时最重要的 3-4 个发展）、市场与投资（昨日收盘和盘前方向、标普、纳斯达克、突出标的等）、科技（除 AI 外的重大科技故事）、体育（凯尔特人队最新动态）。”并且指定格式：“以一行情绪摘要开头，使用粗体、项目符号。”如果使用原始提示“告诉我 AI 新闻”，你可能会得到两个月甚至一年前的旧闻。而通过优化后的提示，你能确保时间是“过去 24 小时”。所以，**大脑倾泻 + 反向提示**，你会得到更好的提示、定时任务和代理输出。

**GI**：很好的建议。

**AF**：不客气。还有什么呢？定时任务真的很酷，非常好用。你可以轻松查看**上下文窗口使用情况**，方便你在窗口快满时开始新会话，以节省成本。你还可以快速切换**模型**。这是 Hermes 的另一个巨大优势：他们架构代理的方式让你可以比 OpenClaw 更容易地切换模型。OpenClaw 中每个模型几乎是硬编码的，你需要请求切换模型，新模型推出时要等待 OpenClaw 更新。而 Hermes 动态工作，可以即时切换模型和思考设置，一旦新模型发布，你可以立即切换。现在在这里，你可以进一步定制：如果任务简单，不需要太多思考，可以用 Opus 4.8 的最小努力模式或 Haiku 的最小努力模式。这是为代理省钱的另一种好方法。

## 子代理与配置文件的区别

**AF**：还有一个**代理窗口**。当你多任务处理时，可以在这里看到所有**子代理（Sub-agents）** 被启动。通常我会被问到：子代理和配置文件有什么区别？**配置文件是真正的独立 Hermes 代理**，有自己的技能、个性等。**子代理是主代理的副本**。所以当我让 Hermes 去做“研究并基于研究构建应用”时，它会启动子代理，这些子代理只是主代理的副本，拥有相同的技能、个性等。但配置文件有自己的技能集、上下文和记忆。所以你应该这么想：任务是否需要一套独立的技能？如果需要，就创建新配置文件。如果任务可以用同一套技能但需要同时进行多个实例？那就用子代理。代理窗口就是查看那些子代理工作的地方。

**GI**：能举个具体例子吗？什么时候需要用多个子代理？

**AF**：假设我想构建一个微型 SaaS，它有五六个不同功能，比如连接 X API 拉取推文，格式化推文，然后允许分享到 Twitter 等。对于这种任务，我会使用子代理。我对我的 Hermes 说：“为我构建这个微型 SaaS，这里有 10 个功能，启动子代理同时构建每个功能。”它会启动五六个子代理，因为每个子代理都拥有相同的编码技能，所以不需要不同技能，只需要同一技能的多个实例一起工作。

而如果我想做 YouTube 视频：我需要一个**研究员**来搜集信息，一个**编剧**来写脚本，一个**图形设计师**来制作缩略图（连接 Grok 图像生成 API）。这时候我需要多个**配置文件**，因为需要不同的技能集。我会说：研究员用 Quen（本地模型，做研究），然后把结果交给 Hermes（Opus 4.8 模型）来写脚本，再把结果交给 GPTM（ChatGPT 模型）用 ChatGPT 图像生成器制作缩略图。这就是配置文件的使用场景。

所以总结：**需要不同技能集** -> 用不同配置文件；**同一技能集，但需要同时多个实例** -> 用子代理。
