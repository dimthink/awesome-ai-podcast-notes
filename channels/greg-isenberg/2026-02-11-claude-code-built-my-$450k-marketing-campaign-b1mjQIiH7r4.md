### 开篇引入

在当今这个AI工具层出不穷、令人眼花缭乱的时代，无数创业者和“构建者”（builders）沉浸在技术创新的狂喜之中，用AI搭建各种精巧的系统和产品。然而，当这些精心打造的项目上线后，却常常只剩下“蟋蟀的叫声”——没有客户，没有收入，更谈不上被动收入。这究竟是哪里出了问题？是技术不够先进？还是产品不够完美？

本期节目，Greg Isenberg 邀请到了营销专家 Jonathan "Jice Cream" Courtney，一位通过实践证明其营销策略有效性的连续创业者。Jonathan 不仅运营着数个达到六位数乃至七位数收入的企业，更拥有长达 14 年将“不卖座”的产品转化为“畅销品”的丰富经验。他将以其独特的视角，揭示一个被普遍忽视的真相：作为一名 CEO 或创业者，你的核心职责并非仅仅是构建，而是**推广**。AI 工具的真正价值，在于如何赋能这一核心职责，将你的“Vibe Coded”项目从无人问津的“蟋蟀叫”阶段，推向“美元符号”的成功彼岸。这不仅仅是一场关于AI工具使用的技术分享，更是一次关于创业本质、营销策略和CEO思维模式的深度重塑。

### CEO 的真正使命：从“构建者”到“推广者” `[02:00-11:00]`

**核心观点**

Jonathan 强调，当前存在一个巨大的误解：许多人认为 CEO 的工作就是利用 AI 工具构建复杂系统，然后钱就会自然而然地来。然而，他通过自身经验和对成功创业者的观察指出，CEO 的核心职责是推广业务，获取用户和收入，而构建只是支持这一目标的手段。

**深度阐述**

Jonathan 一针见血地指出，他经常在社交媒体上看到一种普遍的错误观念，即认为作为 CEO 或创业者，你的主要任务是精通 AI 工具，构建出超级复杂、超级酷炫的产品和自动化系统，然后收入就会随之而来。他与许多同样对 AI 充满热情的 CEO 朋友交流时，总是会问：“你花了四五天时间构建了这些酷炫的系统，现在你的收入增加了吗？或者说，你现在有收入了吗？” 他的答案往往令人警醒：如果使用不当，如果 CEO 不理解自己的真正角色，这些 AI 工具很可能变成“拖延症机器”，让你在无意义的构建中浪费大量时间。

为了进一步论证这一观点，Jonathan 抛出了一个引人深思的问题：“你知道 Anthropic 的 CEO 叫什么名字吗？他长什么样？” `[04:40]` 多数人可能答不上来。但当他问及 OpenAI 的 CEO Sam Altman 或 Levels.io 的 Peter Levels 时，大家却能脱口而出。他指出，这些在 AI 领域取得巨大成功的企业家，他们有一个共同点，却很少有人提及：他们至少 50% 的工作时间都花在**推广**自己的业务上。

“你的核心工作，就是走出去推广你的业务。” `[05:30]` Jonathan 强调，虽然构建酷炫的产品很重要，但那只是围绕核心推广任务展开的辅助工作。他以 Peter Levels 为例，指出其 X（原 Twitter）账号置顶的帖子，就是对其 Photo AI 工具的长期推广。这与许多“构建者”认为“营销很烂，只要产品好，人们自然会来”的观念截然相反。现实是，我们之所以知道这些工具的创始人，正是因为他们频繁出现在各种播客、采访中，积极地推广自己和产品。

Jonathan 用一个生动的比喻来解释这种误解的危害：想象你开了一家餐厅，花了一年时间将所有机器自动化，做出完美的食物，系统完美无瑕，在线预订便捷。但你却从未告诉过任何人这家餐厅的存在。结果可想而知，这家餐厅会很快倒闭，尽管你享受了“玩乐”和“拖延”的过程。

因此，作为 CEO，你的职责不是成为 AI 工具的专家，也不是仅仅构建酷炫的产品。如果无法推广这些产品，它们将无人问津，最终成为一个“自我封闭的拖延站”。Jonathan 明确表示，他热爱 AI 工具，并将展示如何使用它们，但前提是这些工具必须服务于他作为 CEO 的核心职责：获取用户、创造收入、达成交易。他甚至反问主持人 Greg Isenberg，作为 Ideer.com 的创始人，Greg 花了多少时间推广自己的业务？答案显而易见：这个播客本身就是推广工作的重要组成部分。

**个人感受**

Jonathan 在阐述这一部分时，语气坚定而充满洞察力，他似乎对这种“构建者陷阱”感到一丝无奈，但更多的是一种希望通过分享经验来纠正这种普遍误解的使命感。他用“拖延症机器”和“餐厅比喻”形象地描绘了错误的焦点可能带来的后果，让人感同身受。他承认自己也是一个“经典 ADHD 创业者”，不爱做准备，但即使如此，他也深知推广的重要性，并努力将 AI 工具融入到这一核心任务中。

**延伸思考**

这一观点挑战了许多技术驱动型创业者的固有思维。在技术门槛降低、人人都能“Vibe Code”的时代，产品的差异化可能不再仅仅是技术本身，而是如何有效地将产品推向市场。这引发了一个更深层次的问题：在高度内卷的市场中，推广能力是否已经成为比技术能力更稀缺、更核心的竞争力？一个 CEO 如何平衡构建和推广的时间与精力，是决定企业生死的关键。

**精华收获**

- **CEO 的核心职责是推广业务，获取用户和收入。** 构建产品是支持这一目标的手段，而非终极目的。
- **AI 工具若使用不当，可能成为“拖延症机器”。** 避免沉迷于技术构建而忽视市场推广。
- **成功创业者都是积极的推广者。** 观察行业领袖，学习他们的推广策略。
- **推广是业务的“氧气”。** 缺乏推广，再好的产品也无法生存。

### 从“蟋蟀叫”到“美元符号”：三步营销蓝图 `[11:00-26:00]`

**核心观点**

Jonathan 提出了一个清晰的三步营销框架，旨在将潜在客户从最初的接触点，通过培养和转化，最终引导至购买行为，并强调 AI 工具如何支持这一流程。

**深度阐述**

Jonathan 拿出了他为客户展示的“Vibe Coded 蓝图”，一个最初手绘在纸上，后用 AI 精心制作的营销流程图。他强调，理解这个蓝图是利用 AI 进行销售的基础。这个蓝图包含三个核心步骤，形成一个循环：

1. **获取流量 (Traffic)** `[13:00]`
1. **建立“等待模式”/培养池 (Holding Pattern)** `[14:00]`
1. **销售事件 (Selling Event)** `[16:00]`
Jonathan 以 Peter Levels 的 X 账号为例，再次印证了这个循环：Levels 通过参加播客等方式获取流量，将人们引入他的 X 账号或通讯，在那里他发布个人生活（如如何用空气炸锅烹饪牛排）和产品相关内容，保持用户参与。然后，每隔一段时间，他会明确地进行一次销售推广，例如他置顶的关于“e-girl”帖子的推广，并展示了随之而来的销售增长。他强调，这并非是 Peter Levels 刻意算计的“销售活动”，而是从“等待模式”中自然发生的转化活动。

他进一步解释了 Anthropic CEO 参加达沃斯论坛接受采访的深层原因：这并非为了“人类的福祉”，而是为了推广业务，将企业客户引入 Anthropic 的“世界”，让他们理解 Claude 是企业级 AI 解决方案。对于普通创业者，虽然没有 Anthropic 那样的预算，但这个“流量-等待-销售”的循环是普遍适用的。Jonathan 甚至以自己为例，说明他参加 Greg 播客的目的就是获取流量，引导听众了解他，进入他的“等待模式”（播客、邮件通讯），最终通过销售事件（如网络研讨会）转化为客户。

他将这个流程比作“业务的氧气”和“马达”。如果缺乏流量，等待模式中的人数就会不足，销售事件的效果就会大打折扣。他分享了一个量化指标：当邮件通讯新增 5000 人时，他可以预估将有 200 人参加下一次网络研讨会，并有 12 人完成购买。这种可预测性让整个营销机器变得清晰。

**视觉/结构信息描述**

Jonathan 在屏幕上展示了一个“Vibe Coded 蓝图”，这是一个结构清晰的流程图。图的顶部是“流量”，分为“有机”和“付费”两大分支。流量汇入中间的“等待模式”（Holding Pattern），其中包含邮件通讯、播客、YouTube 内容等。从“等待模式”引出“销售事件”（Selling Event），包括网络研讨会、邮件营销、再营销、直接外联。如果销售成功，则转化为“购买/免费试用/预约通话”；如果未转化，则箭头指向返回“等待模式”，形成一个循环。当他开启“AI Use Cases”模式时，蓝图的每个环节下方都出现了 AI 工具的具体应用建议，背景还出现了他自己的“飞行贴纸”和“冰淇淋光标”，增添了趣味性。

**个人感受**

Jonathan 在讲解这个蓝图时显得非常兴奋和自信，仿佛在揭示一个他多年实践的秘密。他用“蟋蟀叫”和“美元符号”的对比，以及“业务的氧气”的比喻，让这个略显复杂的营销流程变得生动易懂。他对于 Peter Levels 和 Anthropic CEO 行为的解读，展现了他作为营销专家的敏锐洞察力，将看似无关的公共活动与核心营销目标联系起来。

**延伸思考**

这个三步蓝图提供了一个普适性的营销框架，无论产品大小、行业如何，其核心逻辑都是相通的。AI 工具的介入，并非颠覆了这一逻辑，而是极大地提升了每个环节的效率和规模。这促使我们思考，如何在自己的业务中识别并优化这三个环节，并积极探索 AI 在其中的应用潜力。

**精华收获**

- **营销流程三步走：流量 -> 等待模式 -> 销售事件。** 这是一个持续循环，确保潜在客户的不断转化。
- **不要直接推销，先培养关系。** “等待模式”是建立信任和价值的关键。
- **销售事件是转化的桥梁。** 利用网络研讨会、邮件营销等方式促成购买。
- **量化营销效果，建立可预测的“马达”。** 了解每个环节的转化率，优化整体效率。
- **AI 是营销流程的强大辅助工具。** 它能提升效率，但不能取代核心的营销策略和CEO的推广职责。

### Claude 与 Claude Code：我的 45 万美元营销战役幕后 `[26:00-40:00]`

**核心观点**

Jonathan 详细展示了他如何利用 Claude 和 Claude Code 这两款 AI 工具，从准备播客内容到实际构建营销活动，实现高效、大规模的营销工作，并分享了他在使用这些工具时的具体流程和心得。

**深度阐述**

Jonathan 终于打开了“AI Use Cases”开关，展示了 AI 如何作为 CEO 营销工作的强大支持。他以自己准备参加 Greg 播客为例，详细阐述了 Claude 和 Claude Code 的实际应用。

**1. Claude 作为营销助理：播客准备流程 ****`[27:00]`**

Jonathan 将 Claude 视为他的“营销助理”。他首先将关于“推广者概念”的所有思考和笔记，以及过去与 Greg 合作播客的数据（观看量、评论等），全部输入 Claude。他的目标是：

- **创建视觉图表**：用图表解释“推广者”概念。
- **研究受众**：分析 Greg 播客的受众特点，找出可能失去他们注意力的点。
- **ADHD 友好型准备**：由于他有 ADHD，不喜欢阅读脚本，他希望 Claude 能帮助他创建一个“可扫描的、ADHD 友好型”的概述文档，以便在播客进行时快速查阅，不遗漏要点。
他的具体操作步骤如下：

- **语音输入**：他直接用手机录制了 15 分钟的“脑力倾泻”，将所有想谈论的内容倾倒出来，然后将音频文件导入 Claude。他强烈推荐使用 WhisperFlow 等工具进行语音输入，因为这能让他更自然地与 AI 交流。
- **迭代指令**：他给 Claude 设置了一个初始指令：“我将作为嘉宾参加 Greg Isenberg 的 Startup Ideas 播客，你将协助我组织强大的内容角度、想法和策略，为他的听众提供最大价值。” `[30:00]` 每次对话结束后，他都会让 Claude 生成一组可粘贴的指令，不断更新其对项目的理解，使其在后续对话中保持上下文。
- **背景研究**：在另一个独立的聊天中，他让 Claude 对 Greg 进行了背景研究，分析了 Greg 的 X 账号和 YouTube 频道，生成了一个研究文件，避免每次新聊天都重复研究。
- **生成内容**：这些准备工作最终帮助他创建了一个简单的概述，并进一步生成了他在播客中展示的“推广者蓝图”信息图。他强调，这种详细的提示只有通过口语交流才能真正实现。
**2. 从 Claude 到 Claude Code：构建营销元素 ****`[32:00]`**

当他对 Claude 生成的蓝图初稿满意后，他便将其从标准 Claude 转移到 Claude Code。

- **上下文迁移**：他让 Claude 生成一个 `claude.md` 文件，其中包含项目的所有上下文信息，以及一个 HTML 文件。他下载了这些文件，然后导入 Claude Code。
- **Claude Code 流程**：在 Claude Code 中，他创建了一个项目文件夹，并指示 Claude Code 读取 `claude.md` 文件以获取上下文。然后，他利用 Claude Code 的 `ask`*`user`*`question` 或 `plan` 模式，让 AI 进一步“拷问”他对播客内容的想法，从而获得更多细节和优化。
- **部署**：最终，Claude Code 输出了优化后的蓝图，并将其部署到 GitHub 和 Vercel 上，形成了他在播客中展示的互动页面。整个过程，从研究到生成营销元素，仅用时约一小时，且大部分时间 AI 都在后台运行，他可以同时处理其他事务。
**3. 复杂营销活动的构建：45 万美元战役 ****`[35:00]`**

Jonathan 进一步展示了如何使用 Claude 进行更复杂的营销活动构建。他透露，他正在策划一个有望带来 45 万美元（至少 35 万美元）收入的营销活动，目前已有 4000 人注册了网络研讨会。

- **“AJ and Smart 营销专家大脑”**：他有一个专门的 Claude 项目，称之为“AJ and Smart 营销专家大脑”，专注于营销文案和策略。他更喜欢这个界面，因为它能提供更好的概览。
- **丰富的上下文文件**：这个项目包含了大量的上下文信息，例如他自己撰写的书籍、一个长达 17,766 行的营销邮件合集（他从别人的营销邮件中学习并提炼精华），这些都被 Claude 消化并添加到 `claude.md` 文件中。
- **生成潜在客户磁铁 (Lead Magnet)**：他演示了如何利用 Claude 快速生成与播客内容相关的免费可下载资料（lead magnet），以吸引听众订阅他的邮件通讯。他将播客的上下文信息（通过 `claude.md` 文件）导入，让 Claude 提出三个潜在客户磁铁的例子。Claude 迅速给出了“推广者蓝图一页纸”、“50 个推广者 CEO 提示”和“洞穴居民审计：你在构建还是推广？”等富有洞察力的建议。
- **效率提升**：过去，一个完整的潜在客户磁铁营销活动可能需要两到三天，涉及与团队沟通、设计师制作图表、选择落地页工具等。现在，借助 AI，整个过程只需半天，且能快速创建高度定制化的营销活动。
**4. Claude 与 Claude Code 的选择 ****`[38:00]`**

Jonathan 承认，他仍在摸索何时在 Claude 和 Claude Code 之间切换。他发现标准 Claude 界面更适合营销文案和策略的概览，而 Claude Code 则在需要快速迭代和构建特定功能（如落地页、复杂的问答流程）时表现出色。他提到一位朋友的公司 Pipex，其联合创始人建议他使用 Claude Code 的 `ask`*`user`*`question` 技能来设计网络研讨会，让 AI 像导师一样“拷问”他，从而得到更优质的结果。

**个人感受**

Jonathan 在展示 AI 工具使用时，显得非常熟练和兴奋。他坦诚自己有 ADHD，这使得他更倾向于利用 AI 来弥补个人习惯上的不足，例如通过语音输入和 ADHD 友好型文档来提高效率。他对于 AI 带来的效率提升感到惊叹，并将其比作“克隆自己”，能够同时处理更多任务。他对于 AI 能够将原本需要数天的工作压缩到数小时内完成，感到非常振奋。

**延伸思考**

Jonathan 的案例展示了 AI 在营销领域的巨大潜力，它不仅仅是内容生成器，更是战略助手、项目经理和效率倍增器。他的工作流强调了“上下文管理”的重要性，即如何让 AI 持续理解项目背景，从而提供更精准的协助。这促使我们思考，如何在自己的工作流中建立类似的 AI 辅助系统，并根据任务性质灵活选择不同的 AI 工具或模式。

**精华收获**

- **AI 是高效营销的强大助手。** 利用 Claude 进行内容研究、策略制定、文案生成。
- **通过语音输入和迭代指令，优化与 AI 的交互。** 建立持续学习的 AI 助手。
- **Claude Code 适用于快速构建定制化营销元素。** 例如落地页、信息图等。

### 结语：拥抱推广者的角色 `[48:00-50:00]`

Jonathan 再次强调了 CEO 作为“推广者”的不可或缺性，并鼓励创业者拥抱这一角色，或寻找能够承担此职责的伙伴。

**延伸思考**

这个结语是对整篇文章的升华。它提醒我们，无论技术如何发展，人性的基本需求和商业的底层逻辑依然存在。推广不仅仅是销售技巧，更是对产品、对愿景的信念表达。一个 CEO 的推广能力，在很大程度上决定了其愿景能否被市场所接受。
