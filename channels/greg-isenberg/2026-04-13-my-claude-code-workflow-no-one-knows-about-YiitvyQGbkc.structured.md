## 四、使用 Paper 进行设计优化

现在我们要回来使用“**落地页架构师**”技能。对于不知道 **Paper** 的人来说，它是一个非常酷的工具，它是连接在一起的。让我退一步讲：

通常情况下，使用 Figma 时，设计师会在 Figma 中创建静态的落地页设计资产，然后交给工程团队。缺失的环节是：现在有很多人直接用 Claude Code 在代码中构建落地页和网站，并在代码中优化，但会失去对迭代内容的追踪，也无法进行精细的优化。

而使用 Paper，你可以拥有一个连接到 Claude Code 的界面，直接在设计中构思、创建多个迭代版本，然后决定你要朝哪个方向走，最后再把它变成代码。这就像是**之前缺失的中间步骤**——之前人们只是直接在代码中构建。

**Greg**：你可以用 Paper。或者还有别的竞品吗？还是就坚持用 Paper？

**Amir**：Figma 最近刚推出了他们的双向 ACP……MCP，你可以既设计又构建代码，或者从代码到设计双向进行，这基本上就是 Paper 已经做到的事情。但我个人觉得，Paper 的工具、界面和体验要好得多，而且运行得更顺畅。

## 五、如何优化设计：参考图像与设计系统

我们已经捕捉了引导磁铁，添加了落地页架构师技能以及我们想要的 offer，现在我们想用 Paper 来实际构建这个页面。

在这个过程进行的同时，我想谈谈如何优化你的设计。很多时候你会看到人们那些“vibe coded”的设计，他们不完全清楚如何朝着正确的方向去构建一个设计系统。

我通常喜欢做的是：**给 Claude 一张我喜欢的现有设计的参考图片**。例如，我会去一个现有的网站，或者打开我收藏的一些书签，截取屏幕截图，然后丢给 Claude，说：“从这些页面中提取关键的设计元素，帮我创建一个设计系统。”

你现在看到的这个设计系统，就是我基于参考图片用 Claude 创建的，这样我们可以在未来的会话中引用它。如果我想回来为我的落地页创建新的区块或组件，我会说：“参考设计风格指南作为基础来创建新组件”，从而保持一致性。

现在我想展示一个例子。很多人错误地认为“vibe coded”的设计就是糟糕的。**这个页面完全是 vibe coded 的**——我们用 Claude 构建了所有这些动画和组件——但它看起来并不糟糕，反而非常精致、完善。要做到这一点，你需要通过 Paper 来优化它，并提供来自其他网站或库的示例组件和插图来构建。

例如，我经常使用一个叫 **Tail Arc**（Tail Pro）的工具，它有很多现成的优秀区块和插图，你可以安装它们并用作参考。

## 十四、构建自主 CRO 代理

**Amir**：我们在 Humble 使用了这种方法——我们刚刚开始构建一个**自主的 CRO（转化率优化）代理**。我们有直接的技能和 MCP 连接到 Claude Code，直接连接到我们的分析、实验工具、我们的营销网站（这就是我们迁移到自定义代码的原因，在 Webflow 上我们做不到这一点），以及 Google Ads。

思路是：我们如何快速启动多个针对特定广告活动的个性化子页面？例如，如果我在 Google Ads 上运行一个痛点广告活动，有五个广告组，我希望每个广告组都有自己的个性化落地页，并且我想对每个落地页进行 A/B 测试，看看哪个标题或布局最能引起共鸣。

通过自定义代码网站和 Claude Code，你可以非常快速地做到这一点。而使用 Framer 或 Webflow，你需要创建页面、创建区块、回来更新它们。这就是我们之前讨论的论点：**网站的未来是自定义代码，代理就是 CMS**。

然后你可以让它自动运行。我们甚至设置了一个 **cron job** 直接在 Claude Code 中，因为它们支持自动化任务。每周运行一个特定任务。我们有以下 Claude 技能：

- **付费媒体经理**：通过 API 从 Meta 和 Google 拉取数据
- **CRO 优化器**：运行 A/B 测试
- **漏斗报告**：给出报告，比如有多少用户注册，它连接到 Stripe、ChartMogul 以及我们所有不同的工具
- **Humbolytics**：通过 Firecrawl 抓取页面，给出优化建议，帮助你运行实验，并通过 Claude 管理结果
**Greg**：这太酷了，我们竟然生活在可以做到这一切的世界里。

**Amir**：是的。最棒的是，你可以用 Idea Browser 拉取所有这些上下文，并将其作为一个文件保存回 Idea Browser。

**Greg**：Jordan 和我在讨论：“哟，我们现在就能做到这一点。想象一下，你做一个每周报告，拉取所有数据，把它添加到 Idea Browser 中。那么 Idea Browser 就有了关于增长随时间变化的上下文。”这才是真正的解锁。

**Amir**：是的。如果你拥有这些，你就会得到更好的结果，你会产生复利效应，这就是你在这个世界竞争的方式。

## 十六、使用 Humbolytics 运行 A/B 实验

使用 Humbolytics，你可以直接进入 API，创建 API 密钥，获取你的属性详情和指令来访问 Claude。你会得到你通常会看到的基础分析：人们从哪里访问、流量、渠道等。如果你想运行付费广告活动，你可以在这里获得完整的归因，精确地看到哪些渠道在为你赚钱。你还可以有热图来查看滚动深度、人们点击了什么、完整体验。你甚至可以创建漏斗。
