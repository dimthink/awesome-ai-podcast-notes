## Nebula：像使用Slack一样与AI代理协作

对，这就是 **Nebula**，我一直这么叫它。你首先会注意到它看起来有点像 Slack，有这些频道。我的想法是，很多工作都发生在Slack这类消息工具里，我想模仿这种体验。只不过，这里的每个成员都是一个代理，可以帮助我完成工作。

我是一个工程师，花了很多时间在技术层面。我用像 **Cloud Code**、**Cursor** 和 **Codex** 这样的工具，让我的编码变得更强大、更有影响力。我想让所有其他类型的工作也有同样的感受。而对我来说，很多工作都存在于云端，比如我们使用的各种服务：**GitHub**、**Slack**、**Linear**、**Notion**、Google 的各种工具。很多工作就在那里发生。

我认为这种流程，感觉上和我开发时构建工程的方式很相似，但至少在AI方面，我感觉没有类似的工具来解决这些问题。所以，在很多方面，我认为它就像是**针对所有其他事情的“云代码”**——针对非工程任务，针对我想要完成的所有其他工作。

因此，我们模仿了 Slack。你有这些频道，可以创建各种东西。我们可以做一个测试，比如连接 Google Slides，向你展示一下你可以怎么做。

### 演示一：用自然语言创建和编辑PPT

假设我进入一个频道，说：*“我想连接到 Google Slides。你能为我创建一个新幻灯片吗？”*

这个想法的核心是：Nebula 就是一个代理。你让它做事，它会想办法去完成。它拥有整个互联网的知识，可以搜索，有浏览器，能查找文档。我们还赋予了它**编写代码的能力**。所以，即使你不知道如何写代码，它知道。它可以为你编写代码来帮助你完成任务，并且可以执行它。

在这个例子中，它确实创建了一个全新的演示文稿。我等会儿可以把它调出来看看。现在它已经创建了这个演示文稿，我不想直接操作它，而是让它开始修改和做一些事情。

比如说：*“我正在和 Greg 做一个关于 Nebula 的播客，这个播客的听众是关注构建和打造卓越公司的创始人/团队负责人。我想创建一个关于 Nebula 及其用途的三页幻灯片大纲。”*

**Greg：** 这部分，对，没错。这让我们得以一窥未来我们与应用程序交互的方式。

**Furkan：** 我认为是的。文本是我们非常熟悉的交互方式。不过我对音频、屏幕共享和实时同步也特别感兴趣。对我来说，下一步应该是，我不仅可以通过手机异步输入，还可以直接打个电话，花三分钟解释我的整个想法，然后 Nebula 就可以去后台工作了。

我们在后台构建了一个引擎，文本是输入，但未来肯定不止于此。你可以看到，它正在接管这个演示文稿，开始编辑。它说正在创建和更新第一页。后台有很多隐藏的操作，比如它创建了自己的待办事项列表，告诉 Google“嘿，把这一页的所有文本替换成这个”。这些都是你在编写程序时会做的事情，而它现在正在做。它为你执行API请求，让所有本应由工程师完成的事情发生。

现在它添加了所有这些内容，一个基本的幻灯片就做好了。然后我们再加一页：*“创建第四页，并添加一张能展示 Nebula 力量的图片，让这张图片占满整页。”*

这个指令就复杂了一点，对吧？它需要理解我们一直在谈论的上下文。它会创建第四页，生成一张图片。它会想出需要的提示词。在这个例子里，它生成了“AI代理网络的强大可视化，一个发光的节点”。它采纳了一些方向，如果我对图片的视觉效果有非常具体的要求，它也会听从并尝试生成。

这里的酷炫之处在于，很多这类事情你其实可以用 Claude 或 ChatGPT 做一部分，比如研究或网络搜索。但现在，**我们把它连接到各种服务上**，这是它能操作的第一步。它可以操作这些东西，连接到图像或视频生成工具。它还有 Python 环境，如果我给它更复杂的任务，它会变得更高级。

它确实在这里创建了一张图片，你可以看到一个文件被创建了。它有自己的文件系统，所以你不需要依赖于某台具体的电脑。它生成了这张图片。风格上，我不太喜欢，我可以告诉它：“我不喜欢这种风格，换个别的。”然后它会创建第四页，然后继续。

你可以看到它实际上正在写 Python 代码。再次说明，我们不是在做工程，但这个系统为了完成任务，自己写了代码来上传文件到 Google Slides。原本需要工程师来写集成代码，或者你去连接某个无代码服务，而现在它自己就能搞定。

看，这里出了点小故障，它会自我修正，说“让我试试别的办法”。这就是AI系统的一部分——**AI并不完美，但你可以告诉它“一直试到成功为止”**。它可以不断尝试，直到找到完成任务的路径，或者最终告诉你它实在没办法了。它正在第三次尝试，希望这次能成功。

看，它说创建了一个文件，有事情在发生。它写了一个 Python 脚本，这是 Nebula 为自己写的代码：“嘿，这是我们已有的文件，这是演示文稿，我想创建这个图片并去做。”这些代码原本需要人去写。它还在抱怨图片大小的问题。

好的，我们得到了一张图片。虽然不是之前想要的那张，但它尝试了。我可以再推动它一下，让它继续做，最终会实现的。

### 演示二：从“一次性任务”到“自动化工作流”

我认为真正的强大之处在于，工作不总是同步的。所以，我最想要的功能之一是：我可以在其他地方写脚本，但现在我可以做的是：

*“好的，现在每天添加一张关于 Nebula 的新幻灯片。我们希望在一周内把这个演示文稿扩展到15页。”*

它支持**调度功能**，对吧？它会说：“好的，你之前在这里做了什么？上下文是什么？让我把所有这些从这次工作中提取出来，创建一个‘配方’，这样我们就可以一遍又一遍地重复这个过程。然后写下触发器。”它写了一个 Cron 表达式（如果你懂技术，就知道 Cron 是什么）。

我们可以去“触发器”那里看看，这里有一个“每日 Nebula 演示文稿更新”。所以，**它每天都会添加一张新幻灯片**。目标是：从第四页开始，每天增加两页，一周内达到15页。这是执行步骤。它会在每天早上9点自动运行。现在，这就完全“甩手”了。

也许幻灯片不是最合适的应用场景，但如果我们想创建一个博客，我会这么做。我可以去编辑风格，我也可以给你看看那个工作流。我们可以快速创建一个博客，让它每天产生内容。我可以指定信息来源，比如从 Twitter 搜索，从互联网搜索，规定如何分析，博客图片的视觉风格，以及文本应该如何呈现。然后连接上我的 Ghost（博客平台）。之后每天告诉它**自我优化**。

**Greg：** 我想，对很多人来说，这就是梦想：创建一个“装在盒子里的生意”。也许它做的是内容，但也可以是每天的邮件简报。你在 X 上搜索，在新闻稿、Google 新闻上搜索，有自己的写作风格指南。然后每天推送内容，通过联盟广告或卖广告赚钱。当我看到这个时，我想到的就是这些。

**Furkan：** 完全同意。那其实也是我的起点之一：我为什么想创建这些所谓的“智能”脚本和代理？嗯，我想自动化我的工作。如果把这个想法推到极致，那就是我可能想要一个**自动化的公司**。我想创造一些方向和指导，然后与智能系统合作去实现它。

所以，我开始思考：我能做的最小的公司项目是什么？博客是我想到的一个。我可以构思内容方向，确定信息来源，想好视觉风格、编辑风格，以及连接到我的系统。这里我用了 Ghost。然后，我让它每天运行。之后我要做什么？我会查看搜索控制台和 PostHog 数据，优化关键词，然后每天都这么做，对吧？

这其实是我最开始的一个北极星目标：**能不能让一个系统完全自主地运行一个公司或一个项目？** 我确实有一个博客，作为构建这个系统的试验田。它现在已经连续运行了15天，每天发布三篇文章。这个博客现在每天大概有100次访问量。我花了不到30分钟进行设置，然后就搞定了。如果我花更多时间打磨，效果会更好。

## 机会与思考：在AI时代构建单人公司

**Greg：** 我能预感到评论区会有人说：“如果每个人都去创建这种‘盒子里的生意’和博客，那还有机会吗？会不会变得商品化？几年后，当每个人都拥有像 Nebula 这样的工具时，世界会变成什么样？”

**Furkan：** 当然。我认为随着我们越来越先进，一些东西会变得不再那么有趣。曾经有一段时间，仅仅拥有一个网站就是竞争优势。在 .com 时代，拥有一个网址本身就是优势，因为它创造了可访问性。然后，成为内容创作者或博主，在5年或10年前可能也是一种优势。

是的，接下来会发生的是，每个人都能用上这些工具，内容会泛滥。那么，**基础版本的东西就不再有用**。我必须把同样的逻辑应用到下一层级的竞争中。我的内容必须更优质，我的交付方式必须更卓越。也许我会转向视频，或者更高级的形式，然后我会构建代理来帮助我做到这些。

我可以创建一个**博客评论代理**，让它与主博客形成闭环。每次博客生成一篇文章，我可以对它说：“嘿，去问问评论代理，确保这篇文章在某某指标上能达到9分（满分10分），并告诉我如何评判。然后测试你的结果，下周再学习改进。”所以，我认为我们会不断向更高级的方向发展。

就现有的东西而言，比如我连接到这个 Nebula 演示用的 Ghost 博客，这是我的 Ghost 管理后台。我让它写一篇测试草稿，它已经在那里了，现在已经连上了系统。接下来就靠我的创造力了。

比如说：*“这个博客要专注于VR主题，特别是关于VR领域的新应用和游戏。你能研究一下顶级VR影响者的内容，找到最新最棒的游戏，并生成一篇文章和一张图片吗？”*

如果我的指令更高级，比如我给了一大段关于我在意的细节，如果我非常精细地去要求，那么结果会更好。你可以看到它开始并行工作，创建了一系列待办事项，同时进行着5个不同的搜索，把所有信息汇总过来。

现在的用户体验还比较“啰嗦”，我认为将来会改进。但我能看到所有正在进行的工作，这是**真实的工作**。我可以给它非常具体的关注领域。我可以告诉它重新设计我的博客，或者把它整合到我的网站里，并连接我的 GitHub。它能把所有这些部分连接在一起。

这就是我一直想要的能力：一个**存在于云端**的系统，拥有代码执行、文件系统等所有我本地开发环境里有的东西，但它生存在互联网上。我开始把它连接到不同的服务，让系统变得更高级，它开始管理多个调度、多项任务、多个优化。

就像我在开头说的，我们正处于一个**富足的时代**，但我们还没充分利用这种富足。我相信，一旦我们开始利用它，也许信息会变得太多，我们会回归到“有机”的时代，或者回归到“品味”的时代。所以，我不认为这个循环会结束：大量的东西涌现，然后大家转向最顶级、最受欢迎或更精雕细琢的事物。也许我们会变得更以人为本。

这个工具能帮我完成海量的事情。我可以管理我的日历，它有一个邮箱地址，可以代表我联系别人，管理对话。比如我在做潜在客户开发，或者计划一次旅行想联系酒店，我可以给它一套通常需要雇人完成的任务包。然后你只需要想办法让它做得更多。

我认为，如果你每月投入1000美元在一个AI团队上，听起来很多，但我相信，只要你把它指向正确的方向，它就能为你创造5000到10000美元的价值。而这部分——**指明方向**——是我们一直都需要做的，必须由人来完成。**这种创造力永远不会消失**。所有其他琐碎的工作，比如研究、写文章、生成图片、安排调度，这些都会消失，而且现在已经有能力做到了。这就是为什么我最终构建了这个系统：既然我手上有这种力量，我怎么能让每个人都拥有它？

**Greg：** 我同意。关键在于如何把它指向正确的事情，而这往往是最难的部分。对于单人公司来说，博客、新闻简报，这些都是机会。在未来的12到18个月里，如果你试图在AI时代建立一个单人公司，你还会想到什么其他的机会？你是如何思考和构思的？

**Furkan：** 是的。在一家公司里，博客可能只占不到2%的工作。我们有博客，但我们还有其他99件事要做。我在想所有这些工作流程：**哪里需要创造力？** 那是我必须把最聪明的人才放进去，用工具武装他们的地方。他们产出什么，这就是核心。

我之所以想让这个东西用起来像 Slack，就是因为这个想法很酷。我有了博客频道，现在我可以创建我的“潜在客户开发”频道，开始启动潜在客户开发工作：去研究这些，每天做，撰写邮件，发送东西。然后，我可能会想，我们需要做产品分析工作，所以我连接我的 PostHog。创建一个能分析我网站流量的代理。

我就是想，怎么能快速生成许多这样的代理，放在不同的频道里？就像这里显示的：“让我连接你的 PostHog。”这就是我的想法。我认为很多工作会变得更复杂，但很多都可以归结为**指令和工作流**。

我之前对无代码运动非常着迷。作为工程师，我和组织里的每个团队（市场、业务、销售）合作，帮助他们获得工程团队才有的能力。这需要大量工作，很难承诺资源去做这些事情。无代码和SaaS的出现真正解锁了这一点。我认为AI的下一步就是，**把它提升到公司的每一个角落**。我感觉我们正在进入一个阶段，这里不再有限制。问题只是：我们想让他们做什么工作？

所以，你看，我有我的博客频道，它正在生成所有这些内容。就在我们谈话的时候，评论应该也进来了。它生成了一张图片，尽管我们没给任何具体指示。我们绝对可以通过更精细的指令来提高编辑质量。它实际上为我做了研究，产出了内容。我可以直接说：“嘿，去发布这个。”看起来不错，那就发布吧。每天发三篇。

现在，我们的博客工作者开始工作了。接下来我们可能去处理分析代理。这个新代理会出现在聊天里，是一个代理创建器生成的。它正在构建定制的东西。
