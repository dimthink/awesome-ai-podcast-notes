## 详细内容

### 第一部分：为什么是 Hermes？OpenClaw 的三大痛点 `[对话节选：视频开头 - 问题陈述]`

**核心观点**

Imran 从 OpenClaw 迁移到 Hermes，因为 OpenClaw 存在三个无法忍受的缺陷：无内置记忆、频繁崩溃、以及 token 消耗完全不透明。

**深度阐述**

Imran 并不是为了尝鲜而使用 Hermes 的。他最初是一名 OpenClaw 用户，但在实际使用中几乎被逼疯。他用几乎无奈的语气描述了三个问题：

1. **没有内置记忆系统**：“我一直在告诉它做同样的事情，一遍又一遍。” 这是所有 AI 代理的通用痛点——模型本身没有长期记忆，每次对话都从零开始。你需要重复指令、重复提供上下文，甚至重复给出 API 密钥。这种体验让自动化失去了意义，因为你投入的“再训练时间”远远超过了节省的时间。
1. **稳定性灾难**：“我记得有一天，我不得不每小时重启一次网关。” Imran 的比喻很生动：“我感觉花在设置 OpenClaw 上的时间，比实际用它改善生活的时间还要多。” 对于想要“随手可用”的用户来说，这种不稳定性是致命的。代理本应是一个后台默默运行的角色，而不是需要你不断干预的“主子”。
1. **Token 消耗的黑箱**：Imran 提到了一个让所有预算敏感的用户都紧张的问题：“我不知道它用了多少 token，也不知道用在了哪里。” 没有透明度，意味着你无法优化、无法控制成本。尤其是当你使用付费 API 时，这种不确定性会让月度账单变得像一个黑洞。
> **原话**：“I felt like I spent more time setting up open claw than I was actually like using it to make my life better.” - [对话节选]
> **翻译**：“我感觉花在设置 OpenClaw 上的时间，比实际用它改善生活的时间还要多。”
**个人感受**

Imran 的真实体验触动了很多人。他并非工程师在挑剔技术细节，而是一个普通用户追求一个简单目标：让工具为生活服务，而不是反过来。他流露出的 frustration 是真实的——就像你买了一个高级咖啡机，结果每次煮咖啡前都要花半个小时清洗校准一样。

**延伸思考**

这三大痛点其实反映了当前 AI 代理软件的一个普遍困境：工具本身还在快速迭代，开发者倾向于先推出功能再考虑体验。但用户要的不是“能跑就行”的实验品，而是稳定、透明、可以依赖的“数字员工”。任何新工具如果要脱颖而出，必须在这三个维度上有根本性突破。Hermes 正是抓住了这一点。

**精华收获**

选择个人代理时，优先关注三点：**是否具备长期记忆**（能否从历史任务中学习）、**是否稳定可靠**（能否连续运行数天不用重启）、以及 **成本是否透明**（能否清晰看到 token 流向）。这三项比功能数量更重要。

### 第二部分：Hermes 如何完美反杀？ `[对话节选：三大问题 - 解决方案介绍]`

**核心观点**

Hermes 通过内置记忆系统、高度稳定性以及精细的成本控制，彻底解决了 OpenClaw 的三大痛点。

**深度阐述**

Imran 在描述 Hermes 的解决方案时，语气明显变亮。他逐一对应问题，给出了令人信服的回答。

- **记忆系统：主动写入 + 实时搜索**
每次任务成功完成后，Hermes 会自动写入自己的记忆存储。它使用标准的 SQLite 数据库——和普通 web 应用一样的机制，因此搜索速度快、结构清晰。更厉害的是，如果任务没有主动保存到记忆，它仍然可以搜索所有日志（logs）来回顾它曾经为你做过什么。Imran 举例说：“即使是你忘记保存到环境变量里的 API 密钥，只要曾经传递给代理，它也能从日志中找到。” 这意味着记忆不再是随机存取，而是有索引、可查询、可回溯的。

- **稳定性：一周不重启**
“我已经一个多星期没重启过 Hermes 了，”Imran 笑着说。对经历过 OpenClaw 每小时都要重启的用户来说，这个对比简直像从地狱到天堂。他承认自己曾像很多用户一样在不同代理之间跳来跳去，直到遇到 Hermes 才停下来。“我用了超过三周，在这个领域里这算很长了。” 注意，他说的是“在这个领域”——AI 代理工具每周可能都会出现新对手，能留在一个平台上超过三周，已经是一种强烈的认可。

- **成本透明度：模型选择 + 规划能力**
Hermes 集成了多个模型提供商，尤其是 OpenRouter。在 OpenRouter 中，你可以看到每种模型每百万 token 的具体价格（例如“Quen 3.6 Plus 的输入 token 成本约为 33 美分，输出约为 1.95 美元”）。Imran 强调：“这和 Sonnet 的价格差距几乎有十倍。” 同时他还指出 Hermes 有一个非常独特的省钱策略：“对于已知的重复任务，你可以让代理一次性写好代码，以后直接用代码执行，而不需要每次都调用 LLM 进行推理。这样后续所有运行都不消耗 token。” 这个思路本质上就是“先智能后确定”——用 AI 生成一次自动化脚本，然后用脚本无限次免费运行。

**个人感受**

Imran 在展示 Hetmes 的终端时，语气中带着自豪。他打开 Hermes 的命令行，界面整洁地列出了 40 多个内置工具和大量预装 skill。他特意提到，许多他在 OpenClaw 中需要手动寻找安装的技能，在 Hermes 中“开箱即用”：“我没有去技能市场下载，它们已经在那里了。” 这种“不用找”的体验是巨大的吸引力。

**延伸思考**

记忆系统的实现方式特别值得关注：它不仅仅是存聊天记录，而是写入结构化的数据库，并且具备日志回溯能力。这意味着 Hermes 的记忆不仅仅是信息仓库，更是一个可审计的工作流记录。对于需要“为什么它做了这个决定”的用户来说，这种设计提供了可追溯性。另外，成本控制中的“写代码替代重复调用”的思路，可以被扩展到任何高频率任务上，是 token 优化的核心方法论。

**精华收获**

- **记忆**：不是被动保存，而是主动写入 + 日志搜索双保险。
- **稳定**：开源项目但已经达到准生产级稳定性。
- **成本**：通过模型选择、使用 OpenRouter 的免费模型、以及任务代码固化，能实现 90% 以上的成本削减（Imran 后续会详细说）。

### 第三部分：安全性与运行模式 `[对话节选：安全提问 - 容器化选项]`

**核心观点**

Hermes 提供多种安全部署方式，包括 Docker 隔离、Modal serverless 以及裸机运行，还支持主动的安全审计。

**深度阐述**

Greg 提出了一个所有用户都会关心的问题：“如果我安装这些技能，怎么确保安全？” Imran 的回答非常具体：

首先，你可以要求代理对自己进行安全审计。“你可以问它‘这个设置安全吗？为什么？’它会检查密钥是否以明文存储、防火墙配置是否有漏洞等。” 这类似于元提示（meta-prompting），让 AI 自动检查环境。

其次，Hermes 原生支持多种运行环境：

- 如果你想隔离，可以塞进 Docker 容器，“和其他文件隔离开”。
- 如果想按需付费，可以部署在 Modal 上作为 serverless 服务。
- 当然，也允许在裸机上直接运行（Imran 自己就是这么干的，他承认自己“有点冒险”，但他通过每日更新和定期的自我审计来平衡风险）。
Imran 的自嘲很有趣：“我只是让它把整个设置弄得更安全一些。” 这其实给出了一个务实的原则：即使用户没有系统安全背景，也可以利用代理自身来增强保护。

**延伸思考**

安全审计的能力是很多用户忽视的增值功能。很多人担心把密钥放在本地不安全，但 Hermes 的方法论是：让 AI 成为你的安全顾问。这种“用 AI 管理 AI 风险”的思路很灵活。同时，三种运行模式覆盖了从技术小白（裸机简单）到专业人士（Docker 隔离）到企业级（serverless 零运维）的需求，显示了设计的周全。

**精华收获**

不要害怕安装代理，而是要学会用代理本身来检查安全性。另外，对绝大多数个人用户来说，裸机运行加每日更新就已经足够了。

### 第四部分：安装入门与模型配置 `[对话节选：安装命令 - 模型列表]`

**核心观点**

Hermes 的安装极其简单，一行命令即可；模型选择透明直观，大幅降低 token 成本。

**深度阐述**

安装步骤几乎是所有教程里最枯燥的部分，但 Imran 处理得非常清爽。

对于 Mac/Linux/WSL 用户：“只需要这一串命令，复制粘贴就行了。” 官方文档在 New Research 网站，初次使用可能需要安装 Xcode 开发者工具（Xcode-select –install）。他为此专门做了前置视频。当然，如果已经安装过，直接更新即可。

安装完成后，最重要的命令是 `hermes model`。Imran 打开一个运行在游戏 PC 上的 Hermes 实例，输入该命令后，屏幕显示了一大堆可用模型提供商，全部都是开箱即用，无需额外安装。这个列表包括 Anthropic（“之前你不能在 OpenClaw 上用 Anthropic 了”）、OpenRouter、新 Portal 等。

在 OpenRouter 中，他甚至展示了免费模型：“Nvidia 的 Neumotron 这周免费，如果你只想随便跑点儿什么，完全可以零成本。” 显然，利用免费模型作为基础和实验，是节省 token 最直接的方法。

**个人感受**

Imran 反复强调“都是开箱即用”时，带有一种“我不必折腾”的舒爽感。当他看到自己长时间没有更新的实例还显示“535 分钟落后”时，他自嘲地笑了——这已经是 beta 软件，但即便如此，也比以前稳定得多。

**延伸思考**

一行命令安装 + 40+ 内置工具 + 多模型选择，这样的开箱体验实际上是决定一个工具能否从 geek 圈走向大众的关键。OpenClaw 的繁琐安装曾吓退多少人？Heremes 显然在“易用性”上下了很大功夫。

**精华收获**

- 安装：一行命令，最多加个开发者工具。
- 模型选择：用 `hermes model` 查看所有提供商，优先选 OpenRouter 以获取价格透明度和免费模型。
- 成本控制第一课：习惯用便宜模型执行简单子任务，用贵模型处理关键推理。

### 第五部分：Android 手机上的个人代理 `[对话节选：Android 安装 - 用例探讨]`

**核心观点**

通过 Termux，您可以在一台廉价 Android 手机上完整运行 Hermes，获得一个始终在线、低功耗的物理代理设备，并能直接调用手机传感器和 SIM 功能。

**深度阐述**

这可能是整个视频中最激动人心的部分。Imran 在屏幕上展示了一个名为“Cookie Monster”的代理，它正在一台 Solana Seeker Android 手机上运行。当被问到“为什么要在手机上跑代理”时，他给出了令人信服的理由。

首先，需要两个应用：

- **Termux**：Android 上的终端模拟器，界面和 Mac 终端几乎一样。
- **Termux API**（从 F-Droid 开源商店获取）：给予代理访问手机物理传感器的能力——电池状态、Wi-Fi 切换、音量调节、拍照、振动马达、亮度调节等等。“基本上手机能做的事，代理都能做。”
安装 Hermes 的脚本和在电脑上相同，只是多几个前置步骤。一旦运行成功，你就拥有了一台**始终在线、超低功耗、自带移动网络和 GPS 的 AI 代理**。

Imran 列出了令人兴奋的应用场景：

- **SIM 卡集成**：放一张 SIM 卡，代理就能读取短信。这意味着可以自动化处理二步验证。他更具体地指出：“你可以想象一个世界，不用 Mac Mini，用一台便宜的 Android 手机就能搞定。”
- **直接发布社交媒体内容**：目前所有调度工具都会抑制触达率（因为通过 API），但通过 Hermes 直接在手机上操作，内容显示来自真实设备和 MAC 地址，相当于模拟了真人操作。你想运行多少个账户，就可以用多少台廉价手机。
- **无需 Mac Mini**：当 Mac Mini 缺货时，一台几百元的 Android 手机+Termux，你就有了一个便携的、随时带着的代理硬件。
**个人感受**

当 Imran 说到“我昨天才把这个搭建好”时，声音里充满了发现新大陆般的兴奋。这个方案不仅是技术可能性，更像是对“个人代理未来形态”的预告：代理不应该被锁在云服务器或特定硬件上，而应该可以部署在任何你拥有的计算设备上——哪怕是一部旧手机。

**延伸思考**

手机本身是移动、接近用户的硬件，自带摄像头、GPS、麦克风、网络。当代理能直接控制这些传感器，它的能力将超越电脑上的纯软件代理，例如“帮我拍张照并分析内容”、“当我的位置靠近超市时提醒我买东西”。这是物理世界和数字代理的结合点。对于创业者，这可能是一个新的计算形态——手机就是个人服务器，代理是操作系统之上的智能层。

**精华收获**

- Android + Termux + Termux API = 低成本、低功耗、随身携带的专用代理设备。
- 它能够接管 SMS 验证、直接操控手机屏幕、利用手机传感器，甚至伪装真人发帖。
- 对于不想买 Mac Mini 或服务器的人来说，旧手机是一个极佳的选择。

### 第六部分：真实生活中的自动化：从厨房到邮件 `[对话节选：个人用例 - Obsidian 整合]`

**核心观点**

使用 Hermes 解决生活中的小麻烦（例如“该做什么菜”）比宏大项目更能体现代理的价值，而且给代理时间积累记忆，它就会变得越来越有用。

**深度阐述**

Imran 分享了他在个人生活中最早获得的“大胜利”。他和妻子喜欢点外卖，但既不健康又费钱。于是他做了一件很多人想不到的事：他对着手机用语音笔记录了 8 分钟的 Telegram 消息，详细描述了自己冰箱和储藏室里所有食材。然后把这条消息发给了自己的 Hermes 代理。

“我说，从今往后每天，根据我的冰箱库存和我当前的健身目标，推荐三个菜谱给我。”代理做到了。这个看似微小的自动化节省了巨大的“决策脑力”。Imran 笑着说：“它听起来很小，但真的减轻了每天‘今天吃什么’的心理负担。”

另一个自动化亮点是邮件分类代理：每天早上自动扫描邮件，删掉垃圾、退订无用的订阅，然后生成一封摘要邮件，列出重要事项。这每天为他节省 30 分钟到一小时。

最关键的一点：**代理需要时间学习**。Imran 强调：“刚装好的 Hermes 不会变得亲密；但随着你每天使用它完成任务，它会把记忆写入数据库，开始理解你的模式。” 他甚至会在每晚问代理：“请告诉我一件或两件你明天可以为我建造的、让我生活更轻松的东西。” 代理会根据自身的记忆，提出新的自动化建议。这形成了“使用 → 记忆 → 建议 → 新自动化”的增长飞轮。

**个人感受**

谈到这些用例时，Imran 的声音里充满了满意。他不只是在讲功能，还在描述一种“被代理了解”的感觉——它知道你家有什么食材、你的锻炼目标、工作日程。这种 intimate（亲密）程度让他半开玩笑地说：“这段可能太私人了，剪掉吧。”Greg 马上接话：“不，这正是我们想要的。” 这恰恰说明，当代理真正融入了你的生活，它会产生一种情感价值。

**延伸思考**

“从语音记录冰箱内容开始”是一个极佳的新手引导策略。很多人面对空白的代理，不知道第一个指令该写什么。Imran 暗示了一个方法：不要试图一步到位构建复杂工作流，而是从你每天都需要花 5 分钟决定的微小决定入手。把你的“脑力税”清单写下来，交给代理一件一件处理。

**精华收获**

- 上手建议：挑一个你每天都会纠结的小事（比如吃什么、查看邮件、列出今日待办），立刻用它解决。
- 积累力量：头 7-20 天是关键的数据积累期，越往后代理越聪明。
- 元提示循环：“今晚为我建一个明天能用的工具”可以持续扩展你的自动化组合。

### 第七部分：Obsidian、G Stack 与技能生态 `[对话节选：Obsidian 展示 - G Stack 介绍 - 技能推荐]`

**核心观点**

Hermes 能通过 Obsidian 输出结构化的每日概览，并整合 G Stack 等外部技能，让代理成为你生活的中央仪表盘。

**深度阐述**

Imran 此前对 Obsidian 并不着迷，他以前把所有笔记丢进 Apple Notes。但当他发现 Hermes 可以自动帮他生成一个结构化的 MD 文件，他彻底转变了。他在视频中展示了类似这样的页面：“Obsidian 里有一个主文件，记录了本周所有重要的事情：今日待办、即将到来的旅行、工作项目、个人事项。” 而这些内容完全是代理每天自动整理好的。他不用手动编辑任何 Markdown 文件——直接讲话就能完成组织。

“如果我过去必须每天早上自己拼凑这些信息，我根本不会坚持。但现在代理做到了，Obsidian 就变得有趣了。” 这个过程体现了一个原则：**工具本身不改变习惯，但能消除摩擦的工具会改变习惯**。Obsidian 在代理出现前对 Imran 是“不想碰”的，有了代理后被激活。

**G Stack（Garry Tan 的 YC 方法论打包）**

Imran 将其比作“YC 式创业流程的开源化”：它把创业分解为每周验证流程、问出正确的问题、然后直接指导代码实现。以前只有 YC 创始人能享受到这种系统化指导，现在通过安装一个技能，任何人都能获得。对于不在硅谷的创业者，这相当于一个免费随身合伙人。

**技能生态**

Hermes 有预装技能（Apple 笔记、提醒、iMessage、Find My 等），但你也可以自己构建。Imran 说他自己写了银行账单分析和一些个人财务管理技能。他强调：“与其去找别人开发的技能，不如从你自己已经在付费的东西（比如金融、健康应用）入手，让代理为你定制工具。”

一个非常酷的例子：他加载了一个经典的“Eliza”心理治疗机器人的 Wikipedia 描述作为数据源，让代理用自然语言把它转化成一个 skill，从而能和“Eliza”在终端里对话。“那个程序（Eliza）最初是在 MIT 花了三年建成的，但现在你只需要告诉 Hermes ‘把这篇文章内容变成一个技能’，它瞬间就能做到。” 这展示了如何把外部知识库快速工具化。

**个人感受**

当 Imran 打开 Obsidian 文件时，你能感受到一种自豪——一个原本杂乱无章的笔记系统在代理的帮助下变得井井有条，就像有一个私人秘书在每天早上给你一份打印好的日报。他反复使用“clean”（干净）这个词来形容效果。

**延伸思考**

Obsidian 的例子揭示了未来个人信息管理的核心模式：代理作为数据编排者。你不直接管理文件夹，而是让代理根据上下文自动生成结构。G Stack 则展示了如何把一个机构的隐性知识（YC 的工作方法）固化成一个 AI 技能，任何人都能带在身上。这是一种知识普惠。

**精华收获**

- **必装技能**：Obsidian skill（用于笔记和组织）、G Stack（如果做创业）。
- **自定义技能思路**：想想你日常用的所有付费服务（银行、健身房、Spotify），让代理帮你提取数据、生成报告。
- **技能创建捷径**：给代理一段文章或文档，让它直接变成一个技能。

### 第八部分：代理的架构思维：子代理与优化陷阱 `[对话节选：多个代理 - 工作与个人分离]`

**核心观点**

虽然可以创建多个子代理分别负责不同领域，但最优方案是只设一个主力代理，最多加一个用于分离工作与个人；避免过度定制，而应该专注使用。

**深度阐述**

Imran 在屏幕展示了多个命名的代理（他的都用 Muppets 名字）。但他说：“我用四个是因为我脑袋里爱折腾，但实际上最优化是用一个或两个。” 理由有两个：第一，子代理可以利用不同的模型分配任务。例如，你可以给简单的邮件分类子代理分配一个便宜的模型，给决策性任务分配高级模型，借此节省费用。第二，保留一个个人和一个工作代理的分离，能让生活更清洁。

但重点在于：**不要陷入“定制化陷阱”**。Greg 接过话头：“有很多人沉醉于定制代理本身，而不是用代理完成任务。” Imran 表示赞同：“代理就像 90 年代的改装车文化——你可以去找各种零件装上。但要记住，改装本身不是技能，用改好的车去做什么才是。” 这种观点非常清醒：工具的价值不在于它有多少可配置旋钮，而在于它每天为你创造了多少价值。

**个人感受**

Greg 说“不要成为一个艺术家”，Imran 笑。他们其实在批评目前社区里的一种风气：把调校代理当作最终目的，而非达成目的的手段。这种批评是及时的，尤其对于技术向的爱好者来说。

**延伸思考**

这和低代码平台的演化相似：人们很容易被搭建界面吸引而忘记真正要解决的问题。未来最有效的代理用户，将是那些能用最少配置产生最大产出的人——他们理解“完美是完成的敌人”。

**精华收获**

- 架构建议：1 个主力（或 2 个：工作/个人），而不是 10 个。
- 给子代理分配合适的模型：简单任务用便宜模型，关键任务用高级模型。
- 核心态度：花费时间在配置上，不如花费时间在使用和迭代任务上。

### 第九部分：技能库与元提示黄金清单 `[对话节选：提示示例 - Cron Job 设置 - 结尾]`

**核心观点**

掌握几组关键的元提示（meta-prompts），就能让 Hermes 从被动工具变成主动顾问。

**深度阐述**

Imran 在 Greg 的追问下现场编写了几条他认为每个人都应该对代理说的话。我把它们整理成可用于实践的清单：

1. **“我最近一直在拖延什么事情？”**  （前提是你让代理看到了你的待办；它能从历史记录中发现被你忽略的任务）
1. **“今天最重要的事情是什么？”** （基于你的目标记忆）
1. **“我每天都在重复做的哪些事可以自动化？”** （关键问题：识别重复任务并转为 cron job 或代码）
1. **“你今晚可以为我建造一个什么工具，让我明天更轻松？”** （鼓励代理主动创新）
1. **“今天有没有什么重要的事情被我遗漏了？”** （检查疏忽）
1. **“根据我过去的工作，有没有什么应该被自动化的？”** （扩充自动化的雷达）
他还特别提到 cron job（计划任务）的设置对许多非技术人员是个障碍。但通过元提示把这种需求转给代理：“帮我检查我的日常工作，并为你认为可以自动化的任务设置 cron job。” 代理就会代劳。

**延伸思考**

元提示的本质是**教会代理如何观察和优化你的使用模式**。一旦你养成了每晚或每天问这些问题的习惯，你就等于建立了一个反馈回路：执行 → 反思 → 自动化 → 更多时间 → 执行更多。这远比零散地临时命令更强大。

**精华收获**

- 元提示是解锁代理潜力的钥匙。
- 把这 6 个问题做成每日清单，一周后你就会看到自动化清单自动扩展。

## 精华总结

- **记忆系统**是 Hermes 区分于 OpenClaw 的核心：自动写入 + 日志回溯，让代理持续变聪明。
- **稳定性**足以令人放心运行一周以上，无需手动干预。
- **成本控制**通过模型选择（OpenRouter）、利用免费模型、以及“一次编码无限执行”的策略，可削减 90% 以上 token 开销。
- **Android 方案**将旧手机转化为专用代理设备，能读取短信、控制传感器，打开新场景（如模拟真人发帖）。
- **上手建议**：从一件每日花费你 5 分钟决策的小事开始（如吃什么、整理邮箱），让代理积累记忆。
- **元提示**：每天问代理“今天最重要的事”、“可以自动化的任务”等，构成自我进化的循环。
- **态度**：花时间使用代理，而不是花时间配置代理。最终价值在创造，不在安装。
