### GTM工程：从克莱·舍基到AI代理的进化史 `[00:03:30-00:06:00]`

**核心观点**

“GTM工程”这个概念最初由 Clay.com 创造，用来描述通过数据清洗和级联工作流实现自动化销售 outreach 的工作者。但在AI时代，这个概念正在被彻底重塑。

**深度阐述**

当 Greg 坦言自己并不真正理解“GTM工程”（Go-to-Market Engineering）这个 buzzword 时，Cody 给出了一个令人意想不到的起源故事。这个词最初诞生于数据 enrichment 平台 Clay.com，用来描述那些擅长设置复杂自动化工作流的专家——他们会设计一系列数据清洗、信息补全的“级联”操作，然后通过这些数据触发个性化的邮件、Slack 消息甚至电话 outreach。

但 Cody 指出，这个概念正在经历一场剧烈的进化。“基本上，我现在把一切曾经需要我动手敲键盘的‘中间工作’，都交给某种代理框架去处理。”他说这话时，语气中透露出一种近乎狂热的兴奋。无论是 Claude Code 还是 Cursor 的 Composer，这些工具正在彻底改变他的工作方式。

他的工作流程被重构为三个简单步骤：**产生想法 → 交给 Claude Code → 打磨最终成品**。这种模式的魔力在于，它让 Cody 能够以前所未有的规模去执行那些曾经“人力不可及”的任务。

**个人感受**

Cody 笑着承认：“所以我基本上又不怎么睡觉了。”这句话背后是一个创业者对效率近乎偏执的追求。当他描述自己如何在30分钟内完成“创建100个Facebook广告 → 发布 → 搭建追踪看板 → 分析数据 → 自动关停低绩效广告 → 将高绩效广告移至新广告组并单独分配预算”这一整套闭环时，你能感受到他语气中的那种“我发现了新大陆”般的狂喜。

**延伸思考**

这种工作流的重构，其实映射了人类与机器协作关系的第三次跃迁。第一次是工具的发明，让我们能做之前做不到的事（如锄头）；第二次是机器的发明，让我们能更快地做重复性的事（如工业生产线）；而第三次是AI代理的发明，让我们能够“委派”思考性的中间工作。我们不再需要亲自执行每一个步骤，而是成为定义目标和验收标准的那个人。

### 搭建你的“指挥中心”：环境文件与API优先思维 `[00:07:00-00:09:30]`

**核心观点**

未来软件采购的核心标准不再是漂亮的用户界面，而是API的健壮程度。你需要一个存储所有API密钥的环境文件，这是你与数字世界交互的“中枢神经系统”。

**深度阐述**

“第一步，创建一个文件夹。”Cody 的演示从一个极其简单但又极其根本的动作开始。他命名自己的文件夹为“Graph Growth Agents”，这个文件夹里存放着他所有的“数字资产”。

最关键的文件是 `.env`（环境文件）。在这个文件里，Cody 存储了所有服务的API密钥：Intercom、SendGrid、HubSpot、Clay、Perplexity、Facebook Ads、 Million Verifier、Instantly……这几乎涵盖了他日常使用的全部营销工具栈。他戏谑地说，为了演示安全，他不能展示真实的文件内容，但这份清单本身就足够令人震撼。

`[00:08:10]`

> "This is actually how I'm thinking about everything I do now and like how I buy software in particular is how robust the API is. It's funny. I was talking to a friend recently and he's like, 'If you're looking at Salesforce versus HubSpot right now, Salesforce, even though it's like historically a more clunky CRM, it's actually the better product for this AI foundation because it has a more robust API.'"
> （“这实际上是我现在思考所有事情的方式，尤其是我采购软件的标准——就是看它的API有多强大。有趣的是，我最近和一个朋友聊天，他说：‘如果你现在在Salesforce和HubSpot之间做选择，尽管Salesforce历来是一个更笨重的CRM，但对于这个AI基础架构来说，它实际上是更好的产品，因为它有更强大的API。’”）`[00:08:30]`
这段话揭示了一个深刻的行业转变。在过去，SaaS产品的核心竞争力是用户体验、界面设计和品牌。但在一个你主要和AI代理交互的世界里，UI变成了“nice to have”，而API才是“must have”。你的代理能否24小时不间断地工作，取决于它能否通过API顺畅地读写数据、触发动作。

**精华收获**

- **建立你的API密钥库**：把你所有常用工具的API密钥集中管理在一个环境文件中。这是你成为“代理骑手”的第一步。
- **API优先采购**：在选择新工具时，把API的完整性和文档质量作为首要决策因素，而非仅仅是界面美观度。
- **本地开发环境**：在本地文件夹中组织你的所有“代理项目”，就像软件工程师管理代码仓库一样。

### 多线程工作流：一个代理骑手的日常 `[00:10:00-00:18:00]`

**核心观点**

真正的效率不是单线程地完成一个任务，而是同时启动十几个代理，让它们并行工作，而你只负责“监督”和“启动下一个”。

**深度阐述**

接下来的20分钟，Cody 进行了一场令人眼花缭乱的现场演示。他同时打开了多个终端窗口，每个窗口都运行着一个独立的 Claude Code 实例，每个实例都在执行不同的任务。Greg 在旁边看得目瞪口呆，称其为“代理骑手”（agent jockey）。

**工作流A：LinkedIn 自动回复** `[00:10:30]`

Cody 最近发布了一个关于“邮件分类”的 Notion  giveaway 文档，很多人在评论区留言索要。他启动了一个预先构建好的“LinkedIn 回复软件”，指定了 Notion 文档的 URL 和 LinkedIn 帖子的 URL。代理开始自动扫描所有留言，并为每条留言生成个性化回复，附上 giveaway 的链接。Cody 只“babysit”（照看）了最初几秒钟，确认方向正确后，就让它自己在后台运行。

**工作流B：批量 Facebook 广告生成器** `[00:12:00]`

在第一个代理运行的同时，Cody 转向下一个窗口，开始构建一个新的工具。他通过语音（转录成文字）向 Claude Code 下达指令：

- 创建一个1080x1080像素的图片广告生成器。
- 需要能根据他提供的示例广告，搭建一个UI模板。
- 需要能批量生成文本变体（标题和正文）。
- 输出需要是包含PNG图片的ZIP文件。
- 前端用React组件实现，图片生成用 `html-to-canvas` 库。
**工作流C：播客 outreach 自动化** `[00:15:00]`

Cody 再次打开一个新的终端窗口，构建第三个代理。这个代理的任务是：通过 Refonic 的API爬取“营销”分类下的所有播客节目，获取主持人邮箱；将邮箱列表发送给 Million Verifier 进行验证；然后将验证通过的邮箱导入 Instantly 的指定邮件营销活动。这个代理一旦完成并部署到服务器上，就能持续不断地为他寻找新的播客露面机会。

**个人感受**

当 Cody 在不同窗口间切换，检查每个代理的进度，并为卡住的代理提供新指令时，他显得游刃有余。他说：“这就是我现在的工作方式。如果我能把它们自动化，如果我能搞清楚‘好，这就是你可以专注的特定工作流’，我就会把它部署到服务器上，让它永久运行。”这不再是“工作”，而是一场精心 orchestrated 的交响乐指挥。

**精华收获**

- **并行启动**：不要等待一个任务完成再开始下一个。AI代理是你的“数字员工”，你可以同时雇佣很多个。
- **转录优先**：用语音转文字工具（如 SuperWhisper）来下达指令，速度远超键盘输入，让思考到执行的延迟降到最低。
- **上下文切换的适应**：同时管理十几个代理需要练习，但大脑的可塑性很强，几周后你就会习惯这种新的工作节奏。

### 从创意到上线：广告创意的无限裂变 `[00:20:00-00:28:00]`

**核心观点**

利用AI代理，你可以实现广告创意的“无限裂变”——从竞争对手的广告库中汲取灵感，从Reddit等社区挖掘真实痛点，然后批量生成成百上千个广告变体，而成本几乎为零。

**深度阐述**

当之前启动的“批量广告生成器”完成后，Cody 开始演示如何实际使用它。他需要为一家数据可视化工具（类似 Looker Studio）生成广告创意。

第一步是**挖掘真实痛点**。Cody 没有凭空想象，他让代理使用 Perplexity API 去 Reddit、YouTube 和 Twitter 上搜索增长营销人员在数据报告方面遇到的真实问题：“无法统一数据”、“数据分析太复杂”、“无法获得工程资源支持”等等。

第二步是**生成批量创意**。获取痛点列表后，他让代理基于这些痛点，调用之前构建好的广告生成器，批量生成所有文本和图像的组合。`[00:24:00]`

> "This that I'm doing costs me nothing like it's literally you know maybe a thousand tokens to do all of these like these generations and so this that's a reason for it. So, it's like I can go we could go and create a thousand ad variations right now, G. And like this literally costs nothing."
> （“我现在做的这些几乎不花什么钱，生成所有这些可能只需要几千个token。所以你看，我们可以马上去创建一千个广告变体，Greg，这基本上零成本。”）`[00:24:10]`
第三步是**批量上传**。生成完所有广告创意的图片和文案后，Cody 再次启动另一个代理，通过 Facebook Ads API，将这些广告草稿批量上传到他在 Facebook 广告管理器中预先创建好的一个广告组中。

**延伸思考**

这里出现了一个有趣的讨论点。Greg 问到：“为什么不用最好的图像生成模型（如Nano Banana Pro）来制作广告？为什么用代码生成？”Cody 的回答体现了深刻的营销策略思考。他区分了“寻找获胜角度”和“制作最终创意”两个阶段。在前期测试阶段，快速、低成本地生成大量包含不同痛点和文案的“ugly ads”，比一开始就追求视觉效果精美的“hero ads”更重要。目标是找到那个 ROI 从1美元到150美元的爆款公式，然后再投入资源进行精美化。

**精华收获**

- **痛点来源**：真实的广告创意来源于真实的用户声音。用AI代理去社交媒体上挖掘未被满足的需求。
- **低成本试错**：用代码生成广告变体是测试市场反应的最高效方式。通过 API 批量上传，可以在几分钟内启动数百个 A/B 测试。
- **两阶段策略**：先跑通“产品-市场契合点”（PMF）的语言和角度，再投入资源进行高质量视觉呈现。

### 数据闭环：自动分析与智能优化 `[00:28:00-00:34:00]`

**核心观点**

生成广告只是开始。真正的魔法在于构建一个数据闭环：实时追踪广告表现，自动关停低绩效广告，自动为高绩效广告加预算，并通过自然语言对话随时获取业务洞察。

**深度阐述**

广告上线后，Cody 的工作流进入了最精彩的环节——自动化分析与优化。

首先，他构建了一个**实时数据看板**。他给 Claude Code 一个指令：“用这个广告组ID，创建一个看板。用折线图展示点击量随时间的变化，同时包含成本和CPC。另外添加一个显示总花费和总点击的卡片，再用条形图展示不同年龄段的广告展现量。”几分钟后，一个定制化的数据看板就搭建完成了。

接着是**自动化优化**。Cody 启动了另一个代理，让它通过 Graph MCP（他公司构建的实时数据管道）拉取刚刚那个广告组的数据。指令非常明确：“找出 CPM（千次展现成本）最高的低绩效广告。”代理返回了一份列表。Cody 随后下达了第二个指令：“用 Facebook Ads API 把这些广告关掉。”

`[00:32:30]`

> "And so it went and it paused those ads for me. And so it just just to walk through what we just did to kind of reiterate this. We just did ideation. We just did bulk ad creation. We just analyze the data for the performers. We just turn those off and on based on that. And at this point you're probably starting to have the epiphany like, oh, I can just turn this in a into a repeatable process."
> （“它就去帮我暂停了那些广告。我们来快速回顾一下刚才做了什么：我们做了创意构思，批量生成了广告，分析了表现数据，然后根据数据关停了低绩效广告。这时候你可能会恍然大悟：哦，我可以把这个变成一个可重复的自动化流程。”）`[00:32:50]`
Cody 进一步描绘了这个流程的理想形态：一个持续运行的“测试”广告组，不断接收新创意；一个每日运行的定时任务（cron job），自动关停低绩效广告；自动将高绩效广告移动到新的、拥有独立预算的“胜出者”广告组。整个过程无需人工干预。

最后是**自然语言交互**。Cody 展示了如何通过手机上的 Claude App 与 Graph MCP 对话：“用 Google Analytics 4 告诉我，昨天有多少新用户访问了网站主页。”每天早上，他都能像和一个数据分析师交谈一样，获取定制的业务简报。

**精华收获**

- **实时数据管道**：不要直接通过API连接广告平台做分析，你会遇到分页和速率限制问题。构建一个数据仓库和实时数据管道（如 Graph MCP）是规模化分析的基础。
- **定义优化规则**：把你的优化逻辑（如“关停 CPM 最高的10%广告”）写成指令，让代理自动执行。
- **对话式 BI**：未来商业智能（BI）的终极形态是对话。能用自然语言提问并从实时数据中获得答案，这种能力将成为决策者的标准配置。

### 临时软件：未来工作的新范式 `[00:34:00-00:37:00]`

**核心观点**

我们正在进入“临时软件”（ephemeral software）的时代。为了一个一次性任务，你可以即时创建UI、即时生成数据库，用完即弃。这彻底改变了我们解决临时问题的效率。

**深度阐述**

Cody 分享了一个让他自己都感到“aha moment”的经历。前几天他需要做一些数据分析工作。在过去，他会下载数据到Excel，然后花几个小时做数据透视表。但这次他换了一种方式：

1. 直接通过 Railway 的 API 即时创建了一个 Postgres 数据库。
1. 让 Claude Code 把 URL 中的数据直接灌入这个新数据库。
1. 和 Claude 一起在这个数据库上做联合分析。
1. 将分析结果推送到目标位置。
1. **用完就把数据库销毁**。
原本需要5小时的数据清洗和分析工作，他在20-30分钟内就完成了。这种“即时创建，用完即焚”的能力，正是未来工作的缩影。Cody 称之为“随需应变的UI，随需应变的数据库，随需应变的软件”。

**延伸思考**

这背后揭示了一个更深层的哲学转变：**从“拥有”软件到“调用”软件**。在过去，为了解决一个问题，你需要购买（或构建）一个永久性的软件工具。在未来，你可以像调用一个函数一样，为单一任务临时生成一个完全定制化的工具。任务完成后，这个工具就消失在数字空间中，不留痕迹，也不占资源。这种灵活性将释放巨大的创造力。

**精华收获**

- **按需创建资源**：学习使用 Railway 或 Vercel 等平台的 API，它们允许你通过代码动态创建和销毁服务器、数据库等基础设施。
- **组合式工作流**：将“临时数据库”+“AI分析”+“结果输出”组合成一个标准工作流，用于处理各种一次性的复杂数据处理任务。

### 赢家与输家：AI时代的职业重构 `[00:37:00-00:45:00]`

**核心观点**

AI代理将彻底重塑劳动力市场。赢家将是那些能用AI武装自己领域知识的“超级个体”；输家将是那些只做重复性“中间工作”的岗位。短期内，我们可能会经历剧烈的就业震荡。

**深度阐述**

在演示的后半段，Greg 和 Cody 进行了一场关于未来的深刻对话。Greg 认为，Cody 展示的这套东西预示了“自主营销”的未来。那些曾经需要人类完成的“工作”（job to be done），现在可以被AI代理无缝地 stitching 在一起。

那么，谁是赢家，谁是输家？

Greg 认为，赢家是“一人公司”和小团队。他们能用极低的成本撬动巨大的营销产出。而对于那些年薪10万美元的营销负责人来说，如果他们能掌握这套技能，证明自己一个人能顶十个人用，那么他们完全有理由要求三倍薪水。这是一个巨大的价值创造机会。

但 Greg 也直言不讳地指出了残酷的一面：**会有真实的、剧烈的岗位流失**。他类比了工业革命初期英国经历的动荡——旧岗位被取代，新岗位被创造，但中间的过渡期充满了混乱和痛苦。

Cody 补充了一个身边的故事：一个创始人朋友告诉他，打算解雇50个人，这占了他团队的70%。因为他认为可以用“代理集群”（agent swarms）来自动化他们所有的工作。而 Cody 意识到，他自己构建的那些并行运行的代理，本质上就是一个“代理集群”。

**个人感受**

面对这种未来，Cody 的语气中既有兴奋，也有一种清醒的敬畏。他特别强调了“领域知识”在新范式下的决定性作用。

`[00:41:30]`

> "Especially if you have domain knowledge is the other thing that I'm finding is like just because you can like it it can be built doesn't mean that you can build it because you don't have the vocabulary... If you can incorporate these tools into what you're doing for work and have that domain expertise, that knowledge that's like on top of that, that actually is what makes you like incredible."

他用自己和联合创始人 Max 做对比。Max 是技术出身，他能用极其精确的技术词汇向编程代理描述问题，得到的输出质量是 Cody 永远无法企及的。同样，如果一个拥有20年平面设计经验的设计师，能用精准的行业术语描述“我想要一种老式电视的噪点纹理”，AI就能瞬间完美呈现。而一个外行再怎么描述，结果也可能惨不忍睹。

**精华收获**

- **超级个体公式**：领域知识 + AI工具应用能力 = 不可替代的价值。
- **API即护城河**：掌握API交互能力，将成为区分新旧两代职场人的关键分界线。
- **拥抱临时性**：未来的工作模式将是项目导向、工具即用即弃的。适应这种流动性，放弃对固定工具和固定流程的执念。
## 精华总结

这场对谈和演示的价值远不止于介绍几个工具。它描绘了一幅正在成型的未来工作图景：

1. **从执行者到定义者**：你的核心价值不再是“如何做”，而是“做什么”和“为什么做”。你把“如何做”的中间过程交给了AI代理。
1. **API是新时代的氧气**：在一个由代理驱动的世界里，软件必须有呼吸的“鼻孔”——强大的API。这是你构建自动化机器的基本积木。
1. **复合型人才的胜利**：仅懂营销不够，仅懂技术也不够。真正稀缺的是那些能用精准的领域词汇调动AI代理，解决复杂问题的“翻译官”和“指挥家”。
1. **自主营销成为现实**：Cody 演示的这套从洞察、创意、发布、分析到优化的完整闭环，完全可以被设定为永续运行的自动化流程。24/7 工作的营销团队，不再是梦想，而是现实。
正如 Cody 最后所说，未来12个月即将发生的变化，很多人可能还没有真正意识到。但现在，你已经看到了它的模样。你可以选择被动等待，也可以选择像 Cody 一样，打开你的终端，创建你的第一个环境文件，开始成为你自己的“代理骑手”。
