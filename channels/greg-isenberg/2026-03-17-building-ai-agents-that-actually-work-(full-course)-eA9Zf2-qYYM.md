### 从“乒乓球”到“执行者”：重新理解 AI Agent `[00:00 - 03:15]`

**核心观点**

AI Agent 与传统的聊天模型有本质区别：聊天模型是“问题到答案”的乒乓游戏，而 AI Agent 是“目标到结果”的自主执行者。

**深度阐述**

视频一开场，Remy 就点破了一个核心困惑：AI Agent 到底是什么？它和我们现在天天用的 ChatGPT、Claude 有什么不同？为了讲清这个关键区别，Remy 用了一个绝妙的比喻。

想象一下你在打乒乓球。你打过去一个球（提问），对方回过来一个球（回答）。你再打过去一个追问，对方再回一个。这就是传统的聊天模型——**问题到答案**，一来一回，所有的工作流都由你在中间推动。你需要不断思考下一个问题是什么，然后把模型的回答拼凑起来，自己去完成最终的任务。

而 AI Agent 则完全不同。它更像一个你雇佣的、有自主能力的员工。你不需要跟它一来一回地打乒乓球，你只需要给它一个**目标**，它就能自己去想办法，直到把最终**结果**放到你面前。这就是 Remy 给出的清晰定义：**聊天是“问题到答案”，而 Agent 是“目标到结果”**。`[03:15]`

那么，当 Agent 拿到一个目标后，它内部到底发生了什么？Remy 引入了“**Agent 循环**”这个概念。这个循环由三步构成：**观察、思考、行动**。`[03:48]`

为了让你亲眼看到这个循环，Remy 现场做了一个演示。他给三个不同的 Agent 平台（Claude Code, Codex, 和 anti-gravity）下达了同一个任务：“**为 Greg Eisenberg 建立一个极简主义的作品集网站，并把它发布到网上预览。**” `[09:05]`

1. **观察**：Agent 接收到任务，首先会检查自己拥有什么。工作区是空的？没有关于 Greg 的任何信息。
1. **思考**：Agent 会想：“要建 Greg 的网站，可 Greg 是谁啊？” 于是它决定，第一步应该是去研究 Greg Eisenberg。
1. **行动**：Agent 调用一个搜索工具（比如 Perplexity）去网上搜索 Greg 的所有信息。
当搜索结果返回后，Agent 又回到了**观察**步骤，它现在拥有了“任务”和“研究资料”。然后它再次**思考**：“好了，我知道 Greg 是谁了，下一步该做什么？我应该先写一个建站计划。”于是它再次**行动**，生成一个计划文档。接着又**观察**（有了计划）、**思考**（下一步是写代码）、**行动**（编写 HTML/CSS/JS 文件）……这个循环会一直持续，直到 Agent 检查自己的输出，确认网站已经建成并成功发布，满足了你在任务中设定的所有参数（如极简风格、可预览），它才会停止，并最终把结果呈现在你面前。`[04:00-07:00]`

这个循环的魔力就在于它的自主性。你不再需要像保姆一样盯着它，告诉它每一步怎么做。你只需要设定好目标，它就会像一台自动驾驶的汽车，自己导航、自己避障、自己到达目的地。

**个人感受**

Remy 在演示这个循环时，语气里充满了兴奋和耐心。他反复切换着不同的软件界面，像个兴奋的孩子在展示自己的新玩具。你能感受到他想要传递的不仅仅是一个技术概念，更是一种“哇，这太酷了”的惊喜感。特别是当他在 Claude Code 里看到 Agent 一步步列出自己的思考过程——“首先，Greg Eisenberg 是谁？”、“现在我需要创建 HTML 文件”、“最后，检查网站是否完成”——时，那种将 AI 的“思考过程”可视化的体验，无疑拉近了我们与这个黑箱的距离。

**延伸思考**

这个“观察-思考-行动”的循环，其实和我们人类解决问题的过程惊人地相似。我们每天做任何复杂任务，都是在不断重复这个循环：看看有什么（观察），想想该怎么做（思考），然后动手干（行动），再检查结果（观察），调整策略（思考），继续干（行动）... 理解了这一点，你就理解了 Agent 的本质：它是对人类工作流程的一种模拟和自动化。

### Agent 的四大组件与“驾驶”比喻 `[07:35-09:05]`

**核心观点**

一个完整的 AI Agent 由大语言模型、Agent 循环、工具和上下文四部分组成。而承载这一切的平台，就是“Agent 驾驶舱”（Agent Harness）。学会使用一个，就能轻松上手所有。

**深度阐述**

在理解了 Agent 的核心工作流程后，Remy 将其拆解为四个基本组件：

1. **LLM（大语言模型）**：这是 Agent 的“大脑”，负责思考和推理。你可以把它想象成 Claude、GPT 或 Gemini 这样的模型。
1. **The Loop（循环）**：这是 Agent 的“驱动程序”，让它能持续工作直到任务完成，而不是只回复一次就停下来。
1. **Tools（工具）**：这是 Agent 的“手脚”，让它能连接外部世界，比如访问网络、发送邮件、操作你的电脑文件。
1. **Context（上下文）**：这是 Agent 的“记忆和工作台”，包含了你给它的所有信息、文件和历史对话。
而将这些组件整合在一起，让 Agent 循环得以顺畅运行的那个平台，就叫做 **Agent 驾驶舱**。`[08:35]` 现在市面上所有热门的 AI Agent 产品，无论是视频中演示的 Claude Code、Codex、anti-gravity，还是后面提到的 OpenClaw、Manus，本质上都是不同的 Agent 驾驶舱。

为了让这个概念更接地气，Remy 用了一个非常生动的比喻：**学开车**。`[13:30]` 我们今天要学习的，就是如何驾驶汽车——如何踩油门（下达指令）、如何打方向盘（理解 Agent 循环）、如何使用刹车（设置安全边界）。一旦你掌握了这些核心的驾驶技能，你就能轻松驾驭任何品牌的汽车。有的车（比如某个驾驶舱）可能座椅加热更好（集成了更多工具），有的车巡航控制更智能（自动化程度更高），但基本的驾驶逻辑是完全相通的。你今天在这里学会用 Claude Code，明天就能无缝切换到 Codex 或 OpenClaw。

这个比喻极大地降低了学习门槛，也点明了学习的关键：**理解概念，而非死记硬背某个软件的操作**。你是在学习驾驶，而不是学习开某一辆特定的车。

### 像 onboarding 员工一样，给你的 Agent 做入职培训：agents.md `[19:15-26:30]`

**核心观点**

要让 Agent 高效工作，首先要像 onboarding 新员工一样，通过一个名为 `agents.md`（或其他类似名称）的配置文件，给它提供关于你、你的业务和工作偏好的完整上下文。

**深度阐述**

Remy 提出了一个极具启发性的观点：**构建 Agent 的最佳方式，就是像 onboarding（入职培训）一个真实员工一样去对待它。**`[19:45]` 想象一下，如果你新招了一个行政助理，你会第一天就直接让他去处理客户邮件吗？当然不会。你会先花时间介绍你自己、公司的业务、核心客户、常用工具以及你的工作习惯。

AI Agent 也一样。如果你不给它任何背景信息就直接问“帮我写一封开发信”，它就会一脸茫然地反问：“你是做什么的？目标客户是谁？语气要怎样？” `[20:45]` 这正是从聊天模型转向 Agent 时一个重要的思维转变。聊天模型（如 ChatGPT）为了方便，内置了自动记忆功能，但它会把你关于不同公司、甚至个人生活的所有对话都混在一起，导致上下文混乱。而 Agent 则要求你手动、精确地给它提供上下文，这反而是一个巨大的优势——因为它确保了信息的纯净和可控。`[22:20]`

这个“入职培训”的关键，就是一个名为 `agents.md` 的纯文本文件。`[23:25]` 它本质上就是一个“**系统提示词**”，作用和你在 ChatGPT 里设置的“自定义指令”或“项目”里的说明完全一样。

- **在 Claude Code 里，它叫 ****`claude.md`**
- **在 Gemini 里，它叫 ****`gemini.md`**
- **在 Codex 或 OpenClaw 里，它叫 ****`agents.md`**
无论叫什么，它的功能都是相同的。Remy 展示了他提前准备好的一个 `agents.md` 文件，里面包含了：`[24:10]`

- **角色**：你是我的行政助理。
- **背景信息**：关于 Greg 的一切，他的公司做什么。
- **工作偏好**：常用的工具（Notion 做项目管理，Stripe 收付款），沟通风格（邮件正式，Slack 轻松）。
当你把这个文件放在 Agent 的工作文件夹里后，神奇的事情就发生了。当你再次下达“写一封开发信”这样简单的指令时，Agent 会在“观察”步骤中自动加载这个文件，然后基于这些信息，生成一封完全符合 Greg 业务背景和口吻的专业邮件，甚至还会贴心地询问：“这封邮件的对象是品牌赞助商、潜在合伙人还是咨询客户？” 因为它知道 Greg 的生意里有这几类人。`[25:50]`

这就是从“提示词工程”到“**上下文工程**”的转变。`[25:10]` 过去，我们花费大量精力琢磨如何写出完美的提示词。而现在，我们只需要构建好完美的上下文，我们的指令就可以变得极其简单，而结果却依然出色。

### 让 Agent 拥有长久记忆：memory.md 与自我进化 `[26:30-33:45]`

**核心观点**

仅仅有初始的入职培训还不够。Agent 需要能够跨会话记住你的偏好和纠正，这需要一个自更新的 `memory.md` 文件来让它像优秀员工一样持续学习和进化。

**深度阐述**

`agents.md` 解决了 Agent 入职培训的问题，但它就像一个静态的员工手册，无法记录你与 Agent 在日常互动中产生的动态信息。Remy 指出了一个新问题：当你告诉 Agent “我的最爱颜色是薰衣草紫”后，它在当前会话中记住了，但当你开启一个新会话再问它时，它又会忘得一干二净。`[27:45]` 同样，如果你纠正它“不要再在邮件末尾写‘Cheers’，要用‘Best regards’”，它当时会答应，但第二天又会犯同样的错误。这就像你的员工每天来上班都会失忆一样，让人崩溃。

为了解决这个问题，Remy 引入了一个新的概念：**memory.md**。`[29:50]` 它的工作方式非常巧妙。首先，你需要在 `agents.md`（或 `claude.md`）里添加一段特殊的指令，就像这样：

> “在执行任何任务前，读取 `memory.md` 文件。当我纠正你或你学到新东西时，更新 `memory.md` 中的相关内容。保持 `memory.md` 为最新状态。” `[29:15]`
然后，你再创建一个空白的 `memory.md` 文件放在同一个文件夹里。现在，魔法再次发生。当你告诉 Agent “我的最爱颜色是薰衣草紫” 时，它不仅会回复“记住了”，还会真的去打开 `memory.md` 文件，把这条信息写进去。当你下次开启新会话并询问同样问题时，Agent 会遵循 `agents.md` 的指令，先去读取 `memory.md`，然后准确回答出你的最爱颜色。`[31:45]`

这个小小的 `memory.md` 文件，就成了 Agent 的“长期记忆”。随着时间的推移，它会不断累积你的各种偏好、纠正过的错误、特定的流程要求。

- 对于行政助理，它会记住“永远不要通过 Slack 和客户沟通，只用邮件”。
- 对于营销主管，它会记住“Facebook 广告的标题必须用问句形式”。
- 对于开发助手，它会记住“代码缩进用两个空格，不要用 Tab”。
这些微小的规则不断叠加，Agent 的错误就会越来越少，变得越来越懂你，越来越像你的“左膀右臂”。`[32:40]` 这就是 AI 员工的“自我进化”过程。虽然文件最终可能会变得很庞大，但在可预见的未来，它带来的好处远远大于管理成本。

### 连接世界的万能插头：MCP（模型上下文协议）`[33:45-40:00]`

**核心观点**

MCP（模型上下文协议）是一个万能翻译器，它让 Agent 能够通过统一的方式，无障碍地连接和使用 Gmail、Calendar、Notion 等任何外部工具，从而实现真正的自动化。

**深度阐述**

现在，你的 Agent 已经了解了你是谁（`agents.md`），并能记住你的偏好（`memory.md`）。但它还被困在你的电脑里，无法触及外部世界。要想让它真正帮你处理工作，它必须能访问你的邮件、日历、项目管理系统等等。这就引出了视频中最关键的技术概念之一：**MCP（模型上下文协议，Model Context Protocol）**。`[34:15]`

Remy 借用另一位嘉宾的精彩比喻来解释 MCP：`[34:30]` 想象一下，你的 Agent（比如 Claude）说的是英语，但你的 Gmail 说法语，Notion 说西班牙语，浏览器说日语，Slack 说中文。在没有 MCP 之前，Agent 虽然理论上能和所有这些工具对话，但它必须为每一个工具单独学习一门“外语”，这需要大量复杂的定制开发。

而 Anthropic 公司创造的 MCP，就像是一个全能的**翻译官**。它站在 Agent 和所有工具之间。Agent 只需要用它的母语（英语）向 MCP 发出请求，MCP 就能把这个请求翻译成对应工具能理解的语言，然后把工具的回应再翻译回英语给 Agent。这就为连接工具建立了一个**统一、标准化的简单方式**。

在实践层面，这意味着你不需要写任何代码。在大多数现代 Agent 驾驶舱（如视频中演示的 Co-wrok, Codex, Manus）里，你只需要进入“设置”或“连接器”界面，就能看到成百上千个常用应用的列表。找到你需要的（Gmail, Google Calendar, Notion, Stripe, Slack...），点击“连接”，授权登录，就搞定了。`[35:30]` 所有这些操作，背后都是 MCP 在默默工作。

为了展示 MCP 连接后的威力，Remy 进行了一个震撼的演示。他在 Claude Code 中，连接好了 Gmail、Granola（会议笔记工具）、Stripe 和 Notion。然后，他下达了一个极其复杂的任务：`[37:15]`

“**总结一下我今天收到的邮件。**”（Agent 通过 MCP 访问 Gmail）。

“**好的，有一封来自潜在客户 Maltoshi 的邮件，说对我们的通话很满意，想知道下一步。现在，查看我今天和 Maltoshi 在 Granola 里的会议笔记，然后起草一封邮件，附上提案和 Stripe 付款链接，最后去 Notion 里为这个项目创建一个新项目面板。**”

我们看到，Agent 开始有条不紊地工作：它从 Granola 里找到了会议记录，提炼了关键信息；接着它访问 Stripe，自动生成了一个付款链接；然后它起草了一封包含所有信息的专业邮件；最后，它又跳转到 Notion，创建了一个新的项目条目。整个过程，Remy 只需要动动嘴皮子，下达最终指令：“发送这封邮件”。`[39:50]`

这就是未来工作的模样。你不再需要穿梭于十几个应用之间，复制粘贴信息。你的 Agent 成了你唯一的“操作系统”，你所有的工具都成了它的“外设”。你只需在一个地方发号施令，它就能调动所有资源为你服务。

### 固化流程，倍增效率：Skills（技能）`[40:00-51:00]`

**核心观点**

Skills 是 AI 版的“标准操作流程”。它能将任何重复性的多步骤工作流程（如写提案、分析竞品广告、转介绍客户）打包成一个可重复调用的“技能”，让你一次构建，终身受用。

**深度阐述**

如果说 MCP 让 Agent 有了“手脚”，那么 **Skills** 就是让 Agent 有了“肌肉记忆”。Remy 将 Skills 定义为 **AI 的标准操作流程**。`[41:05]`

想象一下你要为客户写一份提案。第一次，你可能要和 Agent 来回沟通很多次：调整格式、修改颜色、把价格从顶部移到底部……经过半小时的磨合，你终于得到了一份完美的提案。但一周后，当你需要写另一份提案时，除非你找到并继续使用同一个会话，否则 Agent 会把之前的所有偏好忘得一干二净，你又得从头开始调教它。

这就是 Skills 大显身手的地方。它的核心理念是：**一次解释，永不再说。**`[41:15]`

有两种方式可以创建 Skills：

1. **从零创建**：如果你有一个现成的知识库，比如一个关于“如何写爆款标题”的课程，你可以直接把整个课程文稿上传给 Agent，然后说：“**基于这个课程，为我创建一个‘写爆款标题’的技能。**”`[44:10]` Agent 会利用它内置的“技能创建技能”，自动分析、提炼、打包，生成一个包含操作指南和参考示例的完整技能文件夹。
1. **从流程创建**：这是最常用也最强大的方式。当你刚刚手动完成一个复杂的多步骤任务后（比如刚才那个从邮件、会议纪要、到生成提案链接的流程），你只需要对 Agent 说：“**为我们刚才做的整个流程创建一个技能。**”`[44:45]` Agent 就会回溯刚才的每一步操作、每一个决策点，把它们封装成一个可重复使用的“技能”文件。
Remy 展示了他自己创建的无数技能，其中一个令人印象深刻的例子是“**广告分析技能**”。`[48:40]` 他曾经花了两小时，手动为一个大型电商品牌分析其所有的广告创意和落地页。在完成那次分析后，他让 Agent 把整个过程打包成了一个技能。现在，他只需要输入“运行广告分析技能”，然后粘贴一个竞争对手的广告库链接，他的 Agent 就会自动：

- 抓取该品牌投放的所有 200 多个广告。
- 对每个广告进行视觉和文案分析。
- 截图所有相关的落地页。
- 生成一份详尽的分析报告，总结哪些广告有效、为什么有效、以及可以如何改进。
这个原本需要耗费数小时人工的工作，现在只需要几分钟，而且 Agent 会变得越来越熟练。你可以为你的行政助理创建“日程简报技能”、“转介绍客户技能”，为你的内容团队创建“竞品调研技能”，为你的销售创建“提案生成技能”。这些 Skills 可以相互串联，构建出更复杂的工作流。比如，你的“晨间简报技能”可以自动调用“会议准备技能”来为当天的会议做足功课。`[50:15]`

**个人感受**

在讲述 Skills 时，Remy 的语气中充满了创造者的自豪和兴奋。他像是在展示自己的珍宝库一样，一个个点开那些技能文件夹。你能感受到，对他来说，这已经不是一项工作，而是一种创造和积累的乐趣。每创建一个新技能，都像是在为自己打造一把更称手的工具，日积月累，这些工具最终会构建出一个专属于他的、无比强大的自动化王国。这种“积累”和“复利”的感觉，正是 AI Agent 带来的最性感的魅力。

### 未来已来：你的 AI 操作系统与行动指南 `[51:00-结束]`

**核心观点**

未来每个人都会拥有一个属于自己的 AI 操作系统，通过在不同层级（全局/项目）配置上下文、工具和技能，我们每个人都能成为“10 倍效率”的员工。现在，就是开始构建你的第一个 AI 员工的最佳时机。

**深度阐述**

课程接近尾声，Remy 描绘了他眼中 AI 驱动的未来工作方式。他认为，每个人都将拥有一个自己的 **AI 操作系统**。`[36:40]` 这个操作系统就安在你的电脑里，由一个个文件夹和 Markdown 文件组成。在这些文件夹里，是你为不同公司、不同部门训练的 AI 员工——你的行政助理、你的营销主管、你的财务总监。你不再需要频繁打开那些 SaaS 应用，只需要坐镇在这个操作系统的中央，向你的 AI 员工们发号施令。

为了让这个系统更优雅，Remy 还介绍了两个最后的概念：**全局 vs 项目级**。`[53:20]`

- **全局（Global）**：有些技能、上下文和工具是你希望在每一个项目中都能使用的。比如，一个通用的“文本精简技能”，或者你常用的浏览器搜索工具。这些可以配置在全局层级，让你在任何文件夹里都能调用。
- **项目级（Project-level）**：而像“转介绍给 Sebastian”这种技能，显然只对你的行政助理有用，把它放在营销主管的项目里只会浪费上下文空间。同样，你为 A 公司连接的 Slack 工具，也不应该出现在 B 公司的项目里。
这种精细化的管理，让你的 AI 操作系统既强大又井井有条。

最后，Remy 给所有初学者提供了明确的行动指南：`[54:45]`

1. **选择一个角色**：想清楚你想先为哪个部门或哪个重复性工作打造一个 AI 员工？先从“行政助理”或“内容助手”这样通用的角色开始。
1. **构建入职文件**：打开你最顺手的聊天模型（如 Claude 或 ChatGPT），告诉它：“我想为我的 [角色名] 创建一个入职文件（`agents.md`），请以面试的方式向我提问，来收集所有必要的信息。”让它通过问答帮你生成这个文件。
1. **连接工具**：选择一个你喜欢的 Agent 驾驶舱（建议从 Co-wrok 或 Claude Code 这样界面友好的开始），找到“连接器”或 MCP 设置，把你日常工作最常用的 2-3 个工具（如 Gmail, Calendar）连接上。
1. **在日常使用中构建技能**：从现在开始，每当你完成一个你觉得以后还会重复做的任务（哪怕它很小），都尝试对 Agent 说：“为这个流程创建一个技能”。日积月累，你的技能库会越来越丰富。
**精华收获**

- **核心范式转移**：AI 的使用方式正在从“问题-答案”的聊天模式，转向“目标-结果”的代理模式。学会给 Agent 设定目标，而不是指导步骤。
- **上下文即一切**：在 Agent 时代，好的提示词不再是长篇大论的指令，而是精心构建的上下文。`agents.md` 和 `memory.md` 是你最重要的资产。
- **构建你的数字员工**：像管理真实员工一样管理你的 Agent——入职培训（`agents.md`）、记录偏好（`memory.md`）、赋予工具（MCP）、固化流程（Skills）。这套方法论通用且持久。
- **复利效应**：每一个你创建的技能，每一次你纠正的错误，都在为你的 AI 操作系统增加价值。时间越长，你的效率优势就越难以被超越。
这场对话不仅仅是关于技术，更是关于一种全新的工作和思维方式。现在，钥匙已经交到你手里，是时候启动引擎，驶入这个由 AI Agent 驱动的未来了。
