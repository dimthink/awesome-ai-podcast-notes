## 第一部分：AI代理的万亿级市场机会

### 当前AI应用领域的渗透率不足

这张图来自 **Sequoia（红杉资本）**。问题是：**AI代理在哪些领域被部署了？** 你可以看到，软件工程领域几乎占 **50%**，后台办公占 **9%**，营销和文案撰写占 **4%**，销售和CRM占 **4.3%**，其余领域占比更低。

当看到这些数据时，你的反应是什么？

我的意思是，我认为这反映了**两点**：

**第一**，它绝对反映了AI在那些即使以今天的AI能力也显然可以被颠覆或受益的行业中，**渗透率严重不足**。对吧？如果你把前沿的代理今天就直接部署到这些类别中的每一个，你应该能达到100%。

**第二**，我认为即使是像软件工程这样较高的数字，实际上也有点被**高估**了。我的意思是，你会发现，前沿开发者和应用前沿代理开发实践的公司正在发现，新的软件开发模式甚至不仅仅是每个工程师使用AI自动补全（像GitHub Copilot那样，我们在3年前就搞明白了）。现在的模式是，你甚至**不需要IDE**（集成开发环境）。我在 Hyper Agent 上的开发方式是，我有30个不同的云代码实例并行运行，每一个都连接到一个完全自主的浏览器。它可以去获取其他代理对其创建的任何PR（拉取请求）进行评论。

所以，这种模式转变——从无AI，到我认为的**第一代AI**（即基本上仍以人为中心的开发流程的AI增强）——正在发生。Andre Karpathy 谈到，在去年10月、11月，他完全颠覆了从“大部分仍是人类编写代码，AI增强”到完全相反的模式。这就是我们在前沿公司看到的那种飞跃。我认为即使是50%也是一个低估，因为实际上切换到那种新前沿模式的公司甚至人员的数量，肯定不到今天软件工程领域的50%。

所以，我认为我们实际看到的是，**前沿技术发展如此之快**，而许多公司、许多行业和许多职能甚至 barely 赶上了三年前的最先进水平，更不用说用新的最先进技术来自我颠覆了。

### “副驾驶” vs. “自动驾驶”：万亿级机会的两种形态

另一种思考方式是，这存在 **“副驾驶”** 领域和 **“自动驾驶”** 领域。这些图表来自红杉资本。你怎么看？红杉说，代理领域有万亿美元的机会等着被发掘。但它们非常不同。你对此有何反应？

我的意思是，对我来说，这些代理真正取得突破是在**大约四五个月前**，对吧？我认为开发者从 **Opus** 那里感受到了这一点。Opus 4.5 刚刚设定了一个新的高位线，让人感觉：“哇，这个东西第一次真正像一个真正的软件工程师，能够处理一个真正的人类工程师可能需要很多小时甚至几天的任务。”它可以完全自主地完成，并给我一个完美、干净的 PR（拉取请求），我就可以像评审者一样审阅它。

我认为这种体验将会在每一个其他领域被解锁，而且实际上**已经被解锁了**，对吧？因为我们基本上已经达到了一个点，**模型足够聪明**了。即使是在更同步的聊天交互中（而不是自主代理交互），你也可以问它最先进的事情，给它非常复杂的主题内容，比如管理咨询。你给它一些真正困难、有分量的问题，它会给你真正智能的答案，确实是专家级别的。

所以很明显，**模型的智能已经到位**。这些模型也足够聪明，能够利用大量工具和上下文，连贯地跨多个步骤执行任务。因此，我认为这更多是一个**如何以及多快**能将代理部署到行业中的每一个角色，直到我们真正可以用代理完成人类在每个职能中能做的几乎所有事情的问题。而这个的**TAM（总可用市场）甚至不是一万亿**。它可能是所有白领劳动力的整个GDP，显然达到了数十万亿美元，对吧？仅仅在西方半球就是如此。

### 为什么人们还不行动起来？

这就有点儿像，我不理解人们怎么还没有动力去创业。在听这个节目的人会想……是的，Howie。但感觉就是，我想不出比现在更好的创业时代了。完全同意。

我认为奇怪的是，这几乎就像 **“使用即相信”** 。如果你没有真正亲自动手，至少花一个完整的周末**玩转代理**，你真的很难完全领会这里的潜力。对吧？这意味着，不仅仅是肤浅的、一次性的提问，比如“嘿，谁会赢得下一届总统选举？”这种你可以问聊天机器人的问题。我认为人们并没有真正投入进来。他们只是做了些轻量级的实验。他们没有把一个足够有野心、足够复杂的提示或任务放在前沿代理面前。他们仍然像使用第一代聊天机器人一样使用它。

在你真正体验到这些前沿代理的全部力量和自主性之前，我认为你很难完全推断出，现在可以构建什么类型的、以前在结构上不可能实现的公司。比如，你怎样才能用**一个人和数百个代理**建立一个数十亿美元收入的企业？你必须真正去使用它，才能理解它。

### 经济学：人力 vs. AI代理，毛利率的颠覆

还有，这是另一张我无法停止思考的图表，那就是当你比较一个人类和一个AI代理时，**单位经济学被彻底颠覆了**。你可以在其之上建立起一些毛利率极高的业务。

**完全正确。** 有趣的是，我看到很多人抱怨前沿模型的每个Token成本。对吧？所以 Opus 4.6 现在7了，显然是最贵的模型。然后 GB 5.4 也非常好，但仍然有点贵。即使是开源的，虽然更便宜，但也不是免费的。

我认为有些人正在努力适应这种新的思维模式。在过去软件时代，很多东西是免费的。即使是 ChatGPT 也有免费版本，你可以随意使用，但你得到的是一个廉价、愚蠢的模型。因为它实际上并没有做自主的、多步骤的工作，不会每几天就消耗十亿个Token。它消耗的Token要少得多。

我认为我们必须克服这个障碍，不要将我们对AI的价格期望锚定在传统的订阅软件上（比如，“哦，我的天，我现在每月要付20美元给Netflix，而以前是12.99美元”）。相反，应该像你指出的那样思考：**让一个人做这件事要花多少钱？**

举个例子，如果我想去创造一个完整的营销活动，或者实际上在我作为CEO的角色中，有趣的是，我最近写的一份并发送给整个董事会和主要投资者名单的董事会备忘录，其中很大一部分是由 Hyper Agent 研究和起草的。显然，我的直觉和背景等都被注入到了代理中。当然，我最终会监督它。但我从一些最好的投资者那里得到的反馈是，那是我写过的最好的备忘录。

我当时就想，“是啊，因为是一个代理做的。而且，我只用了十分之一的时间就完成了。”所以，即使它花费了我比方说**150美元**的Token来生成那个输出，想想看我时间的机会成本。因此，我认为这是一个需要进行的**真正的思维重构**。让我们这样思考：人力等值时间成本是多少？与每月10美元的订阅费相比，150美元听起来真的很贵吗？100%同意。

我思考这个问题的方式总是锚定在**价值**上。我从那里面获得了什么价值？事实上，对你的董事会文件或任何东西来说，很可能它就是最好的，因为你得到了那么多的研究支持。没错。

是的。这也引出了我的最后一点：如果你能帮助一家公司运营一支由20个代理组成的团队，负责**客户情报、内容生产、竞争研究、潜在客户 enrichment** 等等各种任务，这就是一幅图景所展示的**工作的未来**——“**代理指挥中心**”。没错。

所以，当你看到这个，你的反应是什么？

我的意思是，那 literally 就是 Hyper Agent 中的一个视图。我感觉我就是在看 Hyper Agent。我认为这是未来。我们正在朝着这样一个世界前进：也许并非每家公司都 literally 只有一个人（虽然会有很多一人公司），但我确实认为**每家公司都会拥有一支代理舰队**。

有趣的是，代理正趋向于这些有目的的、几乎映射到人类曾经扮演的工作角色上。也许这有点像为什么硬件机器人会收敛于**人形形态**？部分原因是，我们家、建筑工地、工厂里的很多基础设施都是为人类工程学建造的。所以，为了让机器人能够无缝地插入现有基础设施，它必须拥有人类尺度的能力。

我认为代理领域也在发生非常类似的现象。不像5年前人们谈论超级智能时，我总是想象会有一个单一的、全能的AI，一下子搞定所有事情，像《瞬息全宇宙》那样。现在，我越来越相信，**上下文窗口**等方面将存在根本性的、永远存在的限制。我认为我们永远无法达到AI模型拥有无限上下文窗口的程度。我认为这背后有物理学原理：你一次只能拥有那么多的注意力和那么多的上下文。

这意味着，出于同样的原因，我们将人类划分为不同的角色或结构，这样公司里的每个人就不必同时知道所有事情和处理所有事情。我认为代理也是如此。因此，你会得到这个代理的概览，它实际上很好地映射到直观的人类扮演的角色。对我来说，这是一个真正有趣的、正在浮现的现象。我最近花了一些时间玩 **Paperclip**，挺有趣的，因为它 literally 创建了**组织架构图的隐喻**。

我认为这非常令人兴奋。一方面，它既熟悉，因为我们并没有完全颠覆我们在旧世界中对工作职能和角色的所有认知。另一方面，也确实需要重新思考和重新定位：“我怎样才能用代理来扮演那个内容生产的角色？”

## 第二部分：HyperAgent 深度解析

我认为现在应该进入 Hyper Agent 的话题了。所以，对于听众来说，什么是 Hyper Agent？你为什么要构建它？这是一个展示和讲解的播客。在本部分的最后，你能承诺给出所有关于如何使用 Hyper Agent 来构建业务的秘诀吗？

### 什么是 Hyper Agent？—— AI代理界的“Macintosh”

这就是 Hyper Agent。我现在在一个会话线程里。我待会儿会缩小，向你展示入口点是什么样的。

你可以把 Hyper Agent 想象成：如果所有其他代理产品（如 OpenClaw 等）更像是 **Linux**，那么 Hyper Agent 就是我们打造的 **Mac 版本**。我们希望它**开箱即用**、**安全**、**云原生**。你不需要运行一台 Mac Mini。也许最重要的是，Hyper Agent 正在应用许多我们在10年前应用于无代码应用类别的相同设计理念和对优秀用户体验的痴迷，但现在应用于代理。

意思是，应用是相当复杂的。如果你是一个开发者，即使在那个时候，你也可以构建一个 Rails 应用，它有一个数据层、一个逻辑层、一个视图层，但它仍然相当技术性。我们非常技术性。Airtable 的整个理念就是将其提炼为一种真正直观的体验。事实上，我们深受 Macintosh 和 GUI（图形用户界面）的启发，将基于终端的命令行计算变成人们可以立即理解的东西。

所以，Hyper Agent 的意图是成为**一种非常直观、可视化的使用代理的方式**。

### 实战演示：从商业研究到产品开发

这实际上是我稍早运行的一个任务线程。这实际上是你（Greg）的一个创业点子，我们让 Hyper Agent 来处理它。基本的想法是：为房地产经纪人从公共数据生成**超本地市场报告**，对吧？

基本上，这个代理到处进行研究，了解市场格局。它运行了一系列分析。它拥有**完整的编码能力**，一个完整的**沙盒环境**。所以，它正在运行一台完整的计算机，只不过是在云端的，不是你自己的电脑。如果需要，你可以将它连接到你的所有账户。它可以访问你的 Slack、Granola 和电子邮件。它可以代表你发送东西，或者只是预先起草邮件。它已经预先配置了能力，比如从 Twitter 拉取信息，使用高级工具如生成图像或使用 Google 地图等。

基本上，它到处研究了这个机会，然后创建了这份**研究简报**。让我给你看看这个是什么样子的。这是为你那个想法准备的商业案例。我有点喜欢它，因为我认为这些我称之为**中等规模**的市场，它不是一个千亿美元的市场，那会超级竞争，会有大型现有企业进入。但我真的很喜欢这个想法，它可能不是微型，而是更小型或中型市场：**几十亿美元的TAM**。这意味着即使只抓住两位数的百分比份额，你也可以建立一个非常有利可图的企业，每年赚取数亿美元。然而，它又足够小，以至于真正的大公司不会来竞争。

所以，这个代理为它创建了一个商业案例。它发现了一些很酷的用户对问题的验证。比如，它查了 Reddit，找到了一些真正在说“我需要这个产品”的房地产人士。这验证了市场需求。

这是当前的问题。我之前甚至不知道这个，但显然有一件法律上的事情改变了市场的动态。人们不想要更多的软件，比如另一个带界面的工具。它还做了一些竞争分析：还有谁在这个领域。

然后它把这件事情的商业案例整合在一起。但更好的是，你不必止步于此。你可以直接告诉它去构建产品的 **V1 版本**。在这种情况下，因为 Hyper Agent 有完整的编码能力，它就直接去创建了这个产品的 V1 版本。

> “没错，而且说实话，很多工作其实很简单：如果你有一个运行前沿模型（如 Opus 4.7 或 GD 5.4）的优秀前沿代理，它开箱就能很好地完成很多这样的事情。所以，任何由前沿模型驱动的前沿代理都应该能够创造出这种质量的应用。Hyper Agent 的独特之处在于，它不仅能完美地做到这一点，而且还能在工作流程中做到：它不仅仅是一个应用构建器。**应用构建现在只是一个功能，一个已经被商品化的功能**。它真正能做的是，去研究从端到端的业务背景：‘我试图做什么？’然后基于此构建应用。对吧？所以，在这种情况下，Hyper Agent 更像是**创始人**，而不仅仅是开发者。”
### 强大的内置工具与代理舰队管理

我还喜欢 Hyper Agent 的一点是，它**开箱即用**，带有真正强大的工具。比如，它有 Google 地图作为工具。它可以去实际找到广告牌位置的街景图像。它知道如何使用街景来找到实际的兴趣点，然后利用该图像作为种子图像，用于AI图像生成或视频生成。

另一个你可以用 Hyper Agent 做的酷事是：告诉它‘拿着这栋房子，我想让你使用 Zillow 的室内照片或外部照片重新设计房子’。它会做得非常好。

这就是 Hyper Agent 的简要介绍。

一旦你真正构建了许多代理，你就能开始看到这种能力：不仅仅是查看我的单个代理，而是**所有代理的概览**。这并非一个非常完善的账户，这大概是你使用 Hyper Agent 第一周的样子。但是，就是我们谈到的那个**指挥中心视图**——我们希望你能够创建许多扮演不同角色的代理：内容营销人员、市场研究员、客户邮件回复者，然后**管理和监督一整支代理舰队**。

通过我们拥有的能力，你可以从每次运行中**策划记忆和技能改进**，不断改进它们。最后，还能够将它们部署到**团队环境**中。例如，如果你想拿任何一个代理，赋予它在 Slack 中发言的能力，你可以说：“让我把这个放到 Slack 里，让它始终在线，甚至始终监听。”它就坐在我的频道里，听着我和我的团队谈论的一切。当有相关的内容要添加时，它会自动插话，然后人们可以像与真正的虚拟同事一样与我互动。我见过一些 OpenClaw 的高级用户实现了这种相当神奇的体验——你 Slack 里的同事现在除了人类，还有代理，它们非常聪明，有自己的专业知识和背景。你在 Hyper Agent 中构建的任何代理，只需**一键点击**就能获得这种能力。

### 技能（Skills）：代理能力复用的核心概念

你提到了技能。Hyper Agent 上的技能是如何工作的？人们应该如何看待它？

是的。技能我认为是**前沿代理世界中最重要概念或原语**。意思是，这些模型普遍足够聪明。就像找到阿尔伯特·爱因斯坦，他显然在一般意义上超级聪明，但他可能不知道如何解决房地产领域的问题。但如果你给他一份恰到好处的简报：“这是一本手册，你需要学习所有关于房地产这份工作的知识”，他会去做得相当好。

所以，**技能**的强大之处在于，它是一个真正**可组合**的概念。你可以**交互式地创建技能**。例如，假设我真的要创建一个新的线程。保持简洁：“帮我创建一个技能，用于发布 Greg Isenberg 风格的 AI 内容。”

> “不，别创建这个。别……”
>

> “不。但说真的，我们不会抢 Greg 的生意。”
但这里真正酷的是，它不会只是说“好吧，我有一个提示，假装自己是 Greg Isenberg”。它可以实际去**研究**你实际是如何做内容的。它会制定一个计划：“我首先要去研究你的风格，弄清楚你关心哪个平台，看你的一些实际帖子，然后把所有这些东西提炼成一个技能。”然后我可以把这个技能固定到一个代理上，或者在任何需要的时候按需使用。

所以，假设只是为了好玩：“你想发布到哪些平台？”我们先说 X。“我们让这个技能只生成草稿，所以它不会自动为你发布。”“你希望你的代理人 Eisenberg 专注于什么类型的内容？”

> “嗯，做**逆向思维的AI观点**吧。”
>

> “好的，酷。除此之外还有别的主题吗？”
>

> “**单人创业家，自力更生，生活方式**。”
“然后你想如何使用这个代理？如果你最终使用这个代理，你是想从一个想法开始？还是希望它自己为你产生想法？”

> “我不想做任何事。我们就走**全自主**模式。总有一天我们得看看，真正的 Greg 是不是整天就坐在泳池边，而全是 Greg 的AI化身版本在自主做所有事。”
现在，它会去研究你，找出如何为这个技能提炼出完美的技能。

### 竞品对比：Hyper Agent vs. Perplexity Computer, Manis, OpenClaw

人们应该如何比较 Hyper Agent 与 Perplexity Computer、Manis 或 OpenClaw 本身？

是的。所以，针对 **Codex**，很简单：Hyper Agent 是一个**更通用的代理平台**。

针对 **OpenClaw**：这更像是**交钥匙**方案，**开箱即用**，默认**安全可靠**，**云原生**，并且更专注于**优秀的用户体验**。OpenClaw 当你真正需要进入配置，或者尝试编辑记忆，或做任何策展或配置时，它相当原始。它是一个非常原始的产品，感觉更像是给那些已经成为专家的、非常技术性的人用的。

我认为 **Perplexity Computer** 和 **Manis** 是最接近 Hyper Agent 的竞争对手。主要区别在于：

1. Hyper Agent 拥有**更强大的开箱即用工具**。
1. 它更注重**开箱即用的用户体验**。
其次，我们也做了更多工作，让 Hyper Agent **立即准备好运行**的不仅仅是单个代理。我认为 Manis 和 Perplexity Computer 的常规体验仍然是：你使用这些产品，有一个相当棒的代理，你直接使用它。你可以在 Hyper Agent 中做到这一点，这正是我们在这里做的。但它也从第一天起就设计得更注重**可扩展性和可部署性**。意思是，一旦我有了一个对我有用的代理，我现在可以**一键部署**到我的 Slack 频道，然后我公司的每个人都可以从这个始终在线、插话的代理中受益。他们可以问它问题，它会回答。你有**指挥中心**，那个**舰队视图**，你不仅可以监督一个代理，而是可以监督你一整支代理舰队。

我们甚至还有能力来监督和策划**学习成果**，这些学习成果让每个代理不断变得更好。所以它们有一种自动的**自我改进循环**，随着时间的推移，它们不仅积累新的记忆，还会向你建议：“嘿，也许你应该添加这个额外的技能”，或者“更新或调整这个技能”，甚至“也许你应该去尝试改变我的代理系统提示，或者给我访问不同工具的权限，这样我就能更好地完成这类工作。”

最棒的是，我们实际上有一个我们称之为 **Rubrics（评分标准）** 的概念。顾名思义，就是一个**评估标准**。你可以用 rubrics 做的一件非常强大的事情是，为某类任务定义“**好的标准是什么**”。例如，我可以创建一个“什么是优秀的 Greg Isenberg 风格内容的标准”。

它的作用是，我可以有一个完整的**评估循环**。每次我的代理运行时，比如 Greg Isenberg 的技能准备好了，我可以说：“我正在创建虚拟 Greg 代理，我要把一个 rubric 固定到这个代理上。”这个 rubric 会说：“每次 Greg 创建一条内容时，我都想用你关心的维度，使用一个作为评判员的独立 LLM 来给那条内容打分。”然后我可以 literally 监督我的代理表现如何，随着时间的推移。如果我想要深入了解并检查任何一个任务运行，看看它是如何被打分的，我也可以做到。

所以，我们基本上有了一个**完整的闭环**：

- 不仅仅是一个开箱即用的代理或线程体验。
- 不仅仅是可以策展代理、部署它们并随时间改进它们。
- 更是拥有一个完整的**可观测性层**和编排故事，你可以查看所有代理的运行情况，看看它们做得怎么样。
如果我把评估标准固定到任何一个代理上，我会看到它的评分趋势线。然后我可以自动建议：“嘿，也许我可以降低模型质量，从 Opus 降到 Sonnet，成本降低五倍，而分数并没有下降多少。”所以，一旦人们真正开始大规模运行代理，这些次级能力就变得至关重要。这不仅仅是我能让一个代理做一件事，而是我如何**监督和运营一个拥有许多不同代理的整个企业**，并确保一致的质量。这很重要，因为如果你使用 Manis，谁是输出的评判者？评判者是你，人类，而不是 Opus 4.6。这是一个巨大的区别。

所以，如果你真的想创建我们之前谈到的“代理优先”企业，管理大量的代理，现实地说，你不可能有带宽去查看每一个输出。

没错。这基本上是**管理学的101**，但现在应用于代理。当你扩大规模时，如果你是一个企业的CEO，你就是没有时间去查看公司里每个人做的每一件事。所以你需要创建更好的**自动化的制衡机制**来监督代理在做什么，并检查工作质量。如果你有一支由人类内容创作者组成的庞大军队，你会希望有一种可扩展的方式来检测他们发布的内容是好是坏，然后知道“好吧，我们必须调整每个人的指导方针”。

### 从实践到精通：不要放弃第一次尝试

好了，现在我们有 Greg Isenberg 的逆向思维草稿技能。我要保存这个技能，然后试运行一下。它会扫描今天的AI新闻和趋势，然后创建一些逆向思维的草稿。

这里的想法是，第一次尝试可能做得还行。它对你做了一些研究，它有很多关于你工作方式的背景信息。如果我想查看这个技能的更多细节，我可以打开它：“什么时候应该使用它”，实际的技能内容。“Greg 的声音像晚餐时一个聪明的朋友，说出大家心照不宣的话，而不是一个企业传播者”，我同意这一点。“你进过所有这些公司的内部……这不意味着要刻薄”，我认为这非常敏锐。你声音大但不烦人或粗鲁。

然后我很好奇你是否同意这些风格上的东西：“你必须在**前七个字**抓住人”。“你不想要大段的文字”，我对此很内疚，所以我应该接受一些 Greg 的技能并应用到我自己身上。“你喜欢有序列表”。“永远不要以‘你怎么看？’结尾”，这太泛泛了。

所以，假设这是一个不错的 V1，也许它只达到了50%的程度。但关键是，这些技能应该是**常青**的。它不是一次性的。重点是，每次我使用一个技能，要么通过 LLM 自动生成学习点并自我改进的建议，要么因为我看着内容说“不太对，这就是你错的原因”，你可以**交互地调整和改进**技能以及代理的性能。

我认为很多人面临的挑战是，他们一次性尝试某事，结果没有他们希望的那么深刻，他们就**放弃了**。我强烈敦促大家，这也是与你的想法非常一致的：**不要在第一次尝试后就放弃**。因为很明显，代理足够强大，可以做你想让它做的几乎任何事情。问题不在于它是否有能力，或者你是否应该放弃它。问题在于你是否有能力投入时间、指导和策展让它达到目标。

我认为这**非常值得**。如果你做到了，它显然会给你带来巨大的杠杆作用。拥有一个**永远在线的员工**，在后台一直做你关心的事情，相对于雇佣一个新员工的成本来说，运行成本微不足道。

> “这和现实生活一样。当我刚开始打网球时，我打得很差。当我去打网球时，我几乎不想去，因为我心想‘我打得不好’。但你会经历那个**混乱的中间阶段**，然后变得越来越好。随着时间的推移，你会想‘哇，这真有趣’。所以我认为，一旦你达到了一个真正有趣、产出确实很好的点，事实上，99%的人并没有投入工作去获得好的产出。所以，这就是**套利机会**。它属于那些真正投入时间优化，并将其带到高质量水平的人。”
>

> “**绝对正确**。这很有趣。Benchmark 的一位合伙人发了一份备忘录，基本上是对所有被投公司的**警钟**，要求他们‘跟上进度，立即从根本上重新思考你如何用AI运营你的业务’。假设是，你可能认为自己正在做一些AI的事情，或者有一些AI功能，但这不够。他们结尾的寓言是：想象有两个朋友，回到2003年左右，他们都去**挨家挨户卖刀**。其中一人决定每晚和周末花30分钟尝试这个新的 Google Adwords 东西，试图为他的业务获得一些额外线索。一个月后，他们的收入从 SEO/SEM 中增长了一点。下个月，他们得到了更多。而另一个人想，‘这个东西太棒了，SEM 太棒了，而且现在还很早，我需要把它搞清楚’。于是他们**完全停止了挨家挨户卖刀**，接下来的几个月只专注于如何让整个互联网业务运转起来。在早期，他们有两个月零收入，靠积蓄生活。但他们慢慢开始让这个东西运转起来，变得精通 SEO 和 SEM 的最佳技术，以及如何创建一个电子商务网站，让人们可以直接在那里交易。故事的结尾是：把时间快进5年，你认为这两个人分别在哪儿？显然，第二个人可能已经建立了一个早期的数十亿美元的电商企业，切下了一块‘下一个亚马逊’。而另一个人可能还在挨家挨户卖刀，这个市场越来越难，而且正在萎缩。所以，我认为这就是那些时刻之一：你必须**按下重置按钮**。那些感觉像实验、实际上并没有带来实际收入的事情，实际上可能是你在短期内（甚至不是两年，也许是六个月）创造真正商业杠杆所能做的最有意义的事情。”
我很好奇，在你的经验中，或者当你看到单人创业家这样做时，平均的**盈亏平衡点**在哪里？无论是 literal 地达到可以全职自给自足，成为你的收入来源，还是甚至只是开始感觉它要成功了。

我认为人们会经历几个**里程碑**，这是一个**信心的游戏**。

- 当你赚到你第一个**互联网美元**时，无论它是什么，都会**重新连接你的大脑**。所以，如果你能从一个陌生人那里为一个想法赚到1美元，仅仅1美元，它就会重新连接你的大脑。
- 然后我认为，一旦你达到每月 **1万美元**，这个数字就有某种魔力。在大多数情况下，一旦你达到那个数字，你很可能会**辞职**，你会全身心投入。你会想：“好吧，这里有点名堂，有一条通向更大事业的道路。”
我认为，对于代理产品和像这样的产品，很多人犯的错误是他们尝试得太**零散**。所以我鼓励人们做的是，**每天**都尝试这个产品，持续一段时间。所以，承诺30天、60天、90天。每天都把它放在你的日历上，比如“这里30分钟，那里30分钟”。这就是让你成为**前1%的代理构建者**的方法。因为你让它成为你工作流程的一部分，然后你会看到超额的回报，因为它是**复利**的。

这很有道理。这有点像，我不是作家，但我听作家朋友说，最重要的事情不是等待一个周末，指望灵光一闪，一口气写完整个剧本或整本书。而是你**强迫自己每天都写几页**，不间断。其中一些页面会很糟糕，但强迫的习惯会让你越来越好，然后它就变成了自然。所以我认为这非常适用，也类似于“变得精通代理”的世界。

### 实战反馈循环：从草稿到符合风格的推文

我们有一些推文吗？好吧，我们看看。共识叙事越来越响亮。每一篇 Medium 文章读起来都和上一篇一样。

这里有一条：“每月1万美元的AI单人创业家热潮，大部分是内容农场的虚构作品。他们说82%的美国企业有零员工。”你对这条有什么看法？

我喜欢它的一点是，当我作为人类发推文时，基本上没有数据，只是有一个**热点看法**。所以很酷的是，这里有**研究**和事实。显然，人们希望他们的推文有数据支持。

> “是的，也许有了一个 Hyper Agent 团队为你做所有研究和产生内容创意，你现在就有时间了。”
哦，这挺酷的。这是真的吗？“Medv 实际上不是一个合法的企业”。我实际上没有关注那个故事的第一阶段，那是“哦，我的天，这个东西太大了”。但对于一个十亿美元创业故事来说，这有点令人失望。但也许有一种观点认为：“不，但真正的成功仍然是可能的，这家伙只是给了我们所有人一个坏名声。”

“你的AI代理没有取代你的虚拟助理”等等。这挺有意思的。

是的，这些都是我称之为**内核**的东西，是产生真正好的推文的素材。酷的是，我可以给它反馈。例如，我想给你关于你技能的反馈。你想给它什么反馈？

> “嗯，我会说，那些表现好的推文听起来非常**朋友对朋友**。这些推文是不是感觉都太……太不像是口语化的了？它们感觉太正式或太僵硬了。”
>

> “**完全正确。** 这正是我会注意到的。”
>

> “我们可以把它放到评估里。你可以立即去修改或根据这个反馈更新技能。你也可以让它立即根据这个反馈生成新的推文草稿，使其更口语化。最后，就像你指出的，我可以去创建一个评分标准，定义我关心的五个维度，然后自动评估未来的每一个输出。你有多种选择，取决于你现在想走多远。如果你现在只想把工作完成，你没必要麻烦去搞 Rubric。但最终，你会想建立一个可扩展的系统，让它持续工作并变得更好。那时你就会使用 Rubric，这其实并不难。你可以通过UI构建一个，或者在这个聊天里说：‘帮我创建一个评分标准，用来评估优秀的 Greg 风格内容。’然后它会帮我去创建那个标准，保存它，把它固定到这个代理或技能上，然后在我每次创建内容时自动运行。”
### 开启“活力模式”与自动化工作流

是否有可能，比如，每天早上8点收到一封带有一些想法的邮件？

**是的，绝对可以。** 方法是，你可以直接在对话线程中告诉它：“你能把这个变成每天早上8点的**重复性每日邮件**吗？”然后它会做的是，说：“我现在要把这个线程保存为一个代理，这个代理会被赋予一个运行计划：每天上午8点去做这件事。”

我们实际上即将推出一个我们称之为 **“活力模式”** 的功能，其灵感来自 OpenClaw 的那种“心跳”行为。你已经可以配置一个代理通过说“我想每30分钟拉取一次”来做到这一点，但我们正在把它变成 Hyper Agent 中的一个**一等公民**功能。你可以 literally 点击一个按钮，**让任何代理或任何线程变“活”**。然后感觉会是，这个东西就是**24/7始终在线**，查看所有新推文，产生新想法，并在它有新东西时通过 Telegram、电子邮件或 Slack 推送给我。所以，用户体验或思维模型是：“哇，这变成了一个始终在线、全天候的代理，向我推送想法”，甚至可以去**预先起草和发布内容**。如果你想要完全冒险（YOLO）模式，你实际上可以让它自己去发推文。

### 连接器（Connectors）与API集成能力

我认为你们在 Hyper Agent 中内置了大量的连接器，对吧？

是的。真正酷的是，我们不仅有大量开箱即用的连接器，你只需在对话线程中**点击一个按钮**进行 OAuth 认证。例如，我可以新建一个线程问：“你想用 Hyper Agent 使用什么工具？可以是 Granola。” “Notion 也许。” “好。我能连接到 Notion 并拉入我所有的笔记吗？”然后它会在线程中说：“嘿，这里有一个 OAuth 链接，连接到你的 Notion。”

但可以说最强大的部分之一是，即使对于我们没有连接器的东西，比如某个非常**晦涩的API**，你也可以让 Hyper Agent **去学习那个API**。例如，我说：“实际上，别管这个了。你能帮我建立一个到某个相当新工具的API集成吗？该工具有API。”

> “我猜你们这里内置了 Linear 吗？”
>

> “我们确实有到 Linear 的连接，但也许 Twilio 是个好例子。我不认为你可以通过 OAuth 连接 Twilio，所以它必须是一个 **API 技能**。我们可能有一个预建的连接器，但我还是让它构建一个自定义技能。‘咨询一下，帮我构建一个通过 API 集成 Twilio 的自定义技能。’”
现在会发生的是，它可以去研究 Twilio API 文档，为自己创建一个使用该 API 的技能，然后以安全的方式询问我输入我的凭证，然后就能够完全使用 Twilio API。

所以，我认为现在的强大之处在于，一个前沿代理应该能够 literal 地做任何事。但这只是你是否能给它访问正确背景信息的问题。你必须告诉它：“嘿，你应该为此构建一个技能。”这样它将来每次都能毫不费力地做到。

假设我们想做 SMS 语音，也许电话号码，我们会关掉 API 密钥。任何特定的工作流？也许我想建立一个**语音和短信服务**，可以打电话给餐厅预订之类。

如果你听到这个还不兴奋于现在就去创业，那真是疯了。

### 如何开始使用 Hyper Agent：克服“空白画布”问题

如果有人第一次听说 Hyper Agent，他们想开始使用，他们应该做什么？他们如何开始？如何从 Hyper Agent 中获得最大收益？

我认为最难的往往不是如何使用产品。我们的用户说：“哇，这个产品超级直观。我通常只需要问代理去解决某事，它就会去做。”所以我不需要学习大量新的配置或UI。我认为最难的部分实际上是**选择正确的问题或商业机会**，你想用 Hyper Agent 去攻克的。而 Hyper Agent 实际上可以帮助你**头脑风暴**。

事实上，我们刚推出了一个更好的**入门流程**。不再是简单地将你带入一个通用的、空白的画布，让你必须选择一个新线程，或者我们有一些模板等等。现在，当你第一次进入时，它会建议：“嘿，你想让我连接到你的所有联系人吗？所以，让我连接到你的 Gmail、Slack、Notion 和 Granola。”它会提议去**研究你**和你的背景。它会阅读你过去几周的大量邮件、Slack 消息，查看你过去的 Granola 会议。当然，所有这些背景信息对你都是私密的。但然后 Hyper Agent 能够向你建议：“嘿，根据我了解到的关于你的一切，这里有一些可能与你相关的用例。”

例如，看起来你是个 VC，你可能做很多交易流工作。我可以创建一个代理，自动去总结和研究你收到的每份投资推介。你可以让我一直开着，我会在后台运行，每次你收到一个 incoming pitch 时就提醒你。你甚至可以让它学会这种行为，在邮件线程中给你回复一条**私密回复**：“嘿，我研究了这个公司，我也总结了他们所有的材料，这是你应该知道的。”

但整个想法是，Hyper Agent 本身可以帮助你识别用例，或者你可以直接带着一个非常宽泛的提示进来，比如“我对建立一个单人创业家生意感兴趣，但我不确定。我有点对房地产感兴趣。我想选一个 Greg 开源的想法，帮我规划一下。”它会非常擅长与你一起推进。

所以，我认为主要的事情是：**不要卡在空白画布的起点问题上**。就进来，找个地方开始。也许是你的个人联系人，也许是你带着一个想法进来。但一旦你开始进入，它就会把你吸得更深，因为你会意识到你所能做的一切，它是如此强大。你会忍不住变得越来越擅长。
