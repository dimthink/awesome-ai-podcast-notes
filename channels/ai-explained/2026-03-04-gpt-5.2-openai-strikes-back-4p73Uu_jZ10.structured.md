# GPT 5.2 发布深度解析：OpenAI 的反击与AI基准测试的迷思

在过去的24小时里，OpenAI 发布了一款新模型以及大量破纪录的测试结果。然而，**GPT-5.2** 可能并非一个圣诞奇迹，因为为了获得前沿性能，它通常需要花费更多的令牌（Tokens）进行思考。但暂且不谈令牌，在许多基准测试中，**GPT-5.2** 已经是目前最好的语言模型之一。

对我而言，这有点像我们所有人都收到了奢侈的圣诞礼物——我们不知道哪些结果是实验室耗尽最后智力或财务透支“买来”的，哪些结果又会在新年伊始就被更耀眼的新事物所取代。无论哪种情况，这确实是一个好模型。

接下来，我将为你提供关于 **GPT-5.2** 的九个细节，这些是你从新闻标题中无法获取的，希望能帮助你自行判断。最后，我还会用一个我认为相当贴切的羊群比喻来作结。

## 一、官方发布的核心亮点与谨慎声明

首先，让我们谈谈 **GPT-5.2** 发布页面顶端那个大胆的声明：**GPT-5.2（思考模型）在“GDP-vow”基准测试上创下了新的最先进水平（State-of-the-Art，SOTA）得分，并且是第一个性能达到或超过人类专家水平的模型**。根据专家评审员的判断，在该基准测试的71%的比较中，它击败了或与顶级行业专业人士持平。显然，它是目前最适合实际专业应用的模型。

我必须指出，OpenAI 及其团队在描述这项基准测试的成果时，措辞是相对具体的。他们称其为一项评估，衡量的是**涵盖44种职业的、明确指定的知识工作任务**。然而，看到模型在实际专业任务上超越专家水平，可能会导致许多人误读这张图表和这项基准测试。

我曾在之前的视频中详细测试过 **GPT-5.2**，并深入介绍过这个基准。现在用十秒钟快速回顾一下：

- **问题来源**：GDP-vow 的问题确实由行业专家撰写，但选择的职业必须主要是数字化职业，非数字化的已被排除。
- **任务选择**：只选取了每个职业中的一部分任务。
- **“明确指定”的含义**：官方使用“明确指定”（well-specified）这个形容词是刻意的，因为每个任务的完整上下文会提前提供给模型。OpenAI 在发布说明中也承认，现实任务通常涉及**隐性知识（Tacit Knowledge）**，你需要自己搜寻、直觉感知或了解相关的背景信息才能解决问题。
- **忽略灾难性错误**：最后，该基准测试明确排除了模型可能造成的**灾难性错误**的影响。你可能最近听说过模型删除了人们整个硬盘的新闻，这类影响在这种基准测试中是难以计算的。
公平地说，这确实意味着，对于像“在进行一些网络研究后创建一个电子表格”这样的任务，模型已经变得极其出色。

## 二、实测表现：惊喜与局限

我让 **GPT-5.2 Pro** 创建一个以足球为主题的交互矩阵，即根据某俱乐部在本赛季与其他所有俱乐部比赛的结果生成一个结果矩阵。

结果令我印象深刻：它不仅生成了比赛列表，还成功创建了交互矩阵。我亲自核对了许多结果，确认准确无误。我还使用其他模型进行了多次深入研究，都得到了相同的结果。

然而，也有一件让我略感失望的事。十月份论文发布时，我赞扬了 OpenAI 将当时最好的模型 **GPT-5 High** 与表现更好的 **Claude Opus 4.1** 进行了对比，这是一种真正的学术诚实。但这次发布 **GPT-5.2**，他们没有将其与 **Claude Opus 4.5** 或 **Gemini 3 Pro** 进行比较。

这自然引发了网友们的“调皮”对比，尤其是在视觉理解方面。**GPT-5.2** 的发布页面展示了模型理解一块主板并能相当准确地进行分割。但随后，现任职于谷歌、曾任 OpenAI 成员的 Logan Kilpatrick 戏谑地表示 **Gemini 3 Pro 在多模态理解方面仍然是 SOTA**，并展示了由 **Gemini 3 Pro** 对同一张图片进行的、精细得多的分割结果。

回到电子表格的例子，我必须说，我也给 **GPT-5.2**（基础版）下达了同样的挑战（因为并非所有人都订阅了每月200美元的Pro tier）。它能获得结果，但**无法创建交互矩阵**。它的令牌预算更小，思考时间也更少，这也许是不可避免的。

## 三、基准测试的迷思：思考时间决定一切？

这就引出了我认为我们都应该开始理解的下一个基本点：如今，AI在基准测试中的表现**越来越（尽管并非完全）由思考时间或使用的令牌数量驱动**。用更专业的术语说，这是**测试时计算（Test Time Compute）**的函数，即模型提供商分配给回答基准问题的计算预算。

正如 OpenAI 的 Noam Brown 指出的，这只是比较基准性能变得越来越困难的原因之一。他说：“OpenAI 发布单一的基准测试分数是因为这样更简单，且人们期望看到它。但理想情况下，所有评估都应该有一个X轴，可能是完成基准测试所用的令牌或词元数量，或是所涉及的成本。”

以旨在测试模型**流体智力（Fluid Intelligence）**的原始基准 **ARC-AGI-1** 为例，模型无法通过记忆来获得高分。在这个基准上，结果几乎随着思考投入的美元或令牌数量增加而** uniformly better**。模型思考时间越长，就能从训练数据中尝试越多的想法，或对一个想法进行越多的排列组合。

因此，在取名略显滑稽的 **GPT-5.2 Pro Extra High 推理模式**下（在 SimpleBench 测试中我会再提到它），它取得了超过90%的最佳性能。尽管如此，必须指出的是，由于各种计算和算法效率的提升，**性价比（Price Performance Ratio）** 仍在持续下降。去年此时，大多数人对 03 模型及其在 ARC AGI1 上 88% 的得分印象深刻。一年后，我们看到了 **390 倍的效率提升**。

## 四、ARC-AGI 2：新纪录与新视角

这就带我们来到了 **ARC-AGI 2**。如果你还没听说过 ARC-AGI，它是一种**模式识别练习**，同样旨在测试模型在训练数据之外的能力。例如，如果第一张图变成第二张图，那么这张图该如何变换？

结果非常相似：**GPT-5.2** 创造了新纪录，且性能几乎与投入的金钱和令牌数量成正比增长。请仔细观察 **Gemini 3 Pro** 与 **GPT-5.2** 的表现。哪个模型更好？一个（**GPT-5.2**）花了更多的令牌和金钱进行思考，得到了更好的结果。这是否意味着它比 Gemini 3 更好？你可能不知道，一家名为 Poetic 的外部公司围绕 **Gemini 3 Pro** 构建了一个脚手架（Scaffold），通过增加令牌开销也得到了类似的结果。

## 五、基准选择的难题：该信谁？

如果思考预算使比较变得复杂，那么模型提供商对基准的选择呢？OpenAI 昨天出来说，不，真正重要的是 **SweetBench Pro**，它很严谨。与仅测试 Python 的 Software Engineering Bench Verified 不同，SweepBench Pro 测试四种语言，旨在更好地防止数据污染。你会从图表中注意到，同样是**更多的输出令牌带来了更高的性能**。这并非说模型在使用令牌的效率上没有提高，但总的来说，花费的令牌越多，效果越好，这仍然是事实。

即使我们使用完全相同的基准进行直接对比，也不总能轻易判断哪个模型更好。这不仅因为某些模型在某基准上更强，更因为**即使是声称测试同一能力的基准，也会给出不同的结果**。比如分析表格和图表。

- **MMU Pro**：旨在测试模型分析表格、图表、图形的能力。**Gemini 3 Pro** 以 81% 的成绩达到 SOTA，高于 **GPT-5.2（思考模型）** 的 80.4%。
- **Chartive Reasoning**：这是一个全新的基准，测试模型进行**现实图表理解**的能力。在这个基准上，**GPT-5.2** 以 88.7% 对 81% 的成绩遥遥领先。奇怪的是，它测试的也是图表理解。那么，**该相信哪个基准**就成了另一个问题。
再来看看像 **Humanity's Last Exam** 和 **GPQA** 这类非常知名的基准，它们都测试极其晦涩的知识和推理能力，尤其是在科学领域。

- 在带工具的 **Humanity's Last Exam** 上，两个模型的结果旗鼓相当，都在 45-46% 左右。
- 在 **GPQA Diamond** 上，**GPT-5.2** 似乎略胜 **Gemini 3 Pro**。但即使是该基准的主要作者之一 David Ryan 也表示，有时很难判断结果，因为你必须相信模型提供商没有在答案上进行训练。他过去也曾提到，可能有 5% 到 10% 的问题只是“噪音”，即正确答案并未在基准答案中真正体现。
## 六、独立基准 SimpleBench 的挑战

那么，一个完全私有、让模型提供商极难作弊的外部基准呢？我拥有自己的基准，名叫 **SimpleBench**。你可以把它想象成常识问题或涉及**时空推理（Spatio-temporal Reasoning）**的陷阱题。我大约 18 个月前设计它时，就是为了直接利用当时已知的模型弱点。

你们会很高兴知道，我几乎花光了预算，让 **GPT-5.2 Pro** 运行了五次，它得到了 **57.4%** 的分数。粗略估计，人类基线大约在 84% 左右。你可以看到，**Gemini 3 Pro** 的表现远好于 **GPT-5.2**，达到了 **76.4%**。模型提供商很难在这个基准上作弊，因为我们不会在 API 调用中给模型提供答案。我们会提取它们的答案，然后与我们自己的答案表进行比较，这个比较是由程序完成的，而不是由大语言模型（LLM）进行的。

顺便提一下，你们大多数人会使用的 **GPT-5.2** 基础版，得分是 **45.8%**。是的，如果你想知道，这是在推理努力程度设为“超高”（Extra High）而非仅仅“高”的情况下得出的。你可能会惊讶地发现它略低于 **GPT-5.1**。这在 SimpleBench 上并非首次，因为 **GPT-5.1** 本身的表现也略逊于获得 56.7% 的 **GPT-5**。对于其他模型提供商，进步则更为均匀：**Opus 4.1** 超越 **Opus 4**，**Opus 4.5** 超越 **Opus 4.1**，**Gemini 3** 超越 **Gemini 2.5**，以此类推。

如果你特别挑剔，可能会怀疑是否存在“基准测试最大化”的现象，即模型在众所周知的、备受关注的编码和数学等基准上的性能被最大化，而损害了核心参数数量和通用知识（你可以称之为模型的**通用智能（General Intelligence Nouse）**）。顺便说一句，这是一个已知的权衡。为了最大化利润率，你通常会希望用**尽可能小的模型（就参数量而言）来满足人们的期望**，这样为数亿人服务的成本更低、更容易。

仅代表我个人观点，尽管 SimpleBench 结果如此，但目前 **Claude Opus 4.5** 仍是我编码的首选模型。

## 七、实战应用与选择指南

现在，你们可能会明智地得出结论：**最好的模型，就是最适合你用例的那个**。因此，我已经将 **GPT-5.2**（基础版）添加到了 LMSYS 平台的免费层级中。你甚至可以在最高层级上访问 Pro 版本，其价格比 OpenAI 的 Pro 层级便宜近五倍。

在这个例子中，我使用了应用的自我聊天功能，让 **Gemini 3**、**GPT-5.2 Pro**、**Claude 4.5 Opus** 和 **Grok 4.1** 互相辩论，以决定哪个模型最聪明。结果可能会让你失望：它们都说对方是最聪明的。除了 **Grok 4.1** 总是认为自己是最好的之外，其他模型都同意大家是平等的。

然后我让它们各自设计一个网站。总的来说，创造最美网站的可能不是 **GPT-5.2 Pro**，我认为大概是 **Claude 4.5 Opus**。你可能知道，至少在网络开发方面，根据 LMSYS 竞技场的数据，**Claude Opus 4.5** 仍然胜过 **GPT-5.2** 和 **Gemini 3 Pro**。

## 八、长上下文能力的突破

**GPT-5.2** 上一个引起我注意的结果是，它在长上下文中回忆细节的能力。正如 OpenAI 所说，它是我们见过的**第一个在“四针挑战”（4 Needle Challenge）中达到近 100% 准确率的模型**。你需要回忆四个不同的信息点（可以想象成散落在近 20 万词文本中的“针”）。你可以看到，无论文本长度如何增加，其性能都保持在相当高的水平。如底部的数据所示，这曾是 **Gemini 3 Pro** 的绝对专长领域之一。所以，至少在 40 万令牌的范围内，**GPT-5.2** 现在有了一个竞争对手。**Gemini** 仍然可以处理高达 100 万令牌的上下文。换句话说，如果你需要中等长度的上下文（最多 40 万令牌），**GPT-5.2** 是值得考虑的；如果需要超长上下文（高达 100 万令牌），则选择 **Gemini 3**。

## 九、其他基准与行业观察

在我结束基准测试的讨论之前，还有几个结果。

- 如果你关心**递归自我改进（Recursive Self-improvement）**或奇点，那么 **GPT-5.2** 只是一个增量步骤，仅此而已。在成功完成 OpenAI 自身内部拉取请求（Pull Requests）并达到其标准方面，它获得了 55% 的分数，而 **GPT-5.1 Codeex Max** 为 53%。
- 同样，在一个对自动化 AI 研究至关重要的**机器学习工程基准**上，它比 **GPT-5.1** 好，但不如 **GPT-5.1 Codeex Max**。
## 十、行业回顾与未来展望

最后，我想对 **GPT-5.2** 发布的意义做一些更广泛的观察，并分享一个羊群的比喻。

昨天，恰好是 OpenAI 成立十周年的日子。Sam Altman 本人表示：“我相信再过 10 年，我们几乎肯定会构建出超级智能。” 当然，我们不必等 10 年才能看到他们的下一个模型。他们的研究主管表示，OpenAI 已经从 5.2 转向开发一个更大更好的模型，这得益于在 **GPT-5.2** 上汲取的经验教训。

尽管性能有所提升，但 **GPT-5.2** 通过 API 的价格涨幅却得到了很好的控制。它仍然比 **Opus** 便宜，在输入令牌方面也比 **Gemini 3 Pro** 便宜。我当然也赞赏 OpenAI 关注心理健康评估（鉴于最近的新闻），而 **GPT-5.2** 在这方面表现更好。

## 总结：羊群的比喻

将视角放得更远，许多人会有一个更基本的问题：这真的是我们通往 AGI 的路径吗？一个接一个地完成任务，一次接一次地获得增量性能提升？

首先，我不会排除性能出现阶跃变化的可能性。可以看看我最近做的关于嵌套学习和持续学习的视频。但也可以想想数羊的比喻。

你可能会看到一片起伏的广阔田野，里面满是羊，想数清它们有多少只。每一只羊都像是一项人类的事业，一项我们可能希望用 AI 自动化的人类任务。

一个团队出发进入田野，手动清点每一只羊。这有点像我们目前用 LLM 所做的事——它们在一个接一个的任务上变得越来越好。目前可能主要是数字化任务（如 GDP-vow 所示），但我昨天刚为 Patreon 采访了 Tony Zhao，他是**Sunday Robotics** 的创始人，他们是已知第一家创造出机器人模型和“Model Act One”的公司，该模型可以用易碎的酒杯装满洗碗机。他们也使用模仿数据来掌握现实世界的物理任务。

所以我画的比喻是，在勾选人类任务清单方面，我们可能已经走过了大约一半的“田野”。在 LLM 兴起之前，许多人希望有一种更“灵光一闪”的方法：某人写下一个算法，瞬间扫描所有田野，数清每一只羊——一种一次性的超级智能和奇点。

但是，即使那种灵光一闪永远不会出现，或者永远不会来自人类，而我们暂时不得不依赖增量式的进步——打破一个又一个基准，超越一个又一个人为基准——那么，**最终，最终我们也会数清所有的羊**。

祝贺 OpenAI 发布了 **GPT-5.2**。祝大家一切顺利！
