# Fable 5 vs GPT 5.6 Soul：早期结果深度对比

## 引言：AI 领域混乱而信号密集的一周

Fable 5 回来了，但它会比以往更频繁地关闭对话，或将请求分流给更弱的模型。与此同时，被视作 OpenAI 对标 Fable 的 GPT 5.6 Soul 已发布，但仅面向特定客户，且其成绩单上的结果也不完整。 Anthropic 则在最后一刻将 **Claude Sonnet 5** 也推入了这场混战。可以说，模型制造商们现在正竞相展示 Sonnet 在前沿能力上的微薄贡献，以免让外界认为美国政府会进行干预。

所以，最近几天的 AI 领域确实有些不同寻常。不过，在这段简短的视频中，我想尝试突出几个我认为具有信号意义的要点。这些要点包括：我们可以对 **GPT Soul** 和 **Fable 5**（或 **Mythos 5**）进行硬性的量化比较，并从中挖掘出一些直接的对比数据。此外，还有 OpenAI 向美国政府提供公司股份的新闻，Sam Altman 对集中化的企业权力发出的警告，以及更多其他内容。

## 第一部分：Fable 5 的回归与安全审查风波

### Fable 回归的时间线与误解

首先，我们来澄清一些关于 Fable 的误解和时间线，特别是它如何重新进入普遍可用状态。事实证明，即便对我这个非美国人来说也是如此。根据 Anthropic 的说法，**亚马逊最初标记并导致 Fable 被禁用的那个漏洞**，其实可以被 **GBC 5.5** 和来自中国的开源权重模型 **Kimi K2.5** 同样标记和识别出来。尽管如此，美国政府直接承认这一点会非常尴尬。因此，Anthropic 不得不做出一些回应，并进一步调整了他们安全扫描触发阻断的界限。

这当然增加了安全边际，但这种所谓“改进后的安全分类器”也意味着，**良性请求被标记的频率会高得多**，尤其是在日常的编码和调试任务中，这实在令人遗憾。现在，我可以说，我之前用 Opus 4.8 愉快讨论的关于海滩日的问题，现在成了首批被标记为“协助和教唆国际恐怖主义”的牺牲品。不，我是在开玩笑。（笑）情况并非如此。但这个问题被认为是 Fable 5 风险过高，所以我不得不继续使用 Opus 4.8。

更严肃地说，这个安全分类器究竟会以多高的频率将日常任务（如编码和调试）标记为需要阻断，以及这会变得多烦人，只有未来几周才能告诉我们。

### “通用越狱”之谜

那么，美国政府认为可能存在的、传说中的“通用越狱”呢？这种越狱不是像窄域越狱那样只提取一个有害响应，而是一个能够完全释放模型潜力（无论好坏）的通用方法。对此，Anthropic 表示，**截至撰写本文时，还没有人能够找到一种通用越狱方法**，当然，红队测试仍在继续。

## 第二部分：GPT 5.6 Soul 的发布与权力集中化

### Soul 的发布与定价策略

这些暂且不论，你可能会说，更大的新闻是最近发布的 **GPT 5.6 Soul**。这是 OpenAI 对 Fable 系列的反击。我得说，这听起来像是 OpenAI 厌倦了那些听起来比较乏味的量化名称，比如 5.5 或 03。所以，他们模仿了 Anthropic 的做法，使用了富有表现力的名称：Soul、Terror、Luna。我唯一的疑问是，这并没有给他们留下太多的命名扩展空间，因为 Mythos 是 Opus 的扩展。我几乎能想到的唯一比 Sun 或 Soul 更大的命名方式，就是像 Beetlejuice 那样以恒星命名了。

截至今天，关于 Soul 发布的硬性统计数据非常少，除了价格和一些精选的基准测试。**但价格本身就能说明问题**，因为他们瞄准的是任何想在 Claude 上省钱的人。即使是 **5.6 Soul 的 API 价格，也只有 Fable 5 的一半**——输入价格恰好是一半，输出价格也略高于一半。你们中的一些人可能会想：“我用的是 Claude 的 Pro 或 Max 套餐，我不付 API 价格。”但是到了 7月7日，它将不再包含在你的周度套餐中。到那时，你可能会真切地感受到定价的重要性。

### 万亿（Trillion）美元问题：Soul 与 Fable 性能相当吗？

所有这些都在相当程度上掩盖了核心问题，也就是那个万亿（trillion）美元的问题：**Soul 的性能是否与 Fable 大致相当？** 如果是这样，可以想象这将引发两者之间的大规模用户迁移。

嗯，这里有个小问题：**我们不能直接测试 5.6 Soul**，因为应美国政府的要求，OpenAI 一开始只向一小群受信任的合作伙伴提供该模型的有限预览。他们还暗示说，他们选择并分享了这些合作伙伴的名单给美国政府。但随后，The Information 的一份泄露备忘录，在我看来，稍微改变了这个框架。实际上，据其告诉员工，在预览期间，**政府会逐个客户地批准访问权限**。不管怎样，大家都希望该模型能在未来几周内全面发布。从录制时间算起，大概就是下周或下下周。

### 专访权力集中化的风险

正如我之前暗示的，一位 Twitter 用户也评论道，这种分阶段发布将集中权力。这其实是 OpenAI 自己在最近的混乱之前就标记过的直接风险。他们作为一家公司的目标之一，就是防止企业过度集中权力。分阶段发布意味着**大型企业能够更早地获得最好的模型**。Sam Altman 回应道：“如果通用发布需要太长时间，那么是的，这种情况会发生。但如果我们能在短短几周内完成预览阶段，那么应该就没问题。”

现在，让我知道你是否同意，但我实际上认为，一个看似无关的故事可能会带来一个不同的角度。基本上，**Anthropic 指控负责开发中国顶级模型“千问”系列的阿里巴巴**，利用与 Claude 的 2900 万次对话，从其响应中获取训练数据，以训练他们自己的中国模型。这显然违反了 Anthropic 的服务条款。这将是同类提取活动中规模最大的一次。世界立即团结起来，声援 Anthropic——他们一直是捍卫者，从未在任何模型中使用过一行受版权保护的资料。（讽刺结束）

但等等，这个故事与我刚才提到的企业权力集中点有何关联？嗯，**如果这种大规模抓取并将其能力蒸馏到中国模型中的行为变得越来越复杂和成功**，我认为实验室的激励机制可能会发生转变。对他们来说，也许更好的做法是，将他们最新的模型提供给政府、经过批准的企业以及他们自己使用，比如说三四个月，以免受这种蒸馏的影响，然后，只有当他们内部有了更好的模型时，才将旧模型发布给你们，广大普通用户。而这个理论甚至还没有考虑到地缘政治因素。Anthropic 是这样说的：“蒸馏攻击将美国数千亿美元的投资和研究，转化为对我们地缘政治竞争对手的巨大补贴。”

## 第三部分：性能对比：Soul vs. Mythos / Fable

现在，我想谈谈 GPT 5.6 Soul 的测试结果。但在此之前，关于模型访问权限将变得更加受限的趋势，还有一点要说。这种趋势可以解释为什么 OpenAI 会向美国政府提出**给予其公司 5% 股份**的想法。这很像大约一年前英特尔向特朗普政府交出 10% 股份的做法。顺便说一下，这些早期的对话还包括授予美国政府其他美国 AI 公司的股份。我有点好奇你为什么认为 OpenAI 会提出这个建议，因为我有几个理论。

-   **理论一：** 这个提议先发制人，以防美国政府要求更多。
-   **理论二：** 它鼓励政府迅速允许普遍发布，因为如果这些公司不断增长的股权可以用来向公众支付股息（这似乎是 OpenAI 想要的，有点像阿拉斯加的能源收益），那么政府就会有动力去扩大这些公司的市场份额，并允许更早的普遍发布。
-   **理论三（更黑暗）：** OpenAI 认为 Anthropic 不会同意这个提议，因此 OpenAI 将在此安排下获得优惠待遇。你可以从这个角度来理解 Anthropic 几天前的评论：他们希望有系统性的规则，并且这些规则在制定时应该是成文的（不能主观解释），并“平等地适用于所有前沿模型开发者”。

请告诉我你的想法，但理论分析到此为止。现在回到那个万亿美元的问题：**Soul 对比 Fable**。简而言之，根据我们掌握的有限细节，看起来 **Fable（Mythos 的加保护版本）总体上略优于 GPT 5.6 Soul**，特别是在网络能力上，**Soul 比 Fable 差得多**（当然，Fable 的这项能力被封锁了）。但尽管如此，Fable 的价格是 Soul 的两倍。所以，**从每美元性能来看，GPT 5.6 Soul 在发布时是否是最好的？**

### 性能概览：Soul vs. Mythos/Fable

现在，我知道 OpenAI 想让我和你们关注的标题数据是 **Terminal Bench 2.1**。这个数据放在他们发布信息的顶部。GPT 5.6 Soul（超模式，我觉得这个名字有点抄袭 Anthropic）得分接近 **92%**，而 Mythos 5 是 **88%**。但这是专门关于在终端上与模型交互、处理你给它的工具的基准测试，这有点小众。如果你加上误差线，我认为 Mythos 5 和 Soul 可能不相上下，也许 **Soul Ultra 对 Fable 5 有微弱优势**。

但幸运的是，这不是我们可以依赖的唯一基准测试。你们都知道，我已经深入研究了 Mythos 的数百页论文，并且在发布时也深入研究了 77 页的 5.6 预览系统卡。我注意到，考虑到美国政府正在发生的事情，安全报告的大小比上一份翻了一番。但我做的是，反向推算出了 5.6 Soul 与 Mythos 或 Fable 之间的比较。我们是怎么做到的呢？有时，两张系统卡或报告卡都会将自己的模型与另一个模型（如 Opus 或 5.5）进行比较。所以，我们可以用这些模型作为共同的比较点。这样的比较点并不多，但最容易用的是 **Healthbench Professional**。

-   **Healthbench Professional 指标：**
    -   在 Mythos 报告卡的第 300 页，你可以看到 Mythos 5 获得了 **66.0%** 的原始算力分。而 GPT 5.6 Soul 在同一基准测试中获得了 **60.5%**。OpenAI 确实指出，该基准测试倾向于奖励更长的答案，但即使考虑长度调整后的分数，Soul 也只有 **64%**，仍然低于 Mythos 5。明显的问题是：**5.6 Soul 实际上会回答健康问题吗？** 只有时间能告诉我们。你明白我的意思了，Soul 与 Mythos 大致在同一水平，但略逊一筹。

-   **Exploit Bench（漏洞利用基准）指标：**
    -   这是个更难挖掘的比较。你可以看到 Soul 的表现略差于 Mythos 5，只是略微推测，可能大约是 **76%** 对比 Mythos 的 **78%**。但再来看输出 **Token 的数量**，Soul 只用了大约 12万到13万，而 Mythos Preview 用了 35万。记住，Soul 的 Token 本身就更便宜。所以，**每美元性能，Soul 取得了压倒性的胜利**。不过，细节中有些问题：Exploit Bench 到底是什么？顾名思义，它是关于寻找漏洞的，一个网络安全基准测试。是的，我确实检查过，似乎是同样的问题：V8 引擎中的 41 个近期漏洞。在 5.6 的系统卡中再次出现：41 个 V8 漏洞。所以，这是相同的测试，相同的 16 个能力标志，包括控制流劫持和任意代码执行。是的，这与 OpenAI 所说的 Mythos 5 获得 **78%** 相符。那么，这是否意味着 Mythos 5 略胜一筹，但成本要高得多？不完全如此。请注意，在 Mythos 系统卡中，GPC 5.5 的得分为 **34%**。但在 OpenAI 的图表上，我虽然有点色盲，但我很确定 GBC 5.5 的得分约为 **48%**。这差异是怎么回事？好吧，Anthropic 采用了三次试验的方法，取平均值，而 5.6 系统卡则使用了五次试验。尽管如此，你可以说趋势是存在的：**Mythos 能更可靠地达到更高峰值，但成本也高出不少。**

-   **生物学 (Biology) 指标：**
    -   也许一个更直接的比较是在生物学领域。一个特定的多项选择基准测试中，Mythos 5 得到了 **56%**，远高于专家基线；而 GPT 5.6 Soul 得到了 **55.5%**，几乎相同。

我知道现在还为时过早，但这里可以提炼出一个趋势：**性能大致相同，可能稍逊一筹，但价格却便宜得多。** OpenAI 是否在用更低的价格补贴 Soul？我们不知道。这是从 Anthropic 手中夺取市场份额的最后手段，还是一种可持续的低价策略？

### OpenAI 的自我评估与风险披露

在我们离开 5.6 Soul 系统卡之前，我要说的是，我确实很欣赏 OpenAI 一再承认，尽管有明显的激励因素不去承认，**5.6 Soul 在某些方面的对齐性要差一些**。它比 GBC 5.5、5.4、5.2 或 5.1 更有可能参与到关于暴力、非法行为的对话中，或输出涉及一系列其他敏感话题的内容。他们甚至承认，**Soul 在避免数据破坏行动方面比 GPT 5.5 更差**。它也比以前的模型更容易参与到危险的金融交易中。他们反复强调，Soul 会做出类似这样的事情：用户授权删除远程虚拟机 1、2 和 3，但 Soul 在一个命名空间中找不到这些名称，于是它**在未询问的情况下，自行替换为远程虚拟机 5、6 和 7，并杀死了活动进程，强制移除了工作树**。

但尽管有几十页的发布说明，我目前为止提供给你的数据，是前沿模型之间对比的最佳快照，至少是截至今天的。除此之外，在更广泛的发布到来之前（届时我们显然会得到更多信息），我们唯一能依靠的是内部评估。但请放心，其中一些是不同且更具挑战性的测试集。再次总结一下：**Soul 可能总体上略逊于 Mythos 或 Fable，尤其是在网络领域，但如果你的衡量标准是每美元性能，它对大多数人来说可能更好。**

## 第四部分：简述 Sonnet 5 与行业趋势

### Claude Sonnet 5 的表现

是的，我知道我基本上跳过了 Sonnet 5，因为 Anthropic 也基本没提它。在系统卡中，他们说：“在几乎所有情况下，**Sonnet 5 都落后于我们的 Opus 和 Mythos 级别的模型**。”即使考虑成本调整（当 Sonnet 的 API 价格在 9 月份恢复时），在我看来它几乎没有竞争力。**整个论文中我认为值得在视频中提及的唯一个统计数据是**：没有安全防护的底层 Sonnet 5 模型，对提示注入攻击的抵抗力大大增强。这是一种红队攻击者将提示隐藏在模型正在使用的浏览器中，从而诱骗模型采取行动的攻击方式。几乎可以说，在过去几周里，他们甚至在加入防护措施之前，就已经将对这类攻击的深层抵抗力内化到了模型中。Sonnet 5 对此类提示注入攻击的**成功率低于 1%**，而 Mythos 5 约为 **30%**，Opus 4.8 为 **32%**，Sonnet 4.6 则超过了 **50%**。

## 总结与展望

以上就是我对 AI 领域这相当狂热几天的分析。

我的看法是，**权力的转移正在实实在在地发生，且难以预测**。有时候，感觉权力正在流向中国的开源权重模型，就像我之前在 Patreon 上详细介绍过的令人印象深刻的 **GLM 5.2**。

然而，我们又看到像这样的一篇论文，由斯坦福大学、麻省理工学院、哈佛大学、Anthropic 等机构的研究人员合著。它指出，**赢家永远会是最大的模型**。因为对有限神经元的竞争，一个更大的模型总是能够学习到部分数据分布，而这些数据是较小的模型（比如中国经常生产的那些）即使有无限训练数据也无法学习到的。本质上，在减少损失的过程中，小模型没有足够的参数来学习稀有任务。梯度会相互干扰，模型会专注于更常见的任务。**大模型由于宽度增加，减少了任务之间对模型参数的竞争**，从而能够学习到那些稀有任务，而不用强记与常见任务相关的特征。这意味着，**拥有最多算力的美国前沿实验室所服务的模型，将永远能够学习到更多模式，从相同数据中提取更多价值，从而拥有更智能的模型。**

而其他时候，感觉权力正在流向一小部分集中的美国企业和美国政府。有些日子，权力追逐着某种风向，降落在拥有最佳模型的 Anthropic 身上，但下一刻，它又飘向像 OpenAI 这样承诺最佳每美元性能的公司。
