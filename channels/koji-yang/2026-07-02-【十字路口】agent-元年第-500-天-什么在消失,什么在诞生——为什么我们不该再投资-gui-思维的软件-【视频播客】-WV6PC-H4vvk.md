2024 年 1 月 1 日，Koji Yang 与两位朋友录了一期名为“Agent 元年”的播客，那时大家还在猜测未来会怎样。500 天后，他们再次坐在一起，发现世界已经变得认不出来——不是因为什么惊天动地的发明，而是因为所有软件的存在方式都在悄悄解体。当模型拥有了近乎无限的上下文窗口，当命令行接口（CLI）因为 AI 而生出了“眼睛和手”，当一个叫“无头”的概念变得理所当然——我们终于看清了那些正在消失的和正在诞生的东西。这场对话的价值不在于预测未来，而在于它精确地画出了一条分界线：在这条线的一边，是过去二十年我们对 GUI 的全部信仰；在另一边，是 AI 原生世界里关于“界面”的全新定义。真格基金的天杰 Jack 用一篇文章激起了巨大的讨论——《我们或许不应该再投资 GUI 思维的软件》。这 500 天到底发生了什么？为什么一个投资人会说出这样的话？而他口中的“无头软件”“CLI 新浪潮”“Skill 重组”到底意味着什么？本文基于播客的全部内容，为你一层层拆解这一切。

# 详细内容

## Agent 元年 500 天：超出预期的疯狂与尚未填补的空白 `[章节 1]`

### 核心观点
过去 500 天，Agent 的能力发展远超最乐观的想象——它让一个前端开发者能维护几十万行代码，让一个曾经坚持不用 AI 写内容的人把整条内容管线全部自动化。但同时，市场上那些直接从 GUI 时代平移到 AI 时代的产品，尤其是信息消费类产品，依然没有给出令人满意的答案。

### 深度阐述

Koji 在开场抛出了一个直截了当的问题：这 500 天的发展，是超出你的预期，还是不及预期？天杰的回答带着一种投资人的冷静反思：“现在市场上绝大多数产品仍然是以人作为最终使用者来设计的。你还是要不停地把自己的注意力灌入一个产品，才能很好地使用它。至少从信息消费这个类别来说，我还没有看到一个让我自己满意的产品出现。” 这句话道出了一个悖论：**Agent 已经可以替人完成很多事，但市面上最主流的工具却依然在把你按在屏幕上，让你手动消耗内容。** 这是天杰口中的“低于预期的部分”——不是因为技术不够，而是因为软件产品的哲学没有跟上。

臧师傅的回答则完全是另一个方向：“事实上，我觉得整体都是高于预期。我如果在当时，根本无法想象现在这个状态。你无法想象现在的工作状态和效率提升——它极大地拓展了我能做的事情。” 他举了两个具体的例子：“从 Manas 开始，我发现自己居然能维护一个几十万行的代码库，还给这么多人提供服务。这对一个只懂基础前端的人来说，以前是完全不可能的。另一个不可能是我现在的工作流程自动化高到了这个程度。之前我坚决不用 AI 写内容，我相信很多做内容的都有这种坚持，但现在我发现这种坚持没有意义。”

为什么坚持会消失？臧师傅的解释带着深刻的洞察：“对于公众号写作来说，它的本质就是把信息传递给你。什么样的方式传递效率最高、摩擦最低，它一定会流向那个方向。当然如果我要写小说，写比较有文学性的东西，还是需要人自己去写，因为 AI 达不到。但如果说你只是在表达观点、写新闻，AI 有时候比我写得好。” 他说自己最近看到一篇最没有“AI 味”的文章叫做《置身钉内》——这句话揭示了当前内容领域的一个诡异现实：**大家都在试图避开 AI 味，但大多数人已经离不开 AI 了。**

当 Koji 让每个人用一句话总结过去 500 天 Agent 行业的状态时，三个人给出了三个极其精准的角度：

天杰说：“变得不方便。” 紧接着补充：“每个东西都在变，除了上下文（Context）越来越重要。”

臧师傅说：“不变的是模型能力的进步会持续解锁一些我们之前没有想到的技术奇观。”

Koji 则引用了一个经典笑话：“所有还没有被驾驭的技术，我们都可以叫它 AI；等它真正稳定高效地落到工作流里，我们就说那不过是软件而已。不变的就是始终有一些新的概念在被去魅。”

三个人最终汇向同一个词：**上下文（Context）**。“不管是模型本身的上下文，还是 Agent 里面的上下文管理，所有东西都回到了 Context 里面。” 这个概念成为理解随后所有讨论的关键：当模型可以记住更长、更动态的对话历史和信息拼图，软件界面存在的意义开始被抽空。

### 个人感受

臧师傅最动人的地方在于他毫不掩饰自己认知变化的轨迹。从“坚信自己写内容才是对的”到“完全把内容管线交给 AI”，这种转变不是妥协，而是对工具理性的一种通透认知。但同时他保留了底线——纯文学依然要人写。这种“分化”恰恰是我们这个时代最诚实的注脚：**工具可以取代技术性表达，但取代不了具有个人风格和情感张力的创作。**

### 延伸思考

天杰的“低于预期”其实是一种更高层次的要求。当 Agent 本身已经能替人类消费信息时，市面上却依然用推荐流、订阅列表来敷衍用户——这暗示着一个尚未被充分开发的巨大市场：**下一代信息消费产品不太可能是一个更好的 RSS 阅读器，而应该是一个能理解你此时此刻认知需求的 Agent。** 谁填补了这个空白，谁就有可能成为 Agent 时代的“微信”。

## Headless：软件只剩一副“身子”，GUI 是人类认知缺陷的补丁 `[章节 2]`

### 核心观点
“无头软件”意味着一个软件不再优先考虑人类用户的操作界面，而是优先考虑被 AI 调用。GUI 在 AI 执行任务的场景下变得不再重要，甚至成为负担——因为 GUI 的终极本质，就是为人类有限的注意力和记忆力打上的补丁。

### 深度阐述

“Headless”——软件没有了头。没有头的意思是：没有了界面，只剩下数据库和一些工具。这个概念因为天杰那篇刷屏的文章《我们或许不应该再投资 GUI 思维的软件》而成了行业共识。三个月后，Koji 当面问他：你的想法改变了吗？

天杰首先做了一件事非常重要的事：重新定义他文章的讨论范围。“当时我之所以加上‘思维’两个字，是因为我们之前有很多效率工具，它们的设计是为了方便人使用。但当 Agent 的能力超越了部分人类时，GUI 对这类以 Agent 为本位的工作流来说，就变成了一个额外的负担。**但不是说不重要。** AutoGPT 和很多框架类的产品诞生在命令行里，用户的接触门槛很高。最终出圈的像 OpenAI 的 Cloud（指 Cursor 或类似产品？）和 Manas，本质上都是非常好的 GUI。所以 GUI 本身的价值并没有退潮，反而因为它能封装更多的能力，用户需要更好的 GUI 来接触这些能力。”

随后他说出了那句最锋利的话：“当时我在文章里面说，**GUI 是人类认知缺陷的补丁。**” 他展开的逻辑极其清晰：“GUI 本质上是基于人类注意力的机制去优化的产物。人类天生就 context 有限——最多只能记住七个并列的单词，要记住第八个第九个，认知挑战陡增。所以像 Notion、Figma、Slack，它们的用户体验做得那么好，本质上是在帮人类克服这些缺陷。一旦某件事物不再需要人类参与的时候，它就不需要再为人类的缺陷去服务了。所以在生产效率的场景里，当人不再长时间聚焦工作时，GUI 就变得没那么重要。”

**正反案例：为什么 Notion 很痛苦，而飞书 CLI 和 Gmail + Codex 很流畅**

Koji 作为一个十年 Notion 重度用户，说出了很多人的心声：“最早用印象笔记，后来换到 Quip，再换到 Notion——你会发现一个趋势：在 GUI 思维下，软件的复杂度层级越来越高。直到 Notion 用 Block 的方式组织信息时，它达到了 GUI 思维的巅峰。但在最近的几个版本里，它在强行往一个已经很极致的 GUI 产品上加 Agent 和 AI 概念，这是在破坏老用户的价值网络，而不是开创新的价值。” 臧师傅接话：“两边人都会很难受。”

正面的例子在哪里？天杰提到了自己的实践：“我通过 Codex 授权了 Gmail。之前我花了大量精力去设置标签规则——标题有关键词的标 A，促销的跳过 inbox 不提醒。但维护这套标签很快就崩溃了，因为你订阅的速度永远超过你维护标签的速度。后来有一天我无意设了一个 filter，导致所有邮件都不出现在 inbox 里了。这在两年前，我每周需要花三个小时去清理 inbox，把所有信息分门别类打上 tag——**这本身就是人类使用 GUI 时一个巨大摩擦的体现。**” 现在他的做法是：每天早上一个定时任务，通过 Agent 把过去 24 小时的所有邮件读完，然后用自己的方式告诉他什么不能错过。“所以某种意义上，Gmail 的 GUI 就‘死’了——它把自己内部的能力献祭给了 Codex。我不再打开 Gmail 这个界面，而是让 Agent 替我去访问信息。”

臧师傅补充了 Codex 在体验上的细节：“它能在 Chat 页面输出视频，里面嵌了视频播放器，一点就能放。图片也融合到了信息流里，先输出图片再说一句话，图文混排。文件夹直接是一个链接，点一下就打开。这些设计其实仍然是一种 GUI，**但它们是为 Agent 的输出而设计的，不是让人去点按钮的。**”

### 个人感受

天杰在分享那篇文章受到的负面意见时，说很多人觉得他太极端了。但随后他话锋一转，聊到了 BP（商业计划书）的 PPT：最近收到了很多 BP，连一张图上的中文都还是半年前那种 AI 生成的有乱码的效果。“一个做 AI Native 产品的创始人，居然不用最好的模型来给自己的产品做展示图，这本身就有问题。在这些礼仪性的场合，BP、官网，是你的门面。你用不用 AI 不重要，但你必须把它打磨好，**你得把你的心血注入进去。**” 他提到了星巴克创始人舒尔茨写的《将心注入》——当公司变大后大家不再认真做咖啡的时候，用这本书重拾士气。这段话说出了一个重要但容易被忽略的观点：**Headless 和去 GUI 是针对 AI 工作流说的，当你要面对投资人、面对用户建立信任时，Human-to-Human 的界面（PPT、官网）依然需要极致的打磨。** 你的品味和用心，是 Agent 无法替代的。

### 延伸思考

Headless 并不等于“不要交互”。交互的对象从人类转向了 AI Agent，意味着你投资软件时必须先问一个问题：**这个产品的最终“用户”是谁？** 如果它是一个被 AI 调用的后端服务，API、数据结构、CLI 才是核心。如果它最终服务于人类信任（比如融资、品牌），GUI 依然决定生死。这可能是未来产品设计中最难把握的平衡：你如何在一个产品里同时让 AI 和人类都满意？也许答案是——把它们拆成两个产品。

## CLI 复兴：当命令行遇上了“全记忆” AI `[章节 3]`

### 核心观点
CLI（命令行界面）因为 AI 变得可以被所有人使用。CLI 是纯文本的，AI 也是纯文本的，这构成了天然的匹配。越来越多的消费级和企业级软件开始推出 CLI，标志着 Agent 时代的新入口竞争已经打响。谁掌控了 CLI，谁就可能掌控 AI 时代的第一触点。

### 深度阐述

“对于普通用户来说，理解 CLI 其实很简单。” 臧师傅用最直白的方式做了解释：“GUI 是你在软件里点按钮，它给你视觉反馈。CLI 是通过命令来执行的，纯文本，你给它一句话，它就帮你剪完视频。但那句话需要固定的格式。” 过去为什么普通用户用不了 CLI？不是因为不想用，是因为记不住。FFMPEG 的创作者自己也记不住自己写的所有命令——那个文档上千页。但是 AI 能记住。**AI 天然擅长文本理解，它能记住并执行任何 CLI 命令。** 所以你只需要说“把视频码率降到 1080p”，AI 就会自动翻译成你根本看不懂但机器能跑的那串代码。

这就解释了为什么飞书、Google Suite，甚至连瑞幸和肯德基都开始推出自己的 CLI。Koji 举了一个非常生活化的例子：“以前你要去小程序点单，现在 AI 顺手就帮你点了。瑞幸和肯德基这么做，是想在 AI 时代拥有自己的专属渠道，摆脱之前被美团等第三方渠道的‘剥削’。” 臧师傅说自己实际用过，体验非常好。

但大部分垄断级软件依然没有推出 CLI。天杰列出了名单：“微信、美团、滴滴、携程、飞猪，都没有。” 为什么？因为一旦推出 CLI 就好像用户入口不在我这里了，变成了一个无头服务提供商，别人还能用 CLI 去拿你的数据。**这需要极大的决心和自信。** 所以天杰对那些已经主流、依然开放 CLI 的公司——Google、飞书——表达了极度的敬佩。

**最希望谁开放 CLI？**

当 Koji 要求每个人许一个愿、说一个最希望开放 CLI 的名字时，臧师傅毫不犹豫地说了两个字：“微信。” 他说：“太痛苦了。你在所有事情都可以 AI 自动化的时候，你跟人的沟通、客户的沟通、上下游的沟通，依然在用非常痛苦的 GUI 方式——点进去、打字、回复。如果能用 CLI 接入，会完全不一样。”

Koji 问：“要不要给张小龙喊个话？” 臧师傅无奈地笑了笑：“算了，太难说服他了。” 但他提出了一个很有意思的“侧翼竞争”思路：“也许我们不再期待传统服务转型，而是希望出现完全基于 Agent 的新服务。比如如果有一天我们仨在豆瓣上互相加了好友，而且豆瓣开放了 CLI 让我们的 Agent 可以去读聊天记录——那我可能真会把一部分对话迁移到豆瓣上去。微信如果感受到了用户流失，也许会被迫行动。”

天杰补充了一个真实正在发生的趋势：“现在我和我身边的年轻人开始在抖音上聊天了，而且频率越来越高。抖音的聊天 DAU 已经是一个亿了。扎克伯格在内部分享时说低估了 TikTok 的冲击，他以为那只是一个内容平台，没想到大家会在上面聊天——因为看到内容、分享给朋友，自动就开启了对话。这直接冲击了 WhatsApp。**内容消费在哪里，聊天就会在哪里。**” 这对微信是同样的威胁。如果微信不能很好地支持 AI 时代的交互方式，它的用户关系链有可能被慢慢侵蚀。

### 个人感受

说到微信时，三个人都带着那种又渴望又无奈的语气。微信是中国互联网最成功的产品之一，它的封闭生态让它极度强大，但也让它成为 Agent 自动化链条上最大的一个“UI 孤岛”。一个让人心情复杂的事实是：**越是成功的旧时代产品，越没有动力去开放 CLI，因为那意味着主动交出自己赖以生存的用户界面。** 这反过来给了创业公司一个清晰的信号：如果你能提供一个开放 CLI 的通信或服务工具，并且能让 Agent 无缝接入，你就有机会切下微信、美团们不愿意放弃的那块蛋糕。

### 延伸思考

消费级 CLI 的出现，意味着**任何企业现在都拥有了一个全新的“渠道选择”**：是继续在别人的超级 App 里（如小程序生态）做争夺流量的奴隶，还是自建 CLI 成为 Agent 时代的第一入口？瑞幸和肯德基的做法给出了一个示范：拥抱 CLI，意味着你拥抱了 Agent 背后的整个用户群——因为 AI 会在用户想喝咖啡时，自动调用你的 CLI 帮你下单。这不是损失，而是一种新型的发放和转化。

## Skill：Agent 时代的“微程序”与永不消失的品味 `[章节 4]`

### 核心观点
Skill 是当前 Agent 领域最具魔力的概念——它把重复性任务封装成可复用的能力单元。虽然长期来看模型会消化掉 Skill 的显性存在，但在可预见的未来（半年到一年），Skill 依然是提升效率最直接的方式，而且它的价值高度依赖于用户品味的调配。

### 深度阐述

臧师傅的 PPT Skill 是这一轮讨论的起点。“第一个版本是分享前我给了它一句话，它基于我的 memory 和我以前的代码项目，从里边掏了点样式，整合成了一个符合我审美的 PPT。第一个版本就已经非常惊艳了。” 天杰分享了他的“Grill Me” Skill：“当你有一个问题想不明白的时候，你让它来向你提问。它会用各种角度榨干你，把你脑海里的想法引导出来，最后帮你做出更好的决定。” 这本质上是元认知的 Agent 化——让 AI 成为你的苏格拉底。

Skill 是否会长期存在？讨论进入了最深刻的阶段。臧师傅认为：“长期来看，模型会逐渐把 Skill 吃掉，变成不用复杂提示就能出现的结果。就像 GPT-4 时代生成好图片要写很长的提示词，GPT-4o 一句话就能给你很好的结果。Skill 这边会慢一些。但 Skill 已经成为了共识——一旦成为共识，它就会被接受、被消化，存在时间会比想象的长。” 天杰则从用户角度提出了不同的看法：“最后的主体应该是一个帮你完成某个任务的 Agent 品牌。你信赖它能完成这件事，至于它用了什么 Skill 组合——**人类很难去分辨两个 Skill 的细微差异。** 你奶奶不会去安装测试 Skill，但如果有一个 Agent 帮她煮咖啡、帮她读邮件，她信任的是那个 Agent。所以 Skill 很可能是过渡概念，最终会被‘Agent 能力’本身覆盖。”

关于 Skill 能否自动沉淀和进化，现场给出了一个清醒的回应。臧师傅说：“AI 可以帮你总结重复流程生成 Skill，但它缺少了你调整的过程。它生成的很可能不符合你的实际情况，因为它在猜。所以最后还是需要人去调。Codex 的做法其实很好：它根据你的聊天记录，发现重复的部分，帮你总结成五六个 Skill，然后问你装不装——这个流程成立，但依然需要你参与。”

### 个人感受

当臧师傅说“我不用别人的 Skill，都是自己做的”时，现场笑出了声。这暴露了 Skill 生态的现实：它目前还是极客和高阶玩家的游戏。天杰的“奶奶测试法”一针见血——任何“需要用户去理解、去选择”的设计，都成不了大众市场。一个真正的 Agent 产品，应该在后台自动帮你管理 Skill，而不需要让用户知道 Skill 的存在。**用户要的从来不是工具链的复杂性，而是“我想要的现在就得到”。**

### 延伸思考

Skill 和 App Store 的逻辑非常相似。App Store 的出现让手机变成了一个“安装应用”的设备；Agent 时代的 Skill Store 会接棒，让 AI 变成“安装能力”的设备。但问题在于：当模型本身可以理解上下文并动态生成能力时，为什么还需要用户去手动“安装”一个 Skill？答案可能是：**Skill 是对权限和品味的显式声明。** 你安装一个“帮我读邮件”的 Skill，等于授权 Agent 去访问你的邮件箱并对它进行某种风格的处理。这种显式授权在安全性和个性化上的意义，是模型自动推测无法替代的。

## 将心注入：AI 时代的人类 GUI 美学 `[章节 5]`

### 核心观点
尽管我们不再为 AI 优先投资 GUI，但在人类面向人类的交互场合——BP、官网、播客封面——GUI 依然是信任和品味的第一道门槛，绝不能因为 AI 而偷懒。

### 深度阐述

这是播客中最容易被忽略但最重要的一个反差段落。天杰在批评 BP 中的粗糙 AI 生成时说：“PPT 本质上也是一种人与人之间的 GUI。如果连这个都不打磨，我觉得有问题。我昨天看一个 BP，图上的中文还是半年前的 AI 生成效果。一个做 AI Native 产品的创始人，居然不用最好的模型来做图，这也反映了问题。”

臧师傅划了一个非常精准的标准：“**用 AI 生成的内容，不要让人看出来是 AI 生成的。**” 你可以用 AI 提高效率，但最终的交付物必须经过你的眼睛和手。星巴克创始人舒尔茨在《将心注入》里写的是同一个道理：当企业扩大后大家不再认真做咖啡时，你需要的不是更好的咖啡机，而是重新找回大家对咖啡的敬畏和用心。**在 AI 时代，这种用心正在变成一种稀缺而且容易被识别的竞争力。**

### 个人感受

这段对话让“无头”“CLI”“Skill”这些冷硬的词汇一下子有了温度。一个全 AI 化的世界如果失去了人类品味的“打磨”，就只是一个高效但乏味的机器。三位嘉宾在用实际行动表明：**他们反对的是“为 GUI 而 GUI”的思维定式，而不是反对交互本身的品质。** 真正优秀的创业者，都会像真正的匠人一样，把心注入到那些需要与人连接的地方——无论是 PPT 上的排版、官网上的文案，还是播客里的声音。

### 延伸思考

未来的优秀产品经理和创始人可能需要双修：**左手打造 AI 原生的 Headless 后端，右手打磨人类共情的前端美学。** 这两者不矛盾，恰恰是时代给每个人提出的新要求。你能让 Agent 工作得多流畅，决定了你的产品上限；你又能让人类认可你、信任你，决定了你的产品能不能在市场上活下来。

---

# 精华收获

1. **不要再以 GUI 思维去创建新软件了。** 尤其是当产品的主要使用场景由 AI 驱动时，核心应该放在 API、CLI、数据结构上，而不是界面的点、划、拖。

2. **CLI 因为 AI 而迎来了它的黄金时代。** AI 记住了所有命令，普通人只需说出意图。消费级 CLI（瑞幸、肯德基）已经开始成为 Agent 时代的流量入口。

3. **Skill 是当前 Agent 落地最快的“甜点”。** 把重复任务封装成 Skill，效率能提升 10 倍以上。但长期来看，Skill 会被模型逐步内化，最终品牌化的 Agent 才是终点。

4. **保持红线：面对人类时，GUI 必须用心。** BP、官网、产品演示——当你和一个活生生的人建立信任时，你的品味和细节不可被 AI 替代。

5. **谁愿意开放 CLI，谁就可能拿到 Agent 时代的连接者红利。** 微信只是一个极端的例子，但它暴露了一个结构性问题：封闭系统在 AI 时代会面临被 Agent 绕过的巨大风险。

6. **上下文（Context）才是新一代产品的核心。** 不再是一行代码、一个界面，而是能用多少动态信息来理解用户的真实需求——这将是所有 AI 时代产品的终极护城河。
