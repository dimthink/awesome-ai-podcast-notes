## 详细内容

### 一、从 M1 到 M3：吞吐量背后的“战栗”与战略定力 `[对话第一部分]`

**核心观点**  
模型系列从 M1 到 M3 的突破并非一帆风顺，最客观的指标是吞吐量：M2.7 实现了日消耗 token 从 1 万亿到 10 万亿的十倍跨越；而 M3 的目标更进一步——让所有人可以无限制、不关心成本地使用模型。

**深度阐述**  
当主持人询问“从 M1 到 M3 哪些突破是虚构的”时，闫俊杰（IO）直率地回应：“其实突破是还没有来到预期。” 他选择了一个极简的测量标尺——吞吐量。“M1 的时候吞吐量其实比较低，M2 坦白说效果不太好，” IO 说，“但我们在做 M1 时就感受到了那个东西。” 他描述的是一种“经典的桥段”——瘫坐在椅子上，看着后面又发了一辆车。那一刻发生在去年五一假期，他们第一次跑通神经网络时。“那个时候就感觉第一次感受到了这个东西，” 但模型并没有成功。这种先验的“感觉”在后来被验证：当团队在 M2 时期面临巨大的内外质疑——所有人都在关注 M Arena 排名，团队却做出一个看似偏执的决定：**只做 Coding Agent**，不做其他无关对话。“我们真的是这么决定的，” IO 回忆，“这个决定让开发者有很多质疑，为什么不管这个东西？” 但正是这种聚焦带来了超预期的结果：M2 刚上线时目标是日消耗 1 万亿 token，到 M2.7 时达到了 10 万亿，十倍于目标。M3 的目标更加宏大：**实现无限成本体验，让所有人无负担地使用模型**，再加上整个行业的大幅增长，总量能到可观的量级。“我们非常接近了。”

这背后是战略的定力：在行业浮躁地追逐各种评测指标时，MiniMax 抓住的核心是“用起来”。吞吐量代表的不是技术炫耀，而是真实的用户黏性和生产关系重构。IO 在另一处补充道：“AI 的核心生产力是提供工具，就像发明汽车让赶路变快一样。” 模型与 Agent 的进化不是互斥的，而是共同进步。“如果没有 Cloud Code，模型的 Agent 可能也不会那么火；没有 GPT-3.5，Code Agent 也做不起来。” 从这个角度看，M 系列的演化是一条双螺旋：能力提升推动应用场景，应用需求又反过来定义模型的优化方向。

**个人感受**  
IO 在回忆第一次跑通神经网络时的“战栗”是非常个人化的。那种先知先觉又无法言说的感受，几乎是所有技术开创者的共同心跳。他坦诚地承认“去年这个时候想不到现在的模型什么样”，这既是谦逊，也道出了 AI 行业充满非线性跃迁的本质。

**延伸思考**  
注意 IO 提到的“聚焦”策略——在 M2 时只做 Coding Agent。这是典型的“单点突破”方法论。在资源有限时，拒绝诱惑比选择方向更难。对于创业者而言，这种“战略减法”在 AI 领域尤其关键：模型能力的边界尚不清晰，过早的多元化可能意味着全面的平庸。

### 二、AI 编程的“普惠”与“工程”之辩 `[对话：嘉园、河套讨论 Engineering]`

**核心观点**  
AI 编程不应停留在“Vibe Coding”的一次性交付，而应回归软件工程本质——长期可维护、可持续的生命周期。当前的 Benchmark（如 SWE-bench）只考核一次性解决能力，严重缺失对代码维护、架构演化的评估。

**深度阐述**  
嘉园首先指出一个历史周期：第一代 Vibe Coding 用户是产品经理，他们通过驱动程序员来“编程”；而今 AI 让每个人都成了产品经理。但真正优秀的决策标准不是“做多少事”，而是“不做哪些事”。当 AI 可以帮你完成一切代码生成时，**筛选与放弃**的能力变得前所未有的重要。

河套则抛出了一个“暴论”：“Vibe Coding 从来没有人说是 Vibe Engineering，但写代码从来都是 Engineering 工作。” 工程师的职责不是一次性交付，而是长期交付且有生命力的系统。他批评当下的 Benchmark 文化：“模型很容易变成在 Coding 任务上使命必达——就像独立的人一样，但最后你发现他做完了，Just so so。而且后面的人根本接不了他的代码。” 因此他推荐回归经典——Clean Code 和 Code Complete 中的原则，将这些“先哲的东西”凝练成 Skill，配合 Agent 赋予其 Context。“每个人写出来的代码跟他的性格很像。你还要加入公司的 Best Practice。” 他特别讨厌一句话：“这是 Agent 做的，你不要怪我。” “它用的是你的账号，代表着你的责任心和对世界的态度。”

**个人感受**  
河套的情绪非常真实。当一个人投入大量时间去构建、维护一个开源项目（Dior Flow 已有超过 1000 位贡献者），却面临“史山代码”的困境时，他深知代码质量不是魔法棒能解决的。他呼吁“Take it seriously”和“工程学心态”，拒绝将 Agent 神秘化。这不仅是技术观点，更是一种职业尊严的捍卫。

**延伸思考**  
这里涉及一个深层矛盾：AI 带来的是“生产效率大爆炸”，但“验证效率”却严重滞后。正如后面阿岛（MiniMax 内部成员）所说：“生产的能力在极大提升，但在验证和交付、在 Review 这方面，模型做的还是不够。” 一个项目在去年几千行代码时就崩溃，今天能撑到十几万行，但最终一样会陷入混乱。这提示我们：**未来最稀缺的能力不是生产代码，而是理解、验证和维护代码的系统性思维。**

### 三、Dior Flow：开源项目“爆火”的玄学与决心 `[河套介绍 Dior Flow]`

**核心观点**  
Dior Flow 的火爆是“上帝握着你的手”的偶然，但背后是明确的初衷——用开源模式降低多模态任务成本，并用中国模型“打样”。它目前已成长为拥有超 1000 名贡献者的全球社区。

**深度阐述**  
当被问及 Dior Flow 为何如此受欢迎时，河套坦言：“为什么火，我们也没追究出原因。很多时候 Go viral 就是上帝握着你的手，写下一行代码。” 但除了玄学，项目有清晰的“两个身份”：

1. **开源项目**：2025 年他们发现 Research 任务极度昂贵——仅需一个 200 美元的 GPT Pro 订阅才能用几次。这种“奸商”行为激发了他们用开源来颠覆的决心，并且“必须用中国的模型给你打个样”。初衷是完成桌面工作者的所有任务：写报告、做图表、生成优质播客、动画片甚至音乐。“每个人的日常工作就是一个多模态的工作。” 因此他们选择了 MiniMax 的多模态 Token ——“得表扬一下 MiniMax”。这种覆盖所有表达形式的野心，即使一开始残缺且会被喷，也坚持要有。

2. **社区**：如今 Dior Flow 有超过 1000 名贡献者，覆盖除南北极外所有大陆。甚至有人用他们看不懂的语言提需求。“怎样用 Agent 来管理这样一个开源项目，也成了我们探讨的方向。” 此外，代码已经被很多人投诉为“史山”，如何清扫重构是当前重点。

**个人感受**  
河套的坦诚令人动容。他没有把成功归为方法论，而是承认“玄学”成分。但同时，他用“桌面工作者所有工作”这种极致愿景，揭示了开源项目的核心理念：解决自己真实、高频且多模态的痛苦。他是务实的理想主义者。他也暗示了社区管理的新范式——用 Agent 来做项目管理，这正是 AI 与开源精神交融的有趣实验。

**延伸思考**  
Dior Flow 案例说明：在一个折叠的市场里（AI 工具昂贵、国产模型可及性仍低），一个开源项目如果切中**多模态桌面自动化**这一高痛度场景，即使早期体验不佳，也极易形成口碑。而且它天然利用了国产模型的低价优势（MiniMax Token），形成生态绑定。

### 四、金融 AI 的悖论：用户连问题都提不出来 `[渔阳谈金融应用]`

**核心观点**  
在金融场景中，AI 最大的价值不是直接给出投资建议，而是筛选信息、降低门槛，并在用户亏钱时提供“陪伴”。实际的投资决策涉及实时变化和步骤链，AI 更适合做“预测可能性”和“应对预案”。

**深度阐述**  
作为上市公司金融科技的 AI 负责人，渔阳开门见山：“大多数用户根本不知道他今天打开应用想干什么。他们唯一的明确目标是赚钱，但问出的问题是‘帮我选几只个股’。” 这种含糊的问题让 AI 无解——对选股而言，信息、个人画像、实时动态缺一不可。更普遍的做法是：用户问“某只个股怎么样”，AI 罗列一堆信息，然后没有然后。渔阳认为，AI 应该做两件事：

- **筛选信息**：识别用户真正需要关注的核心变量。
- **降低门槛**：用自然语言解释市盈率、ROE 等术语的意义，让数字变成决策洞察。

当被问及“你作为负责人会用 AI 炒股吗”，渔阳说，受合规限制不能直接炒股，但公司内部有回测机制，输出不带投资建议。他个人对回测胜率“无比有自信”，因为真实数据可以支撑。但随即话锋一转：“工资远远跟不上模型告诉你的投资建议，你怎么看？” 他回答，炒股不是文字推理，而是多步骤、实时变化的系统。最重要的能力不是预测“涨或跌”，而是**当黑天鹅事件发生（如 OpenAI 突然 IPO）时，你要做什么**。这个应对方案的价值远胜于一个静态判断。

**个人感受**  
渔阳承认“工资跟不上模型建议”的荒诞，但并未因此鼓吹 AI 万能。他反复强调“陪伴”——无论是赚钱时的分享欲，还是亏钱时的安抚。这种情感视角在金融科技中常被忽略，却是用户真实的心理需求。他将 AI 的角色从“决策者”降维为“陪伴者与解释者”是非常明智的定位。

**延伸思考**  
金融领域的 AI 落地是典型的“高监管、高门槛、高风险”场景。渔阳的策略是**用陪伴建立信任，用解释降低焦虑，用回测展示能力**。这种渐进式渗透比直接下结论更可持续。同时他揭示了一个普适规律：用户常常不知道自己要什么。AI 的终极挑战不是给出答案，而是帮助用户提出更好的问题。

### 五、数据、专家与模型进化的三层协同 `[IO 谈数据与专家角色]`

**核心观点**  
模型能力的提升已将数据需求从“标注数据”推向“专家 Agent 协作”。决定模型在特定领域效果的关键，不再是算法同学，而是该领域真正的专家（如金融师、心理医生、核物理学家）。

**深度阐述**  
IO 阐述了一个演化：去年大家对数据的理解还是“数据标注”，但现在需要“专家 Agent 来入”。因为“AI 的核心生产力是工具，发明汽车让赶路更快，AI 让思考过程和做验证的过程更快。但实质问题还是来自行业本身。” 光靠模型或者 researcher 是不够的。

他举了 Coding 领域的例子：“对 Coding 的理解，开发工程师显然比算法同学更理解什么叫好的 Coding。” 因此做评测、Code Review、构建环境这些事情，真正的软件工程师比算法研究者做得更好。“做 Coding 时需要两个角色：researcher 与 engineer。engineer 决定了最关键的评价与 test。” 扩展到更多领域，就需要三个角色：算法、工程师、**真正的专家**。MiniMax 内部的“专家 Agent”实验就是将专家拉入模型迭代流程。

他接着被追问：“还有哪些类似特殊专家角色？” 他提到，半年多前 MiniMax 也不理解为什么 OpenAI 有那么多非 researcher 和非 engineer 的人——金融师、心理医生、核物理学家。核物理学家每次对齐测试时会检查模型能否用于制造核武器。现在他们理解了这个逻辑，并且正在组建自己的专家网络，覆盖网络安全、金融、法律等。“我们在这块接下来的深度应该比现在深很多。”

**个人感受**  
IO 在说“半年前我们并不意识到”时，语气是谦逊而兴奋的。这是一个技术领袖承认认知边界的瞬间。他没有假装一切尽在掌握，而是真诚地分享团队是如何在实战中发现“专家角色”的不可或缺。这给创投圈一个深刻信号：未来模型公司的稀缺资源不再是算力，而是能跟模型对话的行业专家。

**延伸思考**  
这揭示了“行业大模型”的真实路径：不是通用模型 + 垂直数据 fine-tune，而是**通用模型 + 专家 Agent 系统**。后者是一种新的组织模式：模型提供推理框架，专家提供经验与判断标准，工程师负责系统集成。这种“人机联盟”也许是下一个十年的软件范式。

### 六、模型的“持续卡点”：不是某个障碍，而是时间积累的系统工程 `[IO 谈持续模型]`

**核心观点**  
大型模型的持续提升不存在单一技术卡点，而是需要时间、实验和系统性积累。每一代模型只能在已有规模上外推几倍，十倍以上就需靠经验重新构建。中美模型差距大约十倍（两代），中国公司需要循序提升。

**深度阐述**  
被问到“持续模型的最大卡点”时，IO 的回答出人意料：“它没有特别的卡点。卡点就是时间，就是要积累。” 他说 AI 已经是一个很大的产业，不存在一个障碍让事情实现不了。存在的障碍是：为了做出来，需要时间、积攒数据、实验，脚踏实地。

具体到积累什么？主要是数据和算法。模型的特点是：每提升一代，参数量可以放大 3–5 倍，但 Scale up 的外推能力有限（只能外推几倍）。如果突然十倍以上，外推逻辑可能失效，模型就成了开盲盒。所以必须一代一代稳妥推进。IO 透露，目前美国模型比中国大十倍（约两代差距），国内公司至少需要提升两代。第一代把参数模型做扎实，第二代基于积累经验再做大实体。

在技术细节上，他提到 M3 目前的训练量较低但 KV cache 较大，下一版会压缩 KV cache 以优化推理效率。更大模型需要参数量 × 20 的数据量（200T 级别），而这种数量级的数据目前不存在，同时对数据质量要求极高——越大的模型越易拟合噪声。因此这是一个系统性挑战：“我们非常确定我们是过去几个迭代中进步速度最快的，所以我们觉得应该是能到的。”

**个人感受**  
IO 的回答没有常见的推销味道。他不避讳中美差距，也不夸大难度。“应该能到”后面加上“忐忑”二字，真实得让人紧张。这种身处技术深水区的不确定性，反而比拍胸脯更有说服力。

**延伸思考**  
这里传达了一个残酷真相：Scale Law 不是万能钥匙。当模型规模跨过某个阈值，依赖“大力出奇迹”的简单外推失效了，必须依靠工程经验的迭代。这在所有复杂系统中都存在（如芯片工艺、火箭发射）。AI 正在从“科研游戏”走向“工业级工程”。

---

### 七、Vibe Coding 的陷阱：代码生产快，但验证与长期维护跟不上 `[嘉园、河套、阿岛讨论 Engineering 时间]`

**核心观点**  
完全依赖 AI 做复杂项目时，最危险的幻觉是“魔法棒心态”——模型很会一次性交付，但不懂得维护。未来竞争力在于产品质量决策、代码整洁、系统化验证，而非单纯的生产速度。

**深度阐述**  
嘉园首先指出，产品经理的第一代 Vibe Coding 是驱动程序员；现在人人都能 Vibe Coding，相当于人人都成了产品经理。好产品经理的标准不是提需求多，而是**判断做什么和不做什么**。当 AI 让你几乎能做任何事，“不做什么”的决策标准至关重要。他推荐了《Code Complete》和《Clean Code》，强调这些先哲经验不能丢。

河套补充说，Benchmark 如 SWE-bench 只测一次性解决，导致模型变成“使命必达”的机器。而一个优秀的 Agent 应该理解长程任务，懂得维护 code base，甚至有“代码品位”。他呼吁将先哲法则凝练成 Skill，配合 Agent 使用。他还强调责任心——“Agent 做的”不能成为借口，因为账号是人的，责任必须人担。

阿岛（MiniMax 内部成员）提出了一个更尖锐的观点：现在生产效率暴增，但验证、测试、Review 的效率提升远滞后。“以前一个人改一个 bug，现在一个人咔一下提几百个，没人测不动。” 过去他带团队时每行代码都要逐步检验，现在根本做不到，只能做架构。“验证上面应该投入甚至与研发接近的能力，否则项目到十几万行代码就崩溃。” 他表示 MiniMax 训练系统也在用 Vibe Coding，并且遇到了类似问题。解法包括测试自动化、建立项目“什么做、什么不做”的原则，并且“因为人看不出来，所以必须通过系统来保证”。

**个人感受**  
河套的那句“把先哲的东西都凝练成 Skill”极有画面感。他是在试图用 Agent 的工具化来对抗人性的懒惰。阿岛提到“以前每行代码我都能看出来”时的怀念，折射出一个技术管理者面对生产力失控的焦虑。这些情感都指向同一个方向：**工具越强大，越需要更坚固的纪律**。

**延伸思考****  
这是一个被普遍忽视的“生产力悖论”：生产速度暴涨，但系统复杂度非线性上升，导致维护成本爆炸。未来的软件工程可能不再关注“写代码”，而是关注**代码治理** —— 如何用 Agent 自动生成、自动测试、自动重构、自动度量质量。这将催生新的角色：AI Training Data 伦理师、代码质量架构师等。

---

### 八、终局思考：AI 社会的“温暖”与“内耗” `[结尾分享]`

**核心观点**  
下半年的两个关键方向：一是金融场景中的陪伴（尤其在亏钱时），二是从“限制模型发挥”转向“引领模型发挥创造力”。同时，AI 正让很多人变得内耗和焦虑，而真正能带来温暖的是个人如何用 AI 解决具体人的问题，而非追逐宏大叙事。

**深度阐述**  
在最后的自由分享环节，三位嘉宾给出了完全不同的视角：

- **渔阳**：用 AI 炒股的数据回测令人自信，但他强调“陪伴”——无论赚钱还是亏钱，用户都需要陪伴。亏钱时更需要。“工资跟不上模型建议”的荒诞已经出现，但他认为金融 AI 的下半年目标有两个：一是从限制模型转变为引领模型思考；二是强化陪伴属性。

- **河套**：他写专栏《How AI shapes our society》，直观感觉到 AI 让很多人变得内耗和焦虑。他提到自己太太看国内大模型咨询脱发问题，结果更焦虑。于是他用 Code X 给太太建了一个专属 Wiki，导入所有体检报告、用药记录、聊天记录，接到飞书让她专属使用。之后太太看医生沟通顺畅了，焦虑也缓解了。河套说：“我们普通人都有机会让身边的人变得更好，只要愿意。Vibe Coding 不是替代人，而是让好的想法绽放，让远离技术的人感受到一种温暖。这是从冰冷的软件工程里发现的人类温暖。” 他随后坦言自己是“AI 降临派”，认定 AI 能力终将超越人，但更值得思考的是在这种趋势下，人如何不把思考完全外包给 AI。

- **IO**：延续“预测未来的最好方式是创造”的理念，他认为未来一年无法预测，但确定的是模型 Agent 和人的关系会越来越紧密。他强调模型与 Agent 都应作为“实现更高智能的手段”，最终智能还是为人类服务。

**个人感受**  
河套讲述太太的故事时，声音明显柔软了。那种从宏大叙事回归到个人关怀的转折，是整场对话最打动人心的段落。它提醒我们：无论模型多强、Agent 多聪明，技术最终的价值是解决一个具体人的具体痛苦。他还坦言“AI 让很多人内耗”，这是对技术的警醒——不是所有进步都自动带来幸福。

**延伸思考**  
河套那句“未来不是跟人竞争，而是跟 Agent 竞争”以及“会有人把思考外包给 AI”的担忧，触及了一个哲学问题：当 AI 能够更高效地推理时，人类的批判性思考能力是否会萎缩？这也许需要教育和社会架构的前瞻性调整。同时，渔阳提到的“引领模型发挥创造力”是一种新的管理范式：人类从“操作者”变为“策展人”，负责设定边界、目标与价值观。

---

## 精华收获

1. **聚焦战略的价值**：MiniMax 在 M2 时期只聚焦 Coding Agent，拒绝了分散的评测追逐，最终取得十倍吞吐量突破。在不确定的时代，收缩边界可能比扩展更有效。

2. **工具越强，纪律需要越严**：Vibe Coding 和 Agent 大幅降低了写代码的门槛，但项目的长期健康依赖代码质量决策、验证系统和工程纪律。AI 开发需要从“写一次性代码”转向“设计可维护系统”。

3. **专家角色将成为模型公司的竞争护城河**：从数据标注到专家 Agent，行业知识不是 fine-tune 能替代的。引入核物理学家、金融师、心理医生等参与模型迭代，是提升模型在垂直领域信任度的关键。

4. **AI 产品的核心是陪伴而非神化**：无论是在金融场景的“亏钱安慰”还是河套给太太建专属 Wiki，真实需求隐藏在功能之下——用户需要被理解、被支持，而非只有答案。

5. **人应该承担 Agent 行为的责任**：AI 生成代码、建议、决策，但最终的账号和责任是人的。保持 Ownership 是避免系统失控的底线。
