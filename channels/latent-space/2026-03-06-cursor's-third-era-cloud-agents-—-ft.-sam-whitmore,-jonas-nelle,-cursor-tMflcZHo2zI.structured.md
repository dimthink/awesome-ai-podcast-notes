# Cursor 发布 Cloud Agents：开启 AI 驱动开发的新时代

在人工智能飞速发展的今天，编程工具正在经历一场深刻的变革。近日，知名编程工具 Cursor 推出了其革命性的新功能——**Cloud Agents（云端智能体）**。本期播客深入探讨了这一功能背后的理念、技术细节以及它将如何重塑软件开发流程。来自 Cursor 的 Sam Whitmore 和 Jonas Nelle 分享了他们的真知灼见，为我们描绘了一个由 AI 智能体主导的未来开发图景。

## Cloud Agents：从“代码补全”到“自主开发”的跨越

### 什么是 Cloud Agents？

简单来说，**Cloud Agents 为 Cursor 配备了一台专属的云端计算机**。与以往仅能在本地“阅读”代码不同，现在的智能体拥有了一个完整的、可自主操作的虚拟机环境。它可以：

- **自主搭建开发环境**：为代码仓库配置所需的运行环境。
- **端到端测试**：启动开发服务器，迭代修改，并实际运行代码来验证更改。
- **生成演示视频**：将它的操作过程和结果录制成视频，供开发者审阅。
- **提供远程桌面访问**：开发者可以随时远程连接到这个云端桌面，进行实时预览或手动调整。
### 核心理念：将测试和验证交给智能体

Cloud Agents 的一个核心转变是让智能体自己负责测试。想象一下，如果一个同事提交了一个未经测试的 PR（Pull Request，拉取请求）让你审查，你肯定会感到不满。Cursor 将同样的逻辑应用到了智能体上。

- **默认进行测试**：对于非简单的修改，Cloud Agents 会默认进行测试，确保交付的代码是可运行的。
- **校准测试力度**：通过预设的提示词和用户自定义的规则（如 `agents.mdc` 文件），智能体可以判断哪些更改需要测试，哪些（如简单的文案修改）可以跳过。
- **`/notest`**** 命令**：如果用户明确不需要测试，可以使用此命令跳过。
## 革命性的工作流：视频、并行与协作

Cloud Agents 不仅改变了智能体的工作方式，也彻底革新了开发者的工作流程。

### 1. 视频优先的代码审查

当智能体完成一项任务后，它会返回一个**演示视频**。这个视频成为了代码审查的绝佳切入点。

- **降低认知负担**：与直接审查数百行的代码差异（diff）相比，观看一个 20 秒的视频能让人秒懂智能体做了什么。
- **快速对齐意图**：视频是开发者与智能体之间的“共享工件”，能立刻发现理解上的偏差。例如，你本想要一个“开关”，但智能体却实现了一个“复选框”，视频能立刻揭示这个问题。
- **多模型对比**：开发者可以让不同模型（如 Opus, Claude, Gemini）同时处理同一个 Prompt（提示词），然后通过对比它们生成的视频，快速选出最满意的结果进行迭代。
### 2. 并行与异步开发

Cloud Agents 让“并行”成为了可能。开发者可以像水管工拓宽管道一样，同时启动多个智能体处理不同的任务。

- **告别上下文切换**：在传统的本地开发中，切换分支、等待环境重建会打断思路。而现在，开发者可以在早上同时启动 10 个 Cloud Agents。
- **高效的任务管理**：当一个智能体完成任务后，开发者可以观看其视频，给出“改成红色”这样的简单反馈，然后让它继续工作，自己则无缝切换到下一个智能体的审查中。
- **利用碎片化时间**：由于智能体在云端运行，即使关闭本地电脑，任务仍在继续。开发者可以利用通勤等碎片化时间，通过 Web 应用查看进度、观看视频，甚至进行简单的“放行”操作。
### 3. 团队协作的新模式

Cursor 已经将 Cloud Agents 集成到了其团队协作工具 Slack 中，创造了一种全新的开发模式。

- **Slack 即 IDE**：在 Cursor 内部，Slack 频道已经成为开发的中心。在讨论问题的线程中，只要 `@cursor`，就能启动一个 Cloud Agent 来处理任务。
- **人机协同讨论**：团队成员（包括设计、销售等非技术人员）可以在一个智能体创建的线程里共同讨论。人类负责决策“是否要发布这个功能？”、“这个 UX 对吗？”，而智能体则负责实现细节。
- **智能体引入协作者**：智能体甚至可以通过 `git blame` 找到相关代码的负责人，并直接在 Slack 中 `@` 他们，将更广泛的人类专家拉入讨论。

### `/repro` 命令：让 Bug 无处遁形

这是一个专门用于处理 Bug 的 Slash 命令。

- 当开发者使用 `/repro` 命令描述一个 Bug 时，智能体首先会**尝试复现这个 Bug 并录制视频**，证明 Bug 确实存在。
- 然后，它才会去修复代码，并在修复后**再次录制视频，证明 Bug 已被成功修复**。
- 整个过程遵循经典的 TDD（测试驱动开发）“红-绿-重构”模式，极大地增强了对修复结果的信心。
### 内部高级用法：智能体诊断与 MCP 集成

Cursor 团队内部已经在使用更高级的智能体功能。

- **`/cloud-agent-diagnosis`**：这个内部命令利用 **Datadog MCP（Model Context Protocol，模型上下文协议）服务器**，当 Cloud Agent 运行时出现问题，它可以启动子智能体，通过 Datadog MCP 深入分析日志，将原本可能需要数十分钟的调试时间缩短到一次智能体调用的功夫。
- **利用 Transcripts 调试**：一个智能体可以被授予访问另一个智能体完整“思维链”（Transcript）的权限，从而作为“外部调试器”来分析前一个智能体为何失败，或者像“分叉”一样从某一点继续对话。
## 深度思考：Cloud Agents 带来的变革与挑战

### 从“手写代码”到“品味选择”

随着 Cloud Agents 承担了越来越多的编码工作，开发者的角色正在发生根本性的转变。

- **人类负责“品味”，AI 负责“实现”**：开发者不再需要纠结于 `if` 语句应该放在哪里，而是将精力集中在更高层次的决策上，比如“我们真的应该发布这个功能吗？”、“这个用户体验是正确的吗？”。人类的工作变成了不断地“注入品味”，在智能体生成的多个方案中做出选择。
- **“手写代码”成为过去式**：就像现在很少有人会怀念纸带编程一样，“手写代码”（手工敲出每一个字母）可能很快就会被视为一种过时的行为。
### 新瓶颈的出现与应对

当代码生成变得无比容易后，新的瓶颈也随之出现：**如何将海量的、由 AI 生成的代码安全、高效地部署到生产环境？**

- **代码审查的变革**：审查代码本身仍然是必要的，但像 **Bugbot** 这样的工具已经开始扮演“AI 代码审查员”的角色，自动发现潜在问题。当一个开发者多次忽略 Bugbot 的警告而导致线上事故后，他就会学会信任并等待 Bugbot 的审查结果。
- **生产级基础设施的需求**：一个 10 人规模的初创团队，由于使用了 Cloud Agents，其代码产出量可能堪比一个万人规模的巨头公司。因此，他们也同样需要后者所使用的那些基础设施，如**堆叠式差异（Stack Diffs）、合并队列（Merge Queues）、分阶段发布、自动化回归检测**等，以确保软件质量和稳定性。
- **多维度的审查**：未来的审查可能不再是单一的代码审查，而是会细分为**功能审查、性能审查、安全审查**等多个维度，每个维度都由专门的智能体来协助完成。
### “智能体实验室” vs. “模型实验室”

Cursor 被视为“智能体实验室”的典范。它与 Anthropic、OpenAI 等“模型实验室”的区别在于，其核心竞争力不在于训练下一代基础模型，而在于构建一个强大的“智能体层”。

- **模型路由与抽象**：对于普通用户来说，面对层出不穷的新模型可能会感到困惑。Cursor 可以充当“品味仲裁者”，通过像“Auto”这样的默认选项，自动为用户选择当前最适合其任务的最佳模型。这赋予了“智能体实验室”独特的价值：用户忠诚度将建立在平台体验上，而非底层的某个单一模型。
- **多模型协同的潜力**：Cursor 内部曾实验过一个功能：让多个模型为同一个任务生成方案，然后由一个“评审员”智能体综合各个方案的优点，写出一个全新的、更优的代码差异（diff）。实验发现，使用来自**不同提供商**的模型会产生“1+1>2”的协同效应，这预示着未来可能存在一种“模型委员会”或“模型群体智能”的工作模式。
### 对未来预测：Cloud Agents 何时超越本地 Agents？

Sam 预测，到今年年底，在云端运行的智能体（如 Cloud Agents）的处理量将有望超过在本地运行的智能体。尽管目前让云端沙盒环境做到极致完美（如项目初始化、依赖管理、环境一致性等）仍有很多挑战，但这是 Cursor 团队正在全力攻克的方向。

## 总结与展望

Cursor 的 Cloud Agents 不仅仅是一个新功能，它代表着一种**编程范式的转移**。它将开发者从繁琐的“手工艺人”角色中解放出来，转变为能够同时指挥多个智能体的“指挥家”或“产品经理”。

当代码生成变得唾手可得，真正有价值的将是人类的**品味、决策能力和对高层次问题的洞察力**。Cursor 通过 Cloud Agents，正在为所有开发者搭建一个通往这个未来的桥梁。正如访谈中所言，Jevons 悖论正在上演：AI 让开发变得更高效，反而激发了人们更大的雄心，创造出更多、更复杂的软件，从而消耗了更多的 token 和算力。

从软件行业出发，这场由 AI 智能体驱动的变革，也必将溢出到其他行业，重塑未来工作的形态。
