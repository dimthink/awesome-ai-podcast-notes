## 详细内容

### 1. 重回代码的 COO：AI 是领导者的“第二大脑” `[章节 00:00:00 - 00:15:00]`

**核心观点**

Kyle Daigle 并非被管理吞噬的商界精英，他是一个利用 AI 进行“时间回溯”从而重构工作流的重度极客。他发现，通过构建一套连接所有数据源的“回溯循环”，他找到了比年轻写代码时更极致的创造快感。

**深度阐述**

访谈一开场就充满了张力。主持人 Sean 直接甩出了 Kyle 最近暴涨的 Git 贡献图，用一种近乎“捉赃”的语气问道：“这次又是什么情况？你作为一个 COO，怎么会有时间写这么多代码？”

面对这个直击灵魂的发问，Kyle 没有丝毫遮掩。他坦言这 13 年的身份转变——从亲手写 Webhooks 和 API 平台的工程师，到 2018 年带领工程团队上线 GitHub Actions，再到如今接管全公司运营与市场。他始终觉得头衔很滑稽，因为“真正的问题永远是‘人’的问题”。

不过，AI 的出现彻底点燃了他。Kyle 描绘了一个极强的个人工作流：他不再像传统的开发者那样向前构建，而是进行 **“递归循环向后看”**。

“这不是叫它帮我写个博客，”Kyle 说，“我让它去扫描我过去一周所有的 PR，去读我 Obsidian 里的笔记，去抓取我 Teams 里的转录，去读 Slack。然后，它必须给我建立一个报告：告诉我们这周到底发生了什么，什么有效，什么无效。然后基于这种‘向后看’，它告诉我未来几天可以怎么调整。我发现这是极其有价值的。”

**个人感受**

这里我感受到了一种深深的共鸣。当大家都在狂热地讨论“如何让 AI 写出下一行代码”时，Kyle 指向了一个被严重低估的领域：**AI 在“回顾”上的超能力**。大型语言模型在寻找海量非结构化数据中的模式、并生成归结性洞察方面，有着比胡乱生成创意更致命的价值。Kyle 提到了一句很美的话：

> “I just find that the folks that came from a different career... having the power of an AI where I can crank up 15 agents on Saturday, you know, while my kids are doing lacrosse. That's like really powerful and it gets me back to that feeling of creation.”
>

> “我发现那些跨领域而来的人……当拥有 AI 的力量，能在周六孩子练长曲棍球时同时开动 15 个 Agent，这真的太强大了。它把我带回了那种创造的快感。”
他甚至讲了一个有趣的故事：他为了给 CFO 演示营收规划，没有手动做 PPT，而是用 AI 对接了所有数据源构建了一个小 App。但他故意 **“让它看起来不那么 AI 生成”**，甚至有点丑，从而避免了项目的各种询问。这个细节非常反直觉：**在真实的商业世界里，有时候你要对你的工具保持沉默，让结果自己说话。**

**精华收获**

- **回溯高于生成**：AI 最有价值的应用场景不仅仅是生成新内容，而是利用长上下文能力作为“完美的史官”，为知识工作者提供超强的记忆与复盘。
- **脚手架思维**：真正的效率来自将个人多年的业务直觉加上 AI 工具构建的内部“脚手架”。Kyle 证明，在管理者层面，AI 最大的作用是修补“认知带宽”的鸿沟。
- **反直觉的呈现**：在职场中，高度完美的 AI 生成物会引发戒备心。适当降低 AI 的“痕迹感”（Making it look humanly bad），反而能够更顺利地推进实质工作。
### 2. 信任危机与“乐高”架构：微服务的理念正在进入 Agent 圈 `[章节 00:15:00 - 00:30:00]`

**核心观点**

内部推行 AI 最大的灾难在于“技能膨胀”。Kyle 提出，应当彻底摒弃包罗万象的“巨石技能”，转向构建“原子化微技能”。同时，他指出了当前由 AI 生成的代码所引发的“信任危机”，这其实是工业界最深的社会问题。

**深度阐述**

当所有人都在疯狂创建分享自己的技能时，主持人尖锐地提出：“这是一个巨大的混乱吗？” Kyle 笑着回答：“是的，但这混乱正在被共识所纠正。”

他认为，**“巨型完美技能”的时代已经结束了**。以前我们认为《大英百科全书》式的提示词就是王道，但如今 GitHub 内部推行的是 **“微技能”**。微技能就像乐高积木——每一块只做一件极其具体并且完美的事。比如，“识别任何一个 MCP 服务器里最重要的营销信息”。

这背后的逻辑源于矩阵问题：同一个词（如“Summarize”）在不同部门有不同含义。给分析师的总结和给客户的总结完全是两码事。只有原子化的“总结”基座，加上不同的上下文人格化准则，才能解决这种混乱。“微服务”解构了后端，而“微技能”正在解构 AI 交互。

**延伸思考**

紧接着，对话进入了全文最精彩的哲学碰撞之一：**信任**。如果 80% 的拉取请求都是 Agent 写的，当一个 Agent 写代码、另一个 Agent 去审查，人类的签名到底意味着什么？

Kyle 的思考极其深刻：

> “Ultimately we're trying to codify trust... the trust is kind of diffuse... it's a social problem ultimately.”
>

> “归根结底，我们是在尝试编纂信任......信任是高度分散的......而它终究是一个社会问题。”
Kyle 认为，代码领域的信任和自动驾驶或打车软件的信任没什么不同。你坐在一辆标准行驶的出租车里会觉得安心，是因为你信任了社会系统和品牌；而在 Agent 的世界里，针对一个 PR 的信任，来自于“是谁签的字”，而不是“代码是否通过了测试”。

**个人感受**

这段话让我非常触动。他提到了**传递性与验证性**的区别。当 AI 能解决所有的验证问题（单元测试、编译通过），人类情感的“传递性信任”（即我了解 kyle 的品味，他审过了我就信）仍然无可替代。这构成了 GitHub 作为社交网络的绝对护城河。Kyle 也在暗示：目前的 Stars、Commits、Contributor 都太容易被 Gaming，未来的方向可能是某种基于“硬信任”的机制（如赞助、真实的社交身份验证链）。

**精华收获**

- **原子化胜过巨无霸**：提示词工程的终极形态不是万能咒语，而是可组合的、高度可读的微小函数。这能极大降低 Agent 行为的理解成本和维护成本。
- **信任是门槛**：如果 AI 生成的代码在技术和知识产权上已经没有问题，那么人类最后守住的门槛是对“品味”和“历史行为”的信任。GitHub 需要重构的不只是代码，而是点对点的“身份信任体系”。
### 3. 14 倍增长的代价：MySQL 1 的宿命与基础设施的修罗场 `[章节 00:30:00 - 00:50:00]`

**核心观点**

这是全文最扎心也最硬核的真实披露。AI Agent 带来的 14 倍提交量增长正在以一种“物理级”的方式撕碎 GitHub 运行了 15 年的架构假设，迫使他们在对角线上进行重建。

**深度阐述**

Kyle 转到了 GitHub 最近备受干扰的稳定性问题。他承认，这是 GitHub 建立以来面临的最复杂、最有意思的挑战。

他用了一个很有趣的词：**“对角缩放”**。以往，应对增长无非是水平（加机器）或垂直（换设备）。但现在，**服务的性质变了**。

- **Monorepo 的回归**：过去十年我们一直在拆解代码库，但 AI 要求监控全局，所以巨型单体仓库又回来了，这直接冲击了 Git 原有的核心算法。
- **CPU 不足**：大家疯狂讨论 GPU，殊不知由于 Agent 大量调用 Actions 做 CI/CD，CPU 成了新的稀缺资源。
- **老去的灵魂：MySQL 1**：这绝对是内部人士的梗。Kyle 承认，他们的权限和相关核心数据仍然耦合在一个代号为“MySQL 1”的遗留数据库上。他们一直在做解耦，但指数级的增长跟上了解耦的速度。
“我们的可用性在最近几周（相对于发布前）已经好多了，”Kyle 说，“但那个数据库的问题，是物理问题，是需要时间来治愈的。我们必须去切开这些运行了 15 年的服务，重写他们的运行规则。”

**个人感受**

这番谈话充满了工程的美感和阵痛。Kyle 完全没有找借口。他只是在分享：“以前被压垮的是 Web hooks（他一脸坏笑的说那是自己 2014 年写的），现在被压垮的是更深层的东西。” 对任何技术创始人来说，这也是一种安慰：当你平台崩溃时，外面的人骂你，内部的人其实正在拼了老命地解决物理极限导致的坏账。GitHub 的透明度在这里得到了彰显。这种“透明的工程文化”，也是开发者社区愿意给予他们更多包容的原因。

**精华收获**

- **预判的极限**：AI Agent 的爆发让所有的增长假设变成废纸。即使强大如 GitHub，也只能在指数级洪流中边被打脸边重写底层逻辑。
- **基础设施要零假设**：过去的“小而多”的模式被颠覆了。架构设计必须具备极度的弹性来包容不同规模（甚至倒行逆施）的增长模式。
- **拆解是唯一救赎**：摆脱单体数据库依赖（如 MySQL 1）是所有成熟科技公司在面对新时代时不得不补的作业。
### 4. 沉浸在环境中：Copilot 需要停止“等你提问” `[章节 00:50:00 - 01:10:00]`

**核心观点**

Kyle 认为当前所有的 AI 编码工具都非常“近视”。下个时代的 Copilot 必须成为一种“环境智能”：它无时无刻不在监听你的工作上下文，并且在你开口之前就主动为你准备好了方案。

**深度阐述**

主持人抛出问题：“当你看到这些编码 Agent，会觉得历史已经终结了吗？剩下只是优化吗？”

Kyle 毫不犹豫地否定了这个观点。他提出了一个非常了不起的前瞻性概念——**Ambient AI**。

他直言不讳地批评当前的主流工具：它们只关心你 IDE 里打开的那几个文件。

“我认为最有趣的事情是实际的 Ambient AI，而不是一个植入的助手。当我在实现一个新特性时，我希望它知道每一份规格文档、每一个相关邮件、我的每段在线聊天。它必须能听到我们这段对话，然后自动推算出最适合的实现路径。现在没有任何工具能做到。”他说。

这段话完美地解释了为什么微软如此重视 Open Claw 和 Work IQ。Kyle 暗示，微软正在从底层（操作系统沙箱层、数据层）构建这种“全知”的环境。未来的 Copilot 不再是你打开对话框提问的工具，而是渗透在每一个操作按钮后面的智能体。它观察，它建议，它行动。

**个人感受**

读到这段话时，我感受到了一个在 AI 最前线的人对产品形态的终极理解。**“Don’t call me, I’ll call you.”** 这不仅仅是面向对象编程的法则，也是人机交互的未来。我们正在从“用户与计算机对话”变成“计算机与用户的潜意识对话”。Kyle 认为，只要 Agent 还需要人类去给它“喂参数”或“填写任务单”，就还处于极其初级的阶段。真正的解放，来自于一个“永远在线、永远知情”的第二大脑。

**精华收获**

- **洞察环境而非执行指令**：下一代 Agent 的核心竞争力在于**上下文获取能力**，而不是它的推理能力（模型会越来越像）。谁能打通更深层的 OS 级和办公套件级数据，谁就赢了。
- **从触发到预见**：未来的产品设计逻辑将从 PULL（我拉取数据）变为 PUSH（数据推送到我面前）。
### 5. 微软的组织图：“我们是基础设施建造师” `[章节 01:10:00 - 01:30:00]`

**核心观点**

面对微软为何要专设一个 Open Claw 的 CVP 的疑问，Kyle 给出了一个极具大局观的回答：微软已经没有兴趣只做一两个顶层的、垂直的 AI 应用了。它现在的使命是建造从操作系统到数据层到安全沙箱的“平台基石”，让**所有开发者**都能在各自的场景里建 Agent。

**深度阐述**

最后，Sean 提出了一个宏观问题：“你认为未来两三年的真实前景是什么？如果让你问萨提亚，你会问什么？”

Kyle 的回答就像是给微软的 Build 大会做了一个极具深度的注解。他指出，微软清晰地看到了“当机械人替人类按键盘时会面临的安全问题”。因此，微软亲自下场去优化 Windows 沙箱，去构建 Work IQ（企业数据连接层），去赞助并丰富像 Open Claw 这样的开源运行时。Kyle 认为，**“如果我们在为构建者建设，就不能只给他们一个顶部产品，要给他们完整的组件库。”**

他特别骄傲地补了一句：“让我一直留在 GitHub 的原因就是——让所有人都能去建造。我觉得这就像换一个开关。只要你想，任何人都应该能说：‘这破玩意儿不好用，我自己来改。’”

**个人感受**

结尾这段话非常有感染力。Kyle 在描述一个理想的未来：**“How I build on the weekend should be how I build at work.”** 你在周末玩代码的随心所欲，与你在世界级银行里处理敏感数据的严谨，应该是完全相同的体验。平台把合规和约束做了，把自由留给了构建者。这不仅是关于 AI 能力，更是关于**开发者权利**的宣言。这大概就是 Kyle 13 年不离不弃的原因，也是 GitHub 作为平台依然迷人的文化不灭点。

**精华收获**

- **平台的本质**：当所有人都在做 Agent 应用时，真正的护城河是“让 Agent 能安全运行的操作系统”和“能喂饱 Agent 的数据管道”。
- **周末的力量**：组织的重负不应扼杀创造力。AI 时代给了我们巨大的机会，去让大公司的开发环境重新变得像个人项目一样敏捷，这是所有 CTO 的终极命题。

## 精华总结

Kyle Daigle 用这场访谈完成了一件极其难得的事情：他让一个市值万亿公司的平台领导者看起来依然像一个充满热情、富有思考的一线开发者。他没有给出任何关于股价或市场份额的废话，而是呈现了大量关于 AI Agent 带来的**基础设施挑战**（MySQL1 的宿命）、**组织技能重构**（微技能哲学）以及**人类身份与信任**（环境智能的前提）的真实洞察。

**为你留下的 5 个核心洞察：**

1. **提问的艺术不在于生成，在于回顾**：将 AI 视为你每周复盘的第二大脑，远比让它给你写一篇漂亮的散文有价值。
1. **原子化的技能胜过万灵药**：提示词工程在飞速向可组合的代码靠拢。写那些只做一件事、又做得极好的微技能吧。
1. **信任将是 AI 时代最硬的通货**：代码测试可以自动化，但根据一个人的“味觉”去批准代码这件事，在未来很长一段时间内依然是碳基生物的堡垒。
1. **规模是一种新物理**：当你的服务经历 14 倍增长时，不要去修修补补，去重写它的底层逻辑。Infra 即持续重构。
1. **未来的 AI 是环境的，不是触发的**：你在寻找的下一个杀手级应用，不会是一个等你按按钮的机器人，而是一个永远“在听”、永远“懂你”的背景智能。
在这一小时的访谈里，Kyle 不仅定义了 GitHub 的下一步，更定义了一个新时代下，创造者与管理者如何通过 AI 实现自我超越。
