# 对话 AI 研究科学家姚顺宇：模型能力的趋同、应用层的突围与 AI 行业的“靠谱”时代

> **访谈背景**：本期节目录制于 2026 年 3 月。主持人张晓珺对话从 Anthropic 跳槽至 Google DeepMind 的研究科学家姚顺宇。他参与了 Claude 3.7、4.5 及 Gemini 3 等核心模型的开发，并从物理学半路出家投身 AI 行业。访谈中，他分享了对模型能力趋同、Open Cloud 等新形态产品、AI 应用层的壁垒以及行业人才特质等深刻见解。
---

## 开场与嘉宾介绍

哈喽大家好，我是小俊。

AI 业界有两个姚顺宇，一个从 OpenAI 跳槽到了腾讯，一个从 Anthropic 跳槽到了 Google DeepMind。前一位姚顺宇去年来过我们节目，那今年我们邀请了另外一位。

这位姚顺宇毕业于清华和斯坦福大学，曾经的研究方向包括**非厄米系统、量子物理和高能物理**。毕业后从物理转战 AI。2024 年加入 Anthropic，2025 年跳槽到了 Google DeepMind，出任研究科学家，参与了 **Claude 3.7、4.5、Gemini 3** 等模型的开发过程。

除了这些重点模型的开发之外，他有一些很有趣的观点，比如：

- **AI 的本质是简单的**
- **预训练也是一种强化学习**
- **不要迷信“老灯”**
- **AI 个人英雄主义的时代已经过去了**
- **AI 从业者最重要的特质是“靠谱”** 等等。
我们的节目录制于 2026 年 3 月。距离我们这次节目录制完，世界又发生了许多意想不到的变化，比如 Meta 对于 Minus 的收购被撤销，Cursor 可能被 SpaceX 收购，xAI 将终止独立运营、并入 SpaceX 并且更名为 SpaceX AI 等等。

如果我们的谈话内容有一定的滞后性，还请大家能够多多包含，因为 AI 的世界实在是变化太快、也太出乎意料了。可能还要特别说明的一点是，技术细节会涉及到一些企业机密，有一些是嘉宾不方便分享的，所以也请大家能够多多包容。我们在最大的可能范围内和大家一起来学习 AI。

那接下来就是我对话顺宇的访谈。期待 2026 年我们和 AI 共同进步。

---

## 核心观点：AI 从业者最重要的特质是“靠谱”

**Anthropic 作为一个公司来说，它能够实现这种比较 top-down 的机制，是一个很独特的事。** 这对于其他模型公司很难做到。很难，比如说 OpenAI 就干不了。Gemini 也比较难。大公司和 startup 它打法本来就不一样，因为 startup 重要的是 make bet，就是我得赌一件事。

我觉得大家现在每个人都是冲浪的人，本质上是个浪，而不是那个冲浪的人。就 AI 这个事本来也不太需要脑子。

> **需要脑子吗？**
>

> 真的不太需要脑子。
**那需要什么？**

我觉得这个行业最重要的特质就是**靠谱**。就是做事细，然后对自己做的事负责任。这是最重要的特质。

## 嘉宾背景：两个“姚顺宇”的区分

**主持人**: 硅谷不是有两个姚顺宇吗？你要不要先给大家介绍一下你自己，然后给大家科普一下两个姚顺宇的区别？

**姚顺宇**: 可以。我叫姚顺宇，然后显然也有一个跟我几乎同名的朋友，我们俩主要履历也有一些 overlap，所以看起来可能非常难以区分。

我是以前做物理的：

- 本科在清华，那时候做**理论物理**
- 后来去斯坦福做**理论高能物理**，和**量子信息、黑洞**相关那些方面
- 离开斯坦福之后去伯克利短暂地待了两个星期的 postdoc，然后就离职了，去了 **Anthropic**
- 在 Anthropic 待了一年，去年（2025年）9月底、10月初的时候加入了 **Gemini**
如果大家非要区分的话，我觉得最大的区分就是：**那个顺宇一开始就是一直做 CS（计算机科学）相关的事情，而我其实从某种意象上来说是个半路出家——我之前是做理论物理为主的。**

**主持人**: 你们是不是好朋友？你们好像大学就认识，而且是一届的对吧？他是一个什么样的人？你是一个什么样的人？评价一下他，你也评价一下自己。

**姚顺宇**: 对对对，我们本科就认识，因为本科是一届的，在清华。但是他一开始就是学计算机的嘛，所以他在**姚班（计算机科学实验班）**；我是学物理，所以我在**基科班**。后来他去了普林斯顿，我去斯坦福。这可能是另一个有点令人费解的点：好像物理世界里觉得斯坦福应该是学计算机的人该去的地方，然后觉得普林斯顿是学物理的人该去的地方，但我俩正好反过来，所以也可能产生了一些费解的事情。

我倆其实也真的挺不一样，我觉得他（另一位姚顺宇）是一个比我有趣得多的人。我从他身上也学到了很多不一样的 point，比如他花了很多时间去思考**人和 AI 的交互**，包括一些产品上的事情。这对我来说是一个很不一样的朋友。

**主持人**: 你们之前在硅谷的时候多久见一次面？现在还频繁打电话吗？

**姚顺宇**: 我们在硅谷的时候见面确实挺频繁的，可能每几个星期吧。但是见面好像主要是为了凑一块**玩**。就是真的纯玩，可能出去散散步、扯扯有的没的，然后出去吃个饭、打个牌之类的。他回去之后我们其实也还是会经常打电话。

**主持人**: 最近一次电话聊啥了？

**姚顺宇**: 好像就是前一两个星期。可能就是过几个月然后 catch up 一下大家的近况。

**主持人**: 他是不是多次想把你拉过去？

**姚顺宇**: 可能有这个意思吧，但是不关键。

**主持人**: 你为什么不去？

**姚顺宇**: 我觉得对我自己来说我没想清楚吧，多半是我自己的原因。我也没有去任何中国的地方。主要原因是：在去年（2025年）的 8、9 月，我离开 Anthropic 之后，决定要去哪的时候，**最大的动机是我想要学一些不一样的东西**。我没有着重去考虑说我能够去 lead 一个项目或者一个 project，更多的是优先去学习这些东西。所以那个时候我选择去了 Gemini。

**主持人**: 我发现你们俩老被放在一起比较和讨论，对你来说是困扰更多还是享受更多？

**姚顺宇**: 我没什么感觉，因为我这个人也不太关注社交媒体，所以我真的没什么感觉。

## 对 AI 发展阶段的判断：从“能不能做到”到“是否被良好定义”

**主持人**: 顺宇他（另一位姚顺宇）之前在去年（2025年）的时候说 **“AI 进入下半场”** ，这成为了一个非常有名的观点。你觉得今天的 AI 处在一个什么样的时期？你能给它一个定义吗？

**姚顺宇**: 对我来说，我可能看得没有那么清楚什么叫做上半场、什么叫做下半场，或者说这个定义一直以来对我并不是特别清楚。

对我来说，**现在 AI 进入到了一个阶段：大家都已经开始不那么担心一件事“AI 是不是能够做到”，而是担心这件事“是不是被良好定义”**。我觉得这是一个很大的区别。

比如一年之前（2025年初），那时候我在 Anthropic，大家可能担心的事情还是：OpenAI 这个 Reasoning 做得这么强，我们有没有机会追上、有多大的机会超越它？大家很担心这个事。

**我觉得现在，至少在 Gemini、OpenAI 和 Anthropic 这三家当中，没有哪一家会真的担心自己追不上。** 可能现在对大家更难的事情是**想明白要去做什麼**。这是一个 bet，是一个赌，但也是一个很需要人的 insight 的事情。

---

## 模型能力的“拉平”与差异化

**主持人**: 那这也意味着模型的能力被拉平了，对不对？它变得同质化、变得商品化，所以模型没有很大的区别——在好壞上没有很大的区别，但是它需要分化。

**姚顺宇**: 我觉得从用户的实际体验上来说，这三家的模型是能感受到区别的。但难的一点是：**过去这个区别在纸面上也能看出来**。

- **纸面上**：比如像公开的有很多 benchmark（基准测试），大家以前能够去看 SWE-bench，然后数学大家会比像简单一点的 AMMI、难一点的 GSM。那个时候你从纸面上就能看出，哎这个模型好像 Reasoning 强一点，那个模型好像 Coding 强一点。
- **现在**：纸面上大家其实都比较相近。你看纸面上的数据，比如看 SWE-bench，你会发现好像好的会比不好的可能高一个百分点或者两个百分点，但其实大家都在 80% 附近。那个附近数字高一点低一点主要是 **noise（噪声）**，而不是 signal（信号）。
但是从另一方面来说，大家使用上确实还是能体现出区别的。

就我个人了解到的信息而言：

- **Claude**：目前仍然是比较通用的、这种**工具使用类的 agent 表现最好**的。
- **Coding**：在纯 Coding 方面，最近 CodeX 稍微追上了一點，把这个 gap 变小了一点。
- **Gemini**：在**纯 Reasoning** 和一些比较日常的使用环境下，目前还是比较好的。在 Coding 和 Agent 上还处于一个比较接近的状态。
---

## 模型能力的差异：意願 vs. 定义问题

**主持人**: 这些能力是他们有意选择“我优先发展哪个方向”，还是说它就是好和坏的区别？它是能力问题还是意願问题？

**姚顺宇**: 我觉得其实是有一元（意願）的成分在的。

- **在过去**：当大家能从纸面上就看出区别的时候，**意願肯定是佔大多數的**。因为像 Claude 就一直更看重这种使用工具的能力，包括 Coding；OpenAI 有一段时间非常看重 Reasoning（当然它现在也开始看重 Coding了）。你花意願，就意味着你可能花更多的精力去构建合适的计算设施、合适的数据。数据从某种意义上来说是很花时间、很花精力的事。
- **到现在**：两方麵其实都有。因为纸面上看起来都差不多了，甚至你做更内部的测试，这个数字也变没有那么大区别了。这时候**更难的事情就是你该怎么去定义你想要的行为**。在这个事情没有定义得那么清楚的时候，模型很多的差异其实来自一些你想象不到的事。
**主持人**: 想象不到的事指的是？

**姚顺宇**: 就是你现在去问，其实我很難给你一个特别清楚的答案，可能过一段时间之后我回过头来看才能给一个清楚的答案。

我可以举一个例子：退回一两、甚至三年，那时候如果你去网上取预训练的数据，训练一个模型，你会发现模型在写代码（那时候当然没有 Agent 式的写代码，都是写一段代码）。那时候你可能不知道为什麼，但意外的原因可能是：你从网上随便取数据，**如果不做任何数据筛选的话，自然的这个 code 的数据质量就会比别的高一点**——GitHub 的质量是显著比正常网页要高的。

---

## 对 Open Cloud 的看法：模型能力的“自然溢出”

**主持人**: 在进入今天主题之前，我想先聊聊最近模型的一系列进展。你最近看大家都在讨论 Open Cloud，你作为 AI 的研究员，你对这个新的产品形态是怎么看的？你周围有哪些讨论？

**姚顺宇**: 我觉得有趣的是，这个事情的**业外讨论好像比业内讨论更激烈**。

**主持人**: 业内没有人讨论？

**姚顺宇**: 业内有人讨论，但我觉得对业内的人来说它并不是一个特別令人惊讶的事。可能在公司内部也有人做了类似的实验或者 Demo，只是可能并没有作为一个产品去很认真地宣发、打磨，然后发出去。

实际情况就是：你去看 Open Cloud 最早版本的 GitHub 的那个 code，其实写得也不是特别乾淨。但我觉得很重要的事是，**它给大家展示了这种可能性**。展示这种可能性之后，Open Cloud 作者自己也加入了 OpenAI，那这些模型的实验室或者大一点的创业公司会很快跟上，把这个东西打磨成一个真的可用的产品。

**主持人**: 所以我理解其实在 Open Cloud 发布之前，Google 就已经有人在做这个事情了，只是还没有发布，因为大公司的流程比较长。

**姚顺宇**: 我个人的印象是这样的。

**主持人**: 所以这种类 Open Cloud 的产品形态背后，它本身说明了什么？在今年初的这个时间点上。

**姚顺宇**: 其实技术上来说并不说明什麼。Open Cloud 这个产品当然依赖于模型能做的很多事情，但是那些事情的能力其实并不是到今年年初才准备充足的。我觉得可能在去年，像 Opus 4.5 发布的时候（当然那时候 Opus 其实比 OpenAI 和 Gemini 3 在通用能力上都强一些），其实在那个时间点你做这个事就已经是可以展示出来的。而且它一开始发布也没有立即火，就是发布了一段时间之后才火起来。

**所以对我来说，技术上它并不是一个那么令人惊讶的事。它是模型能力的“自然溢出”。**

但它可能对大家的一个惊喜是：以前可能大家都没有意识到这件事可以做。它让大家意识到：**你可以控制很多不一样的模型，做很多不一样的事情，然后把事情汇总之后做一个非常长（long horizon）的任务**。以前大家并没有广泛地对这个事产生共识，这个 Open Cloud 给大家展示了这种可能性。

## 从 Minus 到 Open Cloud：变化是什么？

**主持人**: 你从去年（2025年）初火的是 Minus，今年初火的是 Open Cloud。从 Minus 到 Open Cloud，变化是什么？是模型能力的变化还是产品的变化？

**姚顺宇**: 这也是一个我一直没理解的事。**Minus 和 Open Cloud 之间的质的区别是什么？** 说實话我自己没看太明白。或者换句话說，Open Cloud 这个事火了，但如果你回头问我说“为什么 Minus 做不了这个事”，我不明白 Minus 为什么做不了。可能只是它没做。

---

## 创业公司为何“卖身”给模型公司：壁垒在模型侧

**主持人**: 但是你看不管是 Minus 还是 Open Cloud，他们都选择了 Minus 卖给了 Meta，Open Cloud 卖给了 OpenAI。这个现象说明什麼？为什么他们都卖了？

**姚顺宇**: 我自己感受是：**一个东西如果要长久地生存，还是需要考虑一些壁垒的**。

- **壁壘是模型**。我觉得至少目前来说，很多壁壘都在模型侧。
- 未来会不会产生产品侧的壁壘？我觉得说不准。市场上老生常谈的什么“数据飞轮”之类的事情，目前来说，**我觉得没有哪一个场景真正形成了数据飞轮**。甚至 AI 纯粹原生的应用场景，我觉得目前除了 **AI Coding（写代码）** 之外，没有哪个场景是 AI 真正原生的、变得非常成功的。
因为从某种意义上来说，**ChatGPT 其实是搜索的一个延伸**。大家和 ChatGPT 最多的交互是“我有一个问题”，然后问 ChatGPT——这其实是搜索本来干的事。但它带来的比搜索要远强的一个事情是它变得非常 **interactive（交互性强）**，你可以追问，它甚至可以帮你总结出一些信息、浓缩成回答你的问题的信息。这是以前搜索给不了你的。但从大的需求上来说，是比较类似搜索的。

Minus 和 Open Cloud 都是现在最有名、最火的，但火最后都卖给了模型公司。**这是不是说明“火”还是难以逃脫模型的掌心？逃逸速度不够快？**

我觉得在当前情况下，“火”起来有兩種我能想象的方式：

**第一种：逃得足够快。**

就是我增长速度足够快，以至于在模型公司反应过来的时候，我已经佔領了大量的用户心智；然后在模型公司追你产品形态的时候，我又自己研发出了自己的模型。

**Cursor** 就是试图在走这条路。Cursor 在这种 AI 原生场景下，几乎是我能想到的创业公司里增长得最快的。

**就算这样的公司，它现在也很有危机感。**

**主持人**: 它都有危机感？

**姚顺宇**: 反正我的感受是：对于 Cursor 来说，现在和 Anthropic 已经进入了一个**非常微妙的关係**。

- 曾经它们是亲密无间的合作伙伴：Anthropic 提供模型，Cursor 提供产品。
- 后来 Anthropic 自己有了 **Claude Code**，Claude Code 现在变得非常成功。
- Cursor 现在又自己试图做自己的模型，所以 Cursor 在努力地训练它的 **Composer**。
所以我觉得都不用说未来，**其实现在它们就已经处于一种比较竞争的关系了**。如果在竞争中输掉的话，我觉得是比較麻烦的。因为 Coding 这件事情，本质上是服务于专业用户的专业需求，是一种**效率工具**。效率工具很容易出现的一个场景就是**赢家通吃**。这个不管对 Cursor 还是对 Anthropic 还是对任何一家做 Coding 的公司来说，可能都是它们比较担心的事。

这是刚才说的第一条路：你长得夠快。

## 个人角色与行业心态：AI 是不可阻挡的浪潮

**主持人**: ……我现在知道它是對的，它會影响到其他的模型。你呢？我也不知道這是好事还是坏事。对，我個人來說，我当然很开心。但是从我个人视角来看，**我对任何一个模型的贡献，我觉得我自己对那个事没那么重要**。我觉得我更多的是：我很幸运，有机会在那个时间、在那个时候，加入了一个重要的项目，做了一些事。
