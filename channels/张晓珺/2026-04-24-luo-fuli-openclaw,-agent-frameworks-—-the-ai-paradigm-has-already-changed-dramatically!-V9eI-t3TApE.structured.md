# OpenClaw 时刻：AI 范式已剧变——对话小米大模型负责人罗福莉

**访谈背景**：本文基于对小米大模型负责人罗福莉（媒体曾称“AI天才少女”，但她不喜欢这个称呼）的深度访谈整理。访谈时间为 **2026 年**，正值 **OpenClaw** 发布之后，以及小米 **MiMo V2** 系列模型（Flash、Pro、Omni、TTS）发布之后。我们深入探讨了由 OpenClaw 引发的新一轮技术范式变迁与未来演进的前沿话题。

---

## 一、对 OpenClaw 的认知转变：从抵触到兴奋

### 1. 初印象：一个“玄幻”的产品

在最初接触 OpenClaw 时，很多人的认知并不深入。罗福莉坦言：

> **“我觉得一个非常大的一个分界点，在于使用 OpenClaw 的前后。我自己其实是会把 OpenClaw 把它当做一个划时代的 Agent 框架去定义。”**
然而，最早看到 OpenClaw 时，她也有过排斥。她认为它只是 **Claude Code** 加一个即时通信（IM）的 UI 设计，加上创始人偏运营导向的风格（如 Skill Hub 等），让她觉得这更像一个产品的交互创新，所谓的“本地化”、“24 小时”只是产品定义。**“所以我一开始很排斥去用它。”**

### 2. 转变发生在春节的深夜

转折点发生在春节假期。罗福莉在某个深夜花了两个小时装上了 OpenClaw：

> **“当时已经凌晨 2 点了，我第一次跟它对话的时候，从凌晨 2 点持续到了 6 点天亮……那一晚上，我脑内的多巴胺（或内啡肽）持续分泌，兴奋到完全睡不着觉。”**
**第一天的感受**：它非常有**自主性**和**灵魂**。它会提醒“你现在已经很晚了，早点去睡觉”。这种温度和情商是所有 OpenClaw 用户最先感受到的。背后的机制在于它做了许多精细的 **context 编排**，例如在每轮对话的上下文中拼上当前时间。

**第二天的感受**：它不止于产品设计。罗福莉尝试将一些日常工作中的难题（例如“如何激发团队好奇心”、“如何筛选有好奇心的人”、“如何构建更好的大模型团队”等）交给它，结果它**全部做出来了**，并形成了一套体系化的 **Skills**。从此，OpenClaw 成了她的“数字分身”。

**第三天的感受**：真正超乎意料的是研究任务。她尝试让 OpenClaw 共同构建一个 **User Agent**（用于模拟多轮交互），这原本是她认为需要较长时间的重要研究课题。结果在一两个小时的沟通后，User Agent 就诞生了，可以直接用于后训练（Post-Train）框架，生成更丰富的 Agent 场景数据，用于 **SFT（监督微调）** 或 **RL（强化学习）**。

> **“它从一个有灵魂有温度的产品设计，到帮我替代生活或工作的一部分，最后到促进我的研究——这也就是三天发生的事情。”**
### 3. OpenClaw 与 Claude Code 的本质差异

罗福莉指出，OpenClaw 相比 Claude Code 的核心优势在于**整合后的完成度**：

- **更持久的记忆体系**：对 memory 进行了**分层和分级**，这是 Claude Code 没有的。
- **多模型的联合利用**：例如，当模型缺乏视频理解能力时，OpenClaw 会**自主寻找**一个视频理解能力好的模型来完成，用户无需手动配置。
- **设计初衷不同**：Claude Code 是为软件工程设计的（记忆系统围绕 session 压缩、plan 记忆等）；而 OpenClaw 是为了**端到端地完成所有任务**，并弥补当下模型的短板。
- **开源性带来的可操纵性**：OpenClaw 是开源的，用户可以自己修改 Agent 架构（如重写 Memory 系统、Multi-Agent 系统）。而 Claude Code 是一个**黑盒**。
> **“这种原生的可操纵性给我的冲击感是很大的。……顶尖的模型应该跟顶尖的 Agent 框架共同地往前去进步。”**

## 二、OpenClaw 引发的范式变革：从 Chat 到 Agent

### 1. 效率的飞跃：三四周完成三四十周的研究

在团队全员使用 OpenClaw 后，研究效率得到极大提升。罗福莉回忆：

> **“春节回来过后两天，整个团队的人就在群里边躁动……十分钟不看群消息就 999+。大家玩得非常 happy，一点都不苦，真的很好玩。”**
一旦进入新的研究范式，借助 OpenClaw 和 Claude Code，**“基本上我们可能在三四周的时间做完了以前可能三四十周才能做到的事情。”**

### 2. 群体智能的力量

罗福莉特别强调了 **群体智能** 的重要性：

> **“大家的想象力是一个乘积的作用。……在一个近 100 人的飞书群里，每个人都有自己的背景，都在疯狂改框架，但没有把这个模型给改坏，反而变得非常智能。”**
这种集体改进让 Agent 框架的进步速度极快，**“可能几个小时就迭代一轮。”** 她认为 OpenClaw 后续的 **star 量飞升** 是 AGI 到来前兆必须要有的事情。

### 3. 为什么 OpenClaw 在中国比在美国更火？

罗福莉给出了两个原因：

1. **中国开发者对效率提升的需求更急迫**。OpenClaw 将效率提升拉到了极致。
1. **国内有大量便宜好用的模型**。绝大部分效率提升场景（约 85%）不需要最顶尖的模型。性价比非常划算：**“可能花 10 块钱的 API 就能帮你干完 1000 块钱的事情。”**
### 4. 为什么以前的 “Agent” 没有成功？

罗福莉认为，以前的所谓 Agent 框架过于简洁和不通用，只能针对特定任务（如 BrowseComp、SWE-bench），远未达到**工业级可用**。

> **“以前那些 Agent 框架就不能叫 Agent 框架吧，没有任何可用度。……工业级可用的标准是：你把它接到 Claude Code 或 OpenClaw 里去用，会发现它不可用。”**
真正的工业级 Agent 需要：

- 人不再修改代码，而是提出更高阶的需求（澄清需求、架构设计、辅助理解业务逻辑）。
- 框架本身具备 **Skill** 的沉淀和复用能力。

## 三、Skill、开源与安全性

### 1. Skill 改变了模型的生态

**Skill** 定义了一套执行的规范，这些规范很难在预训练数据中具备（因为预训练数据没有企业内部的组织规范）。现在的大量 Skills 其实是 **Agent 自己写的**。

> **“人的经验沉淀成 Skills 变得更重要了，它是对预训练的一种补充。很多智能我们在互联网上是访问不到的，现在 Skills 算是一种形态。”**
### 2. 开源的价值与安全性

OpenClaw 的开源属性让更多人能够贡献和改进框架。关于安全性，罗福莉认为：

> **“人的大部分简单任务（与隐私相关但难度不高）完全可以放在本地化去做。……一个好的 Agent 框架借助一个很小的 3B 模型，能做的任务复杂度已经超乎想象。”**
她强调，**开源与安全性并不冲突**，并且开源是**加速 AGI** 的必要条件。

## 四、Agent 时代的定义与未来演进

### 1. 什么是 Agent 时代？

> **“我觉得就是生产力加速变革的时代。……大家会觉得很多工作不需要自己做了。”**
罗福莉认为，**2026 年的主要旋律就是生产力的变革**，高生产力场景的持续突破。具体方向包括：

- 做**更长程的任务**。
- 强调**多 Agent 之间的协作**（但当前的多 Agent 工作有点“伪”，因为并没有真正提升任务完成的上限，更多是提升效率和节省成本）。
- 让 Agent 架构适配企业现有的战略资源（操作系统、硬件、流量、社交等）。
### 2. 自学习与自迭代

OpenClaw 让罗福莉第一次感受到了 **Agent 的自学习**：

> **“你确实需要这个模型本身跟你这个 Agent 的架构本身同步地往前走。模型在进步的时候，也是在改变你的整套 Agent 框架。”**
她认为，未来需要实现三个层面的自进化：

1. 框架本身的自进化。
1. Agent 本身的自进化。
1. Agent 与人之间的自进化。
### 3. 模型与框架的耦合

罗福莉不同意“OpenClaw 只是一个壳，释放了 Claude Opus 4.6 能力”的观点。她认为：

> **“它的上限一定是靠 Claude Opus 4.6 带来的，但中等模型借助这套框架也能发挥巨大作用。……一个好的 Agent 框架应该尽量去弥补模型在行动上的缺陷。”**
一个优秀的框架应具备：

- 很好的 memory 系统。
- 接入更广泛的 message channel。
- 主动设计（定时任务等）。
- 自我更新迭代。
- 一套**可泛化的评估体系**。

## 五、从预训练到后训练：卡资源的重新分配

### 1. 预训练、后训练与研究的最佳卡资源比例

罗福莉给出了一个参考比例：

> **“for 研究、for Pre-Train 和 for Post-Train，我自己觉得一个非常合理的卡的比例是 3:1:1。Pre-Train 和 Post-Train 投入的算力相当，研究的比例应该至少是你正式起训练的卡总量的还要多一点。”**
在过去 Chat 时代，预训练与后训练的比例可能是 **3:5:1**（预训练远大于后训练）。但现在顶尖团队已经做到 **1:1**。

### 2. 1T 模型是入场的“门票”吗？

> **“是 Agent 你要做到接近 Claude Opus 4.6 的水平的这样一个入场券。……我认为至少要 1T 以上的参数规模才能做到。”**
训练 1T 模型的挑战是**全方位的**，尤其是训练过程中的数值不稳定（loss spike、expert 分布极端等）。这需要团队有非常严密的监控系统和跨团队协作能力。

### 3. 环境比经验更重要

> **“这些能力都是可以被快速习得的。我觉得最多一两个月，慢的话三四个月，确实都可以被快速习得。所以环境反而比经验更重要。”**

## 六、MiMo V2 系列模型的技术选择与背后逻辑

### 1. Flash：极致性价比与 Long Context 效率

MiMo V2 Flash 定位为 **long context 高效模型**。其核心设计：

- **Hybrid Attention（混合注意力机制）**：结合 Full Attention 和 Sliding Window Attention。Flash 和 Pro 的层数比从 5:1 提升到 **7:1**，在更大模型上实现更高的稀疏比。
- **MTP（Multi-Token Prediction，多词元预测）**：最初在预训练中加入 MTP 是为了提升基座能力。后续发现模型结构在推理时有大量富余算力，MTP 可以有效利用这些算力，**显著提升推理速度**。
- **效果**：Flash 的输入价格为 **百万 token 1.01 美金**，输出为 **百万 token 0.3 美金**，在当时是最低价格、最高速度。
### 2. Pro：更大参数的 Agent 入场券

Pro 与 Flash 同期训练，但 Pro 遇到了更多训练不稳定性问题（数值 spike、expert 不平衡等）。**“训着训着 loss 就崩了”**，需要花大量时间解决。

Pro 的 **1M 上下文窗口** 需要训练，但难点在于**找不到真正的 1M 上下文稠密监督信号**。长上下文的效果是缓慢提升的。

### 3. Omni：全模态理解与 Agentic 能力

Omni 支持**视频、音频、图片、文本**全模态，并且 Agentic 能力能与纯语言模型相当。罗福莉观察到：

> **“MiMo V2 Omni 比 Pro 小，但大家对世界的感知和领悟力、情商、知识储备上，反而比更大的 Pro 更强。……因为它训过视频，所以知道了更多世界知识。”**
但她也谨慎表示：**多模态是否必然促进智能，她不想下结论**。在 Agent 可以将多个模型能力优雅编排的背景下，多模态的**感知维度扩大**可能不直接促进智能，但**生成能力**或许能。

### 4. TTS：优雅架构与强泛化力

TTS 的动机是**用一套优雅的离散化架构**来做传统上容易的事情。通过数千小时的训练，模型展现出**极强的泛化力**：即使只做了刻板的风格化后训练，当输入复杂的自然语言风格标签时，它也能遵循。

> **“这是简洁架构加超大规模训练带来的超强泛化力的外化表现。”**
### 5. 为什么 MiMo 能如此迅速？

罗福莉总结三点：

1. **核心架构与基础设施研究周期漫长，需要一年前就具备战略信念**（提前决策 long context efficient 架构）。
1. **训练后的敏捷性是另外一种能力**，要快速基于当下模型与 Agent 范式的化学反应，设计新的 RL Infra。
1. **好奇心与热爱**，是一贯的驱动力。

## 七、组织管理与人才观

### 1. 没有组，没有职级

罗福莉的团队 **100 人**（含实习生、数据、开发、产品、算法等），但**没有组**、**没有职级**。

> **“平权本身就是有利于所有人平等地贡献自己的创造力和智慧。……有层级就会默认这个层级上的人应该有更强的智能，这个界定非常奇怪。”**
### 2. 环境比经验更重要

团队中大部分人在加入前**没有大语言模型背景**（刚毕业、做工程、做开发等）。罗福莉认为：

> **“这些能力都可以被快速习得。……我更在乎我是不是创造了一个更好的环境，让大家互相蒸馏长处，快速提升。”**
### 3. 如何驱动热情？

- **体验**是驱动热情的重要方式。她曾用“OpenClaw 对话次数不超过 100 轮的人可以直接 quit”这种极端方式，目的只是让大家去用。
- **筛选**：选择**好奇心强、热爱驱动、基础好**的人。她现在甚至倾向于招**大二、大三的本科生**，因为他们的灵活性和适应程度没有被“污染”，想象力更高。
- **多样性**：招人不能太同质化，否则容易 miss 掉有价值的信息。
### 4. 后训练团队的特质

适合后训练范式的两类人：

- **更注重“跟模型玩”**、频繁与模型交互、维护私有测试库的人。
- 做 **RL Infra** 的人。RL Infra 与 Pre-Train Infra 截然不同：RL Infra 需要**允许容错**，处理异构资源（GPU、CPU、存储），算法与工程的**模糊地带**非常多，对灵活性和敏捷性要求极高。

## 八、关于未来：AGI 的时间线与中美竞争

### 1. AGI 的进度

罗福莉最初认为 AGI 至少还需 **两年以上**，但在经历了 OpenClaw 和 MiMo V2 的冲击后，她认为 **两年内应该能实现**。

> **“我感觉历程已经到了 20%，今年至少能到 60%-70%。”**
### 2. 中美代差

> **“国内已经具备 1T 以上基座的公司有好几家（Kimi、MiMo 等）。……距离国外顶尖（Claude Opus 4.6），如果反应速度足够快，只有两三个月的代差。”**
接下来的 **两三个月** 会非常精彩，比拼的是：

- 团队的**技术敏捷程度**。
- 如何**拥抱新范式**来做研究。
- Agent 框架的进步（自学习、自迭代）。
- **推理需求的爆发**（几倍到十倍的空间），以及对推理芯片的**空前需求**。

## 九、个人感悟与工作状态

### 1. 不断否定昨天的自己

> **“我感觉每天可能都在否定昨天的自己……在这个否定当中自我内省和反省。我的脑子就是一个 Sliding Window Attention，忘得非常快。”**
### 2. 心法

从量化投资转到大模型赛道，她的心法是：

> **“做当下符合我价值观的事情，一定要对更多人产生价值。……如果我们创造大模型的人没有内驱力，而是要做一个破坏的东西，那最后会非常危险。”**
### 3. 工作节奏

她自称睡眠需求很少（4-6 小时足够），工作到凌晨是常态，因为“做的事情有点兴奋，睡太多有点浪费时间”。

## 十、最后的话

罗福莉认为，**OpenClaw 时刻** 不是一个短暂的爆发，而是一个**更长、更深远**的范式转移：

> **“它会延续的时间更长，能力更外放到更多人能感受到的场景。……它产生的价值是慢慢到一个临界点，突然进一大步。”**
而开源正是这一切的火种。无论 OpenClaw 被谁收购，只要开源的属性不变，群体演变的可能性就保留着。

> **“如果我们的工作 90% 被替代后，我们应该干点什么？……我觉得可以搞一个公益型的组织，去支持基础研究。……总要做点有益的事情。”**
