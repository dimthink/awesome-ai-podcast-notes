**Michelle**：是的。

**Vishal**：所以，那时我开始深入研究，尝试建立一个数学模型。

**Michelle**：是的。现在你已经发表了一系列论文。我读的第一篇是你提出那种**矩阵抽象模型**的论文。也许我们先谈谈那个，然后再聊聊你更近期的工作。

### 第一部分：大语言模型的矩阵抽象

**Michelle**：也许我们就从第一篇开始，也就是你试图描述，你试图为 LLM 如何工作建立一个数学模型。

**Vishal**：是的。

**Michelle**：当时你实际上是想弄清楚上下文学习是如何工作的。

**Vishal**：没错。

**Michelle**：然后你为 LLM 提出了一种抽象模型，基本上就是一个非常大的矩阵，并用它来进行描述。也许你可以快速地跟我们介绍一下那项工作。

**Vishal**：当然。你可以想象一个巨大无比的矩阵，矩阵的每一行对应一个**提示（prompt）**。这些 LLM 的工作方式是，给定一个提示，它会构建一个**下一个词元（token）** 的概率分布。下一个词元就是下一个词。每个 LLM 都有一个词表，GPT 及其变体的词表大约有 5 万个词元。所以，给定一个提示，它会给出下一个词元应该是什么的分布。然后所有这些模型都会从这个分布中采样。

**Michelle**：是的。那就是**后验分布**。

**Vishal**：对，那就是后验分布，这正是 LLM 的工作方式。所以，这个矩阵的概念就是：对于所有可能的词元组合（即一个提示），都有一行与之对应。

**Michelle**：对。

**Vishal**：而矩阵的列则是对词表中所有词元的概率分布。

**Michelle**：所以如果词表有 5 万个可能的词元，那就是一个关于这 5 万个词元的分布。

**Vishal**：没错。

**Michelle**：说到分布，其实就是概率。

**Vishal**：就是概率。抱歉，是的。就是下一个词元是这个而不是那个的概率。这就是基本概念。当你这样看待它时，至少对我来说，事情就变得更清晰了，让我这种想要建模的人能看明白到底发生了什么。

具体来说，假设你有一个例子：提示只是一个词“**蛋白质**”。那么，下一个词元的分布中，大部分概率会是零，但在比如两个词上会有非零的、不可忽略的概率：一个是“**合成**”，另一个是“**奶昔**”。

现在，LLM 会采样“合成”或“奶昔”作为下一个词元。或者，作为人类，你会给出提示“**蛋白质奶昔**”或“**蛋白质合成**”。现在，取决于你选择了“合成”还是“奶昔”，接下来的那一行会看起来非常不同，对吧？如果你选了“蛋白质合成”，那么接下来概率高的词元都将与生物学有关。但如果你选了“蛋白质奶昔”，那么接下来就会全是关于健身房、锻炼和健身之类的内容。所以，选择“合成”还是“奶昔”会完全改变后续的内容。

**Michelle**：是的。所以这是一个可以称之为**贝叶斯更新**的例子。你从“蛋白质”开始，你对“蛋白质”之后的内容有一个先验概率。一旦你得到新证据，比如下一个词是“合成”或“奶昔”，你就会完全更新这个概率分布。

**Vishal**：现在你可以想象，整个 LLM 就是这个巨大的矩阵，每一行都是像“蛋白质”、“蛋白质奶昔”、“蛋白质合成”、“猫坐在了”、“矮胖墩”等等这样的内容。考虑到这些 LLM 的词表（比如 5 万）和上下文窗口（例如，第一版 ChatGPT 的上下文窗口是 8000 个词元），如果你看看所有可能的 8000 个词元与 5 万词表的组合，这个矩阵的行数将超过全宇宙所有星系中的电子总数！所以，LLM 绝对不可能精确地表示它。幸运的是，这个矩阵非常**稀疏**。为什么？因为任意随机的词元组合是乱码，我们在现实生活中的自然语言里永远不会用到。同样地，矩阵的列也大多是零。如果你有“蛋白质”这个词，那么后面不太可能出现任意的数字或词语。所以，无论是行还是列，它都非常稀疏。

因此，在我看来，所有 LLM 所做的，在某种程度上，就是对这个矩阵提出一个**压缩表示**。当你给出一个提示时，它们试图近似出真正的分布应该是什么，并尝试生成它。至少在我心里，这就是它的本质。

**Michelle**：根据我的理解，如果你有一行是“蛋白质”，然后另一行是“蛋白质奶昔”……

**Vishal**：嗯。

**Michelle**：“蛋白质奶昔”是“蛋白质”的子集，还是不同的行？

**Vishal**：是不同的，它是从“蛋白质”延续下来的。

**Michelle**：我明白了。

**Vishal**：对。

**Michelle**：但我只是说，实际的后验分布，它是子集吗？

**Vishal**：你可以说它是子集。如果你有“蛋白质”，那么“蛋白质奶昔”和“蛋白质合成”都是从“蛋白质”延续下来的。所以“合成”和“奶昔”都有非零的概率。所以你可以，是的，可以某种程度上把它看作是子集。

### 第二部分：上下文学习与贝叶斯更新

**Michelle**：对。你用了这个方法来描述上下文学习是如何工作的。也许先描述一下什么是上下文学习，然后再说说你从中得出的结论。**上下文学习**就是，当你向 LLM 展示一些它从未见过的东西，你给它几个例子说明你想做什么，然后你给出一个与你展示的例子相关的新问题，LLM 能够实时地学会它应该做什么并解决那个问题。顺便说一句，我第一次看到这个的时候，简直惊呆了。而且我刚开始学习的时候，确实用过你的那个 **DSL（领域特定语言）** 例子。能像 DSL 这样工作简直太神奇了。

**Vishal**：它绝对令人叹为观止。回到那个板球问题。90 年代中期，我曾是一个团队的成员，创建了一个名为 Cricinfo 的板球门户网站。板球是一项数据极其丰富的运动，你可以把它想象成棒球再乘以一千，有各种各样的统计数据。我们创建了一个名为 Statsguru 的在线可搜索数据库，你可以搜索任何与板球相关的数据。自 2000 年以来，它一直可用。但是，因为你可以查询任何东西，所有的数据都需要能被访问到。那么，你如何让普通大众也能使用这样的东西呢？他们不可能去写 SQL 查询。当时次优的解决方案是创建一个网页表单。不幸的是，所有东西都被塞进了那个表单里。结果就是，你有大约 20 个下拉菜单、15 个复选框、18 个不同的文本字段。它看起来像一个非常复杂、令人生畏的界面。因此，尽管它可以解决或回答任何查询，但几乎没人用它。使用它的板球迷比例小得可怜，因为它看起来太吓人了。

后来，ESPN 在 2007 年收购了那个网站。我现在仍然认识一些运营该网站的人，我总是跟他们说，你们为什么不对 Statsguru 做点什么？2020 年 1 月，Cricinfo 的主编 Sambit，他是我的朋友，来纽约，我们出去吃饭时，我又跟他提了这事：“你们为什么不对 Statsguru 做点什么？”他看着我，开玩笑地说：“那你为什么不对 Statsguru 做点什么？”但这个想法一直留在我心里。当 GPT-3 发布时，我想，也许我可以用 Statsguru，用 GPT-3 为 Statsguru 创建一个前端。

所以，我所做的是设计了一个 **DSL（领域特定语言）**，它可以将关于板球统计数据的自然语言查询转换成这种 DSL。

**Michelle**：澄清一下，这个 DSL 是你自己创建的，它并不存在于任何 GPT-3 可能见过的在线训练数据中。

**Vishal**：对，GPT 不可能见过。是我创建的。我觉得这很合理，所以我设计了那个 DSL。然后我做了那个少样本学习的事情。我建立了一个数据库，大概有 1500 个自然语言查询以及与之对应的 DSL 查询。所以，当一个新的查询进来，有人用英语问一个数据问题时，我会做的是：遍历这些自然语言查询，进行语义搜索，选出最匹配的前几个，然后用那个自然语言查询及其对应的 DSL 作为前缀一起发送给 GPT-3。如果你还记得，GPT-3 当时的上下文窗口只有 2000 个词元，所以你必须要非常明智地选择例子。选好例子后，我把新的查询发送过去，GPT-3 就会用我设计的那个 DSL 来完成它。而这个 DSL 它在几毫秒前还从未见过。我无法访问 GPT-3 的内部结构，也无法访问它的权重，但它仍然能工作。

**Michelle**：所以，根据你之前的矩阵例子（一个提示对应一个分布），对我来说，上下文学习是如何工作的并不显然。所以我认为你的第一篇论文就是 tackled 这个问题。

**Vishal**：对。

**Michelle**：那么，也许你可以讲解一下你对 LLM 如何进行上下文学习的理解。

**Vishal**：好的。所以当你思考什么是上下文学习时，它是当你看到证据时发生的过程。在第一篇论文中，我也做了件事，我用了这个板球 DSL 的例子，并描绘了当模型被展示越来越多例子时，**下一个词元的概率**是如何变化的。第一次我向它展示这种 DSL（自然语言和对应的 DSL）时，DSL 词元的概率极低，因为 GPT-3 从未见过这种东西。当它看到板球问题时，它在脑海中试图用英语答案来续写。所以那时概率高的都是英语单词。一旦它看到了我的提示（其中包含了问题和 DSL），下一次我在下一行给出问题时，DSL 词元的概率就开始上升。随着每一个例子的给出，这个概率都在上升。最后，当我给出新的查询时，它几乎有 100% 的概率得到正确的词元。

**Michelle**：对。

**Vishal**：这就是一个模型在**实时更新其后验概率**的例子。它正在更新它的知识：“好的，我看到了证据，这是我应该做的。”用通俗的话说，这就是**贝叶斯推理**。**贝叶斯更新**本质上就是，你从一个先验概率开始，当你看到新证据时，你更新你的后验概率。这是数学定义。但在日常语言中，它就是你看到新事物，然后更新你对正在发生的事情的信念。

**Michelle**：对。

**Vishal**：所以对我来说很清楚，LLM 正在做一些类似于贝叶斯更新的事情。所以在第一篇论文中，我有了这个矩阵的公式，并展示了它看起来就像贝叶斯更新。

**Michelle**：是的。

**Vishal**：然后我们就可以进入下一个系列的论文了。

### 第三部分：贝叶斯风洞——严格的数学证明

**Michelle**：没错。所以，嗯，在我看来，那时这已经相当有结论性了。然后你沉寂了一段时间。之后我仍然记得那条 WhatsApp 消息，你说：“Martin，我现在确切地知道这些东西是如何工作的了。”

**Vishal**：是的。

**Michelle**：然后，你发表了一系列论文，可以说在互联网上引起了轰动。你在 Twitter 上超级火，很多人都注意到了。我想马上谈到那个。但在此之前，我记得你的第一篇论文出来时，人们会说：“这些东西肯定不是贝叶斯的。任何东西都可以被看作是贝叶斯的，但它们不是。”你觉得为什么会有这种反应，好像它们不是贝叶斯的？我感觉几乎有一种反弹，只是因为它们被这样定性了。

**Vishal**：对。我认为在概率论和机器学习的整个领域里，一直存在着贝叶斯学派和频率学派两个阵营。我不想卷入那种政治斗争，但“贝叶斯”这个词几乎成了人们的一种条件反射。这是那场古老战争的一部分。

**Michelle**：我明白了。

**Vishal**：所以，这就像古老的贝叶斯与频率学派的斗争。所以人们只是说：“哦不，什么都可以说是贝叶斯的，对吧？”所以我心想，好吧，也许他们有点道理，也许我们所说的并不真的是贝叶斯的。我们如何证明它就是贝叶斯的呢？所以，首先我要感谢你和 Horowitz 的帮助。当我在第一篇论文中展示这些概率时，是因为 OpenAI 在其聊天界面中提供了显示这些概率的选项，但后来他们停止了。

**Michelle**：对。

**Vishal**：所以我们无法再窥探内部发生了什么。出于某种原因他们停止了，我不打算讨论 OpenAI 是开放还是封闭，但他们停止了。所以我们开发了我们自己的界面，叫做 **Token Probe**，它可以让你不仅看到下一个词元的概率，还能看到其熵值。这是基于一个开源模型构建的。是的，所以你可以加载任何类型的开源模型。但是，作为学术界人士，我们之前没有计算资源。多亏了你慷慨的捐赠，我们获得了运行 Token Probe 所需的集群。你可以访问 tokenprobe.cs.columbia.edu。

**Michelle**：它还在运行吗？

**Vishal**：还在运行，仍然有人在用。我在课堂上用它来让学生做作业，他们编写自己的 DSL，然后说这真的帮助他们理解 LLM 是如何工作的。所以，老实说，我对 LLM 的理解就来自于 Token Probe——就坐在那里，看着你填写提示时概率分布的变化。这真的非常非常有启发性。所以对于正在收听的朋友们，网址是什么来着？

**Michelle**：Token Probe。tokenprobe.cs.columbia.edu。去看看吧。这确实是一个非常实用的方法，可以实际看到概率分布是如何随着你填写提示而更新的。

**Vishal**：对。但后来我“作弊”了。你知道，它在运行，但我也可以访问驱动它的 GPU。然后我和哥伦比亚的同事们（其中一位现在在 DeepMind）开始思考，如何才能真正证明它是贝叶斯的。

**Michelle**：你能解释一下吗？实际上我并不知道答案。对我来说，你在第一篇论文里就已经证明了，还缺什么？

**Vishal**：在第一篇论文中，我们展示的是**经验性的**。

**Michelle**：我明白了，是你能看到。

**Vishal**：你可以看到。但对我来说这很明显，是的。但要说服那些会 dismiss 说“任何东西都可以是贝叶斯的”的人，我们必须从数学上精确地展示它。

**Michelle**：懂了，懂了。

**Vishal**：所以我们想出了这个主意，我和我的同事 Naman Agarwal 以及 Siddharth Dalal 一起（这一系列论文都是和他们合著的），我们想出了“**贝叶斯风洞（Bayesian Wind Tunnel）**”这个点子。什么是风洞？在航空航天工业中，风洞是在隔离环境中测试飞机的地方。你不会真的让它飞起来，而是测试它承受各种空气动力学压力的能力，看看它能承受什么，等等。你不想在高空中做这些测试。所以我们说，好吧，我们为什么不创建一个环境，我们拿来这些架构——测试 Transformer、Mamba、LSTM、MLP 等各种架构——我们拿一个空白的架构，给它一个**不可能通过记忆来解决的任务**。这个空间在组合意义上是不可行的，给定参数数量，我们用了非常小的模型。所以它足够困难，以至于它们无法记忆，但又易于处理，以至于我们精确地知道**贝叶斯后验应该是什么**，你可以解析地计算出来。

我们给了这些模型一系列任务，在这些任务中，我们再次表明它们不可能记忆。我们训练了这些模型，发现 **Transformer** 得到了精确的贝叶斯后验分布，精度达到了 **10 的负 3 次方比特**。它完美地匹配了分布。所以，从数学意义上讲，当给定一个需要更新信念的任务时，它实际上是在做贝叶斯更新。**Mamba** 也做得相当不错。**LSTM** 可以做其中一部分。所以在那几篇论文中，我们对贝叶斯任务进行了分类。Transformer 能做所有任务，Mamba 能做大部分，LSTM 只能做一部分，而 **MLP 则完全失败**。

**Michelle**：这是反映了它训练所用的数据，还是更多地反映了机制本身？

**Vishal**：是机制，是**架构**决定的。数据决定了它学习什么任务。所以在第一篇论文中，我们有了这些贝叶斯风洞，并展示了它在做贝叶斯更新。在第二篇论文中，我们展示了它**为什么**能做到。我们观察了 Transformer，观察了梯度，展示了梯度实际上是如何塑造了一种几何结构，使得这种贝叶斯更新成为可能。然后在第三篇论文中，我们做的是，我们拿来了那些**前沿的生产级 LLM**，它们具有开放权重，这样我们就可以窥探内部。我们进行了测试，发现在小型模型中看到的几何结构，在拥有数亿参数的更大模型中依然存在，同样的特征。唯一的区别是，因为它们是在各种数据上训练的，所以会有些杂乱，但你可以看到相同的结构。

所以，**贝叶斯风洞**背后的整个想法是，与这些生产级 LLM 不同——你不知道它们训练过什么，因此你无法从数学上计算后验——风洞为我们解决了这个问题。我们说，好吧，让我们从一个空白的架构开始，给它一个我们知道答案的任务，它无法记忆，让我们看看它会做什么。

### 第四部分：人类 vs. 机器——可塑性、因果关系与 AGI 之路

**Michelle**：那么，你认为这能否为人类如何思考提供某种启示？还是说这些东西是完全独立的？

**Vishal**：不，它确实提供了启示。人类也会在看到新证据时更新我们的信念。所以在某种意义上，我们也在做某种贝叶斯更新，但我们会做得更多，我待会儿会谈到。这些 Transformer 甚至 Mamba 都在做这种贝叶斯更新。但人类的不同之处在于，我们会根据新证据更新后验，但我们大脑经过数亿年进化的优化目标是“**不要死，要繁殖**”。这一直是驱动力。我们的大脑学会了调整，所以当我们看到某种危险——灌木丛里有东西在沙沙作响，不要靠近——我们知道如何应对这种危险，我们知道如何保护自己。我们将这种学习内化了，我们的脑细胞或突触在**整个生命周期中保持可塑性**。

而 LLM 的情况是，一旦训练完成，那些权重就被**冻结**了。当你在进行推理时，例如在对话中进行上下文学习或其他任何事情，你确实在进行贝叶斯推理，但之后你就**忘记了**。下一次新的对话开始时，上下文为零，它不会保留之前那次交互中学到的任何东西。所以，以我做的那个板球 DSL 为例，每一次调用都是全新的，它不记得我上次发送查询时 DSL 是什么样子的。所以这是人类使用贝叶斯更新的一个区别，我们一生都保持可塑性，而 LLM 是被冻结的。

**Michelle**：对。

**Vishal**：还有另一个区别。

**Michelle**：告诉我。

**Vishal**：另一个区别是，首先，正如所说，我们的目标是“不要死，要繁殖”。LLM 的目标是**尽可能准确地预测下一个词元**。所以，你读到的那些可怕的故事，比如“哦，LLM 试图欺骗，试图阻止自己被关闭”，这不是架构的功能，而是训练数据的功能。它被喂食了 Reddit 或社交媒体上的文章，仅此而已。

**Michelle**：顺便说一句，就在今天，Daario 据称说不能排除它们有意识的可能性。

**Vishal**：你可以排除它们……我是说，拜托。Anthropic 生产很棒的产品。Claude Code 非常棒。他们做的很多工作都非常出色。但它们是进行矩阵乘法的**硅晶粒**。它们没有意识。它们没有内心独白。它们不受同样的目标函数驱动：“不要死，要繁殖”。它们受“不要在下个词元上犯错”驱动。而这完全是由训练数据驱动的，对吧？你用社交媒体或 Reddit 上关于“为了生存它会这样做或那样做”的故事来训练 LLM，它就会再现这些。所以这只是一个**反射**，不是心智。而且这些结果，再说一遍，是**完美的贝叶斯**。完美地，是的。

**Michelle**：到小数点后？

**Vishal**：到小数点后。我训练了它 15 万步，精度达到了 10 的负 3 次方比特。我本可以继续训练，但这只用了半个小时，在你为 Token Probe 提供的基础设施上，在后台用那些 GPU 训练的。所以再次感谢你。但是，回到人类，人类是贝叶斯的，但我们还做了别的事。你知道，当我把这支笔扔向你，你会怎么做？你会躲开，对吧？

**Michelle**：是的。

**Vishal**：你为什么要躲开？

**Michelle**：为了避免被击中。

**Vishal**：为了避免被击中。但你的大脑并没有在做贝叶斯计算：“好的，这支笔飞过来了，它击中我的概率是多少，会造成多少疼痛等等。”你实际上在脑海中做的是**模拟**。你看到笔飞过来，你知道它会击中你。你的大脑模拟了这个过程，然后你躲开了它。所以，所有的深度学习都是在做**相关性**，而不是做**因果关系**。因果模型是那些能够进行**模拟和干预**的模型。你知道，Judea Pearl 提出了整个**因果层次结构**，第一个层次是**关联**，也就是建立这些相关模型。深度学习在这方面很出色，非常强大，我的意思是，你每天都能看到这些模型惊人地好。它们做的是关联。第二个层次是**干预**。深度学习模型不做这个。第三个层次是**反事实推理**。干预和反事实推理，你可以想象它们都是某种模拟。你建立了一个关于正在发生的事情的因果模型，然后你能够进行模拟。我们的大脑能做到这一点。目前的架构做不到。

另一个我认为能让这一点更清楚的例子是**香农熵**和**柯尔莫哥洛夫复杂度**这两个技术术语的区别。

**Michelle**：当然。

**Vishal**：如果你看看圆周率 π 的数字的香农熵，它是无限的，不可能预测和学习下一个数字会是什么。香农熵试图建立相关性，试图学习相关性。深度学习做的就是香农熵。另一方面，**柯尔莫哥洛夫复杂度**是能够重现所讨论字符串的**最短程序的长度**。而生成 π 的数字的程序非常小。多亏了拉马努金等人，有很多非常小的程序可以精确地重现它。所以 π 的柯尔莫哥洛夫复杂度非常小，而香农熵是无限的。我认为深度学习仍然停留在香农熵的世界里，还没有跨越到柯尔莫哥洛夫复杂度和因果世界。

**Michelle**：哇，有意思。那么，你认为这在多大程度上为我们改进现有技术提供了研究方向？我给你举个具体的例子，你谈到了人类实际上不会更新那个“矩阵”，不会更新他们的权重。但现在有很多关于**持续学习**的研究。那么，你的工作是否为如何解决这些问题提供了一些指导？特别是，我一直有个疑问：我们用了如此多的数据和算力来创建这些模型，认为可以在实时情况下更新权重并产生有意义的影响，这真的合理吗？我的意思是，似乎你需要多得多的数据才能做到这一点。那么，你能开始回答这些问题吗？

**Vishal**：你可以开始回答其中一些问题。目前存在的一个误解是，**规模可以解决一切问题。规模不能解决一切问题**。你需要一种不同类型的架构。而且这个**持续学习**是一个难题。你必须在学习新东西和**灾难性遗忘**的风险之间取得平衡。如果你更新了权重，却忘记了什么是重要的，忘记了已经学到的东西，那你就没有进步。那它就会变成一个随机的、混乱的模型。所以解决这个问题很困难，这是其中一个方面。

因此，要达到所谓的 **AGI（通用人工智能）**，我认为需要发生两件事。第一是通过持续学习实现这种**可塑性**。第二，我们必须从**相关性走向因果性**。

**Michelle**：对。这与杨立昆谈论的因果与规划有多相似？你知道，预测你的行为会产生什么影响。

**Vishal**：是相关的。他从与 Judea Pearl 模型不同的角度来探讨，但它们是相关的。另一件事是，我第一次来这个播客时，提到过一个 AGI 的测试，叫做**爱因斯坦测试**。我说，你拿一个 LLM，用 **1916 年（或 1911 年）以前的物理学**数据训练它，看看它是否能提出**相对论**。如果它能，那我们就有了 AGI。这是一个很高的标准，但我们应该有高标准。它做不到。

**Michelle**：对。

**Vishal**：我认为 Demis 几周前在印度人工智能峰会上也提到了同样的测试，引起了很多新闻。但为什么是这样？这又如何与我们刚才讨论的香农熵与柯尔莫哥洛夫复杂度的观点联系起来？在爱因斯坦的时代，有很多线索表明牛顿力学有问题。人们知道水星的轨道不合常理，有些不对劲。还有**迈克尔逊-莫雷实验**，他们试图找出一种叫做“以太”的介质，认为光通过它传播。他们觉得如果从不同方向反射光，光速可能会改变，他们可以用非常精密的仪器探测到这种变化。他们尝试了好几次实验，什么也没发现，光速完全没有改变。还有黑洞、引力透镜等问题。所以有很多迹象表明牛顿力学并不能解释一切。但在爱因斯坦提出一个新的**时空容器表示**之前，我们一直被困在原地。

如果你有一个只看重相关性的模型，它看到了所有这些个别的证据，它也无法提出爱因斯坦提出的那个优美方程。我忘了具体是什么了，大概是 `G`*`μν = 8πT`*`μν` 之类的，那个描述时空连续体的张量方程。他提出了一个新的公式。他基本上拒绝了现有的公理，提出了一个关于世界的非常简短的柯尔莫哥洛夫表示。一个方程，其他一切都可以从中推导出来，无论是引力波、黑洞、水星轨道还是 GPS 如何工作。我们每天在手机里使用的 GPS，就用到了相对论的方程。

**Michelle**：所以，这是否最终变成了，你几乎必须**忽略大部分先前数据**才能做到这一点？而 LLM 做不到，因为它们就是在大部分先前数据上训练的。这就像有一种数据引力把你往回拉。就像每个人都说是 X，有一点证据说是 Y，但因为每个人都说是 X，LLM 总会说是 X，它会将 Y 视为异常值。

**Vishal**：没错。实际上，这是一个非常好的说法。它会总是把 Y 当作异常值。

**Michelle**：我现在明白了，你之前说的香农熵与另一个的区别。香农熵总是受限于现有信息的总量，这就是现在发生的情况。而你可以用新的数据，用一个更短的描述来描述一切，这需要一种全新的表示。

**Vishal**：是的，你需要一个新的表示。还记得我们上次谈到的另一个思考方式：宇宙是一个非常非常复杂的高维空间，然后人类以某种方式将其映射到一个**低维流形**上，这个流形相对不那么复杂，然后被记录下来。LLM 学习的就是这个流形。然后它们使用贝叶斯推理在这个流形上移动，但它们被束缚在这个流形上。它们做不到的是**生成一个新的流形**，这需要理解宇宙运作的方式并提出一个新的宇宙表示。而这就是相对论所做的。

**Michelle**：没错。

**Vishal**：爱因斯坦必须创建一个新的流形。如果你固守在牛顿物理学的旧流形上，你会看到这些相关性，但你无法提出一个能解释它们的流形。所以你需要提出一个新的表示。对我来说，AGI 有很多定义。图灵测试，我们已经通过了。执行经济上有用的工作，每天你都能看到 LLM 在做这些。

**Michelle**：我们达到了吗？我不知道。我是说，没有人类干预的情况下？

**Vishal**：不，不，那是另一回事。但尽管如此，这就像汽车能跑得比人快一样。这是一个很肤浅的定义。所以所有这些定义，比如“做有用的事”，也许 6 个月后你就能让 Claude 或 Gemini 在没有干预的情况下完成定义明确、范围清晰的任务，这是可能的。但对我来说，当这两个问题得到解决时，AGI 才会真正实现：通过持续学习实现的**可塑性**，以及以一种更**数据高效**的方式建立**因果模型**。

**Michelle**：我们最近听到人们在谈论涌现性，比如几天前 Donald Knuth 的那个“啊哈时刻”，在 X 上 viral 了。你认为这是否表明我们正在看到涌现性？

**Vishal**：不，不。实际上，这恰恰验证了我一直在谈论的观点。如果你读了他所做的事情，他在一位同事的帮助下，让 LLM 解决一个特定的问题——寻找奇数个顶点的哈密顿循环，我们就不深入细节了——他让 LLM 为一个又一个的奇数 N 持续求解。他还做了一件事：在 LLM 找到特定 N 值的解之后，他让 LLM 用解决那个问题时学到的东西**更新它的记忆**。所以 LLM 尝试了很多不同的东西，有些成功了，就更新记忆。这有点像**拼凑出来的可塑性**。它随着进展学习它已经做过的事情。再说一次，这是一个 hack 出来的版本。你没有改变权重，你只是在改进上下文。但是，随着学习，哈密顿循环及其相关数学的整个空间在这些 LLM 训练过的流形中是**有良好表示的**，你只需要找到正确的连接，而 LLM 很擅长计算，投入足够多的计算，它们就能找到正确的连接。所以 Knuth 能够利用 LLM 的尝试，并最终需要他将所见整合成一个解决方案。LLM 在某个阶段会卡住，你读他写的东西，这刚发表，我想是两天前。

**Michelle**：两三天前。

**Vishal**：但最终，他使用了这些解决方案，并提出了证明。所以，这就像爱因斯坦看到了所有这些证据，然后他思考什么能解释它们，他想出了一个因果模型。所以 Knuth 和他的大脑就是那个**柯尔莫哥洛夫**部分，是人类。而 LLM 在完成**香农**部分时极其高效，它通过尝试各种东西，并更聪明地分解问题，找到了所有的解决方案。

**Michelle**：我想知道，这是否又回到了同样的问题：你认为这为我们下一步要解决的问题提供了一些洞见吗？比如，是否存在一种机制可以获得柯尔莫哥洛夫复杂度？或者说，这告诉了我们方向，但显然没有告诉我们如何做？

**Vishal**：是的，它告诉了我们方向，但没有告诉我们如何做。但即便是柯尔莫哥洛夫复杂度，在很大程度上也仍然是一个理论概念。没有算法，没有找到最短程序的实用方法。我们知道它存在，你可以争论它，但就是这样。所以我认为，这是我的偏见，我们的精力应该集中在那里，而不是更大的模型和更多的词元。

**Michelle**：你能把这两者联系起来吗？这如何与做模拟相关联？还是说模拟是完全正交的？

**Vishal**：不，模拟是相关的。你可以这样想：基本上你做模拟，某种程度上就是朝着做柯尔莫哥洛夫复杂度迈进的一步。**模拟器就是我们创建的程序**。它可能不是完美的程序。但在我们的大脑里，我们创建了这个模拟器，所以当我扔笔时，你知道它朝你飞过来，你就躲开。所以，你并不是在计算概率，而是你建立了一个非常物理的模型。

**Michelle**：概念上，但机制相同？

**Vishal**：确实是相同的机制。你必须建立一个因果模型。

**Michelle**：对。

**Vishal**：对大多数事情来说都是如此。所以，你必须从相关性走向因果性。我们听过这个词无数次了，但在这里，它确实影响了我们看待智能的方式。

**Michelle**：你最近这三篇论文的反响如何？

**Vishal**：我不知道。嗯，arXiv 版本……我可以告诉你，反响很好，很多人读了。我只是好奇你得到了什么样的反馈。

**Vishal**：我得到了很好的反馈。但我是这个领域的**局外人**，对吧？我是个搞网络的。人们会想，为什么他写关于学习、机器学习和深度学习以及贝叶斯的文章？但是那些真正花时间读过这些论文的人，我得到了非常好的反馈。最近 Google Research 有一篇论文，试图通过某种 RLHF 来教会 LLM 正确地进行贝叶斯学习，这正朝着这个方向。我认为人们开始接受 LLM 在做贝叶斯学习这个观点了。我知道有些人也看了贝叶斯风洞论文的 arXiv 版本，并重现了实验。他们读了论文，然后自己做了训练，看到这确实发生了。

**Michelle**：那太好了。下一步是什么？

**Vishal**：下一步是，我希望在这两个并行轨道上取得进展：**可塑性**和**因果性**。因为今天，你已经拿了一个现有的机制，并为它如何工作创建了一个正式模型。所以现在你真正感兴趣的是改进它，创建一个新的机制。

**Michelle**：你认为这会是一个完全不同的架构，还是 LLM 也是解决方案的一部分？

**Vishal**：我认为 LLM 绝对是解决方案的一部分。

**Michelle**：我明白了。

**Vishal**：但必须有更多的东西。所以我之前不感兴趣于 catalog 所有这些 LLM 能做什么，我更有兴趣的是它们**为什么**以及**如何**做到。我认为我们现在对“为什么”和“如何”有了很好的把握。下一步就是如何将它们提升到下一个水平。我们现在，我想，对它们的**局限性**有了相当好的理解。现在，如何进入下一步？

**Michelle**：对于因果关系，是否存在一个类似的理论框架可以应用在这里，就像贝叶斯之于推理一样？

**Vishal**：嗯，Judea Pearl 的整个因果层次结构，我认为那是正确的框架。那是一个非常好的框架。你知道，整个 **do-演算**的方法，我认为这是思考这个问题的好方法。那种关联、干预、反事实的层次结构，在数学上把你从相关性带到实际的模拟。

**Michelle**：太棒了。

**Vishal**：对。

**Vishal**：希望如此。

**Vishal**：谢谢你邀请我。
