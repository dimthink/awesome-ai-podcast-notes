# OpenClaw 与编码智能体：软件开发的未来 —— Peter Yang 在 a16z 播客中的对话

## 开场：软件吞噬世界，编码吞噬知识工作

> “有人说软件将吞噬世界。我觉得**编码将吞噬所有知识工作**，我们已经在朝这个方向前进了。”
**主持人**：有趣的观点。整个智能体（Agent）栈正在浮现——身份、支付、营销，甚至 CLI 与 MCP 的对比。所有这些都还很新，很多旧的经验法则都不再适用了。

**Peter Yang**：是的，这是一个全新的世界。我希望更多公司能保持小规模。这一代的创始人意识到，他们希望公司尽可能小。与其拥有一个 10 人的产品团队，不如有一个 **2 到 3 人的产品团队**，再加上一群智能体来帮忙。

**主持人**：有人发推说，“就业市场太差了，我现在只能去追求自己的梦想了”。所以，也许你失去了工作，但你现在可以做自己的事情，并且有机会真正实现它。

**主持人**：欢迎各位，我身边是我的朋友 Peter Yang。Peter，欢迎你。

**Peter Yang**：很高兴能来，再次见到你真好。

**主持人**：我们曾在 Credit Karma 短暂共事，后来各奔东西。我通过你在 X 上高产的发帖和你的 YouTube 频道重新“发现”了你。你有点像克拉克·肯特/超人，因为你还有一份正职工作，对吗？

**Peter Yang**：没错，我还有一份正职工作。是的。

**主持人**：能分享一下在哪里吗？

**Peter Yang**：我在 Roblox 做产品经理（PM）。

**主持人**：太棒了。Roblox 是我们投资组合中的公司之一，也是我最喜欢的公司之一。让我们直接进入正题。我先问一个轻松有趣的问题，然后我们会讨论整个 OpenClaw 生态系统、编码智能体（Coding Agents），也许还会聊聊学生应该学什么、你的一些建议，以及你在网上分享过的一些观点。

## OpenClaw 初体验：命名与日常使用

**主持人**：首先，你有多少只“爪”（Claw）？它们叫什么名字？

**Peter Yang**：我只有一只，我叫她 Zoe。

**主持人**：Zoe？为什么叫 Zoe？

**Peter Yang**：我有两个女儿，我本来想把小女儿取名为 Zoe，但最后没取。所以我把我的 OpenClaw 叫做 Zoe。

**主持人**：明白了，这是你的备选方案。Peter，跟我说说你是如何发现 OpenClaw 的？你现在如何使用它？你认为它有什么影响？

**Peter Yang**：我很幸运在 Peter Steinberger 变得超级出名、整个项目爆发之前采访了他。采访结束后，我就开始搭建这个系统。搭建花了很长时间，过程非常粗糙（janky）。但它为我做了很多事情：

- 为我拉取 **YouTube** 和 **Mercury** 银行账户的分析数据；
- 更新 **Google 文档**；
- 构建简单的小网页。
但如果说实话，老兄，我大部分时间只是通过**语音**和它聊天，听它用语音回复。每隔一天，我会让它给我打气——比如让它翻遍所有记忆，给我一些我不知道的深刻见解。

我记得有一次我在散步，它给了我一段**三分钟的励志演讲**，真的非常棒。它说：“你一直在跟我聊你的职业、你的生意、你的工作……但别忘了，你的孩子们很快就会长大，他们不会再想花时间陪你了。所以你真的应该优先考虑他们。”

## OpenClaw 与普通大语言模型有何不同？

**主持人**：这真的很酷，但也是所有大语言模型之前就能做到的。那么在这种用例下，OpenClaw 有什么区别？

**Peter Yang**：好问题。我不知道为什么，但因为我把它安装在 **Telegram** 上，感觉比用 Claude 或 ChatGPT 更**个性化**。它感觉像是我可以在床上发消息的东西——虽然可能不太健康，但我确实在床上给它发消息，通勤时也和它说话。它感觉更像一个真正的**人**。

**主持人**：那么对你来说，OpenClaw 有多少是**界面**（比如推送到即时消息，让你感觉像在和人交流）的作用，又有多少是其他组件（自我修改、技能目录等）的作用？

**Peter Yang**：我觉得 **70-80% 是“个性化”部分**，因为我大部分时间只是和它语音聊天。但另一方面，它确实很粗糙，经常忘记事情，我需要不断提醒它。不过，任何疯狂的想法，我只需要和它说，它基本上就能帮我实现。比如有一天我用语音回复时问：“嘿，我们能直接打个实时电话吗？”它说：“你需要连接 Twilio，做这些那些。”然后我就去做了，最后它真的打了我的电话。

**主持人**：哦真的吗？你已经设置好了？我一直很想设置这个。

**Peter Yang**：效果不太好，延迟很严重。但关键是，我竟然能把它跑起来，这本身就挺让人印象深刻的。基本上，我任何疯狂的想法，它都能勉强实现。

## 代码生成与记忆系统：如何实际操作？

**主持人**：在实践中，你是怎么做的？你是让它现场写一个技能（skill），还是去发现一个技能？你实际用了多少代码生成功能？

**Peter Yang**：我用非常随意的方式和它说话，就像跟朋友聊天一样。比如：“嘿，Zoe，我能打个电话吗？”它说：“好的，你得做这些。”我说：“好吧，我打开电脑，弄一下。”然后它说：“给我打个电话。”我们边调试边弄，最后就成功了。用 Claude 时，我会写很长很精细的提示词；但用 OpenClaw，我只是给它发短信。

**主持人**：很有意思。我们刚才谈到了手机消息、记忆系统、代码生成组件。你觉得**记忆系统**做得怎么样？它基于文件存储，有创新性吗？你说它会忘记东西，但大语言模型也会。你认为这个记忆系统是设计得好，还是拖了后腿？

**Peter Yang**：我认为默认的记忆系统其实**不太好**。据我理解，它只是一个 `memory.md` 文本文件，每天更新，而且经常忘记事情。所以我实际上安装了一个**三层记忆系统**——说实话我自己也不太懂——它包含了类似 Toby 的 QMD 搜索工具，还安装了一个 2GB 的东西，然后就好了一点。但我还是得把提示写进 `agents.md` 里，告诉它：“在回答我的任何问题之前，先遍历所有记忆，检查所有内容。”而且它经常忘记自己能做什么。比如我问“你能更新我的 Google 文档吗？”，它说“哦我做不到”。不，你可以的，它就在你的文件里。你得提醒它。

## 争议观点：应用将死，Claw 无处不在

**主持人**：我们来谈谈一些争议。你曾说过“**应用将死，Claw 会无处不在**”。请展开讲讲这个观点。

**Peter Yang**：首先，我在推特上发各种随机的、没怎么深思熟虑的东西。但自从我在 OpenClaw 上设置了 Mercury、MCP 等等这些应用后，我确实**不再怎么打开那些应用了**。我认为最先消亡（或使用量减少）的是那些你打开它只是为了**完成某个任务**的应用。那些用来娱乐的应用可能存活得更久一点。但对于完成任务的应用，给我的智能体发个短信让它去做，要容易得多。就像你有一个超级好用的行政助理。

**主持人**：那么你发现自己的智能手机使用时间（除了 OpenClaw 之外）减少了多少？

**Peter Yang**：没有减少，因为我是个 Twitter 成瘾者，我还是用手机太多。但在使用那些应用方面，确实减少了。

**主持人**：是的，因为你不会让 Zoe“帮我看 X 上有什么有趣的内容”，对吧？

**Peter Yang**：它确实会给我发送**早间简报**，告诉我排名前两的推文和趋势。但我还是会打开 X 自己看。

---

## 应用的情感价值 vs 智能体的单一界面

**主持人**：我一直有个理论：人们打开手机应用是因为他们想**感受某种情绪**。比如 WhatsApp 是为了感受连接，Slack 是为了感受高效，TikTok 是为了娱乐。那么如果只有一个智能体，你如何处理上下文切换？比如什么时候调情，什么时候干活？应用给了你很好的意图划分。

**Peter Yang**：你说得对，Zoe 做不到这一点。但我在 Telegram 上给 Zoe 设置了**多个频道**：一个用于随机语音回复，另一个用于一起做项目，还有一个公开频道用于演示（避免泄露隐私信息）。

**主持人**：这是通过子智能体实现的吗？

**Peter Yang**：不，只是我在网上找到的一个粗糙设置。你可以设置多个 Telegram 频道，但我不确定它是否能在不同频道之间跨上下文记住东西。但至少你可以有独立的对话。

**主持人**：你对你的智能体有多**透明**？它能看到你的个人邮件吗？

**Peter Yang**：我超级透明。我买了一台 Mac Mini，给它设置了独立的邮箱。我给了它对我邮件和日历的**只读权限**，以及对一些文档的**写入权限**。但它不能滚动浏览我整个 Google Drive。

## OpenClaw 如何产品化？

**主持人**：OpenClaw 作为一种架构和原语，你认为它会如何被产品化、打包给全世界？

**Peter Yang**：我认为这正是 Peter Steinberger 在 OpenAI 正在做的事情。他们可能会在 **ChatGPT** 里构建类似功能——让 ChatGPT 真正帮你完成任务，而且感觉更人性化。

**主持人**：老兄，让我吐槽一下 ChatGPT。他们训练模型的方式是，在每次对话结尾都说“如果你需要，我还可以做 X 和 Y”。我对此非常恼火，以至于我**从 ChatGPT 转向了 Claude**。这虽然可能提高了他们的指标，但真的很烦人。为什么不一开始就做了呢？

**Peter Yang**：所以你现在是 Claude 用户了？我也用 **Codex** 来编程。

**主持人**：你更喜欢 Codex？你更喜欢它还是 Claude Code？或者两个都用？

**Peter Yang**：当我想构建真正的东西时用 Codex；Claude Code 则是在我随便玩玩（vibing）的时候用。我觉得它们处于不同的位置。

- **Claude Code 和 Opus 4.6** 更话痨，做更多假设，但在异步体验中更愉快。
- **Codex** 思考更深入，通常更准确，但有时就像和一个每次停顿三分钟才说话的人对话，很难进入心流状态。
**主持人**：我发推说过，**Claude Code 几乎像一台老虎机**——每次都有不同的东西。这让人想起社交网络时代的**变率奖励**（variable scheduled rewards）。编码智能体也有同样的特性：时间也是变化的，有时一秒就出结果，有时要五分钟。这两者加在一起，就给了它那种赌场般的感觉。

**Peter Yang**：另外，Claude Code 有很多**自定义功能**（hooks、skills）。如果你不关注 X，你根本不知道怎么定制。但一旦你定制了，你会感觉它就是你的一部分，很难再换掉。

**主持人**：我用 Claude Code 更多是因为它的**生活质量功能**。比如你可以直接截图粘贴到 Claude Code，而 Codex 需要先把图片保存成文件再粘贴。Claude Code 还加了语音（虽然还有点粗糙）。Claude Code 可以和 Chrome 交互，但 Codex 不能和 Atlas 交互。这些都是 OpenAI 会修复的。**Codex 的模型本身更好**，但那些功能今天还不存在。

## 编码智能体的未来：SaaS 的终结还是玩具？

**主持人**：谈谈编码智能体。你的总体看法是什么？它们是 SaaS 的终结，还是只是玩具？

**Peter Yang**：首先，我不是工程师，我是个新手。但我前几天和一家 **AI 原生初创公司**的人聊过，他们有一堆“氛围编码者”（vibe coders），这些人正试图**构建内部工具来取代他们付费的 SaaS**。这是一家真实存在的公司，他们想摆脱为 SaaS 付费，通过自己写内部工具来实现。

**主持人**：所以他们是应用生成（AppGen）公司，应该用 AppGen 做一切。但你的预测是，普通公司会停止使用 Slack 或 Calendly 吗？

**Peter Yang**：我觉得 **Slack 还有很大生命力**，因为 Slack 本身也可以成为你和智能体对话的地方。但对于一些更简单的应用，比如 **Calendly**，我为什么要付费呢？

**主持人**：反方观点是，它并不贵。你真的想自己维护一个 Calendly 吗？一个月 20 美元，它永远有更新，永远在线。组织里的每个人能投入的精力是有限的。

**Peter Yang**：这倒也是。除非你像那家初创公司一样雇佣专门的“氛围编码者”。但成本和直接付给 Calendly 相比……嗯。

**主持人**：最近很多人提到 **Figma**，股价下跌了，它能挺住吗？

**Peter Yang**：我觉得设计师们还在用 Figma，但作为一个设计师，你需要学习如何“氛围编码”，否则几年后你就会落伍。

**主持人**：我的观点是，我把工具分为**思维工具**和**制作工具**。IDE 曾经是制作工具——执行的场所。现在执行成本趋近于零，很多多智能体 IDE 变成了**尝试和试错**的地方，用试错来促进你的思考。Figma 实际上两者兼具：它是设计执行的地方，也是设计思维的重要场所。这是它在新时代保持高度相关性的机会。

**Peter Yang**：我完全同意。Figma 需要提升它的 AI 工具能力。看着这些智能体与你协作并做事情，真的非常有趣。

## 编码智能体中被低估和被高估的方面

**主持人**：你认为编码智能体哪些方面**被低估**，哪些被**高估**？

**Peter Yang**：也许不算被低估——软件将吞噬世界，编码将吞噬所有知识工作。我们已经在朝这个方向前进了。比如最近 **Lovable** 宣布支持一切和 Replit 类似的功能。每个人都在追赶这个方向。我不想再用 PowerPoint 了，我讨厌写 Google 文档。前几天我写博客时，不再打字，而是用 Claude Code 说：“我给你一堆反馈，你帮我写。”它完成了前 80%，最后 20% 我手动调整。这就是我现在的工作方式：**我从不从零开始，总是先让 AI 完成前 80%**。

**主持人**：有趣的是，Satya 曾说过：**Excel 是世界上最强大、最流行的编程语言**。数以亿计的人会用它，但我们不这么认为。它是一种描述和解决问题的方式。编码智能体将是它的千倍放大——即使是像写 Google 文档这样主观的事情，也可以用编码领域的方式表达出来，让使用智能体来做更令人满意、更高效。

**Peter Yang**：因为 Excel 之所以流行，是因为它**非常平易近人**。而编码智能体让代码基本消失了——你只需要和智能体说话，让它做事。

## 未来公司的形态：小团队 + 智能体

**主持人**：你认为未来的公司是什么样子？只是一群智能体加一个 CEO？CEO 会是智能体吗？人在公司里扮演什么角色？

**Peter Yang**：我有一些**大胆的观点**。随着公司变大，它往往会变成一个**更糟糕的工作场所**，因为有太多人需要协调。我记得我们之前的公司有各种 OKR 会议，我坐在房间里三个小时讨论 OKR，感觉就是在浪费生命。所以我希望**更多公司能保持小规模**。这一代的创始人意识到，他们希望公司尽可能小。与其有一个 10 人的产品团队，不如有一个 2 到 3 人的产品团队，再加上一群智能体来帮忙。因为和智能体跨职能协作比和人协作容易得多。

**主持人**：实际上，智能体因为**不带情绪**，所以更容易。想象一下，我派我的智能体，你派你的智能体去谈判，它们得出一个结论——没有情绪，非常客观。

**Peter Yang**：完全正确。

**主持人**：我们一直在讨论 AI 在工作中的**员工体验价值**——如何提高工作的 NPS（净推荐值）。从 10,000 年前“不被狮子吃掉就是美好一天”，到 100 年前“不在工厂被蒸汽压机压死”，到现在“不被卷入和另一个副总裁下属的高情绪拉锯战”——那种 50 封邮件来回的线程。也许未来的方向是，很多情绪化、主观的工作被智能体处理，我们只是引导过程，而不是置身其中。

**Peter Yang**：我过着 PM 和创作者的“双重生活”。我觉得所有 PM 本质上都想**创造产品**。这就是我们进入这一行的原因。但我大胆地说：我认识的所有 PM 都在晚上和周末试图“氛围编码”。另一个大胆的观点：如果你真的失业了，你反而有更多时间成为一个构建者和创新者，因为你可以真正玩所有这些新东西并学习。

---

## 快速与缓慢：AI 时代的工作节奏

**主持人**：你认为每个人都必须像 Gary 说的那样“不睡觉、10 倍速”吗？还是说深思熟虑也有价值？

**Peter Yang**：现在有了 AI 工具，很容易同时往 10 个不同方向跑。所以有时候你真的需要**慢下来**，想清楚你要去哪里。但我也相信，传统的**年度规划**等流程已经不再适用了。

**主持人**：我的观点是：当你**实现一个局部最大值**时，你应该极快——用智能体迅速爬到山顶。但要找到下一个山丘，你需要慢下来，甚至停下来，去接触真实的世界（touch grass）。所以是**快与慢的结合**，这可能是未来的方式。

**Peter Yang**：是的，你还需要**随机漫步**来寻找产品市场契合点，这需要时间。

---

## “商业一揽子”平台与一人公司

**主持人**：我们之前聊过一些“商业一揽子”平台。你有了解吗？

**Peter Yang**：我看过 PostPulse（？）。它确实让人一窥未来的可能性，但现在还很早期。它让我去跑 Facebook 广告——我为什么要跑 Facebook 广告？

**主持人**：我对它很兴奋，因为它为更多人（即使是**一人公司**）提供了构建公司的路径。想想看：构建一个十亿美元的公司竞争极其激烈，但一百万美元、十万美元市场规模的机会可能遍布各地。这些产品能改变一个人的生活。这虽然不是风险投资级别的公司，但没关系。我希望这个逻辑成立，因为它能让更多人参与进来。

**Peter Yang**：这就是我对我孩子的计划。我希望他们在高中就构建**自力更生的业务**，跳过整个大学和公司生活。

**主持人**：过去 10 年有一种道德恐慌——孩子们都想当 YouTuber。但你就是一个 YouTuber。实际上，孩子们想成为**企业家**或有自主权。如果他们不是程序员，唯一的线上渠道就是制作 YouTube 视频。现在，他们可以做更多了。

## 智能体优先的消费者产品：留存与品牌

**主持人**：很多人说“智能体会先与你的产品交互”。但作为消费者领域的人，你怎么看待**留存**和**品牌资产**？因为用户只是让智能体去调用 API，用户自己可能根本不回来。

**Peter Yang**：我有几个想法：

1. **商业模式简化**：消费者现在愿意为 AI 产品付费（订阅 + 令牌消耗）。有真实的推理成本，所以你从第一天起就必须收费。这反而帮助解决了很多问题。
1. **双重界面**：产品会有 API 供智能体交互，同时也有**消费型界面**（比如移动应用，有信息流，但你也可以切换到“后台”查看执行日志）。
比如我们工作过的 Credit Karma——有时你想看看你的信用分历史和信用卡 offer，有时你只想说：“嘿，帮我搞定所有事情，这周你帮我省了多少钱？”

**主持人**：整个智能体栈正在浮现：**身份、支付、营销**，甚至 CLI 与 MCP 的对比。所有这些都还很新，很多旧的经验法则都不再适用了。

**Peter Yang**：这是一个全新的世界。2025 年我还觉得智能体被高估了，但现在我认为它真的要来了。

**主持人**：我也是。这个词让人沮丧，因为它被过度滥用了。我一直想说：**模型在循环中使用工具**。这才是最好的定义。但没人喜欢听这个，“智能体”更花哨。

## 就业市场的未来：不是失业，而是转型

**Peter Yang**：很多人认为我们会失去工作。我希望所有这些技术能让**人类工作变得更有趣**。

**主持人**：我不认为我们会全部失业。我们看了大量公司，发现两个不同的类别：

- **第一类**：大幅提高个人或团队的效率（比如招聘）。能完成电话筛选、回答公司问题、甚至薪资谈判，但**不能完成 100% 的工作**（比如带候选人参观办公室、入职培训）。
- **第二类**：完成某个**完整工作职能的 100% 自动化**（比如客户支持）。但这种非常罕见。几乎所有 AI 原生产品都能提供巨大的效率提升，但**无法做到 100%**。最后 10% 仍然需要人类。
由于难以达到 100% 自动化，效率提升可能会以其他方式体现——**不是减少工作**，而是可能实现**欧洲式的四天工作周**，或者公司效率翻倍。

**Peter Yang**：但你不觉得会出现从大公司裁员向更多**小公司/个体创业者**的转型吗？

**主持人**：我认为**经济的形态会改变**——集中度会变化。但我认为工作不会减少。**人类的野心没有天花板**，人类的欲望没有天花板。随便读一本有点意思的科幻小说——这不可能是我们所有需求和欲望的顶峰。每天你读到的新东西（奢侈品、肽类等）都会让人想要更多。

**Peter Yang**：我看到一条很棒的推文：“**就业市场太差了，我现在只能去追求自己的梦想了。**”所以，也许你失去了工作，但你现在可以做自己的事情，并且有机会真正实现它。

**主持人**：是的，是的。很棒的积极收尾。谢谢你，Peter。
